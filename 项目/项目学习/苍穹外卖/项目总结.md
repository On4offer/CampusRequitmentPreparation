## 面试常见追问

1. **JWT 和 Session 的区别？**
   - JWT 无状态，前后端分离常用；Session 需要服务端存储，适合传统应用。
2. **为什么要用 nginx 反向代理？**
   - 提高访问速度（缓存）、实现负载均衡、隐藏后端服务保证安全。
3. **为什么密码要加密存储？为什么选 MD5？**
   - 防止数据库泄露时明文密码外泄；MD5 快速不可逆，但实际项目推荐加盐+更安全算法（如 bcrypt）。
4. **为什么要分多模块（common、pojo、server）？**
   - 提高代码复用性、分工清晰、便于维护和扩展。
5. **Swagger 和 YApi 区别？**
   - YApi 偏向设计阶段（团队协作、接口管理），Swagger 偏向开发阶段（生成文档、接口测试）。

------

- 软件开发流程 + 三套环境
- 项目定位（管理端/用户端功能）
- 技术选型（Spring Boot + MyBatis + Redis + JWT + Nginx）
- 模块划分（common/pojo/server）
- 登录流程（Controller → Service → Mapper → JWT）
- nginx 反向代理/负载均衡
- 密码加密（MD5）
- 接口文档（YApi vs Swagger）

## 二、公共字段自动填充

### 背景问题

新增/修改员工、分类、菜品等数据时，都需要手动赋值 `create_time、update_time、create_user、update_user` 字段，导致代码冗余。

### 解决方案

- 使用 **自定义注解 + AOP 切面 + 反射**，在 Mapper 层自动为公共字段赋值。

### 核心实现

**注解定义**

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface AutoFill {
    OperationType value(); // INSERT / UPDATE
}
```

**切面逻辑**

```java
@Before("autoFillPointCut()")
public void autoFill(JoinPoint joinPoint){
    MethodSignature signature = (MethodSignature) joinPoint.getSignature();
    OperationType operationType = signature.getMethod().getAnnotation(AutoFill.class).value();

    Object entity = joinPoint.getArgs()[0];
    LocalDateTime now = LocalDateTime.now();
    Long currentId = BaseContext.getCurrentId();

    if(operationType == OperationType.INSERT){
        // 赋值4个字段
    }else if(operationType == OperationType.UPDATE){
        // 赋值2个字段
    }
}
```

**Mapper 示例**

```java
@AutoFill(value = OperationType.INSERT)
void insert(Category category);

@AutoFill(value = OperationType.UPDATE)
void update(Category category);
```

### 面试追问

1. 为什么把公共字段赋值逻辑放到 AOP，而不是在 Service？
    👉 解耦 & 统一处理，减少重复代码。
2. 如果 `BaseContext` 没有用户信息怎么办？
    👉 需要在登录拦截器中设置当前用户 ID，否则会导致空指针。

## 五、删除菜品

### 业务规则

- 起售中的菜品不能删除
- 套餐关联的菜品不能删除
- 删除菜品时要同步删除口味数据

### 核心代码

```java
@Transactional
public void deleteBatch(List<Long> ids) {
    for(Long id : ids){
        Dish dish = dishMapper.getById(id);
        if(dish.getStatus() == StatusConstant.ENABLE){
            throw new DeletionNotAllowedException("菜品在售，不能删除");
        }
    }
    List<Long> setmealIds = setmealDishMapper.getSetmealIdsByDishIds(ids);
    if(!setmealIds.isEmpty()){
        throw new DeletionNotAllowedException("菜品被套餐关联，不能删除");
    }
    ids.forEach(id -> {
        dishMapper.deleteById(id);
        dishFlavorMapper.deleteByDishId(id);
    });
}
```

## 一、Redis 基础知识

### 1. Redis 特点

- 基于内存，读写性能高（QPS 可达 10w+）。
- 适合存储热点数据：如商品库存、验证码、会话信息。
- 常用作缓存、中间件、分布式锁。
- 数据类型丰富：string、hash、list、set、zset。

### 2. Redis 在项目中的作用

- 存储店铺营业状态（频繁读写，实时性强）。
- 如果放数据库，每次查询都要访问 MySQL，性能差。
- 使用 Redis 可实现高并发下快速读写。

👉 **面试追问**：为什么选择 Redis 存储营业状态而不是 MySQL？
 **答**：Redis 基于内存，读写速度快，适合这种实时性强、频繁查询的场景；MySQL 更新/查询 IO 成本高，不适合做频繁状态开关。

2. 创建请求对象（HttpGet/HttpPost）

## 三、微信登录功能实现

### 3.1 登录流程

1. 小程序调用 `wx.login()` 获取临时授权码 `code`。
2. 前端带 `code` 请求后端 `/user/user/login`。
3. 后端使用 **HttpClient** 请求微信接口（`jscode2session`），携带 `appid + secret + code`。

5. 后端查询用户表：
   - 新用户 → 自动注册
   - 老用户 → 返回已有信息
6. 后端生成 JWT，返回给前端。
7. 前端保存 token，后续请求携带。

```java
public User wxLogin(UserLoginDTO dto) {
    String openid = getOpenid(dto.getCode());
    if(openid == null) throw new LoginFailedException("微信登录失败");
    User user = userMapper.getByOpenid(openid);
    if(user == null){
        user = User.builder().openid(openid).createTime(LocalDateTime.now()).build();
        userMapper.insert(user);
    }
    return user;
}

private String getOpenid(String code){
    Map<String,String> map = new HashMap<>();
    map.put("appid", weChatProperties.getAppid());
    map.put("secret", weChatProperties.getSecret());
    map.put("js_code", code);
    map.put("grant_type", "authorization_code");
    String json = HttpClientUtil.doGet(WX_LOGIN, map);
    return JSON.parseObject(json).getString("openid");
}
```

# 苍穹外卖项目 Day07 面试准备文档（缓存与购物车）

## 一、缓存优化

### 1.1 为什么要用缓存？

- 用户端菜品/套餐展示频繁访问数据库，访问量大 → MySQL 压力大 → 响应慢、体验差。
- 使用 **Redis 缓存热点数据**，降低数据库压力，提高查询性能。

------

### 1.2 缓存菜品实现

#### 核心思路

- 每个分类下的菜品对应一个缓存 key：`dish_{categoryId}`。
- 查询时：先查 Redis，有则直接返回；否则查数据库，并写入 Redis。
- 数据更新时：清理对应缓存，保证一致性。

#### 用户端 Controller

```java
@GetMapping("/list")
@ApiOperation("根据分类id查询菜品")
public Result<List<DishVO>> list(Long categoryId) {
    String key = "dish_" + categoryId;
    List<DishVO> list = (List<DishVO>) redisTemplate.opsForValue().get(key);
    if(list != null && !list.isEmpty()){
        return Result.success(list);
    }
    Dish dish = new Dish();
    dish.setCategoryId(categoryId);
    dish.setStatus(StatusConstant.ENABLE);
    list = dishService.listWithFlavor(dish);
    redisTemplate.opsForValue().set(key, list);
    return Result.success(list);
}
```

#### 管理端清理缓存

```java
private void cleanCache(String pattern){
    Set keys = redisTemplate.keys(pattern);
    redisTemplate.delete(keys);
}
```

调用场景：

- 新增菜品 → 清理该分类缓存
- 修改/删除/起售停售菜品 → 清理所有 `dish_*` 缓存

------

### 1.3 缓存套餐实现（Spring Cache）

```java
    Map map = new HashMap();
    map.put("address", shopAddress);

                              .getJSONObject("result")
                              .getJSONObject("location");

    map.put("address", address);

    String userLngLat = location.getString("lat") + "," + location.getString("lng");

    map.put("origin", shopLngLat);
    map.put("destination", userLngLat);
    String json = HttpClientUtil.doGet("https://api.map.baidu.com/directionlite/v1/driving", map);

    JSONArray routes = JSON.parseObject(json).getJSONObject("result").getJSONArray("routes");
    Integer distance = (Integer) ((JSONObject) routes.get(0)).get("distance");
```

## 一、Spring Task 定时任务

### 1.1 概念

- Spring Task 是 Spring 自带的定时任务框架。
- 本质：基于注解 `@Scheduled` 执行定时逻辑。
- 适合周期性任务（如定时清理、定时通知）。

### 1.2 cron 表达式

- 用于定义任务执行时间。
- 格式：`秒 分 时 日 月 周 [年]`
- 常见例子：
  - `*/5 * * * * ?` → 每 5 秒执行一次
  - `0 0/1 * * * ?` → 每分钟执行一次
  - `0 0 1 * * ?` → 每天凌晨 1 点

### 1.3 核心代码

```java
@Component
@Slf4j
public class MyTask {
    @Scheduled(cron = "0/5 * * * * ?")
    public void executeTask(){
        log.info("定时任务执行：{}", new Date());
    }
}
```

启动类需加注解：

```java
@EnableScheduling
@SpringBootApplication
public class SkyApplication { ... }
```

## 二、订单状态定时处理

### 2.1 场景

1. **待支付订单**：下单超 15 分钟未支付 → 自动取消。
2. **派送中订单**：长时间未完成（如 1 小时）→ 自动完成。

### 2.2 核心代码

```java
@Component
@Slf4j
public class OrderTask {
    @Autowired
    private OrderMapper orderMapper;

    // 每分钟执行，处理支付超时订单
    @Scheduled(cron = "0 * * * * ?")
    public void processTimeoutOrder(){
        LocalDateTime time = LocalDateTime.now().plusMinutes(-15);
        List<Orders> ordersList = orderMapper.getByStatusAndOrdertimeLT(Orders.PENDING_PAYMENT, time);
        ordersList.forEach(order -> {
            order.setStatus(Orders.CANCELLED);
            order.setCancelReason("支付超时，自动取消");
            order.setCancelTime(LocalDateTime.now());
            orderMapper.update(order);
        });
    }

    // 每天凌晨 1 点执行，处理派送中订单
    @Scheduled(cron = "0 0 1 * * ?")
    public void processDeliveryOrder(){
        LocalDateTime time = LocalDateTime.now().plusMinutes(-60);
        List<Orders> ordersList = orderMapper.getByStatusAndOrdertimeLT(Orders.DELIVERY_IN_PROGRESS, time);
        ordersList.forEach(order -> {
            order.setStatus(Orders.COMPLETED);
            orderMapper.update(order);
        });
    }
}
```

## 三、WebSocket

### 3.1 概念

- **WebSocket**：基于 TCP 的全双工通信协议。
- **HTTP vs WebSocket**：
  - HTTP：短连接、单向请求响应。
  - WebSocket：长连接、支持服务端主动推送。

### 3.2 应用场景

### 3.3 核心代码

**WebSocketServer**

```java
@Component
@ServerEndpoint("/ws/{sid}")
public class WebSocketServer {
    private static Map<String, Session> sessionMap = new HashMap<>();

    @OnOpen
    public void onOpen(Session session, @PathParam("sid") String sid) {
        sessionMap.put(sid, session);
    }

    @OnMessage
    public void onMessage(String message, @PathParam("sid") String sid) {
        System.out.println("收到消息:" + message);
    }

    @OnClose
    public void onClose(@PathParam("sid") String sid) {
        sessionMap.remove(sid);
    }

    public void sendToAllClient(String message) {
        for (Session session : sessionMap.values()) {
            session.getBasicRemote().sendText(message);
        }
    }
}
```

   - 确保表格格式统一，填充数据即可，避免动态生成复杂表格结构。
4. **如何保证报表数据的正确性？**
   - 数据来源与统计逻辑要与工作台保持一致。
   - 采用事务保证统计查询一致性。
