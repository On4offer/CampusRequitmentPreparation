# PyCharm 工作目录选错导致一键运行“旧版本”代码

日期：2026-03-19  
项目：个性化情感陪伴智能体（FastAPI/uvicorn）

---

## 1. 现象（Symptoms）

- **一键运行（PyCharm Run Configuration）**：Swagger 里只有 `GET /health`、`POST /chat` 等，**没有** `GET /trace/{trace_id}`、`GET /traces`；访问 `GET /trace/{trace_id}` 返回 `{"detail":"Not Found"}`。
- **命令行启动**（在正确项目目录下执行 uvicorn）：Swagger 里有上述 trace 接口，且能正常查 trace。

即：同一份代码在 IDE 一键运行和命令行下表现不一致，像是“运行了旧版本”。

---

## 2. 影响（Impact）

- 误以为部署/代码未生效，或需要“重启才能加载新路由”。
- 在 PyCharm 里调试、演示时看到的是旧接口集合，排查容易走偏。

---

## 3. 根因（Root Cause）

PyCharm 的 Run Configuration 中 **Working directory（工作目录）** 与 **实际项目路径** 不一致：

- **工作目录** 被设为：`D:\Users\hzr08\Desktop\研二下\个性化情感陪伴智能体`（缺少 `校招学习\项目`）
- **实际项目**（当前编辑、命令行运行的目录）：`D:\Users\hzr08\Desktop\研二下\校招学习\项目\个性化情感陪伴智能体`

以“模块”方式运行 `uvicorn` 时，Python 会把 **工作目录** 加入模块搜索路径。因此：

- 一键运行：从 `研二下\个性化情感陪伴智能体` 下加载 `app` 包 → 若该目录是旧拷贝或未同步，则加载的是**旧代码**（无 trace 路由）。
- 命令行：在 `研二下\校招学习\项目\个性化情感陪伴智能体` 下执行 → 加载的是**当前项目**的 `app`，包含新路由。

解释器和 `.env` 路径可能已指向正确项目，但**进程的当前工作目录**决定“从哪份目录加载 `app`”，因此出现“一键是旧版、命令行是新版”的现象。

---

## 4. 复现步骤（Repro）

1. 在 PyCharm 中新建或编辑 Run Configuration（如 fast2），执行模式为 **Module**，模块名 `uvicorn`，参数 `app.main:app --reload --host 127.0.0.1 --port 8076`。
2. 将 **Working directory** 设为与当前项目**不同**的目录（例如少一层 `校招学习\项目` 的路径）。
3. 保存并一键运行，打开 `http://127.0.0.1:8076/docs`。
4. 观察：Swagger 中缺少新增的 `GET /trace/{trace_id}`、`GET /traces`；用浏览器或工具请求 `GET /trace/<trace_id>` 得到 `{"detail":"Not Found"}`。

---

## 5. 解决方案（Solution）

在 PyCharm 中修正 **Working directory**，使其与**实际项目根目录**一致：

1. **Run** → **Edit Configurations**，选中该配置（如 fast2）。
2. 找到 **Working directory（工作目录）**。
3. 改为项目根目录（与解释器、`.env` 所在路径一致），例如：  
   `D:\Users\hzr08\Desktop\研二下\校招学习\项目\个性化情感陪伴智能体`
4. 确定保存，重新一键运行。

之后 Swagger 会显示全部路由，`GET /trace/{trace_id}` 等接口行为与命令行启动一致。

---

## 6. 预防 / 建议

- 新建 Run Configuration 时，**Working directory** 优先选择“项目根目录”（与打开的项目一致），不要选到同名但路径不同的其它文件夹。
- 若存在多个同名项目（如 `研二下\个性化情感陪伴智能体` 与 `研二下\校招学习\项目\个性化情感陪伴智能体`），在配置中明确核对**解释器、.env、工作目录**三者是否都指向**同一**项目路径。
- 出现“命令行正常、IDE 异常”时，可优先检查 Run Configuration 的 **Working directory** 与当前编辑的项目是否一致。
