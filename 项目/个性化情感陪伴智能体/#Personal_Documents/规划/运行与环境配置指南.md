# 运行与环境配置指南（第一次运行：Cursor/VSCode & PyCharm）

本文档面向第一次跑通本项目的同学，目标是做到：
- **第一次就能跑起来**
- 后续做到 **点击启动 / 点击重启**（不用每次敲命令）

> 本项目后端：FastAPI  
> 启动方式：`uvicorn app.main:app --reload --host 127.0.0.1 --port 8075`

---

## 0. 目录与前置条件（重要）

### 0.1 正确的项目根目录
项目根目录应包含（至少）：
- `app/`
- `requirements.txt`
- `.env`（含密钥）

### 0.2 为什么推荐 `.venv`
Windows 上经常同时装了 Conda / 系统 Python，容易互相污染。推荐每个项目一个 `.venv`，保证依赖可控、可复现。

---

## 1. 第一次运行：通用步骤（所有 IDE 都要做一次）

### 1.1 创建与激活虚拟环境（PowerShell）
在项目根目录打开终端，执行：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

看到提示符前面出现 `(.venv)` 表示激活成功。

### 1.2 配置 `.env`（必须）
打开项目根目录 `.env`，至少填：

```env
LLM_API_KEY=你的真实key
```

如果你用的是 OpenAI 兼容但非 OpenAI 的供应商（DeepSeek/通义/百炼等），通常还需要：
- `LLM_BASE_URL`：改成供应商提供的 OpenAI 兼容地址（例如 `https://api.deepseek.com`）
- `LLM_MODEL`：改成供应商的模型名

> 注意：**修改 `.env` 后需要重启服务一次**（环境变量在进程启动时读取）。

### 1.3 V1 可选配置（按需添加）
从 V1 起，以下项可在 `.env` 中按需配置（不配则用代码默认，不影响基础对话）：

| 变量 | 含义 | 默认 |
|------|------|------|
| `POLICY_CONFIG_PATH` | 共情策略 JSON 路径（如 `config/policy_config.json`），空则用内置 | 空 |
| `EMOTION_LOG_ENABLED` | 是否每轮把情绪写入 JSONL（为情绪曲线/回访预留） | false |
| `EMOTION_LOG_PATH` | 情绪落盘文件路径 | data/emotion_log.jsonl |
| `SAFETY_FORCE_LISTEN` | 高风险是否强制 mode=倾听 | true |
| `WATCH_TIER_FORCE_LISTEN` | 关注档是否优先 mode=倾听 | true |

详见项目根目录 `.env.example` 中的注释；修改后需**重启后端**生效。

## 2. Cursor / VSCode：第一次运行（推荐 F5 一键启动）

### 2.1 打开项目（必须用“文件夹”方式）
1. `File → Open Folder…`
2. 选择项目根目录（能看到 `app/` 的那一层）

### 2.2 选择 Python 解释器（必须做一次）
1. `Ctrl + Shift + P`
2. 搜索：`Python: Select Interpreter`
3. 选择：项目的 `.venv`（路径里包含 `.venv\\Scripts\\python.exe`）

### 2.3 一键启动（F5）
项目已提供：`.vscode/launch.json`（已配置端口 8075）。

启动：
- 直接按 **F5**  
或
- 左侧 `Run and Debug（运行和调试）` → 选择 **FastAPI (uvicorn)** → 点绿色三角

重启：
- 运行后顶部会出现调试栏，点 **Restart（圆形箭头）**

### 2.4 验证
打开浏览器：
- `http://127.0.0.1:8075/docs`（推荐）
- `http://127.0.0.1:8075/health`

在 `/docs` 里测试 `POST /chat`：

```json
{
  "message": "你好"
}
```

停止：
- 调试栏点 Stop  
或终端 `Ctrl + C`

## 3. PyCharm：第一次运行（绿色三角启动 / 重启按钮重启）

### 3.1 打开项目
`File → Open`，选择项目根目录（含 `app/` 的那一层）。

### 3.2 配置解释器为项目 `.venv`（必须做一次）
1. `File → Settings`
2. `Project: <项目名> → Python Interpreter`
3. `Add Interpreter → Existing`
4. 选择：`<项目路径>\\.venv\\Scripts\\python.exe`
5. Apply / OK

### 3.3 安装依赖（第一次必须）
在 PyCharm 底部 `Terminal`：

```powershell
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 3.4 启动（运行配置）
项目已提供运行配置：`.idea/runConfigurations/FastAPI__uvicorn_.xml`。

正常情况：
1. 右上角运行配置下拉 → 选 **FastAPI (uvicorn)**
2. 点绿色三角 Run（或 `Shift + F10`）

重启：
- 运行后点工具栏的 **Rerun/Restart（圆形箭头）**

### 3.5 若看不到 “FastAPI (uvicorn)”（少数情况）
手动新建（最稳）：
1. `Run → Edit Configurations…`
2. `+ → Python`
3. 填：
   - **Name**：`FastAPI (uvicorn)`
   - **Module name**：`uvicorn`
   - **Parameters**：`app.main:app --reload --host 127.0.0.1 --port 8075`
   - **Working directory**：项目根目录（含 `app/`）
   - **Python interpreter**：项目 `.venv`
4. Apply / OK → 再点绿色三角启动

## 4. 常见问题（FAQ）

### 4.1 `/chat` 返回 “Missing LLM_API_KEY”
原因：`.env` 没填 key 或服务没重启。  
解决：填 `LLM_API_KEY` 后，**重启 uvicorn**。

### 4.2 WinError 10013（端口权限/占用）
表现：启动监听端口时报错。  
解决：
- 换端口（推荐）：例如 `--port 8075`（本项目已默认用 8075）
- 或排查 8000 端口占用/防火墙拦截

### 4.3 `ModuleNotFoundError: app`
原因：工作目录不在项目根目录。  
解决：确保运行时的 Working Directory 是项目根（含 `app/` 的那一层）。

