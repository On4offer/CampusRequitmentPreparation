# V3 开发规划：工具调用闭环（Function Calling + 护栏 + 可观测）

> 当前基线：V2 已具备 LTM 检索、/chat 证据注入、citations、trace memory_hits、最小 UI 展示。  
> V3 目标：把“工具占位”升级为“可执行工具链路”，做到 **可控调用、失败可降级、链路可解释**。  
> 本文包含明确时间排期（按周），可直接作为执行与验收文档。

---

## 0. V3 目标与总验收（DoD）

- **目标**
  - 在 `/chat` 中接入工具路由与执行（先做 `time` + `weather` 两个工具）
  - 完成参数校验、超时、重试、最大步数、失败降级
  - trace 可回放工具决策与执行过程
  - UI 能展示本轮工具调用摘要（工具名/状态/耗时）

- **总验收（DoD）**
  1. 用户问“现在几点/今天西安天气”时可稳定触发工具并返回结果；
  2. 工具参数无效时不会执行，能回退为澄清或普通回答；
  3. 工具超时/失败时主链路不中断（`/chat` 仍 200），trace 标记 `degraded=true` 或 step.error；
  4. trace 中有 `tool_route`、`tool_execute`、`tool_summarize`（至少前两步）；
  5. 新增 V3 回归用例，`pytest` 全过，且不破坏 V1/V2 关键用例。

---

## 1. 范围与边界

### 1.1 V3 要做

- 工具注册中心（tool registry）
- 工具路由器（何时调用、调用哪个工具）
- 工具执行器（参数校验、超时、重试）
- `/chat` 工具分支接入（先单工具调用，不做多跳 agent loop）
- trace 与 UI 的工具可观测展示
- 回归测试与演示脚本

### 1.2 V3 不做

- 多工具连续规划/反思循环（V4+）
- 外部 MCP 大规模工具生态接入（V4+）
- 复杂权限系统与租户管理（V5）
- 精细成本治理看板（V5）

## 11. 风险与回滚策略

- **风险 1：工具延迟导致整体慢**
  - 策略：`tool_timeout_s` 默认 5s；超时直接降级

- **风险 2：误触发工具**
  - 策略：先关键词高置信命中 + mode 限制；提供 `tool_enabled` 总开关

- **风险 3：外部天气 API 不稳定**
  - 策略：V3 默认 mock，可切真实 API；真实 API 出错走降级

- **风险 4：改动 /chat 导致回归失败**
  - 策略：以 feature flag 上线（`tool_enabled=false` 默认关闭），逐步开启
