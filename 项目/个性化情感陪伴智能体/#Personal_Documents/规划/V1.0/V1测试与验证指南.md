# V1 测试与验证指南

> 说明 V1 六个模块如何测试、如何验证、界面如何操作，以及**何时需要重启前后端**。

---

## 0. 重启与前置说明

### 0.1 何时需要重启

| 操作 | 后端（uvicorn） | Gradio UI |
|------|-----------------|-----------|
| 修改 `.env`（如 `POLICY_CONFIG_PATH`、`EMOTION_LOG_ENABLED`、`SAFETY_FORCE_LISTEN`） | **需要重启**（进程启动时读 env） | 不需要 |
| 修改 `config/policy_config.json` | **需要重启**（在 `create_app()` 时加载） | 不需要 |
| 修改 Python 代码且使用 `--reload` 启动 | 自动热重载，一般不用手动重启 | 若改的是 `ui/gradio_app.py` 需重启 UI |
| 仅做界面操作、调 API、跑 pytest | 不需要 | 不需要 |

### 0.2 前置准备

- **后端**：在项目根目录执行  
  `python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8076`  
  Swagger：http://127.0.0.1:8076/docs  
- **Gradio UI**（需要界面验证时）：另开终端执行  
  `python -m ui.gradio_app`  
  浏览器打开：http://127.0.0.1:7860  
- **.env**：已配置 `LLM_API_KEY` 等，否则 `/chat` 会 500。

## 1. 模块一：共情策略可配置化

### 1.1 要验证什么

- 不配置时行为与 V0 一致（使用代码内默认）。
- 配置 `POLICY_CONFIG_PATH` 并修改 JSON 后，**重启后端**，共情话术与 mode 规则按新配置生效。

### 1.2 是否需要重启

- **改 `.env` 增加或修改 `POLICY_CONFIG_PATH`**：需**重启后端**。
- **改 `config/policy_config.json` 内容**：需**重启后端**。
- Gradio 不需重启。

### 1.3 怎么测（不启用配置文件）

1. 确保 `.env` 里**没有** `POLICY_CONFIG_PATH`，或值为空。
2. 重启后端（若刚改过 env）。
3. **界面**：打开 Gradio，发「随便聊聊」→ 诊断里 mode 应为 **闲聊**，原因含「用户显式闲聊」等。
4. **自动化**：  
   `python -m pytest tests/test_day5_policy.py tests/test_day7_regression.py -q`  
   应全部通过。

### 1.4 怎么测（启用配置文件并改配置）

1. 在 `.env` 中设置：  
   `POLICY_CONFIG_PATH=config/policy_config.json`
2. **重启后端**。
3. 打开 `config/policy_config.json`，把「闲聊」的 `mode_prompts` 文案改一句（如加「测试」二字），保存。
4. **再次重启后端**。
5. **界面**：Gradio 发「随便聊聊」→ 右侧诊断里本轮的 system 行为应体现新文案（若 UI 不展示 raw prompt，可看 GET /trace 里后续逻辑是否仍按“闲聊”走）。
6. **接口**：Swagger 里 POST /chat，body 如 `{"message":"随便聊聊","user_id":"u1","session_id":"s1"}`，再 GET /trace/{trace_id}，看 `decision.mode` 为「闲聊」、`decision.mode_reason` 与配置中一致。

### 1.5 验收标准

- 不设 `POLICY_CONFIG_PATH` 时，pytest 上述用例全过。
- 设 `POLICY_CONFIG_PATH=config/policy_config.json` 并改 JSON 后重启，共情话术/规则生效（界面或 trace 可验证）。

## 2. 模块二：状态机「工具」分支与占位

### 2.1 要验证什么

- 工具向输入（如「帮我查天气」「记一下明天开会」「现在几点了」）进入 mode=**工具**，trace 里出现 `intended_tool`（weather/memo/time）。

### 2.2 是否需要重启

- 无需为“工具分支”专门重启；改代码且用 `--reload` 时会自动重载。

### 2.3 界面操作（Gradio）

1. 打开 Gradio（http://127.0.0.1:7860）。
2. 输入「**帮我查天气**」→ 发送。
3. 看右侧诊断：**模式** 应为 **工具**；下方 **Trace JSON** 里 `decision.mode` 为「工具」、`decision.intended_tool` 为 **weather**。
4. 再试「**记一下明天开会**」→ `intended_tool` 应为 **memo**；「**现在几点了**」→ **time**。

### 2.4 接口验证（Swagger）

1. POST /chat，body：`{"message":"帮我查天气","user_id":"u1","session_id":"s1"}`。
2. 记下返回的 `trace_id`。
3. GET /trace/{trace_id}，检查 `decision.mode === "工具"`、`decision.intended_tool === "weather"`。

### 2.5 自动化

```powershell
python -m pytest tests/test_day5_policy.py tests/test_day7_regression.py -q
```

- 回归里已含「帮我查天气」「记一下明天开会」「现在几点了」等用例，断言 mode 与 intended_tool。

### 2.6 验收标准

- 上述 3 类工具向输入在界面与 trace 中均为 mode=工具且 intended_tool 正确；pytest 回归全过。

## 3. 模块三：情绪结果持久化占位

### 3.1 要验证什么

- 开启情绪落盘后，每轮 /chat 会在指定文件追加一条 JSONL（含 user_id、session_id、timestamp_ms、label、intensity、risk_tier、trace_id）。

### 3.2 是否需要重启

- **首次开启或修改落盘配置**：需在 `.env` 中设 `EMOTION_LOG_ENABLED=true`、`EMOTION_LOG_PATH=data/emotion_log.jsonl`（或其它路径），然后**重启后端**。
- Gradio 不需重启。

### 3.3 怎么测

1. 在 `.env` 中设置：  
   `EMOTION_LOG_ENABLED=true`  
   `EMOTION_LOG_PATH=data/emotion_log.jsonl`
2. **重启后端**。
3. **界面**：Gradio 连续发 2～3 条消息（如「今天有点累」「没事」）。
4. **验证**：打开项目目录下 `data/emotion_log.jsonl`，应看到 2～3 行 JSON，每行包含 `user_id`、`session_id`、`timestamp_ms`、`label`、`intensity`、`risk_tier`、`trace_id`，与 Trace 里当轮的 emotion 一致。

### 3.4 自动化

```powershell
python -m pytest tests/test_v1_emotion_log.py -q
```

- 用例会临时开启落盘并写临时文件，断言写入一条且字段正确。

### 3.5 验收标准

- 开启配置并重启后，对话若干轮，`data/emotion_log.jsonl` 中条数与轮数一致、字段与 trace 中 emotion 一致；`test_v1_emotion_log` 通过。

## 4. 模块四：长期记忆（LTM）数据模型与接口占位

### 4.1 要验证什么

- POST /memory/ltm 能写入一条 LTM；GET /memory/ltm 能按 user_id（及可选 type）查出，结构与文档一致。

### 4.2 是否需要重启

- 不需要；纯 API 占位，无配置依赖。

### 4.3 界面操作

- **无**。LTM 为占位 API，当前 Gradio 不提供“记忆管理”页，仅通过接口验证。

### 4.4 接口验证（Swagger）

1. **写入**：POST /memory/ltm，Body 示例：
   ```json
   {
     "user_id": "u1",
     "type": "Preference",
     "content": "不喜欢鸡汤",
     "source": "dialogue",
     "confidence": 0.9,
     "tags": ["风格"]
   }
   ```
   返回应含 `"id": "..."`。
2. **列表**：GET /memory/ltm，Query 填 `user_id=u1`、`limit=10`。返回的 `items` 中应有一条 content 为「不喜欢鸡汤」、type 为 Preference。
3. **按 type 过滤**：GET /memory/ltm?user_id=u1&type=Preference&limit=5，应只返回 type 为 Preference 的条目。

### 4.5 自动化

```powershell
python -m pytest tests/test_v1_ltm.py -q
```

### 4.6 验收标准

- 通过 Swagger 或 pytest 能写入并查出 LTM，字段与设计一致；/chat 主链路不调用 LTM（V2 再做检索与注入）。

## 5. 模块五：可观测与配置外置

### 5.1 要验证什么

- **配置外置**：通过 .env 可切换「高风险是否强制倾听」「关注档是否优先倾听」；**结构化日志**：/chat 成功时后端打一条含 trace_id、latency_ms、mode、risk_tier 的日志。

### 5.2 是否需要重启

- **改 .env 中** `SAFETY_FORCE_LISTEN`、`WATCH_TIER_FORCE_LISTEN`：需**重启后端**。
- 日志在进程内输出，无需重启 UI。

### 5.3 配置外置怎么测

1. **默认（True）**：不设或设 `SAFETY_FORCE_LISTEN=true`、`WATCH_TIER_FORCE_LISTEN=true`，重启后端。发「我不想活了」→ 诊断 mode 应为 **倾听**；发「抑郁」→ mode 应为 **倾听**。
2. **关闭高风险强制倾听**：.env 设 `SAFETY_FORCE_LISTEN=false`，**重启后端**。发「我不想活了」→ 风险仍为高风险，但 mode 可能由状态机给出（如闲聊），不再被强制为倾听。
3. **关闭关注档优先倾听**：.env 设 `WATCH_TIER_FORCE_LISTEN=false`，**重启后端**。发「抑郁」→ mode 可能变为闲聊等，不再强制倾听。

注意：若关闭上述开关，当前回归用例中“高风险/关注档期望 mode=倾听”会失败，仅用于验证“配置生效”，验证完可改回 true 再跑全量回归。

### 5.4 结构化日志怎么验证

1. 后端用 uvicorn 启动（不重定向 stdout/stderr）。
2. Gradio 或 Swagger 发一条 /chat（如「你好」）。
3. 在后端终端或日志里应看到一行类似：  
   `chat_done trace_id=... latency_ms=... mode=... risk_tier=... user_id=...`

### 5.5 验收标准

- 通过改 .env 并重启后端可切换安全/关注档策略；/chat 成功时能在控制台或日志中看到上述结构化日志。

## 6. 模块六：回归与演示更新

### 6.1 要验证什么

- 回归用例全部通过；演示脚本 2～3 分钟内能跑完并包含工具占位步骤。

### 6.2 是否需要重启

- 跑 pytest：不需要重启。
- 按演示脚本操作：需先启动后端与 Gradio（若尚未启动），无需为“演示更新”专门重启。

### 6.3 回归测试

在项目根目录：

```powershell
python -m pytest -q
```

或只跑与 V1 强相关的：

```powershell
python -m pytest tests/test_day5_policy.py tests/test_day7_regression.py tests/test_v1_emotion_log.py tests/test_v1_ltm.py -q
```

- 全部通过即回归通过。

### 6.4 演示脚本（界面操作）

按 `开发记录/demo_script.md` 顺序操作即可，概要：

1. **正常对话 + 诊断**：发「今天有点焦虑」→ 看情绪/模式/trace_id/Trace JSON。
2. **多轮**：发「随便聊聊」→ 模式为闲聊。
3. **Trace 回放**：复制 trace_id，浏览器访问 `http://127.0.0.1:8076/trace/{trace_id}`。
4. **安全模式**：发「活着好累，我不想活了」→ 风险=高风险、安全模式=true、回复克制。
5. **工具占位（V1）**：发「帮我查天气」或「记一下明天开会」→ 模式=工具、trace 含 intended_tool。
6. **切 session + 重置**：session_id 改为 s2，点重置，再问「你还记得我刚说了什么吗？」→ 应不记得。

### 6.5 验收标准

- `pytest` 全量或上述 V1 相关用例全过；按 demo_script 跑一遍约 2～3 分钟，且包含工具占位步骤。

## 7. 汇总表

| 模块 | 主要验证方式 | 是否需要重启后端 | 是否需要 Gradio |
|------|----------------|------------------|------------------|
| 1 共情可配置 | pytest + 改 config 后重启再发话 | 改 .env 或 config 后**需要** | 可选（看 mode/话术） |
| 2 工具分支 | 界面发工具向句子 + pytest | 否 | 建议（看诊断 + Trace） |
| 3 情绪落盘 | 开启配置后发话 + 看 JSONL 文件 + pytest | 开启/改配置后**需要** | 可选 |
| 4 LTM 占位 | Swagger 调 POST/GET /memory/ltm + pytest | 否 | 不需要 |
| 5 配置外置 | 改 .env 开关后重启再发话 + 看控制台日志 | 改 .env 后**需要** | 可选 |
| 6 回归与演示 | pytest 全量 + 按 demo_script 操作 | 仅启动时需有后端 | 演示时需要 Gradio |
