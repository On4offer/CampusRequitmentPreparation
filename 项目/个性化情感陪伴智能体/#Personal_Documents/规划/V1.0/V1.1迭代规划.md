# V1.1 迭代规划（一期补充：记忆来源 + 全项目待办收口）

> **定位**：在一期/V1～V5 与「显式 LTM + RAG + STM + Redis/MySQL 落库」**已跑通**的前提下，补齐 **长期记忆「从哪里来」** 的产品与工程叙事，并把**阅读代码与规划文档后梳理的待办**归入 **V1.1（本版本）**、**V1.1 可选** 或 **三期及以后**，避免 backlog 散落。  
> **用途**：排期、简历/答辩、与《三期迭代规划》《整体检查后迭代规划》对齐。

---

## 0. 当前基线：算不算「先打通」？

**算。** 可验收闭环：

| 能力 | 行为 |
|------|------|
| **STM** | 每轮对话成功后写入（Redis 或内存），`/sessions` 可列、可拉消息。 |
| **LTM（显式 + 可选隐式）** | 默认 **`POST /memory/ltm`** / Memory Studio「新建记忆」；**`LTM_EXTRACT_ENABLED=true`** 时 `/chat` 可按每 N 轮写入 **`source=dialogue_extract`**（见《常用命令》§8）。 |
| **RAG** | `RAG_ENABLED=true` 时按 query 检索**进程内**向量/BM25 索引；写入 LTM 时 `index_item`，启动时 **warm** 库中活跃条目。 |
| **引用** | 对话里「引用记忆」来自**已存在**的 LTM。 |

**对话自动沉淀 LTM** 已由 **V1.1** 以开关 + 抽取管线 + Trace/撤销/Memory Studio 筛选交付；默认关闭，见 **`LTM_EXTRACT_ENABLED`**。

---

## 1. V1.1 核心要解决什么（记忆向）

1. **体验**：用户期望「聊久了被记住」→ **隐式/半自动写入 LTM**（带门控与开关）。  
2. **可控**：类型、置信度、去重/合并、撤销，避免污染 RAG。  
3. **可观测**：Trace 中可见抽取步骤与失败原因。  
4. **文档**：与《常用命令》§8、Memory Studio 文案同步（部分已在当前代码中修正）。

---

## 2. V1.1 功能范围（按优先级）

### P0（最小可用）

| 项 | 说明 |
|----|------|
| **LTM 抽取开关** | 如 `LTM_EXTRACT_ENABLED`，默认 `false`，避免未就绪时耗 token。 |
| **抽取管线** | 独立 LLM 调用 → 结构化 JSON（type/content/confidence/tags 等）→ 校验 → `ltm_store.put` + `ltm_retriever.index_item`。 |
| **触发策略（二选一先落地）** | 每 N 轮/M 字摘要后触发；或情绪/意图门控后再触发。 |
| **失败隔离** | 抽取失败只打日志/Trace，**不**影响 `/chat` 成功返回。 |
| **单测** | mock LLM，验证开关关=无写入、开=可写入并可列表查询。 |

### P1（产品化）

| 项 | 说明 |
|----|------|
| **去重/更新** | 与已有 LTM 相似时新建 vs `update_item`（先规则后模型均可）。 |
| **UI 反馈** | Toast「已记录 n 条」或「撤销本次写入」。 |
| **配额** | 抽取调用计入日配额或单独小预算。 |

### P2（与三期衔接）

| 项 | 说明 |
|----|------|
| **异步抽取** | **`LTM_EXTRACT_ASYNC=true` + `REDIS_URL`**：`companion:ltm_extract:queue` + 进程内 BRPOP worker（`main` lifespan）；完成后 **合并 Trace**；无 Redis 时 **回退同步** 并打 warning。 |
| **真实 embedding** | 可选 **`RAG_EMBEDDING_API_BASE`**（OpenAI 兼容 `POST .../embeddings`）+ `RAG_EMBEDDING_API_KEY` / `MODEL`；失败 **回退稀疏向量**（`app/rag/embedding_provider.py`）。 |

### 明确不纳入 V1.1（划入三期或长期）

- 无门控「每句都进 LTM」；跨用户写入；完整关系档案/时间线（见《整体规划》）。  
- Trace/Eval/反馈 **持久化与多实例**（见下表「三期」列）。

| 阶段 | 产出 | 验收 |
|------|------|------|
| **V1.1-M1** | 开关 + 抽取模块 + 单测 | 关=行为与现网一致；开=mock 后可写库/列表可见。 |

| **V1.1-M3** | 去重 + 轻提示（可选） | 重复偏好不无限增殖；用户可感知「已记录」。 |
