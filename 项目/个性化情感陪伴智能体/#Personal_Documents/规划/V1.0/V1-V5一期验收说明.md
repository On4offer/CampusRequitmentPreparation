# V1～V5 一期验收说明

> 对照《整体规划.md》**§10 版本路线**与《V5开发规划.md》DoD，说明当前代码里程碑是否可视为 **一期（V1～V5）完成**。  
> 更新日期：以仓库内 pytest 全绿与本文勾选为准。

---

## 1. 结论（一句话）

- **一期（V1～V5 工程里程碑）**：**已完成**——核心后端能力、可观测 trace、pytest 回归、Gradio 对话 + 运营台均已落地，满足 V5 文档中的模块一至七勾选与 DoD 主线条。
- **《整体规划》里的「最终能力 / §12 SaaS 级界面清单」**：**未 100% 对齐**，属于可单独开的 **二期（体验与演示增强）**，不影响把当前状态称为「V5 一期交付」。

---

## 2. 按版本对照《整体规划》§10

| 版本 | 规划要点 | 当前实现情况 |
|------|-----------|----------------|
| **V0** | 多轮 STM、`/chat`、trace、日志 | ✅ 已有（含 Day2/Day3 用例） |
| **V1** | 情绪识别、状态机、可配置策略、trace 可解释 | ✅ `analyze_emotion`、`decide_mode`、`policy_config_path`、trace decision |
| **V2** | LTM 模型、RAG、证据引用、记忆 CRUD API | ✅ LTM API、检索注入、`citations`；**Gradio 侧「记忆面板」**较简，以 API/Swagger 为主 |
| **V3** | 工具 + 护栏、可审计 | ✅ time/weather 等路径、超时重试、trace 步骤；**memo 等**若未接满，属扩展项 |
| **V4** | 混合检索/改写、benchmark、回归 | ✅ 混合检索、Query 改写、评测脚本与测试；**独立「评测 Lab」页**未必等同规划 §12 的完整 UI |
| **V5** | 隔离、配额、内容安全、反馈、运营配置、运营台 UI | ✅ 见《V5开发规划》各模块；配额当前以 **429** 为主（规划里「降级」为可选增强） |
