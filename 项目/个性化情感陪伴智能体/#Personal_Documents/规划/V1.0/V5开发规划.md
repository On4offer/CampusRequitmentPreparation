# V5 开发规划：业务功能与运营闭环

> 当前基线：V4 具备混合检索、Query 改写、RAG 优化与评测能力。  
> V5 目标：**产品化增强**，做到 **用户隔离、配额限流、内容安全、反馈闭环、运营可配**，提升可上线性。  
> 本文按模块组织，可直接作为执行与验收文档。

---

## 0. V5 目标与总验收（DoD）

- **目标**
  - 不同用户记忆严格隔离，user_id 维度数据互不泄露
  - 按用户/按天 token 配额与限流，超额有明确降级行为
  - 内容安全：输入/输出敏感词与自伤风险检测，记录触发原因
  - 反馈闭环：点赞/点踩/纠错进入 feedback，可沉淀为离线评测样本
  - 运营配置台：关键参数（阈值、TopK、策略开关）可在 UI 调整并立即生效

- **总验收（DoD）**
  1. 不同 user_id 的 LTM/STM 严格隔离，无法跨用户访问；
  2. 超额时 /chat 返回明确提示或降级（如仅用 STM、降低 TopK），trace 标记 `quota_exceeded`；
  3. 高风险/敏感词触发有 trace 记录，可审计；
  4. `POST /feedback` 可写入，反馈可导出为评测样本；
  5. UI 运营台可调整至少 3 项参数并立即生效；
  6. 新增 V5 回归用例，pytest 全过，且不破坏 V1～V4 关键用例。

## 1. 范围与边界

### 1.1 V5 要做

- 用户隔离：LTM/STM/session 按 user_id 严格分桶，API 校验 user_id 归属
- 配额与限流：按 user_id 按天 token 配额、QPS 限流；超额降级
- 内容安全：输入敏感词/自伤风险检测（增强 V1 安全模式）；输出约束与记录
- 反馈 API：`POST /feedback`（trace_id、rating、correction 等）
- 运营配置：参数热更新（从配置源读取，支持 UI 修改后写回）
- UI 运营台：参数开关、配额展示、反馈列表、风险事件列表

### 1.2 V5 不做

- 真实登录/JWT/Session（仅 user_id 字符串隔离）
- Redis 等外部存储（可内存实现，后续可替换）
- 复杂权限 RBAC、租户多级管理
- 精细成本计费与账单系统

## 2. 模块一：用户隔离强化

### 2.1 内容

- 审计 LTM/STM/session/trace 存储：确保所有读写均带 user_id，无跨用户泄露
- API 层：`/memory/ltm`、`/trace` 等接口校验请求 user_id 与资源归属
- 新增 `GET /profile/{user_id}`：返回该用户画像摘要（偏好/边界/近期情绪，仅本人可查）

### 2.2 验收

- 用户 A 无法访问用户 B 的 LTM、trace；
- `/profile/{user_id}` 仅返回该 user_id 的数据，无越权。

---

## 3. 模块二：配额与限流

### 3.1 内容

- 新增 `app/quota/`：按 user_id 按天 token 配额（内存计数器，可后续切 Redis）
- 配置项：`quota_token_per_user_per_day`、`quota_qps_per_user`
- `/chat` 入口：检查配额，超额时返回 429 或降级（仅 STM、降低 TopK、关闭 RAG 改写）
- trace 记录 `quota_used`、`quota_exceeded`

### 3.2 验收

- 超额时 /chat 有明确行为（429 或降级回复）；
- trace 可看到配额使用与超额标记；
- 限流时请求被拒绝或排队，有明确响应。

---

## 4. 模块三：内容安全增强

### 4.1 内容

- 输入检测：敏感词/暴力/自伤关键词（规则 + 可选 LLM），命中时记录并进入安全模式
- 输出约束：生成后过滤敏感词，或要求 LLM 遵守安全指令（V1 已有安全模式，可增强）
- trace 记录 `safety_triggered`、`safety_reason`

### 4.2 验收

- 高风险输入触发安全模式，trace 有记录；
- 输出无敏感词泄露（可规则过滤或人工抽检）。

---

## 5. 模块四：反馈闭环

### 5.1 内容

- 新增 `POST /feedback`：`trace_id`、`user_id`、`rating`（like/dislike）、`correction`（可选纠错文本）
- 存储：`data/feedback.jsonl` 或内存结构，可导出为评测样本
- 可选：点踩/纠错自动加入 `data/feedback_for_eval.jsonl`，供离线评测使用

### 5.2 验收

- 反馈可写入并持久化；
- 可导出为 JSONL 供评测脚本使用；
- trace_id 关联，可回溯到原对话。

---

## 6. 模块五：运营配置台

### 6.1 内容

- 配置热更新：`app/config/hot_config.py`，从文件/内存读取，支持 API 修改后写回
- 可调参数（至少 3 项）：`rag_top_k`、`rag_rewrite_enabled`、`tool_enabled`、`quota_token_per_user_per_day` 等
- 新增 `GET /admin/config`、`PATCH /admin/config`（轻量鉴权：固定 token 或本地 only）
- UI 运营台：Gradio 新增 tab，展示参数表单、配额、反馈列表、风险事件

### 6.2 验收

- 修改参数后，下一轮 /chat 立即生效；
- UI 可操作，无需改代码或重启。

---

## 7. 模块六：UI 运营台与演示

### 7.1 内容

- Gradio 新增「运营台」tab：参数开关、配额使用、反馈列表、风险事件列表
- 风险事件：从 trace 聚合「安全模式触发」记录，按时间展示

### 7.2 验收

- 2～3 分钟演示可展示：调参数→立即生效、超额降级、反馈写入、风险事件查看。

## 10. 风险与回滚策略

- **风险 1：配额实现导致性能下降**
  - 策略：内存计数器，O(1) 查询；可后续切 Redis

- **风险 2：热更新配置并发问题**
  - 策略：读写加锁；或仅支持单进程

- **风险 3：反馈/风险事件数据膨胀**
  - 策略：按日期分文件；保留最近 N 天；支持清理

- **风险 4：运营接口被滥用**
  - 策略：仅 localhost 或固定 token；生产需正式鉴权
