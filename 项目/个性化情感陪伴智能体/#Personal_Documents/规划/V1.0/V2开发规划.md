# V2 开发规划：LTM 接入 /chat + 最小 RAG（证据可引用）

> V1 已完成：共情可配置、工具分支占位、情绪落盘占位、LTM 模型与 API 占位、配置外置、回归与演示更新。  
> V2 目标是“开始像个性化”：把 LTM 真正接入对话链路，做到**跨会话命中 + 证据注入 + 可追踪**。  
> 仍按**模块与里程碑**推进，不按天/周排期。

---

## 0. V2 目标与总验收

- **目标**：在不破坏 V1 稳定性的前提下，完成最小 RAG 闭环：
  - LTM 条目可向量化索引与检索
  - /chat 按当前 query 召回相关记忆并注入 prompt
  - 回复中可输出“证据引用”（memory id / type / content 片段）
  - trace 可看到“检索了什么、命中了什么、为什么用这些证据”

- **总验收（DoD）**：
  1. 跨会话命中偏好/边界（例如“别鸡汤”“叫我小张”）；
  2. 回复中能稳定体现记忆内容，并可选返回 `citations`（至少包含 memory_id）；
  3. trace 中可看到检索步骤与命中列表；
  4. 回归集包含 V2 的“记忆命中与引用”用例，pytest 可跑通；
  5. UI 至少能展示“本轮命中记忆/引用证据”（先最小展示，不追求完整管理台）。

## 2. 模块一：LTM 写入策略升级（从“能存”到“可检索、可控”）

### 2.1 内容

- 扩展 LTM 字段（兼容旧结构）：
  - `is_active`（默认 true，供软删除/失效）
  - `updated_at`（支持后续编辑）
  - `embedding_status`（pending/ready/failed，占位）
- 新增“写入策略函数”：
  - 判断是否值得写入（稳定偏好、边界、长期事实优先）
  - 对“短期情绪表达”降置信度或不入 LTM
- API 扩展（可选）：
  - `PATCH /memory/ltm/{id}`（更新 content/tags/confidence/is_active）
  - `DELETE /memory/ltm/{id}`（软删除，`is_active=false`）

### 2.2 验收

- 新老数据兼容读取；
- 可对单条记忆更新/失效；
- 后续检索默认只检索 `is_active=true` 条目。

---

## 3. 模块二：最小向量索引与检索层（RAG Core）

### 3.1 内容

- 引入检索模块（建议新目录 `app/rag/`）：
  - `embedder.py`：统一 embedding 接口（先可本地模型，后可切 API）
  - `index.py`：最小向量索引实现（可先 in-memory）
  - `retriever.py`：按 `user_id` 检索 TopK，返回 `[(item, score)]`
- 数据流：
  1. LTM 写入后触发 embedding（同步或异步占位）；
  2. 向量写入索引（按 `user_id` 分桶）；
  3. /chat 收到 query 后检索 TopK。
- 配置项（新增到 `settings`）：
  - `rag_enabled`（默认 false，灰度开关）
  - `rag_top_k`（默认 3）
  - `rag_min_score`（默认 0.0）
  - `rag_max_chars`（控制注入上下文长度）

### 3.2 验收

- 对同一 user 的 LTM 条目可检索；
- 命中结果包含 score，且能按 TopK 和阈值过滤；
- 关闭 `rag_enabled` 时行为与 V1 一致。

---

## 4. 模块三：/chat 接入 RAG（证据注入与可解释输出）

### 4.1 内容

- 在 `/chat` 增加检索步骤（在组 prompt 前）：
  - `retrieve_ltm`（trace step）
  - 输入摘要：query、top_k、候选数
  - 输出摘要：命中 id/type/score（截断）
- prompt 注入策略（最小版）：
  - 将命中记忆压缩为 evidence blocks（限制总字符）
  - 注入到 system 或 user 前置上下文（固定模板）
- 回复引用：
  - 方案 A（推荐）：在 `ChatResponse.debug` 中加 `citations`（id/type/score）
  - 方案 B：扩展 `ChatResponse` 顶层 `citations` 字段（更清晰）
- Trace 扩展（建议）：
  - 在 `decision` 增加 `memory_hits`（[{id,type,score}]）
  - 或仅放在某个 `TraceStep.output_summary`（最小侵入）

### 4.2 验收

- 当 LTM 有匹配项时，回复显式体现“记得用户偏好/边界”；
- 返回体或 debug 可看到 citations；
- trace 中可回放检索链路和命中结果。

---

## 5. 模块四：记忆检索质量与降级策略

### 5.1 内容

- 最小质量保护：
  - score 太低则不注入，避免“错召回”
  - 命中过多时按 score 截断 + 字符预算截断
- 降级策略：
  - 检索失败/embedding 异常：记录 trace 错误并退化到 V1（仅 STM + 情绪 + 状态机）
  - 不中断主链路，保证可用性
- 冲突处理（最小）：
  - 同类冲突记忆按 `updated_at` 或 `confidence` 优先

### 5.2 验收

- 检索异常不影响 /chat 可用；
- 降级时 trace 有明确标识（如 `degraded=true` 或 step.error）。

## 8. 建议实施顺序（可调）

| 顺序 | 模块 | 说明 |
|------|------|------|
| 1 | 模块二：最小向量检索层 | 先打通“可召回”能力 |
