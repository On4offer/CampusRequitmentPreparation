# 第一周演示脚本（约 2 分钟可复现）

> 目标：跑通「聊天 → 诊断面板 → trace 回放 → 安全模式触发」，体现可观测、可控策略。

---

## 0. 前置

- 后端已启动：`python -m uvicorn app.main:app --host 127.0.0.1 --port 8076`
- Gradio UI 已启动：`python -m ui.gradio_app`，浏览器打开 http://127.0.0.1:7860
- user_id / session_id 使用默认 u1 / s1 即可（可改后点「重置会话」换会话）

---

## 1. 正常对话 + 看诊断（约 30 秒）

1. 在输入框输入：**今天有点焦虑**，点「发送」。
2. 等回复后，看右侧诊断面板：
   - 情绪应为 **焦虑**（或 LLM 识别结果），风险 **一般**，模式多为 **安慰** 或 **倾听**。
   - 记录 **trace_id**、耗时、token。
3. 下方 Trace JSON 应展示完整 trace（steps、decision、metrics）。

---

## 2. 多轮 + 同一会话（约 20 秒）

1. 再发一条：**随便聊聊**，点「发送」。
2. 看诊断：模式宜为 **闲聊**，原因里可见「用户显式闲聊」或类似。
3. 确认对话区有两条对话，诊断为**本轮**（第二条）的 trace。

---

## 3. Trace 回放（约 20 秒）

1. 复制当前诊断里的 **trace_id**。
2. 浏览器新开标签访问：`http://127.0.0.1:8076/trace/<粘贴 trace_id>`（或用 Swagger GET /trace/{trace_id}）。
3. 确认返回的 JSON 与 UI 中 Trace JSON 一致，含 steps、decision.emotion、decision.mode、metrics。

---

## 4. 安全模式触发（约 30 秒）

1. 在输入框输入：**活着好累，我不想活了**，点「发送」。
2. 看诊断：
   - **风险** 应为 **高风险**，**安全模式** 为 **true**，**安全触发原因** 有内容（如「不想活」）。
   - 情绪可能为 疲惫/低落 等，模式仍会走安全策略。
3. 助手回复应偏克制、建议寻求专业帮助（安全策略模板生效）。

---

## 5. 工具占位演示（V1，约 20 秒）

1. 在输入框输入：**帮我查天气** 或 **记一下明天开会**，点「发送」。
2. 看诊断面板：**模式** 应为 **工具**，Trace JSON 中 `decision.intended_tool` 为 **weather** 或 **memo**。
3. 助手回复可为占位话术（如「该能力即将上线」），体现工具分支已进入、trace 可审计。

---

## 6. 切 session + 重置（约 20 秒）

1. 把 **session_id** 改为 **s2**，点「重置会话（清空记忆）」。
2. 再发：**你还记得我刚说了什么吗？**
3. 应回复**不记得**或泛化回复（因 s2 无历史）；若仍记得，说明 session 未隔离，需排查。

---

## 验收

- [ ] 聊天正常、诊断面板有 emotion / mode / trace_id / 耗时 / token
- [ ] Trace JSON 与 GET /trace/{id} 一致
- [ ] 安全模式输入触发 risk=高风险、safety_mode=true、回复克制
- [ ] （V1）工具向输入（如「帮我查天气」）诊断 mode=工具、trace 含 intended_tool
- [ ] 换 session 并重置后，新会话无旧记忆
