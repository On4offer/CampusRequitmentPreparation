# 终极版：7 天手搓「高复杂度 AI 陪伴 Agent」学习 + 开发规划

## 定位：纯手搓、无开源依赖、面试满分、技术深度拉满

## 每天任务：可执行、不加班、学完就能讲原理

------

# 一、先定死：最终项目能力（你 7 天后要做出来）

**个性化情感陪伴智能体**

- 多轮对话 + 记忆不丢失
- 长期记忆（记住你的喜好、习惯、往事）
- 情绪识别（开心 / 难过 / 疲惫 / 生气）
- 工具调用（查时间、记备忘录、查天气）
- 双层记忆（短期 + 长期，解决上下文溢出）
- 对话状态机（闲聊 / 倾听 / 工作模式）
- FastAPI 后端接口
- Gradio 可演示界面
- 全套可面试讲解架构

------

# 二、技术栈（最终确认版，一个都不多）

```
Python 3.10+
FastAPI + Uvicorn（后端服务）
Pydantic（数据校验）
Gradio（前端界面）
大模型API（通义千问/DeepSeek/百炼）
Sentence-Transformers（向量Embedding）
FAISS（轻量向量数据库，长期记忆）
Python-Dotenv（密钥管理）
```

------

# 三、7 天完整规划（每天 2.5～3 小时，稳扎稳打）

------

## Day 1：环境搭建 + 基础对话系统（核心：跑通第一个版本）

目标：让 AI 能聊天、有角色、有记忆列表

学习内容：

- LLM API 调用

- System Prompt 设计

- 对话历史存储

  

  任务：

1. 安装所有依赖库

2. 实现基础多轮对话

3. 手搓「内存列表」，实现上下文对话

   

   产出：

   可聊天的基础 AI

   

   面试知识点：Prompt 工程、API 调用、上下文格式

------

## Day 2：手搓「短期记忆管理器」（核心面试点）

目标：解决**上下文溢出**，实现专业记忆机制

学习内容：

- 滑动窗口记忆

- 对话历史裁剪

- 内存结构设计

  

  任务：

1. 封装 Memory 类

2. 实现 add_message /get_history/clear

3. 限制 token 长度，防止溢出

   

   产出：

   不会丢上下文的专业记忆模块

   

   面试知识点：上下文管理、记忆机制、token 优化

------

## Day 3：手搓「长期记忆 RAG 引擎」（技术深度核心）

目标：永久记住用户信息，真正懂你

学习内容：

- Embedding 原理

- 向量检索

- RAG 流程

  

  任务：

1. 向量库 FAISS 安装

2. 实现记忆存储（喜好、习惯、事件）

3. 实现自动检索相关记忆

4. 与 LLM 对话融合

   

   产出：

   能记住你所有事的长期记忆系统

   

   面试知识点：RAG、向量数据库、Embedding、检索增强

------

## Day 4：手搓「工具调用系统」（Agent 灵魂）

目标：让 AI 不只会聊天，还能**做事**

学习内容：

- Function Calling 原理

- 工具注册、调度、执行

  

  任务：

1. 写 Tool 调度器类

2. 实现 3 个工具：获取时间、备忘录、天气查询

3. LLM 自动判断是否调用工具

   

   产出：

   具备工具能力的标准 Agent

   

   面试知识点：Agent 工作流、函数调用、工具设计

------

## Day 5：手搓「情绪识别 + 对话状态机」（高级亮点）

目标：让 AI 有情绪、有策略、更像人

学习内容：

- 轻量情绪分类

- 状态机设计

- 回复策略切换

  

  任务：

1. 情绪识别模块

2. 状态机：闲聊 / 倾听 / 安慰 / 工具

3. 根据情绪自动调整语气

   

   产出：

   有情感、有状态的高级 AI

   

   面试知识点：意图识别、状态机、系统设计、对话管理

------

## Day 6：FastAPI 后端封装（体现后端能力）

目标：把 AI 变成**可上线的服务**

学习内容：

- API 接口设计

- 入参出参标准化

- 服务化思想

  

  任务：

1. 搭建 FastAPI 服务

2. 封装聊天接口、记忆接口、工具接口

3. 统一异常处理

   

   产出：

   标准后端 API 服务

   

   面试知识点：RESTful API、后端开发、服务封装

------

## Day 7：Gradio 界面 + 整合调试 + 面试准备

目标：可演示、可讲稿、可面试

任务：

1. 接入 Gradio 前端

2. 全链路联调

3. 整理架构图

4. 准备面试高频问答（记忆、RAG、工具、难点）

   

   产出：

   完整可演示项目 + 面试讲解稿
