# 📘《操作系统学习与实战手册》

> 系统学习见下文各章；日常常用概念与使用场景速查见同目录《常用概念与使用场景》。

------

## 第1章：操作系统概述

### 1.1 什么是操作系统

- **操作系统**是管理硬件与软件资源的程序，为应用提供**进程管理、内存管理、文件系统、设备管理**等；屏蔽硬件细节，提供抽象与安全隔离。
- 内核态与用户态：敏感操作（如 I/O、内存映射）在内核态执行；用户程序在用户态，通过**系统调用**进入内核。

### 1.2 多道、分时与并发

- **多道**：多程序交替占用 CPU，提高利用率；**分时**：时间片轮转，多用户共享；**并发**：多个任务在时间上交替执行；**并行**：多核同时执行。

## 第2章：进程与线程

1. **进程**：资源分配的基本单位，有独立地址空间、文件描述符、PCB（进程控制块）；进程间隔离，通信需 IPC（管道、消息队列、共享内存、信号量、Socket）。
2. **线程**：CPU 调度的基本单位，同一进程内共享地址空间与资源；切换开销小，但需考虑同步（互斥、死锁）。
3. **进程与线程区别**：进程有独立空间、切换开销大；线程共享进程资源、切换快；多线程需处理竞态与同步。
4. **进程状态**：就绪、运行、阻塞（等待 I/O 等）；状态转换与调度。
5. **面试要点**：进程与线程区别、用户线程与内核线程、协程简述（用户态调度、轻量）。

------

## 第3章：进程调度

1. **调度目标**：公平性、吞吐量、响应时间、周转时间；批处理与交互式不同侧重。
2. **调度算法**：**FCFS**（先来先服务）、**SJF**（短作业优先）、**时间片轮转**（RR）、**多级反馈队列**（综合、常用）、**优先级调度**。
3. **上下文切换**：保存与恢复寄存器、栈、PCB 等；频繁切换有开销，线程比进程切换轻量。
4. **面试要点**：常见调度算法、多级反馈队列思路、什么是上下文切换。

------

## 第4章：内存管理

1. **连续分配**：固定/动态分区；产生外部碎片。
2. **分页**：逻辑地址分页、物理内存分帧，页表映射；解决外部碎片；多级页表节省页表空间；**缺页**时从外存调入（页面置换）。
3. **分段**：按逻辑段（代码、数据、栈）划分；有内部碎片；可结合分页成段页式。
4. **虚拟内存**：进程可使用比物理内存更大的地址空间；按需调入、页面置换；**页面置换算法**：FIFO、LRU、Clock、LFU。
5. **面试要点**：分页与分段区别、虚拟内存作用、LRU 与实现思路、缺页与抖动。

------

## 第5章：死锁

1. **条件**：互斥、占有且等待、不可剥夺、循环等待；四者同时满足才可能死锁。
2. **预防**：破坏其一，如一次性申请全部资源、资源有序分配（破坏循环等待）。
3. **避免**：银行家算法，分配前检测是否进入不安全状态。
4. **检测与恢复**：允许死锁发生，定期检测（资源分配图）并解除（剥夺、回滚、重启）。
5. **面试要点**：死锁四条件、如何预防与避免、银行家算法思想。

------

## 第6章：同步与互斥

1. **临界区**：访问共享资源的代码段；需**互斥**进入。
2. **同步机制**：**信号量**（P/V 操作）、**互斥量（Mutex）**、**条件变量**、**管程**；生产者-消费者、读者-写者经典模型。
3. **锁**：自旋锁（忙等）、睡眠锁（等不到则睡眠）；读写锁（多读单写）。
4. **面试要点**：信号量与互斥量区别、生产者消费者、如何保证多线程安全。

------

## 第7章：与 Java/后端的关系

1. **JVM 与进程**：一个 Java 进程对应一个 JVM；线程对应操作系统线程（HotSpot 1:1）或绿色线程。
2. **内存**：JVM 堆、栈、方法区与操作系统虚拟内存的关系；GC 与物理内存。
3. **I/O**：阻塞/非阻塞、同步/异步；NIO、多路复用（select/poll/epoll）；Netty 对 OS I/O 的封装。
4. **面试要点**：进程与 JVM、线程与 OS 线程、用户态与内核态、epoll 与 select 区别。
