# 📌 数据结构常用概念与使用场景速查

> 配合《学习笔记》使用，侧重**复杂度、线性表、树、堆、图、哈希、排序**，便于速查与校招笔试面试。

---

## 一、复杂度与稳定性（排序）


| 复杂度            | 含义   | 典型          |
| -------------- | ---- | ----------- |
| **O(1)**       | 常数   | 哈希查找、数组下标   |
| **O(log n)**   | 对数   | 二分、平衡树、堆    |
| **O(n)**       | 线性   | 遍历、计数排序     |
| **O(n log n)** | 线性对数 | 快排、归并、堆排序   |
| **O(n²)**      | 平方   | 冒泡、选择、插入    |
| **O(2^n)**     | 指数   | 暴力递归（如部分子集） |



| 稳定排序             | 不稳定排序     |
| ---------------- | --------- |
| 冒泡、插入、归并、计数、桶、基数 | 选择、快排、堆排序 |


---

## 二、数组与链表


| 操作        | 数组       | 链表         |
| --------- | -------- | ---------- |
| **随机访问**  | O(1)     | O(n)       |
| **头插/头删** | O(n)     | O(1)       |
| **尾插/尾删** | O(1)（摊还） | O(1)（有尾指针） |
| **按值查找**  | O(n)     | O(n)       |


- **Java**：ArrayList 基于数组，适合查与尾操作；LinkedList 双链表，头尾操作 O(1)，随机访问慢。

---

## 三、栈与队列


| 结构          | 规则     | 典型应用                 |
| ----------- | ------ | -------------------- |
| **栈**       | LIFO   | 括号匹配、DFS、单调栈、表达式     |
| **队列**      | FIFO   | BFS、滑动窗口、任务队列        |
| **Deque**   | 两端进出   | 单调队列、滑动窗口最值          |
| **优先队列（堆）** | 按优先级出队 | TopK、Dijkstra、合并 K 路 |


- **单调栈**：维护“下一个更大/更小”元素；**单调队列**：滑动窗口最值。

---

## 四、树


| 类型         | 特点             | 操作复杂度          | 典型                 |
| ---------- | -------------- | -------------- | ------------------ |
| **BST**    | 左小右大           | O(h)，h 为高      | 有序查找、中序即有序         |
| **AVL**    | 严格平衡，左右子树高度差≤1 | O(log n)       | 查多改少               |
| **红黑树**    | 近似平衡、旋转少       | O(log n)       | TreeMap、HashMap 桶内 |
| **B/B+ 树** | 多路、矮胖          | O(log n)，n 为键数 | 数据库索引、文件系统         |


- **遍历**：前序（根左右）、中序（左根右）、后序（左右根）、层序（BFS）。
- **红黑树**：根黑、叶（NIL）黑、红节点子必黑、任意路径黑节点数相同；插入删除通过旋转与变色保持性质。

---

## 五、堆


| 操作       | 复杂度        |
| -------- | ---------- |
| **取最值**  | O(1)       |
| **插入**   | O(log n)   |
| **删除堆顶** | O(log n)   |
| **建堆**   | O(n)（自底向上） |


- **大顶堆**：父≥子；**小顶堆**：父≤子；常用于优先队列、堆排序、**TopK**（维护 K 大小的小顶堆，堆顶即第 K 大）。

---

## 六、图


| 算法           | 条件      | 复杂度                | 用途            |
| ------------ | ------- | ------------------ | ------------- |
| **BFS**      | 无权或权为 1 | O(V+E)             | 层序、最短路径（边权 1） |
| **DFS**      | 通用      | O(V+E)             | 路径、连通分量、拓扑基础  |
| **Dijkstra** | 非负权     | O((V+E)log V)（堆优化） | 单源最短路径        |
| **Floyd**    | 任意权、多源  | O(V³)              | 任意两点最短        |
| **拓扑排序**     | DAG     | O(V+E)             | 依赖顺序、判环       |


- **存储**：邻接表省空间适合稀疏；邻接矩阵适合稠密与判边 O(1)。

---

## 七、哈希


| 冲突解决     | 说明                      |
| -------- | ----------------------- |
| **链地址法** | 同下标拉链表（或红黑树）；HashMap 实现 |
| **开放寻址** | 冲突则探测下一位置（线性/二次）；缓存友好   |
| **负载因子** | 元素数/容量；过大则扩容，一般 0.75 左右 |


- **HashMap**：键 hashCode→桶下标；冲突拉链；超过阈值扩容 2 倍；Java 8 桶内链表过长转红黑树。

---

## 八、排序速查


| 排序      | 平均         | 最坏         | 空间         | 稳定      |
| ------- | ---------- | ---------- | ---------- | ------- |
| **快排**  | O(n log n) | O(n²)      | O(log n) 栈 | 否       |
| **归并**  | O(n log n) | O(n log n) | O(n)       | 是       |
| **堆排序** | O(n log n) | O(n log n) | O(1)       | 否       |
| **插入**  | O(n²)      | O(n²)      | O(1)       | 是       |
| **计数**  | O(n+k)     | O(n+k)     | O(k)       | 是（实现得当） |


- **快排**：选基准、划分、递归；优化：随机基准、小范围用插入。**归并**：分治、合并有序序列。

---

## 九、二分查找

- **条件**：有序（或满足二段性）；**复杂度**：O(log n)。
- **模板**：左闭右闭 `while (l <= r)`、mid 防溢出 `l + (r - l) / 2`、根据比较缩区间。
- **二分答案**：答案单调时，二分答案并 check，如“最大值最小化”。

---

## 十、面试一句话总结

- **数组**：随机访问 O(1)，插入删除 O(n)；**链表**：插入删除 O(1)，查找 O(n)。
- **栈**：LIFO，DFS、单调栈；**队列**：FIFO，BFS、单调队列。
- **BST**：左小右大；**红黑树**：近似平衡，TreeMap/HashMap 桶；**堆**：最值 O(1)，插入删除 O(log n)，TopK。
- **图**：BFS 最短（权 1）、Dijkstra 非负权、拓扑排序 DAG；**哈希**：冲突链地址/开放寻址，HashMap。
- **排序**：快排/归并 O(n log n)；稳定用归并；**二分**：有序或二段性，O(log n)。

## 十二、B+树与数据库索引速查

### 1. B+树特点

| 特性 | 说明 |
|------|------|
| **多路平衡** | 每个节点存多个键，降低树高 |
| **内部节点存键** | 叶子存数据，内部节点只存键 |
| **叶子链表** | 叶子节点链表连接，支持范围查询 |
| **3层存40亿** | 16KB页，int键可存约1600个，3层1600³≈40亿 |

### 2. 聚簇 vs 非聚簇索引

| 类型 | 叶子存什么 | 特点 |
|------|-----------|------|
| **聚簇索引** | 完整行数据 | 主键索引，数据按主键顺序存 |
| **非聚簇索引** | 主键值 | 二级索引，需回表查数据 |

### 3. 面试一句话

> B+树多路降低树高减少磁盘I/O，内部节点只存键，叶子链表支持范围查询；InnoDB主键是聚簇索引，二级索引需回表。

## 十三、跳表与布隆过滤器速查

### 1. 跳表（Skip List）

| 特性 | 说明 |
|------|------|
| **结构** | 有序链表+多级索引 |
| **操作** | 查询/插入/删除 O(log n) |
| **实现** | 随机决定层数，概率1/2逐层上升 |
| **应用** | Redis ZSet |

### 2. 布隆过滤器

| 特性 | 说明 |
|------|------|
| **结构** | m位位数组 + k个哈希函数 |
| **特点** | 无假阴性，有假阳性 |
| **不能删除** | 标准版不能删（计数布隆可删） |
| **应用** | 缓存穿透防护、URL去重 |

### 3. 参数公式

- 最优 k = (m/n) × ln(2)
- m ≈ -n × ln(p) / (ln(2))²

## 十五、动态规划速查

### 1. DP三要素

1. **状态定义**：dp[i]或dp[i][j]表示什么
2. **状态转移**：dp[i]如何从之前状态推导
3. **初始化**：初始状态是什么

### 2. 经典题型

| 题型 | 状态 | 转移 |
|------|------|------|
| **爬楼梯** | dp[i] = 到i阶方法数 | dp[i] = dp[i-1] + dp[i-2] |
| **0/1背包** | dp[j] = 容量j最大价值 | 倒序：dp[j] = max(dp[j], dp[j-w]+v) |
| **股票** | dp[i][0/1] = 持有/不持有 | 状态机转移 |
| **LCS** | dp[i][j] = LCS长度 | 相等+1，否则max |

### 3. DP识别

- 最优子结构
- 重叠子问题
- 无后效性

## 十六、算法题模板速查

### 1. 滑动窗口

```java
int left = 0, right = 0;
while (right < n) {
    // 扩大窗口
    right++;
    while (需要收缩) {
        // 收缩窗口
        left++;
    }
    // 更新结果
}
```

### 2. 双指针

| 类型 | 用法 |
|------|------|
| **快慢指针** | 找中点、判环、fast走2步slow走1步 |
| **左右指针** | 有序数组，left=0, right=n-1，向中间移动 |

### 3. 前缀和

```java
// 构建
prefix[i+1] = prefix[i] + nums[i];
// 区间和
sum(i,j) = prefix[j+1] - prefix[i];
// 和为K（哈希表优化）
count += map.getOrDefault(sum - k, 0);
```
