# 📘《计算机网络学习与实战手册》

> 系统学习见下文各章；日常常用概念与使用场景速查见同目录《常用概念与使用场景》。

------

## 第1章：计算机网络概述

### 1.1 分层模型

- **OSI 七层**：物理层、数据链路层、网络层、传输层、会话层、表示层、应用层。
- **TCP/IP 四层**：网络接口层、网际层（IP）、传输层（TCP/UDP）、应用层（HTTP、DNS、FTP 等）。校招与开发常关注**传输层与应用层**。

### 1.2 各层职责简述

- **网络层**：IP 寻址与路由、分组转发；**传输层**：端到端可靠/不可靠传输（TCP/UDP）、端口区分进程；**应用层**：具体协议与业务（HTTP、DNS、SMTP 等）。

------

## 第2章：TCP 与 UDP

1. **TCP**：面向连接、可靠、有序、面向字节流；三次握手建立连接、四次挥手释放连接；流量控制（滑动窗口）、拥塞控制（慢启动、拥塞避免、快重传、快恢复）；首部 20 字节起。
2. **UDP**：无连接、不可靠、面向报文；首部 8 字节；适用实时、广播、简单查询（DNS 查询等）。
3. **三次握手**：客户端 SYN → 服务端 SYN+ACK → 客户端 ACK；目的：双方确认收发能力、协商初始序列号。
4. **四次挥手**：主动方 FIN → 被动方 ACK → 被动方 FIN → 主动方 ACK；TIME_WAIT 作用：保证对方收到 ACK、让旧报文消散。
5. **面试要点**：TCP 与 UDP 区别、为什么三次握手不能两次、为什么挥手要四次、TIME_WAIT 与 CLOSE_WAIT。

------

## 第3章：HTTP 与 HTTPS

1. **HTTP**：无状态、请求/响应模型；方法（GET、POST、PUT、DELETE 等）、状态码（2xx、3xx、4xx、5xx）、常见头（Content-Type、Cache-Control、Cookie、Connection）。
2. **HTTP/1.0 vs 1.1**：1.1 持久连接（Keep-Alive）、管道化（有限）、分块传输；仍存在队头阻塞（同一连接上请求顺序处理）。
3. **HTTP/2**：多路复用（单连接多流）、头部压缩、服务端推送；基于二进制帧。
4. **HTTPS**：HTTP over TLS/SSL；加密、完整性、身份验证；握手过程：客户端 Hello → 服务端证书与 Hello → 密钥交换 → 加密通信。
5. **面试要点**：GET 与 POST 区别、状态码含义、HTTP 与 HTTPS 区别、HTTPS 握手简述。

------

## 第4章：DNS 与 CDN

1. **DNS**：域名到 IP 的映射；分层（根、顶级域、权威域）；解析过程：本地缓存 → 本地 DNS → 递归/迭代查询；记录类型：A、AAAA、CNAME、MX、NS。
2. **CDN**：将静态资源分发到边缘节点，用户就近访问；回源、缓存策略、命中率；减轻源站压力、降低延迟。
3. **面试要点**：DNS 解析流程、DNS 劫持与污染、CDN 原理与适用场景。

------

## 第5章：网络层与路由

1. **IP**：无连接、尽力而为；IPv4 地址、子网与掩码、公有/私有地址；分片与重组、TTL。
2. **路由**：路由器根据路由表转发；路由选择算法（距离向量、链路状态）；NAT：内网地址与公网地址转换。
3. **面试要点**：IP 与 MAC 作用、ARP、子网划分、NAT 原理。

------

## 第6章：网络安全与常见攻击

1. **对称/非对称加密**：对称（同一密钥）；非对称（公钥加密私钥解密、私钥签名公钥验签）；HTTPS 中混合使用。
2. **常见攻击**：**XSS**（脚本注入）、**CSRF**（跨站请求伪造）、**DDoS**（流量攻击）、**中间人**（HTTPS 防）；防护思路：转义、Token、限流、证书校验。
3. **面试要点**：XSS 与 CSRF 区别与防护、HTTPS 如何防窃听与篡改。

------

## 第7章：与后端开发的关系

1. **Socket 编程**：TCP/UDP 套接字、bind/listen/accept/connect；NIO、Netty 对多路复用的封装。
2. **HTTP 服务**：Tomcat/Spring MVC 处理 HTTP；反向代理（Nginx）与负载均衡。
3. **面试要点**：从输入 URL 到页面展示（DNS、TCP、HTTP、渲染）、长连接与短连接。
