# 📌 计算机网络常用概念与使用场景速查

## 一、TCP vs UDP

| 对比项 | TCP | UDP |
|--------|-----|-----|
| **连接** | 面向连接 | 无连接 |
| **可靠性** | 可靠、重传、有序 | 不可靠、不保证顺序 |
| **首部** | 20 字节起 | 8 字节 |
| **适用** | 文件、网页、邮件 | 实时音视频、DNS 查询、广播 |
| **流量/拥塞控制** | 有 | 无 |

---

## 二、三次握手与四次挥手

| 阶段 | 过程 | 目的/注意 |
|------|------|-----------|
| **三次握手** | 客户端 SYN → 服务端 SYN+ACK → 客户端 ACK | 双方确认收发能力、协商序列号；两次无法防止历史连接与重复 SYN |
| **四次挥手** | 主动方 FIN → 被动方 ACK → 被动方 FIN → 主动方 ACK | 全双工需双向关闭；被动方可能还有数据要发 |
| **TIME_WAIT** | 主动关闭方在收到对方 FIN 后等待 2MSL | 保证对方收到 ACK、让旧报文过期；端口未释放前不能复用 |
| **CLOSE_WAIT** | 被动方收到 FIN 后未发 FIN 的状态 | 多表示应用未正确关闭连接，需排查代码 |

## 三、HTTP 状态码与方法

| 分类 | 常见 | 说明 |
|------|------|------|
| **2xx** | 200 OK | 成功 |
| **3xx** | 301 永久重定向；302 临时重定向；304 协商缓存命中 | 重定向与缓存 |
| **4xx** | 400 请求错误；401 未认证；403 禁止；404 未找到 | 客户端错误 |
| **5xx** | 500 服务器错误；502 网关错误；503 不可用 | 服务端错误 |

| 方法 | 说明 | 幂等/安全 |
|------|------|-----------|
| **GET** | 获取资源 | 安全、幂等 |
| **POST** | 提交数据、创建资源 | 非幂等 |
| **PUT** | 替换资源 | 幂等 |
| **DELETE** | 删除资源 | 幂等 |

## 四、HTTP 与 HTTPS

| 对比项 | HTTP | HTTPS |
|--------|------|-------|
| **端口** | 80 | 443 |
| **传输** | 明文 | TLS 加密 |
| **身份** | 无 | 证书校验服务器身份 |
| **完整性** | 无 | 防篡改 |

- **HTTPS 握手简述**：Client Hello → Server 证书+Hello → 密钥交换（如 ECDHE）→ 双方得到会话密钥 → 加密通信。

---

## 五、HTTP/1.1 与 HTTP/2

| 对比项 | HTTP/1.1 | HTTP/2 |
|--------|----------|--------|
| **连接** | 多连接或 Keep-Alive 单连接顺序 | 单连接多路复用（多流） |
| **队头阻塞** | 同一连接上请求串行 | 流级别，减少队头阻塞 |
| **头部** | 文本、重复多 | 压缩（HPACK） |
| **数据** | 文本 | 二进制帧 |

## 六、DNS 解析流程

1. 查**本地缓存**（浏览器、系统、hosts）。
2. 请求**本地 DNS**（运营商或配置的 DNS）。
3. 本地 DNS **递归/迭代**查根→顶级域→权威域，得到 A 记录（IP）。
4. 结果缓存并返回。

- **CNAME**：域名别名，最终再解析到 A；**CDN** 常用 CNAME 指向调度域名。

---

## 七、从输入 URL 到页面展示（简述）

1. **DNS** 解析域名 → IP。
2. **TCP** 三次握手建立连接（HTTPS 再 TLS 握手）。
3. 发 **HTTP** 请求（方法、URL、头、体）。
4. 服务器处理并返回 **HTTP** 响应（状态码、头、体）。
5. 浏览器**解析**（HTML→DOM、CSS→CSSOM、渲染树→布局→绘制）。
6. **TCP** 四次挥手或 Keep-Alive 复用。

---

## 八、面试一句话总结

- **TCP**：可靠、面向连接、三次握手四次挥手、流量与拥塞控制；**UDP**：无连接、不可靠、低开销。
- **三次握手**：确认双方收发能力与序列号；**四次挥手**：全双工需双向关闭；**TIME_WAIT**：保证 ACK 到达、旧报文消散。
- **HTTP**：无状态、请求响应；**HTTPS**：TLS 加密+身份校验；**HTTP/2**：多路复用、头部压缩。
- **DNS**：域名→IP，本地缓存→递归/迭代查询；**CDN**：边缘节点、就近访问、减轻源站压力。
- **GET 与 POST**：GET 取资源、参数在 URL、可缓存；POST 提交数据、体在 body、非幂等。
