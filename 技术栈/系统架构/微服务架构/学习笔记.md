# 📘《微服务架构学习与实战手册》

> 系统学习见下文各章；日常常用概念与使用场景速查见同目录《常用概念与使用场景》。

------

## 第1章：微服务架构概述

### 1.1 什么是微服务

- **微服务**是将单体应用拆分为一组**小服务**，每个服务围绕单一业务能力、独立部署与扩展、使用轻量通信（HTTP/RPC），可独立技术选型与数据库。
- 与单体对比：单体简单但扩展与发布耦合；微服务灵活但带来分布式复杂度（网络、一致性、运维）。

### 1.2 核心要素

- **服务拆分**：按业务域/限界上下文划分，避免过大或过细；**独立部署**：每个服务可单独发版；**轻量通信**：REST、gRPC、消息队列；**去中心化**：数据、治理可分散，通过网关与注册中心统一入口与发现。

------

## 第2章：服务注册与发现

1. **为什么需要**：服务实例动态扩缩、IP 变动，调用方无法写死地址，需通过“注册中心”发现实例。
2. **流程**：服务启动时向注册中心注册（IP、端口、元数据）；调用方从注册中心拉取或订阅实例列表；健康检查剔除不可用实例。
3. **常见组件**：Nacos、Eureka、Consul、Zookeeper；Nacos 同时支持配置中心，国内使用多。
4. **与 Spring Cloud**：@EnableDiscoveryClient、spring-cloud-starter-alibaba-nacos-discovery，RestTemplate/OpenFeign 配合负载均衡从注册中心取实例。

------

## 第3章：API 网关

1. **职责**：统一入口、路由转发、鉴权、限流、熔断、日志、协议转换；前端/客户端只认网关，后端服务可隐藏与变更。
2. **路由**：按 path/host 将请求转发到不同服务；可配合注册中心做服务发现。
3. **常见组件**：Spring Cloud Gateway（响应式）、Zuul、Kong、Nginx 作网关；Gateway 基于 WebFlux，支持断言、过滤器、限流。
4. **面试要点**：网关与 Nginx 区别（网关偏业务与治理，Nginx 偏流量与静态）、网关能做什么（鉴权、限流、路由、聚合）。

------

## 第4章：配置中心

1. **为什么需要**：配置集中管理、动态刷新、多环境隔离，避免改配置重启、配置散落各机器。
2. **流程**：配置存于配置中心；应用启动或定时拉取/订阅；支持 @RefreshScope 等动态生效。
3. **常见组件**：Nacos Config、Spring Cloud Config、Apollo；Nacos 与注册中心一体，简单场景常用。
4. **注意点**：敏感信息加密、配置版本与回滚、多环境（dev/test/prod）隔离。

------

## 第5章：服务间调用与通信

1. **同步**：REST（OpenFeign）、gRPC；简单直接，需考虑超时、重试、熔断。
2. **异步**：消息队列（RabbitMQ、Kafka）；解耦、削峰、最终一致性；适合事件驱动、日志收集。
3. **OpenFeign**：声明式 HTTP 客户端，集成负载均衡与熔断；定义接口即可调用远程服务。
4. **负载均衡**：客户端（Ribbon、Spring Cloud LoadBalancer）或网关层；策略：轮询、随机、权重、最少连接。

------

## 第6章：熔断、限流与降级

1. **熔断**：下游故障时快速失败、避免雪崩；半开状态探测恢复；Sentinel、Resilience4j、Hystrix（已停更）。
2. **限流**：控制 QPS/并发数，保护系统；令牌桶、漏桶、计数器；网关或应用内（Sentinel、Guava RateLimiter）。
3. **降级**：非核心功能关闭或返回默认值，保证核心可用；与熔断配合，如返回兜底数据。
4. **面试要点**：熔断与限流区别、Sentinel 规则类型（流控、熔断、热点、系统）、如何防止雪崩。

------

## 第7章：分布式事务与链路

1. **分布式事务**：跨服务、跨库的一致性；方案：Seata（AT/TCC/SAGA）、本地消息表、最终一致性；校招需知概念与取舍（强一致 vs 最终一致）。
2. **链路追踪**：请求跨多服务，需 traceId 串联；Sleuth + Zipkin、SkyWalking、Jaeger；便于排查与性能分析。
3. **面试要点**：微服务带来的挑战（分布式事务、链路、运维）、Spring Cloud 与 Dubbo 简要对比。
