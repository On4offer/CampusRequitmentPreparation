# 📌 微服务架构常用概念与使用场景速查

> 配合《学习笔记》使用，侧重**注册发现、网关、配置、熔断限流**，便于速查与校招面试。

---

## 一、服务注册与发现

| 组件 | 特点 | 常用场景 |
|------|------|----------|
| **Nacos** | 注册+配置、AP/CP 可切换、国内生态好 | Spring Cloud Alibaba、多环境配置 |
| **Eureka** | 注册中心、AP、Netflix 系 | Spring Cloud Netflix（已较少新用） |
| **Consul** | 服务发现+健康检查+KV | 多语言、云原生 |
| **Zookeeper** | CP、强一致 |  Dubbo、Kafka 等依赖 ZK 的场景 |

- **流程**：服务启动 → 注册到中心；调用方 → 拉取/订阅实例列表 → 负载均衡选实例 → 发起调用。

---

## 二、API 网关

| 能力 | 说明 |
|------|------|
| **路由** | 按 path/host 转发到后端服务 |
| **鉴权** | 统一校验 Token、放行/拒绝 |
| **限流** | 全局限流、按 path/用户限流 |
| **熔断/降级** | 下游异常时快速失败或返回兜底 |
| **日志/监控** | 统一记录请求、耗时、状态码 |

| 组件 | 说明 |
|------|------|
| **Spring Cloud Gateway** | 基于 WebFlux、断言+过滤器、与 Spring 生态集成 |
| **Kong** | 基于 Nginx/OpenResty、插件丰富 |
| **Nginx** | 可做反向代理与简单路由，偏流量层 |

---

## 三、配置中心

| 能力 | 说明 |
|------|------|
| **集中管理** | 配置存中心，应用拉取或订阅 |
| **动态刷新** | 改配置不重启（如 @RefreshScope） |
| **多环境** | dev/test/prod 命名空间或分组 |

- **Nacos Config**：bootstrap 中配置 spring.cloud.nacos.config；支持 dataId、group、namespace。
- **注意**：敏感配置加密；大促前可锁定配置避免误改。

---

## 四、服务调用

| 方式 | 典型 | 特点 |
|------|------|------|
| **REST + OpenFeign** | Spring Cloud OpenFeign | 声明式、集成负载均衡与熔断 |
| **gRPC** | 跨语言、高性能 | 二进制、流式，适合内部高性能调用 |
| **消息队列** | RabbitMQ、Kafka | 异步、解耦、削峰、最终一致 |

- **负载均衡**：Spring Cloud LoadBalancer（轮询等）；Ribbon 已进入维护，新项目用 LoadBalancer。

## 五、熔断、限流、降级

| 概念 | 目的 | 典型实现 |
|------|------|----------|
| **熔断** | 下游故障时快速失败，避免雪崩 | Sentinel 熔断规则、Resilience4j CircuitBreaker |
| **限流** | 控制 QPS/并发，保护系统 | Sentinel 流控、网关限流、Guava RateLimiter |
| **降级** | 非核心关掉或返回默认，保核心 | 熔断后返回 fallback、开关关闭非核心功能 |

- **Sentinel**：流控规则（QPS/并发数）、熔断规则（慢调用比例/异常比例/异常数）、热点参数限流、系统自适应。

## 六、分布式事务（概念速查）

| 方案 | 思路 | 适用 |
|------|------|------|
| **Seata AT** | 自动回滚、一阶段提交二阶段异步/同步回滚 | 一般业务、对性能要求高 |
| **TCC** |  Try-Confirm-Cancel，业务自己实现补偿 | 强一致、可接受改业务代码 |
| **本地消息表** | 本地事务+发消息、定时扫表补偿 | 最终一致、实现简单 |
| **MQ 事务消息** | 半消息+本地事务+确认/回滚 | 与 MQ 强绑定的场景 |

---

## 七、面试一句话总结

- **注册发现**：服务自注册、调用方从中心拉实例，实现动态扩缩与故障剔除。
- **网关**：统一入口，做路由、鉴权、限流、熔断、日志。
- **配置中心**：配置集中、动态刷新、多环境，避免改配置重启。
- **熔断**：下游挂了快速失败；**限流**：控 QPS/并发；**降级**：非核心关掉保核心。
- **微服务挑战**：分布式事务、链路追踪、运维与部署复杂度。
