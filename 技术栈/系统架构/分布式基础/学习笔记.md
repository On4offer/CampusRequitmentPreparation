# 📘《分布式基础学习与实战手册》

> 系统学习见下文各章；日常常用概念与使用场景速查见同目录《常用概念与使用场景》。

------

## 第1章：分布式概述

### 1.1 什么是分布式系统

- **分布式系统**由多台计算机协同完成同一目标，通过**网络**通信；特点：无全局时钟、节点故障独立、网络延迟与分区。
- 带来的问题：**一致性**、**可用性**、**分区容错**（CAP）；**共识**（谁说了算）；**分布式事务**、**分布式 ID**、**分布式锁**等。

### 1.2 CAP 与 BASE

- **CAP**：在分布式系统中，**C**onsistency（一致性）、**A**vailability（可用性）、**P**artition tolerance（分区容错）三者最多同时满足两个；实际中 P 必须接受，故在 C 与 A 间权衡。
- **BASE**：**B**asically **A**vailable（基本可用）、**S**oft state（软状态）、**E**ventually consistent（最终一致）；用最终一致性替代强一致，提高可用性。

## 第2章：一致性协议与共识

1. **强一致**：读总能读到最新写；代价高，多用于单机或小范围。
2. **最终一致**：允许一段时间不一致，最终达到一致；常用异步复制、消息队列、版本号/向量时钟。
3. **共识算法**：多节点对同一值达成一致；**Paxos**、**Raft**（易理解，Etcd、Consul 用）、**ZAB**（Zookeeper）；用于注册中心、配置中心、分布式协调。
4. **面试要点**：CAP 含义与取舍、Raft 选主与日志复制简述、为什么需要共识（多副本一致）。

## 第3章：分布式 ID

1. **为什么需要**：分库分表、多节点生成主键不能重复；单机自增无法保证全局唯一。
2. **方案**：**UUID**（简单但无序、占空间）；**雪花算法**（Snowflake，时间+机器+序列，趋势递增）；**号段/步长**（DB 或 Redis 取一段）；**Redis 自增**（需持久化与高可用）；**数据库号段表**（如 Leaf）。
3. **选型**：有序与可读性要求高用雪花或号段；简单场景可用 UUID。
4. **面试要点**：雪花算法结构（1+41+10+12）、号段如何减少 DB 压力。

------

## 第4章：分布式锁

1. **为什么需要**：多实例下同一资源只能被一个实例处理，避免重复扣减、重复发奖等。
2. **实现**：**Redis**（SET NX EX、Lua 保证原子性、Redisson 看门狗续期）；**Zookeeper**（临时顺序节点、最小节点获锁）；**数据库**（唯一索引或乐观锁）。
3. **注意点**：锁的粒度、过期时间与续期、可重入、锁误删（只能删自己的锁，用 value 标识）。
4. **面试要点**：Redis 与 ZK 实现锁的优缺点、Redisson 看门狗、如何防止误删。

------

## 第5章：分布式事务

1. **场景**：跨服务、跨库的多个操作要么全成功要么全失败；如订单+扣库存+积分。
2. **强一致方案**：**2PC/3PC**（协调者+参与者，性能与可用性差，少用）；**Seata AT/TCC**（见微服务章节）。
3. **最终一致方案**：**本地消息表**（本地事务写业务+消息表，定时发 MQ，消费端幂等）；**MQ 事务消息**（半消息+本地事务+确认）；**Saga**（长事务拆成多步+补偿）。
4. **面试要点**：强一致 vs 最终一致、Seata 与 MQ 事务消息的适用场景、幂等的必要性。

------

## 第6章：分布式缓存与 Session

1. **缓存一致性**：先更新 DB 再删缓存（Cache-Aside）；延迟双删；设置过期时间兜底；强一致难，多以最终一致+过期为主。
2. **缓存穿透**：查不存在的数据，打到 DB；布隆过滤器、空值缓存。
3. **缓存击穿**：热点 key 过期瞬间大量请求打 DB；互斥锁、逻辑过期（不设过期由逻辑判断）。
4. **缓存雪崩**：大量 key 同时过期或缓存宕机；过期时间打散、多级缓存、限流降级。
5. **分布式 Session**：Session 存 Redis，多实例共享；或 JWT 无状态。

------

## 第7章：分库分表与读写分离

1. **分库分表**：数据量过大时水平拆分；按 user_id、order_id 等取模或范围分片；带来跨库 join、分布式事务、全局唯一 ID 问题。
2. **读写分离**：写主库，读从库；主从复制延迟导致读不一致，可强制读主或接受短暂延迟。
3. **中间件**：ShardingSphere、MyCat 等做分片与读写分离路由；校招需知概念与适用场景。
4. **面试要点**：何时分库分表、分片键选择、读写分离延迟怎么处理。
