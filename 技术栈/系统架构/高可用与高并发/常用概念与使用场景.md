# 📌 高可用与高并发常用概念与使用场景速查

> 配合《学习笔记》使用，侧重**限流、熔断、负载均衡、冗余与压测**，便于速查与校招面试。

---

## 一、限流

| 算法 | 思路 | 特点 |
|------|------|------|
| **固定窗口计数** | 单位时间内计数，超则拒绝 | 实现简单；临界突发（两窗口交界处） |
| **滑动窗口** | 按当前时间往前推窗口内计数 | 更平滑，略复杂 |
| **漏桶** | 请求进桶，以恒定速率流出 | 平滑输出，不允许突发 |
| **令牌桶** | 恒定速率放令牌，请求取令牌 | 允许一定突发，**常用** |

| 实现 | 说明 |
|------|------|
| **Guava RateLimiter** | 单机令牌桶 |
| **Sentinel** | 流控规则、集群流控、与 Spring 集成 |
| **Nginx limit_req** | 漏桶，按 IP/接口限流 |
| **Redis + Lua** | 分布式限流（计数、滑动窗口等） |

## 二、熔断与降级

| 概念 | 说明 |
|------|------|
| **熔断** | 下游故障时快速失败，不继续调用；三态：关闭→打开→半开（探测） |
| **降级** | 关闭非核心或返回兜底数据，保核心可用 |
| **Sentinel 熔断** | 慢调用比例、异常比例、异常数；达到阈值则熔断，过后半开探测 |

- **区别**：限流是“流量过大就拦”；熔断是“下游坏了就停请求”；降级是“主动关功能或返回默认”。

## 三、负载均衡

| 层级 | 典型 | 说明 |
|------|------|------|
| **DNS** | 域名解析到多 IP | 粗粒度、有缓存，变更慢 |
| **四层** | LVS、F5 | TCP/UDP 转发，性能高，不解析 HTTP |
| **七层** | Nginx、Gateway | 按 URL、Host 等转发，可做会话保持、重写 |

| 策略 | 说明 |
|------|------|
| **轮询 / 加权轮询** | 依次或按权重分配 |
| **随机** | 随机选后端 |
| **最少连接** | 选当前连接数最少的 |
| **一致性哈希** | 同一 key 总到同一节点，适合缓存 |

---

## 四、冗余与故障转移

| 手段 | 说明 |
|------|------|
| **多实例** | 应用无状态、多副本，前面负载均衡 |
| **主从** | DB、Redis 主从复制，主挂则提升从 |
| **哨兵/集群** | Redis 哨兵自动选主；集群分片 |
| **健康检查** | 心跳、HTTP 探针；失败则摘除或熔断 |
| **多机房** | 同城双活、异地多活；流量调度与数据同步 |

---

## 五、高并发常用手段

| 手段 | 说明 |
|------|------|
| **缓存** | 减少 DB/下游压力，多级缓存、热点前置 |
| **异步/MQ** | 削峰、解耦、快速响应；注意堆积与幂等 |
| **扩容** | 水平加机器、弹性伸缩 |
| **限流** | 保护自身与下游，防止过载 |
| **连接池** | DB、Redis、HTTP 连接池，避免连接耗尽 |
| **CDN/静态分离** | 静态资源走 CDN，减轻源站压力 |

---

## 六、压测与容量

| 指标 | 说明 |
|------|------|
| **QPS/TPS** | 每秒请求数/事务数 |
| **RT** | 响应时间（平均、P99） |
| **错误率** | 5xx、超时比例 |
| **资源** | CPU、内存、连接数、磁盘 IO |

- **流程**：定目标 QPS/RT → 压测工具模拟 → 观察瓶颈（DB、Redis、线程池、网络）→ 优化或扩容 → 留冗余。

---

## 七、面试一句话总结

- **限流**：控 QPS/并发，算法有计数器、漏桶、令牌桶；网关或应用内实现。
- **熔断**：下游故障快速失败，三态；**降级**：关非核心或兜底，保核心。
- **负载均衡**：四层转发快、七层可按 URL；策略有轮询、最少连接、一致性哈希。
- **高可用**：多副本、主从、健康检查、故障转移、多机房。
- **高并发**：缓存、异步/MQ、扩容、限流、连接池、CDN。
- **压测**：看 QPS、RT、错误率、资源，找瓶颈并预留容量。
