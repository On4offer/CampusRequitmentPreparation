# 📘《高可用与高并发学习与实战手册》

> 系统学习见下文各章；日常常用概念与使用场景速查见同目录《常用概念与使用场景》。

------

## 第1章：高可用与高并发概述

### 1.1 高可用（HA）

- **高可用**指系统在约定时间内持续可用，通常用**可用率**（如 99.9%、99.99%）和**故障恢复时间**（RTO）衡量。
- 手段：**冗余**（多副本、多机房）、**故障转移**（主从、自动摘除）、**熔断降级**、**限流**、**监控与告警**。

### 1.2 高并发

- **高并发**指单位时间内大量请求同时到达，系统需在**响应时间**与**吞吐量**可接受下稳定处理。
- 手段：**水平扩容**、**缓存**、**异步与消息队列**、**限流**、**数据库与连接池优化**、**CDN 与静态资源分离**。

------

## 第2章：限流

1. **目的**：保护系统与下游，防止过载导致雪崩；保证公平性或按用户/接口分配资源。
2. **算法**：**计数器**（固定窗口、滑动窗口）、**漏桶**（恒定速率输出）、**令牌桶**（恒定速率放令牌，请求取令牌）；令牌桶允许一定突发，常用。
3. **实现层级**：网关（Nginx、Gateway）、应用内（Sentinel、Guava RateLimiter）、分布式（Redis+Lua 限流）。
4. **面试要点**：限流与熔断区别、令牌桶与漏桶区别、单机限流与分布式限流。

------

## 第3章：熔断与降级

1. **熔断**：下游故障或慢调用比例高时，快速失败、不再请求下游，避免雪崩；半开状态定期探测恢复。详见微服务章节。
2. **降级**：非核心功能关闭或返回默认/缓存数据，保证核心链路可用；可与熔断、开关配合。
3. **场景**：大促关推荐、关评论；依赖超时则返回兜底数据；读多写少场景读缓存、写失败可稍后重试。
4. **面试要点**：熔断三态（开/关/半开）、降级与限流的配合、如何设计降级策略。

------

## 第4章：负载均衡

1. **目的**：将流量分散到多台实例，提高吞吐与可用性；避免单点过载。
2. **层级**：**DNS**（粗粒度）、**四层**（LVS、F5、TCP/UDP）、**七层**（Nginx、Gateway、HTTP）。
3. **策略**：轮询、加权轮询、随机、最少连接、一致性哈希（同一客户端固定到同一后端、缓存场景常用）。
4. **面试要点**：四层与七层区别、一致性哈希解决什么问题、Nginx 与 LVS 的定位。

------

## 第5章：冗余与故障转移

1. **无单点**：关键组件多副本（应用多实例、DB 主从、Redis 哨兵/集群、MQ 集群）。
2. **故障发现**：健康检查（HTTP/TCP）、心跳、注册中心剔除。
3. **故障转移**：主从切换（DB、Redis 哨兵）、流量摘除故障节点、熔断跳过故障下游。
4. **多机房/多活**：同城双活、异地多活；数据同步与一致性、流量调度（DNS、网关）。

------

## 第6章：异步、消息队列与缓存

1. **异步**：耗时操作异步处理，快速响应；线程池、CompletableFuture、消息队列。
2. **消息队列**：削峰填谷、解耦、异步；Kafka 高吞吐、RabbitMQ 功能丰富；注意堆积、顺序、幂等。
3. **缓存**：多级缓存（本地+Redis）、热点数据、减少 DB 压力；注意穿透/击穿/雪崩（见分布式基础）。
4. **面试要点**：为什么用 MQ、如何保证消息不丢与幂等、缓存与 DB 的配合。

------

## 第7章：容量规划与压测

1. **容量规划**：根据 QPS、响应时间、机器配置估算实例数；留冗余应对峰值。
2. **压测**：模拟高并发（JMeter、ab、wrk）；观察 QPS、RT、错误率、CPU/内存；找到瓶颈（DB、Redis、下游、线程池）。
3. **扩容**：水平扩容（加机器）、垂直扩容（升配）；弹性伸缩（K8s HPA、云厂商自动伸缩）。
4. **面试要点**：如何评估系统能扛多少 QPS、压测关注哪些指标、如何做容量预留。
