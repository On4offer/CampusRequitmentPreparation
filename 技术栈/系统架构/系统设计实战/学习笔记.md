------

## 第1章：系统设计面试概述

### 1.1 考察目的

- 校招/社招中的**系统设计**考察：需求理解、抽象能力、技术选型、权衡（一致性/可用性/成本）、扩展性与落地思路。
- 不要求实现细节，要求**思路清晰、能拆问题、能说 trade-off**；常与高并发、分布式、缓存、MQ 结合。

### 1.2 答题步骤（通用）

1. **澄清需求**：功能（核心/次要）、用户量/QPS、读写比、时延要求、一致性要求。
2. **估算**：QPS、存储量、带宽等（量级即可）。
3. **顶层设计**：画框图——客户端、网关、服务、缓存、DB、MQ；数据流与职责。
4. **核心链路深挖**：如秒杀——如何防超卖、如何限流、如何削峰；如短链——如何生成、如何跳转、如何统计。
5. **扩展与优化**：分库分表、多级缓存、异步、降级；瓶颈与改进。
6. **小结**：方案总结与可替代方案（如强一致 vs 最终一致）。

## 第2章：秒杀系统

1. **特点**：瞬时高并发、读多写多、库存有限、防超卖、防黄牛。
2. **思路**：**前端**限流与防重复提交；**网关**限流；**服务**多级缓存（页面静态化、商品信息缓存）、**库存**预扣减（Redis 扣减 + 异步落库或 MQ 消峰）、**下单**异步化（MQ）+ 幂等；**DB** 乐观锁或分布式锁防超卖。
3. **防超卖**：Redis 预扣减 + 扣减成功再发 MQ 写订单与 DB 扣库存；或 DB 乐观锁 `update ... where stock > 0`。
4. **面试要点**：为什么用 Redis 扣库存、如何保证不超卖、如何削峰、如何限流与降级。

------

## 第3章：短链服务

1. **需求**：长 URL 转短链（如 t.cn/xxx）；跳转时根据短码查长链并 302 重定向；可选：统计访问量、过期时间。
2. **短码生成**：**发号器**（分布式 ID）取唯一 ID，再转 62 进制或 Base62 得到短码；或 **Hash**（MD5 取前 N 位，需处理冲突）。
3. **存储**：短码 → 长链 存 Redis（热）+ DB（持久）；读多写少，缓存为主。
4. **跳转**：GET 短链 → 查 Redis/DB → 302 Location: 长链；可记录 PV/UV（异步写或 MQ）。
5. **面试要点**：短码如何唯一、如何高并发读、如何防重与过期。

------

## 第4章：点赞、计数与 Feed 流

1. **点赞**：读多写多；Redis Set 存某内容被谁点赞、计数用 Redis 或异步汇总；DB 做持久与最终一致；防重复点赞（同一用户同一内容只加一次）。
2. **计数**：高并发下直接写 DB 压力大；用 Redis 计数，定时或批量同步 DB；或 MQ 异步写。
3. **Feed 流**：关注的人发的内容按时间线展示。**推模式**：发帖时写进每个粉丝的 Feed 收件箱（写扩散，写多读少，适合粉丝少）；**拉模式**：读时聚合关注人的动态（读扩散，读多写少，适合大 V）；**推拉结合**：大 V 用拉、普通用户用推。
4. **面试要点**：推拉模式取舍、如何保证时序与去重、存储结构（时间线表、收件箱表）。

------

## 第5章：即时通讯（IM）与消息队列

1. **IM 简述**：客户端与服务器长连接（WebSocket、TCP）；消息可经服务端转发（存储可选）、或 P2P 经服务端信令。多端同步、未读计数、离线消息、群聊与推送。
2. **消息不丢**：客户端 ACK、服务端持久化、重试与幂等；离线消息拉取与已读同步。
3. **高并发**：连接层水平扩展、消息写 MQ 异步落库、读扩散与在线状态（Redis）。
4. **面试要点**：长连接与心跳、消息顺序与去重、如何扩展、与 MQ 的区别（IM 是端到端，MQ 是系统间）。

------

## 第6章：缓存与存储设计

1. **何时用缓存**：读多写少、热点明显、可接受短暂不一致；多级缓存（本地 + Redis）、缓存更新策略（Cache-Aside、过期）。
2. **存储选型**：关系型（事务、复杂查询）、KV（Redis）、搜索引擎（Elasticsearch）、时序/日志（按场景）；分库分表与读写分离（见分布式基础）。
3. **面试要点**：缓存一致性、缓存穿透/击穿/雪崩、为什么选 Redis/MySQL/ES。

------

## 第7章：综合与面试技巧

1. **先总后分**：先给整体框图，再按模块或链路展开；面试官追问哪就深挖哪。
2. **说 trade-off**：如“这里用最终一致是因为可用性更重要”“用 Redis 扣库存是为了抗住瞬时 QPS，再异步落库”。
3. **常见场景补充**：设计一个延迟队列、设计一个搜索建议（前缀匹配）、设计一个排行榜（ZSet）、设计一个分布式 Session。
4. **与前面章节联动**：微服务（网关、注册发现）、分布式（ID、锁、事务）、高可用（限流、熔断、负载均衡）都会在系统设计题里用到。
