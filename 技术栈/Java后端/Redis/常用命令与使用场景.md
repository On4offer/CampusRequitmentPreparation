## 一、CLI 常用命令速查

### 1.1 通用与 Key

| 命令 | 说明 | 示例 |
|------|------|------|
| **SET key value [NX\|XX] [EX s\|PX ms]** | 设值，NX 不存在才设，XX 存在才设，EX/PX 过期 | SET user:1 "{}" EX 3600 |
| **GET key** | 取值 | GET user:1 |
| **DEL key [key ...]** | 删除 | DEL user:1 |
| **EXISTS key** | 是否存在 | EXISTS user:1 |
| **EXPIRE key seconds** | 设置过期时间（秒） | EXPIRE key 60 |
| **TTL key** | 剩余存活时间，-1 永不过期，-2 不存在 | TTL key |
| **KEYS pattern** | 匹配 key（生产慎用，阻塞） | KEYS user:* |
| **SCAN cursor [MATCH pat] [COUNT n]** | 渐进式遍历，替代 KEYS | SCAN 0 MATCH user:* COUNT 100 |
| **TYPE key** | 类型 | TYPE user:1 |

### 1.2 String

| 命令 | 说明 | 示例 |
|------|------|------|
| **INCR / DECR key** | 自增/自减 | INCR count |
| **INCRBY key n** | 步长自增 | INCRBY stock -1 |
| **MSET / MGET** | 批量设/取 | MGET k1 k2 k3 |
| **SETEX key seconds value** | 设值并过期 | SETEX code 300 "1234" |

### 1.3 Hash

| 命令 | 说明 | 示例 |
|------|------|------|
| **HSET key field value** | 设字段 | HSET user:1 name "tom" |
| **HGET key field** | 取字段 | HGET user:1 name |
| **HGETALL key** | 取全部字段 | HGETALL user:1 |
| **HINCRBY key field n** | 字段自增 | HINCRBY user:1 age 1 |
| **HDEL key field [field ...]** | 删字段 | HDEL user:1 remark |

### 1.4 List

| 命令 | 说明 | 示例 |
|------|------|------|
| **LPUSH / RPUSH key value [value ...]** | 左/右入队 | LPUSH queue task1 |
| **LPOP / RPOP key** | 左/右出队 | RPOP queue |
| **BLPOP / BRPOP key timeout** | 阻塞出队 | BLPOP queue 10 |
| **LRANGE key start stop** | 范围查询 | LRANGE list 0 -1 |

### 1.5 Set

| 命令 | 说明 | 示例 |
|------|------|------|
| **SADD key member [member ...]** | 添加成员 | SADD tags java redis |
| **SMEMBERS key** | 所有成员 | SMEMBERS tags |
| **SISMEMBER key member** | 是否成员 | SISMEMBER tags java |
| **SINTER key1 key2** | 交集 | SINTER set1 set2 |

### 1.6 ZSet（有序集合）

| 命令 | 说明 | 示例 |
|------|------|------|
| **ZADD key score member [score member ...]** | 添加，score 为分数 | ZADD rank 100 user1 |
| **ZRANGE key start stop [WITHSCORES]** | 按排名升序取 | ZRANGE rank 0 9 WITHSCORES |
| **ZREVRANGE key start stop** | 按排名降序（排行榜） | ZREVRANGE rank 0 9 |
| **ZINCRBY key increment member** | 分数增加 | ZINCRBY rank 10 user1 |
| **ZRANK key member** | 成员排名（升序） | ZRANK rank user1 |

### 1.7 分布式锁（常用写法）

```bash
# 加锁（推荐 SET 一条命令）
SET lock:key "uuid" NX PX 30000
# 解锁（需 Lua 保证只删自己的锁，避免删别人）
# Lua: if redis.call("get", KEYS[1]) == ARGV[1] then return redis.call("del", KEYS[1]) else return 0 end
```

## 二、Java 侧常用 API（Spring）

### 2.1 StringRedisTemplate（最常用）

| 方法 | 说明 | 示例 |
|------|------|------|
| **opsForValue().set(key, value)** | 设值 | opsForValue().set("k", "v") |
| **opsForValue().set(key, value, timeout, unit)** | 设值带过期 | set("k", "v", 10, TimeUnit.MINUTES) |
| **opsForValue().get(key)** | 取值 | opsForValue().get("k") |
| **opsForValue().setIfAbsent(key, value)** | NX，可做简单锁 | setIfAbsent("lock", "1", 10, TimeUnit.SECONDS) |
| **opsForValue().increment(key)** | INCR | opsForValue().increment("count") |
| **delete(key)** | 删除 | delete("k") |
| **expire(key, timeout, unit)** | 设置过期 | expire("k", 1, TimeUnit.HOURS) |

### 2.2 Hash / List / Set / ZSet

- **opsForHash()**：put/get/entries/increment；**opsForList()**：leftPush/rightPop/range；**opsForSet()**：add/members；**opsForZSet()**：add/rangeByRankWithScores/reverseRange。用法与 CLI 一一对应，见 Spring Data Redis 文档。

### 2.3 Redisson（分布式锁推荐）

- **RLock lock = redisson.getLock("lockKey")**；**lock.lock()** / **lock.tryLock(waitTime, leaseTime, unit)**；**lock.unlock()**；可重入、看门狗自动续期，生产推荐用 Redisson 替代手写 SET NX。

## 三、使用场景对照

| 场景 | 推荐命令/结构 | 说明 |
|------|----------------|------|
| 缓存对象 | String SET/GET，值存 JSON | 设置合理 TTL，更新时先 DB 再 DEL |
| 计数器/库存 | INCR/DECR 或 HINCRBY | 扣减用 Lua 或 Redisson 保证原子性 |
| 分布式锁 | SET NX EX + Lua 解锁，或 Redisson | 避免死锁（过期）、只删自己的锁 |
| 排行榜 | ZSet ZADD/ZREVRANGE/ZINCRBY | 按分数排序，取 Top N |
| 消息队列 | List LPUSH + BRPOP | 简单队列；可靠投递用 Stream 或 MQ |
| 签到/标签 | Set SADD/SISMEMBER 或 Bitmap | 去重、存在性判断 |
| 限流 | INCR + EXPIRE 或滑动窗口 | 固定窗口：key 按秒/分，INCR 超阈值拒绝 |
| Session 共享 | String/Hash 存用户信息 | 设置 session 过期时间 |

## 四、常见问题排查

| 现象 | 可能原因 | 处理思路 |
|------|----------|----------|
| 连接失败 | 端口、防火墙、requirepass | 检查 redis.conf、bind、密码 |
| 内存涨满 | 未设 maxmemory、大 key、无淘汰 | 设 maxmemory + 淘汰策略、查 bigkeys、优化 key |
| 慢查询 | 大 key、KEYS、复杂命令 | SLOWLOG、避免 KEYS 用 SCAN、拆分大 key |
| 主从延迟 | 网络、从库负载、全量同步 | 看 replication offset/lag、优化网络与从库 |
| 缓存不一致 | 先删缓存再更新 DB、并发 | 先更新 DB 再删缓存；或 Canal 失效缓存 |
| 锁失效 | 过期时间过短、未续期 | Redisson 看门狗或业务侧续期；保证只删自己的锁 |
