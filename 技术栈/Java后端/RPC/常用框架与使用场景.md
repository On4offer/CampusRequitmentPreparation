### 服务实现

```java
@Service  // Spring的@Service
@DubboService(  // Dubbo的服务注解
    version = "1.0.0",
    group = "default",
    timeout = 5000,
    retries = 3,
    loadbalance = "random",
    mock = "return null"  // 降级策略
)
@Slf4j
public class UserServiceImpl implements UserService {
    
    @Autowired
    private UserMapper userMapper;
    
    @Override
    public User getUserById(Long id) {
        log.info("Get user by id: {}", id);
        return userMapper.selectById(id);
    }
    
    @Override
    public List<User> listUsers(UserQuery query) {
        PageHelper.startPage(query.getPageNum(), query.getPageSize());
        return userMapper.selectByQuery(query);
    }
    
    @Override
    public User createUser(UserCreateRequest request) {
        User user = new User();
        BeanUtils.copyProperties(request, user);
        userMapper.insert(user);
        return user;
    }
    
    @Override
    @Transactional
    public User updateUser(Long id, UserUpdateRequest request) {
        User user = userMapper.selectById(id);
        if (user == null) {
            throw new UserNotFoundException("User not found: " + id);
        }
        BeanUtils.copyProperties(request, user);
        userMapper.update(user);
        return user;
    }
    
    @Override
    public void deleteUser(Long id) {
        userMapper.deleteById(id);
    }
}
```

### Dubbo高级配置

```yaml
# 集群容错配置
dubbo:
  consumer:
    cluster: failover            # 容错模式：failover/failfast/failsafe/failback/forking/broadcast
    
# 各模式说明：
# failover: 失败自动切换（默认），调用失败自动切换其他实例
# failfast: 快速失败，只调用一次，失败立即报错
# failsafe: 失败安全，调用失败忽略，记录日志
# failback: 失败自动恢复，失败定时重发
# forking: 并行调用多个实例，一个成功即返回
# broadcast: 广播调用所有实例，一个失败即报错
```

### Dubbo异步调用

```java
@Service
public class AsyncOrderService {
    
    @DubboReference(async = true)
    private UserService userService;
    
    public void asyncProcess() {
        // 发起异步调用
        userService.getUserById(1L);
        
        // 获取Future
        Future<User> future = RpcContext.getContext().getFuture();
        
        // 执行其他操作...
        
        try {
            // 获取结果（阻塞等待）
            User user = future.get(3, TimeUnit.SECONDS);
        } catch (Exception e) {
            log.error("获取用户失败", e);
        }
    }
}
```

### gRPC服务端

```java
@Component
@Slf4j
public class GrpcUserService extends UserServiceGrpc.UserServiceImplBase {
    
    @Autowired
    private UserMapper userMapper;
    
    @Override
    public void getUserById(GetUserRequest request, StreamObserver<User> responseObserver) {
        log.info("Get user by id: {}", request.getId());
        
        com.example.entity.User entity = userMapper.selectById(request.getId());
        
        User user = User.newBuilder()
            .setId(entity.getId())
            .setName(entity.getName())
            .setEmail(entity.getEmail())
            .setAge(entity.getAge())
            .build();
        
        responseObserver.onNext(user);
        responseObserver.onCompleted();
    }
    
    @Override
    public void listUsers(ListUsersRequest request, StreamObserver<ListUsersResponse> responseObserver) {
        // 查询逻辑...
        List<com.example.entity.User> entities = userMapper.selectList(query);
        
        ListUsersResponse.Builder responseBuilder = ListUsersResponse.newBuilder();
        for (com.example.entity.User entity : entities) {
            User user = User.newBuilder()
                .setId(entity.getId())
                .setName(entity.getName())
                .build();
            responseBuilder.addUsers(user);
        }
        
        responseObserver.onNext(responseBuilder.build());
        responseObserver.onCompleted();
    }
    
    @Override
    public void createUser(CreateUserRequest request, StreamObserver<User> responseObserver) {
        com.example.entity.User entity = new com.example.entity.User();
        entity.setName(request.getName());
        entity.setEmail(request.getEmail());
        entity.setAge(request.getAge());
        
        userMapper.insert(entity);
        
        User user = User.newBuilder()
            .setId(entity.getId())
            .setName(entity.getName())
            .setEmail(entity.getEmail())
            .setAge(entity.getAge())
            .build();
        
        responseObserver.onNext(user);
        responseObserver.onCompleted();
    }
}
```

### 1. 统一RPC响应处理

```java
@Data
public class RpcResult<T> implements Serializable {
    private static final long serialVersionUID = 1L;
    
    private boolean success;
    private T data;
    private String errorCode;
    private String errorMessage;
    
    public static <T> RpcResult<T> success(T data) {
        RpcResult<T> result = new RpcResult<>();
        result.setSuccess(true);
        result.setData(data);
        return result;
    }
    
    public static <T> RpcResult<T> failure(String errorCode, String errorMessage) {
        RpcResult<T> result = new RpcResult<>();
        result.setSuccess(false);
        result.setErrorCode(errorCode);
        result.setErrorMessage(errorMessage);
        return result;
    }
    
    public T getDataOrThrow() {
        if (!success) {
            throw new RpcException(errorCode, errorMessage);
        }
        return data;
    }
}

// 服务接口使用统一响应
public interface OrderService {
    RpcResult<Order> getOrderById(Long id);
    RpcResult<List<Order>> listOrders(OrderQuery query);
    RpcResult<Order> createOrder(CreateOrderRequest request);
}
```

## ⚠️ 常见坑点速查

| 坑点 | 说明 | 正确做法 |
|------|------|----------|
| 序列化问题 | 对象未实现Serializable或字段不匹配 | 确保DTO实现Serializable，版本号一致 |
| 接口不一致 | 提供方和消费方接口定义不同 | 接口单独打包，双方依赖同一jar |
| 超时设置不当 | 网络抖动导致频繁超时 | 合理设置timeout，配合重试 |
| 线程池耗尽 | 并发高时业务线程池满 | 调整threads参数，或限流 |
| 上下文丢失 | 异步调用时ThreadLocal失效 | 使用Dubbo的Attachment或包装Runnable |
| 循环依赖 | 服务A调B，B又调A | 避免设计循环调用，或重构拆分 |
| 大对象传输 | 传输大量数据导致OOM | 分页查询，或换用流式接口 |
