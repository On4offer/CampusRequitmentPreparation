## 二、配置文件结构速查

### 2.1 主块与 events

```nginx
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /run/nginx.pid;

events {
    worker_connections 1024;
    use epoll;  # Linux 可选
}
```

### 2.2 http 内 server 与 location

```nginx
http {
    include mime.types;
    default_type application/octet-stream;
    sendfile on;
    keepalive_timeout 65;

    server {
        listen 80;
        server_name example.com;
        root /var/www/html;
        index index.html;

        location / {
            try_files $uri $uri/ /index.html;
        }

        location /api/ {
            proxy_pass http://127.0.0.1:8080/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}
```

### 2.3 负载均衡 upstream

```nginx
upstream backend {
    server 127.0.0.1:8080 weight=2;
    server 127.0.0.1:8081 weight=1;
    # ip_hash;        # 同一客户端固定后端
    # least_conn;     # 最少连接
}

server {
    location /api/ {
        proxy_pass http://backend/;
        proxy_connect_timeout 5s;
        proxy_read_timeout 60s;
        proxy_send_timeout 60s;
    }
}
```

## 三、常用指令速查

### 3.1 反向代理

| 指令 | 说明 | 示例 |
|------|------|------|
| **proxy_pass** | 转发到上游 | proxy_pass http://backend/; |
| **proxy_set_header** | 设置请求头 | proxy_set_header Host $host; |
| **proxy_connect_timeout** | 连接超时 | proxy_connect_timeout 10s; |
| **proxy_read_timeout** | 读响应超时 | proxy_read_timeout 60s; |
| **proxy_send_timeout** | 发请求超时 | proxy_send_timeout 60s; |

### 3.2 静态与路径

| 指令 | 说明 | 示例 |
|------|------|------|
| **root** | 根目录，路径=root+URI | root /var/www; |
| **alias** | 路径=alias 替换 location | location /static/ { alias /data/static/; } |
| **try_files** | 依次尝试文件/目录/回退 | try_files $uri $uri/ /index.html; |

### 3.3 重写与返回

| 指令 | 说明 | 示例 |
|------|------|------|
| **return** | 返回状态码或跳转 | return 301 https://$host$request_uri; |
| **rewrite** | 按正则改写 URI | rewrite ^/old/(.*)$ /new/$1 last; |

### 3.4 其他

| 指令 | 说明 |
|------|------|
| **listen** | 监听端口，可加 ssl |
| **server_name** | 虚拟主机名，支持通配与正则 |
| **client_max_body_size** | 请求体上限（上传） |
| **gzip on** | 开启 gzip |
| **expires** | 静态缓存过期，如 expires 30d; |

## 四、location 匹配优先级

| 类型 | 写法 | 优先级 |
|------|------|--------|
| 精确 | **location = /path** | 最高 |
| 前缀（优先） | **location ^~ /path** | 匹配后不再查正则 |
| 正则 | **location ~ \.php$** | 按出现顺序 |
| 普通前缀 | **location /path** | 最低 |

## 五、使用场景对照

| 场景 | 推荐配置 |
|------|----------|
| 前后端分离 | 静态 root/alias + /api proxy_pass 到后端 |
| 多后端负载 | upstream 多 server + weight，location proxy_pass http://upstream/ |
| HTTP 跳 HTTPS | server 80 里 return 301 https://$host$request_uri; |
| 静态缓存 | location ~* \.(js|css|png)$ { expires 30d; } |
| 跨域头 | add_header Access-Control-Allow-Origin *;（或指定域名） |
| 上传大文件 | client_max_body_size 50m; |
| 后端超时 | proxy_connect_timeout / proxy_read_timeout 调大 |
| 单页应用 | try_files $uri $uri/ /index.html; |

## 六、常见问题排查

| 现象 | 可能原因 | 处理思路 |
|------|----------|----------|
| 403 | 权限、root 错误、禁用目录列表 | 检查路径、权限、autoindex |
| 404 | 文件不存在、proxy_pass 路径错 | 查 root/alias、proxy_pass 末尾是否带 / |
| 502 | 后端未启、端口错、连接被拒 | 查 upstream 地址、后端进程 |
| 504 | 后端响应慢 | 调大 proxy_read_timeout |
| 重载失败 | 语法错误 | nginx -t 查错，看 error_log |
