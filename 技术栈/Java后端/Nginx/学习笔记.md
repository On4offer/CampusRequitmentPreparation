# 🧭 《Nginx 学习与实战手册》

> 系统学习见下文各章；日常常用命令、配置与场景速查见同目录《常用配置与使用场景》。

------

## 第1章：Nginx 概述与核心思想

### 1.1 诞生与发展

- Nginx 由 Igor Sysoev 开发，2004 年发布，主打**高并发、低内存**；广泛用于反向代理、负载均衡、静态资源、HTTPS 终结。

### 1.2 与 Apache 对比

- **Nginx**：事件驱动、多 worker 无阻塞、适合高并发与静态；**Apache**：多进程/多线程、模块丰富、适合动态脚本。Nginx 常作前置代理，将动态请求转发给后端（如 Tomcat、Spring Boot）。

### 1.3 核心特性

- **高并发**：基于 epoll/kqueue，单机可支撑数万并发连接；**事件驱动**：非阻塞 I/O，少量进程处理大量连接；**静态资源**：高效发送文件（sendfile）；**模块化**：核心 + 各功能模块，可扩展。

### 1.4 工作流程

- **Master** 管理 **Worker** 进程；Worker 监听端口、接受连接、处理请求；请求经 **location** 匹配后执行静态或 **proxy_pass** 转发。

## 第2章：Nginx 安装与基础配置

### 2.1 安装

- **Linux**：`apt install nginx` / `yum install nginx`，或官网下载源码 `./configure && make && make install`；**Windows**：官网提供 zip，仅建议学习用。

### 2.2 配置文件结构

- **main**：worker_processes、user、error_log 等；**events**：worker_connections；**http**：通用 HTTP 配置；**server**：虚拟主机，监听端口与 server_name；**location**：URL 匹配与具体逻辑（root、proxy_pass 等）。

### 2.3 启停与重载

- **nginx** 启动；**nginx -s stop** 立即停止；**nginx -s reload** 重载配置（不中断服务）；**nginx -t** 检查配置语法。

### 2.4 日志

- **access_log** 访问日志；**error_log** 错误日志；路径与格式可在 http/server/location 中配置。

------

## 第3章：Nginx 的请求处理机制

### 3.1 Master-Worker

- **Master** 读配置、建信号管道、启动 Worker；**Worker** 竞争 accept、处理请求；进程间不共享连接，无锁竞争。

### 3.2 事件驱动

- 使用 **epoll**（Linux）或 **kqueue**（BSD），单 Worker 内事件循环处理多连接；**异步非阻塞**：读写未就绪时不阻塞，处理其他连接。

### 3.3 连接数

- **worker_connections** 单 Worker 最大连接数；理论并发 ≈ worker_processes × worker_connections（受限于文件描述符与系统参数）。

------

## 第4章：反向代理与负载均衡

### 4.1 正向 vs 反向代理

- **正向**：代理客户端访问外网，客户端配置代理；**反向**：代理后端服务，客户端只访问 Nginx，由 Nginx 转发到 upstream。

### 4.2 proxy_pass

- **location /api/ { proxy_pass http://backend/; }** 将 /api/ 前缀转发到 upstream 的 /；**proxy_pass** 末尾带不带 URI 决定是否替换路径；常用 **proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr;**。

### 4.3 负载均衡

- **upstream** 定义后端组；**默认轮询**；**weight** 加权；**ip_hash** 同一客户端固定后端；**least_conn** 最少连接。**proxy_connect_timeout / proxy_read_timeout** 等控制超时。

### 4.4 健康检查

- 商业版或第三方模块支持主动健康检查；开源可配合 **max_fails** 与 **fail_timeout** 做被动失败剔除。

------

## 第5章：静态资源与缓存优化

### 5.1 root 与 alias

- **root**：路径 = root + URI；**alias**：路径 = alias 替换 location 匹配部分；alias 常用于把 URL 映射到不同目录。

### 5.2 缓存与压缩

- **expires 30d** 或 **add_header Cache-Control "max-age=3600"**；**gzip on; gzip_types text/css application/json;** 开启压缩。

### 5.3 防盗链

- **valid_referers** 限制 Referer；非法引用可 **return 403** 或重定向。

------


## 第6章：Nginx 的重定向与重写

### 6.1 rewrite 与 return

- **rewrite regex replacement [flag]**：按正则改写 URI，flag 有 last、break、redirect（302）、permanent（301）；**return code [url]**：直接返回状态码或跳转，不继续执行后续 rewrite。

### 6.2 常见场景

- **HTTP → HTTPS**：server 80 里 **return 301 https://$host$request_uri;**；**非 www → www**：if ($host !~* ^www\.) { rewrite ^(.*)$ https://www.$host$1 permanent; }；**伪静态**：rewrite 将 /article/123 转为内部路径或 proxy_pass 参数。

### 6.3 调试

- **rewrite_log on;** 配合 error_log debug 可看 rewrite 过程。

------

## 第7章：HTTPS 与安全配置

### 7.1 SSL 配置

- **listen 443 ssl; ssl_certificate /path/to/cert.pem; ssl_certificate_key /path/to/key.pem;**；HTTP 自动跳 HTTPS 见第 6 章。

### 7.2 安全强化

- **add_header Strict-Transport-Security "max-age=31536000";**（HSTS）；**ssl_protocols TLSv1.2 TLSv1.3;** 禁用旧协议；**ssl_session_cache** 提升性能。

### 7.3 证书

- 生产用 **Let's Encrypt**（certbot）或购买证书；自签仅测试用。

------

## 第8章：动静分离与反向代理后端联动

### 8.1 动静分离

- **location ~* \.(js|css|png|jpg)$** 用 root/alias 走本地静态；**location /api/** 用 proxy_pass 转 Spring Boot；静态走 Nginx，动态走后端，减轻后端压力。

### 8.2 location 优先级

- **= 精确** > **^~ 前缀** > **~ 正则** > **无修饰前缀**；同类型按出现顺序。

### 8.3 大文件与上传

- **client_max_body_size** 放宽上传大小；**proxy_request_buffering off** 可减少内存缓冲（按需）；**proxy_read_timeout** 调大长耗时接口。

------

## 第9章：性能调优与高并发优化

### 9.1 进程与连接

- **worker_processes auto;** 或等于 CPU 核数；**worker_connections** 单 worker 最大连接；**worker_rlimit_nofile** 可调大以支持更多连接。

### 9.2 网络与缓冲

- **sendfile on;** 零拷贝发文件；**tcp_nopush on;** 与 **tcp_nodelay** 按场景选用；**keepalive_timeout** 长连接；**proxy_buffer_size**、**proxy_buffers** 影响代理缓冲。

### 9.3 压测

- **ab**、**wrk**、**JMeter** 做并发与 QPS 测试；结合 **error_log** 与系统监控排查瓶颈。

------

## 第10章：Nginx 模块与扩展机制

### 10.1 内置模块

- **proxy**、**rewrite**、**gzip**、**upstream**、**ssl** 等为常用内置模块；编译时或动态加载。

### 10.2 状态监控

- **stub_status** 模块提供简单状态页（需 allow/deny 限制访问）；可被 Prometheus 等采集。

### 10.3 OpenResty / Lua

- **OpenResty** 在 Nginx 上集成 Lua，可写复杂逻辑、限流、动态路由；进阶场景使用。

------


## 第11章：日志分析与监控

### 11.1 日志

- **access_log** 可自定义 log_format（如 $remote_addr、$request、$status、$body_bytes_sent）；**error_log** 级别 debug/info/notice/warn/error，排错时可临时调 debug。

### 11.2 切割与监控

- **logrotate** 按天/大小切割日志；**stub_status** 提供连接数、请求数等；可配合 **Prometheus exporter**、**ELK** 做监控与分析。

------

## 第12章：Nginx 高可用架构设计

### 12.1 主备与 VIP

- **Keepalived** 做 VRRP，同一 VIP 在主备 Nginx 间漂移；主挂掉后备接管，实现高可用。

### 12.2 与网关对比

- Nginx 做**七层反向代理与负载均衡**；**Spring Cloud Gateway**、**Kong** 等偏 API 网关（路由、鉴权、限流）；可组合：Nginx 前置做 TLS 与负载，网关做业务路由。

------

## 第13章：实战案例与部署场景

### 13.1 前后端分离

- 前端静态（Vue 打包）用 **root** 或 **alias** 托管；**/api** 用 **proxy_pass** 转 Spring Boot；**add_header Access-Control-Allow-Origin** 解决跨域（或后端 CORS）。

### 13.2 多服务与 Docker

- 多个 **location** 对应不同 **upstream**；Docker 中 Nginx 用 **proxy_pass http://容器名:端口;** 或服务发现；**client_max_body_size** 调大以支持上传。

------

## 第14章：Nginx 故障排查与常见问题

### 14.1 错误码

- **403**：权限、root 路径错误、禁止列出目录；**404**：文件不存在、proxy_pass 路径错误；**502**：upstream 不可达、后端崩溃；**504**：upstream 超时，调大 **proxy_connect/read/send_timeout**。

### 14.2 其他

- **upstream** 超时与 **proxy_buffer** 过小可导致异常；**rewrite_log** 与 **error_log debug** 助查重写；限流与防 DDoS 可用 **limit_req**、**limit_conn** 或配合防火墙。

------

## 第15章：面试与实战总结

| 问题 | 要点 |
|------|------|
| 反向代理与负载均衡？ | 反向代理隐藏后端，统一入口；负载均衡将请求分到多台后端（轮询/权重/ip_hash/least_conn）。 |
| 高并发原理？ | Master-Worker、事件驱动（epoll）、异步非阻塞 I/O、少进程多连接。 |
| rewrite 与 return？ | rewrite 改写 URI 可多次；return 直接返回或跳转，终止处理。 |
| upstream？ | 定义后端组，配合 proxy_pass；支持负载策略、备份、长连接。 |
| 项目应用？ | 反向代理 Spring Boot、静态托管前端、HTTPS、负载均衡、限流与安全头。 |
