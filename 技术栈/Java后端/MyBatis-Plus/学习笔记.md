# 📘 **第一章：MyBatis-Plus 概述**

------

## 1. **MyBatis 与 MyBatis-Plus 的关系**

### 🔍 **MyBatis 存在的问题**

- **繁琐的 XML 配置**：MyBatis 需要大量的 XML 配置，尤其是在复杂查询和 CRUD 操作中，代码重复率高，维护难度大。
- **手动编写 SQL**：开发人员需要手动编写 SQL 语句，容易出错且不易复用，查询的封装不够灵活。
- **分页功能的缺乏**：MyBatis 原生不支持自动分页功能，需要手动编写分页 SQL 语句。
- **对通用方法支持较弱**：MyBatis 没有提供开箱即用的通用方法，很多功能需要额外的代码支持。

### 🚀 **MyBatis-Plus 的设计目标与诞生背景**

- **简化配置**：MyBatis-Plus 在 MyBatis 的基础上，简化了大量的配置和 CRUD 操作。
- **增强功能**：解决了 MyBatis 在自动化 CRUD、分页、动态查询等方面的不足，提供了一些开箱即用的功能。
- **无侵入设计**：MyBatis-Plus 与 MyBatis 完全兼容，可以轻松集成到现有的 MyBatis 项目中，无需修改原有代码。

------

## 2. **MyBatis-Plus 的核心优势**

### ✨ **无侵入设计**

- **MyBatis-Plus 不改变 MyBatis 的核心功能**，通过增强功能实现对 MyBatis 的扩展。
- 用户可以继续使用 MyBatis 的原有功能，同时享受 MyBatis-Plus 提供的增强功能。

### ⚡ **增强 CRUD 功能**

- 提供了 `BaseMapper`，包含了常用的 CRUD 操作（`insert`、`delete`、`update`、`select`）的方法，免去手动编写 SQL 的麻烦。
- 支持 **批量插入**、**批量更新**，减少了数据库访问次数，提高了性能。

### 🔄 **自动化分页与条件构造**

- **分页功能**：通过 `PaginationInterceptor` 插件，MyBatis-Plus 实现了分页的自动化。
- **条件构造器**：`QueryWrapper`、`UpdateWrapper` 和 `LambdaQueryWrapper` 可以方便地构建动态 SQL，支持链式调用，简化了代码。

### 💻 **支持 Lambda 表达式与链式调用**

- **Lambda 表达式**：避免硬编码字段名的错误，使用 `LambdaQueryWrapper` 时，可以通过字段的引用来生成 SQL。
- **链式调用**：支持链式调用条件查询，使代码更加简洁、可读。

------

## 3. **MyBatis-Plus 的架构设计**

### 🛠 **核心模块**

1. **Wrapper**（包装器，封装器）：
   - 用于构建 SQL 条件，可以通过 `QueryWrapper`、`UpdateWrapper` 等对象来动态拼接查询条件。
   - 支持 Lambda 表达式，避免字段名硬编码。
2. **Mapper**（映射器）：
   - 继承自 `BaseMapper` 的接口提供了常见的 CRUD 方法。
   - `BaseMapper` 中包含了诸如 `selectById`、`insert`、`delete` 等基本操作方法，减少了大量重复代码。
3. **Service**：
   - 在 Service 层通过继承 `ServiceImpl`，可以轻松实现对实体类的常规操作。
   - 通过继承，MyBatis-Plus 自动提供了分页、查询、更新等功能。
4. **Interceptor**：
   - 提供自定义插件功能，支持对 SQL 执行过程进行拦截和修改。
   - 常用的插件如分页插件（`PaginationInterceptor`）、性能分析插件（`PerformanceInterceptor`）等。

### 🔄 **执行流程与调用关系图**

1. **执行流程**
   - 请求进入 Controller 层 -> Service 层 -> Mapper 层（通过 `BaseMapper` 执行数据库操作）-> 数据库查询返回 -> Service 层进行数据处理 -> Controller 层返回响应。
2. **调用关系**
   - Service 调用 `BaseMapper` 的方法，Wrapper 动态构建查询条件，最终生成对应的 SQL 执行。
   - 拦截器对 SQL 进行操作，比如分页或性能分析。

------

## 4. **MyBatis-Plus 在项目中的定位**

### ⚙️ **SpringBoot 集成方式**

- **自动配置**：在 Spring Boot 项目中，MyBatis-Plus 可以通过 `@MapperScan` 注解扫描 Mapper 接口，自动装配，无需额外配置。
- **配置文件**：通常在 `application.yml` 或 `application.properties` 中配置数据库连接和 MyBatis-Plus 的相关设置。

### 🔄 **与 MyBatis 原生配置的兼容性**

- MyBatis-Plus 与原生 MyBatis 完全兼容，可以在同一个项目中同时使用。
- 对于不需要的功能，可以通过 MyBatis-Plus 提供的配置关闭或重写。

## 第2章：快速入门与环境搭建

### 2.1 Spring Boot 集成

- 依赖：**mybatis-spring-boot-starter** + **mybatis-plus-boot-starter**（或仅 Plus，已含 MyBatis）；配置 **spring.datasource**；**@MapperScan** 扫描 Mapper 包；可选 **mybatis-plus.global-config**、**mybatis-plus.configuration**（如逻辑删除字段、驼峰等）。

### 2.2 BaseMapper 与 CRUD

- Mapper 接口继承 **BaseMapper&lt;T&gt;** 即拥有 **insert/selectById/selectBatchIds/updateById/deleteById/selectList** 等；Service 可继承 **ServiceImpl&lt;Mapper, Entity&gt;** 获得 **save/update/remove/getById/list** 等；无需写 XML 即可完成单表 CRUD。

### 2.3 实体注解

- **@TableName("表名")**：表名与类名不一致时；**@TableId(type = IdType.AUTO)**：主键策略（AUTO/ASSIGN_ID 等）；**@TableField**：列名映射、**exist=false** 非表字段、**fill = FieldFill.INSERT** 自动填充；**@TableLogic**：逻辑删除字段；**@Version**：乐观锁版本号。

## 第3章：条件构造器与链式查询

### 3.1 Wrapper 类型

- **QueryWrapper**：查询条件，eq/ne/gt/lt/like/in/orderBy 等；**UpdateWrapper**：更新条件与 set；**LambdaQueryWrapper**：Lambda 指定字段，避免硬编码列名；**LambdaUpdateWrapper**：更新 + Lambda。

### 3.2 常用条件方法

- **eq("col", val)** 等于、**ne** 不等于、**gt/ge/lt/le** 大小比较、**like/likeLeft/likeRight** 模糊、**in("col", list)**、**between**、**isNull/isNotNull**、**orderByAsc/orderByDesc**；链式调用返回 this，可连续拼接。

### 3.3 动态条件

- 使用 **condition** 参数：**wrapper.eq(condition, "col", val)**，condition 为 false 时不拼接；或 **if (xxx) wrapper.eq(...)** 按需拼接，避免传 null 导致异常。

------

## 第4章：分页插件与查询优化

### 4.1 分页配置与使用

- 配置类中注册 **MybatisPlusInterceptor**，添加 **PaginationInnerInterceptor**（可指定 DbType）；查询时构造 **Page&lt;T&gt; page = new Page&lt;&gt;(current, size)**，**mapper.selectPage(page, wrapper)**，结果在 **page.getRecords()**，总数在 **page.getTotal()**。

### 4.2 count 与优化

- 默认会查总数；**page.setSearchCount(false)** 可关闭 count 以提升性能（仅需数据时）；大数据量分页可考虑限制最大页码或游标方式。

------

## 第5章：自动填充与乐观锁机制

### 5.1 自动填充

- 实现 **MetaObjectHandler**，重写 **insertFill/updateFill**，根据 **MetaObject** 判断字段是否有 **@TableField(fill = FieldFill.INSERT/UPDATE)**，统一 set 创建时间、更新时间、操作人等；注册为 Spring Bean。

### 5.2 乐观锁

- 实体字段加 **@Version**；配置类中添加 **OptimisticLockerInnerInterceptor**；更新时 Plus 会将 version 作为 WHERE 条件并 set version=version+1，若影响行数为 0 可判定并发冲突，需重试或提示。

------

## 第6章：多表操作与自定义 SQL

### 6.1 多表与自定义 SQL

- 多表联查无内置 Join，需**自定义 SQL**：在 Mapper 接口中定义方法，对应 **XML 或 @Select 等注解**；XML 中可写复杂 SQL，参数可用 **@Param** 传入 Wrapper（**BaseMapper 的 select 不直接支持多表**）。

### 6.2 与 MyBatis 混用

- 同一 Mapper 既可继承 BaseMapper 用 Plus 方法，也可在 XML 中写自定义 SQL；复杂查询、多表、统计等用 XML，简单 CRUD 用 Plus。

------


## 第7章：代码生成器（Code Generator）

### 7.1 AutoGenerator

- 使用 **AutoGenerator** 配置数据源、包路径、策略（表前缀、命名、作者），可生成 **Entity、Mapper、Mapper XML、Service、Controller**；模板可用 Velocity 或 Freemarker；适合新表快速搭建 CRUD 骨架。

### 7.2 自定义与实战

- 通过 **StrategyConfig** 控制表/字段映射、**PackageConfig** 控制包名、**TemplateConfig** 指定模板路径；生成后可按业务修改，避免重复手写样板代码。

------

## 第8章：进阶功能与插件机制

### 8.1 内置插件

- **PaginationInnerInterceptor**：分页；**OptimisticLockerInnerInterceptor**：乐观锁；**BlockAttackInnerInterceptor**：防止全表 update/delete 无 where。通过 **MybatisPlusInterceptor** 的 **addInnerInterceptor** 按需添加。

### 8.2 自定义插件

- 实现 MyBatis 的 **Interceptor**，在 **@Intercepts** 中指定拦截 **Executor/StatementHandler** 等；可做 SQL 日志、慢 SQL 统计、租户过滤等；注意与 Plus 内置插件的顺序。

------

## 第9章：与 Spring Boot 深度整合

### 9.1 配置与事务

- Mapper 与 Service 由 Spring 管理；**@Transactional** 与 MyBatis 事务一致，同一事务内共用 Connection；**mybatis-plus.mapper-locations**、**global-config.db-config**（逻辑删除字段、主键类型等）在 yml 中配置。

### 9.2 统一封装

- 分页结果可封装为统一 **PageResult**（records、total、current、size）；与 **Result** 统一返回、**@RestControllerAdvice** 全局异常配合，接口风格一致。

------

## 第10章：实战篇：项目集成案例

### 10.1 黑马点评 / 苍穹外卖

- **缓存**：Wrapper 查 DB，结果写 Redis，更新/删除时失效缓存；**自动填充**：创建/更新时间、操作人；**分页**：Page + Wrapper 做列表；**乐观锁**：订单状态、库存扣减等高并发更新场景。

### 10.2 迁移与混用

- 老项目可逐步引入 Plus：新 Mapper 继承 BaseMapper，老 XML 保留；复杂 SQL 继续用 MyBatis XML，简单 CRUD 用 Plus，二者兼容。

------

## 第11章：源码与执行机制分析（进阶）

### 11.1 拦截链与 BaseMapper

- Plus 通过 MyBatis 插件在 **Executor** 层拦截，分页插件改写 SQL、统计 count；**BaseMapper** 方法对应内置的 MappedStatement，由 Plus 在启动时注入，调用时走 MyBatis 执行流程。

### 11.2 Wrapper 与 SQL 构建

- Wrapper 将 eq/like 等转为 **AbstractWrapper** 内部表达式，执行时由 **SqlScript** 等拼成 WHERE 片段，与 select/update 组合成完整 SQL。

## 第12章：面试与常见问题

| 问题 | 要点 |
|------|------|
| MyBatis 与 Plus 区别？ | Plus 在 MyBatis 上增强：BaseMapper 通用 CRUD、Wrapper 条件构造、分页/乐观锁/逻辑删除/自动填充等，无侵入可混用。 |
| Wrapper 与 LambdaWrapper？ | Lambda 用方法引用指定列，避免字符串硬编码，重构友好；功能等价。 |
| 乐观锁原理？ | @Version 字段，更新时 WHERE version=旧值并 SET version=version+1，影响行数 0 表示冲突。 |
| 自动填充？ | MetaObjectHandler 在 insert/update 时根据 @TableField(fill) 自动 set 值。 |
| 常见坑？ | Wrapper 条件注意 null 与空字符串；分页大表可关 count；逻辑删除字段与查询条件要一致。 |
