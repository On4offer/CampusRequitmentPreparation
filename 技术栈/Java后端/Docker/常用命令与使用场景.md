# 📌 Docker 常用命令与使用场景速查

> 配合《学习笔记》使用，侧重**日常开发与运维中的命令、Dockerfile 与 Compose**，便于速查与上手。

---

## 一、镜像命令

| 命令 | 说明 | 示例 |
|------|------|------|
| **docker pull** | 拉取镜像 | docker pull redis:7 |
| **docker images** | 列出本地镜像 | docker images |
| **docker rmi** | 删除镜像 | docker rmi 镜像ID |
| **docker build** | 构建镜像 | docker build -t myapp:1.0 . |
| **docker tag** | 打标签 | docker tag myapp:1.0 registry.cn-hangzhou.aliyuncs.com/ns/myapp:1.0 |
| **docker push** | 推送到仓库 | docker push registry.cn-hangzhou.aliyuncs.com/ns/myapp:1.0 |

## 二、容器命令

| 命令 | 说明 | 示例 |
|------|------|------|
| **docker run** | 创建并启动 | docker run -d -p 8080:8080 --name app myapp:1.0 |
| **docker start/stop/restart** | 启停 | docker start app |
| **docker ps** | 运行中容器 | docker ps -a 含已停止 |
| **docker logs** | 查看日志 | docker logs -f app |
| **docker exec** | 进入容器 | docker exec -it app /bin/sh |
| **docker rm** | 删除容器 | docker rm -f app |
| **docker inspect** | 查看详情 | docker inspect app |

### 常用 run 参数

| 参数 | 说明 | 示例 |
|------|------|------|
| **-d** | 后台运行 | -d |
| **-p** | 端口映射 | -p 8080:8080 |
| **--name** | 容器名 | --name my-redis |
| **-v** | 挂载卷 | -v /data:/var/lib/mysql |
| **-e** | 环境变量 | -e MYSQL_ROOT_PASSWORD=123 |
| **--restart** | 重启策略 | --restart unless-stopped |
| **--memory** | 内存限制 | --memory=512m |

## 五、Dockerfile 常用指令

| 指令 | 说明 | 示例 |
|------|------|------|
| **FROM** | 基础镜像 | FROM eclipse-temurin:17-jre |
| **WORKDIR** | 工作目录 | WORKDIR /app |
| **COPY** | 复制文件 | COPY target/*.jar app.jar |
| **RUN** | 执行命令 | RUN apt-get update && apt-get install -y curl |
| **ENV** | 环境变量 | ENV JAVA_OPTS="-Xmx512m" |
| **EXPOSE** | 声明端口 | EXPOSE 8080 |
| **ENTRYPOINT** | 启动命令 | ENTRYPOINT ["java","-jar","/app.jar"] |
| **CMD** | 默认参数（可被覆盖） | CMD ["--server.port=8080"] |

### Spring Boot 多阶段示例（精简）

```dockerfile
FROM eclipse-temurin:17-jre
WORKDIR /app
COPY target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java","-jar","/app.jar"]
```

## 六、Docker Compose 常用

| 命令 | 说明 |
|------|------|
| **docker compose up -d** | 后台启动所有服务 |
| **docker compose down** | 停止并删除容器 |
| **docker compose ps** | 查看服务状态 |
| **docker compose logs -f** | 查看日志 |
| **docker compose build** | 构建服务镜像 |
| **docker compose exec 服务名 sh** | 进入某服务容器 |

### compose 文件要点

```yaml
services:
  app:
    build: .
    ports:
      - "8080:8080"
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - REDIS_HOST=redis
    depends_on:
      - redis
    restart: unless-stopped
  redis:
    image: redis:7
    volumes:
      - redis_data:/data
    restart: unless-stopped

volumes:
  redis_data:
```

## 七、使用场景对照

| 场景 | 推荐做法 |
|------|----------|
| 本地跑 Redis/MySQL | docker run -d -p 6379:6379 -v redis_data:/data --name redis redis:7 |
| 跑 Spring Boot jar | Dockerfile 多阶段或单阶段 COPY jar + ENTRYPOINT java -jar |
| 多服务联调 | docker-compose.yml 定义 app、redis、mysql，up -d 一键启动 |
| 持久化数据 | -v 具名卷或绑定挂载到容器数据目录 |
| 容器间互通 | 同一 compose 或同一自定义 network，用服务名/容器名访问 |
| 生产部署 | 镜像推私有仓库，宿主机 pull 后 run 或 compose，配置 restart 与资源限制 |

## 八、常见问题排查

| 现象 | 可能原因 | 处理思路 |
|------|----------|----------|
| 容器起不来 | 端口占用、镜像不存在、启动命令报错 | docker logs 容器名；docker run 不加 -d 看前台报错 |
| 连不上容器内服务 | 未 -p 映射、防火墙、监听地址 | 确认 -p 宿主机:容器，容器内监听 0.0.0.0 |
| 数据丢了 | 未挂载卷 | 重要数据用 -v 挂载，重启/删容器不丢 |
| 构建慢/镜像大 | 层多、上下文大、基础镜像大 | 多阶段构建、.dockerignore、alpine 基础镜像 |
| 空间占满 | 未用镜像与容器堆积 | docker system prune -a（慎用，会删未用镜像） |
