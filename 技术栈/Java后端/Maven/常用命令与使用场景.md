# 📌 Maven 常用命令与使用场景速查

> 配合《学习笔记》使用，侧重日常开发中的**命令、配置与场景**，便于速查与上手。

---

## 一、常用命令速查

### 1.1 生命周期与构建

| 命令 | 说明 | 使用场景 |
|------|------|----------|
| `mvn clean` | 删除 `target/` 目录 | 清理旧构建产物，解决“改代码不生效” |
| `mvn compile` | 仅编译主代码 | 快速检查编译是否通过 |
| `mvn test` | 编译并执行测试 | 本地验证单测 |
| `mvn package` | 编译 + 测试 + 打包（jar/war） | 本地打可运行包 |
| `mvn clean package` | 清理后重新打包 | **最常用**，发布前或 CI 中 |
| `mvn clean package -DskipTests` | 打包但跳过测试 | 赶进度时慎用，仅当测试无关本次修改 |
| `mvn install` | 打包并安装到本地仓库 | 多模块项目，供其他模块依赖 |
| `mvn deploy` | 发布到远程仓库 | 发布到 Nexus/私服，供他人使用 |

### 1.2 依赖与插件

| 命令 | 说明 | 使用场景 |
|------|------|----------|
| `mvn dependency:tree` | 打印依赖树 | **排查依赖冲突**、看传递依赖 |
| `mvn dependency:resolve` | 解析并下载依赖 | 确认依赖能拉下来 |
| `mvn dependency:purge-local-repository` | 清除本地某依赖并重新下载 | 依赖损坏或版本异常时 |
| `mvn help:effective-pom` | 查看合并后的最终 POM | 看父 POM/属性继承后的真实配置 |
| `mvn versions:display-dependency-updates` | 检查依赖是否有新版本 | 定期升级依赖时 |

### 1.3 运行与调试

| 命令 | 说明 | 使用场景 |
|------|------|----------|
| `mvn spring-boot:run` | 运行 Spring Boot 应用 | 无 IDE 时直接命令行启动 |
| `mvn clean compile exec:java -Dexec.mainClass="com.xxx.Main"` | 运行指定主类 | 非 Spring Boot 普通 Java 项目 |

### 1.4 常用参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `-DskipTests` | 跳过测试执行（仍会编译测试代码） | `mvn package -DskipTests` |
| `-Dmaven.test.skip=true` | 完全跳过测试（不编译也不执行） | 比 `-DskipTests` 更“彻底” |
| `-P profileId` | 激活指定 profile | `mvn package -Pprod` |
| `-U` | 强制更新快照与 Release | `mvn clean install -U` |
| `-o` / `--offline` | 离线模式，只用本地仓库 | 无网或加快构建 |
| `-q` | 安静模式，少输出 | 脚本/CI 中减少日志 |
| `-X` | Debug 模式，详细日志 | 排查构建失败原因 |

## 二、POM 常用标签速查

### 2.1 项目坐标与基本信息

```xml
<groupId>com.company.project</groupId>      <!-- 组织/公司 -->
<artifactId>my-service</artifactId>          <!-- 模块名 -->
<version>1.0.0-SNAPSHOT</version>            <!-- 版本，SNAPSHOT 表示开发中 -->
<packaging>jar</packaging>                   <!-- jar / war / pom -->
<name>My Service</name>                      <!-- 项目名，可选 -->
<description>项目描述</description>         <!-- 可选 -->
```

### 2.2 依赖声明

```xml
<dependencies>
  <dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <version>2.7.x</version>   <!-- 若父 POM 已管理可省略 -->
    <scope>compile</scope>    <!-- 默认 compile，可选 provided/runtime/test -->
    <optional>true</optional> <!-- 可选依赖，不传递 -->
    <exclusions>
      <exclusion>
        <groupId>xxx</groupId>
        <artifactId>yyy</artifactId>
      </exclusion>
    </exclusions>
  </dependency>
</dependencies>
```

### 2.3 依赖作用域（scope）

| scope | 编译 | 测试 | 运行 | 传递 | 典型用途 |
|-------|------|------|------|------|----------|
| **compile** | ✓ | ✓ | ✓ | ✓ | 默认，普通业务依赖 |
| **provided** | ✓ | ✓ | ✗ | ✗ | Servlet API、Lombok（容器/编译期提供） |
| **runtime** | ✗ | ✓ | ✓ | ✓ | JDBC 驱动、日志实现 |
| **test** | ✗ | ✓ | ✗ | ✗ | JUnit、Mockito |

### 2.4 统一版本：dependencyManagement

在父 POM 或 BOM 中统一版本，子模块不写 `<version>`：

```xml
<dependencyManagement>
  <dependencies>
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-dependencies</artifactId>
      <version>2.7.18</version>
      <type>pom</type>
      <scope>import</scope>
    </dependency>
  </dependencies>
</dependencyManagement>
```

### 2.5 属性与资源过滤

```xml
<properties>
  <java.version>17</java.version>
  <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
</properties>

<build>
  <resources>
    <resource>
      <directory>src/main/resources</directory>
      <filtering>true</filtering>  <!-- 允许 ${xxx} 占位符替换 -->
    </resource>
  </resources>
</build>
```

### 2.6 多环境 Profile

```xml
<profiles>
  <profile>
    <id>dev</id>
    <properties>
      <env>dev</env>
    </properties>
    <activation>
      <activeByDefault>true</activeByDefault>
    </activation>
  </profile>
  <profile>
    <id>prod</id>
    <properties>
      <env>prod</env>
    </properties>
  </profile>
</profiles>
```

使用：`mvn package -Pprod`。

## 三、使用场景对照

| 场景 | 推荐做法 | 命令/配置 |
|------|----------|-----------|
| 本地开发打包运行 | 清理后打包，跳过测试（慎用） | `mvn clean package -DskipTests` |
| 多模块项目本地联调 | 先在本模块执行 install | `mvn clean install` |
| 依赖冲突、类找不到 | 看依赖树，排除或统一版本 | `mvn dependency:tree` |
| 看最终生效的 POM | 查看合并后的 POM | `mvn help:effective-pom` |
| 发布到私服 | 配置 `distributionManagement` 后 | `mvn clean deploy` |
| 使用公司/阿里镜像 | 在 `settings.xml` 配置 `<mirror>` | 见下方常见问题 |
| 指定 JDK 版本编译 | `maven-compiler-plugin` 或 `properties` | `<maven.compiler.source>17</maven.compiler.source>` |

## 四、常见问题排查

| 现象 | 可能原因 | 处理思路 |
|------|----------|----------|
| 依赖下载失败、超时 | 网络、中央仓库慢、未配镜像 | 配置阿里云等镜像；检查 `settings.xml` 与公司代理 |
| 本地有 jar 仍报找不到 | 损坏或版本不一致 | `dependency:purge-local-repository` 或删本地仓库对应目录后重新 `install` |
| 多模块中其他模块找不到 | 依赖模块未 install 到本地 | 先对被依赖模块执行 `mvn install` |
| 打包后运行缺依赖 | 依赖 scope 为 provided 或未打进包 | 检查 scope；Spring Boot 用 `spring-boot-maven-plugin` 打可执行 jar |
| 测试失败导致无法打包 | 单测未通过 | 先修测试或临时 `-DskipTests`（仅作权宜之计） |
| 改代码后不生效 | 未重新编译或旧 class | `mvn clean compile` 或 `clean package` |

## 五、settings.xml 常用配置（本地/镜像）

- **镜像**（加速或内网）：在 `~/.m2/settings.xml` 的 `<mirrors>` 中配置阿里云等。
- **本地仓库路径**：`<localRepository>D:/maven-repo</localRepository>`。
- **私服认证**：`<server>` 中配置 id 与用户名密码，POM 中 `distributionManagement` 的 id 与之对应。
