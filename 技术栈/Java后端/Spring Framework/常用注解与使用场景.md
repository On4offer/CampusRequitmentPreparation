# 📌 Spring Framework 常用注解与使用场景速查

> 配合《学习笔记》使用，侧重**日常开发中的注解、API 与场景**，便于速查与上手。

## 一、Bean 定义与装配

### 1.1 组件声明（被扫描为 Bean）

| 注解 | 说明 | 使用场景 |
|------|------|----------|
| **@Component** | 通用组件 | 无法归到下面几类时用 |
| **@Controller** | Web 控制器 | 表现层，常与 @RequestMapping 配合 |
| **@Service** | 业务层 | Service 类 |
| **@Repository** | 数据访问层 | Dao/Mapper 实现，可带异常转换语义 |

以上均可在 `@ComponentScan` 范围内被扫描并注册为 Bean。

### 1.2 配置类与显式 Bean

| 注解 | 说明 | 使用场景 |
|------|------|----------|
| **@Configuration** | 配置类，内部 @Bean 可互相引用且单例 | JavaConfig 替代 XML |
| **@Bean** | 方法返回值注册为 Bean，方法名为默认 bean 名 | 第三方类、复杂构造、多实例 |

### 1.3 依赖注入

| 注解 | 说明 | 使用场景 |
|------|------|----------|
| **@Autowired** | 按类型注入，可配合 @Qualifier 指定名称 | 构造器/字段/Setter；推荐构造器注入 |
| **@Qualifier("beanName")** | 指定要注入的 Bean 名称 | 同类型多实现时 |
| **@Resource(name="beanName")** | JSR-250，默认按名称再按类型 | 需要按名称注入时 |
| **@Value("${key}")** | 注入配置项或 SpEL | 配置驱动、简单占位符 |

### 1.4 作用域与加载方式

| 注解 | 说明 | 使用场景 |
|------|------|----------|
| **@Scope("singleton")** | 单例（默认） | 无状态 Bean |
| **@Scope("prototype")** | 多例，每次获取新建 | 有状态或需隔离时 |
| **@Lazy** | 延迟初始化，首次使用时创建 | 降低启动时间、按需加载 |

## 二、AOP 相关

### 2.1 启用与切面

| 注解 | 说明 | 使用场景 |
|------|------|----------|
| **@EnableAspectJAutoProxy** | 启用基于 AspectJ 的 AOP（Spring Boot 已自动启用） | 纯 Spring 项目需显式加 |
| **@Aspect** | 标记切面类 | 与 @Before/@Around 等配合 |
| **@Order(n)** | 多个切面时执行顺序，数字小先执行 | 日志、权限、事务等顺序控制 |

### 2.2 切入点与通知

| 注解 | 说明 | 使用场景 |
|------|------|----------|
| **@Before("execution(...)")** | 前置通知 | 参数校验、日志入参 |
| **@After("...")** | 后置通知（无论是否异常） | 清理资源 |
| **@AfterReturning("...", returning="ret")** | 正常返回后 | 记录返回值、指标 |
| **@AfterThrowing("...", throwing="ex")** | 异常抛出后 | 统一异常日志、告警 |
| **@Around("...")** | 环绕，可控制是否执行目标、改返回值 | 耗时统计、权限、限流 |

常用切入点表达式：`execution(* com.example.service.*.*(..))` 表示 service 包下任意类的任意方法。

## 三、事务

### 3.1 声明式事务

| 注解 | 说明 | 使用场景 |
|------|------|----------|
| **@Transactional** | 声明式事务，可加在类或方法上，方法覆盖类 | 需要事务边界时 |
| **@EnableTransactionManagement** | 启用事务管理（Spring Boot 已自动启用） | 纯 Spring 项目需显式加 |

### 3.2 @Transactional 常用属性

| 属性 | 说明 | 示例 |
|------|------|------|
| **propagation** | 传播行为 | REQUIRED（默认）、REQUIRES_NEW、NESTED |
| **isolation** | 隔离级别 | READ_COMMITTED、REPEATABLE_READ 等 |
| **rollbackFor** | 回滚异常类型 | rollbackFor = Exception.class |
| **noRollbackFor** | 不回滚异常 | noRollbackFor = SomeBizException.class |
| **timeout** | 超时秒数 | timeout = 5 |
| **readOnly** | 只读事务优化 | readOnly = true 查询方法 |

⚠️ **事务失效常见原因**：同类内部调用（未走代理）、方法非 public、异常被捕获未抛出、未配置事务管理器。

## 四、配置与环境

### 4.1 属性与配置

| 注解 | 说明 | 使用场景 |
|------|------|----------|
| **@PropertySource("classpath:app.properties")** | 加载 properties 到 Environment | 多配置文件 |
| **@ConfigurationProperties(prefix="app")** | 将配置前缀绑定到 Bean 属性 | 结构化配置（如数据源、Redis） |
| **@Profile("dev")** | 仅在指定 profile 激活时注册 Bean | 多环境不同实现（如 DataSource） |

### 4.2 条件装配（多用于自动配置）

| 注解 | 说明 | 使用场景 |
|------|------|----------|
| **@ConditionalOnClass(X.class)** | classpath 存在某类时才生效 | 有某依赖才创建 Bean |
| **@ConditionalOnProperty("key")** | 配置项存在且匹配时才生效 | 通过配置开关启用功能 |
| **@ConditionalOnMissingBean** | 容器中不存在某类型 Bean 时才创建 | 允许用户自定义覆盖默认 Bean |

## 五、事件与异步

### 5.1 事件

| 注解/API | 说明 | 使用场景 |
|----------|------|----------|
| **@EventListener** | 方法监听某事件类型 | 解耦：发布领域事件后异步发短信、写日志 |
| **ApplicationEventPublisher.publishEvent(event)** | 发布事件 | 业务中发布自定义 ApplicationEvent |

### 5.2 异步与定时

| 注解 | 说明 | 使用场景 |
|------|------|----------|
| **@EnableAsync** | 启用 @Async | 需在配置类上启用 |
| **@Async** | 方法异步执行（不同线程） | 非关键路径操作（发通知、写审计） |
| **@EnableScheduling** | 启用 @Scheduled | 需在配置类上启用 |
| **@Scheduled(cron="0 0 2 * * ?")** | 按 cron 定时执行 | 对账、统计、过期数据清理 |

## 六、常用 API 与接口（速查）

| 接口/类 | 说明 | 使用场景 |
|---------|------|----------|
| **ApplicationContext** | 容器入口，getBean、getEnvironment、publishEvent | 需要从代码中取 Bean 或发事件时 |
| **Environment** | 环境与配置，getProperty("key") | 运行时读配置 |
| **MessageSource** | 国际化 getMessage(code, args, locale) | 多语言文案 |
| **ApplicationEventPublisher** | publishEvent | 发布领域事件 |
| **PlatformTransactionManager** | 编程式事务 getTransaction、commit、rollback | 细粒度事务控制 |

## 七、使用场景对照

| 场景 | 推荐做法 |
|------|----------|
| 新建 Service/Controller | 用 @Service / @Controller，保证在 ComponentScan 包下 |
| 同类型多实现（如多种 MessageService） | 用 @Qualifier 或 @Resource(name="xxx") 指定注入哪一个 |
| 第三方类要进容器 | 在 @Configuration 类中写 @Bean 方法 |
| 统一日志/耗时/权限 | 用 @Aspect + @Around 或 @Before，execution 指定包或注解 |
| 方法需要事务 | 在类或方法上加 @Transactional，确保从外部通过代理调用 |
| 多环境不同数据源 | 不同 @Configuration 上 @Profile("dev"/"prod")，内部分别 @Bean DataSource |
| 配置项绑定到对象 | 用 @ConfigurationProperties(prefix="app.redis") 注入到配置 Bean |
| 异步执行非关键逻辑 | @EnableAsync + 方法上 @Async |
| 定时任务 | @EnableScheduling + 方法上 @Scheduled(cron="...") |
| 业务解耦（如下单后发短信） | 定义事件类继承 ApplicationEvent，publishEvent 发布，@EventListener 监听 |
