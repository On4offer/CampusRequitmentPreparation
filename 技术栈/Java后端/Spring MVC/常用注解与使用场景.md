# 📌 Spring MVC 常用注解与使用场景速查

> 配合《学习笔记》使用，侧重**日常开发中的注解、参数与接口设计**，便于速查与上手。

---

## 一、控制器与请求映射

### 1.1 类级注解

| 注解 | 说明 | 使用场景 |
|------|------|----------|
| **@Controller** | 标识为控制器，方法返回值默认作视图名 | 传统 MVC 返回页面 |
| **@RestController** | @Controller + @ResponseBody，类下方法默认写响应体 | **RESTful API**（前后端分离） |
| **@RequestMapping("/api")** | 类上统一路径前缀 | 如 `/api/user`、`/api/order` |

### 1.2 请求映射

| 注解 | 说明 | 使用场景 |
|------|------|----------|
| **@GetMapping("/list")** | GET 请求，等价 @RequestMapping(method=GET) | 查询列表 |
| **@PostMapping("/save")** | POST 请求 | 新增、提交表单 |
| **@PutMapping("/{id}")** | PUT 请求 | 全量更新 |
| **@DeleteMapping("/{id}")** | DELETE 请求 | 删除 |
| **@RequestMapping(path, method, params, headers)** | 通用映射，可限定参数、请求头 | 需要细粒度条件时 |

## 二、参数绑定

### 2.1 常用参数注解

| 注解 | 说明 | 示例 |
|------|------|------|
| **@RequestParam("name")** | 查询参数或表单字段，可 default、required | `?page=1&size=10` |
| **@PathVariable("id")** | URI 模板变量 | `/user/{id}` → id |
| **@RequestBody** | 请求体反序列化为对象（多为 JSON） | POST JSON 体 |
| **@RequestHeader("Authorization")** | 请求头 | Token、语言等 |
| **@CookieValue("token")** | Cookie 值 | 登录态（少用，多用 Header） |

### 2.2 示例

```java
// GET 分页
@GetMapping("/list")
public Result<Page<User>> list(
    @RequestParam(defaultValue = "1") int page,
    @RequestParam(defaultValue = "10") int size) { ... }

// REST 路径
@GetMapping("/{id}")
public Result<User> get(@PathVariable Long id) { ... }

// POST JSON
@PostMapping("/save")
public Result<Long> save(@RequestBody @Valid UserDTO dto) { ... }
```

## 三、返回值与响应

### 3.1 响应体与状态

| 方式 | 说明 | 使用场景 |
|------|------|----------|
| **@ResponseBody** | 方法返回值序列化后写响应体（类上有 @RestController 可省略） | 返回 JSON |
| **ResponseEntity<T>** | 可设置状态码、响应头、body | 需 201/204/自定义头时 |
| **Result<T>** | 自定义统一结构（code、msg、data） | 前后端约定统一格式 |

### 3.2 示例

```java
// 统一返回
return Result.success(user);
return Result.fail(400, "参数错误");

// 精确控制
return ResponseEntity.status(HttpStatus.CREATED).body(id);
return ResponseEntity.noContent().build();
```

## 四、数据校验

| 注解 | 说明 | 使用场景 |
|------|------|----------|
| **@Valid** | 触发 JSR-303/380 校验（方法参数） | DTO 参数校验 |
| **@Validated** | 支持分组、方法级校验 | 类上或方法参数 |
| **@NotNull / @NotBlank** | 非空 | 必填字段 |
| **@Size(min, max)** | 长度 | 字符串、集合 |
| **@Email / @Pattern** | 格式 | 邮箱、正则 |
| **BindingResult** | 方法参数，紧跟 @Valid 参数，接收校验错误 | 局部处理校验失败（也可交给全局异常） |

---

## 五、异常与全局处理

| 注解 | 说明 | 使用场景 |
|------|------|----------|
| **@RestControllerAdvice** | 全局 REST 增强（异常处理、绑定等） | 统一异常转 JSON |
| **@ExceptionHandler(XxxException.class)** | 处理指定异常 | 业务异常、参数校验异常、兜底 Exception |
| **@ControllerAdvice** | 全局 Controller 增强（可不写 body） | 需返回页面时 |

---

## 六、文件上传与下载

| 类型 | 说明 | 使用场景 |
|------|------|----------|
| **MultipartFile** | 方法参数，接收上传文件 | 单文件/多文件上传 |
| **file.getOriginalFilename()** | 原始文件名（注意安全） | 校验扩展名、生成新文件名 |
| **file.transferTo(File)** | 保存到本地 | 存储到磁盘或 OSS 前落盘 |
| **ResponseEntity<Resource>** + **Content-Disposition** | 下载 | 设置 attachment 与 filename |

---

## 七、跨域与 CORS

| 方式 | 说明 | 使用场景 |
|------|------|----------|
| **@CrossOrigin** | 方法或类上，允许跨域 | 单接口或单 Controller |
| **WebMvcConfigurer.addCorsMappings** | 全局 CORS 规则（allowedOrigins、methods、headers） | 统一配置 |

---

## 八、拦截器（HandlerInterceptor）

| 方法 | 说明 | 使用场景 |
|------|------|----------|
| **preHandle** | Handler 执行前，return false 则中断 | 登录校验、权限、放行白名单 |
| **postHandle** | Handler 执行后、视图前 | 少用 |
| **afterCompletion** | 请求结束后（视图已渲染或异常已处理） | 清理 ThreadLocal、统计 |

配置：实现 **WebMvcConfigurer**，在 **addInterceptors** 中 **registry.addInterceptor(...).addPathPatterns("/api/**").excludePathPatterns("/api/login")**。

## 九、使用场景对照

| 场景 | 推荐做法 |
|------|----------|
| 提供 REST 接口 | 类用 @RestController，方法用 @GetMapping 等，返回 Result<T> |
| 接收 GET 查询参数 | @RequestParam，可 defaultValue、required=false |
| 接收 POST JSON | @RequestBody + DTO，加 @Valid 做校验 |
| REST 风格资源 | @PathVariable 绑定 id，PUT/DELETE 对应更新/删除 |
| 统一返回格式 | 定义 Result，成功/失败均返回 Result；全局异常也封装为 Result |
| 参数校验失败统一返回 | @RestControllerAdvice + @ExceptionHandler(MethodArgumentNotValidException.class) |
| 登录校验 | 拦截器 preHandle 中解析 token，失败 return false 或写 401 |
| 跨域 | 开发环境 @CrossOrigin 或 addCorsMappings；生产可用网关/Nginx |
| 文件上传 | MultipartFile 参数，校验类型与大小，重命名存储 |
| 文件下载 | ResponseEntity<Resource>，设置 Content-Disposition: attachment |
