## 1️⃣ 分库分表概述

### （1）什么是分库分表

**分库分表**是指将一个大型数据库拆分成多个小型数据库，或者将一个大型表拆分成多个小型表，以解决单库单表性能瓶颈问题。

- **分库**：将数据分散到多个数据库实例中
- **分表**：将数据分散到同一数据库的多个表中
- **分库分表**：同时进行分库和分表

**核心目的**：
- 提升数据库并发处理能力
- 提高查询性能
- 解决单表数据量过大的问题
- 提高系统可用性

### （2）为什么需要分库分表

| 问题 | 表现 | 分库分表解决方案 |
|------|------|------------------|
| **单表数据量大** | 查询慢、索引效率低 | 水平分表，减少单表数据量 |
| **单库连接数不足** | 连接池耗尽、拒绝连接 | 分库，分散连接压力 |
| **磁盘空间不足** | 单机磁盘无法扩容 | 分库，数据分散到多台机器 |
| **单机性能瓶颈** | CPU、内存、IO 达到上限 | 分库分表，利用多机资源 |
| **可用性问题** | 单点故障影响整个系统 | 分库，故障隔离 |

### （3）分库分表 vs 读写分离

| 对比项 | 分库分表 | 读写分离 |
|--------|----------|----------|
| **目的** | 解决数据量大、并发高的问题 | 解决读多写少的问题 |
| **数据量** | 数据分散到多库多表 | 数据仍在主库 |
| **写入能力** | 多库并行写入 | 只能主库写入 |
| **查询能力** | 分片查询、聚合查询复杂 | 从库分担读压力 |
| **复杂度** | 高（需要路由、分片键） | 中（主从复制） |
| **适用场景** | 海量数据、高并发写入 | 读多写少、数据量不大 |

### （4）分库分表的优缺点

**优点**：
- ✅ 提升并发处理能力
- ✅ 提高查询性能
- ✅ 解决单表数据量限制
- ✅ 提高系统可用性
- ✅ 便于水平扩展

**缺点**：
- ❌ 增加系统复杂度
- ❌ 跨分片查询困难
- ❌ 分布式事务问题
- ❌ 数据迁移成本高
- ❌ 运维难度增加

## 2️⃣ 分库分表策略

### （1）垂直拆分 vs 水平拆分

**垂直拆分**：按照业务功能或表关系拆分

```sql
-- 垂直分库：按业务模块拆分
-- 用户库：users, user_profiles, user_addresses
-- 订单库：orders, order_items, order_payments
-- 商品库：products, categories, product_attributes

-- 垂直分表：按字段拆分
-- users_base: id, username, email, created_at
-- users_extend: id, age, gender, avatar, bio
```

**水平拆分**：按照数据行拆分

```sql
-- 水平分库：按用户 ID 取模
-- db0: user_id % 4 = 0
-- db1: user_id % 4 = 1
-- db2: user_id % 4 = 2
-- db3: user_id % 4 = 3

-- 水平分表：按订单 ID 范围
-- orders_2024_q1: order_date BETWEEN '2024-01-01' AND '2024-03-31'
-- orders_2024_q2: order_date BETWEEN '2024-04-01' AND '2024-06-30'
```

### （2）分片键选择

**分片键**：决定数据如何分布到不同分片的关键字段

**选择原则**：
1. **查询频率高**：分片键应该是常用查询条件
2. **数据分布均匀**：避免数据倾斜
3. **不可变性**：分片键不应频繁变化
4. **业务相关性**：与业务逻辑紧密相关

**常见分片键**：
- 用户 ID（user_id）
- 订单 ID（order_id）
- 商户 ID（merchant_id）
- 时间字段（created_at）
- 地区字段（region_id）

### （3）分片算法

**取模分片**

```java
// 简单取模
int shardIndex = userId % shardCount;

// 一致性哈希（解决扩容问题）
int shardIndex = hash(userId) % shardCount;
```

**范围分片**

```java
// 按时间范围
if (orderDate >= '2024-01-01' && orderDate < '2024-04-01') {
    shard = 'orders_2024_q1';
} else if (orderDate >= '2024-04-01' && orderDate < '2024-07-01') {
    shard = 'orders_2024_q2';
}
```

**地理位置分片**

```java
// 按地区分片
if (regionId >= 1 && regionId <= 10) {
    shard = 'orders_region_north';
} else if (regionId >= 11 && regionId <= 20) {
    shard = 'orders_region_south';
}
```

**复合分片**

```java
// 先按用户 ID 取模，再按时间范围
int dbIndex = userId % 4;
String tableName = "orders_" + getQuarter(orderDate);
```

### （4）分片数量规划

**分片数量计算**：

```java
// 单表数据量限制
long maxRowsPerTable = 5000000; // 500 万行

// 预估数据量
long estimatedRows = 100000000; // 1 亿行

// 计算需要的表数量
int tableCount = (int) Math.ceil((double) estimatedRows / maxRowsPerTable);
// tableCount = 20

// 计算需要的库数量
int dbCount = (int) Math.ceil((double) tableCount / tablesPerDb);
```

**分片数量建议**：
- 单表数据量：500 万 - 1000 万行
- 单库数据量：不超过 100 GB
- 分片数量：2 的幂次方（2、4、8、16、32）

## 3️⃣ 分库分表核心问题

### （1）跨分片查询

**问题**：查询条件不包含分片键，需要查询所有分片

```sql
-- 分片键：user_id
-- 问题查询：查询某个地区所有用户
SELECT * FROM users WHERE region_id = 1;

-- 解决方案1：广播查询（查询所有分片）
SELECT * FROM users_0 WHERE region_id = 1
UNION ALL
SELECT * FROM users_1 WHERE region_id = 1
UNION ALL
SELECT * FROM users_2 WHERE region_id = 1
UNION ALL
SELECT * FROM users_3 WHERE region_id = 1;

-- 解决方案2：冗余字段（region_id 也作为分片键）
-- 解决方案3：使用 ES 等搜索引擎
```

### （2）分布式事务

**问题**：跨库事务无法使用本地事务

```java
// 问题场景：用户下单，需要扣减库存（库存库）和创建订单（订单库）
@Transactional
public void placeOrder(Order order) {
    // 扣减库存（库存库）
    inventoryDao.decreaseStock(order.getProductId(), order.getQuantity());
    
    // 创建订单（订单库）
    orderDao.insert(order);
    
    // 如果第二步失败，第一步无法回滚
}
```

**解决方案**：

**1. 两阶段提交（2PC）**

```java
// 准备阶段
transactionManager.begin();
try {
    inventoryDao.decreaseStock(productId, quantity); // 库存库
    orderDao.insert(order); // 订单库
    transactionManager.commit();
} catch (Exception e) {
    transactionManager.rollback();
}
```

**2. TCC（Try-Confirm-Cancel）**

```java
// Try 阶段：预留资源
public void tryDecreaseStock(Long productId, Integer quantity) {
    inventoryDao.freezeStock(productId, quantity);
}

// Confirm 阶段：确认扣减
public void confirmDecreaseStock(Long productId, Integer quantity) {
    inventoryDao.decreaseStock(productId, quantity);
}

// Cancel 阶段：取消预留
public void cancelDecreaseStock(Long productId, Integer quantity) {
    inventoryDao.unfreezeStock(productId, quantity);
}
```

**3. 本地消息表**

```sql
-- 本地消息表
CREATE TABLE local_message (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    business_id VARCHAR(64) NOT NULL,
    business_type VARCHAR(32) NOT NULL,
    payload JSON,
    status TINYINT DEFAULT 0, -- 0: 待处理, 1: 已处理, 2: 处理失败

    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_business (business_id, business_type),
    INDEX idx_status (status, retry_count)
);
```

**4. Saga 模式**


### （3）主键生成策略

**问题**：分库分表后，主键不能自增

**解决方案**：

**1. UUID**

```java
// 优点：全局唯一，无需协调
// 缺点：无序，索引效率低，占用空间大
String id = UUID.randomUUID().toString();
```

**2. 雪花算法（Snowflake）**

```java
// 优点：有序、高性能、分布式友好
// 缺点：依赖机器时钟
public class SnowflakeIdGenerator {
    private final long workerId;
    private final long datacenterId;
    private long sequence = 0;
    private long lastTimestamp = -1L;

    public synchronized long nextId() {
        long timestamp = System.currentTimeMillis();
        
        if (timestamp < lastTimestamp) {
            throw new RuntimeException("Clock moved backwards");
        }
        
        if (timestamp == lastTimestamp) {
            sequence = (sequence + 1) & 0xFFF;
            if (sequence == 0) {
                timestamp = tilNextMillis(lastTimestamp);
            }
        } else {
            sequence = 0;
        }
        
        lastTimestamp = timestamp;
        
        return ((timestamp - 1288834974657L) << 22)
            | (datacenterId << 17)
            | (workerId << 12)
            | sequence;
    }
}
```

**3. 数据库号段模式**

```sql
-- 号段表
CREATE TABLE id_segment (

-- 获取号段
UPDATE id_segment 
SET max_id = max_id + step, version = version + 1 
WHERE biz_tag = 'order' AND version = ?;
```

## 4️⃣ 分库分表中间件

### （1）中间件对比

| 中间件 | 类型 | 优点 | 缺点 | 适用场景 |
|--------|------|------|------|----------|
| **ShardingSphere** | 客户端 | 功能强大、生态好 | 性能损耗 | Java 应用、复杂场景 |
| **MyCAT** | 代理 | 对应用透明 | 性能瓶颈 | 多语言应用 |
| **Vitess** | 代理 | 云原生、高可用 | 复杂度高 | 大规模部署 |
| **Cobar** | 代理 | 轻量级 | 功能简单 | 简单场景 |

### （2）ShardingSphere 介绍

**ShardingSphere** 是 Apache 顶级项目，提供分布式数据库中间件解决方案。

**核心功能**：
- 数据分片
- 读写分离
- 分布式事务
- 影子库（压测）
- 数据加密

**架构**：
- **Sharding-JDBC**：客户端模式，轻量级
- **Sharding-Proxy**：代理模式，对应用透明

### （3）MyCAT 介绍

**MyCAT** 是开源的分布式数据库中间件。

**核心功能**：
- 数据分片
- 读写分离
- 多租户
- SQL 解析与路由

**架构**：
- 代理模式，应用无需修改代码

```yaml
spring:
  shardingsphere:
    datasource:
      names: ds0,ds1
      ds0:
        type: com.zaxxer.hikari.HikariDataSource
        driver-class-name: com.mysql.cj.jdbc.Driver
        jdbc-url: jdbc:mysql://localhost:3306/db_0?useSSL=false&serverTimezone=Asia/Shanghai
        username: root
        password: root
      ds1:
        type: com.zaxxer.hikari.HikariDataSource
        driver-class-name: com.mysql.cj.jdbc.Driver
        jdbc-url: jdbc:mysql://localhost:3306/db_1?useSSL=false&serverTimezone=Asia/Shanghai
        username: root
        password: root
    
    rules:
      sharding:
        tables:
          t_order:
            actual-data-nodes: ds$->{0..1}.t_order_$->{0..1}
            database-strategy:
              standard:
                sharding-column: user_id
                sharding-algorithm-name: db_inline
            table-strategy:
              standard:
                sharding-column: order_id
                sharding-algorithm-name: table_inline
            key-generate-strategy:
              column: order_id
              key-generator-name: snowflake
        
        sharding-algorithms:
          db_inline:
            type: INLINE
            props:
              algorithm-expression: ds$->{user_id % 2}
          table_inline:
            type: INLINE
            props:
              algorithm-expression: t_order_$->{order_id % 2}
        
        key-generators:
          snowflake:
            type: SNOWFLAKE
            props:
              worker-id: 123
    
    props:
      sql-show: true
```

```java
}
```

**Service 层**：

```java
@Service
public class OrderService {
    
    @Autowired
    private OrderMapper orderMapper;
    
    public void createOrder(Order order) {
        order.setOrderStatus(0);
        order.setCreatedAt(LocalDateTime.now());
        orderMapper.insert(order);
    }
    
    public Order getOrderById(Long orderId) {
        return orderMapper.selectById(orderId);
    }
    
    public List<Order> getOrdersByUserId(Long userId) {
        LambdaQueryWrapper<Order> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(Order::getUserId, userId);
        return orderMapper.selectList(wrapper);
    }
}
```

```java
public class ConsistentHash {
    private final TreeMap<Long, Node> ring = new TreeMap<>();
    private final int virtualNodes = 150;

    public void addNode(Node node) {
        for (int i = 0; i < virtualNodes; i++) {
            String virtualNodeName = node.getName() + ":" + i;
            long hash = Hashing.murmur3_128().hashString(virtualNodeName, Charset.defaultCharset()).asLong();
            ring.put(hash, node);
        }
    }

    public Node getNode(String key) {
        if (ring.isEmpty()) {
            return null;
        }
        long hash = Hashing.murmur3_128().hashString(key, Charset.defaultCharset()).asLong();
        Map.Entry<Long, Node> entry = ring.ceilingEntry(hash);
        if (entry == null) {
            entry = ring.firstEntry();
        }
        return entry.getValue();
    }
}

// 扩容时只需添加新节点
```

```java
@Service
public class DynamicShardingService {
    
    @Autowired
    private OldShardMapper oldShardMapper;
    
    @Autowired
    private NewShardMapper newShardMapper;
    
    @Autowired
    private MigrationFlagService migrationFlagService;
    
    // 阶段1：双写
    @Transactional
    public void writeData(Data data) {
        oldShardMapper.insert(data);
        if (migrationFlagService.isWriteNewShard()) {
            newShardMapper.insert(data);
        }
    }
    
    // 阶段2：异步迁移
    @Scheduled(fixedDelay = 60000)
    public void migrateData() {
        List<Data> dataList = oldShardMapper.selectUnmigratedData(1000);
        for (Data data : dataList) {
            try {
                newShardMapper.insert(data);
                oldShardMapper.markAsMigrated(data.getId());
            } catch (Exception e) {
                log.error("Migration failed: {}", data.getId(), e);
            }
        }
    }
    
    // 阶段3：切换读
    public Data readData(Long id) {
        if (migrationFlagService.isReadNewShard()) {
            return newShardMapper.selectById(id);
        } else {
            return oldShardMapper.selectById(id);
        }
    }
    
    // 阶段4：停写旧分片
    public void stopOldShard() {
        migrationFlagService.setWriteNewShard(true);
        migrationFlagService.setReadNewShard(true);
    }
}
```

```java
public class DataConsistencyChecker {
    
    public void checkConsistency(Long startId, Long endId) {
        List<Data> mainDataList = mainDataMapper.selectRange(startId, endId);
        
        for (Data mainData : mainDataList) {
            Data replicaData = replicaDataMapper.selectById(mainData.getId());
            
            if (replicaData == null) {
                log.error("Data missing in replica: {}", mainData.getId());
                recordInconsistency(mainData, null);
            } else if (!mainData.equals(replicaData)) {
                log.error("Data mismatch: {}", mainData.getId());
                recordInconsistency(mainData, replicaData);
            }
        }

    private void recordInconsistency(Data mainData, Data replicaData) {
        InconsistencyRecord record = new InconsistencyRecord();
        record.setDataId(mainData.getId());
        record.setMainData(JSON.toJSONString(mainData));
        record.setReplicaData(JSON.toJSONString(replicaData));
        record.setCreatedAt(LocalDateTime.now());
        inconsistencyMapper.insert(record);
    }
}
```

```java
            return false;
        }
        
        // 3. 使用一致性哈希确保同一用户路由一致
        int hash = Hashing.murmur3_128().hashLong(userId).asInt();
        double ratio = Math.abs(hash) % 10000 / 10000.0;
        
        return ratio < canaryRatio;
    }
    
    public String route(Long userId) {
        if (canRouteToNewShard(userId)) {
            return "new_shard";
        } else {
            return "old_shard";
        }
    }
}
```

### （2）流量切换

```java
@Service
public class TrafficSwitchService {
    
    @Autowired
    private MetricsService metricsService;
    
    @Autowired
    private AlertService alertService;
    
    @Scheduled(fixedDelay = 60000)
    public void monitorAndSwitch() {
        TrafficMetrics oldMetrics = metricsService.getMetrics("old_shard");
        TrafficMetrics newMetrics = metricsService.getMetrics("new_shard");
        
        // 检查新分片是否稳定
        if (isNewShardStable(oldMetrics, newMetrics)) {
            increaseNewShardTraffic();
        } else {
            alertService.alert("New shard unstable, rollback traffic");
            rollbackTraffic();
        }
    }
    
    private boolean isNewShardStable(TrafficMetrics oldMetrics, TrafficMetrics newMetrics) {
        return newMetrics.getErrorRate() < oldMetrics.getErrorRate() * 1.1 &&
               newMetrics.getLatency() < oldMetrics.getLatency() * 1.2 &&
               newMetrics.getQps() >= oldMetrics.getQps() * 0.8;
    }
    
    private void increaseNewShardTraffic() {
        double currentRatio = routingConfigService.getCanaryRatio();
        double newRatio = Math.min(currentRatio + 0.1, 1.0);
        routingConfigService.setCanaryRatio(newRatio);
        log.info("Increased new shard traffic to: {}", newRatio);
    }
    
    private void rollbackTraffic() {
        routingConfigService.setCanaryRatio(0.0);
        log.warn("Rolled back traffic to old shard");
    }
}
```

```java
@Service
public class SagaOrderService {
    
    @Autowired
    private OrderMapper orderMapper;
    
    @Autowired
    private InventoryServiceClient inventoryServiceClient;
    
    @Autowired
    private UserServiceClient userServiceClient;
    
    public void createOrder(Order order) {
        SagaTransaction saga = new SagaTransaction();
        
        try {
            // 步骤1：创建订单
            saga.addStep(
                "createOrder",
                () -> orderMapper.insert(order),
                () -> orderMapper.deleteById(order.getOrderId())
            );
            
            // 步骤2：扣减库存
            saga.addStep(
                "decreaseStock",
                () -> inventoryServiceClient.decreaseStock(
                    order.getProductId(), 
                    order.getQuantity()
                ),
                () -> inventoryServiceClient.increaseStock(
                    order.getProductId(), 
                    order.getQuantity()
                )
            );
            
            // 步骤3：增加积分
            saga.addStep(
                "addPoints",
                () -> userServiceClient.addPoints(
                    order.getUserId(), 
                    order.getOrderAmount().intValue()
                ),
                () -> userServiceClient.deductPoints(
                    order.getUserId(), 
                    order.getOrderAmount().intValue()
                )
            );
            
            // 执行 Saga
            saga.execute();
            
        } catch (Exception e) {
            log.error("Order creation failed", e);
            throw e;
        }
    }
}
```

```java
@Service
public class DistributedCacheService {
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    @Autowired
    private OrderMapper orderMapper;
    
    public Order getOrder(Long orderId) {
        String cacheKey = "order:" + orderId;
        
        // 1. 从缓存获取
        Order order = (Order) redisTemplate.opsForValue().get(cacheKey);
        if (order != null) {
            return order;
        }
        
        // 2. 从数据库获取
        order = orderMapper.selectById(orderId);
        if (order != null) {
            // 3. 写入缓存
            redisTemplate.opsForValue().set(cacheKey, order, 1, TimeUnit.HOURS);
        }
        
        return order;
    }
    
    public void updateOrder(Order order) {
        // 1. 更新数据库
        orderMapper.updateById(order);
        
        // 2. 删除缓存
        String cacheKey = "order:" + order.getOrderId();
        redisTemplate.delete(cacheKey);
    }
}
```

```java
@Service
public class ServerlessShardingService {
    
    @Autowired
    private CloudProviderClient cloudProviderClient;
    
    public void autoScale() {
        ShardingMetrics metrics = getShardingMetrics();
        
        if (metrics.getQps() > metrics.getMaxQps()) {
            // 自动扩容
            scaleUp();
        } else if (metrics.getQps() < metrics.getMinQps()) {
            // 自动缩容
            scaleDown();
        }
    }
    
    private void scaleUp() {
        // 1. 申请新的数据库实例
        DatabaseInstance newInstance = cloudProviderClient.createInstance();
        
        // 2. 配置分片规则
        addShardToRouting(newInstance);
        
        // 3. 迁移数据
        migrateDataToNewShard(newInstance);
    }
    
    private void scaleDown() {
        // 1. 选择空闲分片
        DatabaseInstance idleInstance = findIdleInstance();
        
        // 2. 迁移数据
        migrateDataFromShard(idleInstance);
        
        // 3. 下线实例
        cloudProviderClient.terminateInstance(idleInstance);
    }
}
```
