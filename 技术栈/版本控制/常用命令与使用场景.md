# 📌 Git 常用命令与使用场景速查

> 配合《学习笔记》使用，侧重**日常开发中的命令与场景**，便于速查与校招面试。

---

## 一、初始化与克隆

| 命令 | 说明 |
|------|------|
| **git init** | 当前目录初始化为 Git 仓库 |
| **git clone &lt;url&gt;** | 克隆远程仓库（含完整历史） |
| **git clone &lt;url&gt; &lt;dir&gt;** | 克隆到指定目录 |

---

## 二、状态、暂存与提交

| 命令 | 说明 |
|------|------|
| **git status** | 查看工作区、暂存区状态 |
| **git add &lt;file&gt;** / **git add .** | 将修改加入暂存区 |
| **git restore &lt;file&gt;** | 丢弃工作区修改（未暂存） |
| **git restore --staged &lt;file&gt;** | 取消暂存，保留工作区修改 |
| **git commit -m "message"** | 提交暂存区 |
| **git commit --amend** | 修改最近一次提交（信息或追加文件） |
| **git commit --amend --no-edit** | 追加文件到最近提交，不改信息 |

---

## 三、查看与比较

| 命令 | 说明 |
|------|------|
| **git log** | 提交历史 |
| **git log --oneline** | 单行简略显示 |
| **git log -p** | 带 diff |
| **git show &lt;commit&gt;** | 某次提交的详情与 diff |
| **git diff** | 工作区 vs 暂存区 |
| **git diff --staged** | 暂存区 vs 最近提交 |
| **git diff &lt;branch&gt;** | 当前分支 vs 指定分支 |

---

## 四、分支

| 命令 | 说明 |
|------|------|
| **git branch** | 列出本地分支 |
| **git branch -a** | 列出本地+远程分支 |
| **git branch &lt;name&gt;** | 创建分支 |
| **git checkout &lt;name&gt;** / **git switch &lt;name&gt;** | 切换分支 |
| **git checkout -b &lt;name&gt;** / **git switch -c &lt;name&gt;** | 创建并切换 |
| **git merge &lt;branch&gt;** | 将指定分支合并到当前分支 |
| **git rebase &lt;branch&gt;** | 将当前分支变基到指定分支 |
| **git branch -d &lt;name&gt;** | 删除已合并分支 |
| **git branch -D &lt;name&gt;** | 强制删除分支 |

---

## 五、远程

| 命令 | 说明 |
|------|------|
| **git remote -v** | 查看远程仓库地址 |
| **git remote add origin &lt;url&gt;** | 添加远程 origin |
| **git fetch origin** | 拉取远程更新，不合并 |
| **git pull** | fetch + merge 当前分支上游（常用 git pull origin main） |
| **git push origin &lt;branch&gt;** | 推送本地分支到远程 |
| **git push -u origin &lt;branch&gt;** | 推送并设置上游（之后可直接 git push） |
| **git push origin --delete &lt;branch&gt;** | 删除远程分支 |
| **git pull --rebase** | 拉取时用 rebase 代替 merge，保持历史线性 |

---

## 六、撤销与恢复

| 命令 | 说明 |
|------|------|
| **git restore &lt;file&gt;** | 丢弃工作区修改 |
| **git restore --staged &lt;file&gt;** | 取消暂存 |
| **git reset --soft &lt;commit&gt;** | 回退到某提交，保留暂存区和工作区 |
| **git reset --mixed &lt;commit&gt;** | 回退并清暂存区，保留工作区（默认） |
| **git reset --hard &lt;commit&gt;** | 彻底回退，**未提交修改会丢失** |
| **git revert &lt;commit&gt;** | 新增一次“反向”提交，不改写历史（适合已推送） |

---

## 七、标签与贮藏

| 命令 | 说明 |
|------|------|
| **git tag** | 列出标签 |
| **git tag &lt;name&gt;** | 轻量标签（当前提交） |
| **git tag -a &lt;name&gt; -m "msg"** | 附注标签 |
| **git push origin &lt;tagname&gt;** | 推送标签 |
| **git stash** | 贮藏当前修改 |
| **git stash list** | 列出贮藏 |
| **git stash pop** | 应用并移除最近一次贮藏 |
| **git stash apply** | 应用最近一次贮藏，不移除 |
| **git stash drop** | 删除最近一次贮藏 |

## 九、使用场景对照

| 场景 | 推荐做法 |
|------|----------|
| 改错文件想撤销 | git restore &lt;file&gt; 或 git checkout -- &lt;file&gt; |
| 暂存了想取消 | git restore --staged &lt;file&gt; |
| 漏提交想并入上一笔 | git add . && git commit --amend --no-edit |
| 切分支前有未完成修改 | git stash → 切分支/拉取 → git stash pop |
| 远程更新了 push 被拒 | git pull --rebase 或 git pull 后再 git push |
| 已 push 想“撤销”某提交 | git revert &lt;commit&gt; 再 push（推荐） |
| 发版打点 | git tag -a v1.0.0 -m "release 1.0" && git push origin v1.0.0 |
| 参与开源 | fork → clone → 改 → commit → push → 在平台开 PR |

## 十、面试要点

- **工作区、暂存区、仓库**：add 进暂存区，commit 进仓库；三者关系与撤销方式。
- **merge 与 rebase**：merge 保留分支历史、产生合并提交；rebase 历史线性、不要对已推送的公共分支做。
- **冲突解决**：编辑冲突文件 → git add → git commit（或继续 rebase）。
- **fetch 与 pull**：fetch 只拉取不合并；pull = fetch + merge。
- **reset 与 revert**：reset 移动 HEAD（可改写历史）；revert 新增反向提交（不改写历史，适合已推送）。
