# 📘 《Java 8+ 新特性》学习笔记

系统梳理 Lambda、函数式接口、Stream API、Optional、新日期时间 API、接口默认/静态方法及后续版本重要特性，便于系统学习与校招面试速查。

> **版本说明**：以 **JDK 8** 为核心（Lambda、Stream、Optional、java.time、接口 default/static）；第六章涉及 Java 9+ 会注明。

------

# 🧩 第一章：Lambda 表达式与函数式接口

> **目标：** 掌握 Lambda 语法、函数式接口（@FunctionalInterface）及常用内置接口（Supplier、Consumer、Function、Predicate），理解方法引用与 Lambda 的关系。

> **本章在整体中解决什么问题**：建立 Java 8 函数式编程的基础——Lambda 与函数式接口，为**第二章 Stream API**（大量使用 Lambda）和**第三章 Optional**（orElseGet 等接受 Supplier）打基础。与《Java语法与面向对象》中接口、抽象类可对照；与《泛型反射与注解》中函数式接口一节互补。

------

## 🧠 1. Lambda 表达式

### 📘 语法

- **形式**：`(参数列表) -> 表达式` 或 `(参数列表) -> { 语句块 }`。
- 参数类型可省略（由上下文推断）；单参数可省略括号；单行表达式可省略 return 与分号。

| 写法           | 示例 |
| -------------- | ---- |
| 无参           | `() -> 1` |
| 单参（可省括号） | `x -> x + 1` |
| 多参           | `(a, b) -> a + b` |
| 带块           | `(a, b) -> { int c = a + b; return c; }` |

------

### 📘 本质与使用场景

- Lambda 是**函数式接口的实例**，可替代匿名内部类，使代码更简洁。
- 常见场景：集合 forEach、线程 Runnable、Comparator、Stream 操作、回调等。

#### 🧑‍💻 示例代码

```java
// 替代匿名 Runnable
Runnable r = () -> System.out.println("Hello");
new Thread(r).start();

// 替代 Comparator
List<String> list = Arrays.asList("b", "a", "c");
list.sort((a, b) -> a.compareTo(b));

// 配合 Consumer
list.forEach(s -> System.out.println(s));
```

------

## 🧳 2. 函数式接口

### 📘 定义

- **函数式接口**：有且仅有**一个抽象方法**的接口（可含多个 default/static 方法）。
- 可用 `@FunctionalInterface` 标注，编译器会检查是否只有一个抽象方法。

### 📘 常用内置接口（java.util.function）

| 接口            | 抽象方法        | 含义           | 典型用法     |
| --------------- | --------------- | -------------- | ------------ |
| **Supplier&lt;T&gt;**  | `T get()`       | 无参返回值     | 延迟求值、工厂 |
| **Consumer&lt;T&gt;** | `void accept(T)`| 消费一个参数   | forEach      |
| **Function&lt;T,R&gt;** | `R apply(T)`    | 转换 T→R       | map          |
| **Predicate&lt;T&gt;** | `boolean test(T)` | 判断           | filter       |
| **BiFunction&lt;T,U,R&gt;** | `R apply(T,U)` | 二元转换       | reduce 等    |

#### 🧑‍💻 示例代码

```java
Predicate<Integer> isEven = n -> n % 2 == 0;
Function<String, Integer> len = String::length;
Consumer<String> print = System.out::println;
Supplier<Double> random = Math::random;
```

------

## 🧳 3. 方法引用

### 📘 四种形式

| 形式           | 语法示例           | 说明               |
| -------------- | ------------------ | ------------------ |
| 静态方法引用   | `类名::静态方法`    | `Integer::parseInt` |
| 实例方法引用   | `对象::实例方法`   | `System.out::println` |
| 任意对象实例方法 | `类名::实例方法` | `String::length`   |
| 构造器引用     | `类名::new`        | `ArrayList::new`   |

- 方法引用的**参数与返回值**需与函数式接口的抽象方法一致（或兼容）。

------

## 🧳 4. Lambda 与匿名内部类

- Lambda 只能替代**只有一个抽象方法**的接口对应的匿名内部类。
- Lambda 没有自己的作用域里的 this，this 指向外围类；匿名内部类有自己类的作用域。
- 匿名内部类可多字段、多语句；Lambda 更简洁，适合单逻辑。

------

| 知识点     | 面试关键词                     | 实际应用     |
| ---------- | ------------------------------ | ------------ |
| Lambda     | 参数->体、类型推断、函数式接口实例 | 替代匿名类   |
| 函数式接口 | 单抽象方法、@FunctionalInterface、Supplier/Consumer/Function/Predicate | Stream、回调 |
| 方法引用   | 类名::方法、类名::new          | 简化 Lambda  |

------

**学习要点**：能写出 Lambda 与四种方法引用；能说出 Supplier/Consumer/Function/Predicate 的用途；能解释 Lambda 与匿名内部类的区别。

# 🧩 第二章：Stream API

> **目标：** 掌握 Stream 的创建、中间操作（filter、map、flatMap、sorted 等）、终止操作（forEach、collect、reduce 等）及惰性求值，理解与集合的区别。

> **本章在整体中解决什么问题**：第一章的 Lambda 与函数式接口在本章大量应用；本章讲 **Stream** 的创建、中间/终止操作与惰性求值，是集合处理与校招高频考点。学完本章后**第三章 Optional** 可与 Stream 结合（如 findFirst 返回 Optional）；**第四章**起为相对独立的日期、接口 default 等。

------

## 🧠 1. Stream 的创建

### 📘 常见方式

| 方式                 | 示例 |
| -------------------- | ---- |
| 集合                 | `list.stream()`、`list.parallelStream()` |
| 数组                 | `Arrays.stream(array)` |
| 静态方法             | `Stream.of(1,2,3)`、`Stream.iterate(0, n->n+1)`、`Stream.generate(Math::random)` |
| 数字流               | `IntStream.range(0, 10)` |

------

## 🧳 2. 中间操作（惰性）

- **特点**：返回 Stream，不立刻执行；只有遇到**终止操作**才一起执行。
- **常见**：filter、map、flatMap、distinct、sorted、limit、skip、peek。

| 操作     | 说明                     |
| -------- | ------------------------ |
| filter   | 按条件过滤               |
| map      | 元素一对一转换           |
| flatMap  | 元素映射为流再压平       |
| distinct | 去重（按 equals）         |
| sorted   | 排序（自然或 Comparator）|
| limit/skip | 限流/跳过             |

#### 🧑‍💻 示例代码

```java
List<Integer> list = Arrays.asList(1, 2, 3, 4, 5);
list.stream()
    .filter(n -> n % 2 == 0)
    .map(n -> n * n)
    .forEach(System.out::println);  // 4, 16
```

------

## 🧳 3. 终止操作

- **特点**：触发实际计算，返回非 Stream（或 void）。
- **常见**：forEach、collect、reduce、count、min、max、anyMatch、allMatch、noneMatch、findFirst、findAny。

| 操作      | 说明           |
| --------- | -------------- |
| forEach   | 遍历消费       |
| collect   | 收集为 List/Set/Map 等 |
| reduce    | 归约为一个值   |
| count     | 元素个数       |
| min/max   | 最小/最大（需 Comparator）|

#### 🧑‍💻 示例代码

```java
int sum = list.stream().reduce(0, Integer::sum);
List<String> upper = list.stream().map(String::valueOf).collect(Collectors.toList());
long cnt = list.stream().filter(n -> n > 3).count();
```

------

## 🧳 4. Stream 与集合、惰性求值

| 对比     | 集合（Collection）     | Stream                    |
| -------- | ---------------------- | ------------------------- |
| 存储     | 存储元素               | 不存储，描述计算 pipeline |
| 操作     | 外部迭代（for/iterator）| 内部迭代、链式            |
| 求值     | 立即                    | 惰性，终止操作时一次性求值 |
| 复用     | 可多次遍历             | 一次消费后不可复用        |

> 💡 **惰性求值**：中间操作不立刻执行，遇到终止操作时才从源头开始依次执行整个流水线，可优化（如 filter 后 limit 可能不必处理全部元素）。

------

## 🧳 5. 并行流

- `parallelStream()` 或 `stream().parallel()` 可并行处理；底层用 ForkJoinPool。
- **注意**：数据量小可能更慢；要保证无状态、无共享可变变量，避免线程安全问题。

------

# ✅ 本章小结

| 知识点     | 面试关键词                     | 实际应用     |
| ---------- | ------------------------------ | ------------ |
| 创建       | stream()、of、Arrays.stream    | 从集合/数组建流 |
| 中间操作   | filter、map、flatMap、sorted   | 过滤、转换   |
| 终止操作   | collect、reduce、forEach       | 收集、聚合   |
| 惰性求值   | 中间操作不执行、终止才执行     | 性能、短路   |

------

**学习要点**：能写出 filter+map+collect 的典型链式调用；能解释 Stream 与集合的区别及惰性求值的含义。

------

## 🎯 面试常见追问

| 面试官提问               | 回答思路 |
| ------------------------ | -------- |
| Stream 是流式还是一次性？ | Stream 不存数据，描述计算；终止操作触发一次计算，消费后流关闭不可复用。 |
| 中间操作和终止操作区别？   | 中间操作返回 Stream、惰性；终止操作触发求值、返回非 Stream 或 void。 |

------

### ⚠️ 常见坑与注意点

| 现象 / 易错点 | 原因 | 正确做法 / 怎么记 |
|---------------|------|-------------------|
| 重复消费同一 Stream | Stream 一次终止操作后即关闭，再次用会抛 IllegalStateException | 每次需要遍历时重新 **list.stream()** 获取新流，不要复用已消费过的 Stream。 |
| 在 filter/map 中修改外部可变变量 | 并行流下线程不安全，且违反「无状态」要求 | 中间操作应为**无状态**、不依赖或修改外部变量；结果用 collect 收集，不要用外部 List 在 lambda 里 add。 |
| 用 parallelStream 且数据量很小 | 线程调度与合并有开销，可能比串行更慢 | 数据量大、计算耗时再考虑 parallelStream；先保证正确性再测性能。 |
| 链过长、可读性差 | 多步 filter/map/flatMap 叠在一起难读 | 适当换行、提取 Predicate/Function 为命名变量或方法引用，复杂逻辑可保留 for 循环。 |
| 忘记终止操作 | 只写中间操作不写 forEach/collect 等，流水线不执行 | 中间操作惰性，必须有**终止操作**才会触发计算。 |

------

# 🧩 第三章：Optional

> **目标：** 理解 Optional 的用途（避免 NPE、表达“可能为空”）、创建与取值方式、链式操作及使用注意。

> **本章在整体中解决什么问题**：在 Lambda 与 Stream 之后，本章讲 **Optional**——如何表达「可能为空」、正确取值与链式操作。Stream 的 findFirst/findAny 等返回 Optional；学完本章可避免误用 Optional 导致 NPE 或冗余判空。

------

## 🧠 1. 为什么需要 Optional

- **问题**：频繁的 null 判断与 NPE；语义上“可能没有值”不清晰。
- **Optional**：用类型表达“可能有值可能为空”，强制调用方显式处理空情况，减少 NPE。

------

## 🧳 2. 创建 Optional

| 方法              | 说明                     |
| ------------------ | ------------------------ |
| `Optional.of(x)`   | 非 null 值，x 为 null 抛异常 |
| `Optional.ofNullable(x)` | x 可为 null，为 null 则 empty |
| `Optional.empty()` | 空 Optional              |

> ⚠️ 不要用 `Optional.of(null)`，应使用 `ofNullable`。

------

## 🧳 3. 取值与默认值

| 方法            | 说明                         |
| --------------- | ---------------------------- |
| `get()`         | 有值返回值，无值抛 NoSuchElementException；**不推荐**无判断直接 get |
| `orElse(T other)` | 无值返回 other               |
| `orElseGet(Supplier)` | 无值时通过 Supplier 计算默认值（懒求值） |
| `orElseThrow()` / `orElseThrow(Supplier)` | 无值抛异常   |
| `ifPresent(Consumer)` | 有值则消费，无值什么都不做   |
| `isPresent()`   | 判断是否有值                 |

#### ❌ 错误用法 vs ✅ 正确用法

```java
// 错误：不判空直接 get()，空时抛 NoSuchElementException
Optional<String> opt = Optional.ofNullable(getName());
String name = opt.get();  // 若 getName() 为 null，opt 为空，get() 抛异常

// 错误：用 Optional.of(null)，会直接 NPE
Optional<String> bad = Optional.of(getName());  // getName() 为 null 则 NPE

// 正确：用 orElse / orElseGet 提供默认值，或 ifPresent 消费
Optional<String> opt2 = Optional.ofNullable(getName());
String name2 = opt2.orElse("default");           // 无值用默认值
String name3 = opt2.orElseGet(() -> "default");  // 懒求值，适合默认值计算成本高时
opt2.ifPresent(n -> System.out.println(n));      // 有值才执行，无值不执行
```

#### 🧑‍💻 示例代码

```java
Optional<String> opt = Optional.ofNullable(getName());
String name = opt.orElse("default");
opt.ifPresent(n -> System.out.println(n));
```

------

## 🧳 4. 链式操作

- **map(Function)**：有值则转换，无值仍 empty。
- **flatMap(Function)**：映射结果为 Optional 时压平，避免 Optional&lt;Optional&lt;T&gt;&gt;。
- **filter(Predicate)**：有值且满足条件才保留，否则 empty。

```java
Optional<Integer> len = Optional.ofNullable(str).map(String::length);
Optional<String> upper = Optional.ofNullable(str).filter(s -> s.length() > 0).map(String::toUpperCase);
```

------

## 🧳 5. 使用注意与反例

- **不要**在无 `isPresent()` 等判断时直接 `get()`，易 NPE。
- **不要**用 Optional 做字段（序列化、设计复杂），更适合做**方法返回值**表达可能为空。
- Optional **不能完全避免 NPE**：入参仍可能为 null，需在边界处用 ofNullable 或校验。

------

# ✅ 本章小结

| 知识点     | 面试关键词                     | 实际应用     |
| ---------- | ------------------------------ | ------------ |
| 创建       | of、ofNullable、empty          | 包装可能为 null 的返回值 |
| 取值       | orElse、orElseGet、ifPresent   | 安全取默认值 |
| 链式       | map、flatMap、filter           | 链式处理     |
| 注意       | 不直接 get、作返回值而非字段   | 避免误用     |

------

**学习要点**：能写出 Optional 创建、orElse/ifPresent 及 map 链式；能说明 Optional 不能完全避免 NPE 及适用场景。

------

## 🎯 面试常见追问

| 面试官提问           | 回答思路 |
| -------------------- | -------- |
| Optional 能完全避免 NPE 吗？ | 不能；它表达“可能为空”并强制显式处理，减少 NPE，但入参为 null 或误用 get 仍会出问题。 |
| 什么时候用 Optional？ | 方法返回值可能为 null 时，用 Optional 让调用方必须处理空情况；不推荐做字段或集合元素。 |

------

### ⚠️ 常见坑与注意点

| 现象 / 易错点 | 原因 | 正确做法 / 怎么记 |
|---------------|------|-------------------|
| 无判断直接 get() | 空 Optional 调用 get() 抛 NoSuchElementException | 用 **orElse / orElseGet / orElseThrow** 或先 **isPresent()**；避免「能编译就 get」。 |
| Optional.of(null) | of 要求非 null，传 null 会 NPE | 值可能为 null 时用 **Optional.ofNullable(x)**。 |
| 用 Optional 做字段或方法参数 | 增加无谓包装、序列化与 API 噪音 | Optional 适合做**方法返回值**；字段可用 null 或业务上的空对象；参数一般不必 Optional。 |
| 链式 map 里返回 null | optional.map(x -> null) 得到 Optional.empty()，后续链断裂 | map 中尽量返回非 null；若可能为空用 flatMap 返回 Optional。 |

# 🧩 第四章：新日期时间 API（java.time）

> **目标：** 掌握 LocalDate、LocalTime、LocalDateTime、ZonedDateTime、Period、Duration 及 DateTimeFormatter，了解与旧 API 的对比。

> **本章在整体中解决什么问题**：独立于 Lambda/Stream 的**日期时间**专题——java.time 的不可变、线程安全及与 Date/Calendar 的对比。校招常问「为什么要有新时间 API」「SimpleDateFormat 是否线程安全」；与前面三章无强依赖，可按需先学。

------

## 🧠 1. 为什么要有新时间 API

- **旧 API（Date、Calendar）问题**：可变、线程不安全、月份从 0 开始、API 难用。
- **java.time（JDK 8）**：**不可变**、**线程安全**、语义清晰（Local/Zoned、Period/Duration），ISO 日历系统。

------

## 🧳 2. 常用类

### 📘 本地日期时间（无时区）

| 类              | 说明         | 示例 |
| --------------- | ------------ | ---- |
| **LocalDate**   | 日期（年-月-日） | `LocalDate.now()`、`LocalDate.of(2024, 1, 1)` |
| **LocalTime**   | 时间（时:分:秒） | `LocalTime.now()`、`LocalTime.of(12, 30)` |
| **LocalDateTime** | 日期+时间    | `LocalDateTime.now()`、`LocalDate.now().atStartOfDay()` |

### 📘 时区与周期

| 类              | 说明         |
| --------------- | ------------ |
| **ZonedDateTime** | 带时区的日期时间 |
| **Period**      | 日期间隔（年/月/日） |
| **Duration**    | 时间间隔（时/分/秒/纳秒） |

------

## 🧳 3. 格式化与解析

- **DateTimeFormatter**：替代 SimpleDateFormat，**线程安全**。
- `format(DateTimeFormatter)`、`parse(CharSequence, DateTimeFormatter)`；常用模式：`yyyy-MM-dd HH:mm:ss`。
- 预定义：`DateTimeFormatter.ISO_LOCAL_DATE` 等。

#### 🧑‍💻 示例代码

```java
DateTimeFormatter fmt = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");
String s = LocalDateTime.now().format(fmt);
LocalDateTime t = LocalDateTime.parse("2024-01-01 12:00:00", fmt);
```

------

## 🧳 4. 与旧 API 的转换

- **Date → Instant**：`date.toInstant()`。
- **Instant → LocalDateTime**：`LocalDateTime.ofInstant(instant, ZoneId.systemDefault())`。
- **LocalDateTime → Date**：`Date.from(localDateTime.atZone(ZoneId.systemDefault()).toInstant())`。

------

# ✅ 本章小结

| 知识点     | 面试关键词                     | 实际应用     |
| ---------- | ------------------------------ | ------------ |
| 本地时间   | LocalDate、LocalTime、LocalDateTime | 无时区业务时间 |
| 时区与间隔 | ZonedDateTime、Period、Duration | 跨时区、计算间隔 |
| 格式化     | DateTimeFormatter、线程安全    | 解析与展示   |

------

**学习要点**：能使用 LocalDate/LocalDateTime 与 DateTimeFormatter；能说明新 API 不可变、线程安全及与 Date/Calendar 的对比。

------

## 🎯 面试常见追问

| 面试官提问         | 回答思路 |
| ------------------ | -------- |
| 为什么要有新时间 API？ | Date/Calendar 可变、非线程安全、API 难用；java.time 不可变、线程安全、语义清晰。 |
| SimpleDateFormat 线程安全吗？ | 不安全；新 API 用 DateTimeFormatter，线程安全。 |

# 🧩 第五章：接口默认方法与静态方法（Java 8）

> **目标：** 掌握接口中的 default 方法、static 方法及多接口默认方法冲突时的解决规则。

> **本章在整体中解决什么问题**：补充 **接口的 default/static**，使 Lambda、Stream 所用接口（如 Collection 的 forEach、stream）能扩展而不破坏兼容。与《Java语法与面向对象》第八章「接口」中的默认方法冲突、多继承可对照；校招可能问「为什么接口能有 default」「多接口 default 冲突怎么办」。**若时间紧可先掌握第一～四章，本章与第六章为补充。**

------

## 🧠 1. default 方法

- 接口中可用 **default** 修饰方法，提供**默认实现**；实现类可不重写，直接继承默认行为。
- **目的**：在接口中新增方法时不破坏已有实现类（如集合框架的 List 新增 sort、forEach 等）。

```java
interface A {
    default void foo() { System.out.println("A"); }
}
```

------

## 🧳 2. 静态方法

- 接口中可用 **static** 定义静态方法，通过**接口名.方法名**调用；实现类不继承该静态方法。
- 便于在接口旁提供工具方法（如 Function.identity()）。

------

## 🧳 3. 多接口默认方法冲突

- 若实现类同时实现多个接口，且这些接口有**同签名的 default 方法**，实现类**必须重写**该方法，并选择调用哪一个：`接口名.super.方法名()`。
- 若一个接口是父类提供的实现，一个接口是 default，一般**父类优先**（类优先于接口）。

------

# ✅ 本章小结

| 知识点     | 面试关键词                     | 实际应用     |
| ---------- | ------------------------------ | ------------ |
| default    | 默认实现、向后兼容             | 集合 API 扩展 |
| static     | 接口名调用、不继承              | 工具方法     |
| 冲突       | 实现类必须重写、接口名.super   | 多实现时     |

------

**学习要点**：能写出接口的 default/static 方法；能说明多接口同签名 default 时要在实现类中重写并指定调用谁。

------

## 🎯 面试常见追问

| 面试官提问             | 回答思路 |
| ---------------------- | -------- |
| 为什么接口能有 default？ | 便于在接口中新增方法而不破坏已有实现（如 JDK8 集合接口新增 forEach、stream）。 |
| 多接口 default 冲突怎么办？ | 实现类必须重写该方法，在方法内用 接口名.super.方法名() 指定调用哪一个。 |

# 🧩 第六章：其他重要新特性（选学）

> **目标：** 了解 Java 9+ 的模块系统、var、switch 表达式、文本块、record、sealed 等，便于面试提及。

> **本章在整体中解决什么问题**：在 JDK 8 核心特性（第一～五章）之后，本章为 **选学**——Java 9+ 的模块、var、record、sealed 等，面试能说出用途与版本即可。与前面章节无强依赖；若校招以 JDK 8 为主，可只过一遍要点，需要时再深入。

------

## 🧠 1. 模块系统（Java 9）

- **JPMS（Java Platform Module System）**：以模块（module）为单位封装与依赖；module-info.java 声明模块名、导出包、依赖。
- 目的：强封装、减少依赖冲突、缩小运行时镜像。

------

## 🧳 2. var 局部变量类型推断（Java 10）

- 仅用于**局部变量**，由编译器推断类型；不能用于方法参数、字段等。
- 例：`var list = new ArrayList<String>();` 推断为 `ArrayList<String>`。

------

## 🧳 3. switch 表达式与文本块（Java 12+）

- **switch 表达式**：可返回值，用 `->` 或 `yield`，无需 break。
- **文本块（"""..."""）**：多行字符串字面量，避免转义（Java 15 正式）。

------

## 🧳 4. record、sealed（Java 14+）

- **record**：不可变数据载体，自动生成构造器、getter、equals/hashCode/toString。
- **sealed class/interface**：限定子类/实现类集合，便于模式匹配与穷举。

------

# ✅ 本章小结

| 知识点   | 面试关键词           | 实际应用     |
| -------- | -------------------- | ------------ |
| 模块     | JPMS、module-info    | 封装与依赖   |
| var      | 局部变量类型推断     | 简化书写     |
| record   | 不可变数据载体       | DTO、值对象   |
| sealed   | 限定子类             | 穷举与模式匹配 |

------

**学习要点**：能说出 var 仅用于局部变量、record 用于不可变数据、sealed 用于限定子类；面试能简要提及版本与用途即可。

# 📎 附录：面试题速查

1. **Lambda 本质？** 函数式接口的实例，语法糖。
2. **Stream 和集合区别？** Stream 不存数据、惰性求值、一次消费；集合存数据、可多次遍历。
3. **中间操作和终止操作？** 中间返回 Stream、惰性；终止触发求值、返回非 Stream 或 void。
4. **Optional 能完全避免 NPE 吗？** 不能；表达“可能为空”并强制处理，减少 NPE，误用 get 或入参 null 仍会出问题。
5. **为什么要有新时间 API？** Date/Calendar 可变、非线程安全、难用；java.time 不可变、线程安全、语义清晰。
6. **接口 default 作用？** 接口可带默认实现，向后兼容，如 JDK8 集合接口扩展。
7. **多接口 default 冲突？** 实现类必须重写，用 接口名.super.方法名() 指定调用哪个。
8. **var 能用在字段上吗？** 不能，仅局部变量。
9. **record 适用场景？** 不可变数据载体，如 DTO、配置、值对象。
