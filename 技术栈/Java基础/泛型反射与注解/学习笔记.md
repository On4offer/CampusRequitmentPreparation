# 💠 第一章：泛型（Generics）

------

## 1️⃣ 泛型概述

### 📘 什么是泛型

**泛型（Generics）是 Java 1.5 引入的一种语法机制，用于在类、接口和方法中参数化类型**。
 它允许你在定义类或方法时，不指定具体的数据类型，而是延迟到使用时再指定，从而实现**类型安全**和**代码复用**。

```java
List<String> list = new ArrayList<>();
list.add("Java");
// list.add(123); // 编译错误，类型检查
```

泛型在编译时确定类型，避免了运行时 `ClassCastException`。

------

### 🧠 为什么引入泛型

| 问题               | 泛型带来的改进              |
| ------------------ | --------------------------- |
| 编译时无法检查类型 | 泛型提供 **编译期类型检查** |
| 需要强制类型转换   | 泛型消除 **强制类型转换**   |
| 代码重复           | 泛型提升 **代码复用性**     |

示例对比：

```java
// 泛型前：Object 类型需强制转换
List list = new ArrayList();
list.add("Java");
String s = (String) list.get(0);

// 泛型后：编译期自动检查类型
List<String> list2 = new ArrayList<>();
list2.add("Java");
String s2 = list2.get(0); // 无需强转
```

------

### ⚙️ 泛型的编译时检查机制

- 泛型信息只在**编译期**生效，JVM 运行时会**擦除泛型类型**；
- 编译器会根据泛型定义做类型推断与检查；
- 若类型不匹配，编译时报错。

------

### 💼 泛型在集合框架中的应用

| 类/接口       | 泛型定义     | 示例                   |
| ------------- | ------------ | ---------------------- |
| `List<E>`     | 元素类型泛型 | `List<String>`         |
| `Map<K,V>`    | 键值对泛型   | `Map<String, Integer>` |
| `Iterator<E>` | 遍历泛型元素 | `Iterator<User>`       |

```java
Map<Integer, String> map = new HashMap<>();
map.put(1, "Java");
String value = map.get(1);
```

## 2️⃣ 泛型的基本语法

### 🧩 泛型类（Generic Class）

```java
public class Box<T> {
    private T value;
    public void set(T value) { this.value = value; }
    public T get() { return value; }
}
```

使用：

```java
Box<String> box = new Box<>();
box.set("Hello");
System.out.println(box.get());
```

> 💡 泛型类在实例化时指定类型，编译器根据类型自动推断。

------

### 🧩 泛型方法（Generic Method）

泛型方法在方法级别定义类型参数，使用 `<T>` 声明。

```java
public class Utils {
    public static <T> void printArray(T[] array) {
        for (T t : array) System.out.println(t);
    }
}
```

调用：

```java
Integer[] nums = {1, 2, 3};
Utils.printArray(nums);
```

------

### 🧩 泛型接口（Generic Interface）

```java
public interface Generator<T> {
    T next();
}

public class StringGenerator implements Generator<String> {
    public String next() { return "Hello"; }
}
```

------

### 🧩 类型参数命名约定

| 符号 | 含义            | 示例       |
| ---- | --------------- | ---------- |
| T    | Type（类型）    | `List<T>`  |
| E    | Element（元素） | `List<E>`  |
| K    | Key（键）       | `Map<K,V>` |
| V    | Value（值）     | `Map<K,V>` |
| ?    | 通配符          | `List<?>`  |

## 3️⃣ 通配符与边界

### 💬 无界通配符 `<?>`

表示**任意类型**。

```java
public void printList(List<?> list) {
    for (Object obj : list) System.out.println(obj);
}
```

> 可读但**不可写**（除 `null`）。

------

### 💬 上界通配符 `<? extends T>`

表示**T 或 T 的子类**（用于“读取”数据）。

```java
public void processList(List<? extends Number> list) {
    Number n = list.get(0); // ✅ 可读
    // list.add(1); // ❌ 不可写
}
```

------

### 💬 下界通配符 `<? super T>`

表示**T 或 T 的父类**（用于“写入”数据）。

```java
public void addNumbers(List<? super Integer> list) {
    list.add(10); // ✅ 可以写入 Integer
}
```

------

### 🧠 extends 与 super 的区别

| 通配符          | 含义      | 可读 | 可写 | 典型场景           |
| --------------- | --------- | ---- | ---- | ------------------ |
| `<? extends T>` | T及其子类 | ✅    | ❌    | 生产者（Producer） |
| `<? super T>`   | T及其父类 | ❌    | ✅    | 消费者（Consumer） |

------

### ⚖️ PECS 原则

> **Producer Extends, Consumer Super**
>  当一个对象**生产数据**（读出）时使用 `extends`；
>  当一个对象**消费数据**（写入）时使用 `super`。

------

## 4️⃣ 泛型擦除机制（Type Erasure）

### 📘 什么是类型擦除

Java 的泛型是**伪泛型**，只在**编译期**生效。
 编译器会在编译时将泛型信息擦除，转换为原始类型（通常是 `Object`）。

```java
List<String> list1 = new ArrayList<>();
List<Integer> list2 = new ArrayList<>();
System.out.println(list1.getClass() == list2.getClass()); // true
```

------

### 🧠 擦除前后字节码区别

使用 `javap -c` 可查看反编译结果，泛型类型信息被删除，替换为 `Object`。

------

### ⚠️ 擦除带来的限制

1. 运行时无法判断泛型类型：

   ```java
   if (obj instanceof List<String>) // ❌ 编译错误
   ```

2. 无法创建泛型数组：

   ```java
   List<String>[] arr = new List<String>[10]; // ❌ 不允许
   ```

3. 泛型类的静态成员不能使用类型参数。

------

### 🔍 如何通过反射获取泛型参数类型

```java
Field field = StudentService.class.getDeclaredField("students");
Type type = ((ParameterizedType) field.getGenericType()).getActualTypeArguments()[0];
System.out.println(type.getTypeName());
```

------

### 💡 泛型数组的限制与解决方案

**问题**：泛型数组在擦除后类型不确定。
 **解决方案**：使用 `Array.newInstance()` 或集合替代。

```java
@SuppressWarnings("unchecked")
T[] array = (T[]) Array.newInstance(clazz, size);
```

## 5️⃣ 泛型在实际项目中的应用

### 🧰 泛型 + 集合框架

- `List<T>`、`Map<K,V>`、`Set<E>` 全部基于泛型实现类型安全。

```java
List<User> users = new ArrayList<>();
Map<Long, User> userMap = new HashMap<>();
```

------

### 🧾 泛型 + 工具类（Result / Response）

```java
public class Result<T> {
    private Integer code;
    private String msg;
    private T data;
}
```

使用：

```java
Result<User> result = new Result<>();
result.setData(new User());
```

------

### ⚙️ 泛型 + ORM 框架（MyBatis-Plus 示例）

```java
public interface BaseMapper<T> {
    int insert(T entity);
    T selectById(Serializable id);
}
```

> MyBatis-Plus 的 `BaseMapper<T>` 通过泛型实现通用 CRUD。

------

### 🧩 泛型 + 注解 + 枚举

例如在注解中声明泛型类型限制，实现枚举映射或校验逻辑。

------

## 6️⃣ 泛型常见陷阱与面试高频点

| 典型问题           | 原因                                   | 示例                                       |
| ------------------ | -------------------------------------- | ------------------------------------------ |
| 泛型类型擦除       | 运行时类型丢失                         | `List<String>` 与 `List<Integer>` 类型相同 |
| 泛型数组不允许     | 类型不安全                             | `new List<String>[10]`                     |
| 泛型与多态         | 泛型在编译期检查，多态在运行期绑定     | 不同机制                                   |
| 泛型方法与普通方法 | 泛型方法定义在方法级别，可独立于类泛型 | `<T> void print(T t)`                      |

------

### 💬 示例：泛型类型推断

```java
<T> T getFirst(List<T> list) {
    return list.get(0);
}

var result = getFirst(List.of("A", "B", "C")); // 自动推断T为String
```

------

### 💡 面试延伸问题

1. 泛型擦除原理？
    → 编译期检查，运行期擦除；通过桥接方法保持多态。
2. 泛型数组为何不能直接创建？
    → 因为擦除后无法保证类型安全。
3. 泛型与多态的区别？
    → 泛型在编译期约束类型，多态在运行期分派方法。
