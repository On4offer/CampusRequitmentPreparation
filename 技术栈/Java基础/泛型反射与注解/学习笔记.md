## 🧠 1. 泛型概述

### 📘 什么是泛型

**泛型（Generics）** 是 Java 1.5 引入的**编译期类型参数化**机制：在类、接口或方法上声明类型参数（如 `T`、`E`），在使用时再指定具体类型，从而在**编译期**做类型检查并避免强制转换。

```java
List<String> list = new ArrayList<>();
list.add("Java");
// list.add(123);   // 编译错误
String s = list.get(0);  // 无需强转
```

> 💡 泛型在编译时生效，运行时会被**擦除**，JVM 看不到泛型类型信息。

------

### 🧠 为什么引入泛型

| 问题               | 泛型带来的改进              |
| ------------------ | --------------------------- |
| 编译时无法检查类型 | **编译期类型检查**，减少 ClassCastException |
| 需要强制类型转换   | **消除强转**，取元素即目标类型 |
| 代码重复           | **类型参数化**，一份代码多类型复用 |

#### 🧑‍💻 示例代码（泛型前 vs 后）

```java
// 泛型前：Object，需强转，易出错
List list = new ArrayList();
list.add("Java");
String s = (String) list.get(0);

// 泛型后：编译期检查，无需强转
List<String> list2 = new ArrayList<>();
list2.add("Java");
String s2 = list2.get(0);
```

------

### 💼 泛型在集合框架中的应用

| 类/接口       | 泛型含义     | 示例                   |
| ------------- | ------------ | ---------------------- |
| `List<E>`     | 元素类型     | `List<String>`         |
| `Map<K,V>`    | 键与值类型   | `Map<String, Integer>` |
| `Iterator<E>` | 遍历元素类型 | `Iterator<User>`       |

## 🧳 2. 泛型的基本语法

### 📘 泛型类（Generic Class）

- 在类名后声明类型参数 `<T>`，可在成员变量、方法参数与返回值中使用。

#### 🧑‍💻 示例代码

```java
public class Box<T> {
    private T value;
    public void set(T value) { this.value = value; }
    public T get() { return value; }
}
// 使用：Box<String> box = new Box<>(); box.set("Hello");
```

------

### 📘 泛型方法（Generic Method）

- 在方法修饰符与返回类型之间声明 `<T>`，类型参数仅在该方法内有效，可与类上的泛型不同。

#### 🧑‍💻 示例代码

```java
public static <T> void printArray(T[] array) {
    for (T t : array) System.out.println(t);
}
// 调用：Utils.<Integer>printArray(new Integer[]{1,2,3}); 或根据参数推断
```

------

### 📘 泛型接口（Generic Interface）

- 接口声明类型参数，实现类可指定具体类型或继续保留泛型。

#### 🧑‍💻 示例代码

```java
public interface Generator<T> {
    T next();
}
public class StringGenerator implements Generator<String> {
    @Override
    public String next() { return "Hello"; }
}
```

------

### 📘 类型参数命名约定

| 符号 | 含义     | 常见用法   |
| ---- | -------- | ---------- |
| T    | Type     | 通用类型   |
| E    | Element  | 集合元素   |
| K,V  | Key,Value| Map 键值   |
| ?    | 通配符   | 不确定类型 |

## 🧳 3. 通配符与边界

### 📘 无界通配符 `<?>`

- 表示“任意类型”，常用于只读遍历；**除 null 外不能写入**（编译器不知道实际类型）。

```java
public void printList(List<?> list) {
    for (Object obj : list) System.out.println(obj);
}
```

------

### 📘 上界通配符 `<? extends T>`

- 表示 **T 或其子类**，可**安全读取**为 T，但**不能写入**（不知道具体子类型，避免类型不安全）。

```java
public void process(List<? extends Number> list) {
    Number n = list.get(0);  // ✅ 可读
    // list.add(1);           // ❌ 不可写
}
```

------

### 📘 下界通配符 `<? super T>`

- 表示 **T 或其父类**，可**安全写入** T 及子类，但**读取只能得到 Object**。

```java
public void addNumbers(List<? super Integer> list) {
    list.add(10);  // ✅ 可写
}
```

------

### ⚖️ PECS 原则

> **Producer Extends, Consumer Super**
> - **生产者**（向外提供元素、读取）→ 用 `<? extends T>`
> - **消费者**（接收元素、写入）→ 用 `<? super T>`

| 通配符           | 可读     | 可写     | 典型场景   |
| ---------------- | -------- | -------- | ---------- |
| `<? extends T>` | ✅ 读为 T | ❌       | 生产者     |
| `<? super T>`   | 仅 Object | ✅ 写 T | 消费者     |

## 🧳 4. 类型擦除（Type Erasure）

### 📘 什么是类型擦除

- Java 泛型是**编译期**机制，运行时泛型信息被**擦除**，替换为**原始类型**（如 `Object` 或声明的上界）。
- 因此运行时无法得到“具体泛型类型”（如无法判断 `List<String>` 与 `List<Integer>` 的差别）。

```java
List<String> list1 = new ArrayList<>();
List<Integer> list2 = new ArrayList<>();
System.out.println(list1.getClass() == list2.getClass());  // true
```

### 📘 擦除过程

1. **编译时**：检查泛型类型，生成类型安全的代码
2. **擦除**：将泛型类型参数替换为原始类型
   - 无界类型参数（如 `<T>`）→ 替换为 `Object`
   - 有界类型参数（如 `<T extends Number>`）→ 替换为 `Number`
3. **运行时**：JVM 只看到原始类型，无泛型信息

### 📘 桥接方法（Bridge Method）

- **桥接方法**：编译器为了保持多态性而生成的合成方法
- **作用**：解决类型擦除后方法签名不匹配的问题

```java
// 泛型接口
interface Comparable<T> {
    int compareTo(T o);
}

// 实现类
class Integer implements Comparable<Integer> {
    // 编译器生成的桥接方法
    public int compareTo(Object o) {
        return compareTo((Integer) o);
    }
    
    // 实际实现的方法
    public int compareTo(Integer o) {
        return this.value - o.value;
    }
}
```

### ⚠️ 擦除带来的限制

| 限制           | 说明                         |
| -------------- | ---------------------------- |
| 无法 instanceof 泛型 | 如 `obj instanceof List<String>` 编译错误 |
| 不能直接 new 泛型数组 | 如 `new List<String>[10]` 不允许 |
| 静态成员不能用类型参数 | 泛型属于实例，静态属于类     |
| 不能 new T() 等 | 擦除后 T 为 Object，无法确定构造 |
| 不能捕获泛型异常 | 异常类型不能是泛型，因为异常处理在运行时 |

------

### 🔍 通过反射获取泛型参数类型

- 若字段/方法签名上**保留**了泛型（如 `List<String>`），可通过 `getGenericType()` 得到 `ParameterizedType`，再取 `getActualTypeArguments()`。

#### 🧑‍💻 示例代码

```java
// 从字段获取泛型
Field field = MyClass.class.getDeclaredField("list");
Type type = field.getGenericType();
if (type instanceof ParameterizedType) {
    Type[] args = ((ParameterizedType) type).getActualTypeArguments();
    System.out.println(args[0].getTypeName());  // java.lang.String
}

// 从方法返回值获取泛型
Method method = MyClass.class.getMethod("getList");
Type returnType = method.getGenericReturnType();
if (returnType instanceof ParameterizedType) {
    Type[] args = ((ParameterizedType) returnType).getActualTypeArguments();
    System.out.println(args[0].getTypeName());
}

// 从方法参数获取泛型
Method method = MyClass.class.getMethod("process", List.class);
Type[] paramTypes = method.getGenericParameterTypes();
if (paramTypes[0] instanceof ParameterizedType) {
    Type[] args = ((ParameterizedType) paramTypes[0]).getActualTypeArguments();
    System.out.println(args[0].getTypeName());
}
```

------

### 💡 泛型数组的替代方案

- 使用 `Array.newInstance(Class<?>, int)` 创建与泛型成分类型一致的数组，再强转；或直接用**集合**（如 `ArrayList<T>`）代替数组。

```java
@SuppressWarnings("unchecked")
T[] array = (T[]) Array.newInstance(clazz, size);
```

### 📘 类型推断（Type Inference）

- **钻石操作符（JDK 7+）**：`new ArrayList<>()` 编译器自动推断类型
- **泛型方法类型推断**：根据方法参数自动推断类型参数
- **目标类型推断（JDK 8+）**：根据赋值目标类型推断

```java
// 钻石操作符
List<String> list = new ArrayList<>();

// 方法类型推断
printArray(new String[]{"a", "b", "c"});  // 自动推断 T 为 String

// 目标类型推断
List<String> list = createList();  // 根据左侧类型推断
```

## 🧳 5. 泛型在项目中的常见应用

### 📘 统一响应封装（Result / Response）

```java
public class Result<T> {
    private int code;
    private String msg;
    private T data;
    
    // 成功响应
    public static <T> Result<T> success(T data) {
        Result<T> result = new Result<>();
        result.setCode(200);
        result.setMsg("success");
        result.setData(data);
        return result;
    }
    
    // 失败响应
    public static <T> Result<T> error(int code, String msg) {
        Result<T> result = new Result<>();
        result.setCode(code);
        result.setMsg(msg);
        return result;
    }
    
    // getter/setter
}

// 使用
Result<User> userResult = Result.success(user);
Result<List<Order>> ordersResult = Result.success(orders);
```

### 📘 通用工具类

```java
public class Utils {
    // 泛型方法：打印数组
    public static <T> void printArray(T[] array) {
        for (T t : array) System.out.println(t);
    }
    
    // 泛型方法：获取列表第一个元素
    public static <T> T getFirst(List<T> list) {
        return list.isEmpty() ? null : list.get(0);
    }
    
    // 泛型方法：安全转换
    @SuppressWarnings("unchecked")
    public static <T> T cast(Object obj) {

    }
```
```java
}
```

### 📘 构建器模式（Builder Pattern）

```java
public class UserBuilder<T extends UserBuilder<T>> {
    protected String name;
    protected int age;
    
    @SuppressWarnings("unchecked")
    public T name(String name) {
        this.name = name;
        return (T) this;
    }
    
    @SuppressWarnings("unchecked")
    public T age(int age) {
        this.age = age;
        return (T) this;
    }
    
    public User build() {
        return new User(name, age);
    }
}

// 子类构建器
public class AdminBuilder extends UserBuilder<AdminBuilder> {
    private String role;
    
    public AdminBuilder role(String role) {
        this.role = role;
        return this;
    }
    
    @Override
    public Admin build() {
        return new Admin(name, age, role);
    }
}

// 使用
User user = new UserBuilder<>().name("Alice").age(20).build();
Admin admin = new AdminBuilder().name("Bob").age(30).role("ADMIN").build();
```

### 📘 泛型与设计模式

#### 工厂方法模式

```java
public interface Factory<T> {
    T create();
}

public class CarFactory implements Factory<Car> {
    @Override
    public Car create() {

    }
}

```

#### 装饰器模式

```java
public interface Component<T> {
    T operation();
}

public class ConcreteComponent<T> implements Component<T> {
    private T value;
    
    public ConcreteComponent(T value) {
        this.value = value;
    }
    
    @Override
    public T operation() {
        return value;
    }
}

public class Decorator<T> implements Component<T> {
    private Component<T> component;
    
    public Decorator(Component<T> component) {
        this.component = component;
    }
    
    @Override
    public T operation() {
        // 增强功能
        return component.operation();
    }
}
```

## 🧳 7. 泛型最佳实践

### 📘 使用建议

1. **明确类型参数含义**：使用有意义的类型参数名（如 T、E、K、V）
2. **合理使用通配符**：遵循 PECS 原则
3. **避免裸类型**：尽量使用泛型类型，而非原始类型
4. **注意类型擦除**：避免运行时依赖泛型类型信息
5. **使用泛型方法**：当类型参数仅在方法内使用时
6. **考虑类型推断**：利用钻石操作符和类型推断简化代码

### 📘 性能考虑

- **泛型不影响运行时性能**：类型检查在编译期完成
- **避免过度使用泛型**：简单场景使用具体类型更清晰
- **注意自动装箱/拆箱**：泛型与基本类型结合时的性能开销

### 📘 框架中的泛型应用

- **Spring**：`RestTemplate.getForObject()`、`JdbcTemplate.queryForObject()`
- **MyBatis**：`Mapper<T>` 接口，`SqlSession.selectOne()`
- **Guava**：`ImmutableList<T>`、`Optional<T>`
- **RxJava**：`Observable<T>`、`Flowable<T>`

# 🧩 第二章：反射（Reflection）

> **目标：** 理解反射的概念与用途，掌握 Class 的获取、对象创建、字段/方法/构造器的访问与调用，以及反射在框架中的应用与性能注意点。

> **本章在整体中解决什么问题**：第一章泛型在运行期被擦除，反射可在运行期获取类信息与泛型参数；本章讲**反射 API** 与典型用法，为**第三章注解解析**和**第四章综合应用**打基础。Spring、MyBatis 等框架大量依赖反射。

------

## 🧠 1. 反射的基本概念

### 📘 什么是反射

- **反射**是在**运行时**动态获取类的信息（字段、方法、构造器、注解等），并可以**创建对象、访问/修改成员、调用方法**，无需在编译期确定类型。
- 反射是很多框架（Spring、MyBatis、JUnit、Jackson）实现**动态行为**的基础（如根据配置创建 Bean、根据注解映射字段）。

### 📘 核心类

| 类            | 作用                         |
| ------------- | ---------------------------- |
| **Class**     | 表示类或接口，入口            |
| **Field**     | 表示字段，可读/写             |
| **Method**    | 表示方法，可 invoke 调用     |
| **Constructor** | 表示构造器，可 newInstance 创建对象 |

------

## 🧳 2. 获取 Class 对象

### 📘 三种常见方式

| 方式                     | 示例                         |
| ------------------------ | ---------------------------- |
| 类名.class               | `User.class`                 |
| 对象.getClass()          | `obj.getClass()`             |
| Class.forName("全限定名") | `Class.forName("com.xxx.User")` |

- **forName** 会触发类加载（若未加载）；**class** 字面量不会主动加载其他类。

#### 🧑‍💻 示例代码

```java
Class<?> c1 = String.class;
Class<?> c2 = "hello".getClass();
Class<?> c3 = Class.forName("java.lang.String");
// c1 == c2 == c3 为 true（同一类只加载一次）
```

------

## 🧳 3. 反射的核心操作

### 📘 创建对象

- **无参**：`clazz.getDeclaredConstructor().newInstance()`（JDK 9+ 推荐；旧版 `newInstance()` 已过时）。
- **有参**：先 `getDeclaredConstructor(参数类型...)` 得到 `Constructor`，再 `newInstance(实参...)`。

#### 🧑‍💻 示例代码

```java
Class<?> clazz = User.class;
Constructor<?> cons = clazz.getDeclaredConstructor(String.class, int.class);
Object obj = cons.newInstance("张三", 20);
```

------

### 📘 访问与修改字段

- **getDeclaredField(name)**：获取本类声明字段（含 private）；**getField(name)** 仅 public 且含继承。
- **setAccessible(true)**：突破访问限制，可访问 private。
- **get(obj)** / **set(obj, value)**：读/写实例字段；静态字段时 obj 传 null。

#### 🧑‍💻 示例代码

```java
Field field = clazz.getDeclaredField("name");
field.setAccessible(true);
field.set(obj, "李四");
Object value = field.get(obj);
```

------

### 📘 调用方法

## 🧳 4. 反射的性能优化

### 📘 性能瓶颈

- **反射操作比直接调用慢**：需要运行时解析、类型检查、安全检查
- **主要开销**：方法查找、参数类型匹配、权限检查

### 📘 优化策略

1. **缓存 Class 对象**：避免重复获取
2. **缓存 Field/Method/Constructor**：避免重复查找
3. **使用 setAccessible(true)**：跳过权限检查（仅限可信代码）
4. **使用 MethodHandle**（JDK 7+）：性能接近直接调用
5. **使用反射工具库**：如 Apache Commons Lang、Guava

### 📘 MethodHandle 示例

```java
// 获取 MethodHandle
MethodType methodType = MethodType.methodType(void.class, String.class);
MethodHandles.Lookup lookup = MethodHandles.lookup();
MethodHandle methodHandle = lookup.findVirtual(User.class, "setName", methodType);

// 调用方法
User user = new User();
methodHandle.invoke(user, "Alice");
```

### 📘 缓存示例

```java
public class ReflectionCache {
    private static final Map<Class<?>, Map<String, Field>> fieldCache = new ConcurrentHashMap<>();
    private static final Map<Class<?>, Map<String, Method>> methodCache = new ConcurrentHashMap<>();
    
    public static Field getField(Class<?> clazz, String fieldName) {
        return fieldCache.computeIfAbsent(clazz, k -> new ConcurrentHashMap<>())
            .computeIfAbsent(fieldName, k -> {
                try {
                    Field field = clazz.getDeclaredField(fieldName);
                    field.setAccessible(true);
                    return field;
                } catch (Exception e) {
                    throw new RuntimeException(e);
                }
            });
    }
    
    public static Method getMethod(Class<?> clazz, String methodName, Class<?>... parameterTypes) {
        String key = methodName + Arrays.toString(parameterTypes);
        return methodCache.computeIfAbsent(clazz, k -> new ConcurrentHashMap<>())
            .computeIfAbsent(key, k -> {
                try {
                    Method method = clazz.getDeclaredMethod(methodName, parameterTypes);
                    method.setAccessible(true);
                    return method;
                } catch (Exception e) {
                    throw new RuntimeException(e);
                }
            });
    }
}
```

```java
public class SimpleContainer {
    private final Map<Class<?>, Object> beans = new ConcurrentHashMap<>();
    
    public <T> void registerBean(Class<T> clazz) {
        try {
            T instance = clazz.getDeclaredConstructor().newInstance();
            // 注入依赖（简化版）
            for (Field field : clazz.getDeclaredFields()) {
                if (field.isAnnotationPresent(Autowired.class)) {
                    field.setAccessible(true);
                    Class<?> dependencyClass = field.getType();
                    Object dependency = beans.get(dependencyClass);
                    if (dependency == null) {
                        dependency = dependencyClass.getDeclaredConstructor().newInstance();
                        beans.put(dependencyClass, dependency);
                    }
                    field.set(instance, dependency);
                }
            }
            beans.put(clazz, instance);
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }
    
    public <T> T getBean(Class<T> clazz) {
        return clazz.cast(beans.get(clazz));
    }
}

// 使用
SimpleContainer container = new SimpleContainer();
container.registerBean(UserService.class);
container.registerBean(UserDao.class);
UserService userService = container.getBean(UserService.class);
```

## 🧳 6. 反射的安全考虑

### 📘 访问权限

- **setAccessible(true)**：可以突破 private 访问限制
- **安全管理器**：JVM 可以通过安全策略限制反射操作
- **模块化（JDK 9+）**：对未开放的包，反射访问受限

### 📘 安全建议

1. **仅在可信代码中使用 setAccessible(true)**
2. **避免反射访问敏感字段和方法**
3. **在模块化环境中，确保模块导出了必要的包**
4. **使用反射时进行适当的权限检查**

### 📘 模块化中的反射

- **opens 指令**：允许反射访问包中的类型
- **exports 指令**：允许编译时访问，但不一定允许反射访问

```java
// module-info.java
module com.example.app {
    opens com.example.app.model to java.base;
    exports com.example.app.service;
}
```

## 🧳 7. 反射的实际应用案例

### 📘 通用工厂模式

```java
public class ObjectFactory {
    public static <T> T create(Class<T> clazz, Object... args) {
        try {
            if (args.length == 0) {
                return clazz.getDeclaredConstructor().newInstance();
            }
            Class<?>[] parameterTypes = new Class[args.length];
            for (int i = 0; i < args.length; i++) {
                parameterTypes[i] = args[i].getClass();
            }
            Constructor<T> constructor = clazz.getDeclaredConstructor(parameterTypes);
            return constructor.newInstance(args);
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }
}

// 使用
User user = ObjectFactory.create(User.class, "Alice", 20);
```

### 📘 对象复制

```java
public class ObjectCopier {
    public static <T> T copy(T source, Class<T> targetClass) {
        try {
            T target = targetClass.getDeclaredConstructor().newInstance();
            for (Field field : source.getClass().getDeclaredFields()) {
                field.setAccessible(true);
                Object value = field.get(source);
                try {
                    Field targetField = targetClass.getDeclaredField(field.getName());
                    targetField.setAccessible(true);
                    targetField.set(target, value);
                } catch (NoSuchFieldException e) {
                    // 目标类没有该字段，跳过
                }
            }
            return target;
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }
}
```

## 🎯 反射最佳实践

1. **缓存反射对象**：避免重复查找 Field、Method、Constructor
2. **合理使用 setAccessible**：仅在可信代码中使用，避免破坏封装性

### ⚠️ 常见坑与注意点

| 现象 / 易错点 | 原因 | 正确做法 / 怎么记 |
|---------------|------|-------------------|
| 反射修改 final 字段未先 setAccessible(true) | 默认无法访问 private/final，会抛 IllegalAccessException | 先 **field.setAccessible(true)**（注意模块化下可能被拒绝）；修改 final 后语义上仍是“未初始化完成”的错觉，慎用。 |
| 反射破坏单例（通过反射调私有构造 newInstance） | 私有构造可被 setAccessible 后调用，生成新实例 | 单例构造内可判断“若实例已存在则抛异常”，或用枚举单例防反射。 |
| 反射调用方法未处理 InvocationTargetException | method.invoke 会把被调方法内部异常包装成 InvocationTargetException | 捕获后用 **getCause()** 取出真实异常再处理或重抛。 |
| 频繁反射调用方法、未缓存 Method | 每次 getMethod + invoke 有查找与检查开销，性能差 | 将 **Method/Constructor** 缓存起来（如 Map<方法名, Method>），只反射一次，后续复用。 |
| JDK 9+ 模块化下反射访问未开放包失败 | 未在 module-info 中 opens 的包，反射无法访问 | 运行加 --add-opens，或模块声明 opens 对应包给反射框架。 |

## 🧠 1. 注解的基本概念

### 📘 什么是注解

- **注解**是一种**元数据**，附加在类、方法、字段、参数等之上，本身不直接执行业务逻辑，而是供**编译期或运行期**的工具/框架读取并据此做处理。
- 本质上是继承了 **Annotation** 的**特殊接口**，可包含**属性**（无参方法形式，带默认值）。

### 📘 注解的作用

| 阶段     | 作用举例                           |
| -------- | ---------------------------------- |
| 编译期   | @Override 检查重写、Lombok 生成代码、APT 生成源码 |
| 运行期   | Spring 扫描 @Component、JUnit 执行 @Test、校验 @NotNull |

## 🧳 2. 内置注解与元注解

### 📘 常见内置注解

| 注解               | 作用                         |
| ------------------ | ---------------------------- |
| **@Override**      | 标记重写父类/接口方法，编译期检查 |
| **@Deprecated**   | 标记过时，编译/IDE 告警      |
| **@SuppressWarnings** | 抑制指定类型告警（如 "unchecked"） |
| **@FunctionalInterface** | 标记函数式接口，编译期检查 |
| **@SafeVarargs** | 抑制可变参数的类型安全告警 |
| **@Repeatable** | 允许注解重复使用 |
| **@Native** | 标记原生方法中使用的常量 |

### 📘 元注解

- **元注解**是用于修饰其他注解的注解，定义注解的行为和特性

| 元注解 | 作用 | 可选值 |
|--------|------|--------|
| **@Target** | 指定注解可应用的目标元素 | TYPE, FIELD, METHOD, PARAMETER, CONSTRUCTOR, LOCAL_VARIABLE, ANNOTATION_TYPE, PACKAGE, TYPE_PARAMETER, TYPE_USE, MODULE |
| **@Retention** | 指定注解的保留策略 | SOURCE, CLASS, RUNTIME |
| **@Documented** | 标记注解会被包含在 Javadoc 中 | - |
| **@Inherited** | 标记注解可被继承 | - |
| **@Repeatable** | 允许注解在同一元素上重复使用 | - |

#### 🧑‍💻 元注解示例

```java
// 定义一个运行时保留、可用于类和方法的注解
@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface MyAnnotation {
    String value() default "default";
    int priority() default 1;
}

// 定义可重复的注解
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface Roles {
    Role[] value();
}

@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Repeatable(Roles.class)
public @interface Role {
    String value();
}

// 使用重复注解
@Role("admin")
@Role("user")
public class UserService {
}
```

## 🧳 4. 注解解析

### 📘 反射解析注解

- **运行时注解**（@Retention(RUNTIME)）可通过反射解析
- **编译期注解**（@Retention(SOURCE)）需通过 APT（Annotation Processing Tool）解析

### 📘 解析步骤

1. **获取 Class/Field/Method/Constructor 对象**
2. **调用 getAnnotation(Class) 或 getAnnotations()** 获取注解
3. **调用注解的方法**获取属性值
4. **根据注解信息执行相应逻辑**

### 🧑‍💻 注解解析示例

```java
// 解析类上的注解
Class<?> clazz = UserService.class;
if (clazz.isAnnotationPresent(Roles.class)) {
    Roles rolesAnnotation = clazz.getAnnotation(Roles.class);
    Role[] roles = rolesAnnotation.value();
    for (Role role : roles) {
        System.out.println("Role: " + role.value());
    }
}

// 解析方法上的注解
Method method = UserService.class.getDeclaredMethod("getUserById", long.class);
if (method.isAnnotationPresent(Cacheable.class)) {
    Cacheable cacheable = method.getAnnotation(Cacheable.class);
    String key = cacheable.key();
    int expire = cacheable.expire();
    System.out.println("Cache key: " + key);
    System.out.println("Expire time: " + expire);
}

// 解析字段上的注解
Field field = User.class.getDeclaredField("username");
if (field.isAnnotationPresent(Validate.class)) {
    Validate validate = field.getAnnotation(Validate.class);
    boolean required = validate.required();
    String message = validate.message();
    System.out.println("Required: " + required);
    System.out.println("Message: " + message);
}
```

### 📘 处理可重复注解

```java
// 处理重复注解
Role[] roleAnnotations = UserService.class.getAnnotationsByType(Role.class);
for (Role role : roleAnnotations) {
    System.out.println("Role: " + role.value());
}
```

### 📘 APT 注解处理器

- **APT**（Annotation Processing Tool）用于编译期处理注解
- **步骤**：
  1. 继承 `AbstractProcessor`
  2. 重写 `process()` 方法
  3. 使用 `@SupportedAnnotationTypes` 和 `@SupportedSourceVersion` 注解

```java
@SupportedAnnotationTypes("com.example.Validate")
@SupportedSourceVersion(SourceVersion.RELEASE_8)
public class ValidateProcessor extends AbstractProcessor {
    @Override
    public boolean process(Set<? extends TypeElement> annotations, RoundEnvironment roundEnv) {
        // 处理注解
        for (TypeElement annotation : annotations) {
            Set<? extends Element> elements = roundEnv.getElementsAnnotatedWith(annotation);
            for (Element element : elements) {
                // 生成代码或做其他处理
            }
        }
        return true;
    }
}
```

```java
    public static void process(Object obj) {
        Class<?> clazz = obj.getClass();
        
        // 处理类上的注解
        processClassAnnotations(clazz);
        
        // 处理方法上的注解
        for (Method method : clazz.getDeclaredMethods()) {
            processMethodAnnotations(method, obj);
        }
        
        // 处理字段上的注解
        for (Field field : clazz.getDeclaredFields()) {
            processFieldAnnotations(field, obj);
        }
    }
```

```java
// 获取字段泛型类型
public class GenericTypeResolver {
    public static Class<?> getFieldGenericType(Field field) {
        Type type = field.getGenericType();
        if (type instanceof ParameterizedType) {
            ParameterizedType paramType = (ParameterizedType) type;
            Type[] actualTypes = paramType.getActualTypeArguments();
            if (actualTypes.length > 0 && actualTypes[0] instanceof Class) {
                return (Class<?>) actualTypes[0];
            }
        }
        return null;
    }
    
    public static Class<?> getMethodReturnGenericType(Method method) {
        Type type = method.getGenericReturnType();
        if (type instanceof ParameterizedType) {
            ParameterizedType paramType = (ParameterizedType) type;
            Type[] actualTypes = paramType.getActualTypeArguments();
            if (actualTypes.length > 0 && actualTypes[0] instanceof Class) {
                return (Class<?>) actualTypes[0];
            }
        }
        return null;
    }
}

// 使用

    public List<User> getUsers() {
        return users;
    }
}
```

## 🧳 2. 反射 + 注解

### 📘 简化版 IOC 容器

- **组件扫描**：扫描指定包下的类，发现带有 `@Component` 注解的类
- **依赖注入**：根据 `@Autowired` 注解注入依赖
- **生命周期管理**：支持 `@PostConstruct` 和 `@PreDestroy` 注解

```java
public class SimpleIOC {
    private final Map<Class<?>, Object> beans = new ConcurrentHashMap<>();
    private final List<Object> beanInstances = new ArrayList<>();
    
    public void scan(String basePackage) {
        try {
            // 简化版：实际项目中使用 ClassPathScanningCandidateComponentProvider
            String packagePath = basePackage.replace('.', '/');
            URL url = Thread.currentThread().getContextClassLoader().getResource(packagePath);
            if (url != null) {
                File dir = new File(url.toURI());
                for (File file : dir.listFiles()) {
                    if (file.isDirectory()) {
                        scan(basePackage + "." + file.getName());
                    } else if (file.getName().endsWith(".class")) {
                        String className = basePackage + "." + file.getName().replace(".class", "");
                        Class<?> clazz = Class.forName(className);
                        if (clazz.isAnnotationPresent(Component.class)) {
                            registerBean(clazz);
                        }
                    }
                }
            }
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }
    
    public <T> void registerBean(Class<T> clazz) {
        try {
            T instance = clazz.getDeclaredConstructor().newInstance();
            beans.put(clazz, instance);
            beanInstances.add(instance);
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }
    
    public void injectDependencies() {
        for (Object instance : beanInstances) {
            Class<?> clazz = instance.getClass();
            for (Field field : clazz.getDeclaredFields()) {
                if (field.isAnnotationPresent(Autowired.class)) {
                    field.setAccessible(true);
                    Class<?> dependencyClass = field.getType();
                    Object dependency = beans.get(dependencyClass);
                    if (dependency == null) {
                        throw new RuntimeException("Dependency not found: " + dependencyClass.getName());
                    }
                    try {
                        field.set(instance, dependency);
                    } catch (Exception e) {
                        throw new RuntimeException(e);
                    }
                }
            }
        }
    }
    
    public void initializeBeans() {
        for (Object instance : beanInstances) {
            Class<?> clazz = instance.getClass();
            for (Method method : clazz.getDeclaredMethods()) {
                if (method.isAnnotationPresent(PostConstruct.class)) {
                    method.setAccessible(true);
                    try {
                        method.invoke(instance);
                    } catch (Exception e) {
                        throw new RuntimeException(e);
                    }
                }
            }
        }
    }
    
    @SuppressWarnings("unchecked")
    public <T> T getBean(Class<T> clazz) {
        return (T) beans.get(clazz);
    }
}

// 使用
```

```java
            clazz.getInterfaces(),

                        return result;
                    } catch (Exception e) {
                        System.out.println("Error: " + logAnnotation.value());
                        throw e;
                    }
                } else {
                    return method.invoke(target, args);
                }
            }
```

```java
@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
public @interface ConfigProperty {
    String key();
    String defaultValue() default "";
}

public class ConfigInjector {
    private final Properties properties;
    
    public ConfigInjector(Properties properties) {
        this.properties = properties;
    }
    
    public void inject(Object obj) {
        Class<?> clazz = obj.getClass();
        for (Field field : clazz.getDeclaredFields()) {
            if (field.isAnnotationPresent(ConfigProperty.class)) {
                field.setAccessible(true);
                ConfigProperty annotation = field.getAnnotation(ConfigProperty.class);
                String key = annotation.key();
                String defaultValue = annotation.defaultValue();
                String value = properties.getProperty(key, defaultValue);
                
                try {
                    // 根据字段类型转换值
                    if (field.getType() == String.class) {
                        field.set(obj, value);
                    } else if (field.getType() == int.class || field.getType() == Integer.class) {
                        field.set(obj, Integer.parseInt(value));
                    } else if (field.getType() == boolean.class || field.getType() == Boolean.class) {
                        field.set(obj, Boolean.parseBoolean(value));
                    } else if (field.getType() == long.class || field.getType() == Long.class) {
                        field.set(obj, Long.parseLong(value));
                    } else if (field.getType() == double.class || field.getType() == Double.class) {
                        field.set(obj, Double.parseDouble(value));
                    }
                } catch (Exception e) {
                    throw new RuntimeException("Failed to inject config property: " + key, e);
                }
            }
        }
    }
}
```

```java
                // 检查 @Min
                if (field.isAnnotationPresent(Min.class) && value != null) {
                    Min annotation = field.getAnnotation(Min.class);
                    if (value instanceof Number) {
                        Number number = (Number) value;
                        if (number.intValue() < annotation.value()) {
                            errors.add(annotation.message());
                        }
                    }
                }
                
                // 检查 @Max
                if (field.isAnnotationPresent(Max.class) && value != null) {
                    Max annotation = field.getAnnotation(Max.class);
                    if (value instanceof Number) {
                        Number number = (Number) value;
                        if (number.intValue() > annotation.value()) {
                            errors.add(annotation.message());
                        }
                    }
                }
            } catch (Exception e) {
                errors.add("Error validating field " + field.getName() + ": " + e.getMessage());
            }
        }
        
        return errors;
    }
}

// 使用
public class User {
    @NotNull(message = "Username cannot be null")
    private String username;
    
    @Min(value = 18, message = "Age must be at least 18")
    @Max(value = 60, message = "Age must be at most 60")
    private int age;
    
    // getter/setter
}

User user = new User();
user.setUsername(null);
user.setAge(15);

List<String> errors = Validator.validate(user);
for (String error : errors) {
    System.out.println(error);
}
// 输出:
// Username cannot be null
// Age must be at least 18
```

```java
// 自定义注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Cache {
    String key();
    int expire() default 3600;
}

// 缓存管理器
public class CacheManager {
    private final Map<String, CacheEntry<?>> cache = new ConcurrentHashMap<>();
    
    public <T> void put(String key, T value, int expireSeconds) {
        long expireTime = System.currentTimeMillis() + expireSeconds * 1000;
        cache.put(key, new CacheEntry<>(value, expireTime));
    }
    
    @SuppressWarnings("unchecked")
    public <T> T get(String key) {
        CacheEntry<?> entry = cache.get(key);
        if (entry == null) {
            return null;
        }
        if (System.currentTimeMillis() > entry.getExpireTime()) {
            cache.remove(key);
            return null;
        }
        return (T) entry.getValue();
    }
    
    private static class CacheEntry<T> {
        private final T value;
        private final long expireTime;
        
        public CacheEntry(T value, long expireTime) {
            this.value = value;
            this.expireTime = expireTime;
        }
        
        public T getValue() {
            return value;
        }
        
        public long getExpireTime() {
            return expireTime;
        }
    }
}

// 缓存代理
public class CacheProxy {
    private final CacheManager cacheManager;
    
    public CacheProxy(CacheManager cacheManager) {
        this.cacheManager = cacheManager;
    }
    
    public <T> T createProxy(T target) {
        Class<?> clazz = target.getClass();
        return (T) Proxy.newProxyInstance(
            clazz.getClassLoader(),
            clazz.getInterfaces(),
            (proxy, method, args) -> {
                Method targetMethod = clazz.getMethod(method.getName(), method.getParameterTypes());
                if (targetMethod.isAnnotationPresent(Cache.class)) {
                    Cache cacheAnnotation = targetMethod.getAnnotation(Cache.class);
                    String keyPattern = cacheAnnotation.key();
                    int expire = cacheAnnotation.expire();
                    
                    // 生成缓存 key
                    String key = generateKey(keyPattern, args);
                    
                    // 尝试从缓存获取
                    Object result = cacheManager.get(key);
                    if (result != null) {
                        System.out.println("Cache hit: " + key);
                        return result;
                    }
                    
                    // 缓存未命中，执行方法
                    System.out.println("Cache miss: " + key);
                    result = method.invoke(target, args);
                    
                    // 存入缓存
                    cacheManager.put(key, result, expire);
                    return result;
                } else {
                    return method.invoke(target, args);
                }
            }
        );
    }
    
    private String generateKey(String pattern, Object[] args) {
        // 简单实现：替换 {0}, {1} 等占位符
        String key = pattern;
        for (int i = 0; i < args.length; i++) {
            key = key.replace("{" + i + "}", String.valueOf(args[i]));
        }
        return key;
    }
}

// 使用
public interface UserService {
    User findById(Long id);
    List<User> findAll();
}

public class UserServiceImpl implements UserService {
    @Override
    @Cache(key = "user:{0}", expire = 3600)
    public User findById(Long id) {
        System.out.println("Finding user by id: " + id);
        // 模拟数据库查询
        return new User(id, "User " + id);
    }
    
    @Override
    @Cache(key = "users:all", expire = 600)
    public List<User> findAll() {
        System.out.println("Finding all users");
        // 模拟数据库查询
        return Arrays.asList(new User(1L, "User 1"), new User(2L, "User 2"));
    }
}

// 测试
CacheManager cacheManager = new CacheManager();
CacheProxy cacheProxy = new CacheProxy(cacheManager);
UserService userService = new UserServiceImpl();
UserService proxy = cacheProxy.createProxy(userService);

// 第一次调用：缓存未命中
User user1 = proxy.findById(1L); // 输出: Cache miss: user:1
System.out.println(user1);

// 第二次调用：缓存命中
User user1Cached = proxy.findById(1L); // 输出: Cache hit: user:1
System.out.println(user1Cached);

// 第一次调用：缓存未命中
List<User> users1 = proxy.findAll(); // 输出: Cache miss: users:all
System.out.println(users1);

// 第二次调用：缓存命中
List<User> users1Cached = proxy.findAll(); // 输出: Cache hit: users:all
System.out.println(users1Cached);
```

## 🧳 5. 面试高频题汇总

### 📘 泛型相关

| 问题 | 回答思路 |
|------|----------|
| **泛型擦除是什么？** | 编译期有泛型检查，运行期类型参数被擦除为 Object 或上界，所以运行时无法拿到具体泛型类型。 |
| **PECS 原则是什么？** | Producer Extends, Consumer Super。生产者（读取）用 <? extends T>，消费者（写入）用 <? super T>。 |
| **泛型方法如何定义？** | 在方法修饰符与返回类型之间声明 <T>，类型参数仅在方法内有效。 |
| **通配符的使用场景？** | <?> 任意类型（只读），<? extends T> 子类（安全读取），<? super T> 父类（安全写入）。 |
| **泛型与重载的关系？** | 由于类型擦除，不能仅通过泛型参数不同来重载方法。 |
| **如何获取泛型参数类型？** | 通过反射的 getGenericType() 获取 Type，再强转为 ParameterizedType 解析。 |

### 📘 反射相关

| 问题 | 回答思路 |
|------|----------|
| **反射的作用是什么？** | 运行时动态获取类信息、创建对象、访问/修改成员、调用方法，是框架实现动态行为的基础。 |
| **反射的性能如何？** | 比直接调用慢，主要开销在方法查找、类型检查、安全检查；可通过缓存、MethodHandle 优化。 |
| **如何获取 Class 对象？** | 类名.class、对象.getClass()、Class.forName("全限定名")。 |
| **如何突破 private 访问限制？** | 使用 setAccessible(true) 跳过权限检查，仅限可信代码。 |
| **反射在 Spring 中的应用？** | DI 依赖注入、AOP 方法增强、注解处理、Bean 生命周期管理。 |
| **MethodHandle 与反射的区别？** | MethodHandle 性能接近直接调用，更轻量；反射功能更全面但性能较差。 |

### 📘 注解相关

| 问题 | 回答思路 |
|------|----------|
| **注解的本质是什么？** | 注解本质是继承了 Annotation 的特殊接口，用于提供元数据信息。 |
| **元注解有哪些？** | @Target、@Retention、@Documented、@Inherited、@Repeatable。 |
| **注解的保留策略有哪些？** | SOURCE（源码）、CLASS（字节码）、RUNTIME（运行时）。 |
| **如何解析运行时注解？** | 通过反射获取 Class/Field/Method 对象，再调用 getAnnotation() 方法。 |
| **注解在 Spring 中的应用？** | 组件扫描、依赖注入、AOP、事务管理、配置等。 |
| **编译期注解如何处理？** | 使用 APT（Annotation Processing Tool）在编译期处理。 |

### 📘 综合应用相关

| 问题 | 回答思路 |
|------|----------|
| **Spring 如何实现依赖注入？** | 通过反射扫描 Bean，发现 @Autowired 注解，从容器中获取依赖并注入。 |
| **MyBatis 如何实现 Mapper 接口？** | 通过反射动态生成 Mapper 接口的实现类，处理 SQL 执行和结果映射。 |
| **如何实现一个简单的 IOC 容器？** | 扫描包 → 发现 @Component 注解 → 反射创建实例 → 根据 @Autowired 注入依赖。 |
| **如何实现注解驱动的 AOP？** | 使用动态代理，在方法执行前后检查注解，执行相应的增强逻辑。 |
| **泛型、反射、注解在项目中的应用？** | 泛型用于类型安全，反射用于动态行为，注解用于元数据标记，三者结合实现框架的核心功能。 |
