## 🧠 1. 泛型概述

### 📘 什么是泛型

**泛型（Generics）** 是 Java 1.5 引入的**编译期类型参数化**机制：在类、接口或方法上声明类型参数（如 `T`、`E`），在使用时再指定具体类型，从而在**编译期**做类型检查并避免强制转换。

```java
List<String> list = new ArrayList<>();
list.add("Java");
// list.add(123);   // 编译错误
String s = list.get(0);  // 无需强转
```

> 💡 泛型在编译时生效，运行时会被**擦除**，JVM 看不到泛型类型信息。

------

### 🧠 为什么引入泛型

| 问题               | 泛型带来的改进              |
| ------------------ | --------------------------- |
| 编译时无法检查类型 | **编译期类型检查**，减少 ClassCastException |
| 需要强制类型转换   | **消除强转**，取元素即目标类型 |
| 代码重复           | **类型参数化**，一份代码多类型复用 |

#### 🧑‍💻 示例代码（泛型前 vs 后）

```java
// 泛型前：Object，需强转，易出错
List list = new ArrayList();
list.add("Java");
String s = (String) list.get(0);

// 泛型后：编译期检查，无需强转
List<String> list2 = new ArrayList<>();
list2.add("Java");
String s2 = list2.get(0);
```

------

### 💼 泛型在集合框架中的应用

| 类/接口       | 泛型含义     | 示例                   |
| ------------- | ------------ | ---------------------- |
| `List<E>`     | 元素类型     | `List<String>`         |
| `Map<K,V>`    | 键与值类型   | `Map<String, Integer>` |
| `Iterator<E>` | 遍历元素类型 | `Iterator<User>`       |

------

## 🧳 2. 泛型的基本语法

### 📘 泛型类（Generic Class）

- 在类名后声明类型参数 `<T>`，可在成员变量、方法参数与返回值中使用。

#### 🧑‍💻 示例代码

```java
public class Box<T> {
    private T value;
    public void set(T value) { this.value = value; }
    public T get() { return value; }
}
// 使用：Box<String> box = new Box<>(); box.set("Hello");
```

------

### 📘 泛型方法（Generic Method）

- 在方法修饰符与返回类型之间声明 `<T>`，类型参数仅在该方法内有效，可与类上的泛型不同。

#### 🧑‍💻 示例代码

```java
public static <T> void printArray(T[] array) {
    for (T t : array) System.out.println(t);
}
// 调用：Utils.<Integer>printArray(new Integer[]{1,2,3}); 或根据参数推断
```

------

### 📘 泛型接口（Generic Interface）

- 接口声明类型参数，实现类可指定具体类型或继续保留泛型。

#### 🧑‍💻 示例代码

```java
public interface Generator<T> {
    T next();
}
public class StringGenerator implements Generator<String> {
    @Override
    public String next() { return "Hello"; }
}
```

------

### 📘 类型参数命名约定

| 符号 | 含义     | 常见用法   |
| ---- | -------- | ---------- |
| T    | Type     | 通用类型   |
| E    | Element  | 集合元素   |
| K,V  | Key,Value| Map 键值   |
| ?    | 通配符   | 不确定类型 |

------

## 🧳 3. 通配符与边界

### 📘 无界通配符 `<?>`

- 表示“任意类型”，常用于只读遍历；**除 null 外不能写入**（编译器不知道实际类型）。

```java
public void printList(List<?> list) {
    for (Object obj : list) System.out.println(obj);
}
```

------

### 📘 上界通配符 `<? extends T>`

- 表示 **T 或其子类**，可**安全读取**为 T，但**不能写入**（不知道具体子类型，避免类型不安全）。

```java
public void process(List<? extends Number> list) {
    Number n = list.get(0);  // ✅ 可读
    // list.add(1);           // ❌ 不可写
}
```

------

### 📘 下界通配符 `<? super T>`

- 表示 **T 或其父类**，可**安全写入** T 及子类，但**读取只能得到 Object**。

```java
public void addNumbers(List<? super Integer> list) {
    list.add(10);  // ✅ 可写
}
```

------

### ⚖️ PECS 原则

> **Producer Extends, Consumer Super**
> - **生产者**（向外提供元素、读取）→ 用 `<? extends T>`
> - **消费者**（接收元素、写入）→ 用 `<? super T>`

| 通配符           | 可读     | 可写     | 典型场景   |
| ---------------- | -------- | -------- | ---------- |
| `<? extends T>` | ✅ 读为 T | ❌       | 生产者     |
| `<? super T>`   | 仅 Object | ✅ 写 T | 消费者     |

------

## 🧳 4. 类型擦除（Type Erasure）

### 📘 什么是类型擦除

- Java 泛型是**编译期**机制，运行时泛型信息被**擦除**，替换为**原始类型**（如 `Object` 或声明的上界）。
- 因此运行时无法得到“具体泛型类型”（如无法判断 `List<String>` 与 `List<Integer>` 的差别）。

```java
List<String> list1 = new ArrayList<>();
List<Integer> list2 = new ArrayList<>();
System.out.println(list1.getClass() == list2.getClass());  // true
```

------

### ⚠️ 擦除带来的限制

| 限制           | 说明                         |
| -------------- | ---------------------------- |
| 无法 instanceof 泛型 | 如 `obj instanceof List<String>` 编译错误 |
| 不能直接 new 泛型数组 | 如 `new List<String>[10]` 不允许 |
| 静态成员不能用类型参数 | 泛型属于实例，静态属于类     |
| 不能 new T() 等 | 擦除后 T 为 Object，无法确定构造 |

------

### 🔍 通过反射获取泛型参数类型

- 若字段/方法签名上**保留**了泛型（如 `List<String>`），可通过 `getGenericType()` 得到 `ParameterizedType`，再取 `getActualTypeArguments()`。

#### 🧑‍💻 示例代码

```java
Field field = MyClass.class.getDeclaredField("list");
Type type = field.getGenericType();  // 若声明为 List<String>
if (type instanceof ParameterizedType) {
    Type[] args = ((ParameterizedType) type).getActualTypeArguments();
    System.out.println(args[0].getTypeName());  // java.lang.String
}
```

------

### 💡 泛型数组的替代方案

- 使用 `Array.newInstance(Class<?>, int)` 创建与泛型成分类型一致的数组，再强转；或直接用**集合**（如 `ArrayList<T>`）代替数组。

```java
@SuppressWarnings("unchecked")
T[] array = (T[]) Array.newInstance(clazz, size);
```

------

## 🧳 5. 泛型在项目中的常见应用

### 📘 统一响应封装（Result / Response）

```java
public class Result<T> {
    private int code;
    private String msg;
    private T data;
    // getter/setter
}
// 使用：Result<User> res = ...; User u = res.getData();
```

### 📘 ORM / Mapper 接口（如 MyBatis-Plus）

```java
public interface BaseMapper<T> {
    int insert(T entity);
    T selectById(Serializable id);
}
```

> 泛型保证方法参数与返回值与实体类型一致，避免到处强转。

# 🧩 第二章：反射（Reflection）

> **目标：** 理解反射的概念与用途，掌握 Class 的获取、对象创建、字段/方法/构造器的访问与调用，以及反射在框架中的应用与性能注意点。

> **本章在整体中解决什么问题**：第一章泛型在运行期被擦除，反射可在运行期获取类信息与泛型参数；本章讲**反射 API** 与典型用法，为**第三章注解解析**和**第四章综合应用**打基础。Spring、MyBatis 等框架大量依赖反射。

------

## 🧠 1. 反射的基本概念

### 📘 什么是反射

- **反射**是在**运行时**动态获取类的信息（字段、方法、构造器、注解等），并可以**创建对象、访问/修改成员、调用方法**，无需在编译期确定类型。
- 反射是很多框架（Spring、MyBatis、JUnit、Jackson）实现**动态行为**的基础（如根据配置创建 Bean、根据注解映射字段）。

### 📘 核心类

| 类            | 作用                         |
| ------------- | ---------------------------- |
| **Class**     | 表示类或接口，入口            |
| **Field**     | 表示字段，可读/写             |
| **Method**    | 表示方法，可 invoke 调用     |
| **Constructor** | 表示构造器，可 newInstance 创建对象 |

------

## 🧳 2. 获取 Class 对象

### 📘 三种常见方式

| 方式                     | 示例                         |
| ------------------------ | ---------------------------- |
| 类名.class               | `User.class`                 |
| 对象.getClass()          | `obj.getClass()`             |
| Class.forName("全限定名") | `Class.forName("com.xxx.User")` |

- **forName** 会触发类加载（若未加载）；**class** 字面量不会主动加载其他类。

#### 🧑‍💻 示例代码

```java
Class<?> c1 = String.class;
Class<?> c2 = "hello".getClass();
Class<?> c3 = Class.forName("java.lang.String");
// c1 == c2 == c3 为 true（同一类只加载一次）
```

------

## 🧳 3. 反射的核心操作

### 📘 创建对象

- **无参**：`clazz.getDeclaredConstructor().newInstance()`（JDK 9+ 推荐；旧版 `newInstance()` 已过时）。
- **有参**：先 `getDeclaredConstructor(参数类型...)` 得到 `Constructor`，再 `newInstance(实参...)`。

#### 🧑‍💻 示例代码

```java
Class<?> clazz = User.class;
Constructor<?> cons = clazz.getDeclaredConstructor(String.class, int.class);
Object obj = cons.newInstance("张三", 20);
```

------

### 📘 访问与修改字段

- **getDeclaredField(name)**：获取本类声明字段（含 private）；**getField(name)** 仅 public 且含继承。
- **setAccessible(true)**：突破访问限制，可访问 private。
- **get(obj)** / **set(obj, value)**：读/写实例字段；静态字段时 obj 传 null。

#### 🧑‍💻 示例代码

```java
Field field = clazz.getDeclaredField("name");
field.setAccessible(true);
field.set(obj, "李四");
Object value = field.get(obj);
```

------

### 📘 调用方法

- **getDeclaredMethod(name, 参数类型...)** / **getMethod(...)**：获取方法。
- **invoke(obj, 实参...)**：执行；静态方法 obj 传 null。

#### 🧑‍💻 示例代码

```java
Method method = clazz.getDeclaredMethod("setName", String.class);
method.setAccessible(true);
method.invoke(obj, "王五");
```

------

### 📘 获取泛型参数类型

- 字段/方法若带泛型（如 `List<String>`），用 **getGenericType()** 得到 `Type`，再强转为 `ParameterizedType` 取 `getActualTypeArguments()`。

------

## 🧳 4. 反射与访问控制

### 📘 setAccessible(true)

- **AccessibleObject**（Field、Method、Constructor 的父类）的 **setAccessible(boolean)** 可**绕过**访问权限检查，访问 private 等成员。
- JDK 9+ 模块系统下，对**未开放**的模块中的类，即使 setAccessible 也可能被拒绝，需在启动参数中开放（--add-opens）。

### ⚠️ 安全与性能

- 反射会**破坏封装**，生产代码中应谨慎；仅在对框架、工具、测试等场景下合理使用。
- 反射调用比直接调用**慢**，可对 Method/Constructor 做**缓存**，避免重复 getMethod；高热点路径慎用。

------

## 🧳 5. 反射在框架中的应用

| 框架/场景     | 典型用法                                   |
| ------------- | ------------------------------------------ |
| **Spring**    | 根据配置/注解反射创建 Bean、注入依赖、调用 init 方法 |
| **MyBatis**   | 反射将 ResultSet 映射到实体属性            |
| **JUnit**     | 反射查找并执行带 @Test 的方法              |
| **Jackson/Gson** | 反射读取字段/getter 进行序列化与反序列化 |

## 🎯 面试常见追问

| 面试官提问           | 回答思路 |
| -------------------- | -------- |
| 反射有什么缺点？     | 破坏封装、有安全风险；性能比直接调用差，可缓存 Method/Constructor 减轻。 |
| 如何通过反射获取泛型类型？ | 通过 Field#getGenericType() 或 Method#getGenericParameterTypes() 得到 Type，再转为 ParameterizedType 取 getActualTypeArguments()。 |
| Spring 中反射用在哪？ | Bean 的实例化、属性注入、初始化回调（如 @PostConstruct）、AOP 代理等。 |

------

### ⚠️ 常见坑与注意点

| 现象 / 易错点 | 原因 | 正确做法 / 怎么记 |
|---------------|------|-------------------|
| 反射修改 final 字段未先 setAccessible(true) | 默认无法访问 private/final，会抛 IllegalAccessException | 先 **field.setAccessible(true)**（注意模块化下可能被拒绝）；修改 final 后语义上仍是“未初始化完成”的错觉，慎用。 |
| 反射破坏单例（通过反射调私有构造 newInstance） | 私有构造可被 setAccessible 后调用，生成新实例 | 单例构造内可判断“若实例已存在则抛异常”，或用枚举单例防反射。 |
| 反射调用方法未处理 InvocationTargetException | method.invoke 会把被调方法内部异常包装成 InvocationTargetException | 捕获后用 **getCause()** 取出真实异常再处理或重抛。 |
| 频繁反射调用方法、未缓存 Method | 每次 getMethod + invoke 有查找与检查开销，性能差 | 将 **Method/Constructor** 缓存起来（如 Map<方法名, Method>），只反射一次，后续复用。 |
| JDK 9+ 模块化下反射访问未开放包失败 | 未在 module-info 中 opens 的包，反射无法访问 | 运行加 --add-opens，或模块声明 opens 对应包给反射框架。 |

# 🧩 第三章：注解（Annotation）

> **目标：** 理解注解的本质与保留策略，掌握元注解、自定义注解、通过反射解析注解，以及注解在框架中的典型应用。

> **本章在整体中解决什么问题**：第二章反射可读取类/方法/字段信息；本章讲**注解**——元数据、自定义注解、保留策略及**通过反射解析注解**。框架中“注解生效”本质是反射读取注解再执行逻辑；学完本章后**第四章**会结合泛型+反射+注解做综合应用。

------

## 🧠 1. 注解的基本概念

### 📘 什么是注解

- **注解**是一种**元数据**，附加在类、方法、字段、参数等之上，本身不直接执行业务逻辑，而是供**编译期或运行期**的工具/框架读取并据此做处理。
- 本质上是继承了 **Annotation** 的**特殊接口**，可包含**属性**（无参方法形式，带默认值）。

### 📘 注解的作用

| 阶段     | 作用举例                           |
| -------- | ---------------------------------- |
| 编译期   | @Override 检查重写、Lombok 生成代码、APT 生成源码 |
| 运行期   | Spring 扫描 @Component、JUnit 执行 @Test、校验 @NotNull |

------

## 🧳 2. 内置注解与元注解

### 📘 常见内置注解

| 注解               | 作用                         |
| ------------------ | ---------------------------- |
| **@Override**      | 标记重写父类/接口方法，编译期检查 |
| **@Deprecated**   | 标记过时，编译/IDE 告警      |
| **@SuppressWarnings** | 抑制指定类型告警（如 "unchecked"） |
| **@FunctionalInterface** | 标记函数式接口，编译期检查 |

------

### 📘 元注解（修饰注解的注解）

| 元注解         | 作用                           |
| -------------- | ------------------------------ |
| **@Target**    | 指定注解可作用的目标（TYPE、METHOD、FIELD、PARAMETER 等） |
| **@Retention** | 指定保留策略：SOURCE / CLASS / RUNTIME |
| **@Documented** | 是否出现在 Javadoc 中          |
| **@Inherited** | 子类是否可继承父类上的该注解   |
| **@Repeatable**（JDK 8+） | 同一位置可重复使用该注解（需指定容器注解） |

### 📘 保留策略（@Retention）

| 策略     | 说明                     | 典型用途           |
| -------- | ------------------------ | -------------------- |
| **SOURCE** | 仅源码，编译后丢弃       | @Override、Lombok   |
| **CLASS**  | 保留到 class 文件，运行时不加载 | 部分字节码增强      |
| **RUNTIME** | 保留到运行时，可反射获取 | Spring、JUnit、校验 |

> 💡 只有 **RUNTIME** 才能在运行期通过 **getAnnotation()** 等读取。

------

## 🧳 3. 自定义注解

### 📘 语法与属性

- 使用 **@interface** 定义；属性为**无参方法**，默认值用 **default** 指定。
- 若只有一个属性且名为 **value**，使用时可直接写 `@Anno("值")`。

#### 🧑‍💻 示例代码

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation {
    String value() default "";
    int order() default 0;
}
// 使用：@MyAnnotation(value = "test", order = 1)
```

------

## 🧳 4. 注解的解析（反射）

### 📘 常用 API

| API                          | 说明                           |
| ---------------------------- | ------------------------------ |
| **isAnnotationPresent(Class)** | 是否存在指定注解               |
| **getAnnotation(Class)**     | 获取指定类型注解（不含继承）   |
| **getDeclaredAnnotations()** | 本元素上的全部注解             |
| **getAnnotations()**         | 含继承的注解（仅对类有效，需 @Inherited） |

- 可对 **Class**、**Method**、**Field**、**Constructor**、**Parameter** 等调用上述方法。

#### 🧑‍💻 示例代码

```java
Method method = clazz.getDeclaredMethod("myMethod");
if (method.isAnnotationPresent(MyAnnotation.class)) {
    MyAnnotation ann = method.getAnnotation(MyAnnotation.class);
    String v = ann.value();
}
```

### 📍 自定义注解 + 处理器示例（入参→执行→出参）

下面是一个「自定义注解 + 反射处理器」的完整小示例：用注解标记方法，处理器扫描后根据注解执行逻辑（如打印日志、校验角色）。

**1）自定义注解**

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface RequireRole {
    String value();  // 需要的角色，如 "admin"
}
```

**2）处理器（解析注解并执行）**

```java
public static void process(Object obj) throws Exception {
    Class<?> clazz = obj.getClass();
    for (Method m : clazz.getDeclaredMethods()) {
        if (!m.isAnnotationPresent(RequireRole.class)) continue;
        RequireRole ann = m.getAnnotation(RequireRole.class);
        String role = ann.value();
        // 模拟：根据当前用户角色决定是否调用
        if ("admin".equals(role)) {
            m.setAccessible(true);
            m.invoke(obj);   // 入参：obj 为实例；执行：invoke；出参：方法返回值
        }
    }
}
```

**3）使用**

```java
class Service {
    @RequireRole("admin")
    public void deleteUser() { System.out.println("deleteUser"); }
}
process(new Service());  // 会反射调用 deleteUser()
```

> **面试怎么说**：注解只是元数据；要“生效”需要**处理器**——运行期用反射读取注解（getAnnotation），再根据注解属性执行相应逻辑（如权限校验、日志、注入）。Spring 的 @Autowired、@Transactional 都是“注解 + 反射/代理”实现的。

## 🧳 5. 注解在框架中的典型应用

| 框架/场景   | 常见注解举例                                   |
| ----------- | ------------------------------------------------ |
| **Spring**  | @Component、@Service、@Autowired、@Transactional、@RequestMapping |
| **MyBatis** | @Mapper、@Select、@Insert、@Param                |
| **JUnit**   | @Test、@BeforeEach、@AfterEach、@DisplayName     |
| **校验**    | @NotNull、@Size（JSR 303/380）                  |
| **Jackson** | @JsonProperty、@JsonIgnore                      |

- 框架在**启动或首次使用**时扫描类/方法/字段上的注解，再结合反射完成**注册、注入、代理、校验**等逻辑。

## 🧳 6. 注解 + 反射 简单实战思路

- **权限控制**：自定义 @RequireRole("admin")，在拦截器/切面中通过反射取方法上的注解，再根据当前用户角色决定是否放行。
- **日志/耗时**：自定义 @Log，AOP 切带该注解的方法，在环绕通知中打日志或统计耗时。
- **模拟组件扫描**：遍历包下所有类，对带 @Component 的类反射实例化并放入容器（简化版 IOC）。

## 🧠 1. 泛型 + 反射

- **获取泛型参数类型**：字段声明为 `List<User>` 时，通过 Field#getGenericType() 得到 ParameterizedType，可拿到 `User.class`，用于通用 Mapper、序列化等。

- **通用 Service 层**：`BaseService<T>` 泛型 + 反射获取 T 的 Class，配合注解标识表名/主键，实现通用 CRUD。

# 📎 附录：面试题速查

1. **泛型擦除是什么？** 编译期有类型检查，运行期类型参数被擦除为 Object 或上界，运行时无法得到具体泛型类型。
2. **PECS 是什么？** Producer Extends（只读用 extends），Consumer Super（只写用 super）。
3. **反射获取 Class 的三种方式？** 类名.class、对象.getClass()、Class.forName("全限定名")。
4. **反射为什么慢？** 需要解析、访问检查、方法调用走反射路径；可缓存 Method/Constructor 减轻。
5. **注解的保留策略？** SOURCE（仅源码）、CLASS（class 文件）、RUNTIME（运行时可反射读取）。
6. **Spring 中反射和注解用在哪？** Bean 创建与注入、组件扫描、AOP、事务等；注解标记，反射读取并执行逻辑。
7. **如何通过反射获取泛型参数类型？** getGenericType() 得到 Type，转为 ParameterizedType 后 getActualTypeArguments()。
