# **第一章：Java 多线程与并发概述** 🧵

------

### **1. 什么是多线程与并发** 🤔

- **多线程**：在同一个进程中并发执行多个线程。每个线程是一个独立的执行路径，拥有自己的程序计数器、栈和局部变量，但共享内存资源。
  - **线程**是 CPU 调度的基本单位，每个线程都有独立的执行顺序。
- **并发**：多个任务在同一时间段内交替执行，多个任务并不一定要同时进行，尤其在单核 CPU 上，任务通过时间分片轮流执行。
  - **并行**：多个任务在同一时刻同时执行，通常需要多核 CPU 支持。

**示例：**

- 在 **多核 CPU** 上，多个线程可以并行执行。
- 在 **单核 CPU** 上，多个线程通过操作系统调度轮流执行，表现为并发。

------

### **2. 多线程与并发的应用场景** 🌐

多线程和并发编程广泛应用于各种场景中，以下是一些常见的应用：

| **应用场景**            | **描述**                                               |
| ----------------------- | ------------------------------------------------------ |
| **Web 服务器**          | 处理大量并发请求，每个请求由不同线程处理。             |
| **图形用户界面（GUI）** | 保证界面响应性，后台任务（如数据加载）由独立线程处理。 |
| **数据库连接池**        | 多线程管理多个数据库连接，提升并发查询能力。           |
| **游戏开发**            | 同时处理渲染、物理模拟、输入事件等任务。               |
| **大数据处理**          | 将任务并行化，提高数据处理和计算速度。                 |

------

### **3. Java 中的线程模型与进程模型** 🖥️

在 Java 中，线程和进程是操作系统的基本执行单元。Java 提供了一套线程模型来支持并发编程。

#### **进程模型**

- 每个进程有独立的地址空间和资源。
- 进程间通信（IPC）比较复杂。
- 创建和销毁进程开销较大。

#### **线程模型**

- 线程共享进程的地址空间和资源。
- 线程间通信简单（通过共享内存）。
- 线程的创建和销毁开销较小。

Java 使用操作系统的线程调度机制，通过虚拟机（JVM）管理线程。

------

### **4. 线程与进程的区别** ⚖️

| **特性**     | **进程**                     | **线程**                               |
| ------------ | ---------------------------- | -------------------------------------- |
| **定义**     | 操作系统分配资源的基本单位。 | 执行程序的基本单位。                   |
| **内存空间** | 每个进程有独立内存空间。     | 线程共享进程的内存空间。               |
| **通信方式** | 进程间通信（IPC）复杂。      | 线程间通过共享内存通信简单。           |
| **创建开销** | 创建进程开销大。             | 创建线程开销小。                       |
| **调度开销** | 进程切换开销较大。           | 线程切换开销较小。                     |
| **独立性**   | 进程相互独立。               | 线程相互依赖，出错时可能导致进程崩溃。 |

**总结**：

- **进程**适用于更强的隔离和独立性要求，但开销较大。
- **线程**适合共享资源和高效执行，但需要注意同步和安全性。

------

### **5. 为什么需要并发编程** ⚡

并发编程可以显著提升系统性能，特别是在多核处理器和分布式环境中。具体原因如下：

| **原因**             | **描述**                                                  |
| -------------------- | --------------------------------------------------------- |
| **提高资源利用率**   | 多个任务并发执行，充分利用多核 CPU。                      |
| **提高响应性**       | 在 GUI 或高负载系统中，使用多线程避免界面冻结，保持响应。 |
| **缩短任务完成时间** | 将大任务拆分并行执行，减少总完成时间。                    |
| **处理高并发请求**   | 如 Web 服务器，高并发请求可以通过多线程同时处理。         |

**并发编程的挑战**：

- 需要处理线程安全、死锁、资源竞争等问题。
- 对设计和调试提出更高要求。

# 🧩 **第二章：Java 中的线程基础**

> **目标：**
>  理解 Java 中线程的基本概念与操作，掌握线程的生命周期、创建与启动方式，学会控制线程执行和处理中断与暂停。

## 🧠 **1. 线程的生命周期**

### 📘 **线程生命周期概述**

线程的生命周期指的是从线程创建到线程终止的过程。线程可以处于以下几种状态：

| **状态**                  | **描述**                                                     |
| ------------------------- | ------------------------------------------------------------ |
| **新建（New）**           | 线程对象创建后，尚未调用 `start()` 方法。                    |
| **就绪（Runnable）**      | 线程已启动并等待操作系统调度执行，可能会被 CPU 执行。        |
| **运行（Running）**       | 线程正在执行 `run()` 方法中的代码。                          |
| **阻塞（Blocked）**       | 线程因为等待某个资源（如 I/O、锁等）被挂起，无法继续执行。   |
| **等待（Waiting）**       | 线程处于等待其他线程的某些条件下，通常由 `wait()`、`join()`、`sleep()` 等方法引起。 |
| **超时（Timed Waiting）** | 线程在等待某个条件或时间限制，使用 `sleep()`、`join()` 等带时间参数的调用。 |
| **终止（Terminated）**    | 线程执行完毕，`run()` 方法返回或异常退出，线程进入终止状态。 |

### 📈 **线程状态转换**

线程在生命周期中可以从一个状态转变到另一个状态。例如：

- 从 **新建（New）** 到 **就绪（Runnable）**：通过调用 `start()` 方法。
- 从 **就绪（Runnable）** 到 **运行（Running）**：由操作系统调度 CPU 资源给线程。
- 从 **运行（Running）** 到 **阻塞（Blocked）**：线程等待某个资源，执行被挂起。
- 从 **运行（Running）** 到 **等待（Waiting）** 或 **超时（Timed Waiting）**：线程调用了 `wait()`、`join()`、`sleep()` 等方法。

## 🧳 **2. 创建线程的方式**

Java 提供了多种方式来创建和启动线程，以下是常用的三种方法：

### 📘 **1. 继承 `Thread` 类**

- 通过继承 `Thread` 类并重写 `run()` 方法来创建线程。然后使用 `start()` 方法启动线程。

#### 🧑‍💻 示例代码

```java
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Thread is running...");
    }
}

public class Main {
    public static void main(String[] args) {
        MyThread thread = new MyThread();
        thread.start();  // 启动线程
    }
}
```

- **优点**：代码简洁，适合线程类没有其他继承关系时使用。
- **缺点**：Java 不支持多重继承，继承 `Thread` 类时无法继承其他类。

### 📘 **2. 实现 `Runnable` 接口**

- 通过实现 `Runnable` 接口并重写 `run()` 方法来创建线程，然后将 `Runnable` 对象传递给 `Thread` 类的构造函数，再调用 `start()` 方法启动线程。

#### 🧑‍💻 示例代码

```java
class MyRunnable implements Runnable {
    @Override
    public void run() {
        System.out.println("Thread is running...");
    }
}

public class Main {
    public static void main(String[] args) {
        MyRunnable myRunnable = new MyRunnable();
        Thread thread = new Thread(myRunnable);
        thread.start();  // 启动线程
    }
}
```

- **优点**：支持多实现类，适合需要共享资源的场景。
- **缺点**：不如继承 `Thread` 类那样直接。

### 📘 **3. 使用 `Callable` 和 `Future`**

- `Callable` 是 `Runnable` 的改进版本，支持返回值和抛出异常。通过 `ExecutorService` 提交任务并返回一个 `Future` 对象，可以在未来获取线程的执行结果。

#### 🧑‍💻 示例代码

```java
import java.util.concurrent.*;

class MyCallable implements Callable<String> {
    @Override
    public String call() throws Exception {
        return "Task completed!";
    }
}

public class Main {
    public static void main(String[] args) throws ExecutionException, InterruptedException {
        ExecutorService executor = Executors.newCachedThreadPool();
        MyCallable myCallable = new MyCallable();
        Future<String> future = executor.submit(myCallable);
        
        System.out.println(future.get());  // 获取线程执行结果
        executor.shutdown();
    }
}
```

- **优点**：可以返回结果或处理异常，更适合复杂任务。
- **缺点**：相比 `Runnable`，代码略复杂。

## 🧳 **3. 启动与控制线程**

### 📘 **`start()` 与 `run()` 方法**

- **`start()`**：启动线程，调用操作系统的线程调度机制，线程进入就绪状态，由操作系统调度执行 `run()` 方法中的代码。
- **`run()`**：线程执行的主体方法，包含线程需要完成的任务。`run()` 方法在当前线程中被直接调用时并不会启动新线程，而是直接执行任务。

#### 🧑‍💻 示例代码

```java
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Thread is running...");
    }
}

public class Main {
    public static void main(String[] args) {
        MyThread thread = new MyThread();
        thread.run();  // 不会启动新线程，直接执行
        thread.start();  // 启动线程，调用 run() 方法
    }
}
```

### 📘 **`sleep()` 与 `yield()` 方法**

- **`sleep(long millis)`**：让当前线程暂停指定时间，释放 CPU 控制权，使得其他线程有机会执行。
- **`yield()`**：让当前线程暂时退出运行状态，将 CPU 控制权交给同优先级的线程。

#### 🧑‍💻 示例代码

```java
class MyThread extends Thread {
    @Override
    public void run() {
        try {
            System.out.println("Thread is going to sleep...");
            Thread.sleep(2000);  // 睡眠 2 秒
            System.out.println("Thread woke up!");
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}

public class Main {
    public static void main(String[] args) {
        MyThread thread = new MyThread();
        thread.start();
    }
}
```

- `sleep()` 会导致当前线程 **暂停** 指定时间，期间 CPU 可以分配给其他线程。
- `yield()` 使当前线程让出 CPU 时间片，但不会完全休眠，通常用于 **调度优化**。

## 🧳 **4. 线程的中断与暂停**

### 📘 **`interrupt()` 方法**

- `interrupt()` 方法用于 **中断** 线程。它不会立即停止线程的执行，而是设置线程的中断标志，线程需要在合适的地方检查该标志并处理（例如在 `sleep()`、`wait()` 等阻塞操作中抛出 `InterruptedException`）。

#### 🧑‍💻 示例代码

```java
class MyThread extends Thread {
    @Override
    public void run() {
        try {
            Thread.sleep(5000);
        } catch (InterruptedException e) {
            System.out.println("Thread was interrupted!");
        }
    }
}

public class Main {
    public static void main(String[] args) throws InterruptedException {
        MyThread thread = new MyThread();
        thread.start();
        Thread.sleep(1000);
        thread.interrupt();  // 中断线程
    }
}
```

- **注意**：中断只是设置线程的中断标志，线程本身不主动停止，必须显式检查标志并做出相应的响应。

### 📘 **线程的安全停止**

- 如果线程执行的任务是阻塞性的（如 `sleep()`、`wait()` 等），可以通过 `interrupt()` 中断线程并安全地退出。
- 如果线程执行的是非阻塞性的任务，可以通过添加状态标志来安全停止线程。

#### 🧑‍💻 示例代码（使用状态标志）

```java
class MyThread extends Thread {
    private volatile boolean running = true;

    @Override
    public void run() {
        while (running) {
            System.out.println("Thread is running...");
        }
    }

    public void stopRunning() {
        running = false;
    }
}

public class Main {
    public static void main(String[] args) throws InterruptedException {
        MyThread thread = new MyThread();
        thread.start();
        Thread.sleep(1000);
        thread.stopRunning();  // 安全停止线程
    }
}
```

## 🧳 **5. 线程的优先级与调度**

### 📘 **线程优先级**

- 线程优先级用来表示线程执行的优先顺序。Java 中的线程优先级是一个整数，范围从 `Thread.MIN_PRIORITY`（1）到 `Thread.MAX_PRIORITY`（10），默认值为 `Thread.NORM_PRIORITY`（5）。

#### 🧑‍💻 示例代码

```java
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Thread with priority " + getPriority() + " is running...");
    }
}

public class Main {
    public static void main(String[] args) {
        MyThread thread1 = new MyThread();
        thread1.setPriority(Thread.MIN_PRIORITY);  // 设置最低优先级
        thread1.start();

        MyThread thread2 = new MyThread();
        thread2.setPriority(Thread.MAX_PRIORITY);  // 设置最高优先级
        thread2.start();
    }
}
```

### ⚡ **线程调度**

- 线程调度是操作系统和 JVM 共同管理的过程，调度算法通常是基于时间片轮转，优先级高的线程可以获得更多的 CPU 时间。

## 🧠 **1. 线程安全的概念**

### 📘 **线程安全的定义**

- **线程安全**（Thread Safety）是指多个线程在并发执行时，能够正确地共享资源而不会出现数据不一致或竞争条件的问题。
- 线程安全的代码能够在多线程环境下正确运行，而不需要外部同步机制。

### ⚡ **如何判断是否线程安全**

- 如果一个类的实例在多个线程间共享时，其状态保持一致，并且不出现数据竞争或状态破坏，那么这个类是线程安全的。

#### 🧑‍💻 **线程安全的示例**

```java
class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public synchronized int getCount() {
        return count;
    }
}
```

- 通过 `synchronized` 关键字，保证了 `increment()` 和 `getCount()` 方法在多线程环境下的互斥执行，确保了 `count` 的线程安全性。

### 📊 **线程安全的类型**

1. **内部状态不变**：类的状态通过同步机制保护，状态保持一致。
2. **无竞争条件**：多个线程访问时不会发生不一致的修改。
3. **原子操作**：操作是不可分割的，要么完全执行，要么完全不执行。

------

## 🧳 **2. Java 中的同步机制**

### 📘 **同步的作用**

- **同步（Synchronization）** 是多线程编程中解决资源竞争和数据不一致的主要手段，确保同一时间只有一个线程可以访问共享资源。

#### 🧑‍💻 **Java 中的同步方法和同步块**

Java 提供了多种同步机制，最常用的是 `synchronized` 关键字，用于修饰方法或代码块。

### 🧑‍💻 **使用 `synchronized` 关键字**

- **同步方法**：在方法声明中加上 `synchronized` 关键字，确保同一时间只有一个线程可以执行该方法。

```java
class Counter {
    private int count = 0;

    // 同步方法，保证线程安全
    public synchronized void increment() {
        count++;
    }
}
```

- **同步块**：可以在方法内指定同步代码块，避免整个方法都被锁住，提高性能。同步块通常用于对 **共享资源** 进行同步。

```java
class Counter {
    private int count = 0;

    public void increment() {
        synchronized (this) { // 锁住当前对象
            count++;
        }
    }
}
```

- `synchronized` 的作用是 **对某个资源加锁**，确保同一时刻只能有一个线程执行被锁住的代码区域。

#### 🧑‍💻 **同步块与同步方法的区别**

| 特性     | 同步方法                 | 同步块                   |
| -------- | ------------------------ | ------------------------ |
| 锁定范围 | 锁定整个方法             | 只锁定特定的代码块       |
| 灵活性   | 锁住整个方法，灵活性差   | 锁定部分代码，灵活性较高 |
| 性能影响 | 会锁住整个方法，性能稍差 | 只锁住关键部分，提高性能 |

------

## 🧳 **3. 内存可见性与线程间通信**

### 📘 **内存可见性**

- **内存可见性**是指一个线程对共享变量的修改，其他线程是否能立即看到该修改。
- 在多线程环境中，如果没有适当的同步机制，线程之间可能不会看到对共享变量的修改，导致数据不一致。

#### 🧑‍💻 **解决内存可见性问题：使用 `volatile` 关键字**

- `volatile` 关键字确保变量的更新对所有线程立即可见，禁止将值缓存到线程的本地缓存中。

```java
class SharedResource {
    private volatile boolean flag = false;

    public void setFlag(boolean flag) {
        this.flag = flag;
    }

    public boolean getFlag() {
        return flag;
    }
}
```

- `volatile` 保证了变量的 **内存可见性**，但它 **不能保证原子性**，因此当需要对共享变量进行复合操作时，仍然需要使用同步机制。

### 📘 **线程间通信**

- 线程间通信是指不同线程之间通过某种机制交换数据或协作。
- 常见的线程间通信方法包括：
  1. **`wait()` 和 `notify()`**：通过对象锁进行通信，常用于生产者消费者模式。
  2. **`CountDownLatch`**：线程间同步工具类，用于控制多个线程之间的执行顺序。
  3. **`CyclicBarrier`**：线程同步工具类，多个线程相互等待，直到达到某个条件。

#### 🧑‍💻 **`wait()` 和 `notify()` 示例**

```java
class SharedResource {
    private boolean available = false;

    public synchronized void produce() throws InterruptedException {
        while (available) {
            wait();  // 等待资源可用
        }
        available = true;
        System.out.println("Produced");
        notify();  // 通知消费者
    }

    public synchronized void consume() throws InterruptedException {
        while (!available) {
            wait();  // 等待资源
        }
        available = false;
        System.out.println("Consumed");
        notify();  // 通知生产者
    }
}
```

## 🧳 **4. 死锁及其解决方法**

### 📘 **死锁的定义**

- **死锁**（Deadlock）是指两个或多个线程在执行过程中，因为争夺资源而造成的一种互相等待的局面。此时，线程都无法继续执行，从而造成程序停止响应。

#### 🧑‍💻 **死锁的产生条件**

根据著名的死锁理论，死锁发生的四个必要条件是：

1. **互斥条件**：至少有一个资源处于 **非共享模式**，即只能由一个线程占用。
2. **占有并等待条件**：一个线程持有至少一个资源，并等待获得其他线程持有的资源。
3. **不剥夺条件**：线程已获得的资源在未使用完之前不能被剥夺。
4. **循环等待条件**：多个线程形成环状等待，A 等待 B，B 等待 C，C 等待 A。

#### 🧑‍💻 **死锁示例**

```java
class A {
    public synchronized void methodA(B b) {
        b.last();
    }

    public synchronized void last() {}
}

class B {
    public synchronized void methodB(A a) {
        a.last();
    }

    public synchronized void last() {}
}

public class Main {
    public static void main(String[] args) {
        final A a = new A();
        final B b = new B();

        new Thread(() -> a.methodA(b)).start();
        new Thread(() -> b.methodB(a)).start();
    }
}
```

- 上面的代码中，线程 A 持有锁 `A`，等待锁 `B`，而线程 B 持有锁 `B`，等待锁 `A`，这就是 **死锁**。

### ⚡ **死锁的检测与预防**

#### **死锁的检测**

- 可以通过检测线程状态以及锁的占用情况来检测死锁。一些高级的调试工具（如 `JProfiler`、`VisualVM`）可以帮助分析死锁问题。

#### **死锁的预防**

1. **避免循环等待**：对资源加锁时，按一定顺序进行，避免出现环状等待。
2. **使用 `tryLock()`**：使用 `ReentrantLock` 时，使用 `tryLock()` 方法来尝试获取锁，避免线程无限期等待。

#### 🧑‍💻 **死锁预防示例**

```java
ReentrantLock lockA = new ReentrantLock();
ReentrantLock lockB = new ReentrantLock();

if (lockA.tryLock() && lockB.tryLock()) {
    try {
        // 执行任务
    } finally {
        lockA.unlock();
        lockB.unlock();
    }
} else {
    // 失败后进行适当处理
}
```
