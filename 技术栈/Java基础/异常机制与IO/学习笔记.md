## **第一章：Java异常机制概述**

### 1. **异常的定义与分类**

#### 1.1 受检异常（Checked Exception）

- **定义**：受检异常是指程序员在编写代码时必须处理的异常，通常在编译时由编译器检查。若出现这种异常，必须通过 `try-catch` 语句进行捕获，或使用 `throws` 关键字声明抛出。
- **示例**：`IOException`, `SQLException` 等。
- **特性**：通常是程序中无法预料到的、外部环境相关的异常，如文件操作或数据库连接失败。

#### 1.2 未受检异常（Unchecked Exception）

- **定义**：未受检异常是指运行时异常，通常是程序的逻辑错误，编译器不会强制要求捕获。它们继承自 `RuntimeException` 类。
- **示例**：`NullPointerException`, `ArrayIndexOutOfBoundsException` 等。
- **特性**：通常是程序开发时的错误，如访问 null 对象、数组越界等。这些异常发生后通常表示程序需要修正。

#### 1.3 错误（Error）

- **定义**：错误表示一种严重的问题，通常是无法通过捕获和处理来解决的。它们继承自 `Error` 类。
- **示例**：`OutOfMemoryError`, `StackOverflowError` 等。
- **特性**：错误通常是 JVM 或系统环境的异常，如内存溢出或栈溢出。程序不能做任何有效处理，通常会导致程序崩溃。

### 2. **异常的基本概念**

#### 2.1 异常类层次结构

- **顶层类**：`Throwable` 类

  - **Error**：表示严重的系统错误。
  - **Exception**：表示程序中可处理的异常。
    - **Checked Exception**：需要显式处理或声明的异常。
    - **Unchecked Exception**：程序错误，不需要强制处理。

- **类图**：

  ```
  Throwable
    ├── Error
    └── Exception
         ├── IOException (checked)
         ├── SQLException (checked)
         └── RuntimeException (unchecked)
             ├── NullPointerException
             └── ArrayIndexOutOfBoundsException
  ```

#### 2.2 异常对象的创建与传递

- **创建异常对象**：
  - 通过 `new` 关键字创建异常对象。例如：`new IOException("File not found")`
- **异常的传递**：
  - 当一个方法抛出异常时，可以通过 `throws` 声明抛出异常，调用者可以捕获或继续传递。
  - 异常在调用栈中逐层向上抛出，直到被捕获或程序终止。

### 3. **异常的处理**

#### 3.1 `try-catch` 语句

- **语法**：

  ```java
  try {
      // 可能发生异常的代码
  } catch (ExceptionType1 e1) {
      // 异常处理代码
  } catch (ExceptionType2 e2) {
      // 异常处理代码
  }
  ```

- **流程**：

  - `try` 块中的代码会被执行，如果没有异常发生，`catch` 块会被跳过。
  - 如果 `try` 块抛出异常，程序控制流转到与该异常类型匹配的 `catch` 块中。
  - 如果没有匹配的 `catch` 块，异常会被传递给上层调用者，直到找到合适的处理方式。

#### 3.2 `finally` 语句

- **语法**：

  ```java
  try {
      // 可能发生异常的代码
  } catch (Exception e) {
      // 异常处理代码
  } finally {
      // 始终执行的代码
  }
  ```

- **特点**：

  - `finally` 块中的代码无论是否发生异常都会执行，常用于资源释放、关闭文件等操作。
  - 如果 `try` 或 `catch` 中有 `return` 语句，`finally` 仍然会执行。

#### 3.3 `throw` 和 `throws` 关键字

- **`throw`**：用于抛出一个异常。
  - 语法：`throw new ExceptionType("Exception Message");`
  - 例子：`throw new IOException("File not found");`
- **`throws`**：用于声明一个方法可能抛出的异常，提醒调用者需要进行处理。
  - 语法：`public void method() throws IOException, SQLException { }`
  - 例子：`public void readFile() throws IOException { }`

#### 3.4 异常链与多重捕获

- **异常链**：当一个异常引发另一个异常时，可以通过构造函数传递原始异常，从而创建异常链。

  - 例子：

    ```java
    try {
        // Some code that throws IOException
    } catch (IOException e) {
        throw new CustomException("Custom exception occurred", e);  // 原始异常e传递给新异常
    }
    ```

- **多重捕获**：Java 7 引入了多重异常捕获，允许在一个 `catch` 块中处理多种类型的异常。

  - 语法：`catch (IOException | SQLException e)`

  - 例子：

    ```java
    try {
        // code that might throw IOException or SQLException
    } catch (IOException | SQLException e) {
        // Handle both exceptions
    }
    ```

### 4. **自定义异常类**

#### 4.1 继承 `Exception` 或 `RuntimeException`

- **继承自 `Exception`**：

  - 用于创建**受检异常**。需要强制捕获或声明抛出。

  - 示例：

    ```java
    public class MyCheckedException extends Exception {
        public MyCheckedException(String message) {
            super(message);
        }
    }
    ```

- **继承自 `RuntimeException`**：

  - 用于创建**未受检异常**，一般用于表示程序的逻辑错误。

  - 示例：

    ```java
    public class MyUncheckedException extends RuntimeException {
        public MyUncheckedException(String message) {
            super(message);
        }
    }
    ```

#### 4.2 自定义异常的设计原则

- **明确的命名**：异常类名称应清楚描述错误的类型和上下文。
- **合理的构造函数**：提供带有错误消息和错误码的构造函数，以便于调试和日志记录。
- **文档化**：在类和方法的 Javadoc 中说明该异常的使用场景。
- **避免过度使用**：仅在必要时才创建自定义异常，避免滥用。
