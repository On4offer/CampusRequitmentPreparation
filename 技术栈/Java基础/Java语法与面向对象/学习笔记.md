## 🌱 1. Java 的诞生与发展

### 📖 历史背景

- Java 于 **1995 年**由 **Sun Microsystems** 公司推出（后被 Oracle 收购）。
- 设计初衷：为嵌入式设备（智能家电）开发平台无关的应用程序。
- 后被广泛用于 **Web 开发、企业级系统、移动端（Android）**、大数据、AI后端 等领域。

### 🧩 发展历程

| 版本    | 发布年份 | 特性                               |
| ------- | -------- | ---------------------------------- |
| JDK 1.0 | 1996     | 最初版本，支持基础语法             |
| JDK 1.2 | 1998     | 集合框架（Collections）            |
| JDK 1.5 | 2004     | 引入泛型、注解、枚举、自动装箱     |
| JDK 1.8 | 2014     | Lambda 表达式、Stream API、时间API |
| JDK 11  | 2018     | 长期支持版（LTS）                  |
| JDK 17  | 2021     | 最新 LTS，性能、安全性全面提升     |

### 💡 设计理念

> “Write Once, Run Anywhere”
>  一次编写，到处运行 —— 即 **跨平台特性**。

## ⚙️ 2. JDK、JRE、JVM 三者关系

### 🧱 三者定义

| 组件    | 全称                     | 作用                                                         |
| ------- | ------------------------ | ------------------------------------------------------------ |
| **JVM** | Java Virtual Machine     | 运行 Java 字节码的虚拟机，是 Java 跨平台的核心。             |
| **JRE** | Java Runtime Environment | Java 运行环境，包含 JVM + 核心类库。                         |
| **JDK** | Java Development Kit     | Java 开发工具包，包含 JRE + 编译、调试工具（javac、jar等）。 |

### 🔁 关系图

```
┌───────────────────────────┐
│           JDK             │
│  ┌───────────────────────┐│
│  │         JRE           ││
│  │ ┌────────────────────┐││
│  │ │        JVM         │││
│  │ └────────────────────┘││
│  └───────────────────────┘│
└───────────────────────────┘
```

👉 结论：**开发用 JDK，运行用 JRE，底层靠 JVM。**

## 🌍 3. Java 的跨平台原理（字节码与 JVM 机制）

### ⚙️ 工作流程

1. 源代码（.java）
    ↓ 编译
2. 字节码文件（.class）
    ↓ 由 JVM 执行
3. 不同操作系统运行不同版本的 JVM

### 🧬 跨平台关键

- Java 编译器生成的是 **字节码文件（.class）**，而非机器码；
- 各操作系统安装不同的 **JVM 实现**；
- JVM 将字节码 **解释/编译成当前系统可执行的机器码**。

### 🧠 补充：JIT 即时编译器

- JVM 内置 **JIT（Just-In-Time Compiler）**；
- 将热点代码动态编译为机器码，提升执行性能。

## 🧩 4. Java 程序的执行流程

### 🧭 流程图

```
源代码 (.java)
   ↓ javac 编译
字节码文件 (.class)
   ↓ JVM 类加载器（ClassLoader）
字节码验证与解释
   ↓
JIT 编译为机器码
   ↓
执行程序
```

### 🧠 关键步骤

1. **编译阶段**：`javac Hello.java` → 生成 Hello.class
2. **加载阶段**：JVM 类加载器（ClassLoader）加载字节码
3. **验证阶段**：校验安全性和正确性
4. **执行阶段**：解释/编译执行字节码

## 🧭 6. Java 的关键特性

| 特性           | 含义                          | 示例                        |
| -------------- | ----------------------------- | --------------------------- |
| **跨平台性**   | 一次编写，到处运行            | JVM屏蔽底层差异             |

💬 **面试问答**

> **问：为什么说 Java 是跨平台的？**
>  答：因为 Java 程序编译生成的是平台无关的字节码（.class），JVM 会在不同操作系统上解释执行该字节码，因此可实现一次编写，到处运行。

## 🌱 1. Java 程序的基本结构（类 → 方法 → 语句）

### 📖 结构概述

Java 程序由以下三大元素构成：

1. **类**：Java 程序的基本构建单元，所有功能代码都应在类中。
2. **方法**：类中的功能模块，通过方法定义行为。
3. **语句**：方法体内的执行指令，控制程序的流程。

### 🧩 示例代码

```java
public class Hello {
    // 主方法：程序入口
    public static void main(String[] args) {
        System.out.println("Hello, Java!");
    }
}
```

### 🔍 逐行解析：

- **类声明**：`public class Hello`
   定义一个公共类，类名为 `Hello`。
- **主方法**：`public static void main(String[] args)`
   程序执行的入口方法。
- **语句**：`System.out.println("Hello, Java!");`
   输出语句，打印字符串到控制台。

### ⚙️ 机制说明

- Java 程序运行时，JVM 会自动寻找 `main` 方法作为入口。

## 🔑 2. 标识符与关键字

### 🧩 标识符

标识符是 Java 中用于表示类、方法、变量、包等的名字。
 **规则**：

- 由字母、数字、下划线 `_`、美元符号 `$` 组成，且不能以数字开头。
- 标识符区分大小写，`myVariable` 与 `myvariable` 是不同的标识符。

### ⚠️ 关键字

Java 中有一组保留字，不能作为标识符使用。这些关键字具有特定的意义，用于语言的控制结构等。
 **常见关键字：** `public`, `private`, `class`, `int`, `if`, `else`, `for`, `while`, `return` 等。

### 💬 面试问答

> **问：Java 中关键字与标识符的区别是什么？**
>  **答：** 关键字是 Java 语言的保留字，具有特殊的意义，不能用作标识符。标识符则是程序员用来命名类、方法、变量等的名称，可以自定义，但不能与关键字冲突。

## 🧱 3. 变量与常量定义

### 🧩 变量

Java 变量用来存储数据。变量在使用前必须声明并指定类型。
 **基本语法：**

```java
数据类型 变量名 = 初始值;
```

### 💡 常量

常量是不可变的数据值，在定义时就被赋值，并且之后不能更改。常量使用 `final` 关键字定义。
 **常量定义：**

```java
final 数据类型 常量名 = 初始值;
```

### 🧰 示例代码

```java
public class Variables {
    public static void main(String[] args) {
        // 变量
        int num = 10;
        String name = "Java";
        
        // 常量
        final double PI = 3.14159;
        
        System.out.println(num);  // 输出：10
        System.out.println(name); // 输出：Java
        System.out.println(PI);   // 输出：3.14159
    }
}
```

### ⚙️ 机制说明

- 变量：存储可以变化的值，可以在程序运行过程中被修改。
- 常量：一旦赋值后不能再修改。

## 🔢 4. 数据类型体系（基本类型 + 引用类型）

### 🧩 基本数据类型

Java 中有 **8 种基本数据类型**，分别是：

- **整数类型**：`byte`（1字节）、`short`（2字节）、`int`（4字节）、`long`（8字节）
- **浮点类型**：`float`（4字节）、`double`（8字节）
- **字符类型**：`char`（2字节，用于存储单个字符）
- **布尔类型**：`boolean`（1字节，仅有 `true` 或 `false` 两个值）

### 🧩 引用数据类型

引用类型包括 **类、接口、数组** 等，它们存储的是对象的内存地址。

```java
String str = "Hello";  // 引用类型
```

## 🔄 5. 类型转换（自动类型提升与强制类型转换）

### 🧩 自动类型提升

Java 在进行运算时，会根据需要自动将低精度类型提升为高精度类型。
 例如：`int` 会自动转换为 `long` 或 `double`。

### 💡 强制类型转换

在某些情况下，Java 不会自动进行类型转换，此时需要显式地进行强制转换。

```java
int num = (int) 3.14;  // 强制转换为 int，结果是 3
```

## 📝 8. 注释与文档注释（Javadoc）

### 🧩 注释

注释是用来说明代码功能的非执行性语句，分为三种类型：

1. **单行注释**：`// 这是单行注释`
2. **多行注释**：`/* 这是多行注释 */`
3. **文档注释**：`/** 这是文档注释 */`

### 🧩 Javadoc

`Javadoc` 是 Java 提供的生成 API 文档的工具，它使用特定格式的注释来生成 HTML 格式的文档。

```java
    }
```

> **问：Java 中的 Javadoc 是什么？**

# 🧩 第三章：流程控制语句

> **目标：** 掌握顺序与代码块、if-else/switch 分支、for/while/do-while 循环、break/continue/return 及 for-each，并能做简单优化。

## 🌱 1. 顺序结构与代码块

### 📖 顺序结构

Java 程序的默认执行结构是 **顺序结构**，即从上到下依次执行代码。每一行代码按照书写顺序逐行执行，直到程序结束。

### 🧩 代码块

代码块是由一对大括号 `{}` 包围的代码区域，通常用于定义方法体、控制语句等。

```java
{
    // 代码块内的代码
}
```

代码块的作用：

- 限制变量的作用范围。
- 用于控制流程的范围，例如 `if`、`for` 等。

### 🧰 示例代码

```java
public class SequenceExample {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;
        
        // 顺序执行
        int sum = a + b;
        System.out.println("Sum: " + sum); // 输出：Sum: 30
    }
}
```

------

## 🔑 2. 分支语句：`if-else`、`switch-case`

### 🧩 `if-else` 语句

`if-else` 用于根据条件选择性执行代码块。如果条件为真，执行 `if` 块；如果为假，执行 `else` 块。

#### **语法**：

```java
if (condition) {
    // 条件为真时执行的代码
} else {
    // 条件为假时执行的代码
}
```

### 🧩 `switch-case` 语句

`switch-case` 用于根据变量的不同值执行不同的代码块，适合多个条件判断。

#### **语法**：

```java
switch (expression) {
    case value1:
        // 如果 expression == value1，执行此代码
        break;
    case value2:
        // 如果 expression == value2，执行此代码
        break;
    default:
        // 如果没有匹配到任何值，执行此代码
        break;
}
```

### 💬 面试问答

> **问：`switch` 语句和 `if-else` 的区别是什么？**
>  **答：** `switch` 适用于多重条件的判断，而 `if-else` 可以进行更复杂的逻辑判断。`switch` 更清晰、更高效，尤其在判断值较多时。

------

## 🔄 3. 循环语句：`for`、`while`、`do-while`

### 🧩 `for` 循环

`for` 循环用于已知执行次数的情况。通常用于遍历数组或集合。

#### **语法**：

```java
for (initialization; condition; increment) {
    // 执行的代码
}
```

- **initialization**：初始化语句，仅执行一次。
- **condition**：循环条件，条件为 `true` 时执行循环。
- **increment**：循环迭代操作。

### 🧩 `while` 循环

`while` 循环用于未知执行次数的情况。只要条件为 `true`，就会一直执行。

#### **语法**：

```java
while (condition) {
    // 执行的代码
}
```

### 🧩 `do-while` 循环

`do-while` 与 `while` 类似，但它保证至少执行一次循环体。

#### **语法**：

```java
do {
    // 执行的代码
} while (condition);
```

### 💬 面试问答

> **问：`while` 和 `do-while` 的区别是什么？**
>  **答：** `while` 循环在执行前判断条件，如果条件为 `false` 则不执行循环体；`do-while` 循环则是先执行一次循环体，再判断条件，保证至少执行一次。

### 🧰 示例代码

```java
// for 循环示例
for (int i = 0; i < 5; i++) {
    System.out.println("i: " + i);  // 输出 0 到 4
}

// while 循环示例
int j = 0;
while (j < 5) {
    System.out.println("j: " + j);  // 输出 0 到 4
    j++;
}

// do-while 循环示例
int k = 0;
do {
    System.out.println("k: " + k);  // 输出 0 到 4
    k++;
} while (k < 5);
```

------

## 🔁 4. 跳转语句：`break`、`continue`、`return`

### 🧩 `break` 语句

`break` 用于退出循环或 `switch` 语句。常用于提前终止循环或跳出 `switch` 块。

#### **示例代码**：

```java
for (int i = 0; i < 10; i++) {
    if (i == 5) {
        break;  // 退出循环
    }
    System.out.println(i);  // 输出：0 1 2 3 4
}
```

### 🧩 `continue` 语句

`continue` 用于跳过当前循环的剩余部分，直接进入下次循环。

#### **示例代码**：

```java
for (int i = 0; i < 10; i++) {
    if (i == 5) {
        continue;  // 跳过数字 5
    }
    System.out.println(i);  // 输出：0 1 2 3 4 6 7 8 9
}
```

### 🧩 `return` 语句

`return` 用于终止当前方法的执行并返回一个值。

#### **示例代码**：

```java
public static int sum(int a, int b) {
    return a + b;  // 直接返回结果并结束方法执行
}
```

### 💬 面试问答

> **问：`return`、`break`、`continue` 的区别是什么？**
>  **答：**
>
> - `return`：用于结束方法的执行并返回值。
> - `break`：用于退出循环或 `switch` 语句。
> - `continue`：用于跳过当前循环的剩余部分，进入下次循环。

## 🧩 5. 增强 `for` 循环（for-each）

### 📖 介绍

增强 `for` 循环（`for-each`）是 Java 中用于遍历数组、集合等可迭代对象的简化方式。

#### **语法**：

```java
for (元素类型 元素 : 数组/集合) {
    // 执行的代码
}
```

### 🧰 示例代码

```java
String[] colors = {"Red", "Green", "Blue"};
for (String color : colors) {
    System.out.println(color);  // 输出：Red Green Blue
}
```

### 💬 面试问答

> **问：`for-each` 循环的优点是什么？**
>  **答：** `for-each` 循环简化了数组和集合的遍历，避免了使用下标的麻烦，增强了代码的可读性。

## 🔄 6. 流程控制的嵌套与优化技巧

### 🧩 流程控制嵌套

Java 允许在一个控制结构中嵌套其他控制结构，例如嵌套的 `if`、`for` 或者 `while` 循环。虽然嵌套可以解决复杂问题，但嵌套过深会降低代码可读性与性能。

#### **示例代码**：

```java
for (int i = 0; i < 5; i++) {
    for (int j = 0; j < 5; j++) {
        System.out.println("i: " + i + ", j: " + j);
    }
}
```

### ⚙️ 优化技巧

1. **减少嵌套层次**：尽量避免多层嵌套，考虑将条件判断提取为单独的方法。
2. **使用早期返回（early return）**：通过 `return` 语句提前结束不必要的循环或方法，减少嵌套深度。
3. **优化循环条件**：合并条件判断，减少循环内重复计算。

## 🌱 1. 一维数组与二维数组的声明与初始化

### 🧩 一维数组

数组是相同类型元素的集合。在 Java 中，数组是对象，存储在堆内存中。
 **声明与初始化**：

```java
// 声明并初始化一维数组
int[] arr1 = new int[5];          // 定义一个包含 5 个元素的 int 数组
int[] arr2 = {1, 2, 3, 4, 5};     // 定义并直接初始化一维数组
```

### 🧩 二维数组

 **声明与初始化**：

```java
// 声明并初始化二维数组
```

## 🧱 2. 数组的内存分布与引用机制

### 🧩 内存分布

在 Java 中，数组对象的内存分布是基于 **堆内存** 的，数组的元素存储在连续的内存位置。

- 数组对象的 **引用** 存储在栈中。
- 数组的 **内容**（元素）存储在堆中。

### 🧩 引用机制

- 数组是 **引用类型**，所以赋值数组时是复制引用而非内容。
- 例如，`array1 = array2` 只是让 `array1` 指向 `array2` 所指向的内存地址，改变 `array1` 会影响 `array2`。

### 🧰 示例代码

```java
int[] arr1 = {1, 2, 3};
int[] arr2 = arr1; // arr2 与 arr1 引用同一数组
arr2[0] = 100;     // arr1[0] 也变为 100
System.out.println(arr1[0]); // 输出：100
```

## ⚙️ 3. Arrays 工具类的常用方法

### 🧩 `Arrays` 类

`Arrays` 是 Java 提供的一个工具类，位于 `java.util` 包中，提供了操作数组的常用方法。
 **常用方法**：

- `Arrays.toString(arr)`：将数组转换为字符串。
- `Arrays.sort(arr)`：对数组进行排序。
- `Arrays.copyOf(arr, newLength)`：复制数组并更改其长度。
- `Arrays.equals(arr1, arr2)`：比较两个数组是否相等。

### 🧰 示例代码

```java
import java.util.Arrays;

public class ArraysExample {
    public static void main(String[] args) {
        int[] arr = {4, 2, 9, 1, 7};
        
        // 排序
        Arrays.sort(arr);
        System.out.println(Arrays.toString(arr));  // 输出：[1, 2, 4, 7, 9]
        
        // 数组复制
        int[] arrCopy = Arrays.copyOf(arr, 7);  // 长度为 7
        System.out.println(Arrays.toString(arrCopy));  // 输出：[1, 2, 4, 7, 9, 0, 0]
        
        // 比较数组
        int[] arr2 = {1, 2, 4, 7, 9};
        System.out.println(Arrays.equals(arr, arr2));  // 输出：true
    }
}
```

## 🧩 4. 字符串类 `String`、`StringBuilder`、`StringBuffer` 的区别与使用

### 🧩 `String` 类

- **不可变**：`String` 是不可变类，每次对字符串的修改都会创建新的 `String` 对象。
- **性能问题**：由于不可变性，频繁修改字符串时会产生大量临时对象，效率较低。

### 🧩 `StringBuilder` 类

- **可变**：`StringBuilder` 是一个可变的字符序列，适合于字符串的频繁修改。
- **性能较好**：比 `String` 更高效，尤其是在需要进行大量字符串操作时。

### 🧩 `StringBuffer` 类

- **线程安全**：`StringBuffer` 与 `StringBuilder` 类似，区别在于它是线程安全的，因此性能较差。
- **适用于多线程环境**：在多线程环境中操作字符串时，使用 `StringBuffer` 更安全。

### 💬 面试问答

> **问：`String`、`StringBuilder`、`StringBuffer` 有何区别？**
>  **答：** `String` 是不可变的，适合于少量修改。`StringBuilder` 用于单线程场景，性能更高。`StringBuffer` 与 `StringBuilder` 类似，但它是线程安全的，适用于多线程环境。

### 🧰 示例代码

```java
public class StringExample {
    public static void main(String[] args) {
        // String 示例
        String str1 = "Hello";
        str1 = str1 + " World";  // 创建新字符串

        // StringBuilder 示例
        StringBuilder sb = new StringBuilder("Hello");
        sb.append(" World");  // 修改原字符串

        // StringBuffer 示例
        StringBuffer sbf = new StringBuffer("Hello");
        sbf.append(" World");  // 线程安全的修改
    }
}
```

## 🧩 5. 字符串常量池与不可变性原理

### 🧩 字符串常量池

- **概念**：Java 中的 `String` 是不可变的，所有的字符串字面量（例如 `"Hello"`）都会存储在常量池中。
- **优化**：使用常量池避免了重复创建相同内容的字符串对象，从而节省内存。

### 🧩 不可变性原理

- `String` 对象的不可变性是通过 `final` 关键字实现的。一旦创建了一个 `String` 对象，它的内容无法改变。
- 每次修改字符串时，都会创建一个新的 `String` 对象。

### 💬 面试问答

> **问：为什么 `String` 是不可变的？**
>  **答：** `String` 是不可变的，主要是为了确保安全性、哈希值缓存和性能优化。不可变性可以保证 `String` 对象在多线程环境中的安全性。

## 🧩 6. 数组与字符串的常见面试题（反转、去重、统计等）

### 🧩 反转数组

反转数组是常见的面试题，要求将数组的元素顺序颠倒。

#### **示例代码**：

```java
public class ReverseArray {
    public static void main(String[] args) {
        int[] arr = {1, 2, 3, 4, 5};
        for (int i = 0; i < arr.length / 2; i++) {
            int temp = arr[i];
            arr[i] = arr[arr.length - 1 - i];
            arr[arr.length - 1 - i] = temp;
        }
        System.out.println(Arrays.toString(arr));  // 输出：[5, 4, 3, 2, 1]
    }
}
```

### 🧩 字符串去重

要求去除字符串中的重复字符，只保留第一次出现的字符。

#### **示例代码**：

```java
public class RemoveDuplicateChars {
    public static void main(String[] args) {
        String str = "aabbcc";
        StringBuilder result = new StringBuilder();
        for (int i = 0; i < str.length(); i++) {
            if (result.indexOf(String.valueOf(str.charAt(i))) == -1) {
                result.append(str.charAt(i));
            }
        }
        System.out.println(result.toString());  // 输出：abc
    }
}
```

### 🧩 字符串统计

统计字符串中每个字符的出现次数。

#### **示例代码**：

```java
public class CountChars {
    public static void main(String[] args) {
        String str = "abcaabbcc";
        Map<Character, Integer> countMap = new HashMap<>();
        for (char c : str.toCharArray()) {
            countMap.put(c, countMap.getOrDefault(c, 0) + 1);
        }
        System.out.println(countMap);  // 输出：{a=4, b=3, c=2}
    }
}
```

## 🌱 1. 方法的定义与调用

### 📖 方法概述

在 Java 中，方法是定义一段代码逻辑并可以多次调用的功能单元。方法由返回类型、方法名、参数列表以及方法体组成。

### 🧩 方法定义

**基本语法：**

```java
返回类型 方法名(参数类型 参数名) {
    // 方法体
    return 返回值;
}
```

### 🧩 方法调用

方法调用时需要提供符合方法签名的参数，并在方法体内部执行代码。

#### **示例代码**：

```java
public class MethodExample {
    // 方法定义
    public static int add(int a, int b) {
        return a + b;
    }

    public static void main(String[] args) {
        // 方法调用

}
```

### 💬 面试问答

> **问：方法的定义和调用如何匹配？**
>  **答：** 方法定义时指定了返回类型和参数类型，在调用时根据定义的签名传入相应的参数。

## 🔑 2. 方法的参数与返回值

### 🧩 方法参数

```java
// 单个参数
```

## 🧩 3. 方法重载（Overloading）

### 📖 什么是方法重载？

方法重载是指在同一类中，允许定义多个名称相同、但参数不同（类型或个数不同）的方法。编译器根据方法调用时提供的参数类型和个数来确定使用哪个重载版本。

### 🧩 语法

```java
public class OverloadingExample {
    // 方法重载：相同方法名，参数个数不同
    public static void print(int a) {
        System.out.println("Integer: " + a);
    }

    public static void print(String s) {
        System.out.println("String: " + s);
    }

    public static void main(String[] args) {
        print(10);     // 调用 print(int a)
        print("Hello"); // 调用 print(String s)
    }
}
```

### 💬 面试问答

> **问：Java 中方法重载的规则是什么？**
>  **答：** 方法重载要求方法名相同，但参数个数或类型必须不同。返回类型不能作为重载的依据。

### 📖 可变参数

## 🧩 5. Java 的参数传递机制（值传递 vs 引用传递）

### 📖 参数传递机制

Java 中所有方法的参数传递都是 **值传递**，即传递的是变量的副本。

- **基本数据类型**：传递的是值的副本，修改副本不会影响原变量。
- **引用数据类型**：传递的是对象的引用（即内存地址的副本），修改引用指向的对象会影响原对象，但修改引用本身不会改变原始引用。

#### **示例代码：值传递**

```java
public class ValuePassingExample {
    public static void modifyValue(int x) {
        x = 100;
    }

    public static void main(String[] args) {
        int a = 10;
        modifyValue(a);
        System.out.println(a);  // 输出：10
    }
}
```

#### **示例代码：引用传递**

```java
public class ReferencePassingExample {
    public static void modifyArray(int[] arr) {
        arr[0] = 100;
    }

    public static void main(String[] args) {
        int[] nums = {1, 2, 3};
        modifyArray(nums);
        System.out.println(nums[0]);  // 输出：100
    }
}
```

### 💬 面试问答

> **问：Java 是值传递还是引用传递？**
>  **答：** Java 是值传递。对于基本数据类型，传递的是值的副本；对于对象（引用类型），传递的是引用的副本。

## 🔄 6. 递归方法的实现与优化

### 📖 递归概念

递归是指方法调用自身来解决问题。每个递归问题都应该有明确的结束条件（基准条件）。

### 🧩 递归方法实现

#### **示例代码：计算阶乘**

```java
public class RecursionExample {
    public static int factorial(int n) {
        if (n == 0) {
            return 1;  // 基准条件
        }
        return n * factorial(n - 1);  // 递归调用
    }

    public static void main(String[] args) {
        System.out.println(factorial(5));  // 输出：120
    }
}
```

### ⚙️ 递归优化（尾递归）

尾递归是指递归调用出现在方法的最后一个操作，尾递归可以通过优化避免堆栈溢出。

#### **示例代码：尾递归优化**

```java
public class TailRecursionExample {
    public static int factorialTailRecursive(int n, int result) {
        if (n == 0) {
            return result;  // 基准条件
        }
        return factorialTailRecursive(n - 1, n * result);  // 递归调用
    }

    public static void main(String[] args) {
        System.out.println(factorialTailRecursive(5, 1));  // 输出：120
    }
}
```

### 💬 面试问答

> **问：递归方法如何优化？**
>  **答：** 通过尾递归优化，可以避免堆栈溢出，减少内存使用。

## 🧩 7. main 方法与静态上下文

### 📖 main 方法

`main` 方法是 Java 程序的入口点，JVM 从 `main` 方法开始执行程序。`main` 方法必须是 `public static void` 类型的。

#### **语法**：

```java
public static void main(String[] args) {
    // 程序的入口
}
```

### 🧩 静态上下文

- `static` 关键字表示类级别的成员，属于类而非对象。
- 静态方法不能访问实例变量和实例方法，但可以访问静态变量和静态方法。（实例方法可以访问静态与实例变量）

#### **示例代码**：

```java
public class StaticExample {
    static int counter = 0;  // 静态变量

    public static void increment() {  // 静态方法
        counter++;
    }

    public static void main(String[] args) {
        increment();  // 直接调用静态方法
        System.out.println(counter);  // 输出：1
    }
}
```

### 💬 面试问答

> **问：`static` 关键字的作用是什么？**
>  **答：** `static` 修饰符表示该成员属于类本身，而不是类的实例。静态成员可以在没有创建对象的情况下访问。

# 🧩 第六章：面向对象编程（OOP）核心概念

## 🌱 1. 面向对象思想与面向过程对比

### 📖 面向对象（OOP）

面向对象编程（OOP）是一种以对象为核心的编程思想。在 OOP 中，程序通过定义类来创建对象，并通过对象的交互来解决问题。
 OOP 的核心特性包括 **封装**、**继承**、**多态** 和 **抽象**。

### 🧩 面向过程

面向过程编程（POP）关注的是“做什么”，强调程序的过程和执行顺序，适用于较为简单的任务。
 **缺点**：代码重用性差、维护性差，难以扩展。

### 🔄 面向对象 vs 面向过程

| 特性       | 面向过程（POP） | 面向对象（OOP）        |
| ---------- | --------------- | ---------------------- |
| 编程方式   | 以过程为中心    | 以对象为中心           |
| 重点       | 功能的实现      | 对象的状态和行为       |
| 可扩展性   | 差              | 强                     |
| 代码复用性 | 低              | 高                     |
| 适用场景   | 简单任务        | 复杂系统，需求变化频繁 |

## 🔑 2. 类（Class）与对象（Object）

### 📖 类（Class）

类是对象的模板或蓝图，定义了对象的属性（成员变量）和行为（成员方法）。类是对象的抽象。

**类的声明：**

```java
public class Car {

    }
}
```

### 🧩 对象（Object）

对象是类的实例，拥有类中定义的属性和行为。对象通过 `new` 关键字创建。

**对象的创建：**

```java
Car myCar = new Car();  // 创建 Car 类的对象
myCar.color = "Red";     // 设置属性
myCar.accelerate();      // 调用方法
```

## 🧩 3. 属性（成员变量）与方法（成员函数）

### 📖 属性（成员变量）

属性是类中定义的变量，用于描述对象的状态。每个对象拥有独立的属性。

### 📖 方法（成员函数）

方法是类中的函数，用于描述对象的行为。对象可以通过调用方法来执行某些操作。

**示例代码：**


## 🔄 4. 封装（Encapsulation）

### 📖 封装概念

封装是面向对象编程的核心思想之一，它指将数据（属性）和操作数据的方法封装在一起，并通过访问控制来限制外部对数据的直接访问。

### 🧩 访问控制

Java 提供了 `public`、`private`、`protected` 等访问修饰符来控制属性和方法的可见性。

- **`private`**：只能在类内部访问。
- **`public`**：可以在类外部访问。
- **`protected`**：可以在同一包或子类中访问。

### 🧰 示例代码

```java
public class BankAccount {

    public void deposit(double amount) {  // 公共方法

    }

    public double getBalance() {  // 公共方法
        return balance;
    }
}
```

### 💬 面试问答

> **问：为什么要使用封装？**

## 🔄 5. 构造方法与构造器重载

### 📖 构造方法

构造方法用于初始化新创建的对象。构造方法与类同名，并且没有返回类型。构造方法在创建对象时自动调用。

构造方法也可以进行重载，以适应不同的对象创建需求。

#### **示例代码：**

```java

Person person2 = new Person("Alice", 25);  // 使用带参数构造方法
```

### 💬 面试问答

> **问：构造方法与普通方法有何区别？**

## 🧩 6. `this` 关键字的用法与原理

### 📖 `this` 关键字

`this` 关键字指代当前对象的引用，用于区分实例变量和局部变量、调用当前对象的其他构造方法等。

#### **常见用法**：

1. 区分实例变量和局部变量。
2. 调用当前类的其他构造方法。

```java
public class Car {

    }
}
```

### 💬 面试问答

## 🔄 7. Java 内存模型中对象的存储位置

### 📖 Java 内存模型

Java 程序中的对象存储在 **堆内存** 中，而栈内存用于存储局部变量（如方法中的参数和局部变量）。

- **堆内存**：用于存储对象和数组，所有的对象在堆中创建。
- **栈内存**：用于存储方法调用时的局部变量和方法调用栈。

### 🧩 对象的创建与内存分配

### 💬 面试问答

> **问：对象是如何存储的？**
>  **答：** 对象存储在堆内存中，而对象的引用存储在栈内存中。栈中存储的引用指向堆中的对象。

## 🌱 1. 继承（`extends`）的基本语法与意义

### 📖 继承的概念

继承是面向对象编程中的一种机制，它允许一个类（子类）继承另一个类（父类）的属性和方法，从而实现代码的复用和扩展。通过继承，子类不仅可以拥有父类的成员，还可以对其进行扩展或重写。

### 🧩 继承的基本语法

```java
class 子类 extends 父类 {
    // 子类可以访问父类的非私有成员
}
```

#### **示例代码**：

```java
class Animal {
    String name;

    public void eat() {
        System.out.println("Animal is eating");
    }
}

class Dog extends Animal {
    public void bark() {
        System.out.println("Dog is barking");
    }
}

public class InheritanceExample {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.eat();   // 继承自父类
        dog.bark();  // 自己的方法
    }
}
```

### 💬 面试问答

> **问：Java 中继承的优点是什么？**
>  **答：** 继承实现了代码的复用、提高了开发效率，并且能够构建类之间的层次结构。

## 🔑 2. 方法重写（`Override`）与隐藏

### 📖 方法重写（Override）

方法重写是子类重新定义父类中已有的方法，以便修改或扩展父类的行为。方法重写时，子类的方法必须与父类的方法签名完全一致。

#### **示例代码**：

```java
class Animal {
    public void sound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    @Override
    public void sound() {
        System.out.println("Dog barks");
    }
}

public class MethodOverrideExample {
    public static void main(String[] args) {
        Animal animal = new Dog();
        animal.sound();  // 输出：Dog barks
    }
}
```

### 📖 方法隐藏（Hiding）

方法隐藏发生在父类和子类中都有相同名称和参数的方法时，且在子类方法上没有使用 `@Override` 注解。此时，父类的方法不会被子类的同名方法覆盖，方法的隐藏由方法的调用对象决定。

#### **示例代码**：

```java
class Animal {
    public static void staticMethod() {
        System.out.println("Animal static method");
    }
}

class Dog extends Animal {
    public static void staticMethod() {
        System.out.println("Dog static method");
    }
}

public class MethodHidingExample {
    public static void main(String[] args) {
        Animal.staticMethod();  // 输出：Animal static method
        Dog.staticMethod();     // 输出：Dog static method
    }
}
```

### 💬 面试问答

> **问：方法重写和方法隐藏有什么区别？**
>  **答：** 方法重写是子类覆盖父类的实例方法，而方法隐藏是子类静态方法与父类静态方法同名时的情况。

## 🔄 3. `super` 关键字的作用

### 📖 `super` 的概念

`super` 关键字用于引用父类的成员变量和方法，也可以用来调用父类的构造方法。

#### **常见用法**：

1. **调用父类的方法**：`super.method()`
2. **访问父类的构造方法**：`super()`
3. **访问父类的属性**：`super.attribute`

#### **示例代码**：

```java
class Animal {
    String name;

    public Animal(String name) {
        this.name = name;
    }

    public void sound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    public Dog(String name) {
        super(name);  // 调用父类构造方法
    }

    public void sound() {
        super.sound();  // 调用父类方法
        System.out.println("Dog barks");
    }
}

public class SuperExample {
    public static void main(String[] args) {
        Dog dog = new Dog("Buddy");
        dog.sound();  // 输出：Animal makes a sound \n Dog barks
    }
}
```

### 💬 面试问答

> **问：`super` 关键字可以做什么？**
>  **答：** `super` 用于访问父类的构造方法、成员变量和方法。它也可以用于调用父类的构造方法初始化对象。

## 🔄 4. `Object` 类与常用方法（`equals`、`hashCode`、`toString`）

### 📖 `Object` 类

`Object` 是所有类的父类，所有的 Java 类都直接或间接继承自 `Object` 类，提供了一些常用的方法。

#### **常用方法**：

- **`equals()`**：比较两个对象是否相等（对象的内容）。
- **`hashCode()`**：返回对象的哈希码。
- **`toString()`**：返回对象的字符串表示。

#### **示例代码**：

```java
class Person {
    String name;
    int age;

    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (obj == null || getClass() != obj.getClass()) return false;
        Person person = (Person) obj;
        return age == person.age && name.equals(person.name);
    }

    @Override
    public int hashCode() {
        return name.hashCode() + age;
    }

    @Override
    public String toString() {
        return "Person{name='" + name + "', age=" + age + "}";
    }
}

public class ObjectMethodsExample {
    public static void main(String[] args) {
        Person person1 = new Person("Alice", 25);
        Person person2 = new Person("Alice", 25);

        System.out.println(person1.equals(person2));  // 输出：true
        System.out.println(person1.hashCode());        // 输出哈希码
        System.out.println(person1.toString());       // 输出：Person{name='Alice', age=25}
    }
}
```

### 💬 面试问答

> **问：`equals()` 和 `hashCode()` 的关系是什么？**
>  **答：** 根据 `hashCode()` 方法的定义，相同的对象应该有相同的哈希码。`equals()` 方法用于比较两个对象是否相等，而 `hashCode()` 方法用于为对象生成哈希值，在哈希存储（如 `HashMap`）中非常重要（可能哈希碰撞）。

## 🔄 5. 多态（Polymorphism）的原理与调用机制

### 📖 多态概述

多态是指同一操作作用于不同类型的对象时，表现出不同的行为。多态分为两种类型：

1. **编译时多态**：方法重载（Overloading）。
2. **运行时多态**：方法重写（Override）。

### 🧩 运行时多态

在 Java 中，**运行时多态**指的是父类引用指向子类对象时，调用的方法是子类重写的方法。

#### **示例代码**：

```java
class Animal {
    public void sound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    @Override
    public void sound() {
        System.out.println("Dog barks");
    }
}

public class PolymorphismExample {
    public static void main(String[] args) {
        Animal animal = new Dog();
        animal.sound();  // 输出：Dog barks
    }
}
```

### 💬 面试问答

> **问：什么是多态？**
>  **答：** 多态是指同一个方法调用具有不同的表现，依据对象的实际类型来决定调用哪个方法。通过继承和方法重写实现。

## 🔄 6. 向上转型与向下转型

### 📖 向上转型（Upcasting）

向上转型是将子类对象赋值给父类引用变量，通常发生在继承关系中。向上转型是安全的，编译时能通过。

#### **示例代码**：

```java
Animal animal = new Dog();  // 向上转型
```

### 📖 向下转型（Downcasting）

向下转型是将父类引用转换为子类引用。在进行向下转型时，必须确保对象的实际类型与目标类型一致，否则会抛出 `ClassCastException`。

#### **示例代码**：

```java
Animal animal = new Dog();
Dog dog = (Dog) animal;  // 向下转型
```

### 💬 面试问答

> **问：什么是向上转型和向下转型？**
>  **答：** 向上转型是将子类对象赋给父类引用；向下转型是将父类引用转为子类引用，但需要保证类型一致。

## 🔄 7. 抽象类（`abstract`）与模板设计思想

### 📖 抽象类概述

抽象类是不能实例化的类，通常用于定义一组子类共享的通用接口和行为。抽象类可以包含抽象方法和具体方法。

#### **示例代码**：

```java
abstract class Animal {
    abstract void sound();  // 抽象方法

    public void sleep() {
        System.out.println("Animal sleeps");
    }
}

class Dog extends Animal {
    @Override
    public void sound() {
        System.out.println("Dog barks");
    }
}

public class AbstractClassExample {
    public static void main(String[] args) {
        Animal dog = new Dog();
        dog.sound();  // 输出：Dog barks
        dog.sleep();  // 输出：Animal sleeps
    }
}
```

### 🧩 模板设计模式

模板设计模式通过定义一个方法的执行框架，允许子类实现具体步骤。抽象类提供了模板方法，具体实现由子类完成。

## 🌱 1. 接口的定义与实现

### 📖 接口的概念

接口是 Java 中一种特殊的引用类型，定义了一组抽象方法和常量。它规定了类应该做什么，但不指定具体的实现方式。类通过实现接口来遵循接口定义的契约。

### 🧩 接口的定义

接口的定义使用 `interface` 关键字，并且只能包含常量和抽象方法（Java 8 之后可以有默认方法和静态方法）。

#### **示例代码：接口定义与实现**

```java
interface Animal {
    void sound();  // 抽象方法，不包含方法体
}

class Dog implements Animal {
    @Override
    public void sound() {
        System.out.println("Dog barks");
    }
}

public class InterfaceExample {
    public static void main(String[] args) {
        Animal dog = new Dog();
        dog.sound();  // 输出：Dog barks
    }
}
```

### 💬 面试问答

> **问：Java 中接口的作用是什么？**
>  **答：** 接口定义了类的行为规范，类实现接口后可以保证符合该行为规范，从而实现解耦和多态。

------

## 🔑 2. 接口与抽象类的区别

### 📖 接口与抽象类的区别

- **接口**：不能包含任何实现（除非是默认方法或静态方法），只能定义抽象方法，支持多重继承。
- **抽象类**：可以包含方法实现和抽象方法，支持继承，且一个类只能继承一个抽象类。

| 特性     | 接口                          | 抽象类                                   |
| -------- | ----------------------------- | ---------------------------------------- |
| 继承关系 | 一个类可以实现多个接口        | 一个类只能继承一个抽象类                 |
| 方法实现 | 默认方法（Java 8+）可以有实现 | 可以有实现，也可以有抽象方法             |
| 变量     | 默认是 `public static final`  | 可以是各种访问权限，非 `static`          |
| 访问权限 | 所有成员默认为 `public`       | 可以有 `private`、`protected` 等访问权限 |

#### **示例代码：接口与抽象类对比**

```java
// 接口
interface Animal {
    void sound();
}

// 抽象类
abstract class AnimalClass {
    abstract void sound();  // 抽象方法

    public void sleep() {
        System.out.println("Animal sleeps");
    }
}

class Dog implements Animal {
    public void sound() {
        System.out.println("Dog barks");
    }
}

class Cat extends AnimalClass {
    public void sound() {
        System.out.println("Cat meows");
    }
}

public class InterfaceVsAbstractClass {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.sound();  // 输出：Dog barks

        Cat cat = new Cat();
        cat.sound();  // 输出：Cat meows
        cat.sleep();  // 输出：Animal sleeps
    }
}
```

### 💬 面试问答

> **问：接口与抽象类有何区别，什么时候用接口，什么时候用抽象类？**
>  **答：** 接口用于定义行为规范，适用于多个类共享相同行为的场景；抽象类用于实现类之间的共享代码，适用于有共同部分的类。

## 🔄 3. 多继承冲突与默认方法（Java 8）

### 📖 多继承冲突

在 Java 中，接口支持多继承，但当多个接口定义了同名的默认方法时，会发生冲突。这时，类需要显式地重写冲突的方法。

#### **示例代码：多继承冲突**

```java
interface Animal {
    default void sound() {
        System.out.println("Animal makes a sound");
    }
}

interface Dog {
    default void sound() {
        System.out.println("Dog barks");
    }
}

class AnimalDog implements Animal, Dog {
    @Override
    public void sound() {
        // 解决冲突，选择一个方法
        Dog.super.sound();
    }
}

public class MultipleInheritanceConflict {
    public static void main(String[] args) {
        AnimalDog animalDog = new AnimalDog();
        animalDog.sound();  // 输出：Dog barks
    }
}
```

### 💬 面试问答

> **问：Java 中如何解决接口多继承的冲突？**
>  **答：** Java 8 引入了默认方法（`default`），如果接口有相同的默认方法，类需要显式重写该方法并决定调用哪个方法。

------

## 🔄 4. 函数式接口与 Lambda 表达式（Java 8）

### 📖 函数式接口

函数式接口是指仅包含一个抽象方法的接口。Java 8 引入了函数式接口的概念，可以通过 `@FunctionalInterface` 注解来标识。

#### **示例代码：函数式接口**

```java
@FunctionalInterface
interface Calculator {
    int add(int a, int b);
}

public class FunctionalInterfaceExample {
    public static void main(String[] args) {
        // 使用 Lambda 表达式实现接口
        Calculator calculator = (a, b) -> a + b;
        System.out.println(calculator.add(5, 3));  // 输出：8
    }
}
```

### 📖 Lambda 表达式

Lambda 表达式是 Java 8 引入的一种简洁的函数表示方式，可以用于实现函数式接口。

#### **示例代码：Lambda 表达式**

```java
interface Greeting {
    void greet(String name);
}

public class LambdaExample {
    public static void main(String[] args) {
        // Lambda 表达式实现接口
        Greeting greet = (name) -> System.out.println("Hello, " + name);
        greet.greet("Alice");  // 输出：Hello, Alice
    }
}
```

### 💬 面试问答

> **问：什么是函数式接口，如何在 Java 中使用 Lambda 表达式？**
>  **答：** 函数式接口是只有一个抽象方法的接口，可以通过 `@FunctionalInterface` 注解标记。Lambda 表达式用于简化函数式接口的实现，使用 `->` 符号。

## 🔄 5. 接口的设计原则与常见应用（策略模式、回调机制）

### 📖 接口的设计原则

1. **接口优于继承**：使用接口可以避免类之间的紧耦合，提升可扩展性。
2. **面向接口编程**：通过接口实现多态，降低模块间的耦合度。
3. **高内聚，低耦合**：接口可以实现功能模块的解耦，提高代码的可维护性。

### 🧩 常见应用：策略模式

策略模式是一种行为型设计模式，它定义了一系列算法，并将每个算法封装起来，使它们可以互相替换。策略模式让算法独立于使用它的客户端变化。

#### **示例代码：策略模式**

```java
interface Strategy {
    int execute(int a, int b);
}

class Add implements Strategy {
    public int execute(int a, int b) {
        return a + b;
    }
}

class Subtract implements Strategy {
    public int execute(int a, int b) {
        return a - b;
    }
}

class Context {
    private Strategy strategy;

    public Context(Strategy strategy) {
        this.strategy = strategy;
    }

    public int executeStrategy(int a, int b) {
        return strategy.execute(a, b);
    }
}

public class StrategyPatternExample {
    public static void main(String[] args) {
        Context context = new Context(new Add());
        System.out.println("Addition: " + context.executeStrategy(5, 3));  // 输出：Addition: 8

        context = new Context(new Subtract());
        System.out.println("Subtraction: " + context.executeStrategy(5, 3));  // 输出：Subtraction: 2
    }
}
```

### 🧩 常见应用：回调机制

回调机制是将一个方法作为参数传递给另一个方法，待特定事件发生时，调用传入的方法。

#### **示例代码：回调机制**

```java
interface Callback {
    void call();
}

class Task {
    public void performTask(Callback callback) {
        System.out.println("Performing task...");
        callback.call();  // 执行回调
    }
}

public class CallbackExample {
    public static void main(String[] args) {
        Task task = new Task();
        task.performTask(() -> System.out.println("Task completed!"));
    }
}
```

### 💬 面试问答

> **问：策略模式和回调机制有什么区别？**
>  **答：** 策略模式用于将算法封装成类，使得算法可以互相替换；回调机制是通过接口将方法作为参数传递，允许方法在适当时机调用回调函数。

# 🧩 第九章：内部类与枚举

> **目标：** 理解成员/静态/局部/匿名内部类及枚举的用法，枚举实现单例与状态机等场景。

------

## 🌱 1. 成员内部类、静态内部类、局部内部类、匿名内部类

### 📖 内部类的概念

内部类是定义在另一个类中的类，主要用于表示类与类之间的紧密关联。内部类可以访问外部类的成员，包括私有成员。

### 🧩 成员内部类

成员内部类是定义在外部类中的类，通常作为外部类的一部分。

#### **示例代码：成员内部类**

```java
class Outer {
    private String outerField = "Outer field";

    class Inner {
        void display() {
            System.out.println(outerField);  // 访问外部类的私有成员
        }
    }
}

public class InnerClassExample {
    public static void main(String[] args) {
        Outer outer = new Outer();
        Outer.Inner inner = outer.new Inner();  // 创建内部类对象
        inner.display();  // 输出：Outer field
    }
}
```

### 🧩 静态内部类

静态内部类是定义为静态的类。静态内部类与外部类的实例无关，可以直接通过外部类的类名来创建。

#### **示例代码：静态内部类**

```java
class Outer {
    private static String outerField = "Static Outer field";

    static class Inner {
        void display() {
            System.out.println(outerField);  // 访问外部类的静态成员
        }
    }
}

public class StaticInnerClassExample {
    public static void main(String[] args) {
        Outer.Inner inner = new Outer.Inner();  // 静态内部类不需要外部类实例
        inner.display();  // 输出：Static Outer field
    }
}
```

### 🧩 局部内部类

局部内部类是定义在方法内部的类，通常用于临时的任务，不需要暴露给外部。

#### **示例代码：局部内部类**

```java
class Outer {
    void method() {
        class Inner {
            void display() {
                System.out.println("Inside method");
            }
        }

        Inner inner = new Inner();  // 局部内部类只能在方法内部创建
        inner.display();  // 输出：Inside method
    }
}

public class LocalInnerClassExample {
    public static void main(String[] args) {
        Outer outer = new Outer();
        outer.method();
    }
}
```

### 🧩 匿名内部类

匿名内部类是没有类名的内部类，通常用于实现接口或继承类的实例。它是通过 `new` 关键字在代码中直接定义和实例化的。

#### **示例代码：匿名内部类**

```java
interface Animal {
    void sound();
}

public class AnonymousInnerClassExample {
    public static void main(String[] args) {
        Animal dog = new Animal() {  // 匿名内部类实现接口
            public void sound() {
                System.out.println("Dog barks");
            }
        };
        dog.sound();  // 输出：Dog barks
    }
}
```

### 💬 面试问答

> **问：什么是内部类，有什么作用？**
>  **答：** 内部类是定义在另一个类中的类，通常用于表示类与类之间的紧密关联。它可以简化代码结构，提高封装性，并且可以访问外部类的私有成员。

## 🔑 2. 内部类的访问规则与使用场景

### 📖 内部类访问规则

- **成员内部类** 可以访问外部类的所有成员，包括私有成员。
- **静态内部类** 只能访问外部类的静态成员。
- **局部内部类** 只能访问外部方法的局部变量，但这些局部变量必须是 `final` 或者是有效的 `final` 变量。
- **匿名内部类** 通常用来简化代码，处理临时的接口实现或继承。

### 🧩 使用场景

- **成员内部类**：当类之间有紧密的逻辑关系时，使用成员内部类可以更好地组织代码。
- **静态内部类**：当内部类与外部类的实例无关时，使用静态内部类可以减少内存消耗。
- **局部内部类**：用于方法内部的临时操作，避免过多的全局暴露。
- **匿名内部类**：适用于临时实现接口或继承类，不需要命名的类。

## 🔄 3. 枚举类（`enum`）定义与常见用法

### 📖 枚举类的概念

枚举类是 Java 5 引入的一个特性，允许定义一组常量。枚举类的实例是固定的，并且枚举类型是线程安全的。

### 🧩 枚举类的定义

```java
enum Color {
    RED, GREEN, BLUE;
}

public class EnumExample {
    public static void main(String[] args) {
        Color color = Color.RED;
        System.out.println(color);  // 输出：RED
    }
}
```

### 🧩 枚举常用方法

- **`values()`**：返回所有枚举常量的数组。
- **`valueOf()`**：返回与给定字符串对应的枚举常量。

#### **示例代码：`values()` 与 `valueOf()` 方法**

```java
public class EnumMethodsExample {
    public static void main(String[] args) {
        Color color = Color.valueOf("GREEN");
        System.out.println(color);  // 输出：GREEN

        for (Color c : Color.values()) {
            System.out.println(c);  // 输出：RED GREEN BLUE
        }
    }
}
```

## 🔄 4. 枚举实现接口与构造方法

### 📖 枚举实现接口

枚举类可以实现接口，并且可以在枚举常量中定义接口方法的具体实现。

#### **示例代码：枚举实现接口**

```java
interface Shape {
    void draw();
}

enum ShapeType implements Shape {
    CIRCLE {
        @Override
        public void draw() {
            System.out.println("Drawing a circle");
        }
    },
    SQUARE {
        @Override
        public void draw() {
            System.out.println("Drawing a square");
        }
    };
}

public class EnumWithInterfaceExample {
    public static void main(String[] args) {
        ShapeType circle = ShapeType.CIRCLE;
        circle.draw();  // 输出：Drawing a circle
    }
}
```

### 📖 枚举的构造方法

枚举可以有构造方法，可以为每个枚举常量定义不同的属性。

#### **示例代码：枚举的构造方法**

```java
enum Weekday {
    MONDAY("Start of the week"),
    TUESDAY("Second day"),
    WEDNESDAY("Midweek"),
    THURSDAY("Almost there"),
    FRIDAY("End of the work week");

    private String description;

    Weekday(String description) {
        this.description = description;
    }

    public String getDescription() {
        return description;
    }
}

public class EnumConstructorExample {
    public static void main(String[] args) {
        System.out.println(Weekday.MONDAY.getDescription());  // 输出：Start of the week
    }
}
```

### 💬 面试问答

> **问：枚举类和常规类的区别是什么？**
>  **答：** 枚举类在 Java 中用于表示一组常量，每个枚举实例是固定的。枚举类是线程安全的，常用于表示有限的几个选项，如星期、颜色等。

## 🔄 5. 枚举在设计模式中的应用（单例、状态机）

### 📖 单例模式

单例模式是确保某个类只有一个实例，并且提供一个全局访问点。通过枚举实现的单例模式是线程安全的，并且不会受到反射攻击。

#### **示例代码：单例模式**

```java
enum Singleton {
    INSTANCE;

    public void showMessage() {
        System.out.println("Hello from Singleton");
    }
}

public class SingletonExample {
    public static void main(String[] args) {
        Singleton.INSTANCE.showMessage();  // 输出：Hello from Singleton
    }
}
```

### 📖 状态机模式

枚举可以很好地实现状态机模式，每个枚举常量代表一个状态，并且可以在每个状态中实现不同的行为。

#### **示例代码：状态机模式**

```java
enum State {
    START {
        @Override
        public void handle() {
            System.out.println("Start the process");
        }
    },
    STOP {
        @Override
        public void handle() {
            System.out.println("Stop the process");
        }
    };

    public abstract void handle();
}

public class StateMachineExample {
    public static void main(String[] args) {
        State.START.handle();  // 输出：Start the process
        State.STOP.handle();   // 输出：Stop the process
    }
}
```

### 💬 面试问答

> **问：如何使用枚举实现单例模式？**
>  **答：** 使用枚举的 `INSTANCE` 实例来实现单例模式，枚举是线程安全的，并且防止反射攻击。

| 构造方法能重写吗？ | 不能；构造方法不被继承，子类构造中通过 super() 调用父类构造。 |

# 🧩 第十一章：异常处理机制

> **目标：** 理解受检/非受检异常、try-catch-finally、throws/throw、自定义异常与异常链；详细内容见《异常机制与IO》笔记。

------

## 🌱 1. 异常的分类：受检异常 vs 非受检异常

### 📖 异常的分类

在 Java 中，异常分为两类：

- **受检异常（Checked Exception）**：是指在编译时必须处理的异常，通常通过 `try-catch` 块或 `throws` 声明进行处理。常见的如 `IOException`、`SQLException` 等。
- **非受检异常（Unchecked Exception）**：是指在编译时不强制要求处理的异常，通常继承自 `RuntimeException`。常见的如 `NullPointerException`、`ArrayIndexOutOfBoundsException` 等。

#### **区别**：

- 受检异常是编译时检查的异常，需要显示处理或声明；
- 非受检异常是运行时异常，通常不需要显式处理。

#### **示例代码：受检异常与非受检异常**

```java
// 受检异常
public void readFile() throws IOException {
    FileReader file = new FileReader("file.txt");
}

// 非受检异常
public void divideByZero() {
    int result = 10 / 0;  // 会抛出 ArithmeticException
}
```

### 💬 面试问答

> **问：受检异常和非受检异常的区别是什么？**
>  **答：** 受检异常在编译时必须处理，通常用于可预见的错误，如文件未找到；非受检异常不强制处理，通常用于程序的逻辑错误。

------

## 🔑 2. `try-catch-finally` 机制

### 📖 异常处理机制

Java 使用 `try-catch-finally` 块来处理异常。

- **`try` 块**：包含可能抛出异常的代码。
- **`catch` 块**：捕获并处理异常。
- **`finally` 块**：无论是否有异常发生，都会执行的代码，常用于资源释放。

#### **示例代码：`try-catch-finally`**

```java
public class TryCatchExample {
    public static void main(String[] args) {
        try {
            int result = 10 / 0;  // 可能抛出异常
        } catch (ArithmeticException e) {
            System.out.println("Error: " + e.getMessage());  // 捕获并处理异常
        } finally {
            System.out.println("This will always be executed.");  // 资源释放等操作
        }
    }
}
```

### 💬 面试问答

> **问：`finally` 块的作用是什么？**
>  **答：** `finally` 块用于执行无论是否有异常都需要执行的代码，通常用于资源的清理工作，如关闭文件流或数据库连接。

------

## 🔄 3. `throws` 与 `throw` 关键字

### 📖 `throws` 关键字

`throws` 用于声明方法可能抛出的异常。通常与受检异常一起使用，表明该方法不会处理异常，而是交给调用者来处理。

#### **示例代码：`throws`**

```java
public void readFile() throws IOException {
    FileReader file = new FileReader("file.txt");
}
```

### 📖 `throw` 关键字

`throw` 用于手动抛出一个异常对象，通常用于在特定条件下抛出自定义异常或现有异常。

#### **示例代码：`throw`**

```java
public void checkAge(int age) {
    if (age < 18) {
        throw new IllegalArgumentException("Age must be 18 or older");
    }
}
```

### 💬 面试问答

> **问：`throws` 和 `throw` 有什么区别？**
>  **答：** `throws` 用于声明一个方法可能抛出的异常；`throw` 用于在方法体中抛出一个具体的异常。

------

## 🔄 4. 自定义异常类

### 📖 自定义异常

Java 允许开发者根据需求创建自己的异常类。自定义异常通常继承 `Exception` 或 `RuntimeException` 类。

#### **示例代码：自定义异常**

```java
class InvalidAgeException extends Exception {
    public InvalidAgeException(String message) {
        super(message);  // 调用父类构造方法传入异常信息
    }
}

public class CustomExceptionExample {
    public static void checkAge(int age) throws InvalidAgeException {
        if (age < 18) {
            throw new InvalidAgeException("Age must be 18 or older");
        }
    }

    public static void main(String[] args) {
        try {
            checkAge(15);
        } catch (InvalidAgeException e) {
            System.out.println(e.getMessage());  // 输出：Age must be 18 or older
        }
    }
}
```

### 💬 面试问答

> **问：如何定义和使用自定义异常？**
>  **答：** 自定义异常可以继承 `Exception` 类或 `RuntimeException` 类，通常需要重载构造方法，并传入异常信息。在需要的地方使用 `throw` 语句抛出该异常。

## 🔄 5. 异常链与异常封装

### 📖 异常链

异常链指的是一个异常通过 `initCause()` 或构造方法封装另一个异常，通常用于捕获并抛出新异常时，保留原始异常信息。

#### **示例代码：异常链**

```java
public class ExceptionChainExample {
    public static void main(String[] args) {
        try {
            try {
                throw new ArithmeticException("Inner exception");
            } catch (ArithmeticException e) {
                throw new RuntimeException("Outer exception", e);  // 捕获并抛出新异常，保留原始异常
            }
        } catch (RuntimeException e) {
            System.out.println(e.getCause().getMessage());  // 输出：Inner exception
        }
    }
}
```

### 📖 异常封装

通过自定义异常将底层异常封装起来，只暴露必要的异常信息给上层调用者，从而减少异常信息泄露。

#### **示例代码：异常封装**

```java
class DatabaseException extends Exception {
    public DatabaseException(String message, Throwable cause) {
        super(message, cause);  // 将原始异常封装到新的异常中
    }
}

public class ExceptionWrappingExample {
    public static void main(String[] args) {
        try {
            throw new DatabaseException("Database error occurred", new SQLException("SQL syntax error"));
        } catch (DatabaseException e) {
            System.out.println(e.getCause().getMessage());  // 输出：SQL syntax error
        }
    }
}
```

### 💬 面试问答

> **问：什么是异常链？如何使用异常链？**
>  **答：** 异常链是指在捕获异常后，通过构造方法或 `initCause()` 将原始异常封装到新的异常中，从而保留异常的上下文信息。

## 🔄 6. 常见运行时异常与排查技巧

### 📖 常见运行时异常

- **`NullPointerException`**：尝试访问或操作空对象引用。
- **`ArrayIndexOutOfBoundsException`**：访问数组时索引超出范围。
- **`ClassCastException`**：类型转换错误。
- **`IllegalArgumentException`**：方法传入非法参数。
- **`NumberFormatException`**：尝试将非数字字符串转换为数字。

### 🧩 排查技巧

1. **日志记录**：通过 `try-catch` 块记录详细日志，帮助后期排查问题。
2. **调试工具**：使用 IDE 的调试功能逐步执行程序，定位异常发生位置。
3. **堆栈跟踪**：查看异常堆栈信息，找到异常抛出的具体位置。

#### **示例代码：常见异常**

```java
public class RuntimeExceptionExample {
    public static void main(String[] args) {
        String str = null;
        try {
            str.length();  // 会抛出 NullPointerException
        } catch (NullPointerException e) {
            System.out.println("Caught exception: " + e);  // 输出：Caught exception: java.lang.NullPointerException
        }
    }
}
```

### 💬 面试问答

> **问：如何排查 `NullPointerException`？**
>  **答：** 通过检查空值引用并确保对象已正确初始化，使用调试工具逐步检查变量值，避免在可能为空的对象上调用方法。

# 📎 附录：面试题速查

1. **JDK、JRE、JVM 区别？** JDK 包含 JRE 与开发工具，JRE 包含 JVM 与类库，JVM 执行字节码。
2. **Java 为什么能跨平台？** 源码编译成字节码，由各平台 JVM 执行，与平台无关。
3. **Java 是值传递还是引用传递？** 只有值传递；对象参数传的是引用值的拷贝。
4. **重载和重写区别？** 重载：同类同名不同参；重写：子类覆盖父类方法，签名相同。
5. **== 和 equals？** == 比较引用或基本类型值；equals 可重写比较内容，重写 equals 须重写 hashCode。
6. **Object 常见方法？** equals、hashCode、toString、getClass、clone、notify/wait。
7. **接口和抽象类区别？** 接口多实现、无实例字段、抽象方法为主；抽象类单继承、可有实现与状态。
8. **多态底层？** 父类引用指向子类对象，运行时根据实际对象类型确定调用哪个方法。
9. **String 为什么不可变？** final 类与 final 字符数组、无修改接口；利于常量池、线程安全、做 HashMap 键。
10. **构造方法能重写吗？** 不能；构造方法不继承，子类通过 super 调用父类构造。
11. **final、static、this、super？** final 不可变；static 类级共享；this 当前对象；super 父类引用或构造。
12. **封装、继承、多态一句话？** 封装隐藏实现；继承复用与扩展；多态同一接口不同表现，提高扩展性。
