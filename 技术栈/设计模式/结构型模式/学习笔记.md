# 📘《结构型模式学习与实战手册》

> 系统学习见下文各章；日常常用概念与使用场景速查见同目录《常用概念与使用场景》。

------

## 第1章：结构型模式概述

### 1.1 什么是结构型模式

- **结构型模式**关注**对象与类的组合**，形成更大结构，同时保持灵活与可扩展；解决“如何组装、如何复用、如何适配”等问题。
- GoF 结构型共七种：**适配器**、**桥接**、**组合**、**装饰器**、**外观**、**享元**、**代理**。校招与框架中**代理、装饰器、适配器**出现最多。

### 1.2 与创建型、行为型的关系

- 创建型提供“对象从哪来”；结构型决定“对象如何组织、包装、复用”；行为型决定“对象如何协作与分工”。

------

## 第2章：代理模式（Proxy）

1. **意图**：为对象提供代理，以控制对它的访问；客户端通过代理间接访问真实对象。
2. **类型**：静态代理（手写代理类）、动态代理（JDK 基于接口、CGLib 基于子类）；Spring AOP 底层即动态代理。
3. **应用场景**：权限控制、延迟加载、日志/监控、RPC 存根、Spring 事务与 AOP、MyBatis Mapper 接口。
4. **与装饰器区别**：代理控制访问（同接口，侧重“能不能访问、怎么访问”）；装饰器增强功能（同接口，侧重“加一层能力”），边界可模糊。

------

## 第3章：适配器模式（Adapter）

1. **意图**：将不兼容的接口转换为客户期望的接口，使原本无法一起工作的类可以协作。
2. **结构**：目标接口、适配器（实现目标接口并持有被适配者）、被适配者；类适配器（继承）、对象适配器（组合，常用）。
3. **应用场景**：旧接口与新系统对接、SLF4J 与各日志实现、InputStreamReader（字节流→字符流）、Spring MVC HandlerAdapter。
4. **与装饰器区别**：适配器是“接口转换”；装饰器是“同接口增强”，不改变接口。

------

## 第4章：装饰器模式（Decorator）

1. **意图**：动态地给对象增加职责，比继承更灵活；装饰器与组件实现同一接口，可层层包装。
2. **结构**：抽象组件、具体组件、抽象装饰器（持有一个组件引用）、具体装饰器；Java I/O 中 InputStream/FilterInputStream 是典型。
3. **应用场景**：Java I/O 流（BufferedInputStream 装饰 InputStream）、Servlet 中 HttpServletRequestWrapper、增强请求/响应。
4. **与代理区别**：都持有一个内部对象；装饰器强调“层层加功能”；代理强调“控制访问、中介”。

------

## 第5章：外观模式（Facade）

1. **意图**：为子系统中的一组接口提供统一的高层接口，降低子系统与客户端的耦合，简化调用。
2. **结构**：外观类封装多个子系统的调用，客户端只依赖外观；不改变子系统，只增加一层门面。
3. **应用场景**：API 网关对外统一入口、封装多个服务调用的门面服务、SDK 对底层多模块的封装。
4. **与代理/适配器**：外观是“简化一群接口”；代理是“代表一个对象”；适配器是“转换一个接口”。

------

## 第6章：桥接、组合与享元（简述）

1. **桥接（Bridge）**：将抽象与实现分离，使二者可独立变化；避免类爆炸（如多维度：形状×颜色）。应用：JDBC 驱动与 Connection 抽象、消息发送（抽象）与渠道（实现）。
2. **组合（Composite）**：树形结构，叶子与容器统一对待，客户端可一致地处理单个对象与组合。应用：文件系统、菜单树、组织架构。
3. **享元（Flyweight）**：共享细粒度对象以节省内存；内部状态共享、外部状态传入。应用：池化（连接池、线程池）、字符/包装类缓存（如 Integer 缓存）、棋局中棋子共享类型。

------

## 第7章：实战与面试要点

1. **Spring 中的体现**：AOP 动态代理（JDK/CGLib）、HandlerAdapter 适配多种 Controller、各种 Wrapper 的装饰思想、门面式 API（如 RestTemplate 封装 HTTP）。
2. **面试高频**：静态代理与动态代理、JDK 动态代理与 CGLib 区别、适配器与装饰器区别、Spring AOP 用什么代理。
3. **选择原则**：要控制/延迟访问用代理；要接口转换用适配器；要动态增强能力用装饰器；要简化多子系统调用用外观。
