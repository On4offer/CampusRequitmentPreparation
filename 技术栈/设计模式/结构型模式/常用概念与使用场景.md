# 📌 结构型模式常用概念与使用场景速查

> 配合《学习笔记》使用，侧重**代理、适配器、装饰器、外观**的要点与场景，便于速查与校招面试。

---

## 一、代理模式

| 类型 | 实现方式 | 适用 | 典型 |
|------|----------|------|------|
| **静态代理** | 手写代理类，实现与目标相同接口，内部持有目标 | 接口固定、代理类少 | 简单权限/日志代理 |
| **JDK 动态代理** | Proxy.newProxyInstance，基于**接口** | 目标必须实现接口 | Spring AOP 对接口的代理 |
| **CGLib 动态代理** | 生成目标类的**子类**，重写方法 | 无接口的类 | Spring AOP 对类的代理 |

| 对比项 | JDK 动态代理 | CGLib 动态代理 |
|--------|--------------|----------------|
| **依据** | 接口 | 子类（继承） |
| **目标** | 必须有接口 | 可有可无（不能 final） |
| **性能** | 调用略快 | 生成类略慢，调用略慢（可忽略） |
| **Spring 选择** | Bean 实现接口时默认 JDK | 无接口时 CGLib |

---

## 二、适配器模式

| 类型 | 说明 |
|------|------|
| **类适配器** | 适配器继承被适配类并实现目标接口（多继承在 Java 中不适用，多用组合） |
| **对象适配器** | 适配器持有被适配者实例，实现目标接口；**常用** |

- **典型**：SLF4J 适配 Log4j/Logback；InputStreamReader 适配字节流到字符流；Spring MVC **HandlerAdapter** 适配多种 Handler（Controller、Servlet 等）到统一的 ModelAndView 处理流程。

---

## 三、装饰器 vs 代理 vs 适配器

| 模式 | 目的 | 接口关系 | 典型 |
|------|------|----------|------|
| **装饰器** | 增强功能，层层包装 | 与组件同接口 | Java I/O、RequestWrapper |
| **代理** | 控制访问、中介 | 与目标同接口 | AOP、延迟加载、RPC |
| **适配器** | 接口转换，使能协作 | 实现目标接口，持有被适配者 | HandlerAdapter、Reader(InputStream) |

---

## 四、外观模式

- **一句话**：为多个子系统提供**统一入口**，客户端只调门面，不直接依赖多个子模块。
- **典型**：API 网关、封装多个 Service 的 Facade 类、SDK 对外简单 API。

---

## 五、桥接、组合、享元（速记）

| 模式 | 核心思想 | 典型 |
|------|----------|------|
| **桥接** | 抽象与实现分离，两维度独立变化 | JDBC 驱动与 Connection、消息与发送渠道 |
| **组合** | 树形结构，叶子与容器统一接口 | 文件/目录、菜单树 |
| **享元** | 共享相同内部状态，减少对象数 | 连接池、Integer 缓存、棋子类型共享 |

---

## 六、Spring / 框架中的结构型

| 场景 | 对应模式 |
|------|----------|
| **AOP 切面** | 动态代理（JDK/CGLib） |
| **@Transactional** | 代理包装 Bean，在调用前后做事务 |
| **HandlerAdapter** | 适配器：多种 Controller 统一成 ModelAndView |
| **HttpServletRequestWrapper** | 装饰器：增强请求（如多次读 body） |
| **RestTemplate / 网关** | 外观：封装 HTTP/多服务调用 |

---

## 七、面试一句话总结

- **代理**：控制访问，静态/动态；Spring AOP 用 JDK 或 CGLib 动态代理。
- **适配器**：接口不兼容时做转换，对象适配器常用；如 HandlerAdapter。
- **装饰器**：同接口增强，层层包装；如 I/O 流、Wrapper。
- **外观**：多子系统统一入口，简化调用。
- **桥接**：抽象与实现分离；**组合**：树形一致处理；**享元**：共享内部状态省内存。
