# 📌 创建型模式常用概念与使用场景速查

> 配合《学习笔记》使用，侧重**单例、工厂、建造者、原型**的要点与场景，便于速查与校招面试。

---

## 一、单例模式

| 实现方式 | 要点 | 优缺点 |
|----------|------|--------|
| **饿汉** | 静态成员直接 new | 简单、线程安全；启动即加载，可能浪费 |
| **懒汉 + 同步** | getInstance() 加 synchronized | 安全但每次获取都加锁，性能差 |
| **双重检查锁** | volatile + 两次 if (instance == null) + synchronized 块内再 new | 懒加载、线程安全、性能好；需 JDK5+ volatile 保证可见性 |
| **静态内部类** | 内部类持有一个静态实例，getInstance 返回内部类.INSTANCE | 懒加载、线程安全、代码简洁，推荐 |
| **枚举** | enum 单例，天然单例、防反射 | 不能懒加载；适合简单单例 |

```java
// 静态内部类（推荐）
public class Singleton {
    private Singleton() {}
    private static class Holder {
        private static final Singleton INSTANCE = new Singleton();
    }
    public static Singleton getInstance() {
        return Holder.INSTANCE;
    }
}
```

## 二、工厂方法 vs 抽象工厂

| 对比项 | 工厂方法 | 抽象工厂 |
|--------|----------|----------|
| **抽象层** | 一个工厂方法创建一种产品 | 一个工厂创建一族产品（多个等级） |
| **扩展** | 新增产品 = 新增具体工厂+具体产品 | 新增产品族 = 新增具体工厂（多类产品一起） |
| **典型** | LoggerFactory.getLogger() | 主题工厂（创建按钮+文本框+边框等一族） |

---

## 三、简单工厂 vs 工厂方法

| 对比项 | 简单工厂 | 工厂方法 |
|--------|----------|----------|
| **创建逻辑** | 工厂类内 if/switch 决定创建谁 | 子类实现 factoryMethod，决定创建谁 |
| **开闭原则** | 新增类型需改工厂类 | 新增类型只需新增子类，符合开闭 |
| **适用** | 产品类型少、稳定 | 产品类型多、易扩展 |

## 四、建造者模式要点

- **链式调用**：每个 set 方法 `return this`，便于 `new Builder().a(1).b(2).build()`。
- **与构造器对比**：参数多时构造器可读性差；建造者分步设置、可选参数清晰。
- **常见**：StringBuilder、Lombok @Builder、MyBatis SqlSessionFactoryBuilder。

```java
// 典型用法
User user = User.builder()
    .id(1L)
    .name("张三")
    .age(18)
    .build();
```

## 五、原型模式与拷贝

| 概念 | 说明 |
|------|------|
| **浅拷贝** | 复制对象本身，成员引用仍指向原对象成员；默认 clone() 即浅拷贝 |
| **深拷贝** | 递归复制引用类型，或序列化再反序列化、拷贝工具（BeanUtils、JSON） |
| **Cloneable** | 标记接口；clone() 在 Object 中为 protected，需重写并改为 public（或子类可见） |

## 六、Spring 中的创建型

| 场景 | 对应模式/思想 |
|------|----------------|
| **Bean 单例/原型** | 单例（默认）、原型作用域 |
| **BeanFactory / getBean** | 工厂方法思想 |
| **FactoryBean** | 工厂方法：getObject() 返回实际 Bean |
| **@Configuration + @Bean** | 工厂方法：方法即“创建逻辑” |
| **Builder 风格配置** | 建造者（如 RestTemplateBuilder） |

---

## 七、面试一句话总结

- **单例**：保证全局唯一实例，常用静态内部类或双重检查锁。
- **工厂方法**：创建推迟到子类，扩展新产品只加子类。
- **抽象工厂**：一族相关产品由一个工厂创建，换族即换工厂。
- **建造者**：分步构建复杂对象，链式调用，可选参数多时用。
- **原型**：通过复制已有对象创建新对象，注意深拷贝与浅拷贝。
