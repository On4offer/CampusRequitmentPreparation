# 📘《创建型模式学习与实战手册》

> 系统学习见下文各章；日常常用概念与使用场景速查见同目录《常用概念与使用场景》。

------

## 第1章：创建型模式概述

### 1.1 什么是创建型模式

- **创建型模式**关注**对象的创建**，将创建与使用解耦，使系统不依赖具体类的实例化方式。
- GoF 创建型共五种：**单例**、**工厂方法**、**抽象工厂**、**建造者**、**原型**。校招与 Spring 源码中单例、工厂出现频率最高。

### 1.2 与其它类型模式的关系

- **结构型**关注对象组合与结构；**行为型**关注对象间的职责与通信。创建型解决“谁来建、怎么建”的问题，为后续结构与行为提供对象来源。

------

## 第2章：单例模式（Singleton）

1. **意图**：保证一个类只有一个实例，并提供全局访问点。
2. **实现方式**：懒汉（同步/双重检查）、饿汉、静态内部类、枚举；Java 中推荐**双重检查锁**或**静态内部类**，枚举可防反射破坏。
3. **应用场景**：线程池、配置管理、Spring 默认 Bean 作用域、连接池、日志 Logger。
4. **注意点**：多线程安全、反射与反序列化破坏单例的防护、单例与测试的可测性（依赖注入替代硬编码单例）。

------

## 第3章：工厂方法模式（Factory Method）

1. **意图**：定义创建对象的接口，由子类决定实例化哪一个类，使实例化推迟到子类。
2. **结构**：抽象产品、具体产品、抽象工厂（含 factoryMethod）、具体工厂；客户端依赖抽象工厂与抽象产品。
3. **应用场景**：JDBC Driver、集合 Iterator、Spring BeanFactory 的 getBean、日志框架的 LoggerFactory。
4. **与简单工厂区别**：简单工厂在工厂类中 if/switch 创建；工厂方法把“创建谁”交给子类，符合开闭原则。

------

## 第4章：抽象工厂模式（Abstract Factory）

1. **意图**：提供创建一系列相关或相互依赖对象的接口，无需指定具体类。
2. **结构**：多个抽象产品族、多个具体产品、抽象工厂（多个 create 方法）、具体工厂；一个工厂负责一族产品。
3. **应用场景**：UI 主题（按钮+文本框一族）、数据库访问（Connection+Statement 一族）、跨平台 GUI 组件。
4. **与工厂方法区别**：工厂方法一个工厂生产一种产品；抽象工厂一个工厂生产一族产品（多个等级/类型）。

------

## 第5章：建造者模式（Builder）

1. **意图**：将复杂对象的构建与表示分离，使同一构建过程可创建不同表示。
2. **结构**：产品类、抽象建造者（定义构建步骤）、具体建造者、指挥者 Director（可选，封装构建顺序）；链式调用常见（return this）。
3. **应用场景**：StringBuilder、Lombok @Builder、MyBatis SqlSessionFactoryBuilder、AlertDialog 构建、复杂配置对象。
4. **与工厂区别**：工厂关注“创建哪种对象”；建造者关注“分步构建复杂对象”，强调过程与可选参数。

------

## 第6章：原型模式（Prototype）

1. **意图**：用原型实例指定创建对象的种类，通过复制该原型创建新对象。
2. **实现**：实现 Cloneable，重写 clone()；深拷贝与浅拷贝（浅拷贝只复制引用，深拷贝需递归或序列化/工具）。
3. **应用场景**：对象创建成本高、需要大量相似对象、需要保存与恢复对象状态；Spring 中 prototype 作用域 Bean 的创建思路可类比。
4. **注意点**：Java clone 的缺陷（浅拷贝、不调用构造器）；实际项目中常用 JSON 拷贝、BeanUtils 或手动拷贝替代。

------

## 第7章：实战与面试要点

1. **Spring 中的体现**：单例/原型作用域、BeanFactory/ApplicationContext 与工厂方法、FactoryBean、Builder 风格配置。
2. **面试高频**：手写单例（双重检查、静态内部类）、工厂方法与抽象工厂区别、建造者解决什么问题、深拷贝与浅拷贝。
3. **选择原则**：需要唯一实例用单例；需要解耦创建用工厂；需要分步构建用建造者；需要克隆对象用原型。
