# 📌 行为型模式常用概念与使用场景速查

> 配合《学习笔记》使用，侧重**观察者、策略、模板方法、责任链**的要点与场景，便于速查与校招面试。

---

## 一、观察者模式

| 概念 | 说明 |
|------|------|
| **主题（Subject）** | 维护观察者列表，状态变化时 notify 所有观察者 |
| **观察者（Observer）** | 提供 update，接收主题通知 |
| **推模型** | 主题把数据通过 update 参数推给观察者 |
| **拉模型** | 观察者从主题引用主动拉取需要的数据 |

| 对比 | 观察者 | 发布/订阅 |
|------|--------|-----------|
| **耦合** | 观察者通常知道主题 | 发布者与订阅者通过事件通道解耦，互不知对方 |
| **典型** | 直接 addListener + 回调 | 消息队列、EventBus、Redis Pub/Sub |

- **Spring**：ApplicationEventPublisher 发布事件，@EventListener 或实现 ApplicationListener 订阅。

---

## 二、策略模式

- **核心**：算法族可互换，Context 持有策略接口，客户端注入具体策略。
- **替代 if/switch**：多种支付方式、多种排序/比较、多种路由规则，用策略 + 工厂或配置选择，便于扩展。

```java
// 典型结构
public interface Strategy { void execute(); }
public class Context {
    private Strategy strategy;
    public void setStrategy(Strategy s) { this.strategy = s; }
    public void doWork() { strategy.execute(); }
}
```

| 场景 | 对应 |
|------|------|
| 支付方式 | 策略接口 + 支付宝/微信/银行卡实现 |
| Comparator | 策略：不同比较规则 |
| Spring Resource | ClassPathResource、UrlResource 等不同策略 |

---

## 三、模板方法模式

- **核心**：抽象类定义 templateMethod()，内部调用抽象方法 step1()、step2()；子类实现 step1、step2，**不改变调用顺序**。
- **钩子方法**：可选重写的方法，子类可选择性覆盖以影响流程。

| 场景 | 对应 |
|------|------|
| Servlet | service() 为模板，doGet/doPost 为子类实现的步骤 |
| JUnit | setUp / testXxx / tearDown 为模板步骤 |
| JdbcTemplate | execute(ConnectionCallback) 中“取连接→回调→释放”为模板，可变的是 SQL 执行 |

---

## 四、责任链模式

- **核心**：Handler 接口含 next 与 handleRequest；每个具体 Handler 处理完可转给 next，形成链。
- **建链**：a.setNext(b); b.setNext(c); 然后从 a 开始处理请求。

| 场景 | 对应 |
|------|------|
| Servlet Filter | FilterChain，多个 Filter 顺序执行 |
| Spring Interceptor | HandlerInterceptor 链，preHandle → Controller → postHandle |
| 审批流程 | 多级审批，每级可通过/驳回/转交下一级 |

---

## 五、策略 vs 模板方法 vs 状态

| 模式 | 谁决定“用哪个” | 典型 |
|------|----------------|------|
| **策略** | 客户端从外部注入 | 支付方式、排序规则 |
| **模板方法** | 父类定流程，子类定步骤实现 | JdbcTemplate、Servlet |
| **状态** | 对象内部状态切换，状态类决定行为 | 订单状态机、TCP 状态 |

---

## 六、Spring / 框架中的行为型

| 场景 | 对应模式 |
|------|----------|
| **事件 @EventListener** | 观察者 |
| **Resource 多种实现** | 策略 |
| **JdbcTemplate execute** | 模板方法 |
| **HandlerInterceptor 链** | 责任链 |
| **事务传播、AOP 链** | 责任链思想 |

---

## 七、面试一句话总结

- **观察者**：一对多，主题变则通知所有观察者；与发布/订阅区别在耦合度。
- **策略**：算法可互换，Context 持策略接口，避免 if/switch 膨胀。
- **模板方法**：父类定骨架，子类填步骤；Servlet、JdbcTemplate 即此类。
- **责任链**：请求沿链传递，多节点有机会处理；Filter、Interceptor 即链。
- **命令**：请求封装成对象，可排队、撤销、日志；**状态**：按内部状态切换行为。
