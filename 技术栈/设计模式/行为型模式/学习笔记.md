# 📘《行为型模式学习与实战手册》

> 系统学习见下文各章；日常常用概念与使用场景速查见同目录《常用概念与使用场景》。

------

## 第1章：行为型模式概述

### 1.1 什么是行为型模式

- **行为型模式**关注**对象间的职责划分与通信**，描述“谁做什么、谁通知谁、谁先谁后”，解决算法与职责的分配问题。
- GoF 行为型共十一种：**责任链**、**命令**、**解释器**、**迭代器**、**中介者**、**备忘录**、**观察者**、**状态**、**策略**、**模板方法**、**访问者**。校招与业务中最常考**观察者、策略、模板方法、责任链**。

### 1.2 与创建型、结构型的关系

- 创建型提供对象；结构型组织对象；行为型定义对象间的协作与流程，是“动”的部分。

## 第2章：观察者模式（Observer）

1. **意图**：定义一对多依赖，多个观察者监听主题，主题状态变化时自动通知所有观察者。
2. **结构**：主题（Subject）维护观察者列表，提供 attach/detach/notify；观察者（Observer）提供 update；推模型（主题推送数据）与拉模型（观察者主动拉取）。
3. **应用场景**：事件监听、消息队列消费、Vue/React 响应式与订阅、Spring 事件（ApplicationEventPublisher）、GUI 绑定。
4. **与发布/订阅**：观察者通常直接依赖主题；发布/订阅通过事件总线/消息中间件解耦，发布者与订阅者互不知对方。

------

## 第3章：策略模式（Strategy）

1. **意图**：定义一系列算法并封装，使它们可互换；将算法与使用算法的客户端解耦。
2. **结构**：策略接口、多个具体策略、上下文 Context 持有策略并委托执行；客户端选择策略注入 Context。
3. **应用场景**：支付方式（支付宝/微信/银行卡）、排序/比较器、路由规则、折扣计算、Spring 中 Resource 的多种实现（UrlResource、ClassPathResource）。
4. **与状态模式区别**：策略由客户端在外部指定；状态在运行时由对象内部切换，状态自己决定下一状态。

------

## 第4章：模板方法模式（Template Method）

1. **意图**：在父类中定义算法骨架，将某些步骤推迟到子类实现，子类可重写步骤而不改变流程。
2. **结构**：抽象类定义 templateMethod()，其中调用抽象方法 step1()、step2() 等；子类实现这些抽象方法。
3. **应用场景**：JUnit 的 setUp/tearDown、Servlet 的 service 与 doGet/doPost、Spring JdbcTemplate 的 execute（固定流程，可变的是 SQL 与参数）、抽象类定义流程框架。
4. **与策略区别**：模板方法是“继承 + 重写步骤”；策略是“组合 + 替换算法”，多用组合少用继承。

------

## 第5章：责任链模式（Chain of Responsibility）

1. **意图**：多个对象都有机会处理请求，请求沿链传递直到某对象处理或链结束。
2. **结构**：抽象处理者（含 next 引用与 handleRequest）、具体处理者；客户端组链并提交请求。
3. **应用场景**：过滤器链（Servlet Filter、Spring Interceptor）、审批流程、异常处理层级、日志/权限链式处理。
4. **注意点**：链的顺序、是否必须有人处理、请求是否可被多个节点处理（通常一个处理即结束）。

------

## 第6章：命令、状态与其它（简述）

1. **命令（Command）**：将请求封装为对象，便于参数化、排队、日志、撤销；典型：GUI 菜单命令、任务队列、事务型操作。
2. **状态（State）**：对象根据内部状态改变行为，状态类封装对应行为；典型：订单状态（待支付/已支付/已取消）、TCP 连接状态、工作流引擎。
3. **迭代器（Iterator）**：统一访问聚合对象元素，解耦遍历与聚合；Java 集合的 Iterator、foreach。
4. **中介者（Mediator）**：用中介对象封装多对象交互，减少对象间耦合；典型：聊天室、MVC 中 Controller。
5. **备忘录（Memento）**：保存与恢复对象状态，用于撤销、快照；需注意状态暴露与封装边界。

------

## 第7章：实战与面试要点

1. **Spring 中的体现**：事件机制（观察者）、Resource 策略族、JdbcTemplate 模板方法、拦截器/过滤器链（责任链）、事务与 AOP 中的责任链思想。
2. **面试高频**：观察者与发布订阅区别、策略与模板方法区别、手写责任链或过滤器链、策略模式替代 if/switch 的好处。
3. **选择原则**：一对多通知用观察者；多算法可互换用策略；固定流程可变步骤用模板方法；多级处理/审批用责任链。
