# 📌 GitHub Actions 常用配置与使用场景速查

> 配合《学习笔记》使用，侧重**工作流结构、触发、常用 Action、Secrets**，便于速查与校招面试。

---

## 一、工作流文件位置与结构

- **路径**：仓库根目录 `.github/workflows/*.yml` 或 `.yaml`。
- **最小结构**：

```yaml
name: CI
on: push
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: echo "Hello"
```

---

## 二、触发事件（on）

| 事件 | 说明 | 示例 |
|------|------|------|
| **push** | 推送到分支 | on: push 或 push: branches: [main] |
| **pull_request** | PR 打开/更新 | on: pull_request |
| **workflow_dispatch** | 手动触发 | 可在页面上点 Run |
| **schedule** | 定时（cron） | on: schedule: - cron: '0 * * * *' |
| **release** | 发布 release | on: release |
| **push tags** | 推送 tag | on: push: tags: ['v*'] |

- **路径过滤**：仅某目录变更时触发：
  ```yaml
  on:
    push:
      paths:
        - 'src/**'
        - 'pom.xml'
  ```

## 三、Job 与 Step

| 字段 | 说明 |
|------|------|
| **jobs.&lt;id&gt;.runs-on** | ubuntu-latest、windows-latest、macos-latest |
| **jobs.&lt;id&gt;.steps** | 顺序执行的步骤列表 |
| **step.uses** | 使用 Action，如 actions/checkout@v4 |
| **step.run** | 执行 shell 命令；多行用 \| |
| **step.name** | 步骤名称，日志中显示 |
| **step.env** | 该 step 的环境变量 |
| **jobs.&lt;id&gt;.needs** | 依赖的 job 列表，依赖通过后才运行 |

---

## 四、常用 Action

| Action | 用途 |
|--------|------|
| **actions/checkout@v4** | 拉取仓库代码，通常第一步 |
| **actions/setup-java@v4** | 安装 JDK；with: java-version: '17'、distribution: 'temurin' |
| **actions/setup-node@v4** | 安装 Node；with: node-version: '20'、cache: 'npm' |
| **actions/cache@v4** | 缓存目录；with: path、key（含 hash 可自动失效） |
| **actions/upload-artifact@v4** | 上传制品；with: name、path |
| **actions/download-artifact@v4** | 下载制品；with: name |
| **docker/build-push-action@v5** | 构建并推送 Docker 镜像 |

## 五、表达式与 Secrets

| 表达式 | 说明 |
|--------|------|
| **${{ env.VAR }}** | 环境变量 |
| **${{ secrets.XXX }}** | 仓库/组织 Secrets，不在日志中显示 |
| **${{ github.ref }}** | 触发分支/标签，如 refs/heads/main |
| **${{ github.sha }}** | 提交 SHA |
| **${{ github.event_name }}** | 事件名 push、pull_request 等 |
| **${{ steps.&lt;id&gt;.outputs.xxx }}** | 某 step 的 outputs |
| **${{ success() }}** / **${{ failure() }}** | 上一步是否成功/失败 |

- **Secrets 配置**：仓库 Settings → Secrets and variables → Actions → New repository secret；使用时 `${{ secrets.NAME }}`。

## 六、缓存示例

```yaml
- uses: actions/cache@v4
  with:
    path: |
      ~/.m2
      node_modules
    key: ${{ runner.os }}-build-${{ hashFiles('**/pom.xml', '**/package-lock.json') }}
    restore-keys: ${{ runner.os }}-build-
```

- **key** 含 hashFiles 时，文件变化则缓存失效；**restore-keys** 为前缀匹配，用于回退到旧缓存。

## 七、矩阵与条件

```yaml
strategy:
  matrix:
    java: [8, 17]
    os: [ubuntu-latest, windows-latest]
# 每个组合跑一次
```

```yaml
# 仅 main 分支部署
if: github.ref == 'refs/heads/main'
# 仅上一步成功时执行
if: success()
```

## 八、使用场景对照

| 场景 | 做法 |
|------|------|
| **PR 时跑测试** | on: pull_request；steps: checkout → setup-java/node → 构建 → 测试 |
| **推 main 时构建并部署** | on: push: branches: [main]；build job → deploy job（needs: build） |
| **打 tag 发版** | on: push: tags: ['v*']；构建 → 打镜像 → 推镜像 → 可选 release |
| **手动触发** | on: workflow_dispatch；可选 inputs 参数 |
| **缓存 Maven/npm** | actions/cache，path 为 .m2 或 node_modules，key 含 hashFiles |
| **多版本测试** | strategy.matrix 设 java/node 多版本 |
| **部署用密钥** | SSH_KEY、SERVER_HOST 等存 Secrets；step 里用 ${{ secrets.XXX }} |
