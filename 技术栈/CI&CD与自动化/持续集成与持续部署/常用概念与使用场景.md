# 📌 持续集成与持续部署常用概念与使用场景速查

> 配合《学习笔记》使用，侧重**CI/CD 概念、流水线阶段、环境与发布策略**，便于速查与校招面试。

---

## 一、CI 与 CD 区别

| 概念 | 说明 |
|------|------|
| **CI（持续集成）** | 频繁合并代码，每次自动构建+测试，快速发现错误 |
| **持续交付（Continuous Delivery）** | 流水线通过后随时可发布，发布由**人工**触发 |
| **持续部署（Continuous Deployment）** | 流水线通过后**自动**发布到生产（或目标环境） |

- **关系**：CI 是基础；CD 在 CI 之上加“自动/半自动发布”；多数先做 CI + 持续交付。

---

## 二、流水线典型阶段

| 阶段 | 说明 | 失败影响 |
|------|------|----------|
| **拉取代码** | checkout 仓库、子模块 | 后续不执行 |
| **安装依赖** | npm install、mvn 依赖 | 同上 |
| **构建/编译** | mvn package、npm run build | 同上 |
| **静态检查** | ESLint、Sonar、代码规范 | 可配置为失败或警告 |
| **单元测试** | 单测、覆盖率 | 通常失败即停 |
| **集成测试** | 接口测试、端到端（可选） | 同上 |
| **构建制品** | jar、镜像、dist 打包 | 同上 |
| **部署** | 推到测试/预发/生产 | 按环境与策略 |

- **质量门禁**：某阶段不通过则整条流水线失败，不进入下一阶段或不可发布。

---

## 三、环境与发布策略

| 环境 | 典型用途 |
|------|----------|
| **dev** | 开发联调 |
| **test** | 测试、自动化测试 |
| **staging / 预发** | 与生产配置一致，最终验证 |
| **prod** | 生产 |

| 发布策略 | 说明 |
|----------|------|
| **蓝绿** | 两套环境，切换流量，回滚即切回 |
| **金丝雀/灰度** | 部分流量到新版本，逐步放量 |
| **滚动** | 逐个实例替换，无停机 |
| **回滚** | 保留上一版本，出问题一键回退 |

---

## 四、触发方式

| 触发 | 说明 |
|------|------|
| **push** | 推送到某分支触发（如 main、develop） |
| **merge / PR** | 合并或 PR 创建/更新时触发 |
| **tag** | 打 tag（如 v1.0.0）触发发布流水线 |
| **定时** | 定时构建、定时发布（如每晚） |
| **手动** | 在流水线平台手动执行 |

---

## 五、制品与密钥

| 概念 | 说明 |
|------|------|
| **制品** | 构建产物：jar、镜像、前端 dist、安装包等；存制品库或镜像仓库 |
| **制品库** | Nexus、Harbor、云厂商制品库；版本管理、依赖拉取 |
| **密钥/Secrets** | 流水线中用的密码、Token、证书；存 CI 平台 Secrets，不写进代码与配置 |

- **最佳实践**：敏感信息一律用 Secrets；流水线仅拉取必要权限；构建与部署权限分离。

## 六、常见工具对比（速记）

| 工具 | 特点 | 适用 |
|------|------|------|
| **GitHub Actions** | 与 GitHub 一体、YAML 配置、按分钟计费 | 开源项目、GitHub 仓库 |
| **GitLab CI** | 与 GitLab 一体、.gitlab-ci.yml | 自建 GitLab、私有仓库 |
| **Jenkins** | 自托管、插件多、灵活 | 内网、复杂流水线、自建 |
| **云厂商** | 云效、CODING、CodePipeline | 国内云、一体化 DevOps |

---

## 七、面试一句话总结

- **CI**：频繁集成 + 自动构建与测试，快速反馈；**CD**：在 CI 基础上自动化/半自动化发布。
- **流水线**：拉代码 → 依赖 → 构建 → 检查 → 测试 → 制品 → 部署；某步失败即失败（质量门禁）。
- **环境**：dev/test/staging/prod；**发布**：蓝绿、灰度、滚动；**回滚**：保留上一版本、一键回退。
- **制品**：构建产物；**密钥**：用 Secrets 管理，不落库。
- **工具**：GitHub Actions/GitLab CI 与代码托管一体；Jenkins 自建、灵活。
