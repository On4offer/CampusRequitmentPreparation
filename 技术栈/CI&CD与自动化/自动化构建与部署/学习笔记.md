# 📘《自动化构建与部署学习与实战手册》

> 系统学习见下文各章；日常常用概念与使用场景速查见同目录《常用概念与使用场景》。

------

## 第1章：自动化构建概述

### 1.1 什么是自动化构建

- **自动化构建**指通过**脚本或流水线**一键完成：依赖解析、编译/打包、静态检查、测试、生成制品，无需人工逐条执行命令。
- 目标：可重复、可追溯、与环境一致；与 CI 结合时，每次提交或合并触发构建，保证主干始终可构建。

### 1.2 构建与部署的关系

- **构建**产出**制品**（jar、war、前端 dist、Docker 镜像等）；**部署**将制品放到目标环境并启动。流水线中通常先构建、再部署；部署可自动化（CD）或人工触发。

## 第2章：后端构建（Maven / Gradle）

1. **Maven**：mvn clean install、mvn clean package；生命周期（compile、test、package、install、deploy）；多模块用 reactor；跳过测试 -DskipTests；配置文件 pom.xml、settings.xml（镜像、仓库、认证）。
2. **Gradle**：./gradlew build、./gradlew clean build；任务依赖与增量构建；build.gradle、settings.gradle；与 Maven 仓库兼容，可发布到 Nexus。
3. **制品**：jar/war 产出在 target/ 或 build/libs/；可上传到 Nexus、Harbor 关联的制品库；版本号与 Git tag 或 build number 关联便于追溯。
4. **面试要点**：Maven 生命周期、如何跳过测试、多模块构建顺序。

------

## 第3章：前端构建与静态资源

1. **npm / pnpm**：npm run build、pnpm build；package.json 中 scripts 定义；依赖安装 npm ci（锁文件一致）、npm install；产物通常为 dist/ 或 build/。
2. **构建工具**：Vite、Webpack 等；环境变量（如 VITE_API_BASE）在构建时注入；多环境可配 .env.development、.env.production 或 CI 中传入。
3. **静态部署**：dist 可部署到 Nginx、对象存储（OSS）+ CDN；SPA 需配置 fallback 或 try_files 以支持前端路由。
4. **面试要点**：npm ci 与 npm install 区别、前端多环境变量如何注入、构建产物如何部署。

------

## 第4章：Docker 镜像构建与推送

1. **Dockerfile**：FROM 基础镜像、WORKDIR、COPY/ADD、RUN、EXPOSE、CMD/ENTRYPOINT；多阶段构建（先构建再拷贝产物到运行镜像）减小镜像体积。
2. **构建**：docker build -t name:tag .；在 CI 中由流水线执行；构建参数（如 BUILD_DATE、VERSION）可通过 --build-arg 传入。
3. **推送**：docker push registry/name:tag；需先 docker login； registry 可为 Docker Hub、Harbor、云厂商镜像仓库、GHCR。
4. **最佳实践**：多阶段、非 root 用户、.dockerignore 减少上下文、固定基础镜像 tag（不用 latest）。
5. **面试要点**：多阶段构建目的、如何减小镜像体积、镜像 tag 与版本追溯。

------

## 第5章：部署方式概览

1. **脚本部署**：SSH 登录目标机，拉取制品或 scp 上传，执行启动/重启脚本；简单直接，适合少量机器；需保管 SSH 密钥与脚本。
2. **Docker 部署**：目标机拉取镜像并 docker run 或 docker-compose up；环境一致、易回滚（换镜像 tag）；需镜像仓库与 Docker 环境。
3. **K8s 部署**：通过 Deployment、Service 等资源管理；kubectl set image、helm upgrade 或 GitOps（Argo CD、Flux）同步清单；适合多实例、滚动与灰度。
4. **云厂商**：云效、CODING、或云主机/容器服务的控制台或 API；与流水线集成，发布到云上实例或 K8s 集群。
5. **面试要点**：脚本、Docker、K8s 部署的适用场景、如何做零停机发布与回滚。

------

## 第6章：与 CI/CD 流水线的衔接

1. **流水线中调用**：构建阶段执行 mvn/gradle/npm 命令；部署阶段执行 SSH 命令、docker pull + run、或 kubectl/helm；凭据由 CI 平台 Secrets 提供。
2. **版本与追溯**：构建号或 Git SHA 写入制品或镜像 tag；部署时使用同一 tag，便于“哪次构建上了生产”的追溯与回滚。
3. **环境隔离**：不同环境用不同配置文件或环境变量；流水线按分支/tag 选择部署目标与配置。
4. **面试要点**：流水线如何调用构建与部署、如何保证部署的是“刚构建出的那一版”。
