# 📌 自动化构建与部署常用概念与使用场景速查

> 配合《学习笔记》使用，侧重**构建命令、制品、Docker、部署方式**，便于速查与校招面试。

---

## 一、后端构建（Maven / Gradle）

| 工具 | 常用命令 | 说明 |
|------|----------|------|
| **Maven** | **mvn clean install** | 清理、编译、测试、打包、装到本地库 |
| **Maven** | **mvn clean package** | 到 package 阶段，产出在 target/ |
| **Maven** | **mvn -DskipTests package** | 跳过测试打包 |
| **Maven** | **mvn clean deploy** | 打包并发布到远程仓库 |
| **Gradle** | **./gradlew clean build** | 清理并构建 |
| **Gradle** | **./gradlew -x test build** | 跳过测试 |
| **Gradle** | **./gradlew publish** | 发布到配置的仓库 |

- **多模块**：Maven reactor 按依赖顺序构建；Gradle 按 settings 与依赖关系。

## 二、前端构建

| 命令/配置 | 说明 |
|-----------|------|
| **npm run build** / **pnpm build** | 执行 package.json 中 build 脚本 |
| **npm ci** | 按 package-lock.json 安装，适合 CI 环境，干净可重复 |
| **npm install** | 安装依赖，可能更新锁文件 |
| **环境变量** | Vite: VITE_XXX；构建时注入，如 API 地址 |
| **产物** | 通常 dist/ 或 build/，可部署到 Nginx/OSS |

- **npm ci** 与 **npm install**：ci 不修改锁文件、先删 node_modules 再装，适合流水线；install 可能升级依赖。

## 三、Docker 镜像

| 操作 | 命令/要点 |
|------|-----------|
| **构建** | docker build -t name:tag . |
| **多阶段** | 多 FROM；前一阶段构建，后一阶段只拷贝产物，减小镜像 |
| **推送** | docker login → docker push registry/name:tag |
| **.dockerignore** | 排除无关文件，加快构建、减小上下文 |
| **tag 与版本** | 用 Git SHA 或 build number 打 tag，便于追溯与回滚 |

```dockerfile
# 多阶段示例：构建 + 运行
FROM maven:3-eclipse-temurin-17 AS builder
WORKDIR /app
COPY pom.xml .
RUN mvn dependency:go-offline -B
COPY src ./src
RUN mvn package -DskipTests -B

FROM eclipse-temurin:17-jre-alpine
COPY --from=builder /app/target/*.jar app.jar
ENTRYPOINT ["java","-jar","/app.jar"]
```

## 四、部署方式对比

| 方式 | 适用 | 特点 |
|------|------|------|
| **脚本 + SSH** | 少量机器、简单应用 | 直接执行命令；需管密钥与脚本 |
| **Docker** | 单机/多机容器 | 拉镜像、docker run 或 compose；环境一致、易回滚 |
| **K8s** | 多实例、编排、灰度 | Deployment、Service、Ingress；kubectl/helm/GitOps |
| **云厂商控制台/API** | 云主机、托管服务 | 与云产品集成，按云厂商文档操作 |

- **回滚**：脚本部署保留上一版并切换；Docker 用上一镜像 tag；K8s 用上一版 Deployment 或 helm rollback。

## 五、流水线中的衔接

| 阶段 | 典型做法 |
|------|----------|
| **构建** | checkout → 安装依赖（可缓存）→ 构建命令 → 产出制品 |
| **制品** | 上传 artifact（jar/dist）或 构建并推送镜像 |
| **部署** | 下载制品/拉镜像 → SSH 执行脚本 / docker run / kubectl apply |
| **凭据** | CI Secrets 存 SSH 私钥、镜像仓库密码、kubeconfig 等；不在脚本中写明文 |
| **版本** | 构建号或 Git SHA 写入 jar 元数据或镜像 tag，部署时用同一版本 |

## 六、面试一句话总结

- **构建**：Maven/Gradle/npm 在流水线中执行；产出 jar/dist/镜像；跳过测试用 -DskipTests/-x test、npm ci 保证依赖一致。
- **制品**：构建产物存制品库或镜像仓库；版本与 Git/build 号关联。
- **Docker**：多阶段减小镜像、.dockerignore、固定 base tag；构建并 push 在 CI 中完成。
- **部署**：脚本/Docker/K8s 按规模选型；凭据用 Secrets；回滚用上一版本/上一镜像/rollback。
- **与 CI/CD**：构建阶段产出制品，部署阶段消费同一制品；质量门禁失败则不部署。
