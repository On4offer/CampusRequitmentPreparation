# 📌 Docker（Python 项目）常用命令与使用场景速查

> 配合《学习笔记》使用；通用 Docker 命令见《Java后端/Docker/常用命令与使用场景》。此处侧重 **Python 镜像构建、运行与 Compose**。

---

## 一、镜像与构建

### 1.1 通用命令（与 Java 一致）

| 命令 | 说明 | 示例 |
|------|------|------|
| `docker build -t myapp:1.0 .` | 构建镜像 | 在 Dockerfile 所在目录执行 |
| `docker run -d -p 8000:8000 --name web myapp:1.0` | 运行容器 | 映射 8000 端口 |
| `docker logs -f web` | 查看日志 | 排错 |
| `docker exec -it web /bin/sh` | 进入容器 | 调试 |

### 1.2 Python 镜像构建要点

| 要点 | 说明 |
|------|------|
| 先 COPY requirements.txt 再 `pip install` | 依赖不变时复用缓存 |
| 最后 COPY 应用代码 | 代码变更只重建最后几层 |
| 生产建议用 `python:3.x-slim` | 减小体积与攻击面 |

## 二、Dockerfile 示例（Flask / 通用 WSGI）

### 2.1 单阶段

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 8000
CMD ["gunicorn", "-b", "0.0.0.0:8000", "-w", "4", "run:app"]
```

### 2.2 多阶段（可选）

```dockerfile
# 构建阶段：如需编译依赖可在此装 gcc 等
FROM python:3.11-slim as builder
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir --user -r requirements.txt

# 运行阶段
FROM python:3.11-slim
WORKDIR /app
COPY --from=builder /root/.local /root/.local
ENV PATH=/root/.local/bin:$PATH
COPY . .
EXPOSE 8000
CMD ["gunicorn", "-b", "0.0.0.0:8000", "-w", "4", "run:app"]
```

## 三、Docker Compose 示例（Python + Redis）

| 服务 | 说明 |
|------|------|
| `web` | Flask/FastAPI 应用，依赖 Redis |
| `redis` | Redis 容器，供应用与 Celery 使用 |

```yaml
# docker-compose.yml 示例
services:
  web:
    build: .
    ports:
      - "8000:8000"
    environment:
      - REDIS_URL=redis://redis:6379/0
    depends_on:
      - redis
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
```

### Compose 常用命令

| 命令 | 说明 |
|------|------|
| `docker compose up -d` | 后台启动 |
| `docker compose down` | 停止并删除容器 |
| `docker compose build` | 重新构建镜像 |
| `docker compose logs -f web` | 查看 web 服务日志 |

## 四、常用场景速查

| 场景 | 推荐做法 |
|------|----------|
| 本地构建并跑 | `docker build -t myapp . && docker run -p 8000:8000 myapp` |
| 传入环境变量 | `docker run -e DATABASE_URL=... myapp` |
| 挂载代码做开发 | `docker run -v $(pwd):/app myapp`（可覆盖镜像内代码） |
| 生产部署 | 用 Compose 或 K8s 编排，镜像内用 Gunicorn/uvicorn |
