## **1.1 Flask 的起源与特点**

### 🧩 背景

- **Flask** 由 Armin Ronacher 主导，2010 年发布，是 Python 生态中**轻量级 Web 框架**的代表。
- 设计哲学：**微内核 + 可扩展**，只提供路由、请求/响应、模板等核心能力，其余通过扩展（Flask-SQLAlchemy、Flask-Login 等）按需引入。

------

### 🔹 核心特点

| 特性 | 描述 |
|------|------|
| **轻量** | 核心代码少，依赖少，适合小到中型项目与微服务 |
| **WSGI** | 基于 WSGI，可配合 Gunicorn、uWSGI 部署 |
| **Jinja2** | 内置 Jinja2 模板引擎 |
| **REST 友好** | 配合蓝图与扩展可方便构建 REST API |
| **生态丰富** | 扩展众多：ORM、认证、缓存、API 文档等 |

------

## **1.2 Flask 与 Django、FastAPI 对比**

| 对比项 | Flask | Django | FastAPI |
|--------|--------|--------|--------|
| **定位** | 微框架，灵活 | 全栈框架，大而全 | 现代异步 API 框架 |
| **学习曲线** | 低 | 中高 | 中 |
| **异步支持** | 需扩展 | 3.x 原生支持 | 原生异步 |
| **适用场景** | 中小项目、API、微服务 | 内容站、管理后台、全栈 | 高性能 API、自动文档 |

## **1.3 Flask 核心组件**

### Werkzeug

**定义**：Werkzeug 是一个 WSGI 工具库，为 Flask 提供底层的 HTTP 处理、路由、调试等能力。

**作用**：
- 处理 HTTP 请求和响应
- 路由分发
- 调试工具（DebuggedApplication）
- 安全工具（密码哈希、CSRF 保护）

### Jinja2

**定义**：Jinja2 是一个现代的模板引擎，用于生成 HTML、XML 或其他标记格式。

**特点**：
- 模板继承：支持基础模板和子模板
- 过滤器：提供丰富的内置过滤器
- 自定义扩展：支持自定义标签和过滤器
- 沙盒执行：支持受限的模板执行环境

### Click

**定义**：Click 是一个用于创建命令行界面的 Python 库，Flask 使用它来创建 CLI 命令。

**应用**：
- `flask run`：启动开发服务器
- `flask shell`：启动交互式 Shell
- 自定义命令：通过 `@app.cli.command()` 装饰器创建

## **1.4 Flask 应用生命周期**

### 启动流程

```
用户请求 → WSGI 服务器 → Flask 应用 → 路由分发 → 视图函数 → 响应处理 → WSGI 服务器 → 用户
```

**面试可答**：Flask 应用通过 WSGI 服务器（如 Gunicorn）接收请求，经过路由分发到对应的视图函数，视图函数处理后返回响应，最终由 WSGI 服务器返回给用户。

### 请求处理流程

1. **请求接收**：WSGI 服务器接收 HTTP 请求
2. **请求对象创建**：Flask 创建 `request` 对象，包含请求的所有信息
3. **路由匹配**：根据 URL 和 HTTP 方法匹配对应的视图函数
4. **中间件执行**：执行 `before_request` 钩子
5. **视图函数执行**：执行匹配的视图函数
6. **响应处理**：执行 `after_request` 钩子
7. **响应返回**：将响应返回给 WSGI 服务器

## **1.5 Flask 应用示例**

### 最小 Flask 应用

```python
from flask import Flask

# 创建 Flask 应用实例
app = Flask(__name__)

# 定义路由和视图函数
@app.route('/')
def hello():
    return 'Hello, World!'

# 运行应用
if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `Flask(__name__)`：创建应用实例，`__name__` 用于定位资源文件
- `@app.route('/')`：装饰器，定义路由规则
- `app.run(debug=True)`：启动开发服务器，`debug=True` 启用调试模式

### 带参数的路由

```python
from flask import Flask

app = Flask(__name__)

@app.route('/user/<username>')
def show_user_profile(username):
    return f'User: {username}'

@app.route('/post/<int:post_id>')
def show_post(post_id):
    return f'Post: {post_id}'

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `<username>`：字符串类型的路径参数
- `<int:post_id>`：整数类型的路径参数
- Flask 自动将路径参数转换为指定类型并传递给视图函数

## ⚠️ 常见坑与注意点

1. **现象**：生产环境使用 `app.run()`。**原因**：`app.run()` 是开发服务器，不适合生产环境。**正确做法**：使用 Gunicorn 或 uWSGI 等 WSGI 服务器。

2. **现象**：`debug=True` 导致性能问题。**原因**：调试模式会加载额外的调试工具，影响性能。**正确做法**：生产环境禁用调试模式。

3. **现象**：静态文件路径错误。**原因**：未正确设置 `static_folder` 参数。**正确做法**：使用 `url_for('static', filename='style.css')` 生成静态文件 URL。

4. **现象**：模板文件找不到。**原因**：模板文件未放在 `templates` 目录下。**正确做法**：将模板文件放在 `templates` 目录下，或使用 `template_folder` 参数指定。

## 2️⃣ 路由基础

### （1）路由定义

**定义**：路由是将 URL 映射到视图函数的机制，Flask 使用 `@app.route()` 装饰器定义路由。

**为什么需要路由**：路由是 Web 应用的入口，决定了不同的 URL 请求由哪个函数处理，是 Web 框架的核心功能。

### 基本路由

```python
from flask import Flask

app = Flask(__name__)

@app.route('/')
def index():
    return 'Hello, World!'

@app.route('/hello')
def hello():
    return 'Hello, Flask!'

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `@app.route('/')`：定义根路径 `/` 的路由
- `@app.route('/hello')`：定义 `/hello` 路径的路由
- 每个路由对应一个视图函数，函数返回的内容作为响应

### HTTP 方法

Flask 支持所有 HTTP 方法，通过 `methods` 参数指定。

```python
from flask import Flask, request

app = Flask(__name__)

@app.route('/login', methods=['GET', 'POST'])
def login():
    if request.method == 'POST':
        return 'Login POST request'
    else:
        return 'Login GET request'

@app.route('/create', methods=['POST'])
def create():
    return 'Create resource'

@app.route('/update', methods=['PUT', 'PATCH'])
def update():
    return 'Update resource'

@app.route('/delete', methods=['DELETE'])
def delete():
    return 'Delete resource'

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `methods=['GET', 'POST']`：指定路由支持的 HTTP 方法
- `request.method`：获取当前请求的 HTTP 方法
- 默认情况下，路由只支持 GET 方法

### （2）路由参数

### 路径参数

**定义**：路径参数是 URL 路径的一部分，Flask 会自动提取并传递给视图函数。

```python
from flask import Flask

app = Flask(__name__)

@app.route('/user/<username>')
def show_user(username):
    return f'User: {username}'

@app.route('/post/<int:post_id>')
def show_post(post_id):
    return f'Post: {post_id}'

@app.route('/page/<int:page_number>/<string:page_name>')
def show_page(page_number, page_name):
    return f'Page {page_number}: {page_name}'

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `<username>`：字符串类型的路径参数
- `<int:post_id>`：整数类型的路径参数
- `<string:page_name>`：显式指定字符串类型
- Flask 自动将路径参数转换为指定类型并传递给视图函数

### 路径参数类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `string` | 字符串（默认） | `<username>` |
| `int` | 整数 | `<int:post_id>` |
| `float` | 浮点数 | `<float:price>` |
| `path` | 路径（包含斜杠） | `<path:subpath>` |
| `uuid` | UUID | `<uuid:uuid_value>` |

```python
from flask import Flask

app = Flask(__name__)

@app.route('/price/<float:price>')
def show_price(price):
    return f'Price: {price}'

@app.route('/files/<path:filename>')
def show_file(filename):
    return f'File: {filename}'

if __name__ == '__main__':
    app.run(debug=True)
```

### URL 构建器

**定义**：`url_for()` 函数用于根据视图函数名生成 URL，避免硬编码 URL。

```python
from flask import Flask, url_for

app = Flask(__name__)

@app.route('/')
def index():
    return f'Index page'

@app.route('/user/<username>')
def profile(username):
    return f'Profile: {username}'

@app.route('/post/<int:post_id>')
def post(post_id):
    return f'Post: {post_id}'

@app.route('/test')
def test():
    index_url = url_for('index')
    profile_url = url_for('profile', username='john')
    post_url = url_for('post', post_id=1)
    return f'Index: {index_url}, Profile: {profile_url}, Post: {post_url}'

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `url_for('index')`：生成 `index` 视图函数的 URL
- `url_for('profile', username='john')`：生成带参数的 URL
- 使用 `url_for()` 可以避免硬编码 URL，便于维护

## 3️⃣ 请求处理

### （1）Request 对象

**定义**：`request` 对象包含了当前 HTTP 请求的所有信息，通过 `from flask import request` 导入。

**面试可答**：`request` 对象是 Flask 在请求处理过程中创建的全局对象，包含请求的 URL、参数、头部、表单数据等信息，使用 `request` 对象可以方便地获取请求信息。

### 获取请求信息

```python
from flask import Flask, request

app = Flask(__name__)

@app.route('/info')
def request_info():
    info = {
        'method': request.method,
        'url': request.url,
        'path': request.path,
        'args': request.args,
        'form': request.form,
        'json': request.json,
        'headers': dict(request.headers),
        'cookies': request.cookies,
        'remote_addr': request.remote_addr,
    }
    return info

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `request.method`：获取 HTTP 方法
- `request.url`：获取完整 URL
- `request.path`：获取路径部分
- `request.args`：获取查询参数（GET 参数）
- `request.form`：获取表单数据（POST 表单）
- `request.json`：获取 JSON 数据
- `request.headers`：获取请求头部
- `request.cookies`：获取 Cookie
- `request.remote_addr`：获取客户端 IP 地址

### 查询参数

```python
from flask import Flask, request

app = Flask(__name__)

@app.route('/search')
def search():
    query = request.args.get('q', '')
    page = request.args.get('page', 1, type=int)
    per_page = request.args.get('per_page', 10, type=int)
    return f'Search: {query}, Page: {page}, Per Page: {per_page}'

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `request.args.get('q', '')`：获取查询参数 `q`，默认值为空字符串
- `request.args.get('page', 1, type=int)`：获取查询参数 `page`，默认值为 1，类型转换为整数
- `request.args` 是一个类似字典的对象，支持 `.get()` 方法

### 表单数据

```python
from flask import Flask, request

app = Flask(__name__)

@app.route('/login', methods=['GET', 'POST'])
def login():
    if request.method == 'POST':
        username = request.form.get('username', '')
        password = request.form.get('password', '')
        return f'Username: {username}, Password: {password}'
    else:
        return '''
        <form method="post">
            <input type="text" name="username" placeholder="Username">
            <input type="password" name="password" placeholder="Password">
            <button type="submit">Login</button>
        </form>
        '''

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `request.form.get('username', '')`：获取表单字段 `username`，默认值为空字符串
- `request.form` 是一个类似字典的对象，包含表单提交的数据
- 表单数据通过 POST 方法提交，`Content-Type` 为 `application/x-www-form-urlencoded`

### JSON 数据

```python
from flask import Flask, request, jsonify

app = Flask(__name__)

@app.route('/api/data', methods=['POST'])
def receive_json():
    data = request.get_json()
    if data:
        name = data.get('name', '')
        age = data.get('age', 0)
        return jsonify({'name': name, 'age': age})
    else:
        return jsonify({'error': 'Invalid JSON'}), 400

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `request.get_json()`：获取 JSON 数据，返回 Python 字典
- `jsonify()`：将 Python 字典转换为 JSON 响应
- 如果 JSON 数据无效，返回 400 错误

### 文件上传

```python
from flask import Flask, request, jsonify
import os

app = Flask(__name__)
app.config['UPLOAD_FOLDER'] = 'uploads'
os.makedirs(app.config['UPLOAD_FOLDER'], exist_ok=True)

@app.route('/upload', methods=['POST'])
def upload_file():
    if 'file' not in request.files:
        return jsonify({'error': 'No file part'}), 400
    
    file = request.files['file']
    if file.filename == '':
        return jsonify({'error': 'No selected file'}), 400
    
    if file:
        filename = file.filename
        file.save(os.path.join(app.config['UPLOAD_FOLDER'], filename))
        return jsonify({'message': 'File uploaded successfully', 'filename': filename})

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `request.files`：获取上传的文件
- `file.save()`：保存文件到指定目录
- 需要检查文件是否存在和是否选择了文件

### （2）请求钩子

**定义**：请求钩子是在请求处理的不同阶段执行的函数，用于在请求处理前后执行一些操作。

**面试可答**：Flask 提供了 `before_request`、`after_request`、`teardown_request` 等请求钩子，可以在请求处理前后执行一些操作，如日志记录、权限检查、资源清理等。

### before_request

**定义**：在每个请求处理之前执行的函数，可以用于权限检查、日志记录等。

```python
from flask import Flask, request, g

app = Flask(__name__)

@app.before_request
def before_request():
    g.start_time = time.time()
    print(f'Before request: {request.method} {request.path}')

@app.route('/')
def index():
    return 'Hello, World!'

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `@app.before_request`：装饰器，在每个请求处理之前执行
- `g` 对象：Flask 的全局对象，用于在请求处理过程中存储数据
- 可以在 `before_request` 中进行权限检查，如果检查失败可以返回响应

### after_request

**定义**：在每个请求处理之后执行的函数，可以用于修改响应、添加响应头部等。

```python
from flask import Flask, request, make_response

app = Flask(__name__)

@app.after_request
def after_request(response):
    response.headers['X-Custom-Header'] = 'Custom Value'
    print(f'After request: {request.method} {request.path}')
    return response

@app.route('/')
def index():
    return 'Hello, World!'

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `@app.after_request`：装饰器，在每个请求处理之后执行
- `after_request` 函数接收响应对象，可以修改响应
- 必须返回响应对象

### teardown_request

**定义**：在每个请求处理之后执行的函数，即使发生异常也会执行，用于资源清理。

```python
from flask import Flask, g

app = Flask(__name__)

@app.teardown_request
def teardown_request(exception=None):
    print(f'Teardown request, exception: {exception}')

@app.route('/')
def index():
    return 'Hello, World!'

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `@app.teardown_request`：装饰器，在每个请求处理之后执行
- `teardown_request` 函数接收异常对象，即使发生异常也会执行
- 适合用于数据库连接关闭、文件句柄关闭等资源清理

## 4️⃣ 响应处理

### （1）Response 对象

**定义**：`Response` 对象包含了 HTTP 响应的所有信息，Flask 会自动将视图函数的返回值转换为 `Response` 对象。

**面试可答**：Flask 支持多种响应类型，包括字符串、字典、元组、`Response` 对象等，Flask 会自动将返回值转换为 `Response` 对象并返回给客户端。

### 基本响应

```python
from flask import Flask

app = Flask(__name__)

@app.route('/')
def index():
    return 'Hello, World!'

@app.route('/html')
def html():
    return '<h1>Hello, Flask!</h1>'

@app.route('/json')
def json_response():
    return {'message': 'Hello, Flask!', 'status': 'success'}

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- 返回字符串：Flask 自动转换为 HTML 响应
- 返回字典：Flask 自动转换为 JSON 响应
- 返回 HTML：直接返回 HTML 字符串

### 自定义响应

```python
from flask import Flask, make_response, jsonify

app = Flask(__name__)

@app.route('/custom')
def custom_response():
    response = make_response(jsonify({'message': 'Hello, Flask!'}))
    response.status_code = 200
    response.headers['X-Custom-Header'] = 'Custom Value'
    return response

@app.route('/redirect')
def redirect_response():
    return 'Redirecting...', 302, {'Location': '/'}

if __name__ == '__main__':
    app.run(debug=True)
```

## ⚠️ 常见坑与注意点

1. **现象**：路由参数类型转换失败。**原因**：路径参数类型与实际值不匹配。**正确做法**：使用正确的类型转换器，或在视图函数中处理类型转换错误。

2. **现象**：`request.json` 为 None。**原因**：请求的 `Content-Type` 不是 `application/json`。**正确做法**：确保请求的 `Content-Type` 为 `application/json`，或使用 `request.get_json()`。

3. **现象**：`before_request` 中返回响应后视图函数不执行。**原因**：`before_request` 中返回响应会中断请求处理流程。**正确做法**：`before_request` 中只进行预处理，不返回响应。

4. **现象**：文件上传失败。**原因**：表单未设置 `enctype="multipart/form-data"`。**正确做法**：在 HTML 表单中设置 `enctype="multipart/form-data"`。

5. **现象**：响应头部设置不生效。**原因**：`after_request` 中未返回响应对象。**正确做法**：`after_request` 函数必须返回响应对象。

## 1️⃣ 蓝图基础

### （1）蓝图概述

**定义**：蓝图是 Flask 中用于组织应用的一种机制，允许将应用拆分为多个模块，每个模块包含自己的路由、模板、静态文件等。

**为什么需要蓝图**：随着应用规模的增长，将所有路由放在一个文件中会导致代码难以维护。蓝图允许将应用拆分为多个模块，每个模块负责一部分功能，提高代码的可维护性和可扩展性。

**面试可答**：蓝图是 Flask 中用于组织应用的机制，类似于 Django 的 app，可以将应用拆分为多个模块，每个模块包含自己的路由、模板、静态文件等，提高代码的可维护性和可扩展性。

### 蓝图特点

| 特性 | 描述 |
|------|------|
| **模块化** | 将应用拆分为多个模块，每个模块负责一部分功能 |
| **可重用** | 蓝图可以在多个应用中重用 |
| **命名空间** | 蓝图可以为路由添加 URL 前缀，避免路由冲突 |
| **独立配置** | 蓝图可以有自己的配置和钩子函数 |

### （2）应用工厂

### 应用工厂模式

**定义**：应用工厂是一个函数，用于创建和配置 Flask 应用实例。

**为什么需要应用工厂**：应用工厂模式允许在测试时创建多个应用实例，每个实例有不同的配置，便于测试和部署。

```python
# app/__init__.py
from flask import Flask
from app.auth import bp as auth_bp
from app.blog import bp as blog_bp

def create_app(config_name='default'):
    app = Flask(__name__)
    
    # 加载配置
    from config import config
    app.config.from_object(config[config_name])
    
    # 注册蓝图
    app.register_blueprint(auth_bp, url_prefix='/auth')
    app.register_blueprint(blog_bp, url_prefix='/blog')
    
    return app
```

**代码说明**：
- `create_app(config_name)`：应用工厂函数，接受配置名称
- `app.config.from_object()`：从配置对象加载配置
- `app.register_blueprint()`：注册蓝图到应用

### （1）模板继承基础

**定义**：模板继承允许创建一个基础模板，其他模板可以继承基础模板并重写部分内容。

**为什么需要模板继承**：模板继承可以避免重复代码，提高模板的可维护性。


**代码说明**：
- `{% block title %}{% endblock %}`：定义可重写的块
- `{{ url_for('blog.index') }}`：生成 URL
- `{% block content %}{% endblock %}`：定义内容块

**代码说明**：

## 1️⃣ 配置管理

### （1）配置基础

**定义**：配置管理是 Flask 应用中管理不同环境（开发、测试、生产）配置的机制。

**为什么需要配置管理**：不同环境需要不同的配置，如数据库连接、调试模式、密钥等，配置管理可以方便地切换环境配置。

**面试可答**：Flask 提供了多种配置方式，包括直接设置、从对象加载、从文件加载等，可以根据不同的环境使用不同的配置。

### 配置方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **直接设置** | `app.config['KEY'] = 'value'` | 小型应用、临时配置 |
| **从对象加载** | `app.config.from_object()` | 推荐方式，支持多环境 |
| **从文件加载** | `app.config.from_pyfile()` | 配置文件管理 |
| **从环境变量加载** | `app.config.from_envvar()` | 生产环境、容器化部署 |

### 基本配置

```python
from flask import Flask

app = Flask(__name__)

# 直接设置配置
app.config['DEBUG'] = True
app.config['SECRET_KEY'] = 'dev-secret-key'
app.config['DATABASE_URL'] = 'sqlite:///app.db'

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `app.config['DEBUG'] = True`：设置调试模式
- `app.config['SECRET_KEY'] = 'dev-secret-key'`：设置密钥
- `app.config['DATABASE_URL'] = 'sqlite:///app.db'`：设置数据库连接

```python
basedir = os.path.abspath(os.path.dirname(__file__))

class Config:
    """基础配置类"""
    SECRET_KEY = os.environ.get('SECRET_KEY') or 'dev-secret-key'
    SQLALCHEMY_DATABASE_URI = os.environ.get('DATABASE_URL') or \
        'sqlite:///' + os.path.join(basedir, 'app.db')
    SQLALCHEMY_TRACK_MODIFICATIONS = False
    
    # 邮件配置
    MAIL_SERVER = os.environ.get('MAIL_SERVER')
    MAIL_PORT = int(os.environ.get('MAIL_PORT') or 25)
    MAIL_USE_TLS = os.environ.get('MAIL_USE_TLS', 'true').lower() in ['true', 'on', '1']
    MAIL_USERNAME = os.environ.get('MAIL_USERNAME')
    MAIL_PASSWORD = os.environ.get('MAIL_PASSWORD')
    
    # 分页配置
    ITEMS_PER_PAGE = 10

class DevelopmentConfig(Config):
    """开发环境配置"""
    DEBUG = True
    SQLALCHEMY_ECHO = True

class ProductionConfig(Config):
    """生产环境配置"""
    DEBUG = False
    SQLALCHEMY_ECHO = False

class TestingConfig(Config):
    """测试环境配置"""
    TESTING = True
    SQLALCHEMY_DATABASE_URI = 'sqlite:///:memory:'
    WTF_CSRF_ENABLED = False

config = {
    'development': DevelopmentConfig,
    'production': ProductionConfig,
    'testing': TestingConfig,
    'default': DevelopmentConfig
}
```

**代码说明**：
- `Config`：基础配置类，包含所有环境的通用配置
- `DevelopmentConfig`：开发环境配置，启用调试和 SQL 日志
- `ProductionConfig`：生产环境配置，禁用调试和 SQL 日志
- `TestingConfig`：测试环境配置，使用内存数据库
- `config`：配置字典，根据配置名称返回对应的配置类

### 加载配置

```python
# app/__init__.py
from flask import Flask
from config import config
```

**代码说明**：

- `config_name`：配置名称，如 'development'、'production'、'testing'

### （3）认证扩展

### Flask-Login

**安装依赖**：
```bash
pip install flask-login
```

**基本使用**：
```python
from flask import Flask, render_template, redirect, url_for, request
from flask_login import LoginManager, UserMixin, login_user, login_required, logout_user, current_user
from werkzeug.security import generate_password_hash, check_password_hash

app = Flask(__name__)
app.config['SECRET_KEY'] = 'your-secret-key'

login_manager = LoginManager()
login_manager.init_app(app)
login_manager.login_view = 'login'

# 模拟用户数据库
class User(UserMixin):
    def __init__(self, id, username, password):
        self.id = id
        self.username = username
        self.password_hash = generate_password_hash(password)
    
    def verify_password(self, password):
        return check_password_hash(self.password_hash, password)

users = [
    User(1, 'admin', 'admin'),
    User(2, 'user', 'user')
]

@login_manager.user_loader
def load_user(user_id):
    return next((user for user in users if user.id == int(user_id)), None)

@app.route('/login', methods=['GET', 'POST'])
def login():
    if request.method == 'POST':
        username = request.form.get('username')
        password = request.form.get('password')
        user = next((u for u in users if u.username == username), None)
        if user and user.verify_password(password):
            login_user(user)
            return redirect(url_for('protected'))
    return render_template('login.html')

@app.route('/protected')
@login_required
def protected():
    return f'Hello, {current_user.username}!'

@app.route('/logout')
@login_required
def logout():
    logout_user()
    return redirect(url_for('login'))

if __name__ == '__main__':
    app.run(debug=True)
```

**代码说明**：
- `LoginManager()`：创建登录管理器
- `login_manager.init_app(app)`：初始化登录管理器
- `@login_manager.user_loader`：用户加载器
- `login_user(user)`：登录用户
- `logout_user()`：登出用户
- `@login_required`：登录保护装饰器
- `current_user`：当前登录用户

## ⚠️ 常见坑与注意点

1. **现象**：生产环境使用 `app.run()`。**原因**：`app.run()` 是开发服务器，不适合生产环境。**正确做法**：使用 Gunicorn 或 uWSGI 等 WSGI 服务器。

2. **现象**：配置文件中的敏感信息泄露。**原因**：配置文件包含敏感信息并提交到版本控制系统。**正确做法**：使用环境变量存储敏感信息，配置文件使用默认值。

3. **现象**：数据库迁移失败。**原因**：数据库状态与迁移脚本不匹配。**正确做法**：检查迁移脚本，必要时重置数据库或手动修复。

4. **现象**：表单 CSRF 验证失败。**原因**：未设置 SECRET_KEY 或表单未包含 CSRF 令牌。**正确做法**：设置 SECRET_KEY，在表单中包含 `{{ form.hidden_tag() }}`。

5. **现象**：容器化部署后无法访问数据库。**原因**：容器网络配置错误或数据库服务未启动。**正确做法**：检查 Docker Compose 配置，确保服务依赖和网络配置正确。

- 理解常用扩展的使用，可以快速集成数据库、表单、认证等功能
- 掌握生产部署，可以将 Flask 应用投入生产环境
- 理解容器化部署，可以提高应用的可移植性和可扩展性
- 掌握 CI/CD，可以实现自动化测试和部署

> 💡 **重点**：配置管理、常用扩展和生产部署是将 Flask 应用投入生产的关键，掌握这些功能可以开发完整的生产级应用。
