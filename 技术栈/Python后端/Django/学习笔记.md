# 🧱 第1章：Django 概述与核心思想

------

## **1.1 Django 的起源与特点**

### 🧩 背景

- **Django** 由 Lawrence Journal-World 团队开发，2005 年开源，是 Python 生态中**全栈型 Web 框架**的代表。
- 设计哲学：**Battery Included**（自带电池）——ORM、Admin、认证、表单、模板、中间件等开箱即用。

------

### 🔹 核心特点

| 特性 | 描述 |
|------|------|
| **MTV 架构** | Model（模型）、Template（模板）、View（视图），对应 MVC 的变体 |
| **自带 Admin** | 根据模型自动生成后台管理界面，适合内容与运营后台 |
| **ORM** | 内置 Django ORM，迁移（migrations）管理表结构变更 |
| **安全** | CSRF、XSS、SQL 注入等防护内置，适合企业级应用 |
| **生态成熟** | 用户系统、REST 框架（DRF）、缓存、国际化等生态完善 |

------

## **1.2 Django 与 Flask、FastAPI 对比**

| 对比项 | Django | Flask | FastAPI |
|--------|--------|--------|---------|
| **定位** | 全栈、大而全 | 微框架、灵活 | 现代 API、异步 |
| **学习曲线** | 中高 | 低 | 中 |
| **ORM** | 内置 | 需扩展（如 SQLAlchemy） | 需自行集成 |
| **Admin** | 内置 | 无，需扩展 | 无 |
| **适用场景** | 内容站、管理后台、全栈项目 | 轻量 Web、API、微服务 | 高性能 API、前后端分离 |

------

### 🧠 关键词总结

> Django 是 **“全栈 + 约定优于配置”** 的 Web 框架，适合中大型 Web 应用与后台系统，校招常考 MTV、ORM、中间件、DRF（Django REST framework）。

------

### 📘 面试延展题

**问：Django 的 MTV 和 MVC 有什么区别？**

**答：** MTV 中 Model 对应 M，Template 对应 V（展示），View 对应 C（控制逻辑）；Django 的 View 负责处理请求与返回响应，Template 负责 HTML 渲染，本质与 MVC 思想一致，只是命名不同。
