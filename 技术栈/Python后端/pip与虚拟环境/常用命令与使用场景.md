# 📌 pip 与虚拟环境 常用命令与使用场景速查

> 配合《学习笔记》使用，侧重日常开发中的**命令与场景**，便于速查与上手。

---

## 一、虚拟环境（venv）

### 1.1 创建与激活

| 命令 | 说明 | 使用场景 |
|------|------|----------|
| `python -m venv .venv` | 在当前目录创建 `.venv` 虚拟环境 | 新项目初始化 |
| `python -m venv .venv --without-pip` | 创建不含 pip 的 venv（少见） | 极小化环境 |
| `.venv\Scripts\activate`（Windows） | 激活虚拟环境 | 进入项目前 |
| `source .venv/bin/activate`（Linux/Mac） | 激活虚拟环境 | 进入项目前 |
| `deactivate` | 退出当前虚拟环境 | 切换项目或退出 |

### 1.2 常用参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--system-site-packages` | 继承系统 site-packages | 需用系统已装包时 |
| `--clear` | 清空已有 venv 再创建 | 重建干净环境 |

## 二、pip 常用命令

### 2.1 安装与卸载

| 命令 | 说明 | 使用场景 |
|------|------|----------|
| `pip install <包名>` | 安装最新版 | 临时试用某库 |
| `pip install <包名>==1.0.0` | 安装指定版本 | 锁定版本 |
| `pip install -r requirements.txt` | 按文件安装依赖 | **最常用**，复现环境 |
| `pip uninstall <包名>` | 卸载包 | 移除不需要的依赖 |
| `pip install -e .` | 以可编辑模式安装当前项目 | 本地开发库时 |

### 2.2 查询与导出

| 命令 | 说明 | 使用场景 |
|------|------|----------|
| `pip list` | 列出已安装包 | 查看当前环境 |
| `pip show <包名>` | 显示包信息与依赖 | 排查版本与路径 |
| `pip freeze` | 输出已安装包及版本（pip 格式） | 生成 requirements.txt 初稿 |
| `pip freeze > requirements.txt` | 导出当前环境依赖 | 提交依赖清单 |

### 2.3 其他常用参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `-U` / `--upgrade` | 升级到最新版 | `pip install -U requests` |
| `-i <url>` | 使用国内镜像源 | `pip install -i https://pypi.tuna.tsinghua.edu.cn/simple flask` |
| `--no-cache-dir` | 不使用缓存 | CI 或磁盘紧张时 |
| `-q` | 安静模式 | 减少输出 |

## 三、requirements.txt 常用写法

```text
# 精确版本（推荐生产）
Django==4.2.0
requests==2.31.0

# 兼容版本
flask>=2.0,<3.0
sqlalchemy~=2.0.0   # 兼容 2.0.x

# 从 Git 安装（示例）
git+https://github.com/xxx/yyy.git@v1.0.0
```

## 四、典型使用场景

| 场景 | 推荐步骤 |
|------|----------|
| **新项目搭建** | `python -m venv .venv` → 激活 → `pip install -r requirements.txt` |
| **记录当前依赖** | 激活 venv → `pip freeze > requirements.txt` |
| **CI 中安装依赖** | `pip install -r requirements.txt --no-cache-dir` |
| **国内加速** | `pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple` |
