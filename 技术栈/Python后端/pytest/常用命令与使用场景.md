# 📌 pytest 常用命令与使用场景速查

> 配合《学习笔记》使用，侧重**命令行、常用参数、标记与配置**，便于速查与 CI 集成。

---

## 一、常用命令速查

### 1.1 运行测试

| 命令 | 说明 | 使用场景 |
|------|------|----------|
| `pytest` | 当前目录及子目录收集并运行测试 | 本地默认 |
| `pytest path/to/test_file.py` | 运行指定文件 | 单文件调试 |
| `pytest path/to/test_file.py::test_func` | 运行指定用例 | 单用例调试 |
| `pytest -v` | 详细输出（verbose） | 看每个用例名 |
| `pytest -q` | 安静模式 | CI 中减少输出 |
| `pytest -x` | 首次失败即停止 | 快速定位第一个失败 |
| `pytest --lf` | 只运行上次失败的用例 | 修完 bug 后重跑 |
| `pytest -k "keyword"` | 按名称匹配用例 | 只跑含某关键字的测试 |

### 1.2 覆盖率与报告

| 命令 | 说明 | 使用场景 |
|------|------|----------|
| `pytest --cov=src --cov-report=html` | 生成覆盖率并输出 HTML | 查看未覆盖代码 |
| `pytest --cov=src --cov-report=term-missing` | 终端输出并标出未覆盖行 | 快速看缺口 |
| `pytest -n auto`（需 pytest-xdist） | 多进程并行 | 加快大批量测试 |

### 1.3 其他常用参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `-s` | 不捕获 stdout/stderr | 看到 print 输出 |
| `-m "markname"` | 只运行指定标记的用例 | `pytest -m "slow"` |
| `--ignore=dir` | 忽略目录 | 排除无关目录 |
| `--maxfail=N` | 最多失败 N 个后停止 | `--maxfail=1` 同 `-x` |

## 二、pytest.ini / pyproject.toml 配置片段

### 2.1 常用配置项

```ini
# pytest.ini
[pytest]
testpaths = tests
python_files = test_*.py
python_functions = test_*
addopts = -v --tb=short
markers =
    slow: 标记慢速测试
    integration: 集成测试
```

### 2.2 在 pyproject.toml 中

```toml
[tool.pytest.ini_options]
testpaths = ["tests"]
addopts = "-v"
markers = ["slow: 慢速测试"]
```

## 三、常用场景速查

| 场景 | 推荐命令 |
|------|----------|
| 本地全量测试 | `pytest` 或 `pytest -v` |
| 只跑某模块 | `pytest tests/test_api.py` |
| 只跑某一条 | `pytest tests/test_api.py::test_create_user` |
| 失败后只重跑失败 | `pytest --lf` |
| 看覆盖率 | `pytest --cov=src --cov-report=term-missing` |
| CI 中跑并生成覆盖率 | `pytest -q --cov=src --cov-report=xml` |
