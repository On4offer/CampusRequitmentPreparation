# 🧱 第1章：pytest 概述与核心思想

------

## **1.1 pytest 简介**

### 🧩 背景

- **pytest** 是 Python 生态中最流行的**测试框架**之一，基于「约定优于配置」：以 `test_` 开头的函数/类会被自动发现并执行。
- 相比 unittest，语法更简洁、断言更易读、插件丰富（覆盖率、并行、参数化、Fixture 等）。

------

### 🔹 核心特点

| 特性 | 描述 |
|------|------|
| **零样板** | 无需继承 TestCase，普通函数即可 |
| **断言友好** | 直接写 `assert x == 1`，失败时自动展示上下文 |
| **Fixture** | 依赖注入式的前后置与共享资源 |
| **参数化** | `@pytest.mark.parametrize` 多组输入一次写完 |
| **插件生态** | pytest-cov、pytest-xdist、pytest-asyncio 等 |

------

## **1.2 pytest 与 unittest 对比**

| 对比项 | pytest | unittest |
|--------|--------|----------|
| **写法** | `def test_xxx(): assert ...` | 继承 TestCase，`self.assertEqual()` |
| **发现规则** | `test_*.py`、`*_test.py`、`test_` 函数/类 | 继承 TestCase 的 test 方法 |
| **Fixture** | 内置 fixture + 自定义 | setUp/tearDown |
| **参数化** | 内置 parametrize | 需自己循环或第三方 |

------

### 🧠 关键词总结

> pytest 是 **“断言 + Fixture + 参数化”** 的 Python 测试框架，适合单元测试与接口测试，与 Java 的 JUnit 可类比。

------

### 📘 面试延展题

**问：pytest 的 Fixture 和 unittest 的 setUp 有什么区别？**

**答：** Fixture 可被任意测试函数通过参数按需注入，作用域可选 function/class/module/session，且可复用、组合；setUp 是类内固定生命周期，灵活性不如 Fixture。
