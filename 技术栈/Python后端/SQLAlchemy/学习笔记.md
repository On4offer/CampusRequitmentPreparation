# 🧱 第1章：SQLAlchemy 概述与核心思想

------

## **1.1 SQLAlchemy 简介**

### 🧩 背景

- **SQLAlchemy** 是 Python 生态中最主流的 **ORM（Object-Relational Mapping）** 与 **SQL 工具库**，由 Mike Bayer 等人维护。
- 既支持**声明式 ORM**（类似 Java JPA/Hibernate），也支持**核心 SQL 表达式**（Core），可混用。

------

### 🔹 核心组成

| 层级 | 说明 |
|------|------|
| **ORM** | 通过模型类与 Session 进行增删改查，面向对象 |
| **Core** | 表结构、SQL 表达式、连接池，可脱离 ORM 写原生 SQL |
| **Engine** | 数据库连接与连接池管理 |
| **Session** | 工作单元（Unit of Work），管理事务与对象状态 |

------

## **1.2 SQLAlchemy 1.x 与 2.0**

| 对比项 | 1.x | 2.0（推荐） |
|--------|-----|-------------|
| **风格** | 混用多种写法 | 统一 2.0 风格（select/update/delete 函数） |
| **异步** | 需额外异步层 | 原生 asyncio 支持（AsyncEngine、AsyncSession） |
| **类型与映射** | `Column(Integer)` 等 | 推荐 `Mapped[int]` + `mapped_column()` |

------

### 🧠 关键词总结

> SQLAlchemy 是 Python 的 **ORM + SQL 工具库**，通过 Engine 连接数据库、Session 管理事务与对象生命周期，支持同步与异步，与 Java 的 MyBatis/JPA 可类比。

------

### 📘 面试延展题

**问：ORM 的 N+1 查询问题如何避免？**

**答：** 使用 **eager loading**：`selectinload()`、`joinedload()` 等在一开始加载关联数据，避免在循环中多次访问关联属性触发额外查询。SQLAlchemy 中常用 `options(selectinload(Model.relation))`。
