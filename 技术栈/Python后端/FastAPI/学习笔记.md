# 🧱 第1章：FastAPI 概述与核心思想

------

## **1.1 FastAPI 的起源与特点**

### 🧩 背景

- **FastAPI** 由 Sebastián Ramírez 创建，2018 年发布，基于 **Starlette**（ASGI 框架）与 **Pydantic**（数据校验）。
- 定位：**现代、高性能、带自动文档** 的 Python Web API 框架，原生支持 **async/await**。

------

### 🔹 核心特点

| 特性 | 描述 |
|------|------|
| **高性能** | 基于 ASGI，异步 I/O，性能接近 Node.js/Go 同类框架 |
| **自动文档** | 集成 OpenAPI（Swagger）与 ReDoc，零配置生成交互文档 |
| **类型提示驱动** | 基于 Pydantic 与 type hints，自动校验请求体、查询参数与响应 |
| **依赖注入** | 内置 DI 系统，便于复用鉴权、数据库会话等逻辑 |
| **标准开放** | 基于 OpenAPI 3.0、JSON Schema，便于与前端/网关对接 |

------

## **1.2 FastAPI 与 Flask、Django 对比**

| 对比项 | FastAPI | Flask | Django |
|--------|---------|--------|--------|
| **协议** | ASGI（异步） | WSGI（同步为主） | ASGI/WSGI |
| **类型与校验** | 原生 Pydantic | 需扩展 | 需扩展 |
| **API 文档** | 内置 Swagger/ReDoc | 需扩展 | 需扩展 |
| **适用场景** | 高性能 API、微服务、前后端分离 | 轻量 Web、传统 MVC | 全栈、Admin、内容站 |

------

### 🧠 关键词总结

> FastAPI 是 **“类型安全 + 自动文档 + 异步”** 的 ASGI API 框架，适合校招与生产中的 REST/API 开发，面试常考其与 Flask、异步编程的结合。

------

### 📘 面试延展题

**问：FastAPI 为什么性能好？**

**答：** 基于 ASGI 异步 I/O；路由与校验在 C 扩展（如 Pydantic、ujson）上优化；无同步阻塞时能高并发处理请求。与 Flask 同步模型相比，在 I/O 密集场景下优势明显。

# 🧱 第2章：路由、请求体与响应模型

## 1️⃣ 路由定义

### 基本路由

**基本语法**：
```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/items/{item_id}")
async def read_item(item_id: int):
    return {"item_id": item_id}

@app.post("/items/")
async def create_item(item: Item):
    return item
```

**HTTP 方法**：
- `@app.get()`：GET 请求
- `@app.post()`：POST 请求
- `@app.put()`：PUT 请求
- `@app.delete()`：DELETE 请求
- `@app.patch()`：PATCH 请求
- `@app.options()`：OPTIONS 请求
- `@app.head()`：HEAD 请求
- `@app.trace()`：TRACE 请求

### 路径参数

**路径参数类型**：
```python
# 路径参数类型转换
@app.get("/items/{item_id}")
async def read_item(item_id: int):
    return {"item_id": item_id}

# 路径参数校验
from fastapi import Path

@app.get("/items/{item_id}")
async def read_item(
    item_id: int = Path(..., ge=1, le=1000, description="The ID of the item"
)):
    return {"item_id": item_id}
```

**路径转换器**：
- `int`：整数
- `float`：浮点数
- `str`：字符串
- `bool`：布尔值
- `path`：路径（包含斜杠）

### 查询参数

**基本用法**：
```python
@app.get("/items/")
async def read_items(
    skip: int = 0,
    limit: int = 10,
    q: str = None
):
    return {"skip": skip, "limit": limit, "q": q}
```

**查询参数校验**：
```python
from fastapi import Query

@app.get("/items/")
async def read_items(
    q: str = Query(None, min_length=3, max_length=50, regex="^[a-zA-Z0-9]+$")
):
    return {"q": q}
```

### 混合参数

**路径参数 + 查询参数**：
```python
@app.get("/items/{item_id}")
async def read_item(
    item_id: int,
    q: str = None,
    short: bool = False
):
    item = {"item_id": item_id}
    if q:
        item.update({"q": q})
    if not short:
        item.update({"description": "This is an amazing item"})
    return item
```

## 3️⃣ 响应模型

### 基本响应

**默认响应**：
```python

```


**排除字段**：
```python

    name: str

@app.get("/items/{item_id}", response_model=Item, response_model_exclude={"password"})

    return {"name": "Foo", "price": 50.2, "password": "secret"}
```

**包含字段**：
```python
@app.get("/items/{item_id}", response_model=Item, response_model_include={"name", "price"})
```

**常见状态码**：
- `200 OK`：成功
- `201 Created`：创建成功
- `204 No Content`：无内容
- `400 Bad Request`：请求错误
- `401 Unauthorized`：未授权
- `404 Not Found`：资源不存在

## ⚠️ 常见坑与注意点

1. **现象**：路径参数类型错误。**原因**：路径参数类型与函数参数类型不匹配。**正确做法**：确保路径参数类型与函数参数类型一致，FastAPI 会自动进行类型转换。

2. **现象**：请求体验证失败。**原因**：请求体数据不符合 Pydantic 模型定义。**正确做法**：检查请求体数据结构，确保符合模型定义，查看 FastAPI 返回的详细错误信息。

3. **现象**：响应模型字段缺失。**原因**：返回的数据与响应模型不匹配。**正确做法**：确保返回的数据包含响应模型要求的所有字段，或使用默认值。

4. **现象**：路由冲突。**原因**：多个路由路径模式相同。**正确做法**：确保路由路径唯一，避免路径冲突。

5. **现象**：文件上传失败。**原因**：未安装 python-multipart 依赖。**正确做法**：安装 python-multipart 依赖包。

6. **现象**：参数验证错误。**原因**：参数值不符合验证规则。**正确做法**：查看错误信息，调整参数值或验证规则。

# 🧱 第3章：依赖注入与中间件

## 1️⃣ 依赖注入基础

### 什么是依赖注入

**概念**：依赖注入（Dependency Injection, DI）是一种设计模式，用于将对象的创建和依赖关系的管理与对象的使用分离。在 FastAPI 中，依赖注入系统允许你声明所需的依赖项，FastAPI 会自动处理它们的创建和传递。

**优势**：
- **代码复用**：将重复的逻辑（如认证、数据库连接）提取为依赖项
- **模块化**：依赖项可以独立测试和维护
- **类型安全**：依赖项支持类型提示和验证
- **自动文档**：依赖项会自动集成到 API 文档中

### 基本依赖

**简单依赖**：
```python
from fastapi import Depends

async def get_query_token(token: str):
    return token

@app.get("/items/")
async def read_items(token: str = Depends(get_query_token)):
    return {"token": token}
```

**依赖函数**：
```python
async def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

@app.get("/items/")
async def read_items(db: Session = Depends(get_db)):
    return db.query(Item).all()
```

### 依赖链

**依赖嵌套**：
```python
async def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

async def get_current_user(db: Session = Depends(get_db), token: str = Depends(get_query_token)):
    user = db.query(User).filter(User.token == token).first()
    if not user:
        raise HTTPException(status_code=401, detail="Unauthorized")
    return user

@app.get("/items/")
async def read_items(current_user: User = Depends(get_current_user)):
    return {"user": current_user.username, "items": []}
```

## 3️⃣ 中间件

### 什么是中间件

**概念**：中间件是在请求处理过程中执行的代码，位于客户端和路由处理函数之间。中间件可以修改请求和响应，执行额外的逻辑，如日志记录、认证、CORS 等。

**执行顺序**：
1. 客户端发送请求
2. 中间件处理请求（可以修改请求）
3. 路由处理函数处理请求
4. 中间件处理响应（可以修改响应）
5. 响应返回给客户端

### 创建中间件

**基本中间件**：
```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse

app = FastAPI()

@app.middleware("http")
async def add_process_time_header(request: Request, call_next):
    start_time = time.time()
    response = await call_next(request)
    process_time = time.time() - start_time
    response.headers["X-Process-Time"] = str(process_time)
    return response
```

**中间件逻辑**：
- `request`：当前请求对象
- `call_next`：调用下一个中间件或路由处理函数的函数
- 返回值：修改后的响应对象

### 常用中间件

**CORS 中间件**：
```python
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI()

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 允许的源
    allow_credentials=True,
    allow_methods=["*"],  # 允许的 HTTP 方法
    allow_headers=["*"],  # 允许的 HTTP 头部
)
```

**认证中间件**：
```python
@app.middleware("http")
async def auth_middleware(request: Request, call_next):
    token = request.headers.get("Authorization")
    if not token:
        return JSONResponse(
            status_code=401,
            content={"detail": "Authorization header required"}
        )
    # 验证 token
    if token != "Bearer secret":
        return JSONResponse(
            status_code=401,
            content={"detail": "Invalid token"}
        )
    response = await call_next(request)
    return response
```

**日志中间件**：
```python
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

@app.middleware("http")
async def log_requests(request: Request, call_next):
    logger.info(f"Request: {request.method} {request.url}")
    response = await call_next(request)
    logger.info(f"Response: {response.status_code}")
    return response
```

## 4️⃣ 异常处理

### 基本异常

**HTTP 异常**：
```python
from fastapi import FastAPI, HTTPException

app = FastAPI()

@app.get("/items/{item_id}")
async def read_item(item_id: int):
    if item_id not in items:
        raise HTTPException(
            status_code=404,
            detail="Item not found",
            headers={"X-Error": "Item not found"},
        )
    return {"item": items[item_id]}
```

**自定义异常**：
```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse

class CustomException(Exception):
    def __init__(self, name: str):
        self.name = name

app = FastAPI()

@app.exception_handler(CustomException)
async def custom_exception_handler(request: Request, exc: CustomException):
    return JSONResponse(
        status_code=418,
        content={"message": f"I am a teapot {exc.name}"},
    )

@app.get("/items/{item_id}")
async def read_item(item_id: int):
    if item_id == 42:
        raise CustomException(name="Custom Error")
    return {"item_id": item_id}
```

### 全局异常处理

**全局异常处理器**：
```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse

app = FastAPI()

@app.exception_handler(Exception)
async def global_exception_handler(request: Request, exc: Exception):
    return JSONResponse(
        status_code=500,
        content={"detail": "Internal server error"},
    )

@app.get("/items/{item_id}")
async def read_item(item_id: int):
    if item_id == 42:
        raise Exception("Something went wrong")
```

## ⚠️ 常见坑与注意点

1. **现象**：依赖注入循环依赖。**原因**：依赖项之间相互依赖。**正确做法**：重构代码，避免循环依赖，或使用依赖注入容器。

2. **现象**：中间件执行顺序问题。**原因**：中间件的添加顺序影响执行顺序。**正确做法**：注意中间件的添加顺序，确保逻辑正确。

3. **现象**：依赖注入性能问题。**原因**：复杂的依赖链会影响性能。**正确做法**：简化依赖链，使用缓存或单例模式。

4. **现象**：中间件修改请求/响应失败。**原因**：中间件未正确处理请求/响应对象。**正确做法**：确保中间件正确处理请求和响应，遵循 FastAPI 的中间件规范。

5. **现象**：异常处理不生效。**原因**：异常处理器注册不正确或异常类型不匹配。**正确做法**：确保异常处理器正确注册，异常类型匹配。

6. **现象**：依赖注入与路径参数冲突。**原因**：依赖项的参数与路径参数同名。**正确做法**：使用不同的参数名称，或使用依赖函数封装。

## 1️⃣ 异步编程基础

### 什么是异步编程

**概念**：异步编程是一种编程范式，允许程序在等待I/O操作（如网络请求、数据库查询）时继续执行其他任务，而不是阻塞等待。在FastAPI中，异步编程通过`async/await`语法实现。

**优势**：
- **高并发**：在I/O密集场景下，异步编程可以处理更多并发请求
- **响应快**：避免阻塞，提高应用响应速度
- **资源高效**：减少线程开销，提高资源利用率

### async/await 语法

**基本语法**：
```python
async def get_data():
    # 异步操作
    await asyncio.sleep(1)
    return {"data": "Hello"}

@app.get("/data")
async def read_data():
    result = await get_data()
    return result
```

**注意事项**：
- 只有`async`函数可以使用`await`
- 调用`async`函数时需要使用`await`
- 同步函数不能在`async`函数中直接调用，需要使用`asyncio.to_thread()`或其他方式

### 异步与同步的区别

| 特性 | 同步 | 异步 |
|------|------|------|
| 执行方式 | 顺序执行，阻塞等待 | 非阻塞，并发执行 |
| 线程模型 | 多线程或单线程 | 单线程事件循环 |
| I/O操作 | 阻塞线程 | 让出控制权，继续执行其他任务 |
| 适用场景 | CPU密集型任务 | I/O密集型任务 |

```python
    item_id: int,
    item: ItemUpdate,
    db: AsyncSession = Depends(get_db)

    db_item = result.scalar_one_or_none()

    for key, value in item.dict(exclude_unset=True).items():
        setattr(db_item, key, value)
    
    await db.commit()
    await db.refresh(db_item)
    return db_item
```

## 1️⃣ 认证基础

### 什么是认证与授权

**认证（Authentication）**：验证用户身份，确认用户是谁。
**授权（Authorization）**：确定用户可以访问哪些资源，执行哪些操作。

**核心区别**：
- 认证：验证身份（"你是谁？"）
- 授权：验证权限（"你能做什么？"）

### 常见认证方式

| 认证方式 | 描述 | 适用场景 |
|----------|------|----------|
| **Session认证** | 服务器存储会话状态，客户端通过Cookie识别 | 传统Web应用 |

## 2️⃣ JWT 认证

### JWT 基本概念

**JWT（JSON Web Token）**：一种基于JSON的开放标准，用于在各方之间安全地传输信息。

**结构**：
- **Header**：包含令牌类型和签名算法
- **Payload**：包含声明（如用户ID、过期时间）
- **Signature**：使用密钥签名，确保令牌完整性

**优势**：
- 无状态：服务器不需要存储会话状态
- 可扩展：可以在payload中添加自定义声明
- 跨域：适合分布式系统

### JWT 实现

**创建令牌**：
```python
from datetime import datetime, timedelta
from jose import JWTError, jwt
from passlib.context import CryptContext

SECRET_KEY = "your-secret-key"
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def verify_password(plain_password, hashed_password):
    return pwd_context.verify(plain_password, hashed_password)

def get_password_hash(password):
    return pwd_context.hash(password)

def create_access_token(data: dict, expires_delta: timedelta = None):
    to_encode = data.copy()
    if expires_delta:
        expire = datetime.utcnow() + expires_delta
    else:
        expire = datetime.utcnow() + timedelta(minutes=15)
    to_encode.update({"exp": expire})
    encoded_jwt = jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)
    return encoded_jwt

def verify_token(token: str):
    credentials_exception = HTTPException(
        status_code=status.HTTP_401_UNAUTHORIZED,
        detail="Could not validate credentials",
        headers={"WWW-Authenticate": "Bearer"},
    )
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        username: str = payload.get("sub")
        if username is None:
            raise credentials_exception
        return username
    except JWTError:
        raise credentials_exception
```

## 3️⃣ OAuth2 认证

## 6️⃣ 多因素认证

### 基本概念

## ⚠️ 常见坑与注意点

1. **现象**：密码明文存储。**原因**：未使用密码哈希。**正确做法**：使用 passlib 等库对密码进行哈希处理。

2. **现象**：JWT 密钥泄露。**原因**：密钥硬编码在代码中或权限设置不当。**正确做法**：使用环境变量存储密钥，限制密钥访问权限。

3. **现象**：令牌无过期时间。**原因**：未设置 exp 声明。**正确做法**：为令牌设置合理的过期时间。

4. **现象**：权限检查不严格。**原因**：未在所有需要权限的端点进行检查。**正确做法**：使用依赖注入统一管理权限检查。

5. **现象**：CORS 配置过于宽松。**原因**：允许所有源访问。**正确做法**：明确指定允许的源、方法和头部。

6. **现象**：未实现令牌刷新机制。**原因**：用户需要频繁重新登录。**正确做法**：实现刷新令牌机制，提高用户体验。

## ⚠️ 常见坑与注意点

1. **现象**：测试环境与生产环境不一致。**原因**：配置不同或依赖版本不同。**正确做法**：使用环境变量和容器化确保环境一致性。

2. **现象**：部署后应用无法启动。**原因**：端口被占用、依赖缺失、配置错误。**正确做法**：检查日志，确保依赖安装完整，配置正确。

3. **现象**：生产环境性能差。**原因**：未优化服务器配置、数据库连接池配置不当。**正确做法**：调整 Uvicorn/Gunicorn 配置，优化数据库连接池。

4. **现象**：CI/CD 流水线失败。**原因**：测试失败、依赖安装失败、部署配置错误。**正确做法**：检查流水线日志，修复测试和配置问题。

5. **现象**：应用安全性问题。**原因**：未配置 HTTPS、CORS 配置过于宽松、安全头部缺失。**正确做法**：配置 HTTPS，设置合理的 CORS 策略，添加安全头部。

6. **现象**：监控缺失。**原因**：未配置日志和监控工具。**正确做法**：集成日志系统和监控工具，设置告警机制。

## 1️⃣ 性能优化基础

### 性能指标

**响应时间**：从请求发送到收到响应的时间。

**错误率**：请求失败的比例。

### 性能优化原则

- **可测量**：优化前后进行性能测试，验证效果
- **持续监控**：在生产环境持续监控性能

## 2️⃣ 代码优化

### 异步编程优化

**正确使用 async/await**：
- 只在 I/O 操作上使用 `await`
- 避免在 `async` 函数中执行 CPU 密集型任务
- 使用 `asyncio.gather()` 并发执行多个异步操作

**示例**：
```python
# 优化前
async def process_items(items):
    results = []
    for item in items:
        result = await process_item(item)
        results.append(result)
    return results

# 优化后
async def process_items(items):
    tasks = [process_item(item) for item in items]
    results = await asyncio.gather(*tasks)
    return results
```

### 数据结构优化

**选择合适的数据结构**：
- 使用 `dict` 或 `set` 进行快速查找（O(1)）
- 使用 `deque` 进行高效的队列操作
- 使用 `defaultdict` 或 `Counter` 简化代码

**示例**：
```python
# 优化前
def count_words(text):
    words = text.split()
    counts = {}
    for word in words:
        if word in counts:
            counts[word] += 1
        else:
            counts[word] = 1
    return counts

# 优化后
from collections import Counter

def count_words(text):
    words = text.split()
    return Counter(words)
```

### 代码结构优化

**减少函数调用开销**：
- 避免不必要的函数调用
- 使用局部变量减少属性查找
- 避免在循环中进行复杂计算

**示例**：
```python
# 优化前
def process_data(data):
    result = 0
    for item in data:
        result += calculate_value(item)
    return result

def calculate_value(item):
    return item * 2 + 1

# 优化后
def process_data(data):
    result = 0
    for item in data:
        result += item * 2 + 1
    return result
```

## 3️⃣ 数据库优化

### 查询优化

**索引优化**：
- 为频繁查询的字段添加索引
- 避免过度索引，影响写入性能
- 使用复合索引优化多字段查询

**查询语句优化**：
- 只查询需要的字段
- 使用 `LIMIT` 限制返回数据量
- 避免使用 `SELECT *`
- 使用 `JOIN` 优化关联查询

**示例**：
```python
# 优化前
@app.get("/items/")
async def read_items(db: Session = Depends(get_db)):
    items = db.query(Item).all()
    return items

# 优化后
@app.get("/items/")
async def read_items(
    skip: int = 0,
    limit: int = 100,
    db: Session = Depends(get_db)
):
    items = db.query(Item.id, Item.name, Item.price).offset(skip).limit(limit).all()
    return items
```

### 连接池优化

**配置连接池**：
- 根据应用规模调整连接池大小
- 设置合理的连接超时和回收时间
- 监控连接池状态，避免连接泄漏

**示例**：
```python
from sqlalchemy import create_engine

engine = create_engine(
    SQLALCHEMY_DATABASE_URL,
    pool_size=10,           # 连接池大小
    max_overflow=20,         # 最大溢出连接数
    pool_timeout=30,         # 连接超时时间
    pool_recycle=1800,       # 连接回收时间
    echo=False               # 是否打印SQL语句
)
```

### 批量操作

**批量插入**：
- 使用 `add_all()` 批量添加数据
- 减少数据库交互次数

**示例**：
```python
# 优化前
async def create_items(items, db: Session):
    for item_data in items:
        item = Item(**item_data)
        db.add(item)
    await db.commit()

# 优化后
async def create_items(items, db: Session):
    db_items = [Item(**item_data) for item_data in items]
    db.add_all(db_items)
    await db.commit()
```

## 4️⃣ 缓存策略

### 缓存类型

**内存缓存**：
- **Redis**：高性能内存数据库，支持多种数据结构
- **Memcached**：简单的键值存储
- **内存字典**：适合简单场景，进程内缓存

**浏览器缓存**：
- 设置 `Cache-Control` 头部
- 使用 ETag 进行缓存验证
- 使用 CDN 缓存静态资源

### Redis 缓存

**安装依赖**：
```bash
pip install redis
```

**基本使用**：
```python
import redis

redis_client = redis.Redis(host="localhost", port=6379, db=0)

# 设置缓存
redis_client.set("key", "value", ex=3600)  # 过期时间 3600 秒

# 获取缓存
value = redis_client.get("key")

# 删除缓存
redis_client.delete("key")
```

**缓存装饰器**：
```python
import functools
import redis

redis_client = redis.Redis(host="localhost", port=6379, db=0)

def cache(expire=3600):
    def decorator(func):
        @functools.wraps(func)
        async def wrapper(*args, **kwargs):
            # 生成缓存键
            cache_key = f"{func.__name__}:{args}:{kwargs}"
            
            # 尝试从缓存获取
            cached_value = redis_client.get(cache_key)
            if cached_value:
                return eval(cached_value)
            
            # 执行函数
            result = await func(*args, **kwargs)
            
            # 缓存结果
            redis_client.set(cache_key, str(result), ex=expire)
            
            return result
        return wrapper
    return decorator

@cache(expire=3600)
async def get_item(item_id: int, db: Session):
    return db.query(Item).filter(Item.id == item_id).first()
```

### 缓存策略

**缓存失效策略**：
- **时间过期**：设置缓存过期时间
- **主动更新**：数据变更时主动更新缓存
- **缓存穿透**：使用布隆过滤器防止缓存穿透
- **缓存雪崩**：使用随机过期时间避免缓存雪崩

**示例**：
```python
# 缓存穿透防护
from bloom_filter import BloomFilter

# 创建布隆过滤器
bf = BloomFilter(max_elements=10000, error_rate=0.01)

@app.get("/items/{item_id}")
async def read_item(item_id: int, db: Session = Depends(get_db)):
    # 检查布隆过滤器
    if item_id not in bf:
        raise HTTPException(status_code=404, detail="Item not found")
    
    # 尝试从缓存获取
    cache_key = f"item:{item_id}"
    cached_item = redis_client.get(cache_key)
    if cached_item:
        return eval(cached_item)
    
    # 从数据库获取
    item = db.query(Item).filter(Item.id == item_id).first()
    if not item:
        raise HTTPException(status_code=404, detail="Item not found")
    
    # 缓存结果
    redis_client.set(cache_key, str(item.__dict__), ex=3600)
    
    return item
```

## 5️⃣ 并发处理

### 并发模型

**异步 I/O**：
- 基于事件循环的非阻塞 I/O
- 适合 I/O 密集型任务
- 使用 `async/await` 语法

**多线程**：
- 适合 CPU 密集型任务
- 使用 `concurrent.futures.ThreadPoolExecutor`

**多进程**：
- 适合 CPU 密集型任务
- 使用 `concurrent.futures.ProcessPoolExecutor`

### 异步并发

**并发执行多个任务**：
```python
import asyncio

async def fetch_data(url):
    # 模拟网络请求
    await asyncio.sleep(1)
    return f"Data from {url}"

async def main():
    urls = ["http://example.com/1", "http://example.com/2", "http://example.com/3"]
    tasks = [fetch_data(url) for url in urls]
    results = await asyncio.gather(*tasks)
    return results

# 执行
results = asyncio.run(main())
print(results)
```

**限制并发数**：
```python
import asyncio
from asyncio import Semaphore

async def fetch_data(url, semaphore):
    async with semaphore:
        # 模拟网络请求
        await asyncio.sleep(1)
        return f"Data from {url}"

async def main():
    urls = ["http://example.com/1", "http://example.com/2", "http://example.com/3", "http://example.com/4", "http://example.com/5"]
    semaphore = Semaphore(2)  # 限制并发数为 2
    tasks = [fetch_data(url, semaphore) for url in urls]
    results = await asyncio.gather(*tasks)
    return results

# 执行
results = asyncio.run(main())
print(results)
```

### 线程池

**使用线程池处理 CPU 密集型任务**：
```python
import asyncio
from concurrent.futures import ThreadPoolExecutor

def cpu_intensive_task(n):
    # 模拟 CPU 密集型任务
    result = 0
    for i in range(n):
        result += i
    return result

async def main():
    loop = asyncio.get_event_loop()
    with ThreadPoolExecutor() as executor:
        # 在线程池中执行 CPU 密集型任务
        result = await loop.run_in_executor(executor, cpu_intensive_task, 10**8)
        print(f"Result: {result}")

# 执行
asyncio.run(main())
```

## ⚠️ 常见坑与注意点

1. **现象**：过度优化。**原因**：在没有数据支持的情况下进行优化。**正确做法**：使用性能分析工具找出瓶颈，基于数据进行优化。

2. **现象**：缓存不一致。**原因**：数据更新时未同步更新缓存。**正确做法**：实现缓存失效策略，确保数据一致性。

3. **现象**：并发安全问题。**原因**：多线程或多进程操作共享资源。**正确做法**：使用锁或其他同步机制确保并发安全。

4. **现象**：内存泄漏。**原因**：未释放资源或循环引用。**正确做法**：使用内存分析工具检测内存泄漏，及时释放资源。

5. **现象**：数据库连接池耗尽。**原因**：连接池配置不当或连接未及时释放。**正确做法**：调整连接池大小，确保连接正确释放。

6. **现象**：部署环境性能差。**原因**：服务器配置不当或资源不足。**正确做法**：根据应用需求配置合适的服务器资源，优化服务器参数。
