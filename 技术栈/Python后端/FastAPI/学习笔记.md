# 🧱 第1章：FastAPI 概述与核心思想

------

## **1.1 FastAPI 的起源与特点**

### 🧩 背景

- **FastAPI** 由 Sebastián Ramírez 创建，2018 年发布，基于 **Starlette**（ASGI 框架）与 **Pydantic**（数据校验）。
- 定位：**现代、高性能、带自动文档** 的 Python Web API 框架，原生支持 **async/await**。

------

### 🔹 核心特点

| 特性 | 描述 |
|------|------|
| **高性能** | 基于 ASGI，异步 I/O，性能接近 Node.js/Go 同类框架 |
| **自动文档** | 集成 OpenAPI（Swagger）与 ReDoc，零配置生成交互文档 |
| **类型提示驱动** | 基于 Pydantic 与 type hints，自动校验请求体、查询参数与响应 |
| **依赖注入** | 内置 DI 系统，便于复用鉴权、数据库会话等逻辑 |
| **标准开放** | 基于 OpenAPI 3.0、JSON Schema，便于与前端/网关对接 |

------

## **1.2 FastAPI 与 Flask、Django 对比**

| 对比项 | FastAPI | Flask | Django |
|--------|---------|--------|--------|
| **协议** | ASGI（异步） | WSGI（同步为主） | ASGI/WSGI |
| **类型与校验** | 原生 Pydantic | 需扩展 | 需扩展 |
| **API 文档** | 内置 Swagger/ReDoc | 需扩展 | 需扩展 |
| **适用场景** | 高性能 API、微服务、前后端分离 | 轻量 Web、传统 MVC | 全栈、Admin、内容站 |

------

### 🧠 关键词总结

> FastAPI 是 **“类型安全 + 自动文档 + 异步”** 的 ASGI API 框架，适合校招与生产中的 REST/API 开发，面试常考其与 Flask、异步编程的结合。

------

### 📘 面试延展题

**问：FastAPI 为什么性能好？**

**答：** 基于 ASGI 异步 I/O；路由与校验在 C 扩展（如 Pydantic、ujson）上优化；无同步阻塞时能高并发处理请求。与 Flask 同步模型相比，在 I/O 密集场景下优势明显。
