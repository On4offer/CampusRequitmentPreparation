# 📌 Gunicorn 常用配置与使用场景速查

> 配合《学习笔记》使用，侧重**启动命令、常用参数、配置文件**，便于生产部署速查。

---

## 一、常用命令速查

### 1.1 基本启动

| 命令 | 说明 | 使用场景 |
|------|------|----------|
| `gunicorn run:app` | 运行 `run` 模块中的 `app` 对象 | Flask 默认 |
| `gunicorn myproject.wsgi:application` | Django 项目 | Django 部署 |
| `gunicorn run:app -b 0.0.0.0:8000` | 绑定地址与端口 | 指定监听 |
| `gunicorn run:app -w 4` | 4 个 Worker 进程 | 生产常用 |
| `gunicorn run:app -w 4 -k gthread` | 4 进程 + 多线程 Worker | 多线程模型 |
| `gunicorn run:app -c gunicorn.conf.py` | 使用配置文件 | 生产推荐 |

### 1.2 常用参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `-b` / `--bind` | 绑定地址:端口 | `-b 0.0.0.0:8000` |
| `-w` / `--workers` | Worker 进程数 | `-w 4` |
| `-k` / `--worker-class` | Worker 类型 | `sync`、`gthread`、`gevent`、`uvicorn.workers.UvicornWorker`（ASGI） |
| `--threads` | 每 Worker 线程数（gthread 时） | `--threads 2` |
| `--timeout` | 请求超时（秒） | `--timeout 30` |
| `--daemon` | 后台运行 | 不用 systemd 时 |
| `-c` | 配置文件 | `-c gunicorn.conf.py` |
| `--access-logfile` / `--error-logfile` | 访问/错误日志路径 | 生产写文件 |

---

## 二、配置文件示例（gunicorn.conf.py）

```python
bind = "0.0.0.0:8000"
workers = 4
worker_class = "sync"  # 或 "gthread", "gevent"
threads = 2
timeout = 30
accesslog = "/var/log/gunicorn/access.log"
errorlog = "/var/log/gunicorn/error.log"
loglevel = "info"
```

## 三、与框架结合

### 3.1 Flask

| 场景 | 命令 / 说明 |
|------|-------------|
| 应用对象在 `run.py` | `gunicorn run:app` |
| 应用对象在 `app/__init__.py` 的 `create_app()` | `gunicorn "app:create_app()"` 或 在代码里返回 app |

### 3.2 Django

| 场景 | 命令 |
|------|------|
| 标准项目 | `gunicorn myproject.wsgi:application -b 0.0.0.0:8000` |

### 3.3 FastAPI（ASGI）

| 说明 | 命令 |
|------|------|
| 需安装 `uvicorn[standard]`，用 uvicorn worker | `gunicorn run:app -k uvicorn.workers.UvicornWorker` |

## 四、常用场景速查

| 场景 | 推荐命令 / 配置 |
|------|-----------------|
| 本地试跑 | `gunicorn run:app -b 127.0.0.1:8000 -w 2` |
| 生产 4 进程 | `gunicorn run:app -w 4 -b 0.0.0.0:8000 --timeout 30` |
| 使用配置文件 | `gunicorn run:app -c gunicorn.conf.py` |
| 配合 Nginx | Gunicorn 只监听 127.0.0.1:8000，Nginx 反向代理到该端口 |
