# 🧱 第1章：Gunicorn 概述与核心思想

------

## **1.1 Gunicorn 简介**

### 🧩 背景

- **Gunicorn**（Green Unicorn）是 Python 生态中广泛使用的 **WSGI HTTP 服务器**，用于在生产环境中运行 Flask、Django 等 WSGI 应用。
- 开发时常用 `flask run` 或 `python manage.py runserver`，这些是单进程调试服务器；**生产环境应使用 Gunicorn、uWSGI 等** 多进程/多线程服务器。

------

### 🔹 核心特点

| 特性 | 描述 |
|------|------|
| **WSGI 标准** | 实现 PEP 3333，与任何 WSGI 应用兼容 |
| **多 Worker** | 预 fork 多进程（或协程 Worker），提高并发 |
| **部署简单** | 无依赖系统库，`pip install gunicorn` 即可 |
| **与 Nginx 搭配** | 常作为上游应用服务器，由 Nginx 做反向代理与静态资源 |

------

## **1.2 Gunicorn 与 uWSGI、uvicorn 对比**

| 对比项 | Gunicorn | uWSGI | uvicorn |
|--------|----------|-------|---------|
| **协议** | WSGI | WSGI/自有协议 | ASGI |
| **适用框架** | Flask、Django | Flask、Django | FastAPI、Starlette |
| **配置方式** | 命令行 + 配置文件 | 配置丰富、略复杂 | 命令行为主 |
| **使用广泛度** | 非常常见 | 常见 | 异步框架标配 |

------

### 🧠 关键词总结

> Gunicorn 是 **“WSGI 多进程”** 应用服务器，用于在生产环境运行 Flask/Django，通常与 Nginx 反向代理配合使用。

------

### 📘 面试延展题

**问：为什么生产环境不用 Flask 自带的 runserver？**

**答：** runserver 是单线程、未做安全加固，仅适合开发；生产需要多 Worker 并发、进程管理、与 Nginx 配合做静态与负载均衡，因此用 Gunicorn/uWSGI 等 WSGI 服务器。
