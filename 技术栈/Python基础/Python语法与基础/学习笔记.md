## 1️⃣ Python简介

### （1）Python的特点

- **解释型语言**：Python代码不需要编译，由解释器逐行执行，便于快速开发和调试。
  - **为什么需要**：编译型语言需要先编译再运行，开发周期长；解释型语言可以边写边运行，提高开发效率。
  - **面试可答**：Python是解释型语言，代码由解释器逐行执行，便于快速迭代和调试。

- **动态类型**：变量无需声明类型，类型在运行时自动确定。
  - **为什么需要**：静态类型语言需要预先声明类型，代码冗长；动态类型语言更灵活，开发效率高。
  - **面试可答**：Python是动态类型语言，变量类型在运行时确定，使用更灵活但需要类型检查。

- **简洁优雅**：Python强调代码可读性，语法简洁明了。
  - **为什么需要**：代码维护成本远高于开发成本，可读性强的代码更易于维护和协作。
  - **面试可答**：Python的设计哲学是"优雅胜于丑陋"，强调代码的可读性和简洁性。

### （2）Python的应用领域

| 应用领域 | 说明 | 常用库/框架 |
|---------|------|-------------|
| Web开发 | 快速构建Web应用和API | Django、Flask、FastAPI |
| 数据分析 | 数据处理、可视化和统计分析 | Pandas、NumPy、Matplotlib |
| 人工智能/机器学习 | 深度学习、自然语言处理、计算机视觉 | TensorFlow、PyTorch、Scikit-learn |
| 自动化脚本 | 自动化运维、测试、数据处理 | Selenium、Requests、BeautifulSoup |
| 科学计算 | 数值计算、科学模拟 | SciPy、SymPy |

### （3）Python版本

- **Python 2 vs Python 3**：
  - Python 2已于2020年停止维护，不再推荐使用
  - Python 3是当前主流版本，持续更新和优化
  - **面试可答**：Python 2已停止维护，目前主要使用Python 3，两者在语法上有一些差异，如print函数、整数除法等。

- **版本选择建议**：
  - 生产环境：使用Python 3.8+（推荐3.10+）
  - 新项目：使用Python 3.11+（性能优化和语法改进）
  - 兼容性考虑：根据项目依赖选择合适的版本

### （4）Python解释器

- **CPython**：官方解释器，使用C语言实现，是最常用的Python解释器。
- **其他解释器**：
  - **PyPy**：使用JIT技术，执行速度快，适合CPU密集型任务
  - **Jython**：运行在Java平台上，与Java代码互操作
  - **IronPython**：运行在.NET平台上
  - **面试可答**：CPython是官方解释器，使用C语言实现；PyPy使用JIT技术，执行速度更快但兼容性略差。

## 2️⃣ 运行与开发环境

### （1）Python安装

- **Windows系统**：
  - 访问Python官网（https://www.python.org/downloads/）下载安装包
  - 安装时勾选"Add Python to PATH"
  - 验证安装：打开命令行输入`python --version`

- **macOS系统**：
  - 使用Homebrew安装：`brew install python`
  - 或从官网下载安装包安装
  - 验证安装：打开终端输入`python3 --version`

- **Linux系统**：
  - 大多数Linux发行版预装Python
  - 使用包管理器安装：`sudo apt-get install python3`（Ubuntu/Debian）
  - 验证安装：`python3 --version`

### （2）包管理工具pip

- **pip简介**：Python的包管理工具，用于安装、升级和卸载第三方库。
  - **为什么需要**：手动管理依赖复杂且容易出错，pip提供统一的依赖管理方式。
  - **面试可答**：pip是Python的包管理工具，类似于npm（Node.js）或maven（Java），用于管理第三方库。

- **常用pip命令**：
  ```bash
  # 安装包
  pip install package_name
  
  # 安装指定版本
  pip install package_name==1.0.0
  
  # 升级包
  pip install --upgrade package_name
  
  # 卸载包
  pip uninstall package_name
  
  # 查看已安装的包
  pip list
  
  # 查看包信息
  pip show package_name
  
  # 导出依赖列表
  pip freeze > requirements.txt
  
  # 从依赖列表安装
  pip install -r requirements.txt
  ```

- **国内镜像源**：
  ```bash
  # 使用清华镜像源
  pip install -i https://pypi.tuna.tsinghua.edu.cn/simple package_name
  
  # 永久配置镜像源
  pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
  ```

### （3）虚拟环境

- **虚拟环境简介**：为每个项目创建独立的Python环境，隔离项目依赖。
  - **为什么需要**：不同项目可能需要不同版本的库，全局安装会导致依赖冲突。
  - **面试可答**：虚拟环境可以隔离项目依赖，避免不同项目之间的依赖冲突，类似于Java的Maven多模块管理。

- **venv虚拟环境**：
  ```bash
  # 创建虚拟环境
  python -m venv myenv
  
  # 激活虚拟环境（Windows）
  myenv\Scripts\activate
  
  # 激活虚拟环境（macOS/Linux）
  source myenv/bin/activate
  
  # 退出虚拟环境
  deactivate
  ```

- **conda虚拟环境**：
  ```bash
  # 创建虚拟环境
  conda create -n myenv python=3.10
  
  # 激活虚拟环境
  conda activate myenv
  
  # 退出虚拟环境
  conda deactivate
  
  # 删除虚拟环境
  conda remove -n myenv --all
  ```

### （4）开发工具

- **VS Code**：
  - 轻量级、插件丰富、免费开源
  - 推荐插件：Python、Pylance、Jupyter
  - **配置示例**：
    ```json
    {
      "python.defaultInterpreterPath": "./venv/Scripts/python.exe",
      "python.linting.enabled": true,
      "python.linting.pylintEnabled": true
    }
    ```

- **PyCharm**：
  - 功能强大的IDE，适合大型项目
  - 社区版免费，专业版收费
  - **配置步骤**：
    1. 打开PyCharm，创建新项目
    2. 选择Python解释器（虚拟环境）
    3. 安装项目依赖
    4. 配置代码风格和检查工具

- **Jupyter Notebook**：
  - 交互式开发环境，适合数据分析和机器学习
  - 支持代码、文本、图表混合展示
  - **安装**：`pip install jupyter`
  - **启动**：`jupyter notebook`

### （5）运行Python代码

- **交互式REPL**：
  ```bash
  # 启动Python交互式环境
  python
  
  # 或使用IPython（增强版）
  pip install ipython
  ipython
  
  # 退出
  exit()
  ```

- **脚本执行**：
  ```bash
  # 执行Python脚本
  python script.py
  
  # 或使用python3
  python3 script.py
  ```

- **模块导入**：
  ```python
  # script.py
  def hello():
      print("Hello, World!")
  
  if __name__ == "__main__":
      hello()
  
  # 其他文件导入
  import script
  script.hello()
  ```

- **__name__ == "__main__"**：
  - **作用**：区分脚本是被直接运行还是被导入
  - **为什么需要**：避免导入模块时执行测试代码或初始化代码。
  - **面试可答**：`if __name__ == "__main__"`用于判断脚本是否被直接运行，避免导入时执行不必要的代码。

## ⚠️ 常见坑与注意点

1. **现象**：Python命令找不到。**原因**：安装时未勾选"Add Python to PATH"。**正确做法**：重新安装Python并勾选该选项，或手动添加Python路径到系统环境变量。

2. **现象**：pip安装速度慢或失败。**原因**：默认使用国外源，网络不稳定。**正确做法**：使用国内镜像源（如清华、阿里云）加速下载。

3. **现象**：不同项目依赖冲突。**原因**：全局安装依赖，版本不兼容。**正确做法**：为每个项目创建独立的虚拟环境，隔离项目依赖。

4. **现象**：导入模块时执行测试代码。**原因**：未使用`if __name__ == "__main__"`判断。**正确做法**：将测试代码放在该判断下，避免导入时执行。

5. **现象**：代码缩进错误。**原因**：混用Tab和空格，或缩进不一致。**正确做法**：统一使用4个空格缩进，配置编辑器自动转换Tab为空格。

## 1️⃣ 变量与命名

### （1）变量定义

- **变量简介**：变量是存储数据的容器，通过变量名访问数据。
  - **为什么需要**：数据需要存储和传递，变量提供了方便的数据访问方式。
  - **面试可答**：Python是动态类型语言，变量无需声明类型，赋值即定义。

- **变量赋值**：
  ```python
  # 基本赋值
  x = 10
  name = "Alice"
  is_valid = True
  
  # 多重赋值
  a = b = c = 10
  
  # 元组解包赋值
  x, y, z = 1, 2, 3
  
  # 交换变量
  x, y = y, x
  ```

### （2）命名规范

- **命名规则**：
  - 变量名只能包含字母、数字和下划线
  - 不能以数字开头
  - 不能使用Python关键字
  - 区分大小写

- **命名风格**：
  ```python
  # 变量名：蛇形命名法（小写字母+下划线）
  user_name = "Alice"
  max_count = 100
  
  # 常量：全大写字母+下划线
  MAX_SIZE = 1000
  DEFAULT_TIMEOUT = 30
  
  # 类名：驼峰命名法（首字母大写）
  class UserProfile:
      pass
  
  # 函数名：蛇形命名法
  def get_user_name():
      pass
  ```

- **面试可答**：Python推荐使用蛇形命名法（snake_case）命名变量和函数，类名使用驼峰命名法（CamelCase），常量使用全大写字母。

### （3）变量作用域

- **局部变量**：在函数内部定义的变量，只能在函数内部访问。
- **全局变量**：在函数外部定义的变量，可以在整个模块中访问。
- **global关键字**：在函数内部修改全局变量需要使用global关键字。

```python
# 全局变量
count = 0

def increment():
    global count  # 声明使用全局变量
    count += 1
    print(f"局部count: {count}")

increment()
print(f"全局count: {count}")
```

## 2️⃣ 数据类型

### （1）数值类型

- **整数（int）**：表示整数，如10、-5、0。
- **浮点数（float）**：表示小数，如3.14、-2.5。
- **复数（complex）**：表示复数，如3+4j。

```python
# 数值类型
x = 10  # int
y = 3.14  # float
z = 3 + 4j  # complex

# 类型转换
x = int(3.14)  # 3
y = float(10)  # 10.0
z = complex(3, 4)  # (3+4j)

# 数值运算
a = 10 + 5  # 加法
b = 10 - 5  # 减法
c = 10 * 5  # 乘法
d = 10 / 5  # 除法（结果为float）
e = 10 // 5  # 整除（结果为int）
f = 10 % 5  # 取余
g = 10 ** 5  # 幂运算
```

### （2）字符串（str）

- **字符串定义**：使用单引号、双引号或三引号定义字符串。
  - **为什么需要**：字符串是文本数据的基本表示，需要灵活的定义方式。
  - **面试可答**：Python支持单引号、双引号和三引号定义字符串，三引号可以定义多行字符串。

```python
# 字符串定义
s1 = 'Hello'
s2 = "World"
s3 = '''Multi
line
string'''

# 字符串拼接
s = "Hello" + " " + "World"  # "Hello World"
s = " ".join(["Hello", "World"])  # "Hello World"

# 字符串重复
s = "Hello" * 3  # "HelloHelloHello"

# 字符串切片
s = "Hello World"
print(s[0:5])  # "Hello"
print(s[6:])  # "World"
print(s[-1])  # "d"

# 字符串方法
s = "  Hello World  "
print(s.strip())  # "Hello World"（去除首尾空格）
print(s.split())  # ["Hello", "World"]（分割字符串）
print(s.replace("World", "Python"))  # "  Hello Python  "（替换）

# f-string格式化
name = "Alice"
age = 25
print(f"My name is {name} and I'm {age} years old.")
```

### （3）布尔类型（bool）

- **布尔值**：True和False，用于逻辑判断。
- **真值测试**：非零、非空容器为True，零、空容器为False。

```python
# 布尔值
is_valid = True
is_empty = False

# 真值测试
print(bool(0))  # False
print(bool(1))  # True
print(bool(""))  # False
print(bool("Hello"))  # True
print(bool([]))  # False
print(bool([1, 2, 3]))  # True
```

### （4）None类型

- **None简介**：表示空值或无值。
  - **为什么需要**：需要表示变量没有值或函数没有返回值。
  - **面试可答**：None是Python的空值，类似于Java的null或JavaScript的undefined。

```python
# None类型
x = None

# 判断是否为None
if x is None:
    print("x is None")

# 函数默认返回None
def hello():
    print("Hello")

result = hello()
print(result)  # None
```

### （5）类型判断与转换

```python
# 类型判断
x = 10
print(type(x))  # <class 'int'>
print(isinstance(x, int))  # True
print(isinstance(x, (int, float)))  # True（判断多种类型）

# 类型转换
x = int("10")  # 10
y = float("3.14")  # 3.14
z = str(10)  # "10"
lst = list("Hello")  # ['H', 'e', 'l', 'l', 'o']
d = dict([("a", 1), ("b", 2)])  # {'a': 1, 'b': 2}
```

## 3️⃣ 运算符

### （1）算术运算符

```python
# 算术运算符
a = 10
b = 3

print(a + b)  # 13（加法）
print(a - b)  # 7（减法）
print(a * b)  # 30（乘法）
print(a / b)  # 3.333...（除法）
print(a // b)  # 3（整除）
print(a % b)  # 1（取余）
print(a ** b)  # 1000（幂运算）
```

### （2）比较运算符

```python
# 比较运算符
a = 10
b = 5

print(a == b)  # False（等于）
print(a != b)  # True（不等于）
print(a > b)  # True（大于）
print(a < b)  # False（小于）
print(a >= b)  # True（大于等于）
print(a <= b)  # False（小于等于）
```

### （3）逻辑运算符

```python
# 逻辑运算符
a = True
b = False

print(a and b)  # False（与）
print(a or b)  # True（或）
print(not a)  # False（非）

# 短路求值
print(True and False)  # False（False不执行）
print(False or True)  # True（True不执行）
```

### （4）身份运算符

```python
# 身份运算符
a = [1, 2, 3]
b = [1, 2, 3]
c = a

print(a is b)  # False（不同对象）
print(a is c)  # True（同一对象）
print(a is not b)  # True

# 面试可答：is比较对象身份，==比较对象值
print(a == b)  # True（值相同）
print(a is b)  # False（对象不同）
```

### （5）成员运算符

```python
# 成员运算符
lst = [1, 2, 3]
d = {"a": 1, "b": 2}
s = "Hello"

print(1 in lst)  # True
print(4 not in lst)  # True
print("a" in d)  # True（检查键）
print("H" in s)  # True
```

### （6）运算符优先级

| 优先级 | 运算符 | 说明 |
|--------|--------|------|
| 1 | ** | 幂运算 |
| 2 | +x, -x, ~x | 一元运算符 |
| 3 | *, /, //, % | 乘、除、整除、取余 |
| 4 | +, - | 加、减 |
| 5 | <<, >> | 位运算 |
| 6 | & | 位与 |
| 7 | ^ | 位异或 |
| 8 | \| | 位或 |
| 9 | <, <=, >, >=, <>, !=, == | 比较 |
| 10 | is, is not, in, not in | 身份、成员 |
| 11 | not | 逻辑非 |
| 12 | and | 逻辑与 |
| 13 | or | 逻辑或 |

```python
# 运算符优先级示例
result = 2 + 3 * 4  # 14（先乘后加）
result = (2 + 3) * 4  # 20（括号优先）
```

## 4️⃣ 序列类型

### （1）列表（list）

- **列表简介**：可变、有序的序列类型。
  - **为什么需要**：需要存储多个数据并能够动态修改。
  - **面试可答**：列表是可变的有序序列，支持增删改查操作，时间复杂度为O(1)。

```python
# 列表定义
lst = [1, 2, 3, 4, 5]

squares = [x**2 for x in range(10)]
```

### （2）元组（tuple）

- **元组简介**：不可变、有序的序列类型。

## 5️⃣ 映射类型

### （1）字典（dict）

- **字典简介**：键值对的无序集合，键唯一且不可变。
  - **为什么需要**：需要通过键快速查找值。
  - **面试可答**：字典是键值对的集合，通过键快速查找值，时间复杂度为O(1)，底层使用哈希表实现。

## 6️⃣ 集合类型

### （1）集合（set）

- **集合简介**：无序、不重复的元素集合。

```python
lst = [1, 2, 2, 3, 3, 3]
```

# 🌱 第3章：控制流

本章介绍Python的控制流语句，包括条件判断、循环和推导式。通过本章学习，你将掌握Python的程序控制逻辑，能够编写复杂的程序流程。

## 1️⃣ 条件语句

### （1）if语句

- **if语句简介**：根据条件执行不同的代码块。
  - **为什么需要**：程序需要根据不同的条件执行不同的逻辑。
  - **面试可答**：if语句用于条件判断，根据条件执行不同的代码块。

```python
# if语句
x = 10

if x > 0:
    print("x is positive")
elif x < 0:
    print("x is negative")
else:
    print("x is zero")
```

### （2）条件表达式（三元运算符）

- **条件表达式简介**：简洁的条件判断语法。
  - **为什么需要**：简化简单的条件判断，提高代码可读性。
  - **面试可答**：条件表达式是三元运算符，语法为`value_if_true if condition else value_if_false`。

```python
# 条件表达式
x = 10
result = "positive" if x > 0 else "non-positive"
print(result)  # "positive"
```

### （3）真值测试

- **真值简介**：Python中非零、非空容器为True，零、空容器为False。
  - **为什么需要**：简化条件判断，不需要显式比较。
  - **面试可答**：Python中非零、非空容器为True，零、空容器为False。

```python
# 真值测试
print(bool(0))  # False
print(bool(1))  # True
print(bool(""))  # False
print(bool("Hello"))  # True
print(bool([]))  # False
print(bool([1, 2, 3]))  # True

# 真值在条件判断中的应用
x = 10
if x:  # 等价于 if x != 0
    print("x is non-zero")
```

## 2️⃣ 循环语句

### （1）for循环

- **for循环简介**：遍历序列中的元素。
  - **为什么需要**：需要重复执行相同的操作，处理序列中的每个元素。
  - **面试可答**：for循环用于遍历序列，如列表、元组、字符串、字典等。

```python
# for循环遍历列表
lst = [1, 2, 3, 4, 5]
for item in lst:
    print(item)

# for循环遍历字典
d = {"a": 1, "b": 2, "c": 3}
for key, value in d.items():
    print(f"{key}: {value}")

# for循环遍历字符串
s = "Hello"
for char in s:
    print(char)
```

### （2）while循环

- **while循环简介**：当条件为True时重复执行代码块。
  - **为什么需要**：需要根据条件重复执行操作，不确定循环次数。
  - **面试可答**：while循环用于条件循环，当条件为True时重复执行，适合不确定循环次数的场景。

```python
# while循环
count = 0
while count < 5:
    print(count)
    count += 1

# 无限循环（慎用）
while True:
    user_input = input("Enter 'quit' to exit: ")
    if user_input == "quit":
        break
```

### （3）break和continue

- **break简介**：跳出循环，终止循环执行。
- **continue简介**：跳过当前迭代，继续下一次迭代。
  - **面试可答**：break用于跳出循环，continue用于跳过当前迭代。

```python
# break示例
for i in range(10):
    if i == 5:
        break
    print(i)  # 输出0-4

# continue示例
for i in range(10):
    if i % 2 == 0:
        continue
    print(i)  # 输出奇数1,3,5,7,9
```

### （4）循环的else子句

- **else子句简介**：循环正常结束时执行的代码块。
  - **为什么需要**：区分循环正常结束和被break中断。
  - **面试可答**：循环的else子句在循环正常结束时执行，如果循环被break中断则不执行。

```python
# 循环else子句
for i in range(5):
    print(i)
else:
    print("Loop completed normally")  # 正常结束执行

# break中断循环
for i in range(5):
    if i == 3:
        break
    print(i)
else:
    print("This will not be printed")  # break中断不执行
```

```python
print(message1)  # "Hello, Alice!"

message2 = greet("Bob", "Hi")
print(message2)  # "Hi, Bob!"
```

```python
    print(f"c: {c}, d: {d}")
    print(f"kwargs: {kwargs}")
```

## 4️⃣ 函数作用域

### （1）LEGB规则

- **LEGB简介**：Python变量查找顺序为Local → Enclosing → Global → Built-in。
  - **为什么需要**：理解变量查找顺序，避免作用域错误。
  - **面试可答**：Python变量查找顺序为LEGB（Local、Enclosing、Global、Built-in），先查找局部变量，再查找外层函数变量，然后全局变量，最后内置变量。

```python
# LEGB规则示例
x = "global"

def outer():
    x = "enclosing"
    
    def inner():
        x = "local"
        print(f"Local: {x}")
    
    inner()
    print(f"Enclosing: {x}")

outer()
print(f"Global: {x}")
```

### （2）global关键字

```python
# global关键字
count = 0

def increment():
    global count
    count += 1
    print(f"Count: {count}")

increment()
print(f"Global count: {count}")
```

### （3）nonlocal关键字

```python
# nonlocal关键字
def outer():
    count = 0
    
    def inner():
        nonlocal count
        count += 1
        print(f"Inner count: {count}")
    
    inner()
    print(f"Outer count: {count}")

outer()
```

## ⚠️ 常见坑与注意点

1. **现象**：默认参数可变对象问题。**原因**：默认参数在函数定义时只计算一次，可变对象会被共享。**正确做法**：使用None作为默认值，在函数内部创建可变对象。

2. **现象**：修改全局变量失败。**原因**：未使用global关键字声明。**正确做法**：在函数内部使用global关键字声明要修改的全局变量。

3. **现象**：参数顺序错误。**原因**：违反参数顺序规则（位置 → *args → 默认 → **kwargs）。**正确做法**：按照正确的参数顺序定义函数参数。

4. **现象**：lambda表达式过于复杂。**原因**：lambda只能包含一个表达式，不适合复杂逻辑。**正确做法**：复杂逻辑使用普通函数定义。

5. **现象**：返回值类型错误。**原因**：期望返回多个值，实际返回单个值或元组。**正确做法**：明确返回值的类型，使用元组解包接收多值返回。

## 1️⃣ 列表进阶操作

### （1）列表切片

- **切片简介**：通过索引范围获取列表的子集。
  - **为什么需要**：快速获取列表的一部分数据，避免手动遍历。
  - **面试可答**：列表切片使用[start:end:step]语法，左闭右开，支持负索引和步长。

```python
# 列表切片

# 省略起始或结束
print(numbers[:5])  # [0, 1, 2, 3, 4]
print(numbers[5:])  # [5, 6, 7, 8, 9]

# 负索引
print(numbers[-3:])  # [7, 8, 9]
print(numbers[:-3])  # [0, 1, 2, 3, 4, 5, 6]

# 步长
print(numbers[::2])  # [0, 2, 4, 6, 8]
print(numbers[::-1])  # [9, 8, 7, 6, 5, 4, 3, 2, 1, 0]
```

## ⚠️ 常见坑与注意点

1. **现象**：列表切片超出范围。**原因**：切片索引超出列表长度，但切片不会报错。**正确做法**：切片会自动处理超出范围的索引，无需担心。

2. **现象**：字典键类型错误。**原因**：字典键必须是不可变类型（如数字、字符串、元组）。**正确做法**：使用不可变类型作为字典键。

3. **现象**：集合元素可变。**原因**：集合元素必须是不可变类型。**正确做法**：使用不可变类型作为集合元素，如元组。

4. **现象**：元组修改失败。**原因**：元组是不可变的，不能修改。**正确做法**：将元组转换为列表修改后再转回元组，或创建新元组。

5. **现象**：字符串修改失败。**原因**：字符串是不可变的，不能修改。**正确做法**：使用字符串方法创建新字符串，或使用列表存储可变字符。

6. **现象**：正则表达式匹配失败。**原因**：正则表达式模式不正确或文本格式不符合预期。**正确做法**：使用在线正则表达式测试工具验证模式，或使用re.DEBUG查看匹配过程。

------

**学习要点**：
- 掌握列表的切片、推导式和常用方法，理解列表的时间复杂度
- 熟悉字典的推导式、排序和常用方法，理解字典的哈希表实现
- 掌握集合的运算和推导式，理解集合的去重和快速查找特性
- 理解元组的不可变特性，掌握命名元组的使用
- 学会字符串的格式化和正则表达式，理解字符串的不可变特性
- 能够根据实际需求选择合适的数据结构，理解各数据结构的时间复杂度

### （3）日期时间库（datetime）


## ⚠️ 常见坑与注意点

1. **现象**：导入模块失败。**原因**：模块不在搜索路径中，或模块名拼写错误。**正确做法**：检查模块搜索路径，确认模块名正确，或添加自定义搜索路径。

2. **现象**：循环导入错误。**原因**：两个模块相互导入，导致导入循环。**正确做法**：延迟导入（在函数内部导入），或重构代码提取共享模块。

3. **现象**：包导入失败。**原因**：包目录中没有__init__.py文件。**正确做法**：在包目录中创建__init__.py文件，可以是空文件。

4. **现象**：虚拟环境未激活。**原因**：忘记激活虚拟环境，导致包安装到全局环境。**正确做法**：每次使用虚拟环境前先激活，或使用IDE自动激活。

5. **现象**：依赖版本冲突。**原因**：不同项目依赖不同版本的包。**正确做法**：为每个项目创建独立的虚拟环境，使用requirements.txt管理依赖。

6. **现象**：导入顺序混乱。**原因**：未按照PEP 8规范组织导入语句。**正确做法**：按照标准库、第三方库、本地库的顺序组织导入，每组之间空一行。
