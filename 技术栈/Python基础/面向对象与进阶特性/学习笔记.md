# 📘《Python 面向对象与进阶特性学习与实战手册》

> 系统学习见下文各章；日常常用语法与 API 速查见同目录《常用API与使用场景》。

------

## 第1章：面向对象基础

### 1.1 类与实例

- **类**用 class 定义；**实例**通过调用类创建（obj = MyClass()）；首参数 self 表示实例自身。
- 属性：实例属性（self.xxx）、类属性（在类体直接定义）；实例属性覆盖同名类属性；方法：实例方法、类方法（@classmethod，cls）、静态方法（@staticmethod，无 self/cls）。

### 1.2 构造与封装

- **__init__(self, ...)** 构造方法，创建实例时调用；**__new__** 在 __init__ 之前，控制实例创建，少用。
- 封装：约定单下划线 _ 表示“内部使用”、双下划线 __ 触发名称改写（name mangling），用于避免子类同名冲突。

## 第2章：继承与多态

1. **继承**：class Child(Parent):；子类可重写父类方法；**super()** 调用父类方法，常用于 __init__ 与重写方法中。
2. **多态**：同一接口不同实现；Python 鸭子类型（不强制类型，只要对象有相应方法即可）。
3. **多重继承**：class C(A, B):；**MRO**（方法解析顺序）按 C3 线性化，可用 cls.mro() 查看；注意菱形继承与 super 调用顺序。
4. **面试要点**：super() 作用、MRO 顺序、鸭子类型含义。

------

## 第3章：魔术方法与协议

1. **常用魔术方法**：**__str__** / **__repr__**（字符串表示）、**__len__**、**__getitem__** / **__setitem__**（下标访问）、**__iter__** / **__next__**（迭代器）、**__enter__** / **__exit__**（上下文管理器）、**__call__**（实例可调用）。
2. **比较与运算**：__eq__、__lt__、__add__ 等；实现协议即可被 len()、in、for、with 等使用。
3. **面试要点**：__init__ 与 __new__ 区别、上下文管理器与 with、可迭代与迭代器协议。

------

## 第4章：装饰器

1. **概念**：装饰器是接受函数（或类）并返回新函数（或类）的可调用对象；用于在不改原函数代码的前提下增强或包装行为。
2. **定义**：def deco(func): ... return wrapper；使用 @deco 语法糖；带参装饰器需多一层闭包。
3. **常见用途**：计时、日志、权限、重试、缓存（如 functools.lru_cache）；**functools.wraps** 保留原函数元信息。
4. **类装饰器**：装饰类，可用来注册、单例等。
5. **面试要点**：装饰器原理、带参装饰器、wraps 作用。

------

## 第5章：生成器与迭代器

1. **迭代器**：实现 __iter__ 与 __next__；迭代器也是可迭代对象；StopIteration 表示结束；iter()、next() 内置函数。
2. **生成器**：用 **yield** 的函数，调用时返回生成器对象（迭代器）；惰性求值，节省内存；yield 可带值（生成器.send() 与协程基础）。
3. **生成器表达式**：(x*2 for x in lst) 类似列表推导但惰性；适合大数据流式处理。
4. **面试要点**：迭代器与可迭代区别、生成器与列表推导区别、yield 与 return。

------

## 第6章：异常处理

1. **try / except / else / finally**：except 捕获指定异常；可多个 except、或 except Exception；else 在无异常时执行；finally 无论如何都执行（常用于清理）。
2. **异常类型**：BaseException ← Exception ← 具体异常（ValueError、TypeError、KeyError、IndexError 等）；自定义异常继承 Exception。
3. **raise**：主动抛出异常；**raise from** 保留异常链；**assert** 用于调试断言（可被 -O 关闭）。
4. **面试要点**：except 顺序（子类在前）、finally 与 return、异常最佳实践（具体异常、避免裸 except）。

------

## 第7章：上下文管理器与 with

1. **with** 语句：保证进入与退出时执行固定逻辑（如打开/关闭文件、加锁/解锁）；对象需实现 **__enter__** 与 **__exit__**。
2. **contextlib.contextmanager**：用生成器快速写上下文管理器；yield 前为 __enter__，后为 __exit__。
3. **应用**：文件 open、线程锁、数据库连接；保证资源释放与异常安全。
4. **面试要点**：with 原理、__exit__ 是否吞掉异常（返回 True 则抑制）。
