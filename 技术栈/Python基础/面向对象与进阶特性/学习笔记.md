## 1️⃣ 类与实例

### （1）类的定义与实例化

- **类的定义**：使用`class`关键字定义类，类名通常使用驼峰命名法。
  - **为什么需要**：类是面向对象编程的基本单位，用于封装数据和行为。
  - **面试可答**：类是对象的蓝图，定义了对象的属性和方法，通过实例化创建具体对象。

```python
# 类的定义
class Person:
    # 类属性
    species = "Homo sapiens"
    
    # 构造方法
    def __init__(self, name, age):
        # 实例属性
        self.name = name
        self.age = age
    
    # 实例方法
    def greet(self):
        return f"Hello, my name is {self.name} and I'm {self.age} years old."

# 实例化
person = Person("Alice", 25)
print(person.greet())  # Hello, my name is Alice and I'm 25 years old.
print(person.species)  # Homo sapiens
```

### （2）属性

- **实例属性**：通过`self`访问的属性，每个实例都有独立的实例属性。
- **类属性**：在类体中直接定义的属性，所有实例共享同一个类属性。
- **属性访问**：实例属性优先级高于类属性，同名时实例属性会覆盖类属性。

```python
class Person:
    # 类属性
    species = "Homo sapiens"
    count = 0
    
    def __init__(self, name, age):
        # 实例属性
        self.name = name
        self.age = age
        # 修改类属性
        Person.count += 1

# 创建实例
p1 = Person("Alice", 25)
p2 = Person("Bob", 30)

# 访问属性
print(p1.name)  # Alice（实例属性）
print(p1.species)  # Homo sapiens（类属性）
print(Person.count)  # 2（类属性，所有实例共享）

# 实例属性覆盖类属性
p1.species = "Human"
print(p1.species)  # Human（实例属性）
print(p2.species)  # Homo sapiens（类属性）
```

### （3）方法

- **实例方法**：第一个参数是`self`，表示实例本身。
- **类方法**：使用`@classmethod`装饰器，第一个参数是`cls`，表示类本身。
- **静态方法**：使用`@staticmethod`装饰器，没有特殊的第一个参数。

```python
class Person:
    species = "Homo sapiens"
    
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    # 实例方法
    def greet(self):
        return f"Hello, my name is {self.name}."
    
    # 类方法
    @classmethod
    def get_species(cls):
        return cls.species
    
    # 静态方法
    @staticmethod
    def is_adult(age):
        return age >= 18

# 使用实例方法
p = Person("Alice", 25)
print(p.greet())  # Hello, my name is Alice.

# 使用类方法
print(Person.get_species())  # Homo sapiens

# 使用静态方法
print(Person.is_adult(25))  # True
print(Person.is_adult(15))  # False
```

## 2️⃣ 构造与封装

### （1）构造方法

- **__init__方法**：初始化实例，在创建实例时自动调用。
- **__new__方法**：创建实例，在__init__之前调用，用于控制实例的创建过程。
  - **面试可答**：__new__是类方法，返回新创建的实例；__init__是实例方法，初始化实例属性，无返回值。

```python
class Singleton:
    _instance = None
    
    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance
    
    def __init__(self, value):
        self.value = value

# 单例模式测试
s1 = Singleton(10)
s2 = Singleton(20)
print(s1 is s2)  # True
print(s1.value)  # 20
print(s2.value)  # 20
```

### （2）封装

- **封装**：隐藏对象的内部状态，只暴露必要的接口。
- **访问控制**：
  - 公开属性：直接访问
  - 保护属性：以单下划线`_`开头，表示内部使用
  - 私有属性：以双下划线`__`开头，会触发名称改写（name mangling）

```python
class Person:
    def __init__(self, name, age):
        self.name = name  # 公开属性
        self._age = age  # 保护属性
        self.__private = "secret"  # 私有属性
    
    def get_age(self):
        return self._age
    
    def set_age(self, age):
        if age > 0:
            self._age = age
    
    def get_private(self):
        return self.__private

p = Person("Alice", 25)

# 访问公开属性
print(p.name)  # Alice

# 访问保护属性（不推荐）
print(p._age)  # 25

# 访问私有属性（会触发名称改写）
# print(p.__private)  # 报错：AttributeError
print(p._Person__private)  # secret

# 通过方法访问
print(p.get_age())  # 25
p.set_age(30)
print(p.get_age())  # 30
print(p.get_private())  # secret
```

### （3）属性装饰器

- **@property**：将方法转换为属性，提供更优雅的访问方式。
- **@属性名.setter**：定义属性的设置方法。
- **@属性名.deleter**：定义属性的删除方法。

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self._age = age
    
    @property
    def age(self):
        return self._age
    
    @age.setter
    def age(self, value):
        if value > 0:
            self._age = value
        else:
            raise ValueError("Age must be positive")
    
    @age.deleter
    def age(self):
        del self._age

p = Person("Alice", 25)

# 使用属性装饰器访问
print(p.age)  # 25

# 使用属性装饰器设置
p.age = 30
print(p.age)  # 30

# 尝试设置无效值
try:
    p.age = -5
except ValueError as e:
    print(f"Error: {e}")  # Error: Age must be positive

# 删除属性
del p.age
# print(p.age)  # 报错：AttributeError
```

## ⚠️ 常见坑与注意点

1. **现象**：修改类属性影响所有实例。**原因**：类属性是所有实例共享的。**正确做法**：如需每个实例有独立的属性，应使用实例属性。

2. **现象**：访问私有属性失败。**原因**：双下划线开头的属性会被名称改写。**正确做法**：通过公共方法访问，或使用单下划线表示保护属性。

3. **现象**：__new__方法使用不当。**原因**：__new__需要返回实例，否则__init__不会被调用。**正确做法**：确保__new__返回实例，通常调用super().__new__(cls)。

4. **现象**：属性装饰器设置方法未生效。**原因**：设置方法名称与属性名不一致。**正确做法**：设置方法名称必须与@property装饰的方法名称相同。

5. **现象**：类方法和静态方法混淆。**原因**：不理解两种方法的区别和使用场景。**正确做法**：需要访问类属性时使用类方法，不需要访问类或实例属性时使用静态方法。

## 1️⃣ 继承

### （1）继承的基本语法

- **继承**：子类继承父类的属性和方法，实现代码复用。
  - **为什么需要**：避免代码重复，建立类之间的层次关系，提高代码可维护性。
  - **面试可答**：继承是面向对象编程的三大特性之一，允许子类继承父类的属性和方法，实现代码复用和扩展。

```python
# 父类
class Animal:
    def __init__(self, name):
        self.name = name
    
    def speak(self):
        return "Some generic sound"

# 子类
class Dog(Animal):
    def speak(self):
        return "Woof!"

class Cat(Animal):
    def speak(self):
        return "Meow!"

# 使用
animal = Animal("Generic Animal")
dog = Dog("Rex")
cat = Cat("Whiskers")

print(f"{animal.name}: {animal.speak()}")  # Generic Animal: Some generic sound
print(f"{dog.name}: {dog.speak()}")  # Rex: Woof!
print(f"{cat.name}: {cat.speak()}")  # Whiskers: Meow!
```

### （2）方法重写

- **方法重写**：子类重新实现父类的方法，覆盖父类的行为。
- **注意事项**：子类方法的签名应与父类保持一致，以确保多态的正确实现。

```python
class Shape:
    def area(self):
        raise NotImplementedError("Subclass must implement area()")

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height
    
    def area(self):
        return self.width * self.height

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    
    def area(self):
        import math
        return math.pi * self.radius ** 2

# 使用
shapes = [Rectangle(4, 5), Circle(3)]
for shape in shapes:
    print(f"Area: {shape.area()}")
# Area: 20
# Area: 28.274333882308138
```

### （3）super()函数

- **super()**：调用父类的方法，用于在子类中访问父类的属性和方法。
- **使用场景**：在__init__方法中初始化父类属性，在重写的方法中调用父类方法。

```python
class Parent:
    def __init__(self, name):
        self.name = name
    
    def greet(self):
        return f"Hello, {self.name}!"

class Child(Parent):
    def __init__(self, name, age):
        # 调用父类的__init__方法
        super().__init__(name)
        self.age = age
    
    def greet(self):
        # 调用父类的greet方法
        parent_greet = super().greet()
        return f"{parent_greet} I'm {self.age} years old."

# 使用
child = Child("Alice", 10)
print(child.greet())  # Hello, Alice! I'm 10 years old.
```

## 2️⃣ 多态

### （1）多态的概念

- **多态**：同一接口，不同实现。
  - **为什么需要**：提高代码的灵活性和可扩展性，允许不同类型的对象以统一的方式处理。
  - **面试可答**：多态是面向对象编程的三大特性之一，允许不同类型的对象对同一消息做出不同的响应。

```python
class Animal:
    def speak(self):
        pass

class Dog(Animal):
    def speak(self):
        return "Woof!"

class Cat(Animal):
    def speak(self):
        return "Meow!"

class Cow(Animal):
    def speak(self):
        return "Moo!"

# 多态的体现：统一接口处理不同类型的对象
def make_animal_speak(animal):
    return animal.speak()

# 使用
dog = Dog()
cat = Cat()
cow = Cow()

print(make_animal_speak(dog))  # Woof!
print(make_animal_speak(cat))  # Meow!
print(make_animal_speak(cow))  # Moo!
```

### （2）鸭子类型

- **鸭子类型**：如果一个对象看起来像鸭子，走路像鸭子，叫声像鸭子，那么它就是鸭子。
- **Python特点**：Python是动态类型语言，不强制类型，只要对象有相应的方法即可。

```python
# 鸭子类型示例：只要对象有quack方法，就可以被视为鸭子
def make_quack(duck):
    return duck.quack()

class Duck:
    def quack(self):
        return "Quack!"

class Person:
    def quack(self):
        return "I'm quacking like a duck!"

# 使用
duck = Duck()
person = Person()

print(make_quack(duck))  # Quack!
print(make_quack(person))  # I'm quacking like a duck!
```

## 3️⃣ 多重继承

### （1）多重继承的语法

- **多重继承**：一个子类继承多个父类。
- **语法**：`class Child(Parent1, Parent2, ...):`

```python
class A:
    def method(self):
        return "Method from A"

class B:
    def method(self):
        return "Method from B"

class C(A, B):
    pass

# 使用
c = C()
print(c.method())  # Method from A（MRO决定）
```

### （2）方法解析顺序（MRO）

- **MRO**：Method Resolution Order，决定了多重继承中方法的查找顺序。
- **C3线性化**：Python使用C3算法计算MRO，确保方法查找的一致性。
- **查看MRO**：使用`cls.mro()`或`help(cls)`查看类的MRO。

```python
class A:
    pass

class B(A):
    pass

class C(A):
    pass

class D(B, C):
    pass

# 查看MRO
print(D.mro())  # [<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <class 'object'>]
```

### （3）菱形继承问题

- **菱形继承**：多个子类继承同一个父类，然后一个类继承这多个子类。
- **解决方案**：Python的C3算法确保了方法查找的一致性，super()会按照MRO顺序调用父类方法。

```python
class A:
    def method(self):
        return "A.method"

class B(A):
    def method(self):
        return f"B.method -> {super().method()}"

class C(A):
    def method(self):
        return f"C.method -> {super().method()}"

class D(B, C):
    def method(self):
        return f"D.method -> {super().method()}"

# 使用
d = D()
print(d.method())  # D.method -> B.method -> C.method -> A.method
print(D.mro())  # [<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <class 'object'>]
```

## ⚠️ 常见坑与注意点

1. **现象**：多重继承导致方法冲突。**原因**：多个父类有同名方法，MRO决定调用顺序。**正确做法**：了解MRO，合理设计类层次结构，避免复杂的多重继承。

2. **现象**：super()调用顺序错误。**原因**：不理解MRO，导致super()调用了意外的父类方法。**正确做法**：查看类的MRO，确保super()调用符合预期。

3. **现象**：方法重写时签名不一致。**原因**：子类方法参数与父类不同，破坏多态。**正确做法**：保持方法签名一致，或使用*args和**kwargs兼容不同参数。

4. **现象**：菱形继承中的重复初始化。**原因**：多个父类都调用了共同父类的初始化方法。**正确做法**：使用super()确保共同父类只被初始化一次。

5. **现象**：滥用多重继承。**原因**：过度使用多重继承导致代码复杂，难以维护。**正确做法**：优先使用组合而非继承，或使用接口和混入（mixin）模式。

## 1️⃣ 字符串表示方法

### （1）__str__方法

- **__str__**：返回对象的字符串表示，用于`str()`和`print()`函数。
  - **为什么需要**：提供人类可读的对象描述，方便调试和用户交互。
  - **面试可答**：__str__方法返回对象的字符串表示，用于print()函数和str()转换，应该返回简洁、人类可读的字符串。

### （2）__repr__方法

- **__repr__**：返回对象的官方字符串表示，用于`repr()`函数和交互式解释器。
  - **面试可答**：__repr__方法返回对象的官方字符串表示，应该尽可能包含重建对象所需的信息，用于调试和开发。

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def __str__(self):
        return f"Person(name='{self.name}', age={self.age})"
    
    def __repr__(self):
        return f"Person(name='{self.name}', age={self.age})"

# 使用
p = Person("Alice", 25)
print(p)  # Person(name='Alice', age=25)（调用__str__）
print(str(p))  # Person(name='Alice', age=25)（调用__str__）
print(repr(p))  # Person(name='Alice', age=25)（调用__repr__）
```

## 2️⃣ 容器相关方法

### （1）__len__方法

- **__len__**：返回对象的长度，用于`len()`函数。
  - **为什么需要**：使对象可以被`len()`函数调用，支持容器类型的操作。

### （2）__getitem__和__setitem__方法

- **__getitem__**：支持下标访问，用于`obj[key]`。
- **__setitem__**：支持下标赋值，用于`obj[key] = value`。

### （3）__contains__方法

- **__contains__**：支持`in`操作符，用于`key in obj`。

```python
class MyList:
    def __init__(self, items):
        self.items = items
    
    def __len__(self):
        return len(self.items)
    
    def __getitem__(self, index):
        return self.items[index]
    
    def __setitem__(self, index, value):
        self.items[index] = value
    
    def __contains__(self, item):
        return item in self.items

# 使用
my_list = MyList([1, 2, 3, 4, 5])
print(len(my_list))  # 5（调用__len__）
print(my_list[0])  # 1（调用__getitem__）
my_list[0] = 10  # 调用__setitem__
print(my_list[0])  # 10
print(2 in my_list)  # True（调用__contains__）
print(100 in my_list)  # False（调用__contains__）
```

## 3️⃣ 迭代器协议

### （1）__iter__和__next__方法

- **__iter__**：返回迭代器对象，用于`iter()`函数和`for`循环。
- **__next__**：返回下一个元素，当没有更多元素时抛出`StopIteration`异常。
  - **面试可答**：迭代器协议要求实现__iter__和__next__方法，__iter__返回迭代器自身，__next__返回下一个元素或抛出StopIteration异常。

```python
class MyRange:
    def __init__(self, start, end):
        self.start = start
        self.end = end
        self.current = start
    
    def __iter__(self):
        return self
    
    def __next__(self):
        if self.current >= self.end:
            raise StopIteration
        value = self.current
        self.current += 1
        return value

# 使用
for i in MyRange(1, 5):
    print(i)  # 1, 2, 3, 4

# 或使用iter()和next()
my_range = MyRange(1, 3)
it = iter(my_range)
print(next(it))  # 1
print(next(it))  # 2
# print(next(it))  # 报错：StopIteration
```

## 4️⃣ 上下文管理器协议

### （1）__enter__和__exit__方法

- **__enter__**：进入上下文时调用，返回上下文对象。
- **__exit__**：退出上下文时调用，处理异常和清理工作。
  - **面试可答**：上下文管理器协议要求实现__enter__和__exit__方法，用于with语句，确保资源的正确获取和释放。

### （2）contextlib.contextmanager装饰器

- **contextmanager**：使用生成器快速创建上下文管理器。
  - **为什么需要**：简化上下文管理器的实现，避免编写完整的类。

```python
# 使用类实现上下文管理器
class FileManager:
    def __init__(self, filename, mode):
        self.filename = filename
        self.mode = mode
        self.file = None
    
    def __enter__(self):
        self.file = open(self.filename, self.mode)
        return self.file
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        if self.file:
            self.file.close()

# 使用with语句
with FileManager("example.txt", "w") as f:
    f.write("Hello, world!")

# 使用contextmanager装饰器
from contextlib import contextmanager

@contextmanager
def file_manager(filename, mode):
    file = open(filename, mode)
    try:
        yield file
    finally:
        file.close()

# 使用with语句
with file_manager("example.txt", "r") as f:
    content = f.read()
    print(content)  # Hello, world!
```

## ⚠️ 常见坑与注意点

1. **现象**：__repr__和__str__方法混淆。**原因**：不理解两个方法的用途和区别。**正确做法**：__str__返回人类可读的字符串，__repr__返回官方的、可重建对象的字符串。

2. **现象**：迭代器实现错误。**原因**：__iter__方法没有返回迭代器自身，或__next__方法没有正确处理结束条件。**正确做法**：__iter__应返回self，__next__在没有更多元素时抛出StopIteration。

3. **现象**：上下文管理器没有正确处理异常。**原因**：__exit__方法没有正确处理exc_type、exc_val、exc_tb参数。**正确做法**：在__exit__方法中处理异常，或返回False让异常继续传播。

4. **现象**：比较方法返回值错误。**原因**：__eq__方法没有处理类型不匹配的情况，或没有返回NotImplemented。**正确做法**：在类型不匹配时返回NotImplemented，让Python尝试反向操作。

5. **现象**：算术运算方法实现不完整。**原因**：只实现了__add__而没有实现__radd__，导致反向操作失败。**正确做法**：同时实现正向和反向运算方法，或返回NotImplemented让Python尝试反向操作。

# 🌱 第4章：装饰器

本章介绍Python中的装饰器概念，包括装饰器的定义、实现方法、常见用途以及类装饰器等内容。通过本章学习，你将掌握装饰器的核心原理，能够编写和使用装饰器来增强函数和类的功能。

------

## 1️⃣ 装饰器基础

### （1）装饰器的概念

- **装饰器**：接受函数（或类）并返回新函数（或类）的可调用对象。
  - **为什么需要**：在不修改原函数代码的前提下，增强或包装函数的行为。
  - **面试可答**：装饰器是Python的一种高级特性，用于在不修改原函数代码的情况下，增强函数的功能，实现横切关注点的分离。

### （2）装饰器的定义与使用

- **基本语法**：使用`@decorator`语法糖来应用装饰器。
- **实现原理**：装饰器函数接受原函数作为参数，返回一个包装函数。

```python
# 简单的装饰器
def simple_decorator(func):
    def wrapper(*args, **kwargs):
        print("Before function execution")
        result = func(*args, **kwargs)
        print("After function execution")
        return result
    return wrapper

# 使用装饰器
@simple_decorator
def greet(name):
    return f"Hello, {name}!"

# 调用函数
print(greet("Alice"))
# Before function execution
# Hello, Alice!
# After function execution
```

## 2️⃣ 带参数的装饰器

### （1）带参数装饰器的实现

- **带参数装饰器**：需要多一层闭包来处理参数。
- **实现原理**：外层函数接受装饰器参数，返回一个装饰器函数，装饰器函数再接受原函数并返回包装函数。

```python
# 带参数的装饰器
def repeat(num_times):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for _ in range(num_times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

# 使用带参数的装饰器
@repeat(num_times=3)
def greet(name):
    print(f"Hello, {name}!")

# 调用函数
greet("Alice")
# Hello, Alice!
# Hello, Alice!
# Hello, Alice!
```

## 3️⃣ 保留函数元信息

### （1）functools.wraps装饰器

- **wraps**：保留原函数的元信息，如函数名、文档字符串等。
  - **为什么需要**：避免装饰器包装后丢失原函数的重要信息，方便调试和文档生成。

```python
from functools import wraps

def simple_decorator(func):
    @wraps(func)  # 保留原函数元信息
    def wrapper(*args, **kwargs):
        print("Before function execution")
        result = func(*args, **kwargs)
        print("After function execution")
        return result
    return wrapper

@simple_decorator
def greet(name):
    """Greet someone"""
    return f"Hello, {name}!"

# 查看函数元信息
print(greet.__name__)  # greet（保留了原函数名）
print(greet.__doc__)  # Greet someone（保留了原函数文档）
```

## 4️⃣ 常见装饰器用途

### （1）计时装饰器

- **计时装饰器**：测量函数执行时间。
  - **应用场景**：性能分析、优化代码。

```python
import time
from functools import wraps

def timer(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start_time = time.time()
        result = func(*args, **kwargs)
        end_time = time.time()
        print(f"{func.__name__} took {end_time - start_time:.4f} seconds")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(1)
    return "Done"

slow_function()  # slow_function took 1.0012 seconds
```

### （2）日志装饰器

- **日志装饰器**：记录函数的调用信息。
  - **应用场景**：调试、监控、审计。

```python
import logging
from functools import wraps

# 配置日志
logging.basicConfig(level=logging.INFO)

def logger(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        logging.info(f"Calling {func.__name__} with args={args}, kwargs={kwargs}")
        result = func(*args, **kwargs)
        logging.info(f"{func.__name__} returned {result}")
        return result
    return wrapper

@logger
def add(a, b):
    return a + b

add(3, 4)  # INFO:root:Calling add with args=(3, 4), kwargs={} | INFO:root:add returned 7
```

### （3）缓存装饰器

- **缓存装饰器**：缓存函数的返回结果，避免重复计算。
  - **应用场景**：计算密集型函数、频繁调用的函数。

```python
from functools import wraps, lru_cache

# 自定义缓存装饰器
def cache(func):
    cache_data = {}
    
    @wraps(func)
    def wrapper(*args, **kwargs):
        key = str(args) + str(kwargs)
        if key not in cache_data:
            cache_data[key] = func(*args, **kwargs)
        return cache_data[key]
    
    return wrapper

# 使用自定义缓存装饰器
@cache
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

# 使用内置的lru_cache装饰器
@lru_cache(maxsize=128)
def fibonacci_lru(n):
    if n <= 1:
        return n
    return fibonacci_lru(n-1) + fibonacci_lru(n-2)

print(fibonacci(30))  # 832040
print(fibonacci_lru(30))  # 832040
```

### （4）权限装饰器

- **权限装饰器**：检查用户是否有权限执行函数。
  - **应用场景**：Web应用、API接口。

```python
from functools import wraps

def require_permission(permission):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            # 假设current_user是当前登录用户
            current_user = {"username": "alice", "permissions": ["read", "write"]}
            
            if permission not in current_user["permissions"]:
                raise PermissionError(f"User {current_user['username']} does not have {permission} permission")
            
            return func(*args, **kwargs)
        return wrapper
    return decorator

@require_permission("write")
def create_post(title, content):
    return f"Created post: {title}"

@require_permission("admin")
def delete_post(post_id):
    return f"Deleted post {post_id}"

print(create_post("Hello", "World"))  # Created post: Hello
try:
    delete_post(1)
except PermissionError as e:
    print(f"Error: {e}")  # Error: User alice does not have admin permission
```

## 5️⃣ 类装饰器

### （1）类装饰器的实现

- **类装饰器**：使用类来实现装饰器，需要实现`__call__`方法。
  - **应用场景**：更复杂的装饰逻辑，需要维护状态。

```python
from functools import wraps

class CountCalls:
    def __init__(self, func):
        self.func = func
        self.calls = 0
        wraps(func)(self)  # 保留原函数元信息
    
    def __call__(self, *args, **kwargs):
        self.calls += 1
        print(f"Call {self.calls} of {self.func.__name__}")
        return self.func(*args, **kwargs)

@CountCalls
def greet(name):
    return f"Hello, {name}!"

print(greet("Alice"))  # Call 1 of greet | Hello, Alice!
print(greet("Bob"))  # Call 2 of greet | Hello, Bob!
print(greet.calls)  # 2
```

### （2）类装饰器的应用

- **单例模式**：确保类只有一个实例。
- **注册模式**：自动注册类或函数。

```python
# 单例装饰器
class Singleton:
    def __init__(self, cls):
        self.cls = cls
        self.instance = None
    
    def __call__(self, *args, **kwargs):
        if self.instance is None:
            self.instance = self.cls(*args, **kwargs)
        return self.instance

@Singleton
class Database:
    def __init__(self, connection_string):
        self.connection_string = connection_string
        print(f"Connecting to {connection_string}")

# 创建实例
db1 = Database("localhost:5432")  # Connecting to localhost:5432
db2 = Database("localhost:5432")
print(db1 is db2)  # True（同一个实例）
```

## ⚠️ 常见坑与注意点

1. **现象**：装饰器包装后丢失函数元信息。**原因**：包装函数替换了原函数，导致函数名、文档等信息丢失。**正确做法**：使用functools.wraps装饰器保留原函数元信息。

2. **现象**：带参数装饰器实现错误。**原因**：没有正确实现多层闭包结构。**正确做法**：确保外层函数接受装饰器参数，返回装饰器函数，装饰器函数再接受原函数并返回包装函数。

3. **现象**：装饰器执行顺序问题。**原因**：多个装饰器应用时，执行顺序是从下到上。**正确做法**：注意装饰器的应用顺序，确保功能正确。

4. **现象**：装饰器导致函数签名改变。**原因**：包装函数使用*args和**kwargs，导致函数签名不明确。**正确做法**：使用functools.wraps保留原函数签名，或使用inspect模块处理签名。

5. **现象**：装饰器与异步函数不兼容。**原因**：普通装饰器无法正确处理异步函数。**正确做法**：使用asyncio.coroutine装饰器，或确保包装函数是异步的。

## 🎯 面试常见追问

| 面试官提问 | 回答思路 |
|------------|----------|
| 什么是装饰器？装饰器的作用是什么？ | 装饰器是接受函数并返回新函数的可调用对象，用于在不修改原函数代码的情况下增强函数功能，实现横切关注点的分离。 |
| 装饰器的实现原理是什么？ | 装饰器利用闭包原理，接受原函数作为参数，返回一个包装函数，包装函数在调用原函数前后添加额外逻辑。 |
| functools.wraps的作用是什么？ | functools.wraps装饰器用于保留原函数的元信息，如函数名、文档字符串、参数签名等，避免装饰器包装后丢失这些信息。 |
| 如何实现带参数的装饰器？ | 带参数的装饰器需要多一层闭包：外层函数接受装饰器参数，返回装饰器函数，装饰器函数再接受原函数并返回包装函数。 |
| 装饰器的执行顺序是怎样的？ | 多个装饰器应用时，执行顺序是从下到上（或从内到外）。例如，@decorator1 @decorator2 def func(): 会先应用decorator2，再应用decorator1。 |
| 类装饰器和函数装饰器的区别是什么？ | 函数装饰器使用函数实现，类装饰器使用类实现并需要__call__方法。类装饰器可以维护状态，更适合复杂的装饰逻辑。 |

## 第5章：生成器与迭代器

1. **迭代器**：实现 __iter__ 与 __next__；迭代器也是可迭代对象；StopIteration 表示结束；iter()、next() 内置函数。
2. **生成器**：用 **yield** 的函数，调用时返回生成器对象（迭代器）；惰性求值，节省内存；yield 可带值（生成器.send() 与协程基础）。
3. **生成器表达式**：(x*2 for x in lst) 类似列表推导但惰性；适合大数据流式处理。
4. **面试要点**：迭代器与可迭代区别、生成器与列表推导区别、yield 与 return。

------

## 第6章：异常处理

1. **try / except / else / finally**：except 捕获指定异常；可多个 except、或 except Exception；else 在无异常时执行；finally 无论如何都执行（常用于清理）。
2. **异常类型**：BaseException ← Exception ← 具体异常（ValueError、TypeError、KeyError、IndexError 等）；自定义异常继承 Exception。
3. **raise**：主动抛出异常；**raise from** 保留异常链；**assert** 用于调试断言（可被 -O 关闭）。
4. **面试要点**：except 顺序（子类在前）、finally 与 return、异常最佳实践（具体异常、避免裸 except）。

------

## 第7章：上下文管理器与 with

1. **with** 语句：保证进入与退出时执行固定逻辑（如打开/关闭文件、加锁/解锁）；对象需实现 **__enter__** 与 **__exit__**。
2. **contextlib.contextmanager**：用生成器快速写上下文管理器；yield 前为 __enter__，后为 __exit__。
3. **应用**：文件 open、线程锁、数据库连接；保证资源释放与异常安全。
4. **面试要点**：with 原理、__exit__ 是否吞掉异常（返回 True 则抑制）。
