### 1.1 Vue 简介

#### （1）定义与定位

**Vue** 是一套用于构建用户界面的**渐进式 JavaScript 框架**，由尤雨溪（Evan You）创建并维护，核心库只关注**视图层**，易于与现有项目或其它库（如 Vue Router、Pinia）集成。

| 项目 | 说明 |
|------|------|
| **核心能力** | 响应式数据绑定、组件化、声明式模板、虚拟 DOM 与 diff、Composition API。 |
| **典型场景** | 中后台管理系统、移动端 H5、与 Java 后端配合的前端项目、渐进增强的现有站点。 |
| **与 React/Angular** | Vue 模板语法友好、上手快；React 生态大、JSX 灵活；Angular 全家桶、企业级。 |

> 💬 **一句话**：Vue 解决**数据驱动视图、组件复用与维护**问题，适合「视图层 + 状态」场景，通过响应式与组件化把 UI 与数据绑定在一起，减少手写 DOM 与重复代码。

#### （2）传统开发痛点（为什么需要 Vue 或类似框架）

| 痛点 | 说明 |
|------|------|
| **DOM 操作繁琐** | 原生 JS / jQuery 需手动 getElementById、innerHTML、addEventListener，逻辑与视图耦合，难以维护。 |
| **数据与视图不同步** | 数据变更需手动更新多处 DOM，易遗漏、重复代码多，易出 Bug。 |
| **无组件边界** | 页面逻辑全堆在一起，复用靠复制粘贴，协作与测试困难。 |
| **前后端未分离时** | 服务端渲染整页 HTML，交互与局部更新需大量手写，难以做复杂 SPA。 |

- **Vue 的应对**：**声明式**——用模板描述「数据与视图的对应关系」，数据变更时由框架**自动**更新 DOM；**组件化**——页面拆成可复用组件，便于协作与测试；**响应式**——改数据即生效，无需手动选中元素再改。**类比**：jQuery 像「手把手教机器人每一步」，Vue 像「告诉机器人目标状态，由它自己算怎么改」。

------

### 1.2 核心特点

| 特点 | 说明 | 实际意义 |
|------|------|----------|
| **响应式** | 数据驱动视图，数据变更自动更新 DOM | 避免手写 DOM 更新、减少 Bug |
| **组件化** | 页面由组件拼装，可复用与组合 | 可维护、可测试、多人协作 |
| **声明式** | 模板 + 数据绑定，描述「长什么样」而非「怎么改 DOM」 | 代码更清晰、意图更明确 |
| **渐进式** | 可从局部使用（只引入核心）到整站框架（Router、Pinia、Vite） | 既能改造老项目，也能做新项目 |

> 💡 **重点**：Vue 3 为主流版本，提供 **Composition API**（`ref`、`reactive`、`computed`、`watch`）、更好的 **TypeScript** 支持、**Tree-shaking** 与性能优化；新项目建议直接用 Vue 3 + Vite。

### 1.3 Vue 3 与 Vue 2 简要对比

| 对比项 | Vue 2 | Vue 3 |
|--------|--------|--------|
| **API 风格** | Options API（data、methods、computed） | Composition API（ref、reactive、setup）可选，与 Options 并存 |
| **响应式** | Object.defineProperty，无法检测数组下标与新增属性 | Proxy，全面响应式 |
| **TypeScript** | 支持一般 | 源码 TS 重写，类型推导更好 |
| **打包** | 全量或按需 | Tree-shaking，未用到的 API 可被摇掉 |
| **多根节点** | 单根（template 只能一个根元素） | Fragment，支持多根 |

------

### 1.4 与 React、Angular 简要对比

| 对比项 | Vue | React | Angular |
|--------|-----|--------|--------|
| **模板/视图** | 单文件组件（.vue）：template + script + style | JSX，JS 中写类 HTML | 模板 + TypeScript，强约定 |
| **学习曲线** | 低，中文文档与生态友好 | 中，概念少但生态多 | 高，全家桶、依赖注入等 |
| **状态管理** | Pinia（官方推荐）/ Vuex | Redux、Zustand 等 | 内置 RxJS、Service |
| **路由** | Vue Router | React Router | 内置 Router |
| **适用场景** | 中后台、H5、与后端配合 | 大厂、复杂交互、跨端 | 企业级、大型应用 |

> 🔹 **注意**：Vue 适合中后台、移动端 H5、与 Java 后端配合的前端项目；选型时看团队技术栈与历史项目，不必盲目追新。

- **2014** 年尤雨溪发布 Vue 0.x；**2015** 年 Vue 1.0、**2016** 年 Vue 2.0（组件化、虚拟 DOM、响应式成熟）；**2020** 年 Vue 3.0（Composition API、Proxy 响应式、TS、Monorepo）。

### 2.1 创建应用与挂载

#### （1）最小可运行示例

> **一句话**：Vue 3 使用 **createApp** 创建应用实例，再通过 **mount** 挂载到某个 DOM 节点；挂载后该节点内的内容由 Vue 接管。

```html
<!-- index.html 中留一个挂载点 -->
<div id="app"></div>
```

```javascript
// main.js：创建应用并挂载
import { createApp } from 'vue'
import App from './App.vue'

const app = createApp(App)
app.mount('#app')
```

> 🔹 **注意**：一个页面可以多次 `createApp` 挂载到不同 `#id`，但通常 SPA 只挂载一个根组件 **App**；路由、全局组件、use(Pinia) 等在 `app` 上配置后再 `mount`。

#### （2）与 Vue 2 的差异

| 对比项 | Vue 2 | Vue 3 |
|--------|--------|--------|
| **创建实例** | `new Vue({ render: h => h(App) }).$mount('#app')` | `createApp(App).mount('#app')` |
| **全局 API** | Vue.component、Vue.use、Vue.prototype | app.component、app.use、app.config.globalProperties |

### 2.2 模板语法：插值与指令

#### （1）插值

- **文本插值**：`{{ message }}`，将组件的响应式数据渲染为文本；双大括号内支持**单个表达式**（如 `{{ count + 1 }}`），不能写语句。
- **原始 HTML**：`v-html="rawHtml"`，将数据当作 HTML 渲染；**仅可信内容**使用，否则有 XSS 风险。

```html
<p>文本：{{ message }}</p>
<p>表达式：{{ count * 2 }}</p>
<div v-html="rawHtml"></div>
```

#### （2）常用指令速览

| 指令 | 说明 | 示例 |
|------|------|------|
| **v-if / v-else-if / v-else** | 条件渲染，不满足的节点不渲染 | `v-if="show"` |
| **v-show** | 切换 `display: none`，节点始终在 DOM 中 | `v-show="visible"` |
| **v-for** | 列表渲染，必须配合 **:key**（唯一，建议 id） | `v-for="(item, i) in list" :key="item.id"` |
| **v-bind** 或 **:** | 绑定属性 | `:class="cls"` `:style="sty"` |
| **v-on** 或 **@** | 绑定事件 | `@click="fn"` `@submit.prevent="fn"` |
| **v-model** | 双向绑定（见 2.5） | `v-model="text"` |

- **:class** 常用写法：对象 `:class="{ active: isActive }"`、数组 `:class="[clsA, clsB]"`、对象+数组混用；**:style** 可绑对象 `{ color: colorVal }` 或数组。
- **事件修饰符**：`.stop`（阻止冒泡）、`.prevent`（阻止默认，如 form 提交）、`.once`（只触发一次）、`.self`（仅事件源自身触发才执行）；按键修饰符 `.enter`、`.tab` 等。

> 💡 **重点**：**v-if** 切换会销毁/重建节点，**v-show** 只改 CSS；频繁切换用 v-show，否则用 v-if。**v-for** 的 key 不要用 index（列表增删会导致错位与性能问题），用唯一 id。

### 2.3 响应式基础：ref、reactive、computed、watch

#### （1）ref 与 reactive

| API | 适用类型 | 使用方式 | 说明 |
|-----|----------|----------|------|
| **ref** | 基本类型（number、string、boolean）或任意值 | 在 script 中 `.value` 读写；模板中自动解包，无需 .value | 响应式引用，始终通过 .value 访问/修改 |
| **reactive** | 对象（含数组） | 直接改属性 `state.name = 'x'` | 深度响应式，不能整体替换（会丢响应式） |

```javascript
// script setup 示例
import { ref, reactive } from 'vue'

const count = ref(0)
count.value++   // script 中必须 .value

const state = reactive({ name: 'Vue', list: [] })
state.name = 'Vue 3'   // 直接改属性
state.list.push(1)     // 数组同样
```

```html
<!-- 模板中 ref 自动解包 -->
<p>{{ count }}</p>
<p>{{ state.name }}</p>
```

> 💬 **面试怎么说**：ref 适合基本类型和「需要整体替换」的引用；reactive 适合对象/数组且不会整体替换。ref 在模板中自动解包，在 script 里要 .value；reactive 不能整体赋值，否则失去响应式。

#### （2）computed 与 watch

| API | 用途 | 特点 |
|-----|------|------|
| **computed** | 依赖响应式数据计算得出新值 | 有缓存，依赖不变不重算；只读，如需可写可用 get/set |
| **watch** | 侦听某个源，执行副作用 | 可拿到新旧值、可配置 deep/immediate |
| **watchEffect** | 自动收集依赖，依赖变就执行 | 无需显式指定源，适合「用到的都侦听」 |

```javascript
import { ref, computed, watch, watchEffect } from 'vue'

const count = ref(0)
const double = computed(() => count.value * 2)  // 只读

watch(count, (newVal, oldVal) => {
  console.log(newVal, oldVal)
})

watchEffect(() => {
  console.log(count.value)  // 自动依赖 count
})
```

### 2.4 生命周期钩子（Composition API）

| 钩子 | 时机 |
|------|------|
| **onBeforeMount** | 挂载前 |
| **onMounted** | 挂载完成，可访问 DOM、可请求接口 |
| **onBeforeUpdate** | 响应式数据变更、DOM 更新前 |
| **onUpdated** | DOM 更新后 |
| **onBeforeUnmount** | 卸载前 |
| **onUnmounted** | 卸载后，清理定时器、解绑事件等 |

**生命周期执行顺序（面试可答）**：

1. **创建阶段**：setup 执行（Composition API）→ beforeCreate / created（Options API，若有）→ 模板编译、生成渲染函数。
2. **挂载阶段**：onBeforeMount → 创建 VNode、挂载到 DOM → onMounted（此时可访问 DOM、适合请求接口）。
3. **更新阶段**：数据变更 → onBeforeUpdate → 虚拟 DOM diff、patch 真实 DOM → onUpdated。
4. **卸载阶段**：onBeforeUnmount → 卸载子组件、解绑事件、销毁 → onUnmounted（此时清理定时器、取消订阅）。

```javascript
import { onMounted, onUnmounted } from 'vue'

onMounted(() => {
  console.log('组件已挂载')
  // 常见：请求接口、订阅、操作 DOM
})

onUnmounted(() => {
  // 清理：clearInterval、removeEventListener
})
```

> 🔹 **注意**：Composition API 中钩子按**注册顺序**执行；若与 Options API 混用，setup 先于 beforeCreate/created 执行，onMounted 在 mounted 之前。

### 2.5 常见错误写法 vs 正确写法

| 场景 | 错误写法 / 现象 | 正确写法 / 做法 |
|------|-----------------|-----------------|
| **reactive 整体替换** | `state = { name: 'new' }` 后视图不更新 | 改属性：`state.name = 'new'`；或改用 **ref** 包对象，整体赋 `state.value = { name: 'new' }`。 |
| **v-for 用 index 当 key** | 列表增删后出现错位、复用了错误节点 | **key 用唯一 id**：`:key="item.id"`；无 id 时可用业务唯一字段或生成稳定 key。 |
| **script 里 ref 忘记 .value** | `count++` 报错或无效（count 是 ref 对象） | 在 script 中读写用 **count.value++**；模板中自动解包，不需 .value。 |
| **组件上 v-model 不生效** | 子组件没有接收也没有 emit | 子组件 **defineProps(['modelValue'])**，在输入时 **emit('update:modelValue', $event.target.value)**。 |
| **在子组件里直接改 props** | 控制台警告或父组件状态不同步 | props 只读；用**本地 ref 拷贝**或 **emit** 让父组件改；需要双向用 v-model。 |

### 2.6 表单双向绑定（v-model）

- **v-model** 是 **:value + @input**（或 :value + @change）的语法糖，用于**表单控件**与响应式数据的双向绑定：数据变→视图变，用户输入→数据变。
- 在**组件**上使用时，默认对应 **modelValue** prop 与 **update:modelValue** 事件；可改为自定义名（如 `v-model:title="title"`）。

```html
<input v-model="text" />
<!-- 等价于 -->
<input :value="text" @input="text = $event.target.value" />
```

```javascript
const text = ref('')
```

### 常见坑与注意点

| 现象 / 易错点 | 原因 | 怎么改 / 怎么记 |
|---------------|------|-----------------|
| reactive 整体赋值后不更新视图 | 整体赋值会断开响应式引用 | 改属性而不是 `state = { ... }`；或改用 ref 包对象，整体赋 `xxx.value = { ... }`。 |
| v-for 用 index 当 key，列表增删错乱 | diff 用 key 认节点，index 会变 | **key 用唯一 id**（如 item.id），不要用 index。 |
| 在 script 里忘记 ref 的 .value | ref 在 script 中要通过 .value 访问 | 模板里不用 .value，script 里必须 count.value。 |
| v-model 在自定义组件上不生效 | 未定义 modelValue + update:modelValue | 组件内 defineProps(['modelValue'])、emit('update:modelValue', newVal)。 |

### 3.1 单文件组件（SFC）与 script setup

#### （1）什么是 SFC

- **单文件组件**：一个 `.vue` 文件包含 **template**（模板）、**script**（逻辑）、**style**（样式），由 Vite/Vue CLI 编译成 JS 与 CSS。**为什么用**：组件边界清晰、可复用、支持 scoped 样式与预处理器。
- **script setup**：在 `<script setup>` 中写的顶层变量、函数、import 会**自动暴露给模板**，无需 `return`；**defineProps**、**defineEmits** 为编译器宏，无需导入。

```vue
<!-- HelloWorld.vue -->
<script setup>
import { ref } from 'vue'
const count = ref(0)
const props = defineProps({ title: String })
const emit = defineEmits(['update'])
</script>

<template>
  <h1>{{ props.title }}</h1>
  <button @click="emit('update')">点击</button>
</template>

<style scoped>
h1 { color: #42b983; }
</style>
```

> 🔹 **注意**：`scoped` 使样式只作用于当前组件（通过 data 属性选择器实现）；父组件无法直接覆盖，需用 `:deep()` 或全局类。

#### （2）组件注册与使用

| 方式 | 说明 | 示例 |
|------|------|------|
| **局部注册** | 在父组件中 import 后使用 | `import Child from './Child.vue'`，模板中 `<Child />` |
| **全局注册** | app.component('Name', Comp)，任意模板可用 | 通用基础组件（按钮、图标）可全局注册 |

### 3.2 Props 与事件（defineProps、defineEmits）

#### （1）Props：父 → 子

- **defineProps** 定义组件接收的 props；可写**对象**（带类型与默认值）或**数组**（仅名字）。props 为**只读**，子组件不应修改，若需「双向」用 v-model 或事件回传。

```javascript
// 对象写法，推荐
const props = defineProps({
  title: { type: String, required: true },
  count: { type: Number, default: 0 }
})
```

```html
<!-- 父组件 -->
<Child :title="msg" :count="num" />
```

#### （2）Emits：子 → 父

- **defineEmits** 声明组件会触发的事件；父组件通过 **@事件名** 监听。**推荐**在 emits 中声明事件名，便于文档与校验。

```javascript
const emit = defineEmits(['update', 'delete'])
emit('update', newValue)
```

```html
<!-- 父组件 -->
<Child @update="handleUpdate" @delete="handleDelete" />
```

> 💬 **面试怎么说**：父子通信——父用 **props** 传数据给子，子用 **emit** 触发事件把数据或动作通知父；Vue 3 推荐 defineProps/defineEmits，在 script setup 中无需 return。

### 3.3 v-model 在组件上的实现

- 父组件 **v-model="xxx"** 等价于 **:modelValue="xxx"** + **@update:modelValue="xxx = $event"**。子组件需：**defineProps(['modelValue'])**，在需要更新时 **emit('update:modelValue', newVal)**。
- **多个 v-model**：如 `v-model:title="title"` 对应 **title** prop 与 **update:title** 事件。

```vue
<!-- 子组件 CustomInput.vue -->
<script setup>
const props = defineProps(['modelValue'])
const emit = defineEmits(['update:modelValue'])
</script>
<template>
  <input :value="props.modelValue" @input="emit('update:modelValue', $event.target.value)" />
</template>
```

```html
<!-- 父组件 -->
<CustomInput v-model="text" />
```

### 3.4 插槽（slot）与透传（inheritAttrs）

| 概念 | 说明 | 典型用法 |
|------|------|----------|
| **默认插槽** | 子组件内 `<slot />`，父组件把内容填进去 | 布局组件、卡片内容 |
| **具名插槽** | 子组件 `<slot name="header" />`，父用 `v-slot:header` 或 `#header` | 表头、表体、页脚 |
| **作用域插槽** | 子组件 `<slot :row="row" />`，父用 `#default="{ row }"` 拿到数据 | 表格行内自定义渲染 |
| **透传** | 父打在子组件上的 class、style、事件等，未在 props/emits 声明的会落到子根元素 | 用 `inheritAttrs: false` + `$attrs` 可自定义落点 |

```vue
<!-- 子组件 -->
<template>
  <div>
    <slot name="header" />
    <slot :item="item" />
  </div>
</template>
```

```html
<!-- 父组件 -->
<MyComp>
  <template #header>标题</template>
  <template #default="{ item }">{{ item.name }}</template>
</MyComp>
```

### 3.5 依赖注入（provide / inject）

- **跨层级**传值：祖先 **provide(key, value)**，后代 **inject(key)**，无需逐层 props。适合主题、locale、全局配置等。
- **注意**：inject 到的仍是**提供时的引用**，若提供的是响应式对象，后代改属性会生效；若提供的是原始值，则非响应式，可 provide 一个 ref/reactive。

```javascript
// 祖先
import { provide, ref } from 'vue'
const theme = ref('light')
provide('theme', theme)

// 后代
import { inject } from 'vue'
const theme = inject('theme')
```

### 3.6 组件通信小结与选型

| 场景 | 推荐方式 | 说明 |
|------|----------|------|
| **父 → 子** | props | 单一数据源，类型可校验 |
| **子 → 父** | emit 事件 | 子不直接改 props，通过事件通知父 |
| **父 ↔ 子 双向** | v-model（modelValue + update:modelValue） | 封装输入类组件时标准写法 |
| **兄弟** | 提升状态到共同父组件 + props/emit，或 Pinia | 简单用提升状态，多组件共享用 Pinia |
| **跨多层** | provide / inject 或 Pinia | 主题、locale、配置用 provide/inject；全局用户态用 Pinia |
| **任意组件共享** | Pinia | 用户信息、购物车、全局 UI 状态 |

### 3.7 组件通信常见错误 vs 正确写法

| 场景 | 错误写法 / 现象 | 正确写法 / 做法 |
|------|-----------------|-----------------|
| **子组件修改 props** | 在子组件里 `props.count++` 或 `props.obj.x = 1` | props 只读；用**本地 ref** 拷贝初始值再改，或 **emit** 让父组件改；需要双向用 v-model。 |
| **v-model 在组件上不生效** | 子组件没定义 modelValue 与 update:modelValue | **defineProps(['modelValue'])**，输入时 **emit('update:modelValue', newVal)**；等价于父 :modelValue + @update:modelValue。 |
| **插槽不显示** | 子组件没写 `<slot />` 或 name 与父不匹配 | 默认内容用 `<slot />`；具名用 `<slot name="header" />`，父用 `#header` 或 `v-slot:header`。 |
| **provide 原始值，inject 不响应** | `provide('theme', 'dark')`，后代改不了 | 提供 **ref** 或 **reactive**：`provide('theme', ref('dark'))`，后代 inject 后 .value 改即可。 |
| **emit 未声明** | 直接 emit('update')，无类型与文档提示 | **defineEmits(['update', 'delete'])** 声明后再 emit，便于校验与维护。 |

### 4.1 Vue Router 入门

#### （1）基本配置

- **Vue Router** 是 Vue 官方的**路由管理器**，实现 SPA 的路径与组件映射。安装后通过 **createRouter** 创建路由表，再 **app.use(router)** 挂载。

```javascript
// router/index.js
import { createRouter, createWebHistory } from 'vue-router'
import Home from '../views/Home.vue'

const routes = [
  { path: '/', name: 'Home', component: Home },
  { path: '/about', name: 'About', component: () => import('../views/About.vue') }
]
const router = createRouter({ history: createWebHistory(), routes })
export default router
```

```javascript
// main.js
import { createApp } from 'vue'
import App from './App.vue'
import router from './router'
createApp(App).use(router).mount('#app')
```

```html
<!-- App.vue：路由出口与链接 -->
<RouterView />
<RouterLink to="/">首页</RouterLink>
<RouterLink to="/about">关于</RouterLink>
```

#### （2）动态路由与参数

- **动态段**：`path: '/user/:id'`，组件内通过 **useRoute()** 取 **route.params.id**。**query**：`/search?q=vue`，用 **route.query.q**。

```javascript
{ path: '/user/:id', name: 'User', component: () => import('@/views/User.vue') }
```

```javascript
import { useRoute } from 'vue-router'
const route = useRoute()
console.log(route.params.id, route.query)
```

#### （3）导航守卫

| 守卫 | 时机 | 常见用途 |
|------|------|----------|
| **beforeEach** | 每次导航前 | 鉴权：无 token 跳登录、有权限才进某页 |
| **beforeResolve** | 导航确认前、组件内守卫之后 | 最后统一校验 |
| **afterEach** | 导航完成后 | 埋点、改标题 |

**守卫执行顺序（面试可答）**：**全局 beforeEach** → **路由独享 beforeEnter** → **组件内 beforeRouteEnter** → **beforeResolve** → 导航确认 → **afterEach** → 组件 **beforeRouteUpdate** / **beforeRouteLeave**（离开时）。鉴权一般放在 **beforeEach** 即可。

```javascript
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')
  if (to.meta.requiresAuth && !token) {
    next({ name: 'Login' })
  } else {
    next()
  }
})
```

- **路由元信息**：在路由配置中加 **meta: { requiresAuth: true }**，守卫里用 **to.meta** 判断。

#### （4）懒加载

- 用 **() => import('@/views/XXX.vue')** 替代直接 import，打包时会拆成独立 chunk，首屏只加载当前路由组件，**加快首屏**。

### 4.2 Pinia 入门

- **Pinia** 是 Vue 官方推荐的**状态管理**库（Vuex 的继任），支持 Composition API、TypeScript、无 mutations、模块化自然（每个 store 独立）。

#### （1）定义 store

```javascript
// stores/user.js
import { defineStore } from 'pinia'

export const useUserStore = defineStore('user', {
  state: () => ({ name: '', token: '' }),
  getters: {
    isLoggedIn: (state) => !!state.token
  },
  actions: {
    setUser(name, token) {
      this.name = name
      this.token = token
    },
    logout() {
      this.name = ''
      this.token = ''
    }
  }
})
```

#### （2）在组件中使用

```javascript
import { useUserStore } from '@/stores/user'
import { storeToRefs } from 'pinia'

const userStore = useUserStore()
// 直接读 state、调 actions
userStore.name
userStore.setUser('Tom', 'xxx')

// 解构保持响应式：用 storeToRefs，否则会丢响应式
const { name, isLoggedIn } = storeToRefs(userStore)
```

**完整示例（含 getters 与 actions 调用）**：

```javascript
// stores/counter.js
import { defineStore } from 'pinia'
export const useCounterStore = defineStore('counter', {
  state: () => ({ count: 0 }),
  getters: {
    double: (state) => state.count * 2
  },
  actions: {
    increment() { this.count++ }
  }
})
```

```vue
<!-- 组件内 -->
<script setup>
import { useCounterStore } from '@/stores/counter'
import { storeToRefs } from 'pinia'
const store = useCounterStore()
const { count, double } = storeToRefs(store)  // 解构仍响应式
</script>
<template>
  <p>{{ count }} / {{ double }}</p>
  <button @click="store.increment()">+1</button>
</template>
```

### 4.3 与后端接口联调

- **axios**：在 **onMounted** 或事件中 **axios.get/post**，结果用 **ref/reactive** 存，模板中展示；**环境变量**：Vite 用 **import.meta.env.VITE_XXX**，区分开发/生产接口地址。
- **跨域**：开发阶段用 Vite 的 **server.proxy** 代理到后端；生产由 Nginx 或后端 CORS 处理。

**请求示例**：

```javascript
import axios from 'axios'
const list = ref([])
onMounted(async () => {
  const { data } = await axios.get(`${import.meta.env.VITE_API_BASE}/api/list`)
  list.value = data
})
```

**axios 拦截器（鉴权 + 401 跳登录）**：

```javascript
// 请求拦截：统一带 token
axios.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})
// 响应拦截：401 清 token 并跳登录
axios.interceptors.response.use(
  res => res,
  err => {
    if (err.response?.status === 401) {
      localStorage.removeItem('token')
      router.push({ name: 'Login' })
    }
    return Promise.reject(err)
  }
)
```

```javascript
// vite.config.js 开发代理
export default {
  server: {
    proxy: { '/api': { target: 'http://localhost:8080', changeOrigin: true } }
  }
}
```

### 4.4 路由与状态常见错误 vs 正确写法

| 场景 | 错误写法 / 现象 | 正确写法 / 做法 |
|------|-----------------|-----------------|
| **解构 Pinia 后不响应** | `const { name } = useUserStore()`，name 改后视图不更新 | 用 **storeToRefs(store)** 解构 state/getters；actions 可直接从 store 解构调用。 |
| **守卫里 next 忘写或写两次** | 导航卡住或报错 "NavigationDuplicated" | 每个分支**必须调用一次 next()**；不能在同一分支里 next() 两次。 |
| **生产环境接口 404** | 开发用 proxy，生产请求相对路径或 localhost | 生产接口用 **import.meta.env.VITE_API_BASE**，在 `.env.production` 中配置真实域名。 |
| **路由 meta 未声明 requiresAuth** | 所有页面都能进，鉴权形同虚设 | 需登录的 route 加 **meta: { requiresAuth: true }**，beforeEach 里判断 to.meta.requiresAuth。 |

### 5.1 Vite 与项目结构

#### （1）创建项目与脚本

- **Vite** 是 Vue 官方推荐的构建工具：开发时基于 **ESM** 按需编译，**启动快**；生产用 **Rollup** 打包。创建：`npm create vue@latest` 或 `npm create vite@latest my-app -- --template vue`。

| 脚本 | 说明 |
|------|------|
| **npm run dev** | 启动开发服务器，默认 http://localhost:5173 |
| **npm run build** | 生产构建，输出到 dist |
| **npm run preview** | 本地预览 dist 产物 |

#### （2）典型项目结构

```
my-vue-app/
├── public/           # 静态资源，不经过构建
├── src/
│   ├── assets/       # 需构建的静态资源（图片、样式）
│   ├── components/   # 公共组件
│   ├── views/        # 页面级组件（配合路由）
│   ├── stores/       # Pinia stores
│   ├── router/       # Vue Router 配置
│   ├── App.vue       # 根组件
│   └── main.js       # 入口：createApp、use(router)、use(pinia)、mount
├── index.html        # 挂载点 <div id="app"></div>
├── vite.config.js    # Vite 配置（proxy、alias、env 等）
├── .env.development  # 开发环境变量
└── .env.production   # 生产环境变量
```

#### （3）环境变量

- 以 **VITE_** 开头的变量会暴露给客户端；使用 **import.meta.env.VITE_XXX**。`.env.development` / `.env.production` 区分环境。

```bash
# .env.development
VITE_API_BASE=http://localhost:5173

# .env.production
VITE_API_BASE=https://api.example.com
```

### 5.3 构建与部署

- **npm run build** 生成 **dist** 目录（静态 HTML/JS/CSS）。部署：将 dist 内容放到 **Nginx** 等静态服务器；若为 SPA，需配置 **try_files** 让所有路径回退到 index.html，避免刷新 404。

```nginx
location / {
  root /var/www/dist;
  try_files $uri $uri/ /index.html;
}
```

### 5.4 单元测试与 E2E 简介

- **单元测试**：Vue 推荐 **Vitest** + **Vue Test Utils**，测组件渲染与交互。**E2E**：**Cypress** 或 **Playwright**，测完整用户流程。校招能说出「用 Vitest 测组件、用 Cypress 测关键流程」即可。

### 5.5 工程化常见错误 vs 正确写法

| 场景 | 错误写法 / 现象 | 正确写法 / 做法 |
|------|-----------------|-----------------|
| **生产接口地址错** | 写死 `localhost` 或相对路径 `/api` | 用 **import.meta.env.VITE_API_BASE**，在 `.env.production` 配置生产域名。 |
| **部署后刷新 404** | SPA 只有 index.html，直接访问 /user 等路径 404 | Nginx 配置 **try_files $uri $uri/ /index.html**，让所有路径回退到 index.html。 |
| **环境变量不生效** | 写了 `API_BASE` 未加 VITE_ 前缀 | 只有 **VITE_** 开头的变量才会暴露给客户端，且需重启 dev。 |

### 6.1 前后端分离项目中的 Vue 角色

- **接口**：axios 在 onMounted 或事件中调用，数据存 ref/reactive，错误用拦截器统一提示。
- **鉴权**：登录后把 token 存 localStorage 或 Pinia；**路由 beforeEach** 里判断 to.meta.requiresAuth，无 token 跳登录页；**axios 拦截器** 请求头带 Authorization，401 时清 token 并跳登录。
- **路由守卫**：见第 4 章；可配合角色/权限做细粒度控制。

------

### 6.2 性能优化

| 手段 | 原理 / 说明 | 注意 |
|------|-------------|------|
| **路由懒加载** | 路由 component 用 `() => import()`，打包成独立 chunk，首屏只加载当前路由 | 避免首屏 bundle 过大 |
| **组件懒加载** | 大组件用 **defineAsyncComponent** 或 v-if 按需挂载，减少首屏 JS | 弹窗、抽屉等可懒加载 |
| **虚拟列表** | 长列表只渲染可视区（如 vue-virtual-scroller），滚动时复用 DOM | 列表项高度固定或可测时效果好 |
| **按需引入** | Element Plus 等用 **自动按需** 或手动 import 组件与样式，避免全量 | 可配合 unplugin-vue-components |
| **computed 缓存** | 派生数据用 computed，依赖不变不重算 | 避免在模板里写复杂表达式或重复计算 |
| **v-for key 稳定** | key 用唯一 id，便于 diff 复用节点 | 不用 index，避免增删导致错位与多余更新 |

### 6.3 常见坑与调试

| 现象 | 原因 | 处理 |
|------|------|------|
| **响应式「丢失」** | reactive 整体赋值、解构 Pinia 未用 storeToRefs | 改属性或 ref 包对象；storeToRefs |
| **列表更新错乱** | v-for 用 index 当 key | key 用唯一 id |
| **DevTools** | 不显示组件树/状态 | 安装 Vue DevTools 扩展，确认运行环境为开发模式 |

### 6.4 面试要点：响应式原理、Composition vs Options、虚拟 DOM

#### （1）响应式原理（Vue 3）

- **Vue 3** 用 **Proxy** 劫持对象：读属性时**收集依赖**（当前运行的 effect），写属性时**触发依赖**（重新执行 effect）。**ref** 内部用 reactive 包一层或单独用 getter/setter。

**依赖收集与触发流程（面试可答）**：

1. **初始化**：组件 setup 执行时，访问响应式数据（如 ref.value、reactive 属性）会触发 **get**。
2. **收集依赖**：get 里把当前**副作用（effect）** 加入该属性的依赖集合（Dep）；computed、watch、模板渲染都会创建 effect。
3. **触发更新**：修改响应式数据时触发 **set**，遍历依赖集合，**重新执行** 各 effect（如重新渲染、执行 watch 回调）。
4. **ref**：基本类型无法被 Proxy，用 **{ value }** 对象包装，对 .value 的 get/set 做依赖收集与触发。

**流程简图（面试可画）**：

```
   [组件 setup / 渲染]
          │
          ▼
   读取响应式数据 ──► get 被调用 ──► 当前 effect 加入 Dep（依赖收集）
          │
   修改响应式数据 ──► set 被调用 ──► 遍历 Dep 执行各 effect（触发更新）
          │
          ▼
   [重新渲染 / watch 回调 / computed 重算]
```

- **面试怎么说**：Vue 3 用 Proxy 做响应式，get 时收集依赖、set 时触发更新；ref 对基本类型用对象包装，.value 触发 get/set。

#### （2）Composition API 与 Options API

- **Options**：data、methods、computed 按选项组织，逻辑分散。**Composition**：按「逻辑」组织，ref、computed、watch 等可抽成 **composable** 复用；同一逻辑的代码聚在一起，适合复杂组件。
- **面试怎么说**：Composition API 按逻辑组织、易复用（composable）；Options 按选项组织、上手简单。Vue 3 两种都支持。

#### （3）虚拟 DOM 与 diff

- **虚拟 DOM**：用 JS 对象描述 DOM 树，数据变时生成新树，**diff** 新旧树得到最小变更再 **patch** 到真实 DOM。**为什么**：跨平台、集中批改减少重排、声明式写法。

**diff 大致过程（面试可答）**：

1. **同层比较**：只比较同一层级的节点，不跨层对比，复杂度 O(n)；若节点类型不同则直接替换整棵子树。
2. **key 的作用**：列表节点用 **key** 标识身份，diff 时按 key 匹配，可复用节点、调整顺序，避免「就地复用」导致状态错乱（如 input 光标、勾选状态）。
3. **双端比较**：Vue 2 采用双端比较（头头、尾尾、头尾、尾头），减少移动次数；Vue 3 用 **最长递增子序列** 优化移动。
4. **patch**：diff 得到 **补丁**（增、删、改、移），再对真实 DOM 做最小化更新。

- **面试怎么说**：用 JS 对象表示 DOM，更新时 diff 算出最小变更再 patch；同层比较、key 复用、双端或递增子序列优化移动。
