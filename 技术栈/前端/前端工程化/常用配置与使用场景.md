# 📌 前端工程化常用配置与使用场景速查

> 配合《学习笔记》使用，侧重 **npm/pnpm、Vite、ESLint/Prettier** 的常用命令与配置。

---

## 一、包管理常用命令（npm / pnpm）

| 命令 | 说明 |
|------|------|
| **npm init -y** / **pnpm init** | 初始化 package.json |
| **npm install** / **pnpm i** | 安装依赖 |
| **npm install xxx -D** / **pnpm add -D xxx** | 安装为 dev 依赖 |
| **npm run &lt;script&gt;** / **pnpm &lt;script&gt;** | 执行 package.json 中 scripts |
| **npm run build** / **pnpm build** | 通常为构建命令 |
| **npm run dev** / **pnpm dev** | 通常为开发服务器 |
| **npx xxx** / **pnpm dlx xxx** | 临时执行包（如 create-vite） |

---

## 二、Vite 常用配置（vite.config.ts）

| 配置项 | 说明 | 示例 |
|--------|------|------|
| **resolve.alias** | 路径别名 | alias: { '@': path.resolve(__dirname, 'src') } |
| **server.proxy** | 开发代理 | proxy: { '/api': { target: 'http://localhost:8080', changeOrigin: true } } |
| **define** | 全局常量 | define: { __APP_VERSION__: JSON.stringify('1.0') } |
| **build.outDir** | 输出目录 | outDir: 'dist' |
| **build.rollupOptions.output** | 分包、文件名 | manualChunks、chunkFileNames |
| **envDir** | 环境变量文件目录 | envDir: '.'（.env 所在目录） |

---

## 三、环境变量（Vite）

| 文件 | 说明 |
|------|------|
| **.env** | 所有环境 |
| **.env.local** | 本地覆盖，不提交 |
| **.env.development** | 开发环境 |
| **.env.production** | 生产环境 |

- 使用：**import.meta.env.VITE_XXX**（仅 VITE_ 前缀会暴露给客户端）。
- 类型：在 `env.d.ts` 中声明 `Interface ImportMetaEnv { VITE_XXX: string }`。

---

## 四、ESLint / Prettier 要点

| 工具 | 常用 |
|------|------|
| **ESLint** | .eslintrc.cjs 或 eslint.config.js；extends: vue/recommended、@vue/ts；rules 覆盖 |
| **Prettier** | .prettierrc（semi、singleQuote、tabWidth）；与 ESLint 冲突时用 eslint-config-prettier |
| **Husky** | pre-commit 钩子执行 lint-staged；lint-staged 对暂存文件跑 eslint、prettier |

---

## 五、使用场景对照

| 场景 | 推荐做法 |
|------|----------|
| 新建 Vue 项目 | pnpm create vue 或 pnpm create vite@latest -- --template vue |
| 接口代理到后端 | vite.config server.proxy 把 /api 指向后端地址 |
| 多环境接口地址 | .env.development / .env.production 中 VITE_API_BASE，代码用 import.meta.env.VITE_API_BASE |
| 提交前检查 | Husky pre-commit + lint-staged（eslint、prettier） |
| 构建与部署 | pnpm build → 将 dist 部署到 Nginx 或对象存储；SPA 配置 try_files 或 history 路由 fallback |
