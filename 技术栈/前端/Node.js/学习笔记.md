## 第1章：Node.js 概述与环境

### 1.1 Node.js 简介

**Node.js 是什么**：
- Node.js 是基于 Chrome V8 引擎的 JavaScript 运行时环境，允许在服务器端运行 JavaScript
- 由 Ryan Dahl 于 2009 年创建，旨在提供高效的服务器端 JavaScript 运行环境

**核心特点**：
- **非阻塞 I/O**：通过事件循环处理 I/O 操作，避免阻塞主线程
- **事件驱动**：基于事件和回调的编程模型
- **单线程模型**：主线程单线程，通过事件循环处理并发
- **跨平台**：可在 Windows、macOS、Linux 等系统上运行
- **丰富的模块系统**：内置核心模块和第三方模块生态

**应用场景**：
- **后端 API 开发**：构建 RESTful API 和 GraphQL API
- **实时通信**：WebSocket 服务、即时通讯应用
- **微服务架构**：轻量级服务节点
- **命令行工具**：开发各类 CLI 工具
- **构建工具**：Webpack、Gulp 等前端构建工具
- **桌面应用**：通过 Electron 构建跨平台桌面应用
- **IoT 设备**：嵌入式设备的服务端逻辑

**与浏览器的区别**：
| 特性 | Node.js | 浏览器 |
|------|---------|--------|
| 运行环境 | 服务器端 | 客户端 |
| 全局对象 | global | window |
| API 支持 | 核心模块（fs、http 等） | DOM、BOM |
| 模块系统 | CommonJS、ES Module | ES Module |
| 执行模型 | 非阻塞 I/O、事件驱动 | 单线程、事件循环 |

### 1.2 环境搭建与配置

#### 1.2.1 安装 Node.js

**Windows 系统**：
1. 访问 [Node.js 官网](https://nodejs.org/) 下载安装包
2. 运行安装程序，按照向导完成安装
3. 验证安装：`node -v` 和 `npm -v`

**macOS 系统**：
1. 方法一：官网下载安装包
2. 方法二：使用 Homebrew：`brew install node`
3. 验证安装：`node -v` 和 `npm -v`

**Linux 系统**：
1. 使用包管理器：
   - Ubuntu/Debian：`sudo apt install nodejs npm`
   - CentOS/RHEL：`sudo yum install nodejs npm`
2. 或使用 NodeSource 源安装最新版本
3. 验证安装：`node -v` 和 `npm -v`

#### 1.2.2 版本管理

**使用 nvm（Node Version Manager）**：
- **安装 nvm**：
  - Windows：下载 nvm-windows 安装包
  - macOS/Linux：`curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.1/install.sh | bash`

- **常用命令**：
  ```bash
  # 查看可用版本
  nvm ls-remote
  
  # 安装指定版本
  nvm install 16.14.0
  
  # 切换版本
  nvm use 16.14.0
  
  # 查看当前版本
  nvm current
  
  # 设置默认版本
  nvm alias default 16.14.0
  ```

### 1.3 模块系统

#### 1.3.1 CommonJS 模块

**基本用法**：
- **导出**：
  ```javascript
  // utils.js
  const add = (a, b) => a + b;
  const subtract = (a, b) => a - b;
  
  module.exports = {
    add,
    subtract
  };
  
  // 或单个导出
  module.exports = add;
  ```

- **导入**：
  ```javascript
  // app.js
  const utils = require('./utils');
  console.log(utils.add(1, 2)); // 3
  
  // 或解构导入
  const { add, subtract } = require('./utils');
  ```

**模块加载机制**：
1. 缓存机制：模块只加载一次，后续使用缓存
2. 加载顺序：核心模块 → 本地模块 → 第三方模块
3. 路径解析：
   - 绝对路径：直接加载
   - 相对路径：相对于当前文件
   - 模块名：从 node_modules 查找

#### 1.3.2 ES Module

**启用方式**：
1. 在 package.json 中设置：`"type": "module"`
2. 使用 .mjs 扩展名

**基本用法**：
- **导出**：
  ```javascript
  // utils.mjs
  export const add = (a, b) => a + b;
  export const subtract = (a, b) => a - b;
  
  // 默认导出
  export default add;
  ```

- **导入**：
  ```javascript
  // app.mjs
  import { add, subtract } from './utils.mjs';
  console.log(add(1, 2)); // 3
  
  // 默认导入
  import add from './utils.mjs';
  ```

**CommonJS 与 ES Module 的区别**：
| 特性 | CommonJS | ES Module |
|------|----------|-----------|
| 加载方式 | 运行时同步加载 | 编译时静态分析 |
| 导出方式 | module.exports | export / export default |
| 导入方式 | require() | import |
| 顶层 this | module.exports | undefined |
| 循环依赖 | 部分加载 | 实时绑定 |

### 1.4 实践示例

**创建第一个 Node.js 应用**：
1. **初始化项目**：
   ```bash
   mkdir my-node-app
   cd my-node-app
   npm init -y
   ```

2. **创建 app.js**：
   ```javascript
   const http = require('http');
   const port = process.env.PORT || 3000;
   
   const server = http.createServer((req, res) => {
     res.statusCode = 200;
     res.setHeader('Content-Type', 'text/plain');
     res.end('Hello Node.js!\n');
   });
   
   server.listen(port, () => {
     console.log(`Server running at http://localhost:${port}/`);
   });
   ```

3. **运行应用**：
   ```bash
   node app.js
   ```

4. **访问**：浏览器打开 http://localhost:3000

**使用 Express 创建 API**：
1. **安装 Express**：`npm install express`
2. **创建 api.js**：
   ```javascript
   const express = require('express');
   const app = express();
   const port = 3000;
   
   // 解析 JSON 请求体
   app.use(express.json());
   
   // 模拟数据
   let users = [
     { id: 1, name: 'Alice' },
     { id: 2, name: 'Bob' }
   ];
   
   // 获取用户列表
   app.get('/api/users', (req, res) => {
     res.json(users);
   });
   
   // 获取单个用户
   app.get('/api/users/:id', (req, res) => {
     const user = users.find(u => u.id === parseInt(req.params.id));
     if (!user) return res.status(404).json({ message: 'User not found' });
     res.json(user);
   });
   
   // 创建用户
   app.post('/api/users', (req, res) => {
     const newUser = {
       id: users.length + 1,
       name: req.body.name
     };
     users.push(newUser);
     res.status(201).json(newUser);
   });
   
   app.listen(port, () => {
     console.log(`API server running at http://localhost:${port}`);
   });
   ```

3. **运行 API**：`node api.js`

4. **测试 API**：使用 Postman 或 curl 测试端点

### 1.5 常见问题与解决方案

**问题 1：权限错误**
- **症状**：安装全局包时出现权限错误
- **解决方案**：
  - 使用 sudo（Linux/macOS）：`sudo npm install -g package`
  - 或修改 npm 全局目录权限
  - 或使用 nvm 管理 Node.js 版本

**问题 2：版本冲突**
- **症状**：不同项目需要不同 Node.js 版本
- **解决方案**：使用 nvm 管理多个版本

**问题 3：依赖安装失败**
- **症状**：npm install 失败
- **解决方案**：
  - 清除缓存：`npm cache clean --force`
  - 删除 node_modules 和 package-lock.json，重新安装
  - 检查网络连接

**问题 4：环境变量未生效**
- **症状**：process.env 中没有配置的环境变量
- **解决方案**：
  - 确保 dotenv 包已安装
  - 确保 .env 文件在正确位置
  - 重启应用

## 第2章：核心 API 与文件系统

### 2.1 文件系统（fs）

#### 2.1.1 同步与异步操作

**同步操作**：
- 阻塞主线程，适用于初始化操作
- 方法名通常以 `Sync` 结尾

**示例**：
```javascript
const fs = require('fs');

// 同步读取文件
try {
  const data = fs.readFileSync('file.txt', 'utf8');
  console.log(data);
} catch (err) {
  console.error(err);
}

// 同步写入文件
try {
  fs.writeFileSync('output.txt', 'Hello World!');
  console.log('File written successfully');
} catch (err) {
  console.error(err);
}
```

**异步操作**：
- 非阻塞，通过回调函数处理结果
- 推荐在生产环境中使用

**示例**：
```javascript
const fs = require('fs');

// 异步读取文件
fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log(data);
});

// 异步写入文件
fs.writeFile('output.txt', 'Hello World!', (err) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log('File written successfully');
});
```

**Promise  API**：
- Node.js 10+ 支持 Promise 风格的 API

**示例**：
```javascript
const fs = require('fs').promises;

async function fileOperations() {
  try {
    // 读取文件
    const data = await fs.readFile('file.txt', 'utf8');
    console.log(data);
    
    // 写入文件
    await fs.writeFile('output.txt', 'Hello World!');
    console.log('File written successfully');
  } catch (err) {
    console.error(err);
  }
}

fileOperations();
```

#### 2.1.2 文件操作

**读取文件**：
- `fs.readFile(path, options, callback)`
- `fs.readFileSync(path, options)`
- `fs.promises.readFile(path, options)`

**写入文件**：
- `fs.writeFile(file, data, options, callback)`
- `fs.writeFileSync(file, data, options)`
- `fs.promises.writeFile(file, data, options)`

**追加文件**：
- `fs.appendFile(path, data, options, callback)`
- `fs.appendFileSync(path, data, options)`
- `fs.promises.appendFile(path, data, options)`

**删除文件**：
- `fs.unlink(path, callback)`
- `fs.unlinkSync(path)`
- `fs.promises.unlink(path)`

**重命名文件**：
- `fs.rename(oldPath, newPath, callback)`
- `fs.renameSync(oldPath, newPath)`
- `fs.promises.rename(oldPath, newPath)`

**复制文件**：
- `fs.copyFile(src, dest, flags, callback)`
- `fs.copyFileSync(src, dest, flags)`
- `fs.promises.copyFile(src, dest, flags)`

**示例**：
```javascript
const fs = require('fs').promises;

async function fileOperations() {
  try {
    // 写入文件
    await fs.writeFile('example.txt', 'Initial content');
    console.log('File created');
    
    // 追加内容
    await fs.appendFile('example.txt', '\nAppended content');
    console.log('Content appended');
    
    // 读取文件
    const content = await fs.readFile('example.txt', 'utf8');
    console.log('File content:', content);
    
    // 复制文件
    await fs.copyFile('example.txt', 'example-copy.txt');
    console.log('File copied');
    
    // 重命名文件
    await fs.rename('example-copy.txt', 'copy.txt');
    console.log('File renamed');
    
    // 删除文件
    await fs.unlink('copy.txt');
    console.log('File deleted');
  } catch (err) {
    console.error(err);
  }
}

fileOperations();
```

#### 2.1.3 目录操作

**创建目录**：
- `fs.mkdir(path, options, callback)`
- `fs.mkdirSync(path, options)`
- `fs.promises.mkdir(path, options)`

**读取目录**：
- `fs.readdir(path, options, callback)`
- `fs.readdirSync(path, options)`
- `fs.promises.readdir(path, options)`

**删除目录**：
- `fs.rmdir(path, options, callback)`
- `fs.rmdirSync(path, options)`
- `fs.promises.rmdir(path, options)`

**示例**：
```javascript
const fs = require('fs').promises;

async function directoryOperations() {
  try {
    // 创建目录
    await fs.mkdir('new-directory');
    console.log('Directory created');
    
    // 在目录中创建文件
    await fs.writeFile('new-directory/file.txt', 'Hello from directory');
    console.log('File created in directory');
    
    // 读取目录内容
    const files = await fs.readdir('new-directory');
    console.log('Directory contents:', files);
    
    // 删除目录中的文件
    await fs.unlink('new-directory/file.txt');
    console.log('File deleted from directory');
    
    // 删除目录
    await fs.rmdir('new-directory');
    console.log('Directory deleted');
  } catch (err) {
    console.error(err);
  }
}

directoryOperations();
```

#### 2.1.4 文件状态

**获取文件状态**：
- `fs.stat(path, callback)`
- `fs.statSync(path)`
- `fs.promises.stat(path)`

**检查文件是否存在**：
- `fs.existsSync(path)`
- 注意：fs.exists() 已废弃，推荐使用 fs.stat() 或 try/catch

**示例**：
```javascript
const fs = require('fs').promises;

async function checkFileStatus() {
  try {
    // 检查文件状态
    const stats = await fs.stat('file.txt');
    console.log('File exists');
    console.log('Is file:', stats.isFile());
    console.log('Is directory:', stats.isDirectory());
    console.log('File size:', stats.size, 'bytes');
    console.log('Creation time:', stats.birthtime);
    console.log('Modification time:', stats.mtime);
  } catch (err) {
    if (err.code === 'ENOENT') {
      console.log('File does not exist');
    } else {
      console.error(err);
    }
  }
}

checkFileStatus();
```

#### 2.1.5 流操作

**流的类型**：
- **Readable**：可读流
- **Writable**：可写流
- **Duplex**：双向流
- **Transform**：转换流

**可读流示例**：
```javascript
const fs = require('fs');

const readStream = fs.createReadStream('large-file.txt', {
  encoding: 'utf8',
  highWaterMark: 1024 // 1KB 缓冲区
});

let data = '';
readStream.on('data', (chunk) => {
  console.log(`Received ${chunk.length} bytes`);
  data += chunk;
});

readStream.on('end', () => {
  console.log('End of file');
  console.log('Total data length:', data.length);
});

readStream.on('error', (err) => {
  console.error('Error reading file:', err);
});
```

**可写流示例**：
```javascript
const fs = require('fs');

const writeStream = fs.createWriteStream('output.txt');

writeStream.write('Hello ');
writeStream.write('Node.js ');
writeStream.write('Stream!');
writeStream.end();

writeStream.on('finish', () => {
  console.log('File written successfully');
});

writeStream.on('error', (err) => {
  console.error('Error writing file:', err);
});
```

**管道操作**：
```javascript
const fs = require('fs');

const readStream = fs.createReadStream('input.txt');
const writeStream = fs.createWriteStream('output.txt');

readStream.pipe(writeStream);

writeStream.on('finish', () => {
  console.log('File copied successfully');
});

// 链式管道
const zlib = require('zlib');
const gzip = zlib.createGzip();

fs.createReadStream('input.txt')
  .pipe(gzip)
  .pipe(fs.createWriteStream('input.txt.gz'))
  .on('finish', () => {
    console.log('File compressed successfully');
  });
```

### 2.2 路径处理（path）

#### 2.2.1 路径解析

**path.resolve()**：
- 将相对路径解析为绝对路径
- 从右到左处理路径片段

**示例**：
```javascript
const path = require('path');

console.log(path.resolve('.')); // 当前目录的绝对路径
console.log(path.resolve('..')); // 上级目录的绝对路径
console.log(path.resolve('./src', 'index.js')); // 解析相对路径
console.log(path.resolve('/usr', 'local', 'bin')); // 解析绝对路径
```

**path.join()**：
- 连接路径片段，返回规范化的路径
- 不会解析绝对路径

**示例**：
```javascript
const path = require('path');

console.log(path.join('src', 'lib', 'utils.js')); // src/lib/utils.js
console.log(path.join('src', '..', 'lib', 'utils.js')); // lib/utils.js
console.log(path.join('/', 'usr', 'local', 'bin')); // /usr/local/bin
```

#### 2.2.2 路径获取

**path.dirname()**：获取目录名
**path.basename()**：获取文件名
**path.extname()**：获取文件扩展名

**示例**：
```javascript
const path = require('path');
const filePath = '/usr/local/bin/node';

console.log(path.dirname(filePath)); // /usr/local/bin
console.log(path.basename(filePath)); // node
console.log(path.extname(filePath)); // ''

const fileName = '/home/user/documents/report.pdf';
console.log(path.basename(fileName)); // report.pdf
console.log(path.basename(fileName, path.extname(fileName))); // report
console.log(path.extname(fileName)); // .pdf
```

#### 2.2.3 路径规范化

**path.normalize()**：
- 规范化路径，处理 `.`、`..` 和多余的斜杠

**示例**：
```javascript
const path = require('path');

console.log(path.normalize('/usr//local/bin')); // /usr/local/bin
console.log(path.normalize('/usr/../local/bin')); // /local/bin
console.log(path.normalize('src/./lib/../utils.js')); // src/utils.js
```

#### 2.2.4 路径比较

**path.relative()**：
- 返回从 from 到 to 的相对路径

**示例**：
```javascript
const path = require('path');

console.log(path.relative('/usr/local/bin', '/usr/local/lib')); // ../lib
console.log(path.relative('/usr/local', '/usr/local/bin/node')); // bin/node
```

**path.isAbsolute()**：
- 检查路径是否为绝对路径

**示例**：
```javascript
const path = require('path');

console.log(path.isAbsolute('/usr/local')); // true
console.log(path.isAbsolute('./src')); // false
console.log(path.isAbsolute('src/lib')); // false
```

#### 2.2.5 路径格式化与解析

**path.format()**：
- 根据对象创建路径字符串

**示例**：
```javascript
const path = require('path');

const pathObj = {
  dir: '/usr/local',
  base: 'node',
```

### 2.6 常见问题与解决方案

**问题 1：文件路径错误**
- **症状**：`ENOENT` 错误
- **解决方案**：
  - 使用 `path.join()` 或 `path.resolve()` 构建路径
  - 检查文件/目录是否存在
  - 使用绝对路径

**问题 2：文件权限错误**
- **症状**：`EACCES` 错误
- **解决方案**：
  - 检查文件权限
  - 以正确的用户权限运行应用
  - 使用 `chmod` 命令修改权限

**问题 3：大文件处理**
- **症状**：内存不足错误
- **解决方案**：
  - 使用流（Stream）处理大文件
  - 避免一次性读取整个文件到内存
  - 分块处理文件

**问题 4：HTTP 服务器崩溃**
- **症状**：未处理的错误导致服务器崩溃
- **解决方案**：
  - 添加错误处理中间件
  - 监听 `uncaughtException` 事件
  - 使用进程管理器（如 PM2）自动重启

### 3.4 中间件系统

#### 3.4.1 中间件概念

**什么是中间件**：
- 中间件是处理 HTTP 请求的函数
- 可以访问请求对象（req）、响应对象（res）和 next 函数
- 可以执行以下操作：
  - 执行代码
  - 修改请求和响应对象
  - 结束请求-响应周期
  - 调用下一个中间件

**中间件类型**：
- **应用级中间件**：绑定到 app 对象
- **路由级中间件**：绑定到 router 对象
- **错误处理中间件**：处理错误
- **内置中间件**：Express 内置的中间件
- **第三方中间件**：第三方提供的中间件

## 第4章：Node.js 异步编程与高级特性

### 4.1 异步编程模型

#### 4.1.1 回调函数（Callback）

**什么是回调函数**：
- 回调函数是作为参数传递给另一个函数的函数
- 在 Node.js 早期，回调是处理异步操作的主要方式
- 遵循错误优先的约定：`callback(err, result)`

**基本用法**：
```javascript
const fs = require('fs');

// 读取文件 - 回调方式
fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) {
    console.error('Error reading file:', err);
    return;
  }
  console.log('File content:', data);
});

// 写入文件 - 回调方式
fs.writeFile('output.txt', 'Hello World', (err) => {
  if (err) {
    console.error('Error writing file:', err);
    return;
  }
  console.log('File written successfully');
});
```

**回调地狱（Callback Hell）**：
```javascript
// 多层嵌套的回调 - 难以维护
fs.readFile('file1.txt', 'utf8', (err1, data1) => {
  if (err1) {
    console.error('Error reading file1:', err1);
    return;
  }
  
  fs.readFile('file2.txt', 'utf8', (err2, data2) => {
    if (err2) {
      console.error('Error reading file2:', err2);
      return;
    }
    
    fs.writeFile('output.txt', data1 + data2, (err3) => {
      if (err3) {
        console.error('Error writing file:', err3);
        return;
      }
      
      console.log('Files processed successfully');
    });
  });
});
```

**解决方案**：
- 使用命名函数
- 使用 Promise
- 使用 async/await

#### 4.1.2 Promise

**什么是 Promise**：
- Promise 是异步编程的一种解决方案
- 代表一个尚未完成但预期将来会完成的操作
- 有三种状态：pending（进行中）、fulfilled（已成功）、rejected（已失败）

**基本用法**：
```javascript
const fs = require('fs').promises;

// 创建 Promise
const promise = new Promise((resolve, reject) => {
  // 异步操作
  setTimeout(() => {
    const success = true;
    if (success) {
      resolve('Operation successful');
    } else {
      reject(new Error('Operation failed'));
    }
  }, 1000);
});

// 使用 Promise
promise
  .then(result => {
    console.log(result);
  })
  .catch(error => {
    console.error(error);
  })
  .finally(() => {
    console.log('Operation completed');
  });
```

**Promise 链式调用**：
```javascript
const fs = require('fs').promises;

// 链式调用解决回调地狱
fs.readFile('file1.txt', 'utf8')
  .then(data1 => {
    console.log('File1 content:', data1);
    return fs.readFile('file2.txt', 'utf8');
  })
  .then(data2 => {
    console.log('File2 content:', data2);
    return fs.writeFile('output.txt', data1 + data2);
  })
  .then(() => {
    console.log('Files processed successfully');
  })
  .catch(err => {
    console.error('Error:', err);
  });
```

**Promise.all()**：

### 4.2 事件循环（Event Loop）

#### 4.2.1 事件循环概述

**什么是事件循环**：
- Node.js 的核心机制，用于处理异步操作
- 允许 Node.js 执行非阻塞 I/O 操作
- 通过将操作委托给系统内核，实现高并发

**事件循环的工作原理**：
1. 执行同步代码
2. 检查是否有微任务（microtasks），执行所有微任务
3. 进入事件循环，执行宏任务（macrotasks）
4. 检查是否有微任务，执行所有微任务
5. 重复步骤 3-4

#### 4.2.2 事件循环阶段

Node.js 事件循环分为六个阶段：

| 阶段 | 描述 |
|------|------|
| **timers** | 执行 setTimeout 和 setInterval 的回调 |
| **pending callbacks** | 执行 I/O 回调（如 TCP 错误） |
| **idle, prepare** | 内部使用 |
| **poll** | 获取新的 I/O 事件，执行 I/O 回调 |
| **check** | 执行 setImmediate 的回调 |
| **close callbacks** | 执行关闭事件的回调（如 socket.on('close')） |

**执行顺序示例**：
```javascript
console.log('1. Sync code start');

setTimeout(() => {
  console.log('2. setTimeout');
}, 0);

setImmediate(() => {
  console.log('3. setImmediate');
});

process.nextTick(() => {
  console.log('4. process.nextTick');
});

Promise.resolve().then(() => {
  console.log('5. Promise.then');
});

console.log('6. Sync code end');

// 输出顺序：
// 1. Sync code start
// 6. Sync code end
// 4. process.nextTick
// 5. Promise.then
// 2. setTimeout
// 3. setImmediate
```

#### 4.2.3 宏任务与微任务

**宏任务（Macrotasks）**：
- setTimeout
- setInterval
- setImmediate
- I/O 操作
- UI 渲染

**微任务（Microtasks）**：
- Promise.then
- Promise.catch
- Promise.finally
- process.nextTick
- MutationObserver

**执行优先级**：
```javascript
console.log('Start');

setTimeout(() => {
  console.log('setTimeout');
}, 0);

setImmediate(() => {
  console.log('setImmediate');
});

process.nextTick(() => {
  console.log('nextTick 1');
  process.nextTick(() => {
    console.log('nextTick 2');
  });
});

Promise.resolve().then(() => {
  console.log('Promise 1');
  Promise.resolve().then(() => {
    console.log('Promise 2');
  });
});

console.log('End');

// 输出：
// Start
// End
// nextTick 1
// nextTick 2
// Promise 1
// Promise 2
// setTimeout
// setImmediate
```

#### 4.2.4 process.nextTick()

**特点**：
- 在当前操作完成后、事件循环继续之前执行
- 优先级高于 Promise.then
- 可以递归调用，可能导致 I/O 饥饿

**使用场景**：
```javascript
// 确保在事件循环继续之前执行
function apiCall(arg, callback) {
  if (typeof arg !== 'string') {
    process.nextTick(() => {
      callback(new TypeError('Argument must be a string'));
    });
    return;
  }
  // 异步操作
}

// 递归使用（注意：可能导致 I/O 饥饿）
let count = 0;
function recursiveNextTick() {
  if (count < 5) {
    count++;
    console.log('Count:', count);
    process.nextTick(recursiveNextTick);
  }
}

recursiveNextTick();
console.log('This will print after all nextTick callbacks');
```

### 4.3 流（Stream）

#### 4.3.1 流的概念

**什么是流**：
- 流是处理数据的抽象接口
- 允许以连续的方式处理数据
- 特别适合处理大文件或网络数据

**流的类型**：

| 类型 | 描述 | 示例 |
|------|------|------|
| **Readable** | 可读流，用于读取数据 | fs.createReadStream() |
| **Writable** | 可写流，用于写入数据 | fs.createWriteStream() |
| **Duplex** | 双向流，既可读又可写 | net.Socket |
| **Transform** | 转换流，在读写过程中转换数据 | zlib.createGzip() |

#### 4.3.2 可读流（Readable）

**创建可读流**：
```javascript

  encoding: 'utf8',

});
```

**事件处理**：
```javascript
const fs = require('fs');

  console.log(`Received ${chunk.length} bytes`);
  console.log(chunk.toString());
});

});

  console.error('Error reading file:', err);
});

    console.log(`Read ${chunk.length} bytes`);
  }
});
```

**加密转换流**：
```javascript

  return new Transform({
    transform(chunk, encoding, callback) {
      const cipher = crypto.createCipher('aes-256-cbc', key);
      const encrypted = Buffer.concat([
        cipher.update(chunk),
        cipher.final()
      ]);
      this.push(encrypted);
      callback();
    }
  });
}
```

### 4.5 集群（Cluster）

#### 4.5.1 集群概述

**什么是集群**：
- 集群模块允许创建共享服务器端口的子进程
- 充分利用多核 CPU
- 主进程负责管理，工作进程处理请求

**工作原理**：
- 主进程（Master）创建多个工作进程（Worker）
- 工作进程共享同一个服务器端口
- 操作系统负责负载均衡

#### 4.5.2 基本集群示例

**创建集群**：
```javascript
const cluster = require('cluster');
const http = require('http');
const os = require('os');

if (cluster.isMaster) {
  console.log(`Master ${process.pid} is running`);
  
  // 获取 CPU 核心数
  const numCPUs = os.cpus().length;
  console.log(`Starting ${numCPUs} workers...`);
  
  // 创建工作进程
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }
  
  // 工作进程退出
  cluster.on('exit', (worker, code, signal) => {
    console.log(`Worker ${worker.process.pid} died`);
    console.log('Starting a new worker...');
    cluster.fork();
  });
  
  // 工作进程启动
  cluster.on('online', (worker) => {
    console.log(`Worker ${worker.process.pid} is online`);
  });
  
} else {
  // 工作进程创建 HTTP 服务器
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end(`Hello from worker ${process.pid}\n`);
  }).listen(8000);
  
  console.log(`Worker ${process.pid} started`);
}
```

#### 4.5.4 进程间通信

**主进程与工作进程通信**：
```javascript
const cluster = require('cluster');
const http = require('http');
const os = require('os');

if (cluster.isMaster) {
  console.log(`Master ${process.pid} is running`);
  
  const workers = [];
  
  // 创建工作进程
  for (let i = 0; i < os.cpus().length; i++) {
    const worker = cluster.fork();
    workers.push(worker);
    
    // 向工作进程发送消息
    worker.send({ type: 'INIT', workerId: i });
  }
  
  // 接收工作进程的消息
  cluster.on('message', (worker, message) => {
    console.log(`Master received from worker ${worker.process.pid}:`, message);
    
    // 广播给所有工作进程
    if (message.type === 'BROADCAST') {
      workers.forEach(w => {
        if (w !== worker) {
          w.send(message);
        }
      });
    }
  });
  
} else {
  // 接收主进程的消息
  process.on('message', (message) => {
    console.log(`Worker ${process.pid} received:`, message);
  });
  
  http.createServer((req, res) => {
    // 向主进程发送消息
    process.send({ type: 'REQUEST', url: req.url });
    
    res.writeHead(200);
    res.end(`Hello from worker ${process.pid}\n`);
  }).listen(8000);
}
```

class TaskQueue {
  constructor(concurrency) {
    this.concurrency = concurrency;
    this.running = 0;
    this.queue = [];
  }
  
  push(task) {
    this.queue.push(task);
    this.next();
  }
  
  next() {
    while (this.running < this.concurrency && this.queue.length) {
      const task = this.queue.shift();

          console.log('Task completed:', result);

        .catch(err => {

        })
        .finally(() => {
          this.running--;
          this.next();
        });
    }
  }
}

### 6.2 手写代码题

#### 异步编程
1. **实现 Promise**：
   ```javascript
   class MyPromise {
     constructor(executor) {
       this.state = 'pending';
       this.value = undefined;
       this.reason = undefined;
       this.onFulfilledCallbacks = [];
       this.onRejectedCallbacks = [];

       const resolve = (value) => {
         if (this.state === 'pending') {
           this.state = 'fulfilled';
           this.value = value;
           this.onFulfilledCallbacks.forEach(callback => callback(value));
         }
       };

       const reject = (reason) => {
         if (this.state === 'pending') {
           this.state = 'rejected';
           this.reason = reason;
           this.onRejectedCallbacks.forEach(callback => callback(reason));
         }
       };

       try {
         executor(resolve, reject);
       } catch (error) {
         reject(error);
       }
     }

     then(onFulfilled, onRejected) {
       onFulfilled = typeof onFulfilled === 'function' ? onFulfilled : value => value;
       onRejected = typeof onRejected === 'function' ? onRejected : reason => { throw reason };

       const promise2 = new MyPromise((resolve, reject) => {
         if (this.state === 'fulfilled') {
           setTimeout(() => {
             try {
               const x = onFulfilled(this.value);
               this.resolvePromise(promise2, x, resolve, reject);
             } catch (error) {
               reject(error);
             }
           }, 0);
         }

         if (this.state === 'rejected') {
           setTimeout(() => {
             try {
               const x = onRejected(this.reason);
               this.resolvePromise(promise2, x, resolve, reject);
             } catch (error) {
               reject(error);
             }
           }, 0);
         }

         if (this.state === 'pending') {
           this.onFulfilledCallbacks.push(() => {
             setTimeout(() => {
               try {
                 const x = onFulfilled(this.value);
                 this.resolvePromise(promise2, x, resolve, reject);
               } catch (error) {
                 reject(error);
               }
             }, 0);
           });

           this.onRejectedCallbacks.push(() => {
             setTimeout(() => {
               try {
                 const x = onRejected(this.reason);
                 this.resolvePromise(promise2, x, resolve, reject);
               } catch (error) {
                 reject(error);
               }
             }, 0);
           });
         }
       });

       return promise2;
     }

     resolvePromise(promise2, x, resolve, reject) {
       if (promise2 === x) {
         return reject(new TypeError('Chaining cycle detected for promise'));
       }

       if (x instanceof MyPromise) {
         x.then(resolve, reject);
       } else if (x !== null && (typeof x === 'object' || typeof x === 'function')) {
         let called = false;
         try {
           const then = x.then;
           if (typeof then === 'function') {
             then.call(x, (y) => {
               if (called) return;
               called = true;
               this.resolvePromise(promise2, y, resolve, reject);
             }, (r) => {
               if (called) return;
               called = true;
               reject(r);
             });
           } else {
             resolve(x);
           }
         } catch (error) {
           if (called) return;
           called = true;
           reject(error);
         }
       } else {
         resolve(x);
       }
     }

     catch(onRejected) {
       return this.then(null, onRejected);
     }

     static resolve(value) {
       return new MyPromise(resolve => resolve(value));
     }

     static reject(reason) {
       return new MyPromise((resolve, reject) => reject(reason));
     }

     static all(promises) {
       return new MyPromise((resolve, reject) => {
         const results = [];
         let count = 0;

         for (let i = 0; i < promises.length; i++) {
           MyPromise.resolve(promises[i]).then(
             value => {
               results[i] = value;
               count++;
   ```

   ```javascript
             generatorResult = generator[key](arg);
   ```

   ```javascript
             if (count === promises.length) {
   ```

   ```javascript
   class SimpleExpress {
     constructor() {
       this.routes = {};
       this.middlewares = [];
     }

     use(middleware) {
       this.middlewares.push(middleware);
     }

     get(path, handler) {
       this.routes['GET'] = this.routes['GET'] || {};
       this.routes['GET'][path] = handler;
     }

     post(path, handler) {
       this.routes['POST'] = this.routes['POST'] || {};
       this.routes['POST'][path] = handler;
     }

     handleRequest(req, res) {
       let index = 0;
       const next = () => {
         if (index < this.middlewares.length) {
           this.middlewares[index++](req, res, next);
         } else {
           const method = req.method;
           const path = req.url;
           if (this.routes[method] && this.routes[method][path]) {
             this.routes[method][path](req, res);
           } else {
             res.statusCode = 404;
             res.end('Not Found');
           }
         }
       };
       next();
     }

     listen(port, callback) {
       const server = require('http').createServer((req, res) => {
         this.handleRequest(req, res);
       });
       server.listen(port, callback);
     }
   }

   // 使用示例
   const app = new SimpleExpress();

   // 中间件
   app.use((req, res, next) => {
     console.log(`${new Date().toISOString()} - ${req.method} ${req.url}`);
     next();
   });

   // 路由
   app.get('/', (req, res) => {
     res.end('Hello World!');
   });

   app.post('/api', (req, res) => {
     res.end('POST request received');
   });

   app.listen(3000, () => {
     console.log('Server running on port 3000');
   });
   ```

   ```javascript
   class EventEmitter {
     constructor() {
       this.events = {};
     }

     on(event, listener) {
       if (!this.events[event]) {
         this.events[event] = [];
       }
       this.events[event].push(listener);
     }

     once(event, listener) {
       const onceListener = (...args) => {
         listener.apply(this, args);
         this.off(event, onceListener);
       };
       this.on(event, onceListener);
     }

     emit(event, ...args) {
       if (this.events[event]) {
         this.events[event].forEach(listener => {
           listener.apply(this, args);
         });
       }
     }

     off(event, listener) {
       if (this.events[event]) {
         this.events[event] = this.events[event].filter(l => l !== listener);
       }
     }

     removeAllListeners(event) {
       if (event) {
         delete this.events[event];
       } else {
         this.events = {};
       }
     }

     listenerCount(event) {
       return this.events[event] ? this.events[event].length : 0;
     }
   }

   // 使用示例
   const emitter = new EventEmitter();
   emitter.on('test', (message) => {
     console.log('Test event:', message);
   });
   emitter.emit('test', 'Hello World');
   ```
