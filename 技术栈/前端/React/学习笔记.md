### 1.1 React 简介

#### （1）定义与定位

**React** 是 Meta（Facebook）推出的**用于构建用户界面的 JavaScript 库**，采用组件化、声明式与「数据驱动视图」的思想，生态庞大，在 AI 应用、中后台、大厂前端中广泛使用。

| 项目 | 说明 |
|------|------|
| **核心能力** | 组件化、声明式 UI、虚拟 DOM 与 diff、单向数据流、Hooks、Learn Once Write Anywhere。 |
| **典型场景** | 中后台、大厂前端、AI 应用（Vercel AI SDK）、React Native 移动端、SSR。 |
| **与 Vue** | React 用 JSX、Hooks，生态与招聘多；Vue 模板友好、上手快。二者均能胜任复杂前端。 |

> 💬 **一句话**：React 解决**数据驱动视图、组件复用与大规模 UI 工程化**问题，通过声明式与组件化把 UI 与数据绑定在一起，配合 Hooks 与生态支撑复杂应用。

#### （2）传统开发痛点（为什么需要 React 或类似库）

| 痛点 | 说明 |
|------|------|
| **DOM 操作繁琐** | 原生 JS / jQuery 需手动选中、改 innerHTML、绑事件，逻辑与视图耦合，难以维护。 |
| **数据与视图不同步** | 数据变更需手动更新多处 DOM，易遗漏、重复代码多。 |
| **无组件边界** | 页面逻辑堆在一起，复用靠复制粘贴，协作与测试困难。 |
| **跨平台成本高** | Web 与移动端各写一套，逻辑难以复用。 |

- **React 的应对**：**声明式**——用 JSX 描述「数据下的 UI」，由 React 负责更新 DOM；**组件化**——界面由组件组合；**单向数据流**——数据自顶向下、变更通过 setState 触发重渲染；**React Native**——同一套思维写移动端。

------

### 1.2 核心特点

| 特点 | 说明 | 实际意义 |
|------|------|----------|
| **组件化** | 界面由组件组合，可复用与拆分 | 可维护、可测试、多人协作 |
| **声明式** | 描述「UI 在数据下的样子」，由 React 更新 DOM | 代码更清晰、意图更明确 |
| **单向数据流** | 数据自顶向下（props），变更通过 state 更新触发重渲染 | 数据流可预测、易调试 |
| **Learn Once, Write Anywhere** | 同一套思维可写 Web、React Native、SSR | 跨平台、生态统一 |

> 💡 **重点**：**React 18** 为主流，函数组件 + **Hooks** 为默认写法；**Concurrent**（并发渲染）、**自动批处理**（多次 setState 合并更新）等提升体验与性能。

### 2.1 创建项目与挂载

// main.jsx

### 2.2 JSX 语法

#### （1）表达式、条件与列表

- **表达式**：用 **{ }** 包裹 JS 表达式，如 `{count}`、`{user.name}`。**条件**：`{flag && <A />}` 或 `{flag ? <A /> : <B />}`。**列表**：`list.map(item => <li key={item.id}>{item.name}</li>)`，**key 必须稳定唯一**，不用 index。
- **className**：因 class 是保留字，用 **className**；可写 `className={`btn ${active ? 'active' : ''}`}`。**style**：对象写法，驼峰，如 `style={{ marginTop: 8, width: 100 }}`。

```jsx
function App() {
  const [count, setCount] = useState(0)
  const list = [{ id: 1, name: 'a' }, { id: 2, name: 'b' }]
  return (
    <div>
      <p>{count}</p>
      {count > 0 && <span>正数</span>}
      <ul>
        {list.map(item => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
    </div>
  )
}
```

#### （2）事件绑定

- 事件名**驼峰**：**onClick**、**onChange**、**onSubmit**；传函数引用 `onClick={handleClick}` 或内联 `onClick={() => setCount(c => c + 1)}`。**阻止默认**：`e.preventDefault()`。

> 💡 **重点**：**key** 用于 diff 时识别节点身份，用**唯一 id**，不用 index（增删会导致错位与性能问题）；与 Vue 一致。

### 2.3 组件定义与 Props

- **函数组件**：接收 **props** 对象，返回 JSX。**Props 只读**，子组件不直接修改；通过**回调**通知父组件。**children**：组件标签内的内容会作为 **props.children** 传入。

```jsx
function Card({ title, children }) {
  return (
    <div className="card">
      <h2>{title}</h2>
      {children}
    </div>
  )
}
// 使用：<Card title="标题">内容</Card>
```

------

### 2.4 受控与非受控组件

| 类型 | 说明 | 写法 |
|------|------|------|
| **受控** | 表单值由 **state** 控制，**value + onChange** 绑定 | `value={val} onChange={e => setVal(e.target.value)}` |
| **非受控** | 表单值由 **DOM** 自身管理，用 **useRef** 取 DOM 拿值 | `ref={inputRef}`，提交时 `inputRef.current.value` |

**组件渲染与更新顺序（面试可答）**：

1. **首次挂载**：父 → 子 依次执行函数组件体（渲染）→ 生成虚拟 DOM → **commit** 到真实 DOM → **useEffect** 按声明顺序执行（在浏览器绘制之后）。
2. **更新**：state 变更 → 重新执行组件体（渲染）→ diff → patch 真实 DOM → 上一轮 effect 的**清理函数**执行 → 本轮 **useEffect** 执行。
3. **卸载**：组件从树中移除 → 所有 effect 的**清理函数**执行。

**受控示例**：

```jsx
const [text, setText] = useState('')
return (
  <input
    value={text}
    onChange={e => setText(e.target.value)}
  />
)
```

> 💬 **面试怎么说**：受控组件即表单 value 由 state 控制，通过 onChange 更新 state，再驱动视图；React 推荐受控，便于校验与统一状态。

------

### 2.5 常见错误写法 vs 正确写法

| 场景 | 错误写法 / 现象 | 正确写法 / 做法 |
|------|-----------------|-----------------|
| **列表 key 用 index** | 增删后错位、复用了错误节点 | **key 用唯一 id**：`key={item.id}`。 |
| **直接修改 props** | 控制台警告、父组件状态不同步 | props 只读；用**回调**让父组件改，或父传 state + setState。 |
| **class 写成 class** | JSX 中 class 是保留字 | 用 **className**。 |
| **style 写字符串** | style 需要对象 | `style={{ marginTop: 8 }}`，驼峰。 |
| **受控组件 value 未配 onChange** | 只读或警告 | 受控必须 **value + onChange** 成对；或改为非受控（defaultValue）。 |

### 3.1 useState

- **用途**：在函数组件中声明**状态**；返回 `[value, setValue]`。**更新**：传新值或**函数** `setN(n => n + 1)`（依赖前一次状态时用函数，避免闭包旧值）。**不可变更新**：对象/数组不要直接改，应**新对象/新数组**（展开或 map/filter）。

```jsx
const [count, setCount] = useState(0)
setCount(c => c + 1)

const [list, setList] = useState([])
setList(prev => [...prev, newItem])
```

> 💡 **重点**：**批量更新**：React 18 中多次 setState 会在一次渲染中合并（自动批处理）；依赖前一次状态时用 **setX(x => ...)**。

------

### 3.2 useEffect

- **用途**：在函数组件中执行**副作用**（请求、订阅、DOM）。**签名**：`useEffect(() => { ...; return () => {} }, [deps])`。**依赖数组**：空 `[]` 仅挂载时执行一次；不传则每次渲染都执行；传 `[a, b]` 则 a/b 变化时执行。**清理函数**：return 的函数在下次 effect 执行前或卸载时执行，用于取消订阅、清定时器。

```jsx
useEffect(() => {
  const id = setInterval(() => setCount(c => c + 1), 1000)
  return () => clearInterval(id)
}, [])
```

**useEffect 执行时机简图（面试可答）**：

```
  [渲染阶段：组件函数执行，得到新虚拟 DOM]
          │
          ▼
  [commit 阶段：把变更 patch 到真实 DOM，浏览器绘制]
          │
          ▼
  [上一轮 effect 的清理函数执行（若有）]
          │
          ▼
  [本轮 useEffect 按声明顺序执行]
```

> 💬 **面试怎么说**：useEffect 在**渲染、commit、绘制之后**执行；依赖数组控制何时执行；return 的清理函数在依赖变更或卸载时执行。

------

### 3.3 useContext

- **用途**：**跨层级**消费 Context，避免逐层 props。**步骤**：**createContext** → **Provider** 提供 value → 子组件 **useContext(MyContext)** 消费。

```jsx
const ThemeContext = createContext('light')
function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Child />
    </ThemeContext.Provider>
  )
}
function Child() {
  const theme = useContext(ThemeContext)
  return <div>{theme}</div>
}
```

------

### 3.4 useRef

- **用途**：① **DOM 引用**：`const ref = useRef(null)`，`<div ref={ref} />`，通过 `ref.current` 访问 DOM。② **持久化值**：存一个在渲染间**不变**、且**修改不触发重渲染**的值（如定时器 id、上一次的 props）。

```jsx
const inputRef = useRef(null)
inputRef.current?.focus()
```

------

### 3.5 useMemo 与 useCallback

| Hook | 用途 | 说明 |
|------|------|------|
| **useMemo** | 缓存**计算结果** | `const x = useMemo(() => compute(a), [a])`，a 不变则复用上次结果。 |
| **useCallback** | 缓存**函数引用** | `const fn = useCallback(() => {}, [deps])`，避免子组件因回调引用变而无效重渲染（配合 memo）。 |

------

### 3.6 自定义 Hooks

- **目的**：把**可复用的状态与副作用逻辑**抽成函数，函数内可调用其它 Hooks。命名以 **use** 开头。示例：**useFetch**（请求 data/loading/error）、**useLocalStorage**（与 localStorage 同步的 state）。

```jsx
function useCounter(initial = 0) {
  const [count, setCount] = useState(initial)
  const inc = useCallback(() => setCount(c => c + 1), [])
  return { count, inc }
}
```

------

### 3.7 Hooks 规则（面试必答）

1. **只在顶层调用**：不要在循环、条件、嵌套函数里调用 Hooks，保证每次渲染**调用顺序一致**（React 靠顺序对应 state）。
2. **只在 React 函数中调用**：在函数组件或自定义 Hooks 里调用，不在普通函数或类组件里调用。

### 3.8 常见错误写法 vs 正确写法

| 场景 | 错误写法 / 现象 | 正确写法 / 做法 |
|------|-----------------|-----------------|
| **直接改 state** | `list.push(item)` 后 setList(list) | **不可变更新**：`setList(prev => [...prev, item])`。 |
| **useEffect 依赖漏写** | 用了某变量但没放进 deps，导致闭包旧值 | 把 effect 内用到的**响应式值**都放进依赖数组；或用 ref 存最新值。 |
| **条件里调 Hook** | `if (x) useState(0)` 破坏顺序 | Hooks 只在**顶层**调用，条件写在 Hook **内部**。 |
| **依赖数组用对象/数组字面量** | 每次渲染新引用，effect 每次都跑 | 用**原始值**或 **useMemo** 稳定引用；或把依赖拆成具体属性。 |

------

## ✅ 本章小结

| 知识点 | 面试关键词 | 实际应用 |
|--------|------------|----------|
| **useState** | 不可变更新、函数式 setState、批处理 | 状态与列表/对象更新 |
| **useEffect** | 依赖数组、清理函数、挂载/更新/卸载 | 请求、订阅、定时器 |
| **useContext** | createContext、Provider、useContext | 主题、语言、跨层级 |
| **useRef** | DOM 引用、持久化值、不触发渲染 | 聚焦、定时器 id |
| **useMemo/useCallback** | 缓存计算/函数、配合 memo | 性能优化 |
| **Hooks 规则** | 顶层调用、仅 React 函数 | 面试必答 |

**学习要点**：useState 不可变更新；useEffect 看依赖与清理；Hooks 只在顶层、只在 React 函数中调用。下一章讲组件化与数据流（状态提升、Context 深入）。

| 面试官提问 | 回答思路 |
|------------|----------|
| Hooks 为什么不能放在条件里？ | React 靠**调用顺序**对应 state；条件会导致顺序变化，state 错位。 |
| useEffect 依赖数组不写会怎样？ | 不写则**每次渲染后**都执行；空数组则**仅挂载时**执行一次。 |

| 现象 / 易错点 | 原因 | 怎么改 / 怎么记 |

| 在 effect 里拿到旧 state | 闭包捕获了旧渲染的值 | 依赖数组加上该 state；或 setState 用函数式更新。 |
| 清理函数忘写 | 订阅、定时器未取消，内存泄漏或重复执行 | useEffect 里 return 一个函数，在里面对应 clear/取消。 |

### 4.1 状态提升与单一数据源

- **状态提升**：多个组件需要**同一份数据**时，把 state 放到**共同父组件**，通过 props 下传、通过**回调**上传变更，保证**单一数据源**。
- **为什么**：避免多份数据不同步；父组件成为唯一真相来源。

```jsx
function Parent() {
  const [value, setValue] = useState('')
  return (
    <>
      <ChildA value={value} onChange={setValue} />
      <ChildB value={value} />
    </>
  )
}
```

------

### 4.2 Context API 深入

- **创建**：`createContext(defaultValue)`。**提供**：`<MyContext.Provider value={v}>`，value 变化会触发所有消费者重渲染。**消费**：`useContext(MyContext)` 或 `<MyContext.Consumer>`。
- **注意**：Provider 的 **value** 用**新引用**（如对象）时，即使内容没变也会导致消费者重渲染；可把 value 用 **useMemo** 或 **useState** 包一层，避免无意义重渲染。

------

### 4.3 组合 vs 继承

- React 推荐**组合**而非继承。
- **children**：组件用 `props.children` 接收子内容，适合做容器组件（如 Dialog、Card）。
- **render props**：通过 props 传函数，由子组件调用并传入数据，实现逻辑复用。

```jsx
// Render Props 示例
function DataProvider({ render }) {
  const [data] = useState({ name: 'React' })
  return render(data)
}
// 使用
<DataProvider render={data => <div>Hello {data.name}</div>} />
```

- **slot 式**：通过多个 props 传不同区域的 UI（React 没有官方 slot，直接传 JSX 即可）。

```jsx
function Layout({ header, sidebar, content }) {
  return (
    <div className="layout">
      <header>{header}</header>
      <aside>{sidebar}</aside>
      <main>{content}</main>
    </div>
  )
}
// 使用：<Layout header={<Header />} sidebar={<Sidebar />} content={<Main />} />
```

### 4.4 错误边界（Error Boundary）

- **作用**：捕获**子组件树**中的 JS 错误，记录错误并显示 fallback UI，避免整棵子树崩溃（React 16 引入）。
- **实现**：目前只有**类组件**能实现错误边界，需定义 `getDerivedStateFromError`（更新 state 显式降级 UI）或 `componentDidCatch`（记录错误日志）。

```jsx
class ErrorBoundary extends React.Component {
  state = { hasError: false };
  static getDerivedStateFromError(error) {
    return { hasError: true }; // 下次渲染显示 fallback
  }
  componentDidCatch(error, errorInfo) {
    console.error("捕获到错误:", error, errorInfo);
  }
  render() {
    if (this.state.hasError) return <h1>Something went wrong.</h1>;
    return this.props.children;
  }
}
// 使用：<ErrorBoundary><MyComponent /></ErrorBoundary>
```

> 🔹 **注意**：错误边界无法捕获事件处理函数、异步代码（setTimeout）、SSR 以及自身报错。

### 4.5 组件化常见错误 vs 正确写法

| 场景 | 错误写法 / 现象 | 正确写法 / 做法 |
|------|-----------------|-----------------|
| **多组件共享状态各管各的** | 数据不同步 | **状态提升**到共同父组件，props + 回调。 |
| **Context value 每次渲染新对象** | 所有消费者频繁重渲染 | 用 **useState** 或 **useMemo** 稳定 value 引用。 |
| **子组件直接改 props** | 违反单向数据流、难调试 | 通过**回调**让父组件改；父传 setState 或回调函数。 |

### 5.1 React Router v6 入门

#### （1）基本配置

- **BrowserRouter** 包裹根组件；**Routes / Route** 配置路径与组件；**element** 指定要渲染的组件。

```jsx
import { BrowserRouter, Routes, Route } from 'react-router-dom'
function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/user/:id" element={<User />} />
      </Routes>
    </BrowserRouter>
  )
}
```

#### （2）导航与参数

| 用法 | 说明 |
|------|------|
| **useNavigate** | `const nav = useNavigate()`；`nav('/path')`、`nav(-1)` 后退 |
| **useParams** | 动态段：path="/user/:id" → `useParams().id` |
| **useSearchParams** | `const [params, setParams] = useSearchParams()`；`params.get('q')` |
| **Link** | `<Link to="/path">链接</Link>` 声明式导航 |

#### （3）懒加载

- **lazy** + **Suspense**：`const Page = lazy(() => import('./Page'))`，用 `<Suspense fallback={<Loading />}><Page /></Suspense>` 包裹，首屏只加载当前路由组件。

#### （4）路由鉴权示例

- 需登录的页面可包一层**鉴权组件**：检查 token，无则 **navigate** 到登录页；有则渲染 **children**（即实际页面）。

```jsx
function ProtectedRoute({ children }) {
  const navigate = useNavigate()
  const token = localStorage.getItem('token')
  useEffect(() => {
    if (!token) navigate('/login', { replace: true })
  }, [token, navigate])
  if (!token) return null
  return children
}
// 使用：<Route path="/dashboard" element={<ProtectedRoute><Dashboard /></ProtectedRoute>} />
```

### 5.2 全局状态：Redux 与 Zustand 简介

| 方案 | 说明 | 适用 |
|------|------|------|
| **Redux (RTK)** | store、slice (reducer + actions)、useSelector、useDispatch；单一数据源、纯函数 reducer | 大型应用、复杂业务逻辑、需时间旅行调试 |
| **Zustand** | create 创建 store，组件内直接 useXxxStore() 读写，无 Provider | 轻量、快速上手、中小型应用 |
| **Context** | 见第 4 章，React 内置，适合低频更新的数据 | 主题、语言、用户信息 |

**Redux Toolkit (RTK) 核心示例**：

```javascript
// store/counterSlice.js
import { createSlice } from '@reduxjs/toolkit'
export const counterSlice = createSlice({
  name: 'counter',
  initialState: { value: 0 },
  reducers: {
    increment: state => { state.value += 1 } // RTK 内部用 Immer，可直接修改
  }
})
export const { increment } = counterSlice.actions
export default counterSlice.reducer

// 组件中使用
import { useSelector, useDispatch } from 'react-redux'
import { increment } from './counterSlice'

function Counter() {
  const count = useSelector(state => state.counter.value)
  const dispatch = useDispatch()
  return <button onClick={() => dispatch(increment())}>{count}</button>
}
```

**Zustand 最小示例**：

```js
// store/useStore.js
import { create } from 'zustand'
const useStore = create((set) => ({
  count: 0,
  inc: () => set(state => ({ count: state.count + 1 }))
}))

// 组件中使用
// const { count, inc } = useStore()
```

### 5.3 与后端联调

- **请求**：在 **useEffect** 或事件中 **fetch/axios**，用 **useState** 存 data/loading/error。
- **环境变量**：Vite 用 **import.meta.env.VITE_XXX**。
- **鉴权（axios 拦截器）**：与 Vue 类似，请求拦截带 Token，响应拦截处理 401。

```javascript
import axios from 'axios'

// 请求拦截：统一带 token
axios.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// 响应拦截：401 跳登录
axios.interceptors.response.use(
  res => res,
  err => {
    if (err.response?.status === 401) {
      localStorage.removeItem('token')
      window.location.href = '/login' // 或通过其它方式跳转
    }
    return Promise.reject(err)
  }
)
```

### 6.4 面试要点：虚拟 DOM、Hooks 原理、受控组件、React vs Vue

#### （1）虚拟 DOM 与 diff

- **虚拟 DOM**：用 JS 对象描述 DOM 树，数据变时生成新树，**diff** 新旧树得最小变更再 **patch** 到真实 DOM。

**diff 大致过程（面试可答）**：

1. **同层比较**：只比较同一层级节点，不跨层，复杂度 O(n)；节点类型不同则直接替换整棵子树。
2. **key 的作用**：列表节点用 **key** 标识身份，diff 按 key 匹配，可复用节点、调整顺序，避免就地复用导致状态错乱。
3. **协调**：React 通过 **Fiber** 结构（链表）做可中断的协调；Concurrent 下可优先高优先级更新。
4. **patch**：diff 得到变更后，对真实 DOM 做最小化更新（增、删、改、移）。

**面试怎么说**：用对象表示 DOM，diff 算最小变更再 patch；同层比较、key 复用；React 18 还有 Fiber 与并发调度。

#### （2）Hooks 原理（简述）

- **状态如何与组件对应**：React 在组件内部用**链表（或数组）**按 **Hooks 调用顺序**存储 state；每次渲染按顺序取对应项。因此**顺序不能变**（不能放在条件/循环里）。**面试怎么说**：Hooks 靠调用顺序与组件实例对应；顺序一变就会错位，所以只能在顶层调用。

#### （3）受控组件

- 见第 2 章：**value 由 state 控制 + onChange 更新 state**；React 推荐受控，便于校验与统一状态。

#### （4）React 与 Vue 对比

- 见第 1 章：**JSX vs 模板**、**Hooks vs Composition API**、**单向数据流 vs 响应式**、生态与选型。

## 🎯 面试常见追问

| 面试官提问 | 回答思路 |
|------------|----------|
| 虚拟 DOM 是什么？diff 怎么做的？ | 用 JS 对象表示 DOM，更新时 diff 新旧树得最小变更再 patch；同层比较、key 复用。 |
| 为什么 Hooks 不能放在条件里？ | Hooks 靠调用顺序与 state 对应；条件会导致顺序变化，state 错位。 |
| React 和 Vue 区别？ | React JSX+Hooks+单向数据流，Vue 模板+Composition API+响应式；React 生态大，Vue 上手快。 |

------

### 常见坑与注意点

| 现象 / 易错点 | 原因 | 怎么改 / 怎么记 |
|---------------|------|-----------------|
| 过度使用 useMemo/useCallback | 依赖复杂、维护成本高 | 先保证正确，再对瓶颈优化；子组件用 memo 时再稳定回调。 |
| 面试说不清 Hooks 原理 | 未理解顺序对应 | 记住：**调用顺序**与 state 一一对应，所以只能顶层、不能条件。 |
