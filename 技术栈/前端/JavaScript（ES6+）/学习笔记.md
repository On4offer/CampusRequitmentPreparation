## 🌱 1.1 JavaScript 简介

### 📖 核心概念

**JavaScript** 是一种**单线程、动态类型**的脚本语言，运行在浏览器与 Node.js 等环境中，负责页面的交互与逻辑。

**JavaScript的特点**：
- **单线程**：同一时间只能执行一个任务
- **动态类型**：变量类型在运行时确定
- **解释执行**：无需编译，直接由引擎解释执行
- **面向对象**：基于原型的面向对象语言
- **事件驱动**：通过事件触发执行代码

**ES6（ES2015）及后续版本**：
- **ES6 (2015)**：let/const、箭头函数、Promise、模块化、解构、类
- **ES7 (2016)**：Array.includes()、指数运算符
- **ES8 (2017)**：async/await、Object.values()/Object.entries()
- **ES9 (2018)**：Rest/Spread属性、异步迭代
- **ES10 (2019)**：Array.flat()/flatMap()、String.trimStart()/trimEnd()
- **ES11 (2020)**：BigInt、Nullish Coalescing Operator、Optional Chaining
- **ES12 (2021)**：Logical Assignment Operators、Numeric Separator

**1. 浏览器环境**

### 📖 执行机制

**1. 单线程与异步**
- **单线程**：JavaScript只有一个主线程，避免了多线程的复杂性
- **异步任务**：通过任务队列和事件循环实现异步操作

**2. 事件循环（Event Loop）**
```javascript
// 事件循环示意图
while (eventLoop.waitForTask()) {
    eventLoop.processNextTask();
}
```

**3. 任务队列**
- **宏任务（Macro Task）**：setTimeout、setInterval、I/O操作、DOM事件
- **微任务（Micro Task）**：Promise.then()、async/await、process.nextTick（Node.js）

**4. 执行顺序**
1. 执行主线程同步代码
2. 执行微任务队列中的所有任务
3. 执行宏任务队列中的一个任务
4. 重复步骤2-3

**代码示例**：
```javascript
console.log('1'); // 同步代码

setTimeout(() => {
    console.log('2'); // 宏任务
}, 0);

Promise.resolve().then(() => {
    console.log('3'); // 微任务
});

console.log('4'); // 同步代码

// 输出顺序：1 -> 4 -> 3 -> 2
```

### 💬 面试问答

> **问：JavaScript为什么是单线程的？**
> **答**：JavaScript设计为单线程主要是为了避免多线程编程的复杂性，特别是在操作DOM时。如果多个线程同时操作DOM，可能会导致DOM结构混乱。单线程模型也使得代码更容易理解和调试。

> **问：什么是事件循环？**
> **答**：事件循环是JavaScript处理异步任务的机制。主线程执行同步代码后，会从任务队列中取出异步任务执行。任务队列分为宏任务和微任务，微任务优先级高于宏任务，每个宏任务执行后会清空微任务队列。

> **问：宏任务和微任务的区别？**
> **答**：宏任务包括setTimeout、setInterval、I/O操作、DOM事件等，微任务包括Promise.then()、async/await、process.nextTick等。微任务在宏任务执行后立即执行，优先级更高，而宏任务需要等待下一次事件循环。

## 🌱 2.1 变量声明

### 📖 变量声明方式

| 声明方式 | 作用域 | 可重复声明 | 变量提升 | 暂时性死区 | 初始值 |
|----------|--------|------------|----------|------------|--------|
| **var** | 函数作用域 | 是 | 是 | 否 | undefined |
| **let** | 块级作用域 | 否 | 否 | 是 | undefined |
| **const** | 块级作用域 | 否 | 否 | 是 | 必须初始化 |

**代码示例**：
```javascript
// var 示例
console.log(a); // undefined（变量提升）
var a = 1;
console.log(a); // 1

// let 示例
console.log(b); // ReferenceError（暂时性死区）
let b = 2;
console.log(b); // 2

// const 示例
const c = 3;
c = 4; // TypeError（不可修改）

// 块级作用域
if (true) {
    var x = 10; // 函数作用域
    let y = 20; // 块级作用域
    const z = 30; // 块级作用域
}
console.log(x); // 10
console.log(y); // ReferenceError
console.log(z); // ReferenceError
```

### 🔍 最佳实践

- **优先使用const**：避免意外修改，提高代码可维护性
- **let用于需要重新赋值的变量**：如循环计数器、状态变量
- **避免使用var**：函数作用域容易导致变量泄漏和意外覆盖

## 🌱 2.2 数据类型

### 📖 数据类型分类

**1. 原始类型（Primitive Types）**
- **number**：数字（整数、浮点数、NaN、Infinity）
- **string**：字符串
- **boolean**：布尔值（true、false）
- **null**：空值
- **undefined**：未定义
- **symbol**：唯一标识符
- **bigint**：大整数

**2. 引用类型（Reference Types）**
- **object**：对象（普通对象、数组、函数、日期、正则表达式等）

### 🔍 数据类型特点

| 类型 | 存储方式 | 比较方式 | 示例 |
|------|----------|----------|------|
| **原始类型** | 栈内存 | 值比较 | `1 === 1` → true |
| **引用类型** | 堆内存（栈存储引用） | 引用比较 | `{} === {}` → false |

**代码示例**：
```javascript
// 原始类型
const a = 1;
const b = 1;
console.log(a === b); // true

// 引用类型
const obj1 = {};
const obj2 = {};
console.log(obj1 === obj2); // false

// symbol 唯一
const s1 = Symbol('key');
const s2 = Symbol('key');
console.log(s1 === s2); // false

// bigint 大整数
const big1 = 9007199254740991n;
const big2 = BigInt(9007199254740991);
console.log(big1 === big2); // true
```

## 🌱 2.3 类型判断

### 📖 类型判断方法

| 方法 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **typeof** | 原始类型 | 简单直接 | 无法区分null和object |
| **instanceof** | 引用类型 | 可判断具体类型 | 无法判断原始类型 |
| **Object.prototype.toString.call** | 所有类型 | 最准确 | 语法复杂 |
| **Array.isArray** | 数组 | 专门用于数组 | 只适用于数组 |
| **Number.isNaN** | NaN | 准确判断NaN | 只适用于NaN |

**代码示例**：
```javascript
// typeof
console.log(typeof 1); // 'number'
console.log(typeof 'hello'); // 'string'
console.log(typeof true); // 'boolean'
console.log(typeof null); // 'object'（历史遗留问题）
console.log(typeof undefined); // 'undefined'
console.log(typeof Symbol()); // 'symbol'
console.log(typeof 1n); // 'bigint'
console.log(typeof {}); // 'object'
console.log(typeof []); // 'object'
console.log(typeof function() {}); // 'function'

// instanceof
console.log([] instanceof Array); // true
console.log({} instanceof Object); // true
console.log(function() {} instanceof Function); // true

// Object.prototype.toString.call
console.log(Object.prototype.toString.call(1)); // '[object Number]'
console.log(Object.prototype.toString.call('hello')); // '[object String]'
console.log(Object.prototype.toString.call(true)); // '[object Boolean]'
console.log(Object.prototype.toString.call(null)); // '[object Null]'
console.log(Object.prototype.toString.call(undefined)); // '[object Undefined]'
console.log(Object.prototype.toString.call([])); // '[object Array]'
console.log(Object.prototype.toString.call({})); // '[object Object]'

// Array.isArray
console.log(Array.isArray([])); // true
console.log(Array.isArray({})); // false

// Number.isNaN
console.log(Number.isNaN(NaN)); // true
console.log(Number.isNaN(1)); // false
```

## 🌱 2.4 类型转换

### 📖 显式类型转换

**1. 转为数字**
- `Number(value)`：通用转换
- `parseInt(value, radix)`：转为整数
- `parseFloat(value)`：转为浮点数
- `+value`：快捷转换

**2. 转为字符串**
- `String(value)`：通用转换
- `value.toString()`：对象方法
- `` `${value}` ``：模板字符串

**3. 转为布尔值**
- `Boolean(value)`：通用转换
- `!!value`：快捷转换

**代码示例**：
```javascript
// 转为数字
console.log(Number('123')); // 123
console.log(Number('123abc')); // NaN
console.log(Number(true)); // 1
console.log(Number(false)); // 0
console.log(Number(null)); // 0
console.log(Number(undefined)); // NaN

// 转为字符串
console.log(String(123)); // '123'
console.log(String(true)); // 'true'
console.log(String(null)); // 'null'
console.log(String(undefined)); // 'undefined'

// 转为布尔值
console.log(Boolean(0)); // false
console.log(Boolean('')); // false
console.log(Boolean(null)); // false
console.log(Boolean(undefined)); // false
console.log(Boolean(NaN)); // false
console.log(Boolean(1)); // true
console.log(Boolean('hello')); // true
console.log(Boolean({})); // true
console.log(Boolean([])); // true
```

### 📖 隐式类型转换

**1. 算术运算符**
- `+`：遇到字符串会转为字符串
- `-`、`*`、`/`、`%`：转为数字

**2. 比较运算符**
- `==`：会进行类型转换
- `===`：严格相等，不进行类型转换

**3. 逻辑运算符**
- `&&`、`||`、`!`：会转为布尔值

**代码示例**：
```javascript
// 算术运算符
console.log(1 + '2'); // '12'（字符串拼接）
console.log('1' + '2'); // '12'
console.log(1 - '2'); // -1（转为数字）
console.log('10' * '2'); // 20（转为数字）

// 比较运算符
console.log(1 == '1'); // true（类型转换）
console.log(1 === '1'); // false（严格相等）
console.log(null == undefined); // true
console.log(null === undefined); // false
console.log(0 == false); // true
console.log(0 === false); // false

// 逻辑运算符
console.log(!!0); // false
console.log(!!''); // false
console.log(!!null); // false
console.log(!!undefined); // false
console.log(!!NaN); // false
console.log(!!1); // true
console.log(!!'hello'); // true
```

### 🔍 Falsy 值

**Falsy 值**：在布尔上下文中被视为 false 的值
- `false`
- `0`、`-0`
- `''`（空字符串）
- `null`
- `undefined`
- `NaN`

**Truthy 值**：除了 falsy 值以外的所有值

## 🌱 2.5 运算符与表达式

### 📖 常用运算符

**1. 算术运算符**
- `+`、`-`、`*`、`/`、`%`、`**`（指数）

**2. 赋值运算符**
- `=`、`+=`、`-=`、`*=`、`/=`、`%=`

**3. 比较运算符**
- `==`、`===`、`!=`、`!==`、`>`、`<`、`>=`、`<=`

**4. 逻辑运算符**
- `&&`（与）、`||`（或）、`!`（非）

**5. 位运算符**
- `&`（与）、`|`（或）、`^`（异或）、`~`（取反）、`<<`（左移）、`>>`（右移）、`>>>`（无符号右移）

**6. 其他运算符**
- `typeof`：类型判断
- `instanceof`：实例判断
- `in`：属性判断
- `delete`：删除属性
- `void`：返回 undefined

### 📖 ES6+ 新运算符

**1. 展开运算符**（`...`）
```javascript
// 数组展开
const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5]; // [1, 2, 3, 4, 5]

// 对象展开
const obj1 = { a: 1, b: 2 };
const obj2 = { ...obj1, c: 3 }; // { a: 1, b: 2, c: 3 }

// 函数参数
function sum(...args) {
    return args.reduce((a, b) => a + b, 0);
}
console.log(sum(1, 2, 3, 4)); // 10
```

**2. 可选链运算符**（`?.`）
```javascript
const user = {
    name: '张三',
    address: {
        city: '北京'
    }
};

// 传统方式
console.log(user.address && user.address.city); // '北京'

// 可选链
console.log(user.address?.city); // '北京'
console.log(user.address?.street?.name); // undefined
```

**3. 空值合并运算符**（`??`）
```javascript
// 传统方式
const name = user.name || '默认值'; // 会把空字符串、0等falsy值替换为默认值

// 空值合并
const name = user.name ?? '默认值'; // 只有null和undefined才会使用默认值
```

**4. 逻辑赋值运算符**（`&&=`、`||=`、`??=`）
```javascript
// &&=
let a = 1;
a &&= 2; // 等价于 a = a && 2 → 2

// ||=
let b = null;
b ||= '默认值'; // 等价于 b = b || '默认值' → '默认值'

// ??=
let c = undefined;
c ??= '默认值'; // 等价于 c = c ?? '默认值' → '默认值'
```

**5. 空值判断运算符**（`?:`）
```javascript
// 三元运算符
const result = condition ? value1 : value2;

// 嵌套使用
const result = condition1 ? value1 : condition2 ? value2 : value3;
```

## 🌱 3.1 函数定义

### 📖 函数定义方式

| 定义方式 | 语法 | 特点 | 应用场景 |
|----------|------|------|----------|
| **函数声明** | `function fn() {}` | 函数提升，可在定义前调用 | 普通函数定义 |
| **函数表达式** | `const fn = function() {}` | 无函数提升，必须先定义后调用 | 匿名函数、回调函数 |
| **箭头函数** | `const fn = () => {}` | 无自身this，无arguments，不能作为构造函数 | 回调函数、保持this上下文 |
| **Function构造函数** | `new Function('a', 'b', 'return a + b')` | 动态创建函数，性能较差 | 动态代码生成 |
| **立即执行函数(IIFE)** | `(function() {})()` | 创建独立作用域 | 模块化、避免全局污染 |

**代码示例**：
```javascript
// 函数声明
function add(a, b) {
    return a + b;
}

// 函数表达式
const multiply = function(a, b) {
    return a * b;
};

// 箭头函数
const divide = (a, b) => a / b;

// 带函数体的箭头函数
const greet = (name) => {
    const message = `Hello, ${name}!`;
    return message;
};

// 立即执行函数
(function() {
    const privateVar = '私有变量';
    console.log(privateVar);
})();
```

## 🌱 3.2 this 指向

### 📖 this 绑定规则

| 绑定规则 | 触发条件 | this 指向 | 示例 |
|----------|----------|-----------|------|
| **默认绑定** | 独立函数调用 | 全局对象（严格模式为 undefined） | `fn()` |
| **隐式绑定** | 方法调用 | 调用该方法的对象 | `obj.fn()` |
| **显式绑定** | call/apply/bind | 指定的对象 | `fn.call(obj)` |
| **new 绑定** | 构造函数调用 | 新创建的实例对象 | `new Fn()` |
| **箭头函数** | 箭头函数调用 | 继承外层作用域的 this | `() => this.x` |

**代码示例**：
```javascript
// 默认绑定
function defaultBinding() {
    console.log(this); // 全局对象或 undefined（严格模式）
}
defaultBinding();

// 隐式绑定
const obj = {
    name: '张三',
    greet() {
        console.log(this.name); // '张三'
    }
};
obj.greet();

// 显式绑定
function explicitBinding() {
    console.log(this.name);
}
const person = { name: '李四' };
explicitBinding.call(person); // '李四'
explicitBinding.apply(person); // '李四'
const boundFn = explicitBinding.bind(person);
boundFn(); // '李四'

// new 绑定
function Person(name) {
    this.name = name;
}
const p = new Person('王五');
console.log(p.name); // '王五'

// 箭头函数
const arrowObj = {
    name: '赵六',
    greet: () => {
        console.log(this.name); // undefined，继承全局 this
    },
    greetNormal() {
        const arrow = () => {
            console.log(this.name); // '赵六'，继承外层 this
        };
        arrow();
    }
};
```

### 🔍 call、apply、bind 的区别

| 方法 | 调用时机 | 参数传递 | 返回值 |
|------|----------|----------|--------|
| **call** | 立即执行 | 参数列表 | 函数返回值 |
| **apply** | 立即执行 | 参数数组 | 函数返回值 |
| **bind** | 延迟执行 | 参数列表 | 绑定后的新函数 |

**代码示例**：
```javascript
function greet(greeting, punctuation) {
    console.log(`${greeting}, ${this.name}${punctuation}`);
}

const person = { name: '张三' };

// call
greet.call(person, 'Hello', '!'); // 'Hello, 张三!'

// apply
greet.apply(person, ['Hi', '?']); // 'Hi, 张三?'

// bind
const boundGreet = greet.bind(person, 'Hey');
boundGreet('...'); // 'Hey, 张三...'
```

### 💬 面试问答

> **问：箭头函数与普通函数的区别？**
> **答**：箭头函数没有自己的 this，继承外层作用域的 this；没有 arguments 对象；不能作为构造函数使用；没有 prototype 属性；语法更简洁。

> **问：call、apply、bind 的区别？**
> **答**：call 和 apply 会立即执行函数，call 接收参数列表，apply 接收参数数组；bind 返回一个新函数，不会立即执行，可预设参数。

> **问：this 的指向规则有哪些？**
> **答**：默认绑定（独立调用指向全局）、隐式绑定（方法调用指向调用对象）、显式绑定（call/apply/bind 指定）、new 绑定（构造函数指向新实例）、箭头函数（继承外层 this）。

## 🌱 3.3 闭包

### 📖 闭包的概念

**闭包（Closure）**：函数与其词法环境的组合，使得函数可以访问定义时的作用域，即使该作用域已经执行完毕。

**闭包的形成条件**：
1. 函数嵌套（内部函数引用外部函数的变量）
2. 内部函数被返回或传递到外部使用
3. 外部函数执行完毕，但内部函数仍然持有对外部变量的引用

**代码示例**：
```javascript
function createCounter() {
    let count = 0; // 外部函数的局部变量
    
    return function() {
        return ++count; // 内部函数引用外部变量
    };
}

const counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3
```

### 🔍 闭包的应用场景

| 应用场景 | 实现方式 | 优点 |
|----------|----------|------|
| **私有变量** | 通过闭包封装变量 | 实现数据封装和隐藏 |
| **模块化** | 使用 IIFE 创建私有作用域 | 避免全局污染 |
| **柯里化** | 返回嵌套函数，逐步接收参数 | 函数复用、延迟执行 |
| **防抖节流** | 使用闭包保存计时器 | 优化性能 |
| **缓存** | 使用闭包保存计算结果 | 避免重复计算 |

**私有变量示例**：
```javascript
function createPerson(name) {
    let _name = name; // 私有变量
    
    return {
        getName() {
            return _name;
        },
        setName(newName) {
            _name = newName;
        }
    };
}

const person = createPerson('张三');
console.log(person.getName()); // '张三'
person.setName('李四');
console.log(person.getName()); // '李四'
// 无法直接访问 _name
```

**柯里化示例**：
```javascript
function curry(fn) {
    return function curried(...args) {
        if (args.length >= fn.length) {
            return fn.apply(this, args);
        } else {
            return function(...args2) {
                return curried.apply(this, args.concat(args2));
            };
        }
    };
}

// 使用
function sum(a, b, c) {
    return a + b + c;
}

const curriedSum = curry(sum);
console.log(curriedSum(1)(2)(3)); // 6
console.log(curriedSum(1, 2)(3)); // 6
console.log(curriedSum(1)(2, 3)); // 6
```

**防抖函数**：
```javascript
function debounce(fn, delay) {
    let timer;
    return function(...args) {
        clearTimeout(timer);
        timer = setTimeout(() => {
            fn.apply(this, args);
        }, delay);
    };
}

// 使用
const debouncedSearch = debounce(searchFunction, 300);
input.addEventListener('input', debouncedSearch);
```

**节流函数**：
```javascript
function throttle(fn, delay) {
    let lastCall = 0;
    return function(...args) {
        const now = Date.now();
        if (now - lastCall >= delay) {
            lastCall = now;
            return fn.apply(this, args);
        }
    };
}

// 使用
const throttledScroll = throttle(scrollFunction, 100);
window.addEventListener('scroll', throttledScroll);
```

### ⚠️ 闭包的注意事项

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| **内存泄漏** | 闭包引用外部变量，变量无法被垃圾回收 | 及时释放引用，使用 WeakMap/WeakSet |
| **性能问题** | 闭包占用内存，过多闭包影响性能 | 合理使用，避免不必要的闭包 |
| **变量共享** | 循环中使用闭包，变量共享导致意外结果 | 使用 let/const 或立即执行函数 |

**循环中的闭包问题**：
```javascript
// 问题代码
for (var i = 0; i < 5; i++) {
    setTimeout(() => {
        console.log(i); // 输出 5 次 5
    }, 100);
}

// 解决方案1：使用 let
for (let i = 0; i < 5; i++) {
    setTimeout(() => {
        console.log(i); // 输出 0, 1, 2, 3, 4
    }, 100);
}

// 解决方案2：使用立即执行函数
for (var i = 0; i < 5; i++) {
    (function(j) {
        setTimeout(() => {
            console.log(j); // 输出 0, 1, 2, 3, 4
        }, 100);
    })(i);
}
```

## 🌱 3.4 高阶函数与函数式编程

### 📖 高阶函数

**高阶函数**：接受函数作为参数或返回函数的函数。

| 高阶函数 | 说明 | 示例 |
|----------|------|------|
| **map** | 映射数组每个元素 | `arr.map(x => x * 2)` |
| **filter** | 过滤数组元素 | `arr.filter(x => x > 0)` |
| **reduce** | 归并数组元素 | `arr.reduce((a, b) => a + b, 0)` |
| **forEach** | 遍历数组元素 | `arr.forEach(x => console.log(x))` |
| **sort** | 排序数组 | `arr.sort((a, b) => a - b)` |
| **some/every** | 检查数组元素 | `arr.some(x => x > 0)` |
| **find/findIndex** | 查找数组元素 | `arr.find(x => x > 0)` |

**代码示例**：
```javascript
const numbers = [1, 2, 3, 4, 5];

// map
const doubled = numbers.map(x => x * 2); // [2, 4, 6, 8, 10]

// filter
const evens = numbers.filter(x => x % 2 === 0); // [2, 4]

// reduce
const sum = numbers.reduce((acc, curr) => acc + curr, 0); // 15

// 组合使用
const result = numbers
    .filter(x => x > 2)
    .map(x => x * 2)
    .reduce((acc, curr) => acc + curr, 0); // 24
```

### 🔍 函数组合

**函数组合**：将多个函数组合成一个函数，从右到左执行。

```javascript
// 函数组合
function compose(...fns) {
    return function(x) {
        return fns.reduceRight((acc, fn) => fn(acc), x);
    };
}

// 使用
const add5 = x => x + 5;
const multiply2 = x => x * 2;
const toString = x => String(x);

const composed = compose(toString, multiply2, add5);
console.log(composed(3)); // '16' (3 + 5 = 8, 8 * 2 = 16, String(16) = '16')
```

> **问：什么是闭包？**
> **答**：闭包是函数与其词法环境的组合，使得函数可以访问定义时的作用域，即使该作用域已经执行完毕。闭包可以实现私有变量、模块化、柯里化等功能。

> **问：闭包的优缺点？**
> **答**：优点：实现私有变量、延长变量生命周期、模块化、函数柯里化；缺点：可能导致内存泄漏（变量长期被引用）。

> **问：什么是高阶函数？**
> **答**：高阶函数是接受函数作为参数或返回函数的函数。常见的高阶函数有 map、filter、reduce、forEach 等。

### 📖 属性操作

```javascript
}
```

## 🌱 4.2 原型与原型链

### 📖 原型相关概念

| 概念 | 说明 | 访问方式 |
|------|------|----------|
| **prototype** | 函数的属性，指向原型对象 | `Fn.prototype` |
| **__proto__** | 对象的属性，指向构造函数的原型 | `obj.__proto__` |
| **constructor** | 原型对象的属性，指向构造函数 | `obj.constructor` |
| **原型链** | 对象通过 __proto__ 链接形成的链条 | 属性查找路径 |

**代码示例**：
```javascript
function Person(name) {
    this.name = name;
}

Person.prototype.greet = function() {
    console.log(`Hello, I'm ${this.name}`);
};

const person = new Person('张三');

// 原型关系
console.log(person.__proto__ === Person.prototype); // true
console.log(Person.prototype.constructor === Person); // true
console.log(person.__proto__.__proto__ === Object.prototype); // true
console.log(Object.prototype.__proto__); // null
```

### 🔍 原型链查找机制

**属性查找过程**：
1. 在对象自身查找属性
2. 如果找不到，在对象的 __proto__ 中查找
3. 如果还找不到，继续在 __proto__ 的 __proto__ 中查找
4. 直到找到属性或到达原型链顶端（null）

```javascript
function Animal(name) {
    this.name = name;
}

Animal.prototype.eat = function() {
    console.log(`${this.name} is eating`);
};

function Dog(name, breed) {
    Animal.call(this, name);
    this.breed = breed;
}

// 继承 Animal
Dog.prototype = Object.create(Animal.prototype);
Dog.prototype.constructor = Dog;

Dog.prototype.bark = function() {
    console.log(`${this.name} is barking`);
};

const dog = new Dog('旺财', '金毛');
dog.eat();  // 在 Animal.prototype 中找到
dog.bark(); // 在 Dog.prototype 中找到
```

### 💬 面试问答

> **问：prototype 和 __proto__ 的区别？**
> **答**：prototype 是函数的属性，指向原型对象；__proto__ 是对象的属性，指向构造函数的原型。实例对象的 __proto__ 等于构造函数的 prototype。

> **问：什么是原型链？**
> **答**：原型链是对象通过 __proto__ 链接形成的链条，用于实现继承。当访问对象的属性时，如果在自身找不到，会沿着原型链向上查找。

> **问：如何创建一个没有原型的对象？**
> **答**：使用 `Object.create(null)` 可以创建一个没有原型的对象，这样的对象不会继承 Object.prototype 的任何方法。

## 🌱 4.3 继承实现

### 📖 继承方式对比

| 继承方式 | 实现 | 优点 | 缺点 |
|----------|------|------|------|
| **原型链继承** | `Child.prototype = new Parent()` | 简单 | 无法向父类传参，引用类型共享 |
| **构造函数继承** | `Parent.call(this)` | 可传参，不共享引用类型 | 无法继承原型方法 |
| **组合继承** | 原型链 + 构造函数 | 可传参，继承原型方法 | 调用两次父类构造函数 |
| **寄生组合继承** | `Object.create(Parent.prototype)` | 完美继承 | 较复杂 |
| **ES6 class extends** | `class Child extends Parent` | 语法清晰 | 语法糖，底层仍是原型 |

**原型链继承**：
```javascript
function Parent() {
    this.name = 'Parent';
    this.colors = ['red', 'blue'];
}

Parent.prototype.getName = function() {
    return this.name;
};

function Child() {
    this.name = 'Child';
}

// 继承
Child.prototype = new Parent();

const child1 = new Child();
const child2 = new Child();
child1.colors.push('green');
console.log(child2.colors); // ['red', 'blue', 'green']（共享引用类型）
```

**构造函数继承**：
```javascript
function Parent(name) {
    this.name = name;
    this.colors = ['red', 'blue'];
}

Parent.prototype.getName = function() {
    return this.name;
};

function Child(name) {
    Parent.call(this, name);
}

const child1 = new Child('Child1');
const child2 = new Child('Child2');
child1.colors.push('green');
console.log(child2.colors); // ['red', 'blue']（不共享）
console.log(child1.getName); // undefined（无法继承原型方法）
```

**组合继承**：
```javascript
function Parent(name) {
    this.name = name;
    this.colors = ['red', 'blue'];
}

Parent.prototype.getName = function() {
    return this.name;
};

function Child(name, age) {
    Parent.call(this, name); // 第二次调用 Parent
    this.age = age;
}

Child.prototype = new Parent(); // 第一次调用 Parent
Child.prototype.constructor = Child;

const child = new Child('Child', 18);
console.log(child.getName()); // 'Child'
```

**寄生组合继承**（推荐）：
```javascript
function inheritPrototype(Child, Parent) {
    const prototype = Object.create(Parent.prototype);
    prototype.constructor = Child;
    Child.prototype = prototype;
}

function Parent(name) {
    this.name = name;
    this.colors = ['red', 'blue'];
}

Parent.prototype.getName = function() {
    return this.name;
};

function Child(name, age) {
    Parent.call(this, name);
    this.age = age;
}

inheritPrototype(Child, Parent);

Child.prototype.getAge = function() {
    return this.age;
};

const child = new Child('Child', 18);
console.log(child.getName()); // 'Child'
console.log(child.getAge());  // 18
```

## 🌱 4.4 ES6 类

### 📖 class 语法

```javascript
class Person {
    // 构造函数
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
    
    // 实例方法
    greet() {
        console.log(`Hello, I'm ${this.name}`);
    }
    
    // 静态方法
    static isPerson(obj) {
        return obj instanceof Person;
    }
    
    // getter
    get info() {
        return `${this.name}, ${this.age} years old`;
    }
    
    // setter
    set info(value) {
        [this.name, this.age] = value.split(', ');
    }
}

const person = new Person('张三', 25);
person.greet(); // 'Hello, I'm 张三'
console.log(Person.isPerson(person)); // true
console.log(person.info); // '张三, 25 years old'
```

### 📖 继承与 super

```javascript
class Animal {
    constructor(name) {
        this.name = name;
    }
    
    eat() {
        console.log(`${this.name} is eating`);
    }
    
    static isAnimal(obj) {
        return obj instanceof Animal;
    }
}

class Dog extends Animal {
    constructor(name, breed) {
        super(name); // 调用父类构造函数
        this.breed = breed;
    }
    
    bark() {
        console.log(`${this.name} is barking`);
    }
    
    // 重写父类方法
    eat() {
        super.eat(); // 调用父类方法
        console.log(`${this.name} is eating dog food`);
    }
    
    static isDog(obj) {
        return obj instanceof Dog;
    }
}

const dog = new Dog('旺财', '金毛');
dog.eat();  // '旺财 is eating' '旺财 is eating dog food'
dog.bark(); // '旺财 is barking'
console.log(Dog.isDog(dog)); // true
console.log(Animal.isAnimal(dog)); // true
```

### 🔍 class 与构造函数的区别

| 特性 | class | 构造函数 |
|------|-------|----------|
| **语法** | 更清晰，类似其他语言 | 较传统 |
| **提升** | 不存在提升 | 函数声明提升 |
| **调用方式** | 必须使用 new | 可直接调用 |
| **方法枚举** | 方法不可枚举 | 原型方法可枚举 |
| **本质** | 语法糖，底层仍是原型 | 原型机制 |

### 💬 面试问答

> **问：ES6 class 与构造函数的区别？**

> **问：super 的作用？**
> **答**：在构造函数中调用 super() 调用父类构造函数；在方法中调用 super.method() 调用父类方法。

> **问：如何实现多重继承？**
> **答**：JavaScript 不支持多重继承，但可以通过 mixin 模式实现类似功能。

## 🌱 5.1 异步编程基础

### 📖 同步与异步

| 特性 | 同步 | 异步 |
|------|------|------|
| **执行方式** | 按顺序执行，阻塞后续代码 | 不阻塞，后续代码继续执行 |
| **代码复杂度** | 简单直观 | 相对复杂，需要处理回调 |
| **性能** | 阻塞主线程，性能差 | 不阻塞主线程，性能好 |
| **适用场景** | 简单计算、数据处理 | I/O操作、网络请求、定时器 |

**代码示例**：
```javascript
// 同步代码
console.log('开始');
console.log('中间');
console.log('结束');
// 输出：开始 -> 中间 -> 结束

// 异步代码
console.log('开始');
setTimeout(() => {
    console.log('异步操作');
}, 0);
console.log('结束');
// 输出：开始 -> 结束 -> 异步操作
```

### 📖 回调函数

**回调函数**：作为参数传递给另一个函数，在特定时机被调用的函数。

```javascript
// 基本回调
function fetchData(callback) {
    setTimeout(() => {
        const data = { id: 1, name: '张三' };
        callback(data);
    }, 1000);
}

fetchData((data) => {
    console.log(data);
});

// 回调地狱
fetchUser((user) => {
    fetchOrders(user.id, (orders) => {
        fetchProducts(orders[0].id, (products) => {
            fetchDetails(products[0].id, (details) => {
                console.log(details);
            });
        });
    });
});
```

### 🔍 回调地狱的问题

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| **代码嵌套深** | 多层回调嵌套，代码难以阅读 | Promise 链式调用 |
| **错误处理困难** | 每个回调都需要单独处理错误 | Promise.catch / try-catch |
| **代码耦合度高** | 逻辑紧密耦合，难以维护 | async/await |
| **回调执行顺序不清晰** | 异步流程难以追踪 | Promise / async/await |

## 🌱 5.2 Promise

### 📖 Promise 基础

**Promise**：表示一个异步操作的最终完成（或失败）及其结果值的对象。

**Promise 状态**：
| 状态 | 说明 | 转换 |
|------|------|------|
| **pending** | 初始状态，既未完成也未拒绝 | - |
| **fulfilled** | 操作成功完成 | pending -> fulfilled |
| **rejected** | 操作失败 | pending -> rejected |

**Promise 特点**：
- 状态一旦改变，不可再次改变
- 可以链式调用
- 支持异步操作的组合

**代码示例**：
```javascript
// 创建 Promise
const promise = new Promise((resolve, reject) => {
    // 异步操作
    setTimeout(() => {
        const success = true;
        if (success) {
            resolve('操作成功');
        } else {
            reject('操作失败');
        }
    }, 1000);
});

// 使用 Promise
promise
    .then((result) => {
        console.log(result); // '操作成功'
    })
    .catch((error) => {
        console.error(error);
    })
    .finally(() => {
        console.log('无论成功失败都执行');
    });
```

### 📖 Promise API

| API | 说明 | 返回值 | 应用场景 |
|-----|------|--------|----------|
| **Promise.resolve** | 创建已完成的 Promise | Promise | 快速创建成功状态的 Promise |
| **Promise.reject** | 创建已拒绝的 Promise | Promise | 快速创建失败状态的 Promise |
| **Promise.all** | 等待所有 Promise 完成 | Promise | 并行执行多个异步操作 |
| **Promise.race** | 返回最快完成的 Promise | Promise | 超时处理、竞速 |
| **Promise.allSettled** | 等待所有 Promise 结束 | Promise | 获取所有结果，无论成功失败 |
| **Promise.any** | 返回第一个成功的 Promise | Promise | 多个请求，取最快成功的 |

**代码示例**：
```javascript
// Promise.all
const promises = [
    fetch('/api/user'),
    fetch('/api/posts'),
    fetch('/api/comments')
];

Promise.all(promises)
    .then(([user, posts, comments]) => {
        console.log('所有请求完成');
    })
    .catch((error) => {
        console.error('有一个请求失败');
    });

// Promise.race（超时处理）
function fetchWithTimeout(url, timeout) {
    return Promise.race([
        fetch(url),
        new Promise((_, reject) => {
            setTimeout(() => reject(new Error('Timeout')), timeout);
        })
    ]);
}

// Promise.allSettled
Promise.allSettled(promises)
    .then((results) => {
        results.forEach((result) => {
            if (result.status === 'fulfilled') {
                console.log('成功:', result.value);
            } else {
                console.log('失败:', result.reason);
            }
        });
    });
```

### 🔍 Promise 链式调用

**链式调用规则**：
- then/catch/finally 返回新的 Promise
- 可以链式调用多个 then
- 错误会传递到最近的 catch

```javascript
fetch('/api/user')
    .then((response) => {
        if (!response.ok) {
            throw new Error('Network response was not ok');
        }
        return response.json();
    })
    .then((user) => {
        return fetch(`/api/orders/${user.id}`);
    })
    .then((response) => response.json())
    .then((orders) => {
        console.log(orders);
    })
    .catch((error) => {
        console.error('Error:', error);
    });
```

### 💬 面试问答

> **问：Promise 的状态有哪些？**
> **答**：pending（初始状态）、fulfilled（成功）、rejected（失败）。状态一旦改变，不可再次改变。

> **问：Promise.all 和 Promise.race 的区别？**
> **答**：Promise.all 等待所有 Promise 完成，全部成功才成功，一个失败就失败；Promise.race 返回最快完成的 Promise，无论成功失败。

> **问：Promise 如何解决回调地狱？**
> **答**：Promise 提供链式调用，将嵌套的回调转换为平级的 then 链，代码更清晰，错误处理更统一。

## 🌱 5.3 async/await

### 📖 async/await 基础

**async 函数**：
- 自动返回 Promise
- 内部可以使用 await
- 内部抛出错误会被 Promise 捕获

**await 表达式**：
- 等待 Promise 完成
- 只能在 async 函数中使用
- 会暂停 async 函数的执行，但不阻塞主线程

**代码示例**：
```javascript
// async 函数
async function fetchUser() {
    const response = await fetch('/api/user');
    const user = await response.json();
    return user;
}

// 使用
fetchUser()
    .then((user) => console.log(user))
    .catch((error) => console.error(error));

// async 函数返回 Promise
async function getData() {
    return 'data'; // 等价于 return Promise.resolve('data')
}

// await 等待 Promise
async function process() {
    const data = await getData();
    console.log(data); // 'data'
}
```

### 🔍 错误处理

**try-catch 处理错误**：
```javascript
async function fetchData() {
    try {
        const response = await fetch('/api/data');
        if (!response.ok) {
            throw new Error('Network response was not ok');
        }
        const data = await response.json();
        return data;
    } catch (error) {
        console.error('Error:', error);
        throw error; // 重新抛出或返回默认值
    }
}

// 多个 await 的错误处理
async function processMultiple() {
    try {
        const user = await fetchUser();
        const orders = await fetchOrders(user.id);
        const products = await fetchProducts(orders[0].id);
        return products;
    } catch (error) {
        console.error('Process failed:', error);
        throw error;
    }
}
```

**并行执行**：
```javascript
async function fetchAllData() {
    try {
        // 错误方式：串行执行
        // const user = await fetchUser();
        // const posts = await fetchPosts();
        // const comments = await fetchComments();
        
        // 正确方式：并行执行
        const [user, posts, comments] = await Promise.all([
            fetchUser(),
            fetchPosts(),
            fetchComments()
        ]);
        
        return { user, posts, comments };
    } catch (error) {
        console.error('Error:', error);
    }
}
```

### 📖 async/await 与 Promise 的关系

| 特性 | async/await | Promise |
|------|-------------|---------|
| **语法** | 同步写法，更易读 | 链式调用 |
| **错误处理** | try-catch | .catch() |
| **调试** | 更易调试，有堆栈信息 | 相对困难 |
| **本质** | Promise 的语法糖 | 基础机制 |
| **适用场景** | 顺序执行的异步操作 | 复杂的异步组合 |

**转换示例**：
```javascript
// Promise 写法
fetchUser()
    .then((user) => fetchOrders(user.id))
    .then((orders) => fetchProducts(orders[0].id))
    .then((products) => console.log(products))
    .catch((error) => console.error(error));

// async/await 写法
async function getProducts() {
    try {
        const user = await fetchUser();
        const orders = await fetchOrders(user.id);
        const products = await fetchProducts(orders[0].id);
        console.log(products);
    } catch (error) {
        console.error(error);
    }
}
```

## 🌱 5.4 事件循环与任务队列

### 📖 宏任务与微任务

| 任务类型 | 包含内容 | 优先级 | 执行时机 |
|----------|----------|--------|----------|
| **宏任务** | script、setTimeout、setInterval、I/O、UI渲染 | 低 | 每轮事件循环执行一个 |
| **微任务** | Promise.then/catch/finally、queueMicrotask、MutationObserver | 高 | 当前宏任务结束后立即执行 |

**执行顺序**：
1. 执行主线程同步代码
2. 执行所有微任务
3. 执行一个宏任务
4. 重复步骤2-3

**代码示例**：
```javascript
console.log('1'); // 同步

setTimeout(() => {
    console.log('2'); // 宏任务
}, 0);

Promise.resolve().then(() => {
    console.log('3'); // 微任务
});

console.log('4'); // 同步

// 输出：1 -> 4 -> 3 -> 2
```

### 🔍 复杂案例分析

```javascript
async function async1() {
    console.log('async1 start');
    await async2();
    console.log('async1 end');
}

async function async2() {
    console.log('async2');
}

console.log('script start');

setTimeout(() => {
    console.log('setTimeout');
}, 0);

async1();

new Promise((resolve) => {
    console.log('promise1');
    resolve();
}).then(() => {
    console.log('promise2');
});

console.log('script end');

// 输出：
// script start
// async1 start
// async2
// promise1
// script end
// async1 end
// promise2
// setTimeout
```

**解析**：
1. 同步代码：`script start` -> `async1 start` -> `async2` -> `promise1` -> `script end`
2. 微任务：`async1 end` -> `promise2`
3. 宏任务：`setTimeout`

### 💬 面试问答

> **问：宏任务和微任务的区别？**
> **答**：宏任务包括 script、setTimeout、setInterval、I/O 等，微任务包括 Promise.then、queueMicrotask 等。微任务优先级更高，在当前宏任务结束后立即执行，而宏任务需要等待下一轮事件循环。

> **问：async/await 的执行顺序？**
> **答**：await 后面的代码会被放入微任务队列，等待当前同步代码执行完毕后执行。

> **问：setTimeout(fn, 0) 什么时候执行？**
> **答**：setTimeout(fn, 0) 的回调会被放入宏任务队列，在当前同步代码和所有微任务执行完毕后执行。

## 🌱 6.4 Symbol 与迭代器

### 📖 Symbol

**Symbol**：表示唯一的标识符。

| 特性 | 说明 | 示例 |
|------|------|------|
| **唯一性** | 每个 Symbol 都是唯一的 | `Symbol('a') !== Symbol('a')` |
| **不可枚举** | 不会出现在 for...in 中 | - |
| **作为属性键** | 可以用作对象属性键 | `{ [Symbol('id')]: 1 }` |

**代码示例**：
```javascript
// 创建 Symbol
const id = Symbol('id');
const id2 = Symbol('id');
console.log(id === id2); // false

// 作为对象属性
const user = {
    name: '张三',
    [id]: 123
};

console.log(user[id]); // 123
console.log(Object.keys(user)); // ['name']
console.log(Reflect.ownKeys(user)); // ['name', Symbol(id)]

// 全局 Symbol
const globalId = Symbol.for('globalId');
const sameGlobalId = Symbol.for('globalId');
console.log(globalId === sameGlobalId); // true

// 内置 Symbol
const obj = {
    [Symbol.iterator]: function* () {
        yield 1;
        yield 2;
        yield 3;
    }
};
```

### 📖 迭代器与 for...of

**迭代器协议**：
- 对象具有 `Symbol.iterator` 属性
- 该属性返回一个迭代器对象
- 迭代器对象具有 `next()` 方法
- `next()` 返回 `{ value, done }`

**代码示例**：
```javascript
// 自定义迭代器
const range = {
    from: 1,
    to: 5,
    [Symbol.iterator]() {
        return {
            current: this.from,
            last: this.to,
            next() {
                if (this.current <= this.last) {
                    return { done: false, value: this.current++ };
                } else {
                    return { done: true };
                }
            }
        };
    }
};

for (const num of range) {
    console.log(num); // 1, 2, 3, 4, 5
}

// 生成器函数
function* generateId() {
    let id = 1;
    while (true) {
        yield id++;
    }
}

const gen = generateId();
console.log(gen.next().value); // 1
console.log(gen.next().value); // 2
console.log(gen.next().value); // 3
```

## 🌱 6.5 Map、Set、WeakMap、WeakSet

### 📖 Map 与 Set

| 数据结构 | 说明 | 特点 |
|----------|------|------|
| **Map** | 键值对集合 | 键可以是任意类型 |
| **Set** | 唯一值集合 | 值唯一，可用于去重 |
| **WeakMap** | 弱引用 Map | 键必须是对象，垃圾回收友好 |
| **WeakSet** | 弱引用 Set | 值必须是对象，垃圾回收友好 |

**代码示例**：
```javascript
// Map
const map = new Map();
map.set('name', '张三');
map.set(1, 'number key');
map.set({ id: 1 }, 'object key');

console.log(map.get('name')); // '张三'
console.log(map.has('name')); // true
console.log(map.size); // 3

// 遍历
for (const [key, value] of map) {
    console.log(key, value);
}

// Set
const set = new Set([1, 2, 2, 3, 3, 3]);
console.log(set); // Set { 1, 2, 3 }

set.add(4);
set.delete(1);
console.log(set.has(2)); // true

// 去重
const array = [1, 2, 2, 3, 3, 3];
const unique = [...new Set(array)];
console.log(unique); // [1, 2, 3]

// WeakMap
const weakMap = new WeakMap();
const obj = {};
weakMap.set(obj, 'value');
console.log(weakMap.get(obj)); // 'value'
// 当 obj 被垃圾回收，weakMap 中的条目自动删除
```

## 🌱 7.1 防抖与节流

### 📖 概念对比

| 特性 | 防抖（Debounce） | 节流（Throttle） |
|------|------------------|------------------|
| **核心思想** | 延迟执行，只执行最后一次 | 固定频率执行，限制执行次数 |
| **触发时机** | 停止触发后执行 | 按固定间隔执行 |
| **应用场景** | 搜索框输入、窗口调整、表单验证 | 滚动加载、按钮点击、鼠标移动 |
| **实现方式** | 清除定时器，重新计时 | 记录时间戳，判断是否到达间隔 |

### 🔍 防抖实现

**基础版防抖**：
```javascript
function debounce(fn, delay) {
    let timer = null;
    return function(...args) {
        const context = this;
        clearTimeout(timer);
        timer = setTimeout(() => {
            fn.apply(context, args);
        }, delay);
    };
}

// 使用
const debouncedSearch = debounce((query) => {
    console.log('搜索:', query);
    // 发送搜索请求
}, 300);

input.addEventListener('input', (e) => {
    debouncedSearch(e.target.value);
});
```

**立即执行版防抖**：
```javascript
function debounce(fn, delay, immediate = false) {
    let timer = null;
    return function(...args) {
        const context = this;
        const callNow = immediate && !timer;
        
        clearTimeout(timer);
        timer = setTimeout(() => {
            timer = null;
            if (!immediate) {
                fn.apply(context, args);
            }
        }, delay);
        
        if (callNow) {
            fn.apply(context, args);
        }
    };
}

// 立即执行
const debouncedClick = debounce(() => {
    console.log('点击处理');
}, 500, true);

button.addEventListener('click', debouncedClick);
```

**带取消功能的防抖**：
```javascript
function debounce(fn, delay) {
    let timer = null;
    
    const debounced = function(...args) {
        const context = this;
        clearTimeout(timer);
        timer = setTimeout(() => {
            fn.apply(context, args);
        }, delay);
    };
    
    // 取消方法
    debounced.cancel = function() {
        clearTimeout(timer);
        timer = null;
    };
    
    return debounced;
}

// 使用
const debouncedSave = debounce(saveData, 1000);

// 组件卸载时取消
window.addEventListener('beforeunload', () => {
    debouncedSave.cancel();
});
```

### 🔍 节流实现

**时间戳版节流**：
```javascript
function throttle(fn, delay) {
    let lastTime = 0;
    return function(...args) {
        const context = this;
        const now = Date.now();
        
        if (now - lastTime >= delay) {
            fn.apply(context, args);
            lastTime = now;
        }
    };
}

// 使用
const throttledScroll = throttle(() => {
    console.log('滚动位置:', window.scrollY);
}, 100);

window.addEventListener('scroll', throttledScroll);
```

**定时器版节流**：
```javascript
function throttle(fn, delay) {
    let timer = null;
    return function(...args) {
        const context = this;
        
        if (!timer) {
            timer = setTimeout(() => {
                fn.apply(context, args);
                timer = null;
            }, delay);
        }
    };
}
```

**双保险版节流**（时间戳 + 定时器）：
```javascript
function throttle(fn, delay) {
    let lastTime = 0;
    let timer = null;
    
    return function(...args) {
        const context = this;
        const now = Date.now();
        const remaining = delay - (now - lastTime);
        
        if (remaining <= 0) {
            // 超过间隔，立即执行
            if (timer) {
                clearTimeout(timer);
                timer = null;
            }
            fn.apply(context, args);
            lastTime = now;
        } else if (!timer) {
            // 设置定时器，保证最后一次触发也能执行
            timer = setTimeout(() => {
                fn.apply(context, args);
                lastTime = Date.now();
                timer = null;
            }, remaining);
        }
    };
}
```

### 💬 面试问答

> **问：防抖和节流的区别？**
> **答**：防抖是在事件停止触发后才执行，只执行最后一次；节流是固定频率执行，限制执行次数。防抖适用于搜索框输入，节流适用于滚动加载。

> **问：如何实现一个带立即执行和取消功能的防抖？**
> **答**：使用闭包保存timer，通过immediate参数控制是否立即执行，返回的函数添加cancel方法用于取消定时器。

## 🌱 7.2 深拷贝与浅拷贝

### 📖 概念对比

| 特性 | 浅拷贝 | 深拷贝 |
|------|--------|--------|
| **拷贝层级** | 只拷贝第一层 | 递归拷贝所有层级 |
| **引用类型** | 拷贝引用地址 | 创建新对象 |
| **性能** | 快 | 慢 |
| **内存占用** | 少 | 多 |
| **循环引用** | 无影响 | 需要处理 |

### 🔍 浅拷贝实现

```javascript

const shallowCopy1 = { ...obj };

// 方法2：Object.assign

// 方法3：Array.prototype.slice

// 手写浅拷贝
function shallowClone(obj) {
    if (obj === null || typeof obj !== 'object') {
        return obj;
    }
    
    const result = Array.isArray(obj) ? [] : {};
    for (const key in obj) {
        if (obj.hasOwnProperty(key)) {
            result[key] = obj[key];
        }
    }
    return result;
}
```

### 🔍 深拷贝实现

**基础版深拷贝**：
```javascript
function deepClone(obj) {
    if (obj === null || typeof obj !== 'object') {
        return obj;
    }
    
    if (obj instanceof Date) {
        return new Date(obj);
    }
    
    if (obj instanceof RegExp) {
        return new RegExp(obj);
    }
    
    if (Array.isArray(obj)) {
        return obj.map(item => deepClone(item));
    }
    
    const result = {};
    for (const key in obj) {
        if (obj.hasOwnProperty(key)) {
            result[key] = deepClone(obj[key]);
        }
    }
    return result;
}
```

**完整版深拷贝**（处理循环引用）：
```javascript
function deepClone(obj, map = new WeakMap()) {
    // 处理基本类型和null
    if (obj === null || typeof obj !== 'object') {
        return obj;
    }
    
    // 处理日期
    if (obj instanceof Date) {
        return new Date(obj);
    }
    
    // 处理正则
    if (obj instanceof RegExp) {
        return new RegExp(obj);
    }
    
    // 处理Map
    if (obj instanceof Map) {
        const result = new Map();
        map.set(obj, result);
        obj.forEach((value, key) => {
            result.set(deepClone(key, map), deepClone(value, map));
        });
        return result;
    }
    
    // 处理Set
    if (obj instanceof Set) {
        const result = new Set();
        map.set(obj, result);
        obj.forEach(value => {
            result.add(deepClone(value, map));
        });
        return result;
    }
    
    // 处理循环引用
    if (map.has(obj)) {
        return map.get(obj);
    }
    
    // 处理数组
    if (Array.isArray(obj)) {
        const result = [];
        map.set(obj, result);
        for (let i = 0; i < obj.length; i++) {
            result[i] = deepClone(obj[i], map);
        }
        return result;
    }
    
    // 处理对象
    const result = {};
    map.set(obj, result);
    
    // 拷贝Symbol属性
    const symKeys = Object.getOwnPropertySymbols(obj);
    for (const symKey of symKeys) {
        result[symKey] = deepClone(obj[symKey], map);
    }
    
    // 拷贝普通属性
    for (const key in obj) {
        if (obj.hasOwnProperty(key)) {
            result[key] = deepClone(obj[key], map);
        }
    }
    
    return result;
}

// 测试循环引用
const obj = { a: 1 };
obj.self = obj;
const cloned = deepClone(obj);
console.log(cloned.self === cloned); // true（循环引用正确处理）
```

**使用JSON的深拷贝**（简单场景）：
```javascript
function deepCloneJSON(obj) {
    return JSON.parse(JSON.stringify(obj));
}

// 缺点：
// 1. 无法处理函数
// 2. 无法处理undefined、Symbol
// 3. 无法处理循环引用
// 4. 无法处理Date（会变成字符串）
// 5. 无法处理RegExp、Map、Set等
```

### 💬 面试问答

> **问：浅拷贝和深拷贝的区别？**
> **答**：浅拷贝只拷贝第一层，引用类型拷贝的是地址；深拷贝递归拷贝所有层级，创建全新的对象。

> **问：如何实现一个处理循环引用的深拷贝？**
> **答**：使用WeakMap记录已拷贝的对象，遇到循环引用时直接返回已拷贝的对象。

```javascript
        for (let j = 0; j < len - 1 - i; j++) {

function quickSort(arr) {

    const left = arr.filter(x => x < pivot);
    const middle = arr.filter(x => x === pivot);
    const right = arr.filter(x => x > pivot);

}
```

## 🌱 7.4 Promise 手写实现

### 📖 基础版 Promise

```javascript
class MyPromise {
    constructor(executor) {
        this.state = 'pending';
        this.value = undefined;
        this.reason = undefined;
        this.onFulfilledCallbacks = [];
        this.onRejectedCallbacks = [];
        
        const resolve = (value) => {
            if (this.state === 'pending') {
                this.state = 'fulfilled';
                this.value = value;
                this.onFulfilledCallbacks.forEach(fn => fn());
            }
        };
        
        const reject = (reason) => {
            if (this.state === 'pending') {
                this.state = 'rejected';
                this.reason = reason;
                this.onRejectedCallbacks.forEach(fn => fn());
            }
        };
        
        try {
            executor(resolve, reject);
        } catch (error) {
            reject(error);
        }
    }
    
    then(onFulfilled, onRejected) {
        onFulfilled = typeof onFulfilled === 'function' ? onFulfilled : value => value;
        onRejected = typeof onRejected === 'function' ? onRejected : reason => { throw reason };
        
        const promise2 = new MyPromise((resolve, reject) => {
            if (this.state === 'fulfilled') {
                setTimeout(() => {
                    try {
                        const x = onFulfilled(this.value);
                        this.resolvePromise(promise2, x, resolve, reject);
                    } catch (error) {
                        reject(error);
                    }
                }, 0);
            } else if (this.state === 'rejected') {
                setTimeout(() => {
                    try {
                        const x = onRejected(this.reason);
                        this.resolvePromise(promise2, x, resolve, reject);
                    } catch (error) {
                        reject(error);
                    }
                }, 0);
            } else {
                this.onFulfilledCallbacks.push(() => {
                    setTimeout(() => {
                        try {
                            const x = onFulfilled(this.value);
                            this.resolvePromise(promise2, x, resolve, reject);
                        } catch (error) {
                            reject(error);
                        }
                    }, 0);
                });
                
                this.onRejectedCallbacks.push(() => {
                    setTimeout(() => {
                        try {
                            const x = onRejected(this.reason);
                            this.resolvePromise(promise2, x, resolve, reject);
                        } catch (error) {
                            reject(error);
                        }
                    }, 0);
                });
            }
        });
        
        return promise2;
    }
    
    resolvePromise(promise2, x, resolve, reject) {
        if (promise2 === x) {
            return reject(new TypeError('Chaining cycle detected for promise'));
        }
        
        if (x !== null && (typeof x === 'object' || typeof x === 'function')) {
            let called = false;
            try {
                const then = x.then;
                if (typeof then === 'function') {
                    then.call(x, y => {
                        if (called) return;
                        called = true;
                        this.resolvePromise(promise2, y, resolve, reject);
                    }, r => {
                        if (called) return;
                        called = true;
                        reject(r);
                    });
                } else {
                    resolve(x);
                }
            } catch (error) {
                if (called) return;
                called = true;
                reject(error);
            }
        } else {
            resolve(x);
        }
    }
    
    catch(onRejected) {
        return this.then(null, onRejected);
    }
    
    static resolve(value) {
        return new MyPromise((resolve) => resolve(value));
    }
    
    static reject(reason) {
        return new MyPromise((_, reject) => reject(reason));
    }
    
    static all(promises) {
        return new MyPromise((resolve, reject) => {
            const results = [];
            let count = 0;
            
            promises.forEach((promise, index) => {
                MyPromise.resolve(promise).then(value => {
                    results[index] = value;
                    count++;
                    if (count === promises.length) {
                        resolve(results);
                    }
                }, reject);
            });
        });
    }
}
```

```javascript
function jsonStringify(obj) {
    if (obj === null) return 'null';
    if (obj === undefined) return undefined;
    if (typeof obj === 'boolean') return String(obj);
    if (typeof obj === 'number') return String(obj);
    if (typeof obj === 'string') return `"${obj}"`;
    if (typeof obj === 'symbol') return undefined;
    if (typeof obj === 'function') return undefined;
    
    if (obj instanceof Date) {
        return `"${obj.toISOString()}"`;
    }
    
    if (Array.isArray(obj)) {
        const items = obj.map(item => jsonStringify(item) || 'null');
        return `[${items.join(',')}]`;
    }
    
    if (typeof obj === 'object') {
        const items = [];
        for (const key in obj) {
            if (obj.hasOwnProperty(key)) {
                const value = jsonStringify(obj[key]);
                if (value !== undefined) {
                    items.push(`"${key}":${value}`);
                }
            }
        }
        return `{${items.join(',')}}`;
    }
}
```

```javascript
function curry(fn) {
    return function curried(...args) {
        if (args.length >= fn.length) {
            return fn.apply(this, args);
        } else {
            return function(...args2) {
                return curried.apply(this, args.concat(args2));
            };
        }
    };
}
```
