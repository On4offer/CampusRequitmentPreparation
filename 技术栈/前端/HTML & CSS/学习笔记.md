## 🌱 1.1 HTML 简介

### 📖 核心概念

**HTML（HyperText Markup Language）** 是用于创建网页结构的标记语言，由一系列标签组成，描述页面的结构和内容。

**HTML发展历程**：

| 版本 | 发布年份 | 主要特性 |
|------|----------|----------|
| HTML 1.0 | 1991 | 基础标签 |
| HTML 2.0 | 1995 | 表单、表格 |
| HTML 3.2 | 1997 | 样式表、脚本支持 |
| HTML 4.01 | 1999 | 语义化、XHTML |
| HTML5 | 2014 | 语义化标签、Canvas、SVG、LocalStorage、Web Workers |

**HTML5核心特性**：
- **语义化标签**：新增 `<header>`、`<nav>`、`<main>`、`<article>`、`<section>`、`<aside>`、`<footer>` 等
- **多媒体支持**：原生 `<video>`、`<audio>` 标签
- **图形绘制**：`<canvas>`、`<svg>`
- **本地存储**：LocalStorage、SessionStorage
- **离线应用**：Service Workers、Manifest
- **表单增强**：新的输入类型、属性验证
- **Web Workers**：后台 JavaScript 线程
- **地理定位**：Geolocation API

**代码示例**：
```html
<!-- HTML5 多媒体示例 -->
<video controls width="400" poster="poster.jpg">
    <source src="video.mp4" type="video/mp4">
    <source src="video.webm" type="video/webm">
    您的浏览器不支持 video 标签
</video>

<audio controls>
    <source src="music.mp3" type="audio/mpeg">
    您的浏览器不支持 audio 标签
</audio>
```

## 🌱 1.2 HTML 文档结构

### 📖 文档组成

| 组成部分 | 说明 | 必需 |
|----------|------|------|
| `<!DOCTYPE html>` | 文档类型声明 | 是 |
| `<html>` | 根元素 | 是 |
| `<head>` | 头部信息 | 是 |
| `<body>` | 主体内容 | 是 |

**标准文档结构**：
```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <!-- 元信息区域 -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="页面描述">
    <meta name="keywords" content="关键词1,关键词2">
    <meta name="author" content="作者">
    <title>页面标题</title>
    
    <!-- 资源链接 -->
    <link rel="stylesheet" href="style.css">
    <link rel="icon" href="favicon.ico">
    <script src="app.js" defer></script>
</head>
<body>
    <!-- 页面内容 -->
    <header>
        <h1>网站标题</h1>
        <nav>
            <a href="/">首页</a>
            <a href="/about">关于</a>
        </nav>
    </header>
    
    <main>
        <article>
            <h2>文章标题</h2>
            <p>文章内容...</p>
        </article>
    </main>
    
    <footer>
        <p>&copy; 2024 公司名称</p>
    </footer>
</body>
</html>
```

### 🔍 head 标签详解

| 标签 | 作用 | 常见属性 |
|------|------|----------|
| `<meta charset>` | 字符编码 | UTF-8（推荐） |
| `<meta name="viewport">` | 视口设置 | width, initial-scale |
| `<meta name="description">` | 页面描述 | 用于SEO |
| `<meta name="keywords">` | 关键词 | 搜索引擎优化 |
| `<title>` | 页面标题 | 浏览器标签页显示 |
| `<link>` | 外部资源 | stylesheet, icon |
| `<script>` | JavaScript | src, defer, async |
| `<style>` | 内联样式 | - |

**viewport 移动端适配**：
```html
<!-- 基础视口设置 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- 禁止缩放 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">

<!-- 适配不同设备宽度 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0, minimum-scale=1.0, maximum-scale=5.0">
```

### 💬 面试问答

> **问：DOCTYPE 的作用是什么？**
> **答**：DOCTYPE 用于声明文档类型和DTD（文档类型定义），告诉浏览器按照哪种规范来解析文档。没有DOCTYPE声明会导致浏览器使用 Quirks Mode（兼容模式）渲染页面，可能出现样式兼容性问题。

> **问：meta 标签有哪些常见用途？**
> **答**：meta标签用于提供页面的元信息，包括字符编码、视口设置、SEO优化、页面描述、关键词、作者信息、刷新跳转、HTTP头信息等。

## 🌱 1.3 语义化标签

### 📖 语义化的意义

| 方面 | 说明 | 重要性 |
|------|------|--------|
| **代码可读性** | 标签语义清晰，便于开发者理解和维护 | 高 |
| **SEO 优化** | 搜索引擎能更好地理解页面结构 | 高 |
| **无障碍访问** | 屏幕阅读器能准确识别内容区域 | 高 |
| **团队协作** | 统一规范便于团队开发 | 中 |

### 🔍 常用语义化标签

| 标签 | 语义 | 使用场景 |
|------|------|----------|
| `<header>` | 页面或区块头部 | 顶部导航、标题区域 |
| `<nav>` | 导航链接 | 菜单、目录 |
| `<main>` | 主要内容 | 页面核心内容区域 |
| `<article>` | 独立内容 | 博客文章、新闻 |
| `<section>` | 内容区块 | 章节、主题分组 |
| `<aside>` | 侧边内容 | 相关文章、广告 |
| `<footer>` | 页面或区块底部 | 版权、联系方式 |
| `<figure>` | 媒体内容 | 图片、图表 |
| `<figcaption>` | 媒体说明 | 图片标题 |
| `<mark>` | 标记文本 | 搜索高亮 |
| `<time>` | 时间 | 日期时间 |
| `<progress>` | 进度条 | 加载进度 |
| `<meter>` | 度量衡 | 仪表盘 |

**代码示例**：
```html
<!-- 博客文章结构 -->
<article>
    <header>
        <h1>文章标题</h1>
        <p>
            <time datetime="2024-01-15">2024年1月15日</time>
            作者：张三
        </p>
    </header>
    
    <main>
        <p>文章正文内容...</p>
        
        <section>
            <h2>第一章</h2>
            <p>章节内容...</p>
        </section>
        
        <section>
            <h2>第二章</h2>
            <p>章节内容...</p>
        </section>
    </main>
    
    <aside>
        <h3>相关文章</h3>
        <ul>
            <li><a href="#">相关文章1</a></li>
            <li><a href="#">相关文章2</a></li>
        </ul>
    </aside>
    
    <footer>
        <p>Tags: <mark>JavaScript</mark>, <mark>前端</mark></p>
    </footer>
</article>
```

| 场景 | 推荐标签 | 不推荐 |

| 导航菜单 | `<nav>` | `<div class="nav">` |
| 主要内容 | `<main>` | `<div class="main">` |
| 文章内容 | `<article>` | `<div class="article">` |

### 📖 链接与媒体

**链接 `<a>` 标签属性**：

| 属性 | 说明 | 常用值 |
|------|------|--------|
| `href` | 链接地址 | URL、锚点 `#id` |
| `target` | 打开方式 | `_self`（默认）、`_blank`（新窗口） |

**图片 `<img>` 标签属性**：

| 属性 | 说明 | 重要性 |
|------|------|--------|
| `src` | 图片地址 | 必需 |
| `alt` | 替代文本 | 必需（无障碍） |
| `width/height` | 图片尺寸 | 推荐（防止布局抖动） |
| `loading` | 加载方式 | `eager`（默认）、`lazy`（懒加载） |
| `srcset` | 多分辨率 | 响应式图片 |
| `sizes` | 图片尺寸列表 | 响应式布局 |

```html
<!-- 基础用法 -->

<!-- 响应式图片 -->
<img 
    src="photo-400.jpg"
    srcset="photo-400.jpg 400w, photo-800.jpg 800w, photo-1200.jpg 1200w"
    sizes="(max-width: 600px) 400px, (max-width: 1000px) 800px, 1200px"
    alt="响应式图片"
>

<!-- 懒加载 -->
<img src="photo.jpg" loading="lazy" alt="懒加载图片">

<!-- 图片格式选择 -->

    <source srcset="photo.avif" type="image/avif">
    <source srcset="photo.webp" type="image/webp">

```

### 📖 数据属性（data-*）

**访问方式**：
```javascript
// 设置数据
element.setAttribute('data-user-id', '123');
element.dataset.userId = '123';

// 读取数据
element.getAttribute('data-user-id');
element.dataset.userId;

// 多个单词用驼峰
<div data-user-name="zhangsan">用户</div>
element.dataset.userName; // 'zhangsan'

// 删除数据
element.removeAttribute('data-user-id');
delete element.dataset.userId;
```

**实战应用**：
```html

<script>
    document.querySelector('.buy-btn').addEventListener('click', function() {
        const product = this.closest('.product');
        const id = product.dataset.id;
        const price = parseFloat(product.dataset.price);
        const stock = parseInt(product.dataset.stock);
        
        if (stock > 0) {
            console.log(`购买商品 ${id}，价格 ¥${price}`);
        }
    });
</script>
```

### 💬 面试问答

> **问：语义化标签有什么好处？**
> **答**：1. 代码可读性好，便于维护；2. SEO友好，搜索引擎能理解页面结构；3. 无障碍支持，屏幕阅读器能准确识别内容；4. 便于团队协作，统一规范。

> **问：如何选择 div 和语义化标签？**
> **答**：当元素有明确语义时，使用语义化标签（如header、nav、article）；仅作为样式或脚本容器时使用div。语义化标签更利于SEO和无障碍访问。

> **问：data-* 属性有什么用途？**
> **答**：1. 存储页面私有数据；2. 在HTML中绑定数据，避免AJAX请求；3. 用于前端组件数据传递；4. 结合JavaScript实现交互逻辑。

> **问：表单的 GET 和 POST 区别？**
> **答**：GET参数在URL中明文传输，有长度限制（2048字符），用于查询；POST参数在请求体中传输，更安全，无长度限制，用于提交数据。

## 🌱 2.1 CSS 简介

### 📖 核心概念

**CSS（Cascading Style Sheets）** 用于描述 HTML 元素的样式，包括布局、颜色、字体等。

**CSS发展历程**：

| 版本 | 发布年份 | 主要特性 |
|------|----------|----------|
| CSS 1.0 | 1996 | 基础样式、文本、颜色 |
| CSS 2.0 | 1998 | 定位、媒体类型、伪类 |
| CSS 2.1 | 2004 | 修复2.0问题，统一标准 |
| CSS 3.0 | 2011 | 模块化、圆角、阴影、动画、渐变 |

**CSS3 新特性**：

| 类别 | 新特性 |
|------|--------|
| **选择器** | 属性选择器增强、结构伪类、:not() |
| **盒子模型** | box-sizing、resize、outline |
| **背景与边框** | background-size、border-radius、box-shadow、border-image |
| **文本效果** | text-shadow、text-overflow、word-wrap |
| **2D/3D变换** | transform、transition |
| **动画** | @keyframes、animation |
| **布局** | Flexbox、Grid、多列布局 |
| **媒体查询** | @media、响应式设计 |

### 🔍 CSS 工作原理

```
┌─────────────────────────────────────────────────────────┐
│                      渲染过程                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   HTML                    CSS                           │
│     │                      │                            │
│     ▼                      ▼                            │
│  DOM 树    +    CSSOM 树  =  渲染树（Render Tree）     │
│     │                      │                            │
│     └──────────┬───────────┘                           │
│                ▼                                       │
│           布局（Layout）                                │
│                ▼                                       │
│           绘制（Paint）                                │
│                ▼                                       │
│           合成（Composite）                            │
│                ▼                                       │
│           显示页面                                      │
└─────────────────────────────────────────────────────────┘
```

**代码示例**：
```html
<!-- HTML 结构 -->
<div class="container">
    <h1 class="title">标题</h1>
    <p class="content">内容</p>
</div>

<!-- CSS 样式 -->
.container {
    width: 100%;
    padding: 20px;
}

.title {
    font-size: 24px;
    color: #333;
}

.content {
    font-size: 16px;
    line-height: 1.6;
}
```

### 💬 面试问答

> **问：CSS 的工作原理是什么？**
> **答**：浏览器解析HTML生成DOM树，解析CSS生成CSSOM树，两者结合生成渲染树，然后进行布局和绘制，最终显示页面。这个过程包括DOM树构建、CSSOM树构建、渲染树生成、布局、绘制、合成等步骤。

> **问：CSS 和 HTML 的关系？**
> **答**：HTML定义页面结构，CSS定义页面样式。HTML提供语义化的标记，CSS通过选择器作用于这些标记，实现样式控制。两者配合实现结构与表现分离。

## 🌱 2.2 CSS 引入方式

### 📖 四种引入方式对比

| 方式 | 语法 | 优先级 | 优点 | 缺点 |
|------|------|--------|------|------|
| **内联样式** | `style="..."` | 最高 | 精确控制 | 代码冗余、难维护 |
| **内部样式** | `<style>...</style>` | 中 | 页面内复用 | 多页面无法复用 |
| **外部样式** | `<link rel="stylesheet">` | 最低 | 复用、缓存 | 需要HTTP请求 |
| **@import** | `@import url()` | 最低 | 模块化 | 性能较差 |

**代码示例**：
```html
<!-- 1. 内联样式：直接写在元素上 -->
<div style="color: red; font-size: 16px;">内联样式</div>

<!-- 2. 内部样式：写在 head 中 -->
<head>
    <style>
        .container {
            width: 100%;
            margin: 0 auto;
        }
        .title {
            font-size: 24px;
        }
    </style>
</head>

<!-- 3. 外部样式：引入 CSS 文件 -->
<head>
    <link rel="stylesheet" href="styles/main.css">
    <link rel="stylesheet" href="styles/grid.css">
    <!-- 媒体查询适配 -->
    <link rel="stylesheet" href="styles/mobile.css" media="screen and (max-width: 768px)">
</head>

<!-- 4. @import：在 CSS 中引入 -->
<style>
    @import url('base.css');
    @import url('theme.css');
    
    /* 必须在其他规则之前 */
    body {
        margin: 0;
    }
</style>
```

### 📖 link 与 @import 的区别

| 特性 | `<link>` | `@import` |
|------|----------|-----------|
| 加载时机 | HTML 解析时同步加载 | CSS 解析时加载 |
| 并行请求 | 支持 | 不支持（串行） |
| JS 控制 | 可通过 DOM 操作 | 无法控制 |
| 浏览器支持 | 全部支持 | IE5 以下不支持 |
| 权重 | 更高 | 更低 |

```html
<!-- 推荐：使用 link 标签 -->
<link rel="stylesheet" href="style.css">

<!-- 不推荐：@import 嵌套会导致请求阻塞 -->
<style>
    @import url('style.css');
</style>
```

### 📖 CSS 书写位置建议

```html
<!-- 现代最佳实践 -->
<head>
    <!-- 预加载关键 CSS -->
    <link rel="preload" href="styles/main.css" as="style">
    
    <!-- 主样式表 -->
    <link rel="stylesheet" href="styles/main.css">
    
    <!-- 延迟加载非关键样式 -->
    <link rel="stylesheet" href="styles/print.css" media="print">
</head>

<body>
    <!-- 内容 -->
    
    <!-- 脚本（推荐放在 body 末尾） -->
    <script src="app.js" defer></script>
</body>
```

## 🌱 2.3 选择器详解

### 📖 基础选择器

| 选择器 | 语法 | 权重 | 示例 |

| **元素选择器** | `tagname` | 1 | `p { }` |
| **ID 选择器** | `#id` | 100 | `#header { }` |
| **类选择器** | `.classname` | 10 | `.container { }` |
| **通配符选择器** | `*` | 0 | `* { }` |
| **属性选择器** | `[attr]` | 10 | `[type="text"] { }` |

```css
/* ID 选择器 */
```

### 📖 组合选择器

| 选择器 | 语法 | 说明 | 示例 |
|--------|------|------|------|
| **后代选择器** | `A B` | 选择 A 内部的所有 B | `.container p` |
| **子选择器** | `A > B` | 选择 A 的直接子元素 B | `.nav > li` |
| **相邻兄弟** | `A + B` | 选择 A 后面紧邻的 B | `h1 + p` |
| **通用兄弟** | `A ~ B` | 选择 A 后面的所有 B | `.title ~ p` |

**代码示例**：
```css
/* 后代选择器：所有后代 */
.article p {
    margin-bottom: 16px;
}

/* 子选择器：直接子元素 */
.menu > li {
    display: inline-block;
}

/* 相邻兄弟：紧邻的下一个 */
h1 + .subtitle {
    margin-top: -10px;
}

/* 通用兄弟：后面的所有 */
.title ~ .content {
    color: #666;
}
```

### 📖 属性选择器

| 选择器 | 说明 | 示例 |
|--------|------|------|
| `[attr]` | 具有该属性 | `[disabled]` |
| `[attr="value"]` | 属性值等于 | `[type="text"]` |
| `[attr~="value"]` | 属性值包含（空格分隔） | `[class~="active"]` |
| `[attr\|="value"]` | 属性值等于或开头 | `[lang\|="zh"]` |
| `[attr^="value"]` | 属性值开头 | `[href^="https"]` |
| `[attr$="value"]` | 属性值结尾 | `[src$=".png"]` |
| `[attr*="value"]` | 属性值包含 | `[class*="btn"]` |

**代码示例**：
```css

    opacity: 0.5;
    cursor: not-allowed;
}

/* 文本输入框 */
[type="text"] {
    width: 100%;
}

/* HTTPS 链接 */

}

/* PDF 下载链接 */

/* 包含 btn 的类名 */

}
```

### 📖 选择器优先级计算

**优先级权重**：

| 选择器 | 权重 |
|--------|------|
| `!important` | 最高（∞） |
| 内联样式 | 1000 |
| ID 选择器 | 100 |
| 类选择器、属性选择器、伪类 | 10 |
| 元素选择器、伪元素 | 1 |
| 通配符、组合符、否定伪类 | 0 |

**计算规则**：
```
优先级 = (ID数量, 类数量, 元素数量)

例如：
- #header        = (1, 0, 0) = 100
- .nav .link     = (0, 2, 0) = 20  
- div#header     = (1, 0, 1) = 101
- .nav a:hover   = (0, 2, 1) = 21
- #nav ul li.active = (1, 1, 2) = 112
```

**实战计算**：
```css
/* 权重: 1, 0, 0 */
#header { }

/* 权重: 0, 1, 0 */
.nav { }

/* 权重: 0, 1, 1 */
.nav li { }

/* 权重: 0, 2, 1 */
.nav li.active { }

/* 权重: 0, 1, 1 */
.nav li:first-child { }

/* 权重: 1, 0, 1 */
#header .logo { }
```

### 💬 面试问答

> **问：CSS 选择器的优先级如何计算？**
> **答**：优先级由 ID 数量、类/属性/伪类数量、元素/伪元素数量决定。计算方式： specificity = (id, class, element)。内联样式权重最高，其次是 ID 选择器，然后是类/属性/伪类，最后是元素/伪元素。!important 最高优先级。

> **问：有哪些方式可以提高 CSS 选择器的优先级？**
> **答**：1. 增加选择器 specificity（添加更多类、标签）；2. 使用 ID 选择器；3. 使用内联样式（不推荐）；4. 使用 !important（谨慎使用）。最佳实践是避免使用 !important，通过合理的 CSS 架构解决优先级问题。

> **问：:nth-child 和 :nth-of-type 的区别？**
> **答**：:nth-child 选择第 n 个子元素（无论类型）；:nth-of-type 选择同类型中的第 n 个元素。例如 `div:nth-child(2)` 选择作为第二个子元素的 div，而 `div:nth-of-type(2)` 选择第二个 div 元素。

> **问：伪类和伪元素的区别？**
> **答**：伪类表示元素的状态（如:hover、:focus），伪元素表示元素的特定部分（如::before、::first-line）。伪类用单冒号，伪元素用双冒号（CSS3 规范）。

## 🌱 2.4 样式优先级与层叠规则

### 📖 层叠规则

**层叠顺序**（从高到低）：
1. 重要声明（`!important`）
2. 内联样式（`style=""`）
3. ID 选择器
4. 类选择器、属性选择器、伪类
5. 元素选择器、伪元素
6. 通配符、继承样式

**同优先级规则**：
```css
/* 后定义的优先级更高 */
p { color: red; }    /* 被覆盖 */
p { color: blue; }   /* 最终生效 */
```

### 📖 继承机制

**可继承属性**：

| 类别 | 属性 |
|------|------|
| 字体 | font-family、font-size、font-weight、font-style |
| 文本 | line-height、color、text-align、text-indent |
| 列表 | list-style、list-style-type |
| 其他 | visibility、opacity、cursor |

**不可继承属性**：

| 类别 | 属性 |
|------|------|
| 布局 | width、height、margin、padding |
| 边框 | border、border-radius |
| 背景 | background、background-color |
| 定位 | position、float、display |

**强制继承**：
```css
/* 使用 inherit 强制继承 */
.child {
    color: inherit;        /* 继承父元素颜色 */
    border: inherit;       /* 继承父元素边框 */
}

/* 使用 initial 重置为默认值 */
.reset {
    color: initial;        /* 重置为黑色 */
    all: initial;          /* 重置所有属性 */
}

/* 使用 unset 组合 */
.combined {
    color: unset;          /* 可继承则继承，否则重置 */
}
```

### 📖 样式优先级实战

**场景：覆盖第三方库样式**：
```css
/* 原始样式（第三方库） */
.btn {
    padding: 8px 16px;
    background: #007bff;
}

/* 方案1：增加选择器 specificity */
.navbar .btn {
    padding: 10px 20px;
}

/* 方案2：使用 ID */
#header .btn {
    padding: 10px 20px;
}

/* 方案3：使用 !important（不推荐） */
.btn {
    padding: 10px 20px !important;
}
```

**最佳实践**：
```css
/* 推荐：使用 BEM 命名或模块化 CSS */

/* 模块名 + 元素名 + 状态 */
.button {
    /* 基础样式 */
}

.button--primary {
    /* 变体样式 */
    background: #007bff;
}

.button:hover {
    /* 状态样式 */
    background: #0056b3;
}
```

### 💬 面试问答

> **问：CSS 层叠规则是什么？**
> **答**：当多条规则作用于同一元素时，按优先级顺序决定生效的规则。优先级：!important > 内联样式 > ID > 类/属性/伪类 > 元素/伪元素。同优先级时，后定义的规则生效。

> **问：哪些 CSS 属性可以继承？**
> **答**：主要是非布局相关属性：字体属性（font-*）、文本属性（color、line-height、text-*）、列表属性（list-style-*）、光标（cursor）、可见性（visibility）等。布局相关属性（width、height、margin、padding、border）不可继承。

> **问：如何覆盖 !important 声明？**
> **答**：可以添加更多的 !important 或使用更高优先级的选择器。实际上应该尽量避免使用 !important，通过合理的 CSS 架构和 specificity 管理来解决样式冲突。

## 🌱 3.1 盒模型深度解析

### 📖 盒模型基础概念

**盒模型（Box Model）** 是CSS布局的基础核心概念。在CSS中，每个HTML元素都被抽象为一个矩形盒子，这个盒子由**内容区域（Content）**、**内边距（Padding）**、**边框（Border）**和**外边距（Margin）**四个部分组成。理解盒模型的工作原理对于精确控制元素尺寸、实现精确布局至关重要。

| 组成部分 | 说明 | 影响元素尺寸 |
|----------|------|--------------|
| **内容区（Content）** | 元素的实际内容，如文本、图片 | 是（width/height默认值） |
| **内边距（Padding）** | 内容与边框之间的空白区域 | 是（box-sizing: content-box） |
| **边框（Border）** | 围绕内容和内边距的边界 | 是（box-sizing: content-box） |
| **外边距（Margin）** | 元素外部与其他元素之间的空间 | 否（不计入元素尺寸） |

### 🔍 两种盒模型的区别

CSS中存在两种盒模型，它们的核心区别在于**width属性含义的不同**：

| 盒模型类型 | box-sizing值 | width 含义 | 实际占宽 |
|------------|--------------|------------|----------|
| **标准盒模型** | content-box | 仅内容区宽度 | width + padding + border |
| **IE盒模型（替代盒模型）** | border-box | 内容 + 内边距 + 边框 | width（包含padding和border） |

**代码示例**：
```css
/* 标准盒模型（默认） */
.content-box {
    box-sizing: content-box;
    width: 200px;
    padding: 20px;
    border: 10px solid #333;
    /* 实际占宽：200 + 20*2 + 10*2 = 260px */
}

/* IE盒模型（推荐） */
.border-box {
    box-sizing: border-box;
    width: 200px;
    padding: 20px;
    border: 10px solid #333;
    /* 实际占宽：200px（内容区自动调整为 200-20*2-10*2 = 140px） */
}
```

**面试问答**：

> **问：如何理解标准盒模型和IE盒模型的区别？**
> **答**：标准盒模型中，width和height属性只包含内容区域的尺寸，不包括内边距、边框和外边距。而IE盒模型（也称替代盒模型）中，width和height属性包含了内容、内边距和边框的总体尺寸。在实际开发中，由于IE盒模型更符合直觉（设置多少就是占多大空间），推荐使用 `box-sizing: border-box` 并全局应用。

> **问：为什么推荐使用 `box-sizing: border-box`？**
> **答**：主要有两个原因：一是计算更直观，设置的width就是元素实际占用的宽度，无需手动计算padding和border的影响；二是配合百分比宽度和padding实现流式布局时更加方便，例如设置 width: 50% + padding: 20px 不会导致元素溢出容器。

### 📖 盒模型尺寸计算

在实际布局中，精确计算元素的尺寸非常重要。以下是详细的计算公式：

**标准盒模型总尺寸计算**：
```css
/* 水平方向 */
总宽度 = width + padding-left + padding-right + border-left + border-right + margin-left + margin-right

/* 垂直方向 */
总高度 = height + padding-top + padding-bottom + border-top + border-bottom + margin-top + margin-bottom
```

**IE盒模型总尺寸计算**：
```css
/* 水平方向 */
总宽度 = width（已包含padding和border） + margin-left + margin-right

/* 垂直方向 */
总高度 = height（已包含padding和border） + margin-top + margin-bottom
```

**代码示例**：
```css
.box {
    /* 尺寸设置 */
    width: 200px;
    height: 100px;
    
    /* 内边距 */
    padding: 20px 30px;  /* 上下20px，左右30px */
    
    /* 边框 */
    border: 5px solid #007bff;
    
    /* 外边距 */
    margin: 15px 20px;
}

/* 标准盒模型下：
   内容区：200x100px
   总宽度：200 + 30*2 + 5*2 + 20*2 = 310px
   总高度：100 + 20*2 + 5*2 + 15*2 = 175px
*/
```

### 📖 外边距折叠现象

**外边距折叠（Margin Collapsing）** 是CSS布局中一个重要但容易误解的概念，指的是相邻元素的外边距会合并为较大值而非叠加。

| 折叠类型 | 场景 | 折叠结果 |
|----------|------|----------|
| **垂直兄弟折叠** | 两个块级元素上下相邻 | 取较大外边距值 |
| **父子外边距折叠** | 父元素与第一个/最后一个子元素 | 子元素外边距"溢出"到父元素外部 |
| **空元素折叠** | 元素的上下外边距相遇 | 取较大值 |

**代码示例**：
```css
/* 垂直兄弟外边距折叠 */
.box1 {
    margin-bottom: 20px;
}
.box2 {
    margin-top: 30px;
}
/* 实际间距 = max(20px, 30px) = 30px，而非 50px */

/* 父子外边距折叠 */
.parent {
    margin-top: 20px;
    /* 解决方式1：添加border-top */
    /* 解决方式2：添加padding-top */
    /* 解决方式3：overflow: hidden（创建BFC） */
}
.child {
    margin-top: 30px;
}
/* 实际父元素顶部外边距 = max(20px, 30px) = 30px */
```

**面试问答**：

> **问：如何解决父子外边距折叠问题？**
> **答**：有几种常用解决方案：一是在父元素上添加 `border-top` 或 `padding-top`，这样会创建新的边距边界；二是在父元素上使用 `overflow: hidden` 或其他值（除了visible），这会创建新的块级格式化上下文（BFC）；三是使用Flexbox或Grid布局，这些布局方式不会产生外边距折叠。

### 📖 最佳实践建议

| 场景 | 推荐做法 | 说明 |
|------|----------|------|
| 全局盒模型 | `* { box-sizing: border-box; }` | 统一使用IE盒模型 |
| 表单元素 | 继承盒模型 | `input, textarea { box-sizing: inherit; }` |
| 外边距控制 | 统一使用 margin-bottom | 便于维护，避免上下外边距混淆 |
| 负外边距 | 谨慎使用 | 可能导致布局错位和覆盖问题 |
| 外边距折叠 | 理解并利用 | 可用于简单的间距实现 |

## 🌱 3.2 传统布局技术

### 📖 浮动布局（Float）

**浮动布局** 是CSS早期最常用的布局方式之一，通过 `float` 属性使元素脱离文档流并向左或向右移动。

| 属性值 | 说明 | 特点 |
|--------|------|------|
| `left` | 左浮动 | 元素靠左，后续内容环绕右侧 |
| `right` | 右浮动 | 元素靠右，后续内容环绕左侧 |
| `none` | 不浮动 | 默认值，元素正常文档流 |

**浮动的工作原理**：
1. 元素浮动后会脱离文档流
2. 浮动元素会尽可能地向左或向右靠拢
3. 后续未浮动内容会环绕浮动元素
4. 父元素高度塌陷（无法包含浮动子元素）

**代码示例**：
```css
/* 左浮动 */
.float-left {
    float: left;
    width: 200px;
    margin-right: 20px;
}

/* 右浮动 */
.float-right {
    float: right;
    width: 200px;
    margin-left: 20px;
}

/* 清除浮动 - 父元素使用 */
.clearfix::after {
    content: "";
    display: block;
    clear: both;
}

/* 清除特定方向的浮动 */
.clear-left { clear: left; }
.clear-right { clear: right; }
.clear-both { clear: both; }
```

**双栏布局实现**：
```html
<div class="container clearfix">
    <div class="sidebar">
        <h3>侧边栏</h3>
        <p>侧边栏内容</p>
    </div>
    <div class="content">
        <h3>主内容</h3>
        <p>主内容区域，会环绕侧边栏显示</p>
    </div>
</div>

<style>
.container {
    width: 100%;
}
.sidebar {
    float: left;
    width: 250px;
    background: #f5f5f5;
    padding: 20px;
}
.content {
    margin-left: 270px;  /* 留出侧边栏宽度+间距 */
    padding: 20px;
}
</style>
```

### 📖 定位布局（Position）

**定位布局** 通过 `position` 属性控制元素在文档中的位置，是实现特殊效果（如固定导航、弹出层、绝对定位元素）的核心技术。

| 属性值 | 说明 | 定位参考 |
|--------|------|----------|
| `static` | 静态定位 | 默认值，正常文档流 |
| `relative` | 相对定位 | 相对于自身原始位置 |
| `absolute` | 绝对定位 | 相对于最近非static定位祖先 |
| `fixed` | 固定定位 | 相对于浏览器视口 |
| `sticky` | 粘性定位 | 相对于最近滚动祖先和视口 |

**代码示例**：
```css
/* 相对定位 - 元素占据原空间，可用于微调 */
.relative-box {
    position: relative;
    top: 10px;
    left: 20px;
    /* 仍占据原文档流位置，新位置是原位置+偏移 */
}

/* 绝对定位 - 完全脱离文档流 */
.absolute-box {
    position: absolute;
    top: 0;
    right: 0;
    /* 相对于最近position非static的祖先元素定位 */
}

/* 固定定位 - 相对于视口固定 */
.fixed-header {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
    height: 60px;
    /* 滚动页面时元素位置不变，常用于固定导航 */
}

/* 粘性定位 - 混合定位模式 */
.sticky-nav {
    position: sticky;
    top: 0;
    /* 未达到阈值前正常滚动，超过阈值后固定 */
}
```

**居中布局技巧**：
```css
/* 绝对定位居中 - 已知尺寸 */
.center-box {
    position: absolute;
    top: 50%;
    left: 50%;
    width: 300px;
    height: 200px;
    margin-top: -100px;  /* 高度的一半 */
    margin-left: -150px;  /* 宽度的一半 */
}

/* 绝对定位居中 - 未知尺寸（推荐） */
.center-box-flex {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
}
```

**面试问答**：

> **问：absolute定位是相对于哪个元素？**
> **答**：absolute定位是相对于最近的非static定位祖先元素（即position值为relative、absolute、fixed或sticky的祖先元素）。如果没有任何非static定位的祖先，则相对于初始包含块（通常是浏览器视口）。这与fixed定位（始终相对于视口）不同。

> **问：position: relative和position: absolute的主要区别是什么？**
> **答**：主要区别有两点：一是定位参考不同，relative相对于自身原始位置，absolute相对于最近非static祖先；二是是否脱离文档流，relative仍占据原始文档流空间，absolute完全脱离文档流不占据空间。

### 📖 显示属性（Display）

**display属性** 决定了元素的显示类型，是控制布局的最基本属性之一。

| 属性值 | 类型 | 特点 |
|--------|------|------|
| `block` | 块级 | 独占一行，可设置宽高 |
| `inline` | 行内 | 不独占一行，不可设置宽高 |
| `inline-block` | 行内块 | 不独占一行，可设置宽高 |
| `none` | 无 | 完全隐藏，不占据空间 |
| `flex` | 弹性盒 | 创建flex容器 |
| `grid` | 网格 | 创建网格容器 |

**代码示例**：
```css
/* 块级元素 */
.block {
    display: block;
    width: 100%;
    height: 100px;
    background: #e0e0e0;
}

/* 行内元素 */
.inline {
    display: inline;
    /* 不能设置宽高，外边距水平有效 */
}

/* 行内块元素 - 常用于按钮、图标 */
.inline-block {
    display: inline-block;
    width: 100px;
    height: 50px;
    /* 兼具行内和块级特性 */
}

/* 隐藏元素 */
.hidden {
    display: none;
    /* 完全不占据空间 */
}
.hidden-visibility {
    visibility: hidden;
    /* 占据空间但不可见 */
}
```

## 🌱 3.3 Flexbox 弹性布局

### 📖 Flexbox 核心概念

**Flexbox（Flexible Box）** 即弹性盒子布局，是CSS3引入的一维布局模型，专门为解决传统布局中的各种痛点而设计。Flexbox能够简便、完整、响应式地实现各种页面布局，特别适合处理一维方向上的元素排列（行或列）。

| 概念 | 说明 |
|------|------|
| **容器（Flex Container）** | 设置 `display: flex` 或 `display: inline-flex` 的元素 |
| **项目（Flex Item）** | 容器的直接子元素 |
| **主轴（Main Axis）** | 项目排列的主方向（默认水平从左到右） |
| **交叉轴（Cross Axis）** | 垂直于主轴的方向（默认垂直从上到下） |

### 🔍 容器属性详解

**1. flex-direction - 主轴方向**：

| 属性值 | 说明 | 主轴方向 |
|--------|------|----------|
| `row` | 行（默认） | 从左到右 |
| `row-reverse` | 行反向 | 从右到左 |
| `column` | 列 | 从上到下 |
| `column-reverse` | 列反向 | 从下到上 |

```css
.container {
    display: flex;
    flex-direction: row;        /* 默认：水平从左到右 */
    flex-direction: row-reverse; /* 水平从右到左 */
    flex-direction: column;     /* 垂直从上到下 */
    flex-direction: column-reverse; /* 垂直从下到上 */
}
```

**2. justify-content - 主轴对齐方式**：

| 属性值 | 说明 | 效果示例 |
|--------|------|----------|
| `flex-start` | 起始端对齐 | ████░░░░ |
| `flex-end` | 结束端对齐 | ░░░░████ |
| `center` | 居中对齐 | ░░████░░ |
| `space-between` | 两端对齐 | ███░░░██ |
| `space-around` | 环绕间距 | ░██░░░██░ |
| `space-evenly` | 等距分布 | ░█░░█░░█░ |

```css
.container {
    /* 水平居中 */
    justify-content: center;
    
    /* 两端对齐 */
    justify-content: space-between;
    
    /* 等间距 */
    justify-content: space-evenly;
}
```

**3. align-items - 交叉轴对齐方式**：

| 属性值 | 说明 | 效果示例 |
|--------|------|----------|
| `stretch` | 拉伸填满（默认） | ███<br>███ |
| `flex-start` | 起始端对齐 | ███<br>░░░ |
| `flex-end` | 结束端对齐 | ░░░<br>███ |
| `center` | 居中对齐 | ░░░<br>███<br>░░░ |
| `baseline` | 基线对齐 | Abc<br>xxx |

```css
.container {
    /* 垂直居中 */
    align-items: center;
    
    /* 拉伸填满（默认） */
    align-items: stretch;
}
```

**4. flex-wrap - 换行控制**：

| 属性值 | 说明 |
|--------|------|
| `nowrap` | 不换行（默认），项目会收缩 |
| `wrap` | 换行 |
| `wrap-reverse` | 逆向换行 |

**5. align-content - 多行对齐**（仅在flex-wrap: wrap时有效）：

| 属性值 | 说明 |
|--------|------|
| `flex-start` | 整体靠上 |
| `flex-end` | 整体靠下 |
| `center` | 整体居中 |
| `space-between` | 行间等距 |
| `space-around` | 行周围等距 |
| `stretch` | 拉伸填满 |

### 🔍 项目属性详解

| 属性 | 说明 | 常用值 |
|------|------|--------|
| `flex-grow` | 放大比例 | 数值（默认0） |
| `flex-shrink` | 缩小比例 | 数值（默认1） |
| `flex-basis` | 初始尺寸 | 长度值或auto |
| `flex` | 简写属性 | grow shrink basis |
| `align-self` | 自身对齐 | 同align-items |
| `order` | 排列顺序 | 数值（默认0） |

**代码示例**：
```css
/* 基础等宽布局 */
.item {
    flex: 1;
    /* flex: 1 1 0% 的简写 */
}

/* 固定宽度 + 弹性宽度 */
.sidebar {
    flex: 0 0 200px;  /* 不放大、不缩小、固定200px */
}
.content {
    flex: 1;  /* 占据剩余空间 */
}

/* 单独对齐方式 */
.item-self {
    align-self: center;
}

/* 排列顺序 */
.item-order {
    order: -1;  /* 数字越小越靠前 */
}
```

### 📖 常见布局示例

**垂直水平居中**：
```css
.parent {
    display: flex;
    justify-content: center;
    align-items: center;
    height: 100vh;
}
```

**导航菜单**：
```css
.nav {
    display: flex;
    justify-content: space-between;
    align-items: center;
    height: 60px;
    padding: 0 20px;
}

.nav-items {
    display: flex;
    gap: 20px;  /* 项目间距 */
}
```

**Sticky Footer布局**：
```css
.wrapper {
    display: flex;
    flex-direction: column;
    min-height: 100vh;
}

.header {
    flex: 0 0 auto;
}

.main {
    flex: 1 1 auto;
}

.footer {
    flex: 0 0 auto;
}
```

**面试问答**：

> **问：flex: 1 表示什么？**
> **答**：`flex: 1` 是 `flex: 1 1 0%` 的简写，包含三个值：flex-grow（放大比例）为1表示项目会平均分配剩余空间；flex-shrink（缩小比例）为1表示空间不足时项目会缩小；flex-basis为0%表示项目的基础尺寸为0（所有空间都可分配）。

> **问：Flexbox和Grid有什么区别？**
> **答**：Flexbox是一维布局，只能处理一个方向（行或列）的布局；Grid是二维布局，可以同时处理行和列的布局。简单来说，Flexbox适合处理一排或一列的元素（如导航菜单、列表），Grid适合处理需要行列对齐的复杂布局（如整个页面布局、相册）。

## 🌱 3.4 Grid 网格布局

### 📖 Grid 核心概念

**CSS Grid Layout** 是CSS3引入的二维网格布局系统，可以同时控制行和列，实现复杂的页面布局。Grid布局将容器划分为行和列，形成网格单元，项目可以放置在指定的网格位置。

| 概念 | 说明 |
|------|------|
| **网格容器（Grid Container）** | 设置 `display: grid` 的元素 |
| **网格项目（Grid Item）** | 容器的直接子元素 |
| **网格线（Grid Line）** | 网格的分隔线 |
| **网格轨道（Grid Track）** | 相邻网格线之间的区域 |
| **网格单元格（Grid Cell）** | 行和列交叉的区域 |
| **网格区域（Grid Area）** | 由多个单元格组成的区域 |

### 🔍 容器属性详解

**1. grid-template-columns / grid-template-rows - 轨道尺寸**：

```css
.container {
    /* 固定宽度 */
    grid-template-columns: 200px 200px 200px;
    
    /* 比例分配 */
    grid-template-columns: 1fr 2fr 1fr;
    
    /* 混合使用 */
    grid-template-columns: 100px 1fr 100px;
    
    /* 重复写法 */
    grid-template-columns: repeat(3, 1fr);
    
    /* 自动填充 - 响应式布局常用 */
    grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
}
```

| 单位 | 说明 | 示例 |
|------|------|------|
| `px` | 固定像素 | 100px |
| `%` | 百分比 | 33.33% |
| `fr` | 比例单位 | 1fr, 2fr |
| `auto` | 自动计算 | auto |
| `min-content` | 最小内容 | min-content |
| `max-content` | 最大内容 | max-content |
| `minmax(min, max)` | 范围函数 | minmax(100px, 1fr) |

**2. grid-gap / gap - 间距**：

```css
.container {
    /* 行间距 */
    row-gap: 20px;
    
    /* 列间距 */
    column-gap: 20px;
    
    /* 简写 */
    gap: 20px;
    gap: 10px 20px;  /* 行间距10px，列间距20px */
}
```

**3. grid-template-areas - 命名区域**：

```css
.container {
    display: grid;
    grid-template-columns: 200px 1fr 200px;
    grid-template-rows: auto 1fr auto;
    grid-template-areas: 
        "header header header"
        "sidebar main aside"
        "footer footer footer";
}

/* 项目指定区域 */
.header { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main { grid-area: main; }
.aside { grid-area: aside; }
.footer { grid-area: footer; }
```

**4. 对齐属性**：

| 属性 | 作用 | 属性值 |
|------|------|--------|
| `justify-items` | 单元格内容水平对齐 | start, end, center, stretch |
| `align-items` | 单元格内容垂直对齐 | start, end, center, stretch |
| `justify-content` | 整个网格水平对齐 | start, end, center, stretch, space-between |
| `align-content` | 整个网格垂直对齐 | start, end, center, stretch, space-between |

### 🔍 项目属性详解

| 属性 | 说明 | 示例 |
|------|------|------|
| `grid-column-start` | 列起始网格线 | 1, 2, 3 |
| `grid-column-end` | 列结束网格线 | 1, 2, 3, span 2 |
| `grid-row-start` | 行起始网格线 | 1, 2, 3 |
| `grid-row-end` | 行结束网格线 | 1, 2, 3, span 2 |
| `grid-column` | 列简写 | 1 / 3 |
| `grid-row` | 行简写 | 1 / 3 |
| `grid-area` | 指定网格区域 | header |

**代码示例**：
```css
/* 跨列 */
.item {
    grid-column: 1 / 3;  /* 从第1条线到第3条线 */
}

/* 跨行 */
.item {
    grid-row: 1 / 2;
}

/* 跨多行/列 */
.item {
    grid-column: span 2;  /* 跨2列 */
    grid-row: span 2;     /* 跨2行 */
}

/* 命名区域 */
.item {
    grid-area: header;
}
```

### 📖 常见布局示例

**两栏布局**：
```css
.container {
    display: grid;
    grid-template-columns: 250px 1fr;
    gap: 20px;
}
```

**三栏布局**：
```css
.container {
    display: grid;
    grid-template-columns: 200px 1fr 200px;
    gap: 20px;
}
```

**响应式卡片布局**：
```css
.card-grid {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
    gap: 24px;
}
```

**圣杯布局**：
```css
.container {
    display: grid;
    grid-template-columns: 200px 1fr 200px;
    grid-template-rows: auto 1fr auto;
    grid-template-areas: 
        "header header header"
        "nav main aside"
        "footer footer footer";
}

.header { grid-area: header; }
.nav { grid-area: nav; }
.main { grid-area: main; }
.aside { grid-area: aside; }
.footer { grid-area: footer; }
```

**面试问答**：

> **问：Grid布局中的 `fr` 单位是什么？**
> **答**：`fr` 是"fraction（分数）"的缩写，表示可用空间的比例。例如 `grid-template-columns: 1fr 2fr 1fr` 会将可用空间分成4等份，第一列占1份，中间列占2份，最后一列占1份。`fr` 单位可以与固定值混用，如 `200px 1fr 200px`。

> **问：如何实现Grid布局的响应式而不使用媒体查询？**
> **答**：使用 `repeat(auto-fill, minmax(最小值, 最大值))` 语法可以实现自动响应的网格布局。例如 `grid-template-columns: repeat(auto-fill, minmax(200px, 1fr))` 会自动根据容器宽度创建尽可能多的列，每列最小200px，最大平分剩余空间。

## 🌱 3.5 布局方案对比与选择

### 📖 布局技术对比

| 布局方式 | 维度 | 适用场景 | 优点 | 缺点 |
|----------|------|----------|------|------|
| **Float** | 一维 | 简单环绕布局 | 兼容性好 | 需要清除浮动 |
| **Position** | 定位 | 特殊定位元素 | 定位精准 | 脱离文档流 |
| **Flexbox** | 一维 | 导航、列表、居中 | 强大、灵活 | 一维限制 |

### 📖 布局方案选择指南

|----------|----------|
| 页面整体布局 | Grid |
| 导航菜单 | Flexbox |
| 元素居中 | Flexbox |
| 图文环绕 | Float |
| 弹出层定位 | Position |
| 卡片网格 | Grid + Flexbox |
| 等高列 | Flexbox或Grid |
| 响应式布局 | Grid + 媒体查询 |

## 💬 本章面试要点

> **问：解释一下CSS盒模型，如何设置盒模型为IE盒模型？**
> **答**：CSS盒模型由内容区、内边距、边框和外边距四部分组成。标准盒模型中width只包含内容区，IE盒模型中width包含内容区、内边距和边框。设置 `box-sizing: border-box` 可以将盒模型改为IE盒模型。

> **问：Flexbox中 justify-content 和 align-items 的区别是什么？**
> **答**：justify-content控制项目在主轴（Main Axis）上的对齐方式，align-items控制项目在交叉轴（Cross Axis）上的对齐方式。主轴由flex-direction决定（默认是水平方向），交叉轴垂直于主轴。

> **问：如何实现一个完全居中的布局？**
> **答**：有多种实现方式。Flexbox方式：在父容器设置 `display: flex; justify-content: center; align-items: center;`。Grid方式：在父容器设置 `display: grid; place-items: center;`。绝对定位方式：父容器设置 `position: relative`，子元素设置 `position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%);`。

> **问：flex-grow 和 flex-shrink 的作用是什么？**
> **答**：flex-grow定义项目在有剩余空间时的放大比例，默认值为0（不放大）。flex-shrink定义项目在空间不足时的缩小比例，默认值为1（可缩小）。设置为0表示不参与伸缩。

## ⚠️ 常见错误与避坑指南

| 错误类型 | 问题描述 | 解决方案 |
|----------|----------|----------|
| 盒模型误解 | 误以为width包含padding和border | 使用 `box-sizing: border-box` |
| 外边距折叠 | 垂直相邻元素外边距不叠加 | 理解折叠机制，使用BFC或padding |
| Flex换行问题 | 项目被强制压缩不换行 | 添加 `flex-wrap: wrap` |
| Grid兼容 | 旧浏览器不支持Grid | 使用 `@supports` 检测或回退方案 |
| 浮动父元素塌陷 | 父元素无法包含浮动子元素 | 使用clearfix或overflow: hidden |
| absolute定位失效 | 没有设置非static定位祖先 | 确保有relative/absolute父元素 |

## 🌱 4.1 动画与过渡

### 📖 过渡（Transition）

**CSS过渡** 是一种简单而强大的动画实现方式，能够在元素属性值发生变化时创建平滑的过渡效果。过渡不需要复杂的@keyframes定义，适合简单的状态变化动画。

| 属性 | 说明 | 常用值 |
|------|------|--------|
| `transition-property` | 要过渡的CSS属性 | all, width, opacity |
| `transition-duration` | 过渡持续时间 | 0.3s, 300ms |
| `transition-timing-function` | 过渡时间函数 | ease, linear, cubic-bezier() |
| `transition-delay` | 过渡延迟时间 | 0.3s, 300ms |
| `transition` | 简写属性 | all 0.3s ease |

**时间函数对比**：

| 函数 | 效果 | 贝塞尔曲线 |
|------|------|------------|
| `ease` | 慢-快-慢 | cubic-bezier(0.25, 0.1, 0.25, 1) |
| `linear` | 匀速 | cubic-bezier(0, 0, 1, 1) |
| `ease-in` | 慢-快 | cubic-bezier(0.42, 0, 1, 1) |
| `ease-out` | 快-慢 | cubic-bezier(0, 0, 0.58, 1) |
| `ease-in-out` | 慢-快-慢 | cubic-bezier(0.42, 0, 0.58, 1) |

**代码示例**：
```css
/* 基础过渡 */
.btn {
    background-color: #3498db;
    transition: background-color 0.3s ease, transform 0.3s ease;
}

.btn:hover {
    background-color: #2980b9;
    transform: scale(1.05);
}

/* 多个属性过渡 */
.card {
    transition: all 0.3s ease;
}
.card:hover {
    box-shadow: 0 10px 20px rgba(0, 0, 0, 0.2);
    transform: translateY(-5px);
}

/* 自定义时间函数 */
.custom-ease {
    transition: transform 0.5s cubic-bezier(0.68, -0.55, 0.265, 1.55);
}
```

**面试问答**：

> **问：transition 和 animation 的区别是什么？**
> **答**：transition需要依赖属性变化（触发条件）才能执行，animation可以自动执行且可以循环。transition适合简单的状态变化，如hover效果；animation适合复杂的、多阶段的、需要循环的动画。transition是单向的（从A到B），animation可以定义多个关键帧。

### 📖 动画（Animation）

**CSS动画** 通过 `@keyframes` 规则定义关键帧，实现更复杂、更可控的动画效果。

| 属性 | 说明 | 常用值 |
|------|------|--------|
| `animation-name` | 动画名称 | @keyframes名称 |
| `animation-duration` | 动画时长 | 1s, 1000ms |
| `animation-timing-function` | 时间函数 | ease, linear, cubic-bezier |
| `animation-delay` | 延迟时间 | 0.5s |
| `animation-iteration-count` | 循环次数 | 1, 2, infinite |
| `animation-direction` | 播放方向 | normal, reverse, alternate |
| `animation-fill-mode` | 填充模式 | none, forwards, backwards, both |
| `animation-play-state` | 播放状态 | running, paused |
| `animation` | 简写 | name duration timing delay count direction fill |

**@keyframes 语法**：
```css
/* 百分比关键帧 */
@keyframes fadeIn {
    0% { opacity: 0; }
    100% { opacity: 1; }
}

/* from/to 简写 */
@keyframes slideIn {
    from { transform: translateX(-100%); }
    to { transform: translateX(0); }
}

/* 多阶段动画 */
@keyframes bounce {
    0%, 20%, 50%, 80%, 100% { transform: translateY(0); }
    40% { transform: translateY(-30px); }
    60% { transform: translateY(-15px); }
}
```

**代码示例**：
```css
/* 淡入动画 */
.fade-in {
    animation: fadeIn 1s ease-in-out;
}

@keyframes fadeIn {
    from { opacity: 0; }
    to { opacity: 1; }
}

/* 脉冲动画 */
.pulse {
    animation: pulse 2s infinite;
}

@keyframes pulse {
    0%, 100% { transform: scale(1); }
    50% { transform: scale(1.05); }
}

/* 旋转加载动画 */
.spinner {
    width: 40px;
    height: 40px;
    border: 4px solid #f3f3f3;
    border-top: 4px solid #3498db;
    border-radius: 50%;
    animation: spin 1s linear infinite;
}

@keyframes spin {
    0% { transform: rotate(0deg); }
    100% { transform: rotate(360deg); }
}

/* 暂停/播放控制 */
.paused {
    animation-play-state: paused;
}
```

### 📖 动画性能优化

**CSS动画性能** 主要涉及浏览器的渲染管线，了解这些知识可以创建更流畅的动画。

| 属性 | 性能 | 说明 |
|------|------|------|
| `transform` | 高 | 只触发合成，不触发布局和绘制 |
| `opacity` | 高 | 只触发合成 |
| `filter` | 中 | 可能触发布局重计算 |
| `width/height` | 低 | 触发布局重计算 |
| `position` | 低 | 触发布局重计算 |
| `color` | 中 | 触发重绘 |

**代码示例**：
```css
/* 高性能动画 */
.good-animation {
    /* 使用transform和opacity */
    transform: translateX(100px);
    opacity: 0.5;
}

/* 低性能动画 - 避免使用 */
.bad-animation {
    /* 避免动画width/height */
    width: 100px;  /* 差 */
    left: 100px;   /* 差 */
    background: red;  /* 中 */
}
```

**will-change 优化**：
```css
.optimized {
    /* 告诉浏览器即将变化，提前优化 */
    will-change: transform, opacity;
}

/* 注意：不要滥用，使用后及时移除 */
.optimized:hover {
    transform: scale(1.1);
}
```

## 🌱 4.3 变换（Transform）

### 📖 2D 变换

**transform** 属性允许对元素进行旋转、缩放、倾斜、平移等变换操作，而不影响文档流。

| 函数 | 说明 | 示例 |
|------|------|------|
| `translate(x, y)` | 平移 | translate(10px, 20px) |
| `rotate(angle)` | 旋转 | rotate(45deg) |
| `scale(x, y)` | 缩放 | scale(1.5) |
| `skew(x, y)` | 倾斜 | skew(10deg, 5deg) |
| `matrix(a, b, c, d, e, f)` | 矩阵变换 | matrix(1, 0, 0, 1, 0, 0) |

**代码示例**：
```css
/* 平移 */
.translate {
    transform: translate(50px, 100px);
    /* 等同于 translateX(50px) translateY(100px) */
}

/* 旋转 */
.rotate {
    transform: rotate(45deg);
    /* 正值顺时针，负值逆时针 */
}

/* 缩放 */
.scale {
    transform: scale(1.2);
    /* 大于1放大，小于1缩小 */
}

.scale-xy {
    transform: scale(1.5, 0.8);
    /* 水平放大1.5倍，垂直缩小 */
}

/* 倾斜 */
.skew {
    transform: skew(20deg);
    /* 水平倾斜 */
}

/* 组合变换 - 从左到右依次执行 */
.combine {
    transform: translate(50px, 50px) rotate(45deg) scale(1.2);
}
```

**变换原点**：
```css
/* 默认变换原点 - 元素中心 */
.transform-origin {
    transform-origin: center center;
}

/* 自定义变换原点 */
.transform-origin-corner {
    transform-origin: top left;
}

.transform-origin-custom {
    transform-origin: 20% 80%;
}
```

### 📖 3D 变换

**3D变换** 在2D变换的基础上增加了Z轴的支持，可以创建立体效果。

| 函数 | 说明 |
|------|------|
| `translateX(x)` | X轴平移 |
| `translateY(y)` | Y轴平移 |
| `translateZ(z)` | Z轴平移 |
| `translate3d(x, y, z)` | 3D平移 |
| `rotateX(angle)` | X轴旋转 |
| `rotateY(angle)` | Y轴旋转 |
| `rotateZ(angle)` | Z轴旋转 |
| `rotate3d(x, y, z, angle)` | 3D旋转 |
| `scaleX(x)` | X轴缩放 |
| `scaleY(y)` | Y轴缩放 |
| `scaleZ(z)` | Z轴缩放 |
| `scale3d(x, y, z)` | 3D缩放 |
| `perspective(n)` | 透视效果 |

**3D效果示例**：
```css
/* 3D容器 */
.scene {
    perspective: 1000px;
    /* 视距，越小透视效果越明显 */
}

/* 3D旋转 */
.rotate-x {
    transform: rotateX(45deg);
    /* 绕X轴旋转 */
}

.rotate-y {
    transform: rotateY(45deg);
    /* 绕Y轴旋转 */
}

.rotate-z {
    transform: rotateZ(45deg);
    /* 绕Z轴旋转 */
}

/* 3D卡片翻转效果 */
.card-container {
    perspective: 1000px;
    width: 200px;
    height: 300px;
}

.card-inner {
    transition: transform 0.6s;
    transform-style: preserve-3d;
}

.card-container:hover .card-inner {
    transform: rotateY(180deg);
}

.card-front,
.card-back {
    position: absolute;
    backface-visibility: hidden;
}

.card-back {
    transform: rotateY(180deg);
}
```

**面试问答**：

> **问：transform 和 position: relative 都能移动元素，它们有什么区别？**
> **答**：主要区别有两点：一是性能，transform使用GPU加速，position: relative会触发布局重计算，性能较差；二是文档流影响，position: relative仍占据原文档流位置，而transform不会影响文档流布局，只是视觉上的移动。

## 🌱 4.4 媒体查询与响应式设计

### 📖 媒体查询基础

**媒体查询（Media Query）** 是CSS3引入的特性，允许根据设备特性和环境条件应用不同的样式规则，是实现响应式设计的基础。

```css
/* 基本语法 */
@media media-type and (media-feature) {
    /* 样式规则 */
}
```

**媒体类型（Media Types）**：

| 类型 | 说明 |
|------|------|
| `all` | 所有设备 |
| `screen` | 屏幕设备（默认） |
| `print` | 打印预览 |
| `speech` | 屏幕阅读器 |
| `tv` | 电视设备 |

**媒体特性（Media Features）**：

| 特性 | 说明 | 常用值 |
|------|------|--------|
| `width` | 视口宽度 | 768px, 100vw |
| `height` | 视口高度 | 600px, 100vh |
| `min-width` | 最小视口宽度 | 768px |
| `max-width` | 最大视口宽度 | 1200px |
| `min-height` | 最小视口高度 | - |
| `max-height` | 最大视口高度 | - |
| `orientation` | 设备方向 | portrait, landscape |
| `aspect-ratio` | 宽高比 | 16/9 |
| `resolution` | 设备分辨率 | 2dppx, 300dpi |
| `color` | 颜色位数 | 8, 16 |

### 📖 响应式断点设计

**常见断点策略**：

```css

    .hide-mobile { display: none; }
}

@media only screen and (min-width: 576px) and (max-width: 767px) {
    .container { width: 540px; }

    .container { width: 720px; }

    .container { width: 960px; }
}

/* 超大设备（大屏桌面） */
@media only screen and (min-width: 1200px) {
    .container { width: 1140px; }
}
```

## 💬 本章面试要点

> **问：如何实现一个无限循环的动画？**
> **答**：使用 `animation-iteration-count: infinite;` 可以让动画无限循环播放。也可以配合 `animation-direction: alternate;` 实现来回循环的动画效果。

> **问：CSS动画和JavaScript动画有什么区别？应该优先使用哪个？**
> **答**：CSS动画由浏览器线程执行，性能更好，适合简单的、独立的动画效果。JavaScript动画可以控制精确的时间轴、暂停、恢复、倒退等，适合复杂的、交互性强的动画。原则是：能用CSS实现的就用CSS，CSS做不到的再用JavaScript。

> **问：如何优化CSS动画的性能？**
> **答**：主要优化策略包括：优先使用transform和opacity属性进行动画；使用will-change提前告知浏览器；避免动画中触发布局重计算（reflow）和重绘（repaint）的属性；使用硬件加速；减少动画中改变的元素数量。

> **问：响应式设计和自适应设计的区别是什么？**
> **答**：响应式设计（Responsive）使用媒体查询和弹性布局，一套代码适配所有屏幕尺寸。自适应设计（Adaptive）为不同设备提供不同版本的网站。响应式更灵活但可能加载不必要的资源，自适应更精准但维护成本高。

## ⚠️ 常见错误与避坑指南

| 错误类型 | 问题描述 | 解决方案 |
|----------|----------|----------|
| 动画性能差 | 使用left/top做动画 | 改用transform |

| 渐变语法错误 | 忘记写渐变方向 | 添加 to right 等方向 |
| 阴影不生效 | 内阴影忘记加inset | 添加inset关键字 |

| 变量未生效 | 未定义直接使用 | 确保在:root中定义 |
| 变换失效 | 未设置变换原点 | 使用transform-origin |

## 🌱 5.1 常见布局解决方案

### 📖 居中布局

**水平居中**：

| 方法 | 适用场景 | 代码复杂度 |
|------|----------|------------|
| `margin: 0 auto` | 已知宽度的块级元素 | 简单 |
| `text-align: center` | 行内/行内块元素 | 简单 |
| `flex` | 任意场景 | 中等 |
| `grid` | 任意场景 | 中等 |

```css
/* 方法1：margin auto - 适用于已知宽度 */
.center-block {
    width: 200px;
    margin: 0 auto;
}

/* 方法2：text-align - 适用于行内元素 */
.center-inline {
    text-align: center;
}

/* 方法3：Flexbox - 推荐 */
.center-flex {
    display: flex;
    justify-content: center;
}

/* 方法4：Grid */
.center-grid {
    display: grid;
    justify-content: center;
}
```

**垂直居中**：

| 方法 | 适用场景 | 注意事项 |
|------|----------|----------|
| `line-height` | 单行文本 | 只适用于文本 |
| `flex` | 任意场景 | 推荐方案 |
| `grid` | 任意场景 | 最简方案 |
| `absolute` | 定位元素 | 需计算偏移 |

```css
/* 方法1：line-height - 仅限单行文本 */
.center-text {
    height: 50px;
    line-height: 50px;
}

/* 方法2：Flexbox - 推荐 */
.center-flex-v {
    display: flex;
    align-items: center;
    height: 200px;
}

/* 方法3：Grid - 最简方案 */
.center-grid-v {
    display: grid;
    align-content: center;
    height: 200px;
}

/* 方法4：绝对定位 - 适用于定位元素 */
.parent {
    position: relative;
    height: 200px;
}
.child {
    position: absolute;
    top: 50%;
    transform: translateY(-50%);
}
```

**水平垂直居中**：

```css
/* Flexbox 方案 - 推荐 */
.center-both-flex {
    display: flex;
    justify-content: center;
    align-items: center;
    height: 200px;
}

/* Grid 方案 - 最简 */
.center-both-grid {
    display: grid;
    place-items: center;
    height: 200px;
}

/* 绝对定位方案 */
.center-both-absolute {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
}
```

### 📖 两栏布局

| 方案 | 优点 | 缺点 |
|------|------|------|
| Float | 兼容性好 | 需清除浮动 |
| Flexbox | 简单强大 | 旧IE不支持 |
| Grid | 最简洁 | 旧浏览器不支持 |

```css
/* Float 方案 */
.layout-float::after {
    content: "";
    display: block;
    clear: both;
}
.sidebar {
    float: left;
    width: 200px;
}
.content {
    margin-left: 200px;
}

/* Flexbox 方案 - 推荐 */
.layout-flex {
    display: flex;
}
.sidebar {
    flex: 0 0 200px;
}
.content {
    flex: 1;
}

/* Grid 方案 */
.layout-grid {
    display: grid;
    grid-template-columns: 200px 1fr;
}
```

### 📖 三栏布局

```css
/* 圣杯布局 - Flexbox */
.holy-grail-flex {
    display: flex;
    min-height: 100vh;
    flex-direction: column;
}
.header, .footer {
    flex: 0 0 auto;
}
.main-container {
    display: flex;
    flex: 1;
}
.sidebar-left, .sidebar-right {
    flex: 0 0 200px;
}
.content {
    flex: 1;
}

/* 圣杯布局 - Grid */
.holy-grail-grid {
    display: grid;
    grid-template-columns: 200px 1fr 200px;
    grid-template-rows: auto 1fr auto;
    min-height: 100vh;
    grid-template-areas:
        "header header header"
        "nav main aside"
        "footer footer footer";
}
```

### 📖 Sticky Footer 布局

**问题描述**：当页面内容较少时，页脚会跟随内容上移，导致页面底部留白。

```css
/* Flexbox 方案 - 推荐 */
.sticky-footer {
    display: flex;
    flex-direction: column;
    min-height: 100vh;
}
.content {
    flex: 1;
}

/* Grid 方案 */
.sticky-footer-grid {
    display: grid;
    grid-template-rows: auto 1fr auto;
    min-height: 100vh;
}
```

## 🌱 5.2 性能优化

### 📖 CSS 性能优化策略

| 优化策略 | 说明 | 优先级 |
|----------|------|--------|
| 简化选择器 | 减少选择器复杂度 | 高 |
| 使用CSS变量 | 提高可维护性 | 高 |
| 避免@import | 阻塞渲染 | 高 |
| 使用简写属性 | 减少代码量 | 中 |
| 压缩CSS | 减少文件体积 | 高 |
| 内联关键CSS | 加快首屏渲染 | 中 |

```css
/* 避免复杂选择器 */
/* 不推荐 */
.nav ul li a span.active {
    color: red;
}

/* 推荐 */
.nav .active {
    color: red;
}

/* 使用简写属性 */
/* 不推荐 */
margin-top: 10px;
margin-right: 20px;
margin-bottom: 10px;
margin-left: 20px;

/* 推荐 */
margin: 10px 20px;
```

### 📖 渲染性能优化

**浏览器渲染流程**：解析HTML → 构建DOM → 解析CSS → 构建CSSOM → 合成渲染树 → 布局 → 绘制 → 合成

| 概念 | 说明 | 性能影响 |
|------|------|----------|
| **重排（Reflow）** | 重新计算布局 | 高 |
| **重绘（Repaint）** | 重新绘制元素 | 中 |
| **合成（Composite）** | 合成图层 | 低 |

**优化建议**：

```css
/* 高性能动画 - 使用 transform 和 opacity */
.good-animation {
    transform: translateX(100px);  /* 仅合成 */
    opacity: 0.5;                  /* 仅合成 */
}

/* 避免触发布局重计算 */
.bad-animation {
    width: 100px;     /* 触发重排 */
    height: 100px;    /* 触发重排 */
    left: 50px;       /* 触发重排 */
}

/* 使用 will-change 提示浏览器 */
.optimized-element {
    will-change: transform;
    transform: translateZ(0);  /* 开启硬件加速 */
}

/* 使用 CSS  containment 隔离布局 */
.isolated {
    contain: content;
    /* 或 */
    contain: strict;
}
```

### 📖 加载性能优化

```css
/* 预加载关键资源 */
<link rel="preload" href="styles.css" as="style">

/* 异步加载非关键CSS */
<link rel="preload" href="styles.css" as="style" 
      onload="this.onload=null;this.rel='stylesheet'">

/* 字体优化 */
@font-face {
    font-family: 'CustomFont';
    src: url('font.woff2') format('woff2');
    font-display: swap;
}
```

### 📖 兼容性解决方案

**特性检测 vs 浏览器检测**：

```javascript
/* 特性检测 - 推荐 */
if (CSS.supports('display: grid')) {
    // 使用Grid布局
} else {
    // 使用Flexbox回退方案
}

/* 浏览器检测 - 不推荐 */
if (navigator.userAgent.includes('Chrome')) {
    // 不推荐
}
```

**@supports 特性检测**：

```css
/* 基础用法 */
@supports (display: grid) {
    .container {
        display: grid;
    }
}

/* 组合条件 */
@supports (display: grid) and (display: flex) {
    .container {
        display: grid;
    }
}

/*否定条件*/
@supports not (display: grid) {
    .container {
        display: flex;
    }
}
```

**Polyfill 回退方案**：

```html
<!-- 条件加载 Polyfill -->
<script>
    if (!CSS.supports('display: grid')) {
        document.write('<script src="grid-polyfill.js"><\/script>');
    }
</script>
```

## 🌱 5.4 无障碍访问

### 📖 无障碍访问基础

**无障碍访问（Accessibility, a11y）** 确保网页内容对所有用户可访问，包括视力、听力、运动障碍用户。

| 原则 | 说明 |
|------|------|
| **可感知性** | 信息必须以用户可感知的方式呈现 |
| **可操作性** | 用户界面组件必须可操作 |
| **可理解性** | 界面必须可理解 |
| **健壮性** | 内容必须足够健壮 |

### 📖 语义化 HTML 无障碍

```html
<!-- 使用正确的标题层级 -->
<h1>主标题</h1>
<h2>章节标题</h2>
<h3>小节标题</h3>

<!-- 图片必须添加 alt 属性 -->
<img src="photo.jpg" alt="风景照片：蓝天白云下的草原">

<!-- 装饰性图片使用空 alt -->
<img src="decoration.png" alt="">

<!-- 链接要有描述性文本 -->
<!-- 不推荐 -->
<a href="#">点击这里</a>

<!-- 推荐 -->
<a href="/products">查看我们的产品</a>
```

### 📖 表单无障碍

```html
<!-- 正确关联 label 和 input -->
<label for="username">用户名</label>
<input type="text" id="username" name="username">

<!-- 或者使用嵌套 -->
<label>
    邮箱
    <input type="email" name="email">
</label>

<!-- 必填字段标记 -->
<label for="phone">
    电话号码
    <span aria-label="必填">*</span>
</label>
<input type="tel" id="phone" required>

<!-- 错误提示关联 -->
<label for="password">密码</label>
<input type="password" id="password" 
       aria-describedby="password-hint"
       aria-invalid="false">
<p id="password-hint">密码至少8位字符</p>
```

### 📖 ARIA 属性

| 属性 | 作用 | 示例 |
|------|------|------|
| `role` | 定义元素角色 | `role="navigation"` |
| `aria-label` | 提供文本标签 | `aria-label="关闭"` |
| `aria-labelledby` | 关联标题元素 | `aria-labelledby="title"` |
| `aria-describedby` | 关联描述元素 | `aria-describedby="hint"` |
| `aria-hidden` | 隐藏元素 | `aria-hidden="true"` |
| `aria-expanded` | 展开状态 | `aria-expanded="false"` |
| `aria-pressed` | 按下状态 | `aria-pressed="false"` |
| `aria-disabled` | 禁用状态 | `aria-disabled="true"` |

```html
<!-- 折叠面板 -->
<button aria-expanded="false" 
        aria-controls="panel-content">
    展开更多
</button>
<div id="panel-content" hidden>
    展开的内容...
</div>

<!-- 模态框 -->
<div role="dialog" 
     aria-modal="true" 
     aria-labelledby="modal-title">
    <h2 id="modal-title">确认删除</h2>
    <p>确定要删除此项吗？</p>
    <button>取消</button>
    <button>确定</button>
</div>

<!-- 导航菜单 -->
<nav role="navigation" aria-label="主导航">
    <ul>
        <li><a href="/" aria-current="page">首页</a></li>
        <li><a href="/about">关于</a></li>
    </ul>
</nav>
```

### 📖 键盘无障碍

```css
/* 焦点样式 - 不要移除 */
:focus-visible {
    outline: 2px solid #007bff;
    outline-offset: 2px;
}

/* 移除默认焦点样式后必须自定义 */
:focus {
    outline: none;
}

/* 自定义焦点样式 */
button:focus-visible {
    outline: 2px solid #007bff;
    outline-offset: 2px;
}
```

```html
<!-- 跳转链接 - 屏幕阅读器专读 -->
<a href="#main-content" class="skip-link">
    跳到主要内容
</a>

<style>
.skip-link {
    position: absolute;
    top: -40px;
    left: 0;
    padding: 8px;
    background: #000;
    color: #fff;
    z-index: 100;
}
.skip-link:focus {
    top: 0;
}
</style>

<!-- 使用 tabindex 控制焦点 -->
<div tabindex="0">可获得焦点</div>
<div tabindex="-1">可编程获得焦点</div>

```

**面试问答**：

> **问：如何实现一个无障碍的轮播图？**
> **答**：轮播图的无障碍实现需要：1）为整个轮播图添加角色和标签 `role="region" aria-label="图片轮播"`；2）每个幻灯片添加 `aria-hidden="true"`（当不可见时）或 `aria-live="polite"`（自动播放时）；3）提供上一张/下一张按钮，并添加 `aria-label` 描述；4）提供指示器（如圆点）的焦点和键盘导航支持；5）考虑提供暂停/播放控制。

## 💬 本章面试要点

> **问：如何实现一个完全居中的布局？**
> **答**：有多种实现方式。Flexbox方式：在父容器设置 `display: flex; justify-content: center; align-items: center;`。Grid方式：在父容器设置 `display: grid; place-items: center;`。绝对定位方式：父容器设置 `position: relative`，子元素设置 `position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%);`。

> **问：如何优化CSS动画的性能？**
> **答**：主要优化策略包括：1）优先使用transform和opacity属性进行动画，这两个属性只触发合成；2）使用will-change提前告知浏览器要变化的属性；3）避免动画中触发布局重计算（reflow）的属性如width、height、top、left等；4）使用硬件加速（translateZ(0)或translate3d）；5）减少动画中改变的元素数量。

> **问：重排和重绘有什么区别？**
> **答**：重排（Reflow）是指当DOM元素的布局属性（尺寸、位置）发生变化时，浏览器重新计算整个文档的布局；重绘（Repaint）是指当元素的外观属性（如颜色、背景）发生变化但布局不变时，浏览器重新绘制元素。重排比重绘消耗更大，应尽量避免。

> **问：如何实现一个无障碍的表单？**
> **答**：无障碍表单的关键点包括：1）每个表单控件必须有关联的label；2）必填字段使用required属性和aria-required；3）使用aria-describedby关联错误提示；4）使用aria-invalid标识错误状态；5）确保键盘可以完整操作表单；6）提供清晰的错误提示信息。

## 🌱 6.1 HTML 高频面试题

### 📖 HTML 基础概念

> **问：HTML5有哪些新特性？**
> **答**：HTML5引入了众多新特性：1）语义化标签（header、nav、main、article、section、aside、footer）；2）多媒体支持（video、audio）；3）图形绘制（canvas、svg）；4）本地存储（localStorage、sessionStorage）；5）离线应用（Service Workers、Manifest）；6）表单增强（新的输入类型、属性）；7）Web Workers后台线程；8）Geolocation地理定位；9）WebSocket双向通信。

> **问：语义化的意义是什么？**
> **答**：语义化主要有四方面意义：1）代码可读性，标签语义清晰便于开发者理解和维护；2）SEO优化，搜索引擎能更好地理解页面结构，提升搜索排名；3）无障碍访问，屏幕阅读器能准确识别内容区域；4）团队协作，统一规范便于多人协同开发。

> **问：DOCTYPE的作用是什么？**
> **答**：DOCTYPE用于声明文档类型和DTD（文档类型定义），告诉浏览器按照哪种规范来解析文档。HTML5的`<!DOCTYPE html>`声明启用标准模式渲染页面。没有DOCTYPE声明会导致浏览器使用Quirks Mode（兼容模式）渲染，可能出现样式兼容性问题。

> **问：meta标签有哪些常见用途？**
> **答**：meta标签用于提供页面的元信息：1）字符编码 `<meta charset="UTF-8">`；2）视口设置 `<meta name="viewport">`用于移动端适配；3）SEO优化（description、keywords、author）；4）页面描述和社交分享（og:title、og:image）；5）HTTP头信息（refresh、expires）；6）浏览器兼容性（X-UA-Compatible）。

> **问：Canvas和SVG的区别是什么？**
> **答**：Canvas是基于像素的位图，适合大量对象实时渲染（如游戏），放大后失真；SVG是基于矢量的图形，适合静态图表和UI组件，无限放大不失真。Canvas需要手动控制重绘，SVG通过DOM操作；Canvas适合像素操作和热力图，SVG适合图标和矢量图形。

## 🌱 6.2 CSS 高频面试题

### 📖 盒模型与选择器

> **问：解释CSS盒模型，如何切换盒模型？**
> **答**：CSS盒模型由内容区、内边距、边框、外边距四部分组成。标准盒模型中width只包含内容区，IE盒模型中width包含内容区、内边距和边框。通过 `box-sizing` 属性切换：`content-box`（标准盒模型，默认）和 `border-box`（IE盒模型）。

> **问：CSS选择器优先级如何计算？**
> **答**：优先级由四元组(a,b,c,d)决定：a是行内样式（1,0,0,0）；b是ID选择器数量（0,1,0,0）；c是类选择器、属性选择器、伪类数量（0,0,1,0）；d是标签选择器、伪元素数量（0,0,0,1）。`!important` 优先级最高。比较时从a到d逐位比较。

> **问：什么是BFC？如何触发BFC？**
> **答**：BFC（Block Formatting Context，块级格式化上下文）是CSS渲染盒子的区域，内部的布局不影响外部。触发BFC的方式：1）根元素html；2）float不为none；3）position为absolute或fixed；4）display为inline-block、flex、grid；5）overflow不为visible。BFC的应用：防止外边距折叠、清除浮动、阻止文字环绕。

### 📖 布局与定位

> **问：Flexbox和Grid的区别是什么？**
> **答**：Flexbox是一维布局，只能处理行或列单一方向的布局；Grid是二维布局，同时处理行和列的布局。简单来说，Flexbox适合导航、列表等线性布局，Grid适合相册、表格、页面整体布局等需要行列对齐的场景。

> **问：position各属性值的区别？**
> **答**：static是默认值，正常文档流；relative相对于自身原始位置定位，仍占据原空间；absolute相对于最近非static祖先定位，完全脱离文档流；fixed相对于浏览器视口定位，滚动时位置不变；sticky是粘性定位，未超过阈值前正常滚动，超过后固定。

> **问：如何实现三栏布局（左右固定，中间自适应）？**
> **答**：有多种实现方式。Flexbox方案：父容器 `display: flex`，左侧 `flex: 0 0 200px`，中间 `flex: 1`。Grid方案：`grid-template-columns: 200px 1fr 200px`。浮动方案：左右浮动，中间设置margin。

### 📖 动画与性能

> **问：transition和animation的区别？**
> **答**：transition需要依赖属性变化触发，animation可自动执行；transition是单向过渡，animation可定义多关键帧；transition适合简单状态变化，animation适合复杂动画；transition性能通常更好，animation可控性更强。

> **问：如何优化CSS动画性能？**
> **答**：优化策略：1）使用transform和opacity，这两个属性只触发合成；2）避免动画width/height/top/left等触发布局的属性；3）使用will-change提示浏览器；4）使用translateZ(0)开启硬件加速；5）减少动画元素数量。

> **问：什么是重排和重绘？如何优化？**
> **答**：重排是重新计算布局，当元素尺寸、位置变化时触发；重绘是重新绘制元素，当外观变化但布局不变时触发。优化方法：使用transform、opacity做动画；避免频繁读取布局属性；使用文档片段批量操作DOM；使用CSS containment隔离。

## 🌱 6.3 手写代码题

### 📖 常见手写题

**题目1：实现三栏布局**

```css
/* Flexbox 实现 */
.container {
    display: flex;
    height: 100vh;
}
.left, .right {
    width: 200px;
    background: #f0f0f0;
}
.center {
    flex: 1;
    background: #fff;
}

/* Grid 实现 */
.container {
    display: grid;
    grid-template-columns: 200px 1fr 200px;
    height: 100vh;
}
```

**题目2：实现垂直居中**

```css
/* 方案1：Flexbox */
.parent {
    display: flex;
    justify-content: center;
    align-items: center;
    height: 400px;
}

/* 方案2：Grid */
.parent {
    display: grid;
    place-items: center;
    height: 400px;
}

/* 方案3：绝对定位 */
.parent {
    position: relative;
    height: 400px;
}
.child {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
}
```

**题目3：实现Sticky Footer**

```css
/* Flexbox */
.page-wrapper {
    display: flex;
    flex-direction: column;
    min-height: 100vh;
}
.content {
    flex: 1;
}

/* Grid */
.page-wrapper {
    display: grid;
    grid-template-rows: auto 1fr auto;
    min-height: 100vh;
}
```

**题目4：实现双列等高布局**

```css
/* Flexbox */
.columns {
    display: flex;
}
.column {
    flex: 1;
    padding: 20px;
}

/* Grid */
.columns {
    display: grid;
    grid-template-columns: 1fr 1fr;
}
```
