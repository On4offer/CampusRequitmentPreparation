# 📌 计算机网络与浏览器常用概念与使用场景速查

> 配合《学习笔记》使用，侧重 **HTTP、缓存、跨域、安全与浏览器**，便于速查与校招面试。

---

## 一、HTTP 状态码

| 分类 | 常见码 | 说明 |
|------|--------|------|
| **2xx** | 200 | 成功 |
| **3xx** | 301 | 永久重定向；302 临时重定向；304 协商缓存命中 |
| **4xx** | 400 | 请求错误；401 未认证；403 禁止；404 未找到 |
| **5xx** | 500 | 服务器错误；502 网关错误；503 服务不可用 |

---

## 二、常用请求/响应头

| 头 | 说明 | 示例/场景 |
|----|------|-----------|
| **Content-Type** | 请求/响应体类型 | application/json、application/x-www-form-urlencoded、multipart/form-data |
| **Cache-Control** | 缓存策略 | max-age=3600、no-cache、no-store、private、public |
| **Cookie / Set-Cookie** | 携带/设置 Cookie | HttpOnly、Secure、SameSite=Strict |
| **Origin / Access-Control-Allow-Origin** | 跨域 | 响应头允许的源；* 或具体域名 |
| **Access-Control-Allow-Methods / Headers** | CORS 允许的方法与头 | 预检请求后返回 |

---

## 三、强缓存与协商缓存

| 类型 | 相关头 | 行为 |
|------|--------|------|
| **强缓存** | Expires、Cache-Control（max-age） | 未过期直接用本地，不请求 |
| **协商缓存** | Last-Modified / If-Modified-Since | 发请求，服务器比较时间，304 用本地 |
| **协商缓存** | ETag / If-None-Match | 发请求，比较摘要，304 用本地 |

- 流程：先看强缓存 → 过期则发请求 → 带 If-Modified-Since 或 If-None-Match → 服务器返回 304 或 200+ 新内容。

---

## 四、跨域与 CORS

- **同源**：协议 + 域名 + 端口 一致。
- **CORS**：服务端响应头 `Access-Control-Allow-Origin` 等；**简单请求**直接发；**非简单请求**（如自定义头、PUT）先发 **OPTIONS 预检**，通过后再发实际请求。
- **开发常用**：Vite/Webpack 配置 proxy，将 /api 代理到后端，避免浏览器跨域。

---

## 五、安全要点

| 问题 | 防护思路 |
|------|----------|
| **XSS** | 输出转义、CSP、Cookie 设 HttpOnly、避免 innerHTML 插不可信内容 |
| **CSRF** | 校验 Referer/Origin、CSRF Token、Cookie SameSite |
| **HTTPS** | 防止窃听与篡改，校招可简述 TLS 握手与证书校验 |

## 六、浏览器存储

| 存储 | 容量 | 生命周期 | 同源 |
|------|------|----------|------|
| **Cookie** | 约 4KB | 可设过期 | 按域名，可设 path |
| **LocalStorage** | 约 5MB | 持久，除非清除 | 同源 |
| **SessionStorage** | 约 5MB | 标签页关闭即失效 | 同源 |

---

## 七、重排与重绘

- **重排（reflow）**：布局变化（尺寸、位置、增删节点等）；**重绘（repaint）**：外观变化不改变布局（颜色、背景等）。重排一定引发重绘。
- **优化**：批量读 DOM 再批量写、用 transform/opacity 做动画、减少直接改样式触发布局。

---

## 八、从 URL 到页面（面试简述）

1. DNS 解析域名 → IP  
2. TCP 三次握手建立连接（HTTPS 还有 TLS 握手）  
3. 发 HTTP 请求（方法、URL、头、体）  
4. 服务器处理并返回响应（状态码、头、体）  
5. 浏览器解析 HTML 建 DOM、解析 CSS 建 CSSOM、合成渲染树、布局、绘制、合成  
6. 连接关闭（或 Keep-Alive 复用）
