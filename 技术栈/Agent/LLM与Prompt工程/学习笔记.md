> 系统学习见下文各章；日常常用 Prompt 模式与 API 速查见同目录《常用API与使用场景》。

------

## 第1章：大模型与 Prompt 工程概述

### 1.1 大语言模型（LLM）简介

- **LLM** 是基于 Transformer 的预训练模型，通过海量文本学习语言与知识，可完成续写、问答、摘要、翻译、代码生成等任务。
- 常见形态：**通用基座模型**（GPT、Claude、通义、文心等）、**对话/指令微调模型**（Chat 系列）、**开源与闭源**、**API 与本地部署**。

### 1.2 什么是 Prompt 工程

- **Prompt** 即给模型的**输入提示**；**Prompt 工程** 指通过设计提示内容与结构，让模型更稳定、更符合预期的输出。
- 价值：同一模型，不同 Prompt 效果差异大；好的 Prompt 可减少幻觉、约束格式、提升可解析性（如 JSON、表格）。

### 1.3 典型应用场景

- 问答与知识检索、文本摘要与改写、分类与抽取、代码生成与解释、多轮对话、与 RAG/Agent 结合做检索增强与工具调用。

## 第2章：Prompt 设计原则与结构

1. 角色设定（Role）：让模型扮演什么身份
2. 任务描述（Task）：明确要做什么、输出形式（JSON/列表/段落）
3. 上下文与约束（Context & Constraints）：背景信息、格式要求、禁止项
4. 示例（Few-shot）：给 1～3 个输入输出示例，引导格式与风格
5. 思维链（Chain-of-Thought）：让模型“先推理再回答”，提升复杂题准确率
6. 分隔与格式：用清晰的分段、标记（如 ```、---）便于模型解析

------

## 第3章：常见 Prompt 模式与技巧

1. Zero-shot 与 Few-shot；何时加示例、示例数量与质量
2. CoT（Chain-of-Thought）：显式要求“一步步思考”、Let's think step by step
3. 角色 + 规则 + 输出格式：系统 Prompt 与用户 Prompt 分工（API 中的 system/user）
4. 分步与拆任务：复杂任务拆成多轮或子问题
5. 负面约束：不要做什么、不要输出无关内容、不要编造
6. 输出结构化：要求 JSON、XML、Markdown 表格，便于程序解析

------

## 第4章：与 API 的配合使用

1. 开放 API 形态：OpenAI 兼容接口（model、messages、temperature、max_tokens）
2. messages 结构：system / user / assistant；多轮对话的拼接方式
3. 参数：temperature（创造性 vs 稳定）、top_p、max_tokens、stop
4. 流式输出（stream）：边生成边返回，前端逐字展示
5. 国产大模型 API 概览：通义、文心、智谱、月之暗面、DeepSeek 等（鉴权、端点、差异）

------

## 第5章：幻觉、安全与评估

1. 幻觉原因与缓解：知识截止、不确定时要求模型声明“不确定”、用 RAG 补充依据
2. 安全与合规：敏感信息过滤、输出审核、合规使用条款
3. 简单评估方式：人工抽检、关键指标（准确率、格式符合率）、A/B 测试不同 Prompt
4. 迭代优化：根据 badcase 调整角色/示例/约束，形成 Prompt 版本管理

------

## 第6章：实战与最佳实践

1. 从零设计一个任务 Prompt 的流程（目标→角色→任务→示例→约束→测试）
2. 模板化与变量替换：将 Prompt 做成模板，运行时注入业务变量
3. 与 RAG 结合：检索到的文档作为 context 注入 Prompt，要求基于文档回答并引用
4. 面试要点：什么是 Prompt 工程、Few-shot 与 CoT、如何减少幻觉、system/user 分工
