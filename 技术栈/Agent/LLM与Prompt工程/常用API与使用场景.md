# 📌 LLM 与 Prompt 工程常用 API 与使用场景速查

> 配合《学习笔记》使用，侧重 **Prompt 结构、常用模式与 API 调用**，便于速查。

---

## 一、Prompt 结构速查

| 部分 | 说明 | 示例 |
|------|------|------|
| **角色** | 设定模型身份 | 你是一个专业的客服助手。 |
| **任务** | 明确要做什么 | 根据用户问题，从以下文档中提取答案，并以 JSON 格式输出。 |
| **上下文** | 背景与输入材料 | 文档内容：…… |
| **约束** | 格式与禁止项 | 只输出 JSON，不要编造文档中没有的信息。 |
| **示例** | Few-shot | 用户问：… → 输出：{ "answer": "…" } |
| **思维链** | CoT 引导 | 请先简要分析问题，再给出最终答案。 |

---

## 二、常用 Prompt 句式

| 目的 | 示例 |
|------|------|
| 角色设定 | 你是一个{角色}，擅长{能力}。 |
| 输出格式 | 请以 JSON 格式输出，包含字段：title, summary。 |
| 不要编造 | 仅根据上述内容回答，若无法从文中得出答案，请回答“无法从给定内容中得出”。 |
| 分步思考 | 请一步步思考后再给出最终答案。（Let's think step by step） |
| 负面约束 | 不要输出与问题无关的内容；不要包含敏感个人信息。 |

## 三、API 调用要点（OpenAI 兼容）

| 参数 | 说明 | 常用值 |
|------|------|--------|
| **model** | 模型名 | gpt-4o、gpt-3.5-turbo、或国产模型对应 id |
| **messages** | 消息列表 | [{ role: "system", content: "…" }, { role: "user", content: "…" }] |
| **temperature** | 随机性 | 0～1，摘要/抽取用 0.2～0.3，创意用 0.7～0.9 |
| **max_tokens** | 最大生成长度 | 视任务设定，避免过长浪费或截断 |
| **stream** | 流式 | true 时返回 SSE 流，前端逐字展示 |

### 多轮对话

- 将历史 user/assistant 消息按顺序追加到 messages；过长时可保留最近 N 轮或做摘要压缩。

## 四、使用场景对照

| 场景 | 推荐做法 |
|------|----------|
| 问答 | 角色+任务+上下文（或 RAG 检索结果），约束“仅基于上下文回答”。 |
| 摘要 | 明确字数/条数，可要求“分点概括”。 |
| 分类/打标签 | Few-shot 给 1～2 个示例，要求输出固定标签或 JSON。 |
| 抽取 | 规定输出 JSON 字段，必要时给 schema 或示例。 |
| 代码生成 | 说明语言与框架，要求只输出代码块、带注释。 |
| 多轮对话 | system 定角色与规则，user/assistant 历史拼接；可设 max_tokens 控制单轮长度。 |

## 五、国产模型 API 速记

| 平台 | 鉴权方式 | 备注 |
|------|----------|------|
| 阿里通义 | API-KEY，DashScope 端点 | 兼容 OpenAI 格式的 SDK/代理 |

- 具体端点与参数以各平台文档为准；多数提供 OpenAI 兼容的 HTTP 接口，可统一封装 client。
