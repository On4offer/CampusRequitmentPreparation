# 📘《Agent 开发学习与实战手册》

> 系统学习见下文各章；日常常用框架 API、Agent 模式与工具定义速查见同目录《常用API与使用场景》。

------

## 第1章：Agent 概念与典型架构

### 1.1 什么是 Agent

- **Agent** 指能根据目标**自主规划、调用工具、多步执行**的智能体；相比单次问答，具备：感知（输入）、决策（选动作）、执行（调工具）、反馈（观察结果）的循环。
- 典型组成：**LLM**（推理与规划）、**工具集**（Tool/Function）、**记忆**（对话或短期状态）、**控制流**（何时停止、重试）。

### 1.2 ReAct 与 Tool Use

- **ReAct**（Reasoning + Acting）：模型交替输出“思考—行动—观察”，根据观察再决定下一步，直到得出最终答案。
- **Tool Use / Function Calling**：模型输出结构化“工具调用”（名称+参数），由程序执行工具并把结果塞回上下文，模型再基于结果继续生成；这是当前主流实现方式。

### 1.3 与后端的关系

- Agent 常作为**服务层**：接收用户请求 → 调用 LLM+工具（检索、查库、调 API）→ 汇总结果返回；可与 Spring Boot 等后端并存，由后端提供 REST 或内部接口给 Agent 调用。

## 第2章：工具定义与调用

1. 工具描述：名称、说明（description）、参数 schema（JSON Schema），便于模型选择与填参
2. 实现方式：本地函数、HTTP 接口、内部服务调用；参数校验与错误处理
3. 模型侧：OpenAI function calling、国产模型的 tool 字段；多工具时的选择与参数抽取
4. 安全：工具权限、输入校验、敏感操作二次确认、调用次数与超时

------

## 第3章：LangChain / LangGraph 概览

1. **LangChain**：链式组合 LLM、Prompt、工具、记忆；LCEL（LangChain Expression Language）写流水线；AgentExecutor 执行 ReAct 等 Agent
2. **LangGraph**：基于图的 Agent 编排，显式状态与边，支持分支、循环、人工介入；适合多步、多 Agent 协作
3. 核心抽象：Tool、Agent、Chain、Memory、Callback；与国产模型对接（自定义 LLM 封装）
4. 适用场景：快速搭对话 Agent、RAG 问答、简单工作流；复杂流程用 LangGraph 更清晰

------

## 第4章：多 Agent 与编排

1. 多 Agent 分工：如“规划 Agent + 执行 Agent”“检索 Agent + 总结 Agent”；通过共享状态或消息队列协作
2. 编排模式：顺序、分支、循环、人工审核节点；LangGraph 的状态图与 conditional edge
3. 与后端联调：Agent 调用后端提供的工具（REST/内部 RPC），后端不直接依赖 Agent 框架，通过接口契约协作

------

## 第5章：记忆、上下文与生产考量

1. 短期记忆：当轮对话的 messages；长期记忆：向量库、知识库、用户画像（需合规）
2. 上下文窗口与裁剪：摘要压缩、滑动窗口、只保留关键工具调用与结果
3. 稳定性：超时、重试、熔断；成本：token 计数、缓存、模型选型
4. 可观测：日志、trace、工具调用与耗时监控；便于排查与优化

------

## 第6章：实战与面试要点

1. 从零搭一个 Agent：选模型 → 定义工具（描述+实现）→ 选框架（LangChain/LangGraph 或自研循环）→ 联调与迭代
2. 常见问题：工具选错、参数填错、幻觉导致乱调工具；对策：清晰描述、Few-shot、结果校验与重试
3. 面试：Agent 与普通 LLM 区别、ReAct 与 Tool Use、如何设计工具 schema、多 Agent 如何分工与编排
