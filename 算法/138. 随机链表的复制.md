# LeetCode 138. 复制带随机指针的链表（Copy List with Random Pointer）

## 题意

给定链表，每个节点有 `next` 与 `random` 两个指针。返回该链表的**深拷贝**（新链表节点与旧链表节点地址不同，但结构与值一致；`random` 指向也要正确复制，允许为 `null`）。

------

## 思路与指针设计（O(1) 额外空间的“穿插法”最优解）

核心三步（把新节点**穿插**到旧链表里，再拆分）：

1. **穿插复制**：对每个旧节点 `x`，创建新节点 `x'` 并插到 `x` 后面：`x -> x' -> x.next`。
2. **修复 random**：由于 `x'` 在 `x` 后面，`x'.random = x.random == null ? null : x.random.next`。
3. **拆分**：把交织的链表拆成旧链表与新链表两条独立链。

指针：

- `cur`：扫描旧链表；
- 拆分阶段用 `old`、`neo` 两个游标同步前进。

------

## 伪代码

```
if head == null: return null

// 1) 穿插复制
cur = head
while cur != null:
    copy = new Node(cur.val)
    copy.next = cur.next
    cur.next = copy
    cur = copy.next

// 2) 修复 random
cur = head
while cur != null:
    cur.next.random = (cur.random == null) ? null : cur.random.next
    cur = cur.next.next

// 3) 拆分
old = head
neo = head.next
res = neo
while old != null:
    old.next = old.next.next
    neo.next = (neo.next == null) ? null : neo.next.next
    old = old.next
    neo = neo.next

return res
```

## Java 标准实现（O(1) 额外空间）

```java
/*
// Definition for a Node.
class Node {
    int val;
    Node next;
    Node random;

    public Node(int val) {
        this.val = val;
        this.next = null;
        this.random = null;
    }
}
*/

class Solution {
    public Node copyRandomList(Node head) {
        if (head == null) return null;

        // 1) 穿插复制：A->B->C 变为 A->A'->B->B'->C->C'
        Node cur = head;
        while (cur != null) {
            Node copy = new Node(cur.val);
            copy.next = cur.next;
            cur.next = copy;
            cur = copy.next;
        }

        // 2) 修复 random：A'.random = A.random == null ? null : A.random.next
        cur = head;
        while (cur != null) {
            Node copy = cur.next;
            copy.random = (cur.random == null) ? null : cur.random.next;
            cur = copy.next; // 跳到下一个原节点
        }

        // 3) 拆分：恢复原链 + 提取新链
        Node old = head;
        Node neo = head.next;
        Node res = neo;
        while (old != null) {
            old.next = old.next.next;                 // 原链：跳过副本
            neo.next = (neo.next == null) ? null : neo.next.next; // 新链：跳过原节点
            old = old.next;
            neo = neo.next;
        }

        return res;
    }
}
```

------

## 图解要点（文字版）

以 `A → B → C` 为例：

- 步骤1：`A→A'→B→B'→C→C'`；
- 步骤2：`A'.random = A.random?.next`（因为 `A.random` 的副本就在它后面）；
- 步骤3：拆分成两条：
   原：`A → B → C`；
   新：`A' → B' → C'`。

------

## 复杂度

- 时间：O(n)（三次线性扫描）
- 空间：O(1) 额外空间（原地交织再拆分）

------

## 面试回答模板

> **思路**：采用“穿插法”。第一遍把每个原节点后面插入其副本；第二遍利用交织结构设置副本的 `random`（`copy.random = orig.random?.next`）；第三遍把交织链表拆成原与副本两条链。
>  **复杂度**：时间 O(n)，空间 O(1)。
>  **优势**：无需哈希表映射，原地完成。

------

## 常见坑

1. **`random == null` 判空**：`copy.random = (orig.random == null ? null : orig.random.next)`。
2. **拆分时两条链都要连好**：原链 `old.next = old.next.next`；新链 `neo.next = neo.next == null ? null : neo.next.next`。
3. 别忘了**返回新链表头**（第一份副本 `head.next` 保存为 `res`）。
4. 极小输入：`head == null`、单节点（`random` 指向自己）、全 `random = null` 等边界均能覆盖。

------

### 补充：哈希表版本（更直观，O(n) 空间）

面试时若紧张，也可先写 Map 版本，稳妥但占空间：

```java
class Solution {
    public Node copyRandomList(Node head) {
        if (head == null) return null;
        Map<Node, Node> map = new HashMap<>();
        for (Node cur = head; cur != null; cur = cur.next) {
            map.put(cur, new Node(cur.val));
        }
        for (Node cur = head; cur != null; cur = cur.next) {
            Node copy = map.get(cur);
            copy.next = map.get(cur.next);
            copy.random = map.get(cur.random);
        }
        return map.get(head);
    }
}
```
