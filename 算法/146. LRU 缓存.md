## 🧩 一、题意理解

实现一个 **最近最少使用（LRU, Least Recently Used）缓存机制**，
 要求：

- 支持两个操作：
  - `get(key)`：返回 key 对应的值，如果不存在返回 -1。
  - `put(key, value)`：写入键值对；
     如果容量满了，要删除**最久未使用**的键。
- 所有操作都要在 **O(1)** 时间内完成。

------

## 💡 二、思路分析

想要 `O(1)` 时间完成查找、插入、删除 ——
 必须结合两种结构：

| 功能           | 结构         | 说明                               |
| -------------- | ------------ | ---------------------------------- |
| 快速查找       | **HashMap**  | 通过 key 找到对应的节点            |
| 快速移动、删除 | **双向链表** | 用于维护访问顺序（最近使用放尾部） |

------

## ⚙️ 三、核心机制：双向链表 + HashMap

1. **HashMap** 存 key → Node（链表节点）的映射；
2. **双向链表** 存储节点顺序：
   - 最近访问的节点放在**尾部**；
   - 最久未访问的节点在**头部**；
3. 当容量满时：
   - 删除链表头节点；
   - 同时删除对应的 HashMap 键。

------

## 📊 四、关键操作图解

假设容量为 2：

| 操作     | 状态变化                           |
| -------- | ---------------------------------- |
| put(1,1) | [1]                                |
| put(2,2) | [1,2]                              |
| get(1)   | 访问1 → [2,1]                      |
| put(3,3) | 超容量 → 删除最久未使用的2 → [1,3] |
| get(2)   | 返回 -1                            |

------

## 🧱 五、伪代码逻辑

```
1. get(key):
   - 如果不存在：返回 -1
   - 如果存在：
       1. 把该节点移到链表尾部（表示最近使用）
       2. 返回 value

2. put(key, value):
   - 如果 key 已存在：
       1. 更新 value
       2. 把节点移到链表尾部
   - 如果 key 不存在：
       1. 若满了，删除链表头部节点（最久未使用）
       2. 插入新节点到链表尾部
```

## 🧑‍💻 六、Java 实现（手写双向链表版）

```java
import java.util.HashMap;

class LRUCache {
    // 定义双向链表节点
    class Node {
        int key, value;
        Node prev, next;
        Node(int k, int v) { key = k; value = v; }
    }

    private int capacity;
    private HashMap<Integer, Node> map;
    private Node head, tail; // 虚拟头尾节点

    public LRUCache(int capacity) {
        this.capacity = capacity;
        this.map = new HashMap<>();
        // 虚拟头尾节点（哨兵）
        head = new Node(-1, -1);
        tail = new Node(-1, -1);
        head.next = tail;
        tail.prev = head;
    }

    public int get(int key) {
        if (!map.containsKey(key)) return -1;
        Node node = map.get(key);
        moveToTail(node); // 最近使用，移到末尾
        return node.value;
    }

    public void put(int key, int value) {
        if (map.containsKey(key)) {
            Node node = map.get(key);
            node.value = value;
            moveToTail(node);
        } else {
            if (map.size() == capacity) {
                // 删除最久未使用节点（head.next）
                Node lru = head.next;
                removeNode(lru);
                map.remove(lru.key);
            }
            Node newNode = new Node(key, value);
            addToTail(newNode);
            map.put(key, newNode);
        }
    }

    // 把节点移到尾部（最新使用）
    private void moveToTail(Node node) {
        removeNode(node);
        addToTail(node);
    }

    // 删除节点
    private void removeNode(Node node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
    }

    // 在尾部插入节点
    private void addToTail(Node node) {
        Node prev = tail.prev;
        prev.next = node;
        node.prev = prev;
        node.next = tail;
        tail.prev = node;
    }
}
```

## ✅ 九、简洁版：用 `LinkedHashMap` 一行搞定！

```java
import java.util.LinkedHashMap;
import java.util.Map;

class LRUCache extends LinkedHashMap<Integer, Integer> {
    private int capacity;

    public LRUCache(int capacity) {
        super(capacity, 0.75F, true); // accessOrder=true 表示按访问顺序
        this.capacity = capacity;
    }

    public int get(int key) {
        return super.getOrDefault(key, -1);
    }

    public void put(int key, int value) {
        super.put(key, value);
    }

    @Override
    protected boolean removeEldestEntry(Map.Entry<Integer,Integer> eldest) {
        return size() > capacity;
    }
}
```

## 🧠 十、面试回答模板

> 我用的是 HashMap + 双向链表实现 LRU。
>  HashMap 用来 O(1) 查找节点，
>  双向链表维护访问顺序，最近访问的移到尾部，最久未使用的在头部。
>  每次 put 或 get 都把节点移到尾部。
>  当容量满时，删除头节点。
>  这种方式能保证所有操作 O(1) 时间复杂度。
