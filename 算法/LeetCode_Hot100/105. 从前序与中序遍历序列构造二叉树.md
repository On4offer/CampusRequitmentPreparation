# 🧩 LeetCode 105. Construct Binary Tree from Preorder and Inorder Traversal

------

## 📘题意说明

给定两个数组：`preorder` 和 `inorder`，它们分别是二叉树的**前序遍历**和**中序遍历**的结果。根据这两个遍历结果，构造出该二叉树。

**示例：**

```
输入:
preorder = [3,9,20,15,7]
inorder = [9,3,15,20,7]

输出:
       3
      / \
     9  20
        /  \
       15   7
```

------

## 🧠思路与指针设计

这题的核心在于利用前序和中序遍历的特点来重建二叉树。

### ✳️ 核心思想：

1. **前序遍历**：根节点是第一个元素，接下来是左子树，最后是右子树。
2. **中序遍历**：左子树在根节点的左边，右子树在根节点的右边。

### ✳️ 关键指针设计：

- **preorder** 中的第一个元素是当前子树的根节点。
- **inorder** 中根节点将二叉树分成左右两个子树部分。

通过递归处理左右子树，我们能够逐步构建出整个二叉树。

------

## 🧾 步骤伪代码

```
1. 如果 preorder 或 inorder 为空，则返回 null。
2. 从 preorder 中取出第一个元素作为根节点。
3. 在 inorder 中找到该根节点的位置，分割成左右子树。
4. 递归构建左子树和右子树。
5. 返回根节点。
```

## 💻 Java 标准实现

```java
/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
class Solution {
    public TreeNode buildTree(int[] preorder, int[] inorder) {
        return buildTreeHelper(preorder, 0, preorder.length - 1, inorder, 0, inorder.length - 1);
    }

    private TreeNode buildTreeHelper(int[] preorder, int preStart, int preEnd, int[] inorder, int inStart, int inEnd) {
        if (preStart > preEnd || inStart > inEnd) return null;

        // 前序遍历的第一个元素是根节点
        int rootVal = preorder[preStart];
        TreeNode root = new TreeNode(rootVal);

        // 在中序遍历中找到根节点的位置
        int rootIndex = -1;
        for (int i = inStart; i <= inEnd; i++) {
            if (inorder[i] == rootVal) {
                rootIndex = i;
                break;
            }
        }

        // 计算左子树的节点数量
        int leftTreeSize = rootIndex - inStart;

        // 构建左子树和右子树
        root.left = buildTreeHelper(preorder, preStart + 1, preStart + leftTreeSize, inorder, inStart, rootIndex - 1);
        root.right = buildTreeHelper(preorder, preStart + leftTreeSize + 1, preEnd, inorder, rootIndex + 1, inEnd);

        return root;
    }
}
```

## 🧩 图解说明

假设输入如下：

```
preorder = [3,9,20,15,7]
inorder = [9,3,15,20,7]
```

1. **根节点**是`preorder[0] = 3`。
2. 在`inorder`中找到3的位置，分割为`[9]`（左子树）和 `[15, 20, 7]`（右子树）。
3. 左子树由 `preorder[1] = 9` 和 `inorder[0] = 9` 构成。
4. 右子树根节点是`preorder[2] = 20`，在`inorder`中找到20的位置，分割为`[15]`（左子树）和 `[7]`（右子树）。
5. 递归构建左右子树，最终得到：

```
       3
      / \
     9   20
        /  \
       15   7
```

>
> **关键点：**
>
> - 前序遍历的第一个元素是根节点；
> - 中序遍历中找到根节点后，将左右子树划分；
> - 递归构建左子树和右子树。

>  时间复杂度是 O(n)，空间复杂度是 O(n)。

## ⚠️ 常见坑点

1. 在 `inorder` 中查找根节点时要遍历整个子树，注意处理根节点位置的索引。
2. 递归过程中，要确保区间的正确传递，避免越界。
3. 在递归构建左、右子树时，正确地更新 `preStart` 和 `inStart` 等指针。
