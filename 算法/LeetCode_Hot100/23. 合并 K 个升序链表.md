------

## 🧩 一、题意理解

给定一个包含 `k` 个升序链表的数组 `lists`，
 要求将所有链表**合并为一个升序链表**并返回。

------

## 💡 二、思路分析

### 思路一：优先队列（小顶堆）——最常见做法 ✅

- 核心思想：每次从 k 个链表中取当前最小的节点。
- 利用 **PriorityQueue（小顶堆）** 保存所有链表的当前头节点，堆顶是当前最小节点。

#### 思路流程：

1. 将所有非空链表的头节点放入最小堆；
2. 每次取出堆顶节点（即最小值）；
3. 将该节点的 `next` 加入堆；
4. 重复直到堆空。

#### 时间复杂度：

- 每个节点入堆、出堆各一次；
- 堆大小 ≤ k；
- 总复杂度：`O(N log k)`
   （N = 所有链表节点总数）

#### 空间复杂度：

- 堆中最多存 k 个节点：`O(k)`

------

## 🧱 三、伪代码

```
function mergeKLists(lists):
    pq = new PriorityQueue based on node.val
    for list in lists:
        if list != null:
            pq.offer(list)
    
    dummy = new ListNode(-1)
    tail = dummy

    while pq not empty:
        node = pq.poll()     // 最小节点
        tail.next = node
        tail = node
        if node.next != null:
            pq.offer(node.next)

    return dummy.next
```

## 🧑‍💻 四、Java 标准实现

```java
import java.util.PriorityQueue;

class Solution {
    public ListNode mergeKLists(ListNode[] lists) {
        if (lists == null || lists.length == 0) return null;

        // 创建最小堆，按节点值排序
        PriorityQueue<ListNode> pq = new PriorityQueue<>(
            (a, b) -> a.val - b.val
        );

        // 初始化，将每个链表的头节点放入堆中
        for (ListNode head : lists) {
            if (head != null) pq.offer(head);
        }

        // 虚拟头结点，方便拼接
        ListNode dummy = new ListNode(-1);
        ListNode tail = dummy;

        // 逐步取出堆中最小节点
        while (!pq.isEmpty()) {
            ListNode node = pq.poll(); // 取出最小值
            tail.next = node;          // 加入结果链表
            tail = tail.next;          // 移动尾指针

            if (node.next != null) {   // 当前链表未结束
                pq.offer(node.next);
            }
        }

        return dummy.next;
    }
}
```

## 🚀 一、思路二：分治归并法（归并排序思想）

核心思想：

- 两两合并链表，
- 不断缩小问题规模，
- 最终得到一个完整的有序链表。

就像归并排序那样，每一轮都把相邻的链表合并成一个。

------

## 💡 二、流程解析

假设有 `k` 个链表：

1. 如果 `k == 0`：返回 null

2. 如果 `k == 1`：直接返回该链表

3. 否则：将列表分为两半，递归合并：

   ```
   left = mergeKLists(lists[0...mid])
   right = mergeKLists(lists[mid+1...k-1])
   return mergeTwoLists(left, right)
   ```

------

## 🔧 三、辅助函数

需要一个合并两个升序链表的函数（和 LeetCode 21 题相同）：

```java
private ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(-1);
    ListNode cur = dummy;
    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) {
            cur.next = l1;
            l1 = l1.next;
        } else {
            cur.next = l2;
            l2 = l2.next;
        }
        cur = cur.next;
    }
    cur.next = (l1 != null) ? l1 : l2;
    return dummy.next;
}
```

## 🧑‍💻 四、完整 Java 实现

```java
class Solution {
    public ListNode mergeKLists(ListNode[] lists) {
        if (lists == null || lists.length == 0) return null;
        return merge(lists, 0, lists.length - 1);
    }

    // 分治合并函数
    private ListNode merge(ListNode[] lists, int left, int right) {
        if (left == right) return lists[left];
        if (left > right) return null;
        
        int mid = (left + right) / 2;
        ListNode l1 = merge(lists, left, mid);
        ListNode l2 = merge(lists, mid + 1, right);
        return mergeTwoLists(l1, l2);
    }

    // 合并两个链表
    private ListNode mergeTwoLists(ListNode l1, ListNode l2) {
        ListNode dummy = new ListNode(-1);
        ListNode cur = dummy;
        while (l1 != null && l2 != null) {
            if (l1.val <= l2.val) {
                cur.next = l1;
                l1 = l1.next;
            } else {
                cur.next = l2;
                l2 = l2.next;
            }
            cur = cur.next;
        }
        cur.next = (l1 != null) ? l1 : l2;
        return dummy.next;
    }
}
```

## 🧩 七、两种解法对比总结

| 对比项     | 堆解法                 | 分治解法                  |
| ---------- | ---------------------- | ------------------------- |
| 思想       | 最小堆，每次取最小节点 | 归并排序，两两合并        |
| 时间复杂度 | O(N log k)             | O(N log k)                |

| 优势       | 简洁高效，代码短       | 可读性好，结构清晰        |
| 缺点       | 需要堆结构             | 递归可能栈溢出（极大k时） |

## 🧾 八、面试回答模板（总结式回答）

> **回答示例：**
>
> 这题我有两种常用写法。
>  第一种是用 **PriorityQueue（最小堆）**，每次取出当前最小节点，时间复杂度 O(N log k)。
>  第二种是用 **分治归并**，将 k 个链表两两合并，类似归并排序，也能达到 O(N log k)。
>  我更推荐堆的写法，代码更直观，适合高并发下的按序合并场景。
