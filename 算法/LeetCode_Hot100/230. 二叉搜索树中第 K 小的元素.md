------

## 🧩 一、题目理解

> 给定一棵二叉搜索树（BST）的根节点 `root`，以及一个整数 `k`，
>  请返回 **BST 中第 k 小的节点值**。

------

## 🌲 二、核心知识点

**二叉搜索树（BST）的性质：**

> - 中序遍历（左 → 根 → 右）会得到一个**升序序列**。
> - 因此，只要中序遍历到第 k 个节点，就能得到第 k 小的值。

------

## 💡 三、思路分析

有两种常见解法：

### ✅ 方法1：中序遍历（递归）

- 对 BST 进行中序遍历；
- 使用一个计数器记录当前访问到第几个节点；
- 当计数器 == k 时，返回该节点值。

### ✅ 方法2：中序遍历（迭代，用栈模拟）

- 用栈手动模拟递归；
- 每次往左下走到底；
- 弹栈时 ++count；
- 当 count == k 时返回。

> ✅ 推荐：迭代法更适合面试手写，不容易写错。

------

## 🔧 四、关键伪代码

```
stack = new Stack()
cur = root
count = 0

while cur != null or stack not empty:
    while cur != null:
        stack.push(cur)
        cur = cur.left

    cur = stack.pop()
    count++
    if count == k:
        return cur.val

    cur = cur.right
```

------

## 💻 五、Java 实现（迭代中序法）

```java
import java.util.Stack;

class Solution {
    public int kthSmallest(TreeNode root, int k) {
        Stack<TreeNode> stack = new Stack<>();
        TreeNode cur = root;
        int count = 0;

        while (cur != null || !stack.isEmpty()) {
            // 一直往左走到底
            while (cur != null) {
                stack.push(cur);
                cur = cur.left;
            }

            cur = stack.pop(); // 访问当前节点
            count++;

            if (count == k) return cur.val; // 找到第k小

            cur = cur.right; // 转向右子树
        }

        return -1; // 理论不会执行到这里
    }
}
```

### **LeetCode 第230题：二叉搜索树中第K小的元素**

------

## 💡题目要求

> 给定一棵二叉搜索树（BST），找到树中的第 `k` 小的节点值。

**示例：**

```
输入：root = [3,1,4,null,2], k = 1
输出：1
```

------

## 🧠 思路分析（中序遍历）

这是典型的“二叉搜索树中查找第 `k` 小元素”问题。

### 1️⃣ 核心思路

- 在二叉搜索树（BST）中，中序遍历会按升序访问所有节点；
- 因此，我们可以通过中序遍历的顺序来找到第 `k` 小的节点；
- 使用递归或迭代的方式遍历二叉树，并且每访问一个节点就计数，当计数等于 `k` 时返回当前节点的值。

### 2️⃣ 中序遍历步骤：

- 递归地遍历左子树；
- 访问当前节点；
- 递归地遍历右子树；
- 在遍历过程中，记录访问到的节点数，当计数等于 `k` 时，返回当前节点的值。

------

## ✅ 代码实现（最优写法）

```java
class Solution {
    private int count = 0; // 计数器，用于记录访问的节点数
    private int result = -1; // 记录第k小的元素

    public int kthSmallest(TreeNode root, int k) {
        inorderTraversal(root, k); // 从根节点开始中序遍历
        return result; // 返回结果
    }

    private void inorderTraversal(TreeNode node, int k) {
        if (node == null || count >= k) {
            return; // 如果节点为空或者已经找到了第k个元素，则返回
        }

        // 先遍历左子树
        inorderTraversal(node.left, k);

        // 访问当前节点
        count++; // 访问节点后，计数器增加
        if (count == k) {
            result = node.val; // 找到第k个元素，保存结果
            return; // 不再继续遍历
        }

        // 最后遍历右子树
        inorderTraversal(node.right, k);
    }
}
```

## 🧩 复杂度分析

| 项目       | 复杂度                               |
| ---------- | ------------------------------------ |
| 时间复杂度 | O(H + k)（H为树的高度，k为目标位置） |
| 空间复杂度 | O(H)                                 |

- **时间复杂度**：`O(H + k)`，在最坏情况下，需要遍历到 `k` 个节点，`H` 是树的高度。
- **空间复杂度**：`O(H)`，递归调用栈的空间开销，其中 `H` 是树的高度。

## 💬 面试官可能追问

1. **为什么是中序遍历？**
   - 因为二叉搜索树的性质保证了中序遍历按升序顺序访问所有节点，因此可以利用中序遍历来找到第 `k` 小的元素。
2. **能否使用非递归的方式进行中序遍历？**
   - 可以使用栈来模拟递归过程，进行非递归的中序遍历。
3. **如果给定的是普通的二叉树，而非二叉搜索树，能否用中序遍历找第 `k` 小元素？**
   - 在普通二叉树中，中序遍历的顺序是不确定的，因此不能直接使用中序遍历来解决该问题，必须先对树进行排序。
