------

## 💡题目要求

> 给你一个链表，删除链表的**倒数第 n 个节点**，并返回头节点。

示例：

```
输入：1 -> 2 -> 3 -> 4 -> 5, n = 2  
输出：1 -> 2 -> 3 -> 5
```

------

## 🧠思路分析（双指针法）

这是典型的“快慢指针”问题。

### 1️⃣ 核心思路

- 设置两个指针 `fast` 和 `slow`，初始都指向头节点；
- 先让 `fast` 向前移动 `n` 步；
- 然后让 `slow` 和 `fast` 一起移动，直到 `fast` 到达链表末尾（null）；
- 此时 `slow` 的下一个节点就是**要删除的节点**；
- 直接跳过该节点即可（`slow.next = slow.next.next`）。

假设链表m个节点，删除倒数第n个，则左侧有m-n个节点，fast右移了n步之后（左侧有n+1个节点），slow再跟着fast一起走完剩余m-n-1个节点，所以slow走了m-n-1步，指向第m-n-1个节点，下一个节点就是倒数第n个节点，删除此时的slow下一个节点即可。

### 2️⃣ 特殊情况

如果删除的是第一个节点（例如 `n` 恰好等于链表长度），
 我们可以在链表头前加一个**虚拟头节点 dummy**，方便统一处理。

加上头结点之后的链表长度为 m，删除倒数第 n 个节点，fast 先移动 n+1 步，然后 fast、slow 一起走，直到 fast==null，此时 slow 走了 m-n 步，停在第 m-n 个节点，`slow.next` 即是倒数第 n 个节点 → 执行删除

------

## ✅代码实现（最优写法）

```java
class Solution {
    public ListNode removeNthFromEnd(ListNode head, int n) {
        // 创建虚拟头节点，防止删除第一个节点时出错
        ListNode dummy = new ListNode(0, head);
        ListNode fast = dummy;
        ListNode slow = dummy;

        // 1. fast先走n+1步（保证slow停在删除节点的前一个节点）
        for (int i = 0; i <= n; i++) {
            fast = fast.next;
        }

        // 2. fast和slow一起走，直到fast到达末尾
        while (fast != null) {
            fast = fast.next;
            slow = slow.next;
        }

        // 3. 删除slow.next
        slow.next = slow.next.next;

        // 4. 返回真正的头节点
        return dummy.next;
    }
}
```

| 空间复杂度 | O(1)                |

## 💬面试官可能追问

1. **为什么要用dummy节点？**

   - 防止删除的是第一个节点导致`head`丢失；
   - 统一逻辑，避免特殊判断。

2. **如果不想用dummy节点，怎么写？**
    可以先遍历计算链表长度，然后再删除第 `len - n` 个节点，但需要两次遍历，效率略低。

3. **能否写出双遍历法？**

   ```java
   public ListNode removeNthFromEnd(ListNode head, int n) {
       int len = 0;
       ListNode cur = head;
       while (cur != null) {
           len++;
           cur = cur.next;
       }
   
       if (n == len) return head.next;
   
       cur = head;
       for (int i = 1; i < len - n; i++) {
           cur = cur.next;
       }
       cur.next = cur.next.next;
       return head;
   }
   ```

## 🧱总结模板（可背诵）

> 面试题：删除链表倒数第 N 个节点
>  **思路：** 用双指针（快慢指针），fast 先走 n 步，然后 slow 和 fast 一起走，直到 fast 到尾，此时 slow 的下一个节点即要删除。
>  **关键技巧：** 用虚拟头节点 dummy 统一处理删除头节点情况。
>  **时间复杂度 O(L)，空间复杂度 O(1)。**
