# 一、题目表述（面试官口吻）

> 现在有一套 Spring Boot 电商接口，需要对“下单、支付、个人中心”等业务做**登录校验**。要求：
>
> 1. 登录后将用户态保存到 Redis；
> 2. 请求携带 token；
> 3. 未登录直接拦截；
> 4. 已登录为本次请求注入用户上下文并**滑动续期**；
> 5. 便于在 Service 层随取随用，不用层层传参。
>     请设计整体方案与核心代码，并说明你的取舍与考虑。

------

# 二、总体思路（30 秒版）

- **登录态存储**：Redis（Hash）按 `login:user:{token}` 保存 `UserDTO` 字段；
- **鉴权入口**：Spring MVC `HandlerInterceptor`；
- **上下文传递**：`ThreadLocal<UserDTO>`；
- **滑动续期**：每次请求命中用户态则刷新 TTL；
- **路径策略**：开放接口放行，其余强校验；
- **统一清理**：`afterCompletion` 里清 `ThreadLocal` 防脏数据。

------

# 三、系统设计（含时序）

**时序**：
 Client → (带 Authorization: token) → `RefreshTokenInterceptor`（解析+续期+注入） → `LoginInterceptor`（强校验） → Controller → Service(从 ThreadLocal 取用户) → … → `afterCompletion` 清理。

**Redis Key 设计**：

- `login:user:{token}`：Hash，field 为 `id/nick/icon/...`，TTL=30min（示例）
- 可选二级索引：`login:token:uid:{uid} -> {token}`（实现单端登录/踢人下线）

# 四、核心代码（可落地）

### 1) 用户上下文持有者

```java
public class UserHolder {
    private static final ThreadLocal<UserDTO> TL = new ThreadLocal<>();
    public static void save(UserDTO user){ TL.set(user); }
    public static UserDTO get(){ return TL.get(); }
    public static void remove(){ TL.remove(); }
}
```

### 2) 续期&注入（弱校验，放行）

```java
@Component
public class RefreshTokenInterceptor implements HandlerInterceptor {
    @Resource private StringRedisTemplate stringRedisTemplate;
    private static final String LOGIN_USER_KEY = "login:user:";
    private static final long LOGIN_TTL_MIN = 30;

    @Override
    public boolean preHandle(HttpServletRequest req, HttpServletResponse resp, Object h) {
        String token = req.getHeader("Authorization");
        if (token == null || token.isEmpty()) return true;

        String key = LOGIN_USER_KEY + token;
        Map<Object, Object> map = stringRedisTemplate.opsForHash().entries(key);
        if (map == null || map.isEmpty()) return true;

        UserDTO user = BeanUtil.fillBeanWithMap(map, new UserDTO(), true);
        UserHolder.save(user);
        stringRedisTemplate.expire(key, Duration.ofMinutes(LOGIN_TTL_MIN)); // 滑动续期
        return true;
    }

    @Override
    public void afterCompletion(HttpServletRequest req, HttpServletResponse resp, Object h, Exception ex) {
        UserHolder.remove();
    }
}
```

### 3) 强校验拦截器（必须登录）

```java
@Component
public class LoginInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest req, HttpServletResponse resp, Object handler) throws Exception {
        if (UserHolder.get() == null) {
            resp.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            resp.setContentType("application/json;charset=UTF-8");
            resp.getWriter().write("{\"code\":401,\"msg\":\"未登录或已过期\"}");
            return false;
        }
        return true;
    }
}
```

### 4) Web 配置（路径放行/保护）

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Resource private RefreshTokenInterceptor refreshTokenInterceptor;
    @Resource private LoginInterceptor loginInterceptor;

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(refreshTokenInterceptor)
                .addPathPatterns("/**")
                .excludePathPatterns("/user/login", "/user/register", "/captcha/**", "/public/**", "/doc.html", "/swagger/**");

        registry.addInterceptor(loginInterceptor)
                .addPathPatterns("/order/**", "/user/**", "/cart/**", "/pay/**")
                .excludePathPatterns("/user/login", "/user/register");
    }
}
```

### 5) 登录成功写入 Redis（示例）

```java
public String login(LoginForm form){
    User user = userService.verify(form); // 校验账号密码
    String token = UUID.randomUUID().toString(true);
    UserDTO dto = BeanUtil.copyProperties(user, UserDTO.class);
    Map<String, Object> map = BeanUtil.beanToMap(dto, new HashMap<>(),
            CopyOptions.create().setIgnoreNullValue(true)
                    .setFieldValueEditor((k,v) -> v==null? null : v.toString()));
    String key = "login:user:" + token;
    stringRedisTemplate.opsForHash().putAll(key, map);
    stringRedisTemplate.expire(key, Duration.ofMinutes(30));
    return token;
}
```

> 说明：存 **Hash** 是因为用户对象多字段、需局部更新/读取，且内存更省；Java 端天然用 `Map` 对应 Redis Hash 的 `HSET/HGETALL`。

# 五、边界与异常处理

- **Token 缺失/格式非法**：直接按未登录处理；
- **Redis Key 失效**：视为过期登录，返回 401；
- **线程复用污染**：务必在 `afterCompletion` 清 `ThreadLocal`；
- **网关/多层反向代理**：统一在网关层透传 `Authorization`；
- **跨服务调用**：下游服务可复用同拦截器，或在网关做一次鉴权并下发 `X-User` 头（零侵入方案）。

# 六、与 JWT 的对比与取舍（面试高频）

| 方案                  | 优点                                           | 缺点                                                   | 使用场景                         |
| --------------------- | ---------------------------------------------- | ------------------------------------------------------ | -------------------------------- |
| Redis-Session（本文） | 可服务端**主动失效**、可续期、字段可变更即生效 | 依赖 Redis、维护成本略高                               | 管理后台、电商登录、需要踢人下线 |
| JWT（无状态）         | 无集中存储、扩展方便                           | 不能主动失效（需黑名单）、续期复杂、敏感变更不即刻生效 | 高并发、读多写少的公开 API       |

> 若必须 JWT：可采用**短有效期 + 刷新 token**，或维护**黑名单**（仍需 Redis）。

# 七、可扩展能力（加分点）

1. **单点登录/踢人下线**：维护 `uid->token` 索引，新登录先踢旧 token；
2. **注解化控制**：自定义 `@LoginRequired`，在拦截器里识别 `HandlerMethod` 判断是否放行；
3. **多端并发**：改成 `uid -> set(token)` 支持 PC/APP 多端共存；
4. **权限/角色**：在 Redis 再挂 `roles/permissions` 字段，或接入 RBAC；
5. **风控**：对异常 IP/设备指纹拒绝续期；
6. **分布式链路**：把 `UserId` 写入 MDC，日志自动带用户维度；
7. **限流配合**：登录态维度做 QPS 限流（滑动窗口 / 令牌桶）。

# 八、测试清单（落地即用）

- **正常流**：登录→拿 token 访问受保护接口→返回 200；
- **未登录**：不带 token 访问→返回 401；
- **过期**：模拟 TTL 过期→返回 401；
- **续期**：连续访问间隔 < TTL，观察 key 的剩余 TTL 变大；
- **清理**：压力测后随机比对 `ThreadLocal` 是否残留（可打断点/日志）；
- **并发**：100 并发访问校验无竞态异常；
- **踢人**（可选）：旧 token 失效后访问→401。

# 九、标准面试回答模板（背诵版）

> 我采用 **Interceptor + Redis(Hash) + ThreadLocal** 的方案。登录成功后把 `UserDTO` 字段化存到 `login:user:{token}`，TTL 30 分钟。请求到来时，`RefreshTokenInterceptor` 从头里取 token，命中 Redis 则反序列化为 `UserDTO` 注入 `ThreadLocal`，同时滑动续期；随后 `LoginInterceptor` 做强校验，若 `ThreadLocal` 为空则返回 401。之所以用 **Hash** 而不是 JSON String，是因为可以**局部读取/更新**、更省内存；用 `ThreadLocal` 是为了在同线程内**无侵入**传递用户上下文，最后在 `afterCompletion` 清理防止线程复用污染。若需要无状态，我会切到 JWT，但会配合**短期 token + refresh token 或黑名单**来解决主动失效的问题。
