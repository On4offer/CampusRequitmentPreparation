**Answer:**

在 Java 虚拟机 (JVM) 里，**符号引用 (Symbolic Reference)** 和 **直接引用 (Direct Reference)** 是两个重要概念，主要出现在 **类加载过程** 和 **运行时链接** 里。

------

## 1. 符号引用 (Symbolic Reference)

- **定义**：
   在编译期，类并不知道其他类的具体内存地址，所以编译器在 `.class` 文件的常量池中，用 **一组符号（字符串、索引等）来表示引用目标**。
   这种“名字标识”形式的引用就叫 **符号引用**。

- **例子**

  ```java
  String s = new String("hello");
  ```

  编译成字节码后：

  - 常量池里保存 `"java/lang/String"`，这是符号引用；
  - `"hello"` 也是以符号形式保存。

- **特点**

  1. 存在于 **Class 文件常量池**。
  2. 与具体内存地址无关。
  3. 在编译期生成，在类还没加载前就能确定。

------

## 2. 直接引用 (Direct Reference)

- **定义**：
   当类加载器把类加载进 JVM，并经过 **解析 (Resolution)** 阶段后，JVM 会把符号引用解析成 **直接引用**。
   直接引用指向的是 **内存地址、偏移量或句柄**，可以直接定位到目标。
- **特点**
  1. 存在于 **运行时内存**（方法区、堆等）。
  2. 与具体内存布局绑定。
  3. 有了直接引用，就能真正访问目标（类、方法、字段等）。

------

## 3. 两者关系

- **符号引用 → 解析 → 直接引用**
- 过程：
  1. 编译时：只知道名字（符号引用）。
  2. 类加载解析：JVM 查找实际内存位置，把名字翻译成地址（直接引用）。
  3. 运行时：用直接引用访问方法或字段。

------

## 4. 类比理解

- **符号引用**：像写在简历上的“公司名称 + 部门名称”。
- **直接引用**：入职后拿到的“工位号 + 办公室地址”。

前者是“描述”，后者是“内存坐标”。

------

## 5. 举例说明

比如调用一个方法：

```java
obj.hello();
```

- **符号引用**：常量池中保存 `"HelloWorld/hello()V"` 这样的描述符。
- **直接引用**：类加载解析后，变成方法表里的入口地址（偏移量）。

------

## 6. 总结

- **符号引用**：编译期产生，存在常量池里，是“名字形式”的引用，不依赖内存地址。
- **直接引用**：运行期产生，指向实际内存地址或偏移，可以直接访问目标。
- **关系**：JVM 在类加载解析阶段，会把符号引用解析成直接引用。

------

📌 **一句话总结**

> 符号引用是编译期的“名字”，直接引用是运行期的“地址”；类加载解析阶段负责把前者翻译成后者。
