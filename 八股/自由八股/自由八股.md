## ✅ 一、JDK 动态代理

### 🔹 概念

JDK 动态代理是 Java 提供的一种运行时生成代理类的机制，用于拦截接口方法的调用并进行增强（如日志、事务、权限等）。

### 🔹 特点

- 代理类在运行时生成
- 代理的必须是 **接口**
- 基于 `java.lang.reflect.Proxy` 和 `InvocationHandler`

### 🔹 简要代码示例

```java
interface UserService {
    void login(String user);
}

class UserServiceImpl implements UserService {
    public void login(String user) {
        System.out.println("用户登录：" + user);
    }
}

class MyHandler implements InvocationHandler {
    private Object target;

    public MyHandler(Object target) {
        this.target = target;
    }

    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("方法前置增强");
        Object result = method.invoke(target, args);
        System.out.println("方法后置增强");
        return result;
    }
}

// 创建代理对象
UserService proxy = (UserService) Proxy.newProxyInstance(
        UserServiceImpl.class.getClassLoader(),
        new Class[]{UserService.class},
        new MyHandler(new UserServiceImpl())
);

proxy.login("张三");
```

## ✅ 二、MyBatis 中的动态代理机制

### 🔹 作用

MyBatis 使用 JDK 动态代理生成 `Mapper 接口` 的代理对象（实现类），使开发者只需调用接口方法即可执行 SQL，而无需自己写 DAO 实现类。

### 🔹 关键类

- `MapperProxy`：实现了 `InvocationHandler`
- `MapperProxyFactory`：代理工厂，生成代理类
- `MapperRegistry`：管理所有 Mapper 接口的注册信息
- `SqlSession.getMapper(...)`：返回代理对象的核心入口

### 🔹 调用链图解

```
UserMapper mapper = sqlSession.getMapper(UserMapper.class)
            ↓
MapperRegistry.getMapper()
            ↓
MapperProxyFactory.newInstance()
            ↓
Proxy.newProxyInstance(...) ⬅️ JDK 动态代理
            ↓
调用接口方法 → MapperProxy.invoke()
            ↓
从 Configuration 中获取 MappedStatement
            ↓
Executor 执行 SQL → 封装返回结果
```

### 🔹 源码核心：`MapperProxy#invoke`

```java
public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
    if (Object.class.equals(method.getDeclaringClass())) {
        return method.invoke(this, args); // 处理 toString 等基础方法
    }

    MapperMethod mapperMethod = cachedMapperMethod(method); // 缓存方法
    return mapperMethod.execute(sqlSession, args); // 执行 SQL
}
```

## ✅ 三、责任链设计模式（Chain of Responsibility）

### 🔹 概念

责任链是一种行为型设计模式，将多个处理器连成一条链，**每个处理器决定是否处理请求，或者将请求传递给下一个处理器**。

### 🔹 模式结构图

```
Client
  ↓
Handler1 → Handler2 → Handler3 → ... → null
```

### 🔹 应用场景

- Servlet Filter
- Spring Interceptor
- MyBatis 插件机制（核心）

------

## ✅ 四、MyBatis 插件中的责任链模式

### 🔹 可拦截的四大核心接口（注解指定）

```java
@Intercepts({
    @Signature(type = Executor.class, method = "update", args = {MappedStatement.class, Object.class})
})
public class MyPlugin implements Interceptor {
    public Object intercept(Invocation invocation) throws Throwable {
        // 前置增强
        Object result = invocation.proceed(); // 调用下一个插件或目标方法
        // 后置增强
        return result;
    }

    public Object plugin(Object target) {
        return Plugin.wrap(target, this); // 包装为代理
    }
}
```

### 🔹 插件责任链执行机制

```text
Executor executor = new SimpleExecutor();

executor = Plugin.wrap(executor, plugin1);
executor = Plugin.wrap(executor, plugin2);
executor = Plugin.wrap(executor, plugin3);

执行 executor.update(...) 时，链式调用：
plugin3.intercept → plugin2.intercept → plugin1.intercept → SimpleExecutor.update
```

### 🔹 核心类：`Plugin.wrap(Object target, Interceptor interceptor)`

```java
public static Object wrap(Object target, Interceptor interceptor) {
    Class<?> type = target.getClass();
    Class<?>[] interfaces = getAllInterfaces(type);
    return Proxy.newProxyInstance(
        type.getClassLoader(), interfaces,
        new Plugin(target, interceptor)
    );
}
```

## ✅ 五、小结归纳（面试速答模板）

> MyBatis 使用 **JDK 动态代理** 为 Mapper 接口生成实现类，拦截方法调用并委托给内部 SQL 执行逻辑，主要通过 `MapperProxy` 实现。其插件机制基于 **责任链模式**，通过 `Plugin.wrap()` 将拦截器层层包裹目标对象，形成链式调用结构，便于实现功能增强，如分页、日志、安全审计等。

# Mybatis中Dao接口和XML文件的SQL如何建立关联

MyBatis 中通过 **Mapper 接口（即 DAO 接口）** 与 **Mapper XML 文件** 中的 SQL 建立关联，核心有三个条件：

------

## ✅ 一、关联的三个关键条件

| 条件                                                         | 说明                     |
| ------------------------------------------------------------ | ------------------------ |
| ① 接口全限定名与 XML `<mapper>` 的 namespace 一致            | 保证一个接口对应一个 XML |
| ② 方法名与 XML 中 SQL 的 `id` 一致                           | 方法调用 → 映射 SQL      |
| ③ 方法参数和返回值类型与 XML 中 `parameterType`、`resultType` 对应 | 保证类型正确传递和解析   |

------

## ✅ 二、关联机制示意

### 1️⃣ Mapper 接口

```java
// 文件名：UserMapper.java
public interface UserMapper {
    User selectUserById(int id);
}
```

### 2️⃣ Mapper XML 文件

```xml
<!-- 文件名：UserMapper.xml -->
<mapper namespace="com.example.mapper.UserMapper">
    <select id="selectUserById" parameterType="int" resultType="com.example.pojo.User">
        SELECT * FROM users WHERE id = #{id}
    </select>
</mapper>
```

> ✅ **说明：**
>
> - `namespace` = 接口全限定名（包名+类名）
> - `<select id="方法名">` → 建立绑定
> - 参数类型和返回类型与方法保持一致

------

## ✅ 三、加载与使用过程

1. **MyBatis 启动时加载所有 `<mapper>` XML 文件**
2. **把 `<mapper>` 注册进 Configuration 的 MapperRegistry**
3. **调用 `sqlSession.getMapper(UserMapper.class)`**
4. **通过 JDK 动态代理生成接口代理类**
5. **接口方法被调用时，通过 namespace + id 查找 SQL 执行**

------

## ✅ 四、Spring Boot 自动绑定示例（实战）

```yaml
# application.yml
mybatis:
  mapper-locations: classpath:mapper/*.xml
  type-aliases-package: com.example.pojo
@Mapper // 或 @MapperScan("com.example.mapper")
public interface UserMapper {
    User selectUserById(int id);
}
```

Spring Boot 会自动扫描接口与 XML 映射，无需手动注册。

------

## ✅ 五、注意事项（面试高频）

| 项目                     | 说明                           |
| ------------------------ | ------------------------------ |
| `namespace` 不一致       | 会导致接口找不到 SQL 映射      |
| `id` 与方法名不一致      | 调用时抛出绑定异常             |
| 多参数时未使用 `@Param`  | 会导致 SQL 参数无法映射        |
| XML 未加载或路径配置错误 | 接口方法会报空指针或未映射异常 |

------

## ✅ 六、小结口播版（面试总结）

> MyBatis 通过接口与 XML 的绑定实现 SQL 映射，要求 Mapper 接口的 **全限定名必须与 XML 的 namespace 完全一致**，接口方法名必须与 SQL 的 `<select>/<insert>` 的 `id` 相同。底层通过动态代理方式实现方法调用 → SQL 执行的映射查找。Spring Boot 中通常使用 `@Mapper` 或 `@MapperScan` 自动完成绑定。
