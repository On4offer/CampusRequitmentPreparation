------

## 1. 术语解释

在 Java 中，线程一旦启动（`start()`），是不能被强制“杀死”的。
 JDK 提供的 `stop()` 方法早已废弃，因为它会导致资源不一致和数据破坏。
 **正确做法**：通过 **协作式停止**，即线程自己检查一个标志位，优雅地退出。

------

## 2. 原理剖析

线程的运行就是不停执行 `run()` 方法里的逻辑。

- 如果我们需要让它停下来，就要在逻辑中加一个**可见的中断信号**。
- Java 提供了 **`interrupt()` 和 `isInterrupted()`** 来做这件事。
- 或者自己定义一个 **`volatile boolean flag`** 作为“停止标志”。

------

## 3. 使用方式

### 方法 1：使用 `interrupt()`（推荐）

```java
class MyThread extends Thread {
    @Override
    public void run() {
        while (!isInterrupted()) { // 检查是否被中断
            System.out.println("线程运行中...");
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                System.out.println("线程被唤醒并中断");
                break;  // 退出循环
            }
        }
        System.out.println("线程结束");
    }
}

public class Demo {
    public static void main(String[] args) throws InterruptedException {
        MyThread t = new MyThread();
        t.start();
        Thread.sleep(3000);
        t.interrupt();  // 通知线程中断
    }
}
```

### 方法 2：使用 `volatile` 标志位

```java
class MyTask implements Runnable {
    private volatile boolean running = true;

    @Override
    public void run() {
        while (running) {
            System.out.println("执行任务中...");
        }
        System.out.println("任务结束");
    }

    public void stop() {
        running = false;
    }
}

public class Demo {
    public static void main(String[] args) throws InterruptedException {
        MyTask task = new MyTask();
        Thread t = new Thread(task);
        t.start();
        Thread.sleep(2000);
        task.stop(); // 改变标志位，线程退出
    }
}
```

------

## 4. 对比分析

| 方法              | 优点                     | 缺点                   | 场景     |
| ----------------- | ------------------------ | ---------------------- | -------- |
| `stop()`（废弃）  | 直接杀死                 | 破坏数据一致性         | 不推荐   |
| `interrupt()`     | 标准做法，线程能感知中断 | 需要在任务里检查并处理 | 常用     |
| `volatile` 标志位 | 简单直观                 | 不能处理中断阻塞       | 轻量任务 |

------

## 5. 项目实践

- **定时任务**：使用 `ScheduledExecutorService`，任务可取消。
- **大批量任务**：配合 `Future.cancel(true)`，内部也是调用 `interrupt()`。
- **长循环任务**：配合 `volatile` + `interrupt()` 双重保障。

------

## 6. 标准回答（面试可直接背）

> 在 Java 中不能直接强制停止线程，`stop()` 已废弃。常见有两种安全方式：
>
> 1. 使用 **`interrupt()`** 发出中断信号，线程通过 `isInterrupted()` 或 `InterruptedException` 来响应退出；
> 2. 使用 **`volatile` 标志位** 控制线程执行循环的结束。
>     在实际项目中，会根据任务类型选择方式，例如 IO 阻塞任务用 `interrupt()`，计算密集型任务用 `volatile` 标志。
