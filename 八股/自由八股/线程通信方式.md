## 1. 术语解释

**线程间通信**：多个线程之间 **传递数据/协调执行** 的方式。
 Java 提供了多种机制，可以分为：

- **共享内存通信**（最常见）：依赖 `synchronized`、`volatile`、`Lock`、内存模型保证可见性。
- **消息传递通信**：通过队列、管道、工具类来传递数据。

------

## 2. 原理剖析

Java 的线程通信主要依赖 **JMM (Java Memory Model)**：

- **可见性**：一个线程修改的变量，对另一个线程及时可见。
- **有序性**：通过 `happens-before` 保证操作顺序。
- **原子性**：关键操作不可分割。

所以通信方式本质上是围绕 **如何保证数据同步与可见性**。

## 3. 使用方式

### ✅ 常见通信方式

#### (1) `wait() / notify()`（依赖 synchronized）

```java
class Demo {
    private final Object lock = new Object();
    private boolean ready = false;

    public void producer() throws InterruptedException {
        synchronized (lock) {
            ready = true;
            lock.notify(); // 唤醒消费者
        }
    }

    public void consumer() throws InterruptedException {
        synchronized (lock) {
            while (!ready) {
                lock.wait(); // 等待生产者通知
            }
            System.out.println("消费数据");
        }
    }
}
```

- 核心：必须在 **同步块** 内使用，否则抛异常。

------

#### (2) `join()`

让一个线程等待另一个线程结束：

```java
Thread t = new Thread(() -> System.out.println("子线程"));
t.start();
t.join(); // 主线程等待 t 执行完
System.out.println("主线程继续");
```

------

#### (3) `volatile` 关键字

保证 **变量的可见性**，常用于**状态标志**：

```java
volatile boolean running = true;

public void run() {
    while (running) { /* ... */ }
}
```

------

#### (4) 管道流（`PipedInputStream / PipedOutputStream`）

适用于 **字节流通信**：

```java
PipedOutputStream out = new PipedOutputStream();
PipedInputStream in = new PipedInputStream(out);

new Thread(() -> {
    try { out.write("hello".getBytes()); } catch (Exception e) {}
}).start();

new Thread(() -> {
    try { System.out.println(new String(in.readAllBytes())); } catch (Exception e) {}
}).start();
```

------

#### (5) 并发工具类

- `BlockingQueue`：典型生产者-消费者模型
- `CountDownLatch`：一个线程等待多个线程完成
- `CyclicBarrier`：多个线程相互等待，到达屏障点再一起执行
- `Semaphore`：限制同时访问资源的线程数
- `Exchanger`：两个线程交换数据

```java
BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(10);

// 生产者
new Thread(() -> {
    try { queue.put(1); } catch (InterruptedException e) {}
}).start();

// 消费者
new Thread(() -> {
    try { System.out.println(queue.take()); } catch (InterruptedException e) {}
}).start();
```

## 4. 对比分析

| 通信方式         | 特点                                | 适用场景            |
| ---------------- | ----------------------------------- | ------------------- |
| `wait/notify`    | 经典方式，依赖 synchronized，易出错 | 简单的生产者-消费者 |
| `join`           | 等待线程结束                        | 主线程等待子线程    |
| `volatile`       | 保证可见性，轻量级                  | 状态标志位          |
| 管道流           | 数据直接传递                        | 流式通信            |
| `BlockingQueue`  | 高级封装，线程安全                  | 生产者-消费者       |
| `CountDownLatch` | 一等多                              | 主线程等待子任务    |
| `CyclicBarrier`  | 多等多                              | 多线程分阶段任务    |
| `Semaphore`      | 限流控制                            | 控制并发数          |
| `Exchanger`      | 两线程交换                          | 双方协作            |

------

## 5. 项目实践

- **Web 后端任务调度**：使用 `BlockingQueue` 做任务队列。
- **并发测试工具**：`CountDownLatch` 启动多个线程同时执行。
- **批量计算**：`CyclicBarrier` 分任务 → 汇总结果。
- **接口限流**：`Semaphore` 控制最大并发请求数。

------

## 6. 标准回答（面试背诵版）

> Java 中线程间通信方式主要分为两类：
>
> - **共享内存通信**：通过 `synchronized/wait/notify`、`volatile`、`join` 等方式实现状态同步和通知；
> - **消息传递通信**：通过 `BlockingQueue`、`管道流`、并发工具类（`CountDownLatch`、`CyclicBarrier`、`Semaphore`、`Exchanger`）等实现数据交换和协作。
>    在实际开发中，更推荐使用 JUC 提供的并发工具类，因为它们线程安全、可读性高，避免了 `wait/notify` 容易出错的问题。
