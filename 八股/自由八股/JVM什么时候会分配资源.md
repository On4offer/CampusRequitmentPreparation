**Answer:**

这里的“JVM 什么时候分配资源”要分开理解，因为 JVM 管理的资源主要有 **内存** 和 **线程/CPU 时间片** 两大类。

------

### 1. **内存资源的分配**

JVM 在程序运行时会根据对象、方法调用等情况分配内存，常见的有几种情况：

1. **类加载时**
   - 加载 `.class` 文件时，JVM 会在 **方法区 (元空间/Metaspace)** 分配存放类元信息的内存。
2. **对象创建时**
   - 调用 `new` 创建对象时，JVM 会在 **堆 (Heap)** 中分配对象实例内存。
   - 同时分配对象的默认字段初始值。
3. **方法调用时**
   - 每次方法被调用时，JVM 会在 **栈帧 (Stack Frame)** 中分配局部变量表、操作数栈等资源。
   - 方法返回时，栈帧出栈，对应的内存也就释放了。
4. **运行时需要时**
   - 比如字符串拼接，JVM 可能会临时分配缓冲区。
   - JIT 编译器可能会为热点代码申请额外空间做优化。

------

### 2. **线程与 CPU 资源的分配**

- **线程创建时**：
   调用 `new Thread().start()`，JVM 会向操作系统申请一个线程（栈空间 + 程序计数器等），并注册到调度器里。
- **CPU 时间片**：
   JVM 并不会自己决定哪个线程先跑，而是交给 **操作系统调度器**。
  - 操作系统基于时间片轮转、优先级等策略，把 CPU 资源分配给某个线程。
  - JVM 只是把线程交给 OS，等操作系统来分配执行权。

------

### 3. **垃圾回收时的资源回收**

当堆内存不足，或者触发 GC 条件时，JVM 会启动垃圾回收线程：

- 找出不可达对象，释放其占用的堆内存。
- 这样“资源”就被重新收回，等待下一次分配。

------

### 4. 小结

JVM 的资源分配时机：

1. **加载类时** → 分配方法区元信息
2. **new 对象时** → 分配堆内存
3. **方法调用时** → 分配栈帧
4. **启动线程时** → 向 OS 申请线程栈
5. **运行过程中需要时** → 临时分配内存、缓冲区
6. **垃圾回收** → 回收不可达对象内存

------

💡一句话总结：

> **JVM 在程序执行的各个阶段“按需”分配资源**：创建对象时分配堆内存，调用方法时分配栈帧，启动线程时向操作系统申请线程，执行时依赖操作系统调度 CPU。
