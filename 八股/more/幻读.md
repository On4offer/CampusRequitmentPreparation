在数据库事务隔离中，**幻读（Phantom Read）** 是一种特殊的数据一致性问题，常作为高级面试题出现，尤其在 MySQL/InnoDB、事务设计、并发控制中非常关键。

------

## 一、什么是幻读？

**幻读**（Phantom Read）指的是：

> **同一个事务中，两次执行相同条件的查询，第二次却出现了“原本不存在的新行”**，就好像出现了“幻影”一样。

------

### 举例说明：

假设你执行如下 SQL 语句查询符合条件的用户：

```sql
SELECT * FROM users WHERE age > 20;
```

你看到当前有 5 行结果，然后你要对这些结果进行更新：

```sql
UPDATE users SET status = 'verified' WHERE age > 20;
```

但在你两次操作之间，**另一个事务插入了一行 age = 25 的新用户**，此时你会发现你原来认为的“符合条件的用户”数量变了。这就是**幻读**现象。

------

## 二、幻读与其他读现象对比

| 类型       | 描述                                                         |
| ---------- | ------------------------------------------------------------ |
| 脏读       | 读到**未提交事务**修改的数据                                 |
| 不可重复读 | 同一个查询在同一事务中读到**不同数据**（数据被其他事务修改） |
| 幻读       | 同一个查询在同一事务中读到**不同的记录数量**（被插入或删除） |

------

## 三、InnoDB 中如何防止幻读？

- 默认隔离级别：`REPEATABLE READ`（可重复读）
- 在 InnoDB 中，**MVCC + 间隙锁（Next-Key Lock）** 联合用来防止幻读。

### 间隙锁是什么？

- 锁住一个范围，不仅仅是已有记录，还包括“可能插入的空隙”。
- 这样就能防止其他事务在范围内插入新行，从而防止幻读。

------

## 四、如何测试幻读？

使用如下语句组合可模拟幻读：

```sql
-- 事务A
START TRANSACTION;
SELECT * FROM orders WHERE amount > 100;

-- 事务B（此时插入新数据）
INSERT INTO orders (id, amount) VALUES (999, 200);
COMMIT;

-- 事务A再次查询
SELECT * FROM orders WHERE amount > 100;
```

在 READ COMMITTED 下，会看到一条“新行”，产生幻读；而在 REPEATABLE READ 并加锁的情况下，InnoDB 会阻止插入，避免幻读。

------

## 五、面试总结一句话：

> 幻读是“数据数量”层面的变化，而不是内容的变化。InnoDB 使用 Next-Key Lock 机制避免幻读，特别是在 REPEATABLE READ + 加锁读时。
