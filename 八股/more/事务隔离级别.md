在数据库中，**事务隔离级别（Transaction Isolation Level）** 定义了**一个事务在多大程度上能看到其他事务所做的修改**，它是实现并发控制、数据一致性的关键机制。

------

### 一、四种标准隔离级别（由 SQL 标准定义）

| 隔离级别             | 脏读 | 不可重复读 | 幻读                   | 并发性       | 默认值（MySQL） |
| -------------------- | ---- | ---------- | ---------------------- | ------------ | --------------- |
| **READ UNCOMMITTED** | 可能 | 可能       | 可能                   | 高           | 否              |
| **READ COMMITTED**   | 否   | 可能       | 可能                   | 中           | 否              |
| **REPEATABLE READ**  | 否   | 否         | 可能（InnoDB中防止了） | 中等         | **是**          |
| **SERIALIZABLE**     | 否   | 否         | 否                     | 低（性能差） | 否              |

------

### 二、隔离级别详解

#### 1. **READ UNCOMMITTED（读未提交）**

- 事务可以读取其他事务**尚未提交**的数据。
- 存在 **脏读** 问题。
- **不常用**，极限性能优先但风险大。

#### 2. **READ COMMITTED（读已提交）**

- 只能读取其他事务**已提交**的数据。
- 避免脏读，但可能出现 **不可重复读**（两次查询结果不同）。

#### 3. **REPEATABLE READ（可重复读）**

- 同一个事务内多次读取同一数据，结果保持一致。
- 防止脏读和不可重复读，但可能出现 **幻读**。
- **InnoDB 通过 MVCC + 间隙锁防止幻读**。
- 是 **MySQL InnoDB 的默认隔离级别**。

#### 4. **SERIALIZABLE（可串行化）**

- 最严格：所有事务串行执行，相当于加表锁。
- 完全避免脏读、不可重复读、幻读。
- **性能最差、并发最低**，极少使用。

------

### 三、三种并发异常

| 异常类型       | 描述                                          | 避免的最低隔离级别                        |
| -------------- | --------------------------------------------- | ----------------------------------------- |
| **脏读**       | 读取了另一个事务未提交的数据                  | READ COMMITTED                            |
| **不可重复读** | 同一事务中两次读到不同值                      | REPEATABLE READ                           |
| **幻读**       | 同一事务两次读取，记录数量不一致（新增/删除） | SERIALIZABLE 或 InnoDB 的 REPEATABLE READ |

------

### 四、MySQL 设置与查看隔离级别

```sql
-- 查看当前隔离级别
SELECT @@tx_isolation;

-- 设置当前会话隔离级别
SET SESSION TRANSACTION ISOLATION LEVEL REPEATABLE READ;

-- 设置全局隔离级别
SET GLOBAL TRANSACTION ISOLATION LEVEL READ COMMITTED;
```

------

### 五、性能与隔离的权衡

| 隔离级别         | 一致性保障强度 | 性能/并发性 |
| ---------------- | -------------- | ----------- |
| SERIALIZABLE     | 最强           | 最差        |
| REPEATABLE READ  | 强             | 中等        |
| READ COMMITTED   | 中             | 好          |
| READ UNCOMMITTED | 最弱           | 最佳        |

------

### 六、小结一句话：

> **事务隔离级别决定了事务之间数据可见的程度，越高的隔离级别一致性越强，但性能越低；MySQL 默认使用 REPEATABLE READ，是性能与一致性的平衡点。**
