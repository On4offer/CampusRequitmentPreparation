## ✅ 一、什么是三次握手（TCP 三次握手）

三次握手是指：**建立 TCP 连接** 时，客户端与服务器之间相互通信的三个步骤，确保双方**都准备好收发数据**，从而建立可靠连接。

------

### 📦 三次握手过程

1️⃣ **客户端发送 SYN 包（同步请求）**

- 表示：客户端希望建立连接；
- 包含：SYN 标志位、初始序列号 `seq = x`。

2️⃣ **服务器返回 SYN + ACK 包**

- 表示：服务器接受请求，准备好连接；
- 包含：
  - `SYN=1`：回复同步请求；
  - `ACK=1`：确认客户端的 `SYN`；
  - `seq = y`，`ack = x + 1`。

3️⃣ **客户端发送 ACK 包**

- 表示：客户端确认收到服务器的响应；
- 包含：`ACK=1`，`ack = y + 1`。

> 💡 此时连接建立完成，双方可以开始传输数据。

------

### 🧠 三次握手的目的

- 确保客户端和服务端**都能正常发送与接收**；
- 防止**历史连接请求（旧 SYN 包）被误当作新连接**；
- 建立**可靠的全双工通道**。

------

## ❓ 面试高频问：**为什么不是两次？**

因为：

- 第一次握手只表示客户端能发；
- 第二次握手只表示服务端能收；
- **必须第三次握手确认服务端的 ACK 收到了，才能保证双方通信正常。**

------

## ✅ 二、什么是四次挥手（TCP 四次挥手）

四次挥手是指：**断开 TCP 连接** 时，客户端与服务器通过四个阶段优雅关闭连接，确保数据不丢失。

------

### 📦 四次挥手过程

1️⃣ **客户端发送 FIN 包**

- 表示：客户端不再发送数据了，但还能接收；
- 进入 **FIN_WAIT_1** 状态。

2️⃣ **服务器回复 ACK 包**

- 表示：收到客户端关闭请求；
- 客户端进入 **FIN_WAIT_2**；
- 服务器仍可能继续发送剩余数据。

3️⃣ **服务器发送 FIN 包**

- 表示：服务器数据发完，准备断开连接；
- 进入 **LAST_ACK** 状态。

4️⃣ **客户端发送 ACK 包**

- 确认服务器关闭请求，进入 **TIME_WAIT** 状态；
- 2MSL 后彻底关闭连接。

------

### 🧠 四次挥手的目的

- 实现**半关闭机制**：允许一方先关闭，另一方再关闭；
- 确保双方都收到了关闭请求；
- 防止数据丢失或连接残留。

------

## ✅ 三次握手 vs 四次挥手 总结表

| 特性             | 三次握手（连接）    | 四次挥手（断开）            |
| ---------------- | ------------------- | --------------------------- |
| 作用             | 建立连接            | 优雅断开连接                |
| 包含过程         | SYN → SYN+ACK → ACK | FIN → ACK → FIN → ACK       |
| 状态复杂度       | 较简单              | 更复杂，有 TIME_WAIT 等状态 |
| 是否可靠传输前提 | 是                  | 是                          |

------

## 💬 面试标准答题模板（简洁版）

> 三次握手用于建立 TCP 连接，由客户端发送 SYN，服务器回复 SYN+ACK，客户端再发送 ACK，确保双方能正常通信。而四次挥手用于断开连接，由客户端发送 FIN 表示结束发送，服务器确认后自己也发送 FIN，客户端确认后才完全断开。这样设计是为了保证数据完整和通信双方都能优雅地关闭连接。

------

## ✅ 一句话总结

> 三次握手保障“可靠连接建立”，四次挥手确保“数据安全关闭”。
