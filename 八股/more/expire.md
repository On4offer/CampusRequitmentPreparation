好的，我们来介绍 Redis 中的 **`EXPIRE` 命令**，它在缓存过期控制、分布式锁自动释放等场景中非常重要。

------

## 一、什么是 `EXPIRE`？

**`EXPIRE key seconds`** 是 Redis 提供的一个命令，用于**为指定的 key 设置过期时间（单位：秒）**。
 一旦过期，key 会被 Redis 自动删除。

------

## 二、基本语法

```bash
EXPIRE mykey 60
```

- 含义：为 `mykey` 设置 60 秒的生存时间。
- 执行成功会返回 `1`，如果 `mykey` 不存在，返回 `0`。

------

## 三、常见用法

### 1. 设置过期时间（秒）

```bash
EXPIRE session:user123 1800  # 30分钟后自动失效
```

### 2. 设置绝对过期时间（Unix时间戳）

```bash
EXPIREAT mykey 1715539999
```

### 3. 查看剩余时间

```bash
TTL mykey   # 查看 key 剩余秒数（返回 -1 表示永不过期）
```

------

## 四、配合使用场景

### 1. **缓存自动失效**

在设置缓存值之后设置过期时间，避免数据一直占用内存：

```bash
SET product:1001 '{"name":"apple"}'
EXPIRE product:1001 300
```

或直接一条命令完成（推荐）：

```bash
SET product:1001 '{"name":"apple"}' EX 300
```

### 2. **分布式锁**

为避免死锁，必须为锁设置过期时间：

```bash
SET lock:order123 uuid-value NX EX 10
```

- `NX`：只在 key 不存在时设置
- `EX 10`：锁最多存在 10 秒，防止程序异常导致锁永远不释放

------

## 五、注意事项

| 问题/行为           | 说明                                 |
| ------------------- | ------------------------------------ |
| 重复执行 `EXPIRE`   | 会覆盖原有过期时间                   |
| `EXPIRE` 后更新 key | 默认不影响过期时间（除非你重写 key） |
| 删除 key            | 会连同过期时间一起删除               |
| key 过期不马上删除  | Redis 采用惰性删除 + 定期删除策略    |

------
