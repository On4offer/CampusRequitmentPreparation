------

## 一、什么是依赖注入？

**依赖注入（DI）** 是一种设计模式，它的核心思想是：

> **由外部容器（如 Spring）负责将对象需要的“依赖”自动注入进来，而不是由对象自己去创建依赖。**

它是实现 **控制反转（IOC）** 的一种具体方式。

------

## 二、通俗理解

想象一个类中依赖另一个类：

传统写法：

```java
UserService userService = new UserService();
```

依赖注入写法（由框架注入）：

```java
@Autowired
private UserService userService;
```

**区别**：

- 原来你自己控制依赖（手动 new）；
- 现在由框架（容器）控制注入（自动提供）；
- 你只“声明需要”，容器“负责提供”。

------

## 三、依赖注入的三种方式（Spring 支持）

| 注入方式    | 示例说明                                 |
| ----------- | ---------------------------------------- |
| 构造器注入  | 依赖通过构造方法传入（推荐）             |
| Setter 注入 | 通过 setter 方法注入依赖（适合可选依赖） |
| 字段注入    | 通过反射设置字段值，写法最简但封装性差   |

------

## 四、Spring 中的依赖注入示例

```java
@Component
public class OrderService {

    private final UserService userService;

    @Autowired  // 构造器注入
    public OrderService(UserService userService) {
        this.userService = userService;
    }
}
```

- `@Autowired` 表示：请 Spring 容器为我注入一个 `UserService` 类型的 Bean；
- Spring 会自动从上下文中查找匹配的对象并注入。

------

## 五、依赖注入的优点

| 优点           | 说明                                   |
| -------------- | -------------------------------------- |
| **解耦合**     | 组件之间不直接创建彼此，依赖更松散     |
| **更易测试**   | 可使用 mock 对象注入依赖测试           |
| **增强灵活性** | 便于扩展、替换依赖                     |
| **易维护**     | 项目结构清晰，依赖关系统一交由容器管理 |

------

## 六、一句话总结：

> **依赖注入是一种由容器管理对象依赖关系的模式，它是实现 IOC 的核心机制，使得代码更加解耦、灵活、易于测试。**
