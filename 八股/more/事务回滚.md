## ✅ 什么是事务回滚？

> **事务回滚**指的是在一个事务执行过程中，如果发生错误或手动触发回滚命令，数据库会**撤销当前事务中所做的所有操作**，恢复到事务开始前的状态。

它是 **ACID 中的一致性 + 原子性保障机制**的体现。

------

## 💡 举个简单例子：

```sql
START TRANSACTION;
UPDATE account SET money = money - 100 WHERE id = 1;
UPDATE account SET money = money + 100 WHERE id = 2;
ROLLBACK;
```

执行 `ROLLBACK` 后，这两个更新操作都被撤销了，数据就像没改过一样。

------

## 🔁 事务回滚的两种方式

### 1. **自动回滚（出错时）**

如发生异常、数据库断开、死锁等：

```java
@Transactional
public void transfer() {
    accountDao.debit(1, 100);
    int x = 1 / 0; // 抛出异常
    accountDao.credit(2, 100);
}
```

由于抛出运行时异常（`ArithmeticException`），事务会自动回滚。

------

### 2. **手动回滚**

你可以主动声明回滚，例如：

```java
TransactionAspectSupport.currentTransactionStatus().setRollbackOnly();
```

或通过 `PlatformTransactionManager` 编程式控制。

------

## ✅ Spring 中事务回滚的默认规则

| 异常类型           | 是否触发回滚 |
| ------------------ | ------------ |
| `RuntimeException` | ✅ 会         |
| `Error`            | ✅ 会         |
| `CheckedException` | ❌ 不会       |

### ✅ 想让 checked 异常也回滚？

你可以手动指定：

```java
@Transactional(rollbackFor = Exception.class)
```

------

## ⚠️ 常见事务回滚**不生效的原因**

| 问题场景                    | 原因说明                                |
| --------------------------- | --------------------------------------- |
| 异常被 `try-catch` 捕获     | 没有抛出到外层，Spring AOP 无法感知异常 |
| 调用本类方法（非代理）      | 代理机制无效，事务注解不生效            |
| 是 checked 异常             | Spring 默认只回滚 RuntimeException      |
| `@Transactional` 加在接口上 | Spring AOP 默认对实现类增强，接口无效   |
| 异步方法或新线程中抛异常    | 已脱离事务上下文                        |

------

## 🔧 示例代码：自定义异常也回滚

```java
@Transactional(rollbackFor = CustomCheckedException.class)
public void doBusiness() throws CustomCheckedException {
    saveData();
    throw new CustomCheckedException("业务异常");
}
```

## 💬 面试标准答题模板

> 事务回滚是数据库保证原子性的重要机制，在操作失败或异常发生时，通过回滚撤销之前的修改操作，保证数据一致性。在 Spring 中，事务默认会在运行时异常或 Error 出现时回滚，如果是受检异常则需要手动指定 `rollbackFor` 属性。同时需要注意 AOP 的代理限制、异常传播、内部方法调用等问题，否则事务可能不会生效。

------

## ✅ 一句话总结

> 事务回滚 = **撤销未提交的所有操作**，是数据库 ACID 中“一致性和原子性”的守护神。
