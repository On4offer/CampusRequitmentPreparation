## ✅ 二、什么是字段注入？

**字段注入（Field Injection）** 是 Spring 中将依赖对象直接注入到类的字段（属性）上的一种方式。

> Spring 容器在实例化 Bean 时，通过反射将依赖对象注入到被 `@Autowired` 标注的字段中。

------

## ✅ 三、字段注入的使用方式

最常见的使用方式是在成员变量上加 `@Autowired` 注解：

```java
@Service
public class OrderService {

    @Autowired
    private OrderRepository orderRepository;

    public void createOrder() {
        orderRepository.save(...);
    }
}
```

也可结合 `@Qualifier` 指定具体 Bean：

```java
@Autowired
@Qualifier("vipOrderRepository")
private OrderRepository orderRepository;
```

------

## ✅ 四、字段注入的原理简述

字段注入本质是由 Spring 的 **BeanPostProcessor** 在对象初始化后，通过 **Java 反射** 修改私有字段值，实现依赖注入。

流程如下：

```text
1. Spring 创建 Bean 实例（无参构造）
2. 扫描字段上是否有 @Autowired
3. 反射方式注入依赖对象
4. 完成属性设置后执行初始化方法
```

------

## ✅ 五、字段注入的优缺点

### ✅ 优点：

| 优点            | 说明                        |
| --------------- | --------------------------- |
| 写法简洁        | 无需构造器或 Setter，代码少 |
| 常用于简单 Bean | 快速原型开发、高效注入      |

### ❌ 缺点：

| 缺点                    | 说明                                        |
| ----------------------- | ------------------------------------------- |
| **不利于测试**          | 字段是 `private`，难以在单元测试中注入 Mock |
| **不利于维护**          | 隐式依赖，类的依赖关系不明确                |
| **循环依赖更容易出错**  | 尤其在多线程 Bean 中难以调试                |
| **无法声明 final 字段** | 因为反射注入不能改 final 字段               |

------

## ✅ 六、字段注入 vs 构造器注入 vs Setter 注入

| 方式        | 是否推荐 | 可测试性 | 是否明确依赖     | 支持 final 字段 |
| ----------- | -------- | -------- | ---------------- | --------------- |
| 字段注入    | ❌ 不推荐 | 差       | 差（依赖不透明） | 否              |
| 构造器注入  | ✅ 推荐   | 好       | 好（依赖显式）   | ✅               |
| Setter 注入 | ⭕ 可选   | 一般     | 一般             | 否              |

> 🟡 官方推荐 **构造器注入（Constructor Injection）**，特别是在使用 Lombok 的 `@RequiredArgsConstructor` 时最方便。

------

## ✅ 七、实战案例：字段注入带来的测试问题

```java
@Autowired
private UserRepository userRepository;
```

在单元测试中难以注入 mock：

```java
// 不可访问 private 字段
UserService userService = new UserService();
// 不能直接 set UserRepository
```

解决方案：改为构造器注入或使用反射工具手动注入。

## ✅ 十、总结建议

- 字段注入虽简单但隐藏依赖，**不推荐在生产代码中广泛使用**
- **构造器注入是最佳实践**，特别适合配合 Lombok 使用
- 字段注入反射方式注入私有字段，容易导致测试困难
- 项目中应**统一注入方式**，保持可维护性与可测试性
