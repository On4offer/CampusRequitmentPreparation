好的！我们来系统性地介绍一下 **Spring 中 Bean 的实例化**——也就是容器如何创建你定义的对象。

------

## 一、什么是 Bean 的实例化？

**Bean 的实例化**，就是 Spring 容器在启动或运行时，**根据 Bean 定义信息（如类名、作用域、构造方法等），在内存中创建出对象（实例）** 的过程。

简单说：你写了一个类，Spring 把它“变成”一个对象放进容器，这个过程就叫“实例化”。

------

## 二、Spring 实例化 Bean 的三种方式

| 实例化方式            | 示例方式                             | 特点与说明               |
| --------------------- | ------------------------------------ | ------------------------ |
| 1. 构造方法实例化     | 默认无参构造器，或指定构造方法       | 最常用，支持自动注入     |
| 2. 静态工厂方法实例化 | 使用某个类的静态方法返回对象         | 类似工具类的静态创建方法 |
| 3. 实例工厂方法实例化 | 使用某个工厂对象的方法返回 Bean 实例 | 更灵活但配置更复杂       |

------

### 1. **构造方法实例化（最常见）**

#### 默认无参构造方法：

```java
@Component
public class UserService {
    public UserService() {
        System.out.println("UserService 实例化了！");
    }
}
```

Spring 会通过 **反射调用构造方法** 创建这个类的实例。

#### 带参数构造方法（构造器注入）：

```java
@Component
public class OrderService {
    private final PaymentService paymentService;

    @Autowired
    public OrderService(PaymentService paymentService) {
        this.paymentService = paymentService;
    }
}
```

------

### 2. **静态工厂方法实例化**

适用于你有一个工具类提供静态创建方法：

```java
public class UserFactory {
    public static User createUser() {
        return new User("Tom");
    }
}
<bean id="user" class="com.example.UserFactory" factory-method="createUser"/>
```

Spring 会调用 `UserFactory.createUser()` 方法来创建 Bean。

------

### 3. **实例工厂方法实例化**

先定义一个工厂类，再通过这个工厂的实例方法返回 Bean：

```java
public class CarFactory {
    public Car createCar() {
        return new Car("BMW");
    }
}
<bean id="carFactory" class="com.example.CarFactory"/>
<bean id="car" factory-bean="carFactory" factory-method="createCar"/>
```

先创建 `carFactory` Bean，再调用其 `createCar()` 方法生成 `car` Bean。

------

## 三、Bean 实例化的完整流程（核心步骤）

以构造方式为例，Spring 实例化过程如下：

1. **解析配置**：Spring 解析 `@Component`、`@Bean`、XML 等 Bean 定义；
2. **构造实例**：调用构造方法或工厂方法，创建 Bean 实例；
3. **依赖注入**：根据 `@Autowired`、构造参数等注入所需依赖；
4. **回调方法调用**：
   - `@PostConstruct` 方法
   - `InitializingBean.afterPropertiesSet()` 方法
5. **放入容器中**：放入一级缓存，供后续使用；
6. **如果配置了AOP**，还会包装成代理对象返回。

------

## 四、注意点

- 对于 `@Component`、`@Service`、`@Controller`，Spring 默认使用构造器实例化；
- 如果一个类没有无参构造器，也没被自动注入依赖，会抛出 `NoSuchMethodException`；
- 原型（prototype）Bean 每次 `getBean()` 都会触发一次实例化。

------

## 五、常见错误

| 错误或异常                         | 原因说明                            |
| ---------------------------------- | ----------------------------------- |
| `NoSuchMethodException`            | 没有默认构造方法，注入失败          |
| `BeanCurrentlyInCreationException` | 构造器注入 + 循环依赖               |
| `UnsatisfiedDependencyException`   | 缺少必须注入的 Bean，导致实例化失败 |
