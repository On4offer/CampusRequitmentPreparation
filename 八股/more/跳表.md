## ✅ 什么是跳表（Skip List）？

> 跳表是一种 **基于链表、通过增加多级索引加速查询** 的**有序数据结构**，支持快速插入、删除、查找，时间复杂度接近于平衡树，但实现更简单。

通俗来说：

> 跳表 = “链表 + 多层电梯”
>  底层是有序链表，每层往上是“跳跃索引”，可以跳过大量节点，加速查找。

------

## 📚 结构图示（文字版）

假设你有一个普通升序链表：

```
1 → 3 → 5 → 7 → 9 → 11 → 13
```

跳表会建立多层索引，例如：

```
Level 3:       1 -------------> 9
Level 2:       1 -----> 5 -----> 9
Level 1:       1 -> 3 -> 5 -> 7 -> 9 -> 11 -> 13
```

这样查找时可以“先跳大步，再精确查找”，大大提升效率。

------

## ⚙️ 跳表支持的操作及复杂度

| 操作       | 时间复杂度 |
| ---------- | ---------- |
| 查找       | O(log n)   |
| 插入       | O(log n)   |
| 删除       | O(log n)   |
| 空间复杂度 | O(n)       |

跳表通过**随机算法**决定每个节点“出现在多少层”，避免退化为链表。

------

## 🧠 跳表的核心特性

- **有序结构**：内部是升序的链表
- **多级索引**：每层链表是下一层的“稀疏版”
- **查找快速**：从上层向下查，逐层精细定位
- **实现简单**：相比红黑树或 AVL 树更易实现和维护

------

## 🛠 跳表 vs 平衡树（对比）

| 对比点          | 跳表（Skip List）        | 红黑树 / AVL 树      |
| --------------- | ------------------------ | -------------------- |
| 数据结构        | 多层链表 + 随机索引      | 树结构，需旋转平衡   |
| 查找效率        | O(log n)                 | O(log n)             |
| 插入/删除复杂度 | O(log n)，操作简单       | O(log n)，需维护平衡 |
| 实现难度        | 简单                     | 相对复杂             |
| 并发友好性      | 更适合并发优化           | 更难并发分离         |
| Redis 使用      | ✅（用于 ZSet、有序字典） | ❌                    |

## 🚀 跳表在 Redis 中的应用

在 Redis 中，跳表用于实现 **Sorted Set（有序集合）** 的底层数据结构：

```text
ZADD score member
ZRANGE ... WITHSCORES
```

- Redis 使用 **双结构**（跳表 + 哈希表）：
  - 哈希表：快速定位元素；
  - 跳表：按 score 排序，支持范围查找、排名等。

跳表让 Redis 的 ZSet 操作：

- 查找快；
- 范围查询性能高；
- 排名计算也很高效。
