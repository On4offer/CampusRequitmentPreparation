在 Java 中，集合框架（Java Collection Framework）是开发中最常用也最重要的模块之一。下面是**常见集合及其特点**的总结，可用于面试复习或项目选型参考：

------

## 一、List 系列（有序，可重复）

| 类名         | 底层结构   | 是否线程安全 | 是否有序 | 是否可重复 | 特点                        |
| ------------ | ---------- | ------------ | -------- | ---------- | --------------------------- |
| `ArrayList`  | 动态数组   | 否           | 是       | 是         | 查询快，插入/删除慢（中间） |
| `LinkedList` | 双向链表   | 否           | 是       | 是         | 插入/删除快，查询慢         |
| `Vector`     | 动态数组   | 是           | 是       | 是         | 线程安全，效率低            |
| `Stack`      | Vector子类 | 是           | 是       | 是         | 后进先出（LIFO）栈结构      |

------

## 二、Set 系列（无序，不可重复）

| 类名            | 底层结构           | 是否线程安全 | 是否有序               | 是否可重复 | 特点                                |
| --------------- | ------------------ | ------------ | ---------------------- | ---------- | ----------------------------------- |
| `HashSet`       | HashMap            | 否           | 否                     | 否         | 插入快，去重依赖 hashCode 和 equals |
| `LinkedHashSet` | HashMap + 双向链表 | 否           | 是（插入顺序）         | 否         | 保持插入顺序                        |
| `TreeSet`       | 红黑树（TreeMap）  | 否           | 是（自然排序或自定义） | 否         | 可排序，插入/查找略慢               |

------

## 三、Map 系列（键值对）

| 类名                | 底层结构           | 是否线程安全 | 是否有序       | 是否允许 null               | 特点                           |
| ------------------- | ------------------ | ------------ | -------------- | --------------------------- | ------------------------------ |
| `HashMap`           | 数组 + 链表/红黑树 | 否           | 否             | 允许1个key和多个value为null | 查询快，效率高                 |
| `LinkedHashMap`     | HashMap + 双向链表 | 否           | 是（插入顺序） | 同上                        | 有序                           |
| `TreeMap`           | 红黑树             | 否           | 是（key有序）  | 不允许key为null             | 可排序                         |
| `Hashtable`         | 哈希表             | 是           | 否             | 不允许                      | 古老，效率低，不推荐           |
| `ConcurrentHashMap` | 分段锁或CAS        | 是（高并发） | 否             | 不允许 key 为 null          | 并发性能好，推荐在多线程中使用 |

------

## 四、Queue / Deque（队列 / 双端队列）

| 类名            | 类型           | 特点                                     |
| --------------- | -------------- | ---------------------------------------- |
| `PriorityQueue` | 小顶堆（默认） | 元素自动排序，适合任务调度               |
| `ArrayDeque`    | 双端队列       | 可做栈、队列，高效                       |
| `LinkedList`    | 也实现了 Queue | 可当队列、栈使用                         |
| `BlockingQueue` | 接口           | 支持线程间通信，如 `LinkedBlockingQueue` |
| `Deque`         | 接口           | 支持双端操作，如 `ArrayDeque`            |

------

## 五、线程安全集合（java.util.concurrent）

| 类名                    | 特点                                 |
| ----------------------- | ------------------------------------ |
| `CopyOnWriteArrayList`  | 写时复制，适合读多写少               |
| `ConcurrentHashMap`     | 高并发场景推荐使用                   |
| `ConcurrentLinkedQueue` | 非阻塞队列                           |
| `LinkedBlockingQueue`   | 支持阻塞等待的队列                   |
| `SynchronizedList/Map`  | Collections 工具类包装的线程安全集合 |

------

### 常见选择建议（记忆技巧）：

- 需要有序 + 可重复 → `ArrayList` / `LinkedList`
- 需要去重 → `HashSet` / `LinkedHashSet`
- 需要排序 → `TreeSet` / `TreeMap`
- 需要并发安全 → `ConcurrentHashMap` / `CopyOnWriteArrayList`
- 队列场景 → `Queue` / `BlockingQueue`
