### Spring Boot的约定优于配置（Convention over Configuration）

**“约定优于配置”（Convention over Configuration，简称CoC）**是Spring Boot的一个核心理念，它指的是通过默认的行为和配置约定，减少开发者需要做的显式配置，从而提高开发效率和可维护性。开发者只需要遵循框架的默认约定，而不需要手动进行大量的配置。这样，应用程序的开发变得更加简单和一致。

Spring Boot通过自动配置、默认约定和约定优于配置的策略，帮助开发者减少大量的配置工作，使得开发更专注于业务逻辑，而非繁琐的配置文件和类。

### 1. **Spring Boot中的约定优于配置**

Spring Boot通过以下几个方面实现了约定优于配置：

#### 1.1 自动配置（Auto Configuration）

Spring Boot的自动配置机制基于“约定优于配置”的原则，通过自动推断和配置所需的组件。例如，如果项目中包含了`spring-boot-starter-web`依赖，Spring Boot会自动配置Web相关的基础设施，如嵌入式的Tomcat、Spring MVC等。开发者无需显式配置Servlet容器、数据源等常见组件，Spring Boot会根据项目依赖和默认的规则自动配置这些组件。

#### 1.2 默认配置（Default Configuration）

Spring Boot为常见的功能（如数据源、JPA、Web服务器等）提供了合理的默认值。如果开发者没有显式地指定某些配置，Spring Boot会采用这些默认值。例如，Spring Boot默认使用内嵌的Tomcat作为Web服务器，默认端口是8080，默认的数据库连接方式是H2嵌入式数据库等。

开发者可以通过覆盖默认配置来定制化应用，但如果不需要特殊配置，默认的行为通常就足够了。

#### 1.3 项目结构约定

Spring Boot约定了一个常见的项目结构，遵循这一结构可以使开发者在创建应用时减少配置和混乱。例如：

```
src
 └── main
     ├── java
     │   └── com
     │       └── example
     │           └── demo
     │               └── DemoApplication.java
     └── resources
         ├── application.properties
         └── static
             └── (web assets)
```

这种约定使得应用结构一致，开发者可以快速理解和管理项目。

#### 1.4 默认的配置文件（application.properties 或 application.yml）

Spring Boot默认会查找`application.properties`或`application.yml`文件，作为应用的配置文件。开发者可以在此文件中自定义各种配置，但如果不做任何修改，Spring Boot会使用默认的设置。例如，默认的服务器端口为8080，数据库配置为内嵌H2数据库等。

```properties
# 默认端口配置
server.port=8080

# 默认数据库连接配置（假设没有配置其他数据库）
spring.datasource.url=jdbc:h2:mem:testdb
```

#### 1.5 内嵌Web服务器的默认选择

Spring Boot默认提供了对内嵌Web服务器的支持，最常见的是Tomcat，但你可以根据需要切换为Jetty或Undertow。只要依赖了`spring-boot-starter-web`，Spring Boot会自动配置Tomcat作为Web服务器，无需任何额外配置。

如果需要切换到Jetty或Undertow，只需排除Tomcat并引入相应的依赖：

```xml
<!-- 使用Jetty代替Tomcat -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-jetty</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-tomcat</artifactId>
    <scope>provided</scope>
</dependency>
```

#### 1.6 约定的命名约定

Spring Boot使用了“约定的命名”来进一步减少配置的复杂性。例如：

- **Controller类**：默认情况下，Spring Boot会自动扫描`@RestController`或`@Controller`注解的类来提供Web功能，开发者无需手动配置。
- **Service、Repository类**：按照Spring的约定，`@Service`、`@Repository`、`@Component`等注解的类会自动被Spring容器扫描并注入，无需额外的配置。

#### 1.7 JPA和数据库自动配置

Spring Boot使用约定的方式自动配置数据库连接和JPA。只要引入`spring-boot-starter-data-jpa`，并在`application.properties`文件中提供数据库配置，Spring Boot会自动配置数据源、EntityManagerFactory、事务管理等。

```properties
# 配置数据源和JPA
spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=root
spring.datasource.password=root
spring.jpa.hibernate.ddl-auto=update
```

如果没有提供特定配置，Spring Boot会默认配置一个H2内存数据库，并进行初始化。

#### 1.8 `@SpringBootApplication`的作用

`@SpringBootApplication`是Spring Boot的一个关键注解，它是`@Configuration`、`@EnableAutoConfiguration`和`@ComponentScan`的组合，负责自动配置Spring应用上下文和组件扫描。开发者只需要将应用的入口类标注为`@SpringBootApplication`，Spring Boot就会自动进行必要的配置和组件扫描，减少了大量的手动配置。

```java
@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
```

### 2. **Spring Boot的“约定优于配置”带来的好处**

#### 2.1 提高开发效率

由于Spring Boot遵循了“约定优于配置”原则，开发者只需要关注业务逻辑而不是繁琐的配置。这减少了开发中的重复性工作，使开发者能够更加专注于实现功能。

#### 2.2 降低配置错误

大量的自动配置和默认设置减少了因手动配置错误而导致的问题。如果配置出错，Spring Boot通常会通过日志提供详细的错误信息，帮助开发者快速定位问题。

#### 2.3 提供统一的开发体验

Spring Boot的约定和默认设置确保了开发者在不同项目中获得一致的体验。无论是项目结构、配置文件，还是组件的默认配置，都符合约定，使得跨项目开发变得更加简单。

#### 2.4 增强了应用的可维护性

统一的约定和结构让应用更加规范化，降低了代码和配置的复杂性。新的开发者能够更容易理解和维护代码，因为大家遵循相同的约定和规则。

#### 2.5 便于扩展和定制

尽管Spring Boot遵循约定优于配置的原则，但它仍然允许开发者覆盖默认配置。你可以根据需要定制应用的行为，但如果不需要特别配置，Spring Boot会提供合适的默认值。

### 3. **总结**

Spring Boot的“约定优于配置”通过自动配置、默认配置和合理的约定，使得开发变得更加高效、简洁和一致。开发者无需为常见功能编写繁琐的配置，只需要关注业务逻辑和定制化需求。虽然Spring Boot提供了很多默认设置，但它仍然允许开发者灵活地进行定制化配置，确保了高效的开发和可维护性。
