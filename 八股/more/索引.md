“索引”在数据库中是一种**提高查询效率的机制**，可以类比为书的目录，能帮助数据库快速定位到想要的数据行，而不必逐行扫描整个表。

------

### 一、索引的定义

**索引（Index）\**是数据库系统自动维护的\**特殊数据结构**，本质上是以某种排序结构（如 B+ 树、哈希表）保存列值及其所在数据行的指针，用于加速数据检索。

------

### 二、索引的作用

- **加速数据查询**（最主要目的）
- **加快表的连接操作**
- **辅助排序和分组操作**
- **用于约束，如唯一性约束（UNIQUE）**

------

### 三、常见索引类型（以 MySQL 为例）

| 类型     | 特点与用途                           |
| -------- | ------------------------------------ |
| 普通索引 | 最基本的索引，没有任何限制           |
| 唯一索引 | 保证列中的值唯一                     |
| 主键索引 | 表的主键，自带唯一性，不允许为 NULL  |
| 联合索引 | 一个索引包含多个列                   |
| 全文索引 | 用于全文搜索（如搜索文章中的关键词） |
| 空间索引 | 用于地理空间数据（如GIS）            |

------

### 四、索引的底层结构（MySQL InnoDB）

InnoDB 存储引擎中，**索引一般是基于 B+ 树实现的**：

- 叶子节点存储实际的记录数据（聚簇索引）
- 非叶子节点只存储索引键及指向子节点的指针

------

### 五、举个例子

假设有如下表：

```sql
CREATE TABLE user (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    age INT,
    INDEX (name)
);
```

这表示：

- `id` 是主键，自动创建主键索引；
- `name` 字段上单独建了一个普通索引；
- 查询时，如果语句是 `SELECT * FROM user WHERE name = 'Tom';`，就可以利用 `name` 索引快速查找，不用全表扫描。
