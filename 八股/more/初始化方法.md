Spring 中的 **Bean 初始化方法** 是指在 Bean 实例化完成、依赖注入之后，Spring 容器执行的 **“后续初始化操作”**，通常用于进行资源准备、属性校验、连接建立等任务。

------

## 一、什么是 Bean 初始化方法？

Bean 初始化方法发生在 **实例化 → 依赖注入 → 初始化 → 使用** 这个生命周期中的 **“初始化”阶段**。
 它的主要目的是：**在 Bean 完全就绪前执行一些自定义逻辑**。

------

## 二、常见的 Bean 初始化方式（Spring 支持的三种）

| 初始化方式                      | 实现方式                         | 特点与说明                    |
| ------------------------------- | -------------------------------- | ----------------------------- |
| 1. `@PostConstruct`注解         | 在方法上加 `@PostConstruct`      | 推荐方式，简洁、注解驱动      |
| 2. 实现 `InitializingBean` 接口 | 实现 `afterPropertiesSet()` 方法 | 接口式初始化，适合老项目      |
| 3. 指定 `init-method` 方法      | 在 XML 或注解中显式指定方法名    | 灵活，适合配置类或第三方 Bean |

------

### 1. `@PostConstruct` 注解（推荐）

#### 用法示例：

```java
@Component
public class MyService {

    @PostConstruct
    public void init() {
        System.out.println("MyService 初始化完成！");
    }
}
```

#### 特点：

- 简单直观
- 方法名任意，只要加了注解
- 支持与 `@Component`、`@Bean` 一起使用

------

### 2. 实现 `InitializingBean` 接口

#### 用法示例：

```java
@Component
public class MyService implements InitializingBean {

    @Override
    public void afterPropertiesSet() throws Exception {
        System.out.println("通过 InitializingBean 初始化");
    }
}
```

#### 特点：

- 结构明确，但耦合强
- 一个类只能有一个 `afterPropertiesSet()` 方法
- 在 `@PostConstruct` 之后执行

------

### 3. 自定义 `init-method`（XML 或 @Bean 配置中）

#### XML 配置：

```xml
<bean id="myBean" class="com.example.MyBean" init-method="customInit"/>
```

#### 注解配置：

```java
@Bean(initMethod = "customInit")
public MyBean myBean() {
    return new MyBean();
}
```

#### Bean 类定义：

```java
public class MyBean {
    public void customInit() {
        System.out.println("通过 init-method 初始化");
    }
}
```

#### 特点：

- 灵活配置，常用于**第三方类**无法修改时
- 方法名任意，只要在配置中声明

------

## 三、初始化方法的执行时机

```text
实例化 → 依赖注入 → 初始化方法（@PostConstruct、afterPropertiesSet、init-method） → Bean就绪
```

如果多种方式同时存在，**执行顺序如下**：

1. `@PostConstruct`
2. `InitializingBean.afterPropertiesSet()`
3. `init-method`（如果配置了）

------

## 四、初始化方法的应用场景

- 启动时加载本地缓存、配置
- 连接数据库、远程服务或第三方 API
- 检查依赖属性是否完整或合法
- 启动后台任务或定时器

------

## 五、常见错误与注意事项

| 问题                              | 说明                    |
| --------------------------------- | ----------------------- |
| 忘加 `@PostConstruct`             | 方法不会被调用          |
| 方法必须是 `void` 无参数          | 否则无法执行            |
| 原型 Bean 的初始化需要注意        | 每次 getBean() 才初始化 |
| `@PostConstruct` 不适用于静态方法 | 无法注入依赖            |

------
