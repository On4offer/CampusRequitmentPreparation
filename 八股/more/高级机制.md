Spring IOC（控制反转）容器的高级机制，指的是在基础的Bean管理（如创建、注入）之外，**Spring为了实现更强大、灵活、可扩展的依赖注入与生命周期控制**，提供的一系列**底层 SPI 接口、动态代理、AOP协同、Bean处理链等机制**。

------

## 一、Spring IOC 的高级机制总览

| 分类             | 关键机制                                                     | 说明                                                         |
| ---------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| Bean生命周期扩展 | `BeanPostProcessor`、`InstantiationAwareBeanPostProcessor`   | 拦截Bean初始化前后、实例化过程，进行增强                     |
| Bean定义阶段扩展 | `BeanFactoryPostProcessor`、`BeanDefinitionRegistryPostProcessor` | 修改Bean定义或注册新Bean                                     |
| 注解与元数据解析 | `@Autowired`、`@Value`、`@Qualifier` + `AutowiredAnnotationBeanPostProcessor` | 基于反射自动解析注入点                                       |
| 延迟初始化       | `@Lazy`、`ObjectFactory`、`ObjectProvider`                   | 支持按需加载，解决循环依赖                                   |
| 条件注入         | `@Conditional`、`Condition` 接口                             | 根据环境、配置或逻辑判断是否注册Bean                         |
| 作用域扩展       | `Scope`、`ScopeMetadataResolver`                             | 除 singleton/prototype 外支持自定义作用域（如 request、session） |
| 动态代理         | CGLIB、JDK Proxy                                             | 支持AOP、懒加载、接口增强等                                  |
| 工厂Bean机制     | `FactoryBean` 接口                                           | 将复杂对象创建逻辑封装成Bean（如MyBatis的Mapper）            |
| 依赖注入元数据   | `@Import`、`@ImportSelector`、`@Configuration`、`ImportBeanDefinitionRegistrar` | 支持模块化装配和按需Bean注册                                 |
| 事件监听机制     | `ApplicationEventPublisher`、`@EventListener`                | 支持IOC层的事件发布与监听                                    |

------

## 二、重点机制详解

### 1. `BeanPostProcessor`（Bean后置处理器）

- 拦截 **每个Bean的初始化前后过程**；
- 可用于：代理包装、注入自定义逻辑、注解处理等。

```java
public interface BeanPostProcessor {
    Object postProcessBeforeInitialization(Object bean, String beanName);
    Object postProcessAfterInitialization(Object bean, String beanName);
}
```

> 典型应用：`@Autowired` 注解就是通过 `AutowiredAnnotationBeanPostProcessor` 处理的。

------

### 2. `BeanFactoryPostProcessor` 和 `BeanDefinitionRegistryPostProcessor`

- 容器启动早期执行，作用于 **BeanDefinition 级别**；
- 可动态修改/注册新的 Bean 定义。

```java
public interface BeanFactoryPostProcessor {
    void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory);
}
```

------

### 3. `FactoryBean<T>`

- 用于封装复杂Bean的创建过程；
- 容器中通过 `getObject()` 返回实际对象。

```java
public class MyFactoryBean implements FactoryBean<MyType> {
    public MyType getObject() { return new MyType(); }
    public Class<?> getObjectType() { return MyType.class; }
}
```

> 例如：Spring-MyBatis 中 Mapper 接口的动态代理就是基于 `FactoryBean` 实现。

------

### 4. 条件注入：`@Conditional`

- 配合 `Condition` 接口，在运行时判断是否注入某个Bean。

```java
@Configuration
@Conditional(MyCondition.class)
public class MyConfig {}
```

------

### 5. 自定义作用域：`Scope`

- Spring 默认支持 `singleton`、`prototype`，但你可以扩展如 `thread`、`tenant` 等。

```java
public class ThreadLocalScope implements Scope { ... }
```

------

### 6. 延迟依赖注入工具：`ObjectProvider`

用于从单例Bean中延迟获取 prototype Bean 或可选依赖。

```java
@Autowired
private ObjectProvider<TaskHandler> taskHandlerProvider;

taskHandlerProvider.getObject().doWork();
```

------

### 7. 注解驱动注册：`@Import`、`@ImportSelector`

- `@Import`: 直接引入类；
- `ImportSelector`: 通过返回类名数组动态注册；
- `ImportBeanDefinitionRegistrar`: 手动注册BeanDefinition。

------

### 8. ApplicationContext 事件机制

- 自定义事件、监听器，解耦模块间通知；
- 比如服务启动完成、配置变更、用户操作日志通知等。

------

## 三、总结一句话

> **Spring IOC 不只是注入容器对象，它更像是一个“微型平台”**，通过钩子接口、处理链、代理机制实现 Bean 生命周期、注入逻辑和扩展能力的全流程控制。
