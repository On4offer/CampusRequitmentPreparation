**反射机制（Reflection）** 是 Java 提供的一种强大工具，它允许程序在**运行时**动态地获取类的信息（如属性、方法、构造器等），并进行调用或修改，即使这些信息在编译时并不可见。

------

### 一、反射的核心用途

1. **运行时获取类的结构信息**：类名、字段、方法、构造器、注解等
2. **动态创建对象实例**（替代 `new`）
3. **动态调用对象方法**
4. **访问/修改私有字段或方法**
5. **用于框架设计**（如 Spring、MyBatis、JUnit）

------

### 二、反射的核心类（java.lang.reflect 包）

| 类/接口       | 描述                                             |
| ------------- | ------------------------------------------------ |
| `Class<T>`    | 表示类的字节码对象，是获取反射的入口             |
| `Field`       | 表示类的成员变量                                 |
| `Method`      | 表示类的方法                                     |
| `Constructor` | 表示类的构造方法                                 |
| `Modifier`    | 提供访问修饰符相关操作（如 `private`、`public`） |

------

### 三、反射使用示例

```java
// 1. 获取Class对象
Class<?> clazz = Class.forName("com.example.User");

// 2. 创建对象
Object obj = clazz.getDeclaredConstructor().newInstance();

// 3. 获取方法并调用
Method method = clazz.getMethod("setName", String.class);
method.invoke(obj, "张三");

// 4. 获取属性并读取
Field field = clazz.getDeclaredField("name");
field.setAccessible(true); // 访问私有属性
String name = (String) field.get(obj);
System.out.println(name);
```

------

### 四、三种获取 Class 对象的方式

1. `Class<?> clazz = Class.forName("com.xxx.MyClass");`
2. `Class<?> clazz = MyClass.class;`
3. `Class<?> clazz = obj.getClass();`

------

### 五、反射的优缺点

**优点：**

- 提高程序的灵活性和扩展性
- 支持框架和通用工具开发

**缺点：**

- 性能开销较大（比直接调用慢）
- 安全性较低（可访问私有结构）
- 不易维护和调试

------

### 六、应用场景

- **Spring IOC**：反射创建和注入 Bean
- **ORM框架（MyBatis/Hibernate）**：反射映射数据库字段和对象属性
- **JUnit 单元测试**：反射调用测试方法
- **IDE 插件开发**：动态分析代码结构
