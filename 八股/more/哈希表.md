“哈希表”（Hash Table）是一种 **通过哈希函数将键（Key）映射到数组下标** 的数据结构，用于实现高效的查找、插入和删除操作，平均时间复杂度可以达到 **O(1)**。

下面是哈希表的详细介绍：

------

### 一、基本概念

- 哈希表包含一个**数组**（称为“桶”），和一个**哈希函数**。
- 每个键通过哈希函数转换为一个数组下标，将数据存储到该下标对应的位置。

```java
int index = hash(key) % array.length;
```

------

### 二、哈希冲突（Hash Collision）

由于数组长度有限，不同的键可能映射到相同的下标，称为**哈希冲突**。

#### 常见解决方法：

1. **链地址法（拉链法）**
   - 每个桶是一个链表或红黑树，冲突的元素存储在链表中。
2. **开放地址法**
   - 冲突时寻找下一个空位置（线性探测、二次探测等）

------

### 三、常见操作及复杂度

| 操作 | 平均时间复杂度 | 最坏时间复杂度 |
| ---- | -------------- | -------------- |
| 查找 | O(1)           | O(n)（全冲突） |
| 插入 | O(1)           | O(n)           |
| 删除 | O(1)           | O(n)           |

------

### 四、Java 中的哈希表实现

#### 1. `HashMap`

- 使用**链地址法**
- 支持 `null` 键和 `null` 值
- 线程不安全，适用于单线程或配合 `Collections.synchronizedMap` / `ConcurrentHashMap`

#### 2. `Hashtable`

- 使用 `synchronized` 实现线程安全
- 不允许 `null` 键或值
- 线程安全但性能差

#### 3. `ConcurrentHashMap`

- 线程安全、性能优良的哈希表
- 使用分段锁或 CAS 操作实现高并发下的数据一致性

------

### 五、扩容机制

- 当元素数量超过阈值（load factor × 容量）时，会触发 **resize()** 扩容（通常是翻倍），并重新哈希所有元素，代价较高。

------

### 六、总结

| 特点             | 描述                                    |
| ---------------- | --------------------------------------- |
| 高效             | 平均 O(1) 的时间复杂度                  |
| 哈希冲突         | 需要通过链表、红黑树或探测等方式解决    |
| 扩容开销大       | 扩容操作会带来性能抖动                  |
| 多线程场景要慎用 | 使用 `ConcurrentHashMap` 替代 `HashMap` |
