B+树是一种广泛应用于**数据库系统和文件系统**的**多路平衡查找树**，是 B-树（B-Tree）的变种，专为**磁盘存储和范围查询优化**设计。

------

## 一、B+树的基本结构

### 特点总结：

1. **每个节点可包含多个关键字**（非二叉树，是 m 叉树）
2. **所有数据都存放在叶子节点**
3. **非叶子节点只存储索引，不存储数据**
4. **叶子节点间通过指针相连，形成有序链表**，便于范围查找

------

### B+树 vs B-树（区别）

| 特性         | B-树                 | B+树                               |
| ------------ | -------------------- | ---------------------------------- |
| 数据存储位置 | 所有节点都可存数据   | **仅叶子节点存数据**               |
| 叶子节点结构 | 不一定连通           | **叶子节点有序链表结构**           |
| 查询效率     | 查询可能停在中间节点 | 查询一定走到叶子节点（路径更均衡） |
| 范围查询性能 | 不支持链式遍历       | **支持范围查找，效率更高**         |
| 存储密度     | 较低                 | 较高（非叶节点只存索引）           |

------

## 二、B+树的结构示意图（简单版）

```
        [10 | 20 | 30]
        /    |    |   \
   [1 5 9] [11 15] [21 25] [31 40]

叶子节点：所有数据项
内部节点：仅存索引值
叶子节点：有序链表 -> -> ->
```

------

## 三、B+树的核心操作

### 1. 插入操作

- 在叶子节点中插入数据，按顺序排列
- 如果节点满了，则**分裂**，中间值上移到父节点（可能递归分裂）

### 2. 删除操作

- 删除叶子节点数据
- 若元素太少，进行**合并或借值**

### 3. 查询操作

- 从根开始，通过索引定位到叶子节点
- 所有查询**最终都走到叶子节点**

### 4. 范围查询

- 由于叶子节点是链表结构，范围查询非常高效

------

## 四、B+树的参数：m 阶 B+树

- 每个非叶节点最多有 m 个孩子
- 至少有 `⌈m/2⌉` 个孩子（根节点除外）
- 叶子节点存放 `k` 个有序数据项，并与相邻叶子节点相连

------

## 五、为什么数据库使用 B+树而不是二叉树/哈希表？

| 数据结构   | 存储模型适配 | 范围查询   | 内存/磁盘优化         | 索引使用           |
| ---------- | ------------ | ---------- | --------------------- | ------------------ |
| 二叉查找树 | 差           | 支持       | 差（深度大）          | 不常用             |
| 哈希表     | 好           | **不支持** | 差（无顺序）          | 不支持范围查找     |
| **B+树**   | **很好**     | **支持**   | **顺序存储 + 层数少** | **数据库索引主力** |

B+树的**磁盘友好性 + 范围查询能力**，是数据库采用它作为**索引结构**（如 InnoDB 的聚簇索引）的关键原因。

------

## 六、应用场景

- **数据库索引**（MySQL/InnoDB）
- **文件系统目录结构**（如 NTFS、Ext4）
- **键值存储引擎**（如 LevelDB、RocksDB）
