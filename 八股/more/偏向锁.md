### **偏向锁（Biased Lock）简介**

**背景**：偏向锁（Biased Locking）是 JVM 为了提升并发性能而采用的一种优化锁机制。它的核心思想是：**如果一个线程长时间持有锁，并且没有其他线程竞争这个锁，那么这个锁就偏向当前线程，后续该线程再次获取锁时不需要再进行竞争**。偏向锁的出现是为了减少由于锁竞争所带来的性能开销，特别是在多线程中**同一线程多次获取锁**的场景下。

------

### **一、偏向锁的工作原理**

偏向锁的工作原理非常简单，基本过程如下：

1. **线程获得锁**：当某个线程第一次访问一个对象时，JVM 会给该对象分配一个偏向锁，并且**偏向该线程**。
2. **线程继续获取锁**：如果这个线程在后续的执行过程中再次访问该对象，JVM 会**快速地允许该线程获得锁**，不需要进行线程竞争。
3. **线程竞争**：如果在锁偏向过程中，有其他线程尝试获取该锁，JVM 会**撤销偏向锁**，并将其升级为轻量级锁或重量级锁，来处理多个线程的竞争。
4. **撤销偏向锁**：当另一个线程尝试获取同一对象的锁时，JVM 会撤销偏向锁，并将其转换为轻量级锁或重量级锁，进行线程同步。

------

### **二、偏向锁的优缺点**

#### **优点：**

- **无锁操作**：如果某个线程连续多次访问同一对象，它不需要加锁，避免了不必要的锁竞争，提高了性能。
- **减少锁的开销**：通过让同一线程偏向获取锁，减少了锁的获取和释放的开销，减少了内存同步的频繁操作。

#### **缺点：**

- **线程切换时会撤销**：如果一个线程执行完操作后，另一个线程需要访问该对象，偏向锁会被撤销，导致性能开销。
- **不适用于高并发环境**：如果存在频繁的锁竞争，偏向锁会被频繁撤销，导致性能下降，因此在高并发环境下使用偏向锁会带来性能损失。

------

### **三、偏向锁的实现机制**

在 JVM 中，**对象头**中会包含与锁相关的信息。偏向锁的实现是在对象头的 **Mark Word** 中记录当前持有锁的线程 ID。当一个线程第一次访问对象时，JVM 会把当前线程的 ID 写入 Mark Word 中，这样后续相同线程访问该对象时就不需要进行加锁操作。

如果在偏向锁状态下有其他线程尝试获取锁，JVM 会触发 **锁撤销**，将偏向锁升级为 **轻量级锁**（基于 CAS 操作）或者 **重量级锁**（操作系统的互斥量）。

------

### **四、偏向锁的状态转换**

| 状态         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| **无锁**     | 对象没有被任何线程持有锁。                                   |
| **偏向锁**   | 当一个线程首次获取锁后，偏向该线程，并记录线程 ID。          |
| **轻量级锁** | 如果其他线程尝试获取偏向锁，JVM 会撤销偏向锁，转为轻量级锁。 |
| **重量级锁** | 如果有多个线程竞争锁，且轻量级锁无法满足条件，JVM 会将其升级为重量级锁。 |

------

### **五、如何启用或禁用偏向锁**

偏向锁通常是 JVM 的默认行为，可以通过启动参数进行控制。

#### **启用偏向锁（默认启用）**：

```bash
-XX:+UseBiasedLocking
```

#### **禁用偏向锁**：

```bash
-XX:-UseBiasedLocking
```

通过禁用偏向锁，可以禁用 JVM 在对象上启用偏向锁的机制，所有的锁操作将直接进入轻量级锁或者重量级锁的状态。这通常用于需要在高并发环境下进行性能优化的场景，尤其是对于锁竞争较多的应用。

------

### **六、偏向锁的使用场景**

偏向锁特别适合以下场景：

1. **单线程或低并发环境**：如果大部分时间内只有一个线程访问某个资源，偏向锁可以避免不必要的加锁操作。
2. **相同线程频繁访问同一对象**：如果同一线程多次访问某个共享对象，偏向锁可以减少锁的竞争，提高性能。

------

### **七、小结一句话**

> **偏向锁通过将锁偏向于访问某个对象的单个线程，减少了加锁操作的开销，适用于线程竞争较少的场景，提高了性能；但在多线程竞争激烈时，偏向锁会撤销并升级为其他锁机制，可能带来性能损失。**
