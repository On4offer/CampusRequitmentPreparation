好的，我们来系统讲解一下 Java 中的**`synchronized` 关键字**。

------

## 一、什么是 `synchronized`？

`**synchronized**` 是 Java 提供的一种 **内置锁（Monitor Lock）机制**，用于在**多线程并发环境下实现线程间的同步**，保证**同一时间只有一个线程可以访问临界区代码（互斥访问）**，防止线程安全问题。

------

## 二、`synchronized` 的三种用法

| 用法形式                   | 锁对象            | 应用场景                 |
| -------------------------- | ----------------- | ------------------------ |
| `synchronized` 实例方法    | 当前对象（this）  | 多线程访问同一个对象实例 |
| `synchronized` 静态方法    | 当前类的Class对象 | 类级别共享资源           |
| `synchronized(obj)` 代码块 | 指定锁对象        | 精细控制代码范围与锁粒度 |

------

### 1. **同步实例方法**

```java
public synchronized void doSomething() {
    // 此方法同一时刻只能被一个线程访问
}
```

- 锁的是当前对象（`this`）
- 适用于每个对象都有自己的数据副本

------

### 2. **同步静态方法**

```java
public static synchronized void doStatic() {
    // 锁的是类对象(Class)
}
```

- 锁的是类级别（`Class<?> clazz`）
- 适用于所有实例共享同一资源的情况

------

### 3. **同步代码块**

```java
public void doBlock() {
    synchronized (lockObject) {
        // 只锁定这一部分代码
    }
}
```

- 更灵活：只对关键代码加锁，减少锁粒度，提高并发效率

------

## 三、`synchronized` 的实现原理（JVM 层）

` synchronized` 是基于 JVM 中的 **[对象头](对象头)（Object Header）+ [Monitor](Monitor)（监视器锁）** 实现的。

在 HotSpot 虚拟机中，Java 对象头中包含一个 **Mark Word**，它用于存储锁的信息状态。

### 锁的四种状态（从轻到重）：

| 锁类型   | 特点                   | 是否阻塞 | 说明                           |
| -------- | ---------------------- | -------- | ------------------------------ |
| 无锁     | 没有竞争               | 否       | 最快                           |
| 偏向锁   | 偏向第一个获得锁的线程 | 否       | 无需CAS，极快                  |
| 轻量级锁 | 有多个线程但无冲突     | 否       | 使用CAS，自旋避免阻塞          |
| 重量级锁 | 多线程竞争激烈         | 是       | 线程阻塞、上下文切换，性能最低 |

JVM 会根据**锁的使用情况自动在这些状态间升级或降级**。

------

## 四、`synchronized` 的优点和缺点

### 优点：

- 使用简单、语法清晰
- 是 Java 原生支持，JVM 自动优化
- 能有效防止线程并发冲突，保证数据一致性

### 缺点：

- 如果锁的范围太大或锁竞争激烈，会导致**阻塞和性能下降**
- 在高并发场景下效率不如 CAS + Lock 等机制

------

## 五、常见应用场景

- 多线程访问共享变量时
- 控制单例模式（懒汉式）
- 同步缓存读写、数据库更新等操作

------

## 六、和其他同步方式的对比

| 特性         | `synchronized`         | `ReentrantLock`   |
| ------------ | ---------------------- | ----------------- |
| 实现层级     | JVM 内建               | Java API 层       |
| 可中断锁     | 否                     | 是                |
| 可尝试加锁   | 否                     | 是（`tryLock()`） |
| 公平锁支持   | 否                     | 是                |
| 条件变量支持 | 否（用 `wait/notify`） | 是（`Condition`） |

------

### 一句话总结：

> `synchronized` 是 Java 提供的关键字级别的同步机制，通过监视器锁保证线程安全，在开发中使用广泛，适合低~中等并发场景。
