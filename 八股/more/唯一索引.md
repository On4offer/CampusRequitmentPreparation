**唯一索引（UNIQUE INDEX）** 是一种数据库约束，用于确保某一列（或列组合）中的值在整张表中是**唯一的**，即**不能重复**。它是保证数据完整性的重要手段之一。

------

### 一、唯一索引的作用

- **约束数据**：防止插入重复值。
- **加速查询**：作为索引存在，提高搜索效率。
- **支持多列组合**：可以对多个字段创建联合唯一索引。

------

### 二、与主键的区别

| 比较项               | 主键（PRIMARY KEY） | 唯一索引（UNIQUE）      |
| -------------------- | ------------------- | ----------------------- |
| 是否唯一             | 是                  | 是                      |
| 是否允许 NULL        | 否                  | **允许一个或多个 NULL** |
| 一个表是否可有多个   | 只能一个            | **可以多个**            |
| 是否自动创建聚簇索引 | InnoDB 默认是       | 否，非聚簇              |

------

### 三、创建唯一索引的方法

#### 1. 创建表时定义唯一约束：

```sql
CREATE TABLE users (
    id INT PRIMARY KEY,
    email VARCHAR(100) UNIQUE
);
```

#### 2. 表创建后添加唯一索引：

```sql
ALTER TABLE users ADD UNIQUE (email);
```

#### 3. 创建复合唯一索引（联合唯一）：

```sql
CREATE UNIQUE INDEX uniq_name_phone ON users (name, phone);
```

------

### 四、唯一索引中的 NULL 行为

- **允许多个 NULL 值**（因为在 SQL 标准中 NULL != NULL）。
- 示例：

```sql
INSERT INTO users (email) VALUES (NULL);  -- OK
INSERT INTO users (email) VALUES (NULL);  -- OK
```

------

### 五、查询优化作用

- 唯一索引同样是一种索引结构（B+树），可用于加速：
  - 精确查找：`WHERE email = 'abc@example.com'`
  - 覆盖索引优化（当查询字段被索引覆盖时）

------

### 六、设计建议

- 对于自然唯一的字段，如身份证号、邮箱、用户名，应使用唯一索引。
- 避免将可能变化的字段（如昵称）设为唯一。
- 如果业务上要求值不可重复，但不作为主键，就使用唯一索引。
