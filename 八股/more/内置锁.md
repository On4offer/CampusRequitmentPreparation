### **内置锁（Synchronized）简介**

**背景**：在 Java 中，**内置锁（又叫“同步锁”或“Monitor 锁”）** 是通过 `synchronized` 关键字实现的一种最基本的线程同步机制。它是 Java 提供的 **内建并发控制机制**，不依赖额外类库，使用简单，安全可靠。

------

### **一、什么是内置锁？**

内置锁是由 JVM 自动提供的锁机制，**每个 Java 对象都天然拥有一个内置锁（Monitor）**。当你使用 `synchronized` 关键字修饰方法或代码块时，实际上就是在使用这个内置锁。

------

### **二、synchronized 的使用方式**

#### 1. **同步实例方法**

锁的是当前对象（`this`）：

```java
public synchronized void method() {
    // 线程安全代码
}
```

#### 2. **同步静态方法**

锁的是当前类的 `Class` 对象：

```java
public static synchronized void staticMethod() {
    // 同一类的所有实例共享这个锁
}
```

#### 3. **同步代码块**

可以指定锁对象（灵活性最高）：

```java
public void method() {
    synchronized (this) {
        // 线程安全代码
    }
    
    // 或者使用其他对象作为锁
    synchronized (lockObject) {
        // 更细粒度的控制
    }
}
```

------

### **三、内置锁的工作机制**

- 当一个线程进入 `synchronized` 代码块或方法时，它必须**先获取对象的监视器锁（Monitor）**。
- 如果锁已被其他线程持有，当前线程会**阻塞等待**。
- 一旦线程获取到锁，就可以安全地访问同步资源。
- 执行完毕后会**自动释放锁**（即使抛出异常也会释放）。

------

### **四、内置锁的特性**

| 特性         | 描述                                       |
| ------------ | ------------------------------------------ |
| **互斥性**   | 同一时刻只有一个线程可以获得锁。           |
| **可重入性** | 同一个线程可以多次获取同一把锁，不会死锁。 |
| **自动释放** | 代码执行完毕后自动释放，不需要手动操作。   |
| **非公平锁** | 默认是非公平的，不能保证线程获取锁的顺序。 |

------

### **五、内置锁的底层实现（JVM 级别）**

JVM 对 synchronized 的锁实现做了优化，锁有多个状态，**支持锁升级**：

1. **无锁**：对象没有竞争时；
2. **偏向锁**：偏向某线程，性能高；
3. **轻量级锁**：多个线程竞争但无阻塞；
4. **重量级锁**：多线程竞争严重，操作系统互斥锁（Monitor）实现。

> JVM 会根据竞争情况在这些锁之间**自动升级或退化**，提高性能。

------

### **六、适用场景**

- 多线程访问共享变量时（如计数器、缓存、集合等）；
- 实现线程安全的方法或类；
- 对象级同步、类级同步等。

------

### **七、内置锁与显示锁（ReentrantLock）的对比**

| 对比项             | `synchronized`（内置锁） | `ReentrantLock`（显式锁）     |
| ------------------ | ------------------------ | ----------------------------- |
| **是否内置**       | 是（语言关键字）         | 否（需手动使用）              |
| **是否可中断**     | 否                       | 是                            |
| **是否可定时获取** | 否                       | 是（`tryLock(timeout)`）      |
| **是否可公平锁**   | 否                       | 是（构造函数中可设置）        |
| **自动释放**       | 是                       | 否（必须手动调用 `unlock()`） |

------

### **八、小结一句话**

> **内置锁是 Java 提供的最简单可靠的线程同步机制，通过 `synchronized` 关键字自动管理加锁与释放，适用于大多数并发控制场景。**
