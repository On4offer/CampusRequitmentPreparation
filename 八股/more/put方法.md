### 🔹1. **计算键的 hash 值**

调用 `hash(key)` 方法，对 key 的 `hashCode()` 进行高位扰动（hash 扰动函数）：

```java
hash = (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16)
```

目的是让高位信息参与到低位，提高 hash 值的随机性，减少碰撞。

### 🔹2. **定位数组索引（桶位置）**

通过：

```java
index = (n - 1) & hash
```

根据数组长度 `n` 和 hash 值确定该 key 应放入的桶位置（数组下标）。

### 🔹3. **判断该位置是否为空**

- **如果为空**：直接创建新节点（`Node<K,V>`）放入该位置。
- **如果非空**（发生碰撞）：
  - 遍历该位置上的链表或红黑树：
    - 如果找到 key 相等的节点（`equals` 为 true）：更新其 value。
    - 如果未找到：将新节点插入链表尾部（或树上）。

### 🔹4. **判断是否需要树化**

链表长度超过阈值（默认 8），并且数组长度 ≥ 64 时，将链表转为 **红黑树**，提高查询效率。

### 🔹5. **判断是否需要扩容**

插入后判断 size 是否超过 `threshold = capacity * loadFactor`：

- 如果超出：进行扩容（resize），将旧数据重新计算 hash 分布到新数组中。

### 🔹6. **结束 put 操作**

完成插入，返回旧值（若为更新操作）或 `null`（若为新插入）。

### ✍️ 关键细节与面试关注点

| 点位             | 说明                                                    |
| ---------------- | ------------------------------------------------------- |
| `hash()` 方法    | 高位扰动提高分布性，面试官常考其意义和实现              |
| 扩容机制         | 默认负载因子为 0.75，扩容是原数组大小的 2 倍            |
| 红黑树           | JDK 1.8 开始引入，链表长度超过 8 且数组长度 ≥ 64 才触发 |
| 冲突处理         | 使用链表或红黑树（尾插法，防止死循环）                  |
| `put` 时间复杂度 | 平均为 O(1)，最差为 O(log n)（树）或 O(n)（链表）       |
