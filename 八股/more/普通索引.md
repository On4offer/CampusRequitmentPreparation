**普通索引（Normal Index）** 是数据库中最常用、最基础的一种索引类型。它主要用于**加速数据查询**，但**不限制列中的值是否重复**。

------

### 一、普通索引的特点

- **加速查询**：通过创建索引结构（如 B+ 树）提升查询效率。
- **不限制唯一性**：允许列中存在重复值或 NULL。
- **可以单列或多列**：支持联合索引。

------

### 二、普通索引的作用

- 提高 `SELECT` 查询的速度，特别是 `WHERE`、`ORDER BY`、`GROUP BY`、`JOIN` 中使用到的列。
- 适用于高频读取、低频更新的字段。

------

### 三、创建普通索引的方法

#### 1. 创建表时指定：

```sql
CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    INDEX (name)  -- 普通索引
);
```

#### 2. 表已存在时添加：

```sql
ALTER TABLE users ADD INDEX idx_name (name);
```

#### 3. 使用 `CREATE INDEX` 创建：

```sql
CREATE INDEX idx_email ON users(email);
```

------

### 四、普通索引 vs 唯一索引 vs 主键

| 特性          | 普通索引（INDEX） | 唯一索引（UNIQUE） | 主键（PRIMARY KEY） |
| ------------- | ----------------- | ------------------ | ------------------- |
| 值是否唯一    | 否                | 是                 | 是                  |
| 是否允许 NULL | 允许多个          | 允许多个           | 不允许              |
| 可否有多个    | 可以多个          | 可以多个           | **只能有一个**      |

------

### 五、使用示例

```sql
-- 创建普通索引
CREATE INDEX idx_age ON users(age);

-- 查询使用索引
SELECT * FROM users WHERE age = 30;
```

使用普通索引后，MySQL 会在查询优化阶段判断是否走索引。

------

### 六、注意事项

- 索引能加速查询，但也会略微降低插入、更新、删除的性能（因为要维护索引结构）。
- 不要对频繁变化但不参与查询的字段创建索引。
- 合理选择**覆盖索引**和**联合索引**可进一步优化性能。
