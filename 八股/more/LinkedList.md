### **LinkedList 介绍**

**`LinkedList`** 是 Java 集合框架中的一种 **List** 实现，它基于 **双向链表** 数据结构来存储元素。与 `ArrayList` 使用数组存储数据不同，`LinkedList` 通过一系列的 **节点（Node）** 连接起来，每个节点包含数据和指向前后节点的引用。

### **LinkedList 的基本特点**

1. **基于双向链表**：`LinkedList` 使用双向链表来存储元素。每个节点包含两个部分：
   - **数据部分**：存储元素的值。
   - **指针部分**：分别指向下一个节点和上一个节点。
2. **支持快速插入和删除**：由于链表是通过指针连接的，插入和删除元素的操作只需要修改相邻节点的指针，不需要移动大量元素，因此 **插入和删除** 操作非常高效，尤其是在列表的两端。
3. **不支持快速随机访问**：与 `ArrayList` 不同，`LinkedList` 不是基于数组的，所以它不支持常数时间的 **随机访问**。为了访问某个元素，需要从头节点或尾节点开始，按顺序遍历链表，时间复杂度是 **O(n)**。
4. **允许 `null` 元素**：`LinkedList` 允许存储 `null` 元素，和其他 `List` 实现一样，它可以存储重复的元素。
5. **适用于频繁插入和删除的场景**：`LinkedList` 特别适合于需要频繁进行插入和删除操作的场景，尤其是在列表的两端进行插入或删除时，性能尤为出色。

### **LinkedList 的常用方法**

`LinkedList` 提供了与 `List` 接口类似的常用方法，同时还额外提供了一些 **双向链表特有的操作**：

1. **添加元素**：
   - `add(E e)`：将指定元素添加到列表的末尾。
   - `addFirst(E e)`：将指定元素插入到链表的开头。
   - `addLast(E e)`：将指定元素插入到链表的末尾。
2. **删除元素**：
   - `remove()`：移除链表的第一个元素。
   - `removeFirst()`：移除链表的第一个元素。
   - `removeLast()`：移除链表的最后一个元素。
3. **获取元素**：
   - `get(int index)`：获取指定位置的元素。
   - `getFirst()`：获取链表的第一个元素。
   - `getLast()`：获取链表的最后一个元素。
4. **修改元素**：
   - `set(int index, E element)`：替换指定位置的元素。
5. **检查元素**：
   - `contains(Object o)`：检查是否包含指定元素。
   - `indexOf(Object o)`：返回指定元素第一次出现的位置，如果没有返回 -1。
6. **其他操作**：
   - `size()`：返回链表中元素的个数。
   - `isEmpty()`：检查链表是否为空。
   - `clear()`：清空所有元素。

### **LinkedList 与 ArrayList 的对比**

| 特性                        | **LinkedList**                       | **ArrayList**                                  |
| --------------------------- | ------------------------------------ | ---------------------------------------------- |
| **底层数据结构**            | 双向链表                             | 动态数组                                       |
| **随机访问时间复杂度**      | O(n)                                 | O(1)                                           |
| **插入/删除操作时间复杂度** | O(1)（在两端插入删除）               | O(n)（尤其是插入/删除位置靠近中间时）          |
| **内存使用**                | 较大（每个节点存储数据和前后指针）   | 较小（只存储元素，不存储额外的指针）           |
| **适用场景**                | 适合频繁插入和删除操作，特别是在两端 | 适合频繁的随机访问操作，较少进行插入和删除操作 |
| **线程安全**                | 非线程安全                           | 非线程安全                                     |

### **LinkedList 的底层实现**

`LinkedList` 是通过 **双向链表** 来实现的。每个节点（Node）包含：

- **数据部分**：存储元素的值。
- **前指针**：指向前一个节点。
- **后指针**：指向下一个节点。

### **双向链表的结构**：

- **头节点（Head）**：指向链表的第一个节点。
- **尾节点（Tail）**：指向链表的最后一个节点。
- **节点结构**：
  - 每个节点都有两个指针：指向前一个节点（`previous`）和指向下一个节点（`next`），这使得链表在任意一端都可以进行快速的插入和删除。

### **插入与删除操作**

由于 `LinkedList` 基于链表实现，每次插入和删除元素时，都仅需更新相邻节点的指针，效率较高，尤其是在链表两端（头尾插入或删除）进行操作时。

- **在链表的头部插入**：
  - 修改头节点的 `next` 指针，使其指向新节点，同时新节点的 `previous` 指针指向原头节点。
- **在链表的尾部插入**：
  - 修改尾节点的 `previous` 指针，使其指向新节点，同时新节点的 `next` 指针为 `null`。

### **LinkedList 的优缺点**

#### **优点**：

1. **插入和删除效率高**：`LinkedList` 适合频繁进行插入和删除操作，特别是当操作发生在链表的两端时，时间复杂度是 **O(1)**。
2. **动态大小**：不像 `ArrayList` 使用固定大小的数组，`LinkedList` 使用链表的节点，因此可以动态增加或减少元素数量，不需要进行扩容。
3. **无需移动元素**：插入和删除元素时，不需要像 `ArrayList` 一样移动其他元素，减少了不必要的操作。

#### **缺点**：

1. **随机访问性能差**：`LinkedList` 不支持 **O(1)** 的快速随机访问，因为必须从头节点或尾节点开始遍历查找指定元素，时间复杂度是 **O(n)**。
2. **内存开销大**：每个节点除了存储数据，还需要存储前后节点的引用，因此需要更多的内存空间。
3. **不能使用索引直接访问**：`LinkedList` 不支持像 `ArrayList` 那样通过索引直接访问元素，必须进行遍历。

### **使用 LinkedList 的场景**

- **频繁的插入和删除操作**：`LinkedList` 在这类场景中表现更好，尤其是插入和删除发生在列表的两端时。
- **内存不是问题时**：`LinkedList` 比 `ArrayList` 更占用内存，因为每个节点除了存储元素数据，还需要存储指向前后节点的指针。如果内存资源充足，`LinkedList` 是一个很好的选择。
- **顺序访问优先**：如果查询时不频繁使用随机访问，而是以顺序访问为主，`LinkedList` 会非常合适。

### **总结**

- **`LinkedList`** 是基于 **双向链表** 实现的 `List`，适用于需要频繁插入和删除元素的场景，尤其是在列表的两端。
- 与 **`ArrayList`** 相比，`LinkedList` 在 **随机访问** 时的性能较差，但在 **插入和删除操作** 时性能非常优越。
- **内存开销** 较大，因为每个节点都需要存储前后指针。
