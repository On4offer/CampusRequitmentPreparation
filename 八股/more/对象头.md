很好，这是一个理解 Java 对象底层机制的关键点。我们来详细介绍一下 **Java 中的对象头（Object Header）**。

------

## 一、什么是对象头？

在 HotSpot 虚拟机（JVM）中，**Java 中的每个对象在内存中都有一个结构：对象头 + 实例数据 + 对齐填充**。

### 对象头（Object Header）是每个对象的基础组成部分之一，**主要用于 JVM 内部管理对象**，包括：

| 组成部分               | 作用说明                          |
| ---------------------- | --------------------------------- |
| **Mark Word**          | 存储哈希码、GC信息、锁标志等      |
| **Class Pointer**      | 指向对象的类型元数据（class信息） |
| **数组长度（仅数组）** | 数组对象还包含数组长度字段        |

------

## 二、Mark Word 的详细结构（重点）

`Mark Word` 是对象头中最重要的一部分，它在不同状态下会存储不同信息。

| 状态     | 内容示意                       | 说明                       |
| -------- | ------------------------------ | -------------------------- |
| 普通无锁 | 哈希码、GC分代年龄             | 默认状态                   |
| 偏向锁   | 线程ID、时间戳、偏向标志       | 偏向某一线程的优化锁状态   |
| 轻量级锁 | 指向锁记录的指针（栈帧）       | 自旋优化，无阻塞           |
| 重量级锁 | 指向 Monitor 的指针            | 多线程竞争时膨胀为重量级锁 |
| GC标记   | 对象是否可达、是否已标记等信息 | GC 时用于标记清除判断      |

------

### Mark Word 在无锁状态下结构（64位JVM）示意：

| 占位字段   | 位数 | 含义            |
| ---------- | ---- | --------------- |
| 对象哈希码 | 25位 | `hashCode()` 值 |
| 分代年龄   | 4位  | GC 用           |
| 锁状态标志 | 2位  | 标识是否加锁    |
| 保留位     | 1位  | JVM 保留        |

------

## 三、Class Pointer（类型指针）

- 指向对象的 Class 元数据（即该对象是哪一个类的实例）
- JVM 可以通过它找到方法表、字段表等信息
- 这是对象“知道自己是谁”的关键机制

------

## 四、数组对象的特殊字段（数组长度）

数组对象的对象头中还会额外包含一个**数组长度字段**（非数组对象没有）。

------

## 五、对象头的作用总结

| 功能         | 说明                                       |
| ------------ | ------------------------------------------ |
| 锁状态管理   | 控制偏向锁、轻量级锁、重量级锁等互斥行为   |
| GC 元信息    | GC 时标记是否可达、回收年龄等              |
| 类型识别     | 用于查找类定义、方法、字段等元信息         |
| 快速访问优化 | 提供 `hashCode()`、`getClass()` 等加速路径 |

------

## 六、对象头的使用场景

- `synchronized` 使用对象头的 Mark Word 来标识锁状态
- `System.identityHashCode()` 使用对象头中的哈希码
- GC 使用对象头中记录的标志位做对象可达性判断
- 通过 Class Pointer 快速访问类元信息（继承结构、方法表等）

------

### 一句话总结：

> **对象头是 Java 对象的“身份证+控制面板”，承担着锁状态、类信息、GC管理等 JVM 底层核心职责。**
