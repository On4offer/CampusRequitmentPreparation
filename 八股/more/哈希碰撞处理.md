哈希碰撞（Hash Collision）是指：**不同的键（Key）经过哈希函数计算后得到相同的哈希值（或数组下标）**，这时就会发生“碰撞”，必须采取策略来存储这些冲突的数据。

------

### 一、为什么会发生哈希碰撞？

因为哈希函数的输出空间是有限的（例如一个数组长度是16），而输入空间是无限的（任意字符串、对象等），**鸽巢原理**告诉我们，必然会有不同的键映射到同一个桶中。

------

### 二、常见的哈希碰撞处理方法

#### 1. **链地址法（拉链法）——Java HashMap的做法**

- 同一个哈希值的元素被组织成一个链表（或红黑树），放在数组对应的桶里。
- 当发生碰撞时，新元素加入到链表尾部（或树中）。

```text
[数组索引 3] -> key1 -> key2 -> key3
```

- JDK8之后，当链表长度 > 8 且数组长度 ≥ 64 时，会**转为红黑树**，优化查询性能（由O(n)变为O(log n)）。

#### 2. **开放地址法**

- 若当前位置已被占用，就寻找下一个空位置，直到找到为止。

常用策略：

- **线性探测（Linear Probing）**：i, i+1, i+2…
- **二次探测（Quadratic Probing）**：i, i+1², i+2²…
- **双重哈希（Double Hashing）**：使用第二个哈希函数决定步长

这种方法适用于哈希表是一个**连续数组**，不能有链表结构，比如 C 中的实现。

#### 3. **再哈希（Rehashing）**

- 当碰撞严重或负载因子过高时，**重新扩容哈希表并重新计算哈希值**，降低冲突概率。
- 比如 `HashMap` 在元素数超过 `load factor * capacity` 时就会触发 `resize()`。

#### 4. **哈希函数设计优化**

- 减少碰撞的根本办法是**设计更好的哈希函数**，例如减少相似字符串的哈希重合。

------

### 三、Java 中的哈希碰撞处理流程（HashMap）

```java
int hash = hash(key);  // 计算哈希值
int index = hash % table.length;  // 定位桶
Node<K,V> node = table[index];  // 获取桶内容

if (node == null)
    table[index] = new Node<>(...);  // 无碰撞，直接插入
else
    // 遍历链表或红黑树，比较 key 是否相等，否则插入尾部
```

------

### 四、链地址法 vs 开放地址法对比

| 特性       | 链地址法             | 开放地址法                   |
| ---------- | -------------------- | ---------------------------- |
| 结构       | 每个桶存一个链表或树 | 所有数据存入一个数组         |
| 空间利用率 | 需要额外链表结构     | 数组空间利用高               |
| 删除操作   | 简单                 | 复杂（需探测后维护探测路径） |
| 并发友好性 | 高（每个桶独立）     | 差（同一个数组）             |

------

### 五、总结

- **哈希碰撞不可避免，关键是处理策略**
- Java 中普遍使用 **链地址法 + 红黑树优化**
- 高频使用场景要注意选择好的哈希函数和容器实现
