在操作系统中，**父子进程**是指通过创建子进程来实现进程管理的概念。父进程是启动子进程的进程，子进程是由父进程创建的进程。父子进程的关系常见于多任务处理系统，尤其在 Unix 和类 Unix 系统中，父进程通过系统调用来创建子进程，子进程通常会执行与父进程相关的任务或独立的任务。

### 1. **父子进程的基本概念**

- **父进程**：在操作系统中，父进程是通过某种方式（如调用 `fork()` 系统调用）创建一个或多个子进程的进程。父进程可以控制和管理子进程的执行。
- **子进程**：子进程是由父进程创建的进程。子进程通常从父进程复制部分资源和执行环境（如内存映像），并可以继承父进程的部分或全部属性。

### 2. **父子进程的关系**

父子进程在操作系统中通过以下方式进行联系和管理：

- **父进程与子进程的创建**：父进程使用如 `fork()`（在 Linux 和 Unix 系统中）等系统调用创建子进程。此时，子进程会复制父进程的地址空间、打开的文件等资源，但拥有一个不同的进程ID（PID）。
- **进程ID与父进程ID**：每个进程都有一个唯一的进程ID（PID）。父进程的PID在子进程中会被记录为**父进程ID（PPID）**。父进程通过 PPID 来识别子进程。
- **父进程与子进程的生命周期**：通常，子进程会在父进程的指导下执行任务，并在完成任务后退出。父进程可以通过 `wait()` 等系统调用来等待子进程结束，并获取子进程的退出状态。

### 3. **常见的系统调用**

- **`fork()`**：在 Unix 和 Linux 系统中，`fork()` 系统调用用于创建一个子进程。调用 `fork()` 后，父进程返回子进程的进程ID，而子进程返回 0。
- **`exec()`**：子进程可以使用 `exec()` 系列系统调用来加载新的程序，这会替换子进程的内存映像。
- **`wait()`**：父进程可以使用 `wait()` 系统调用来等待子进程的终止，并获取子进程的退出状态。
- **`exit()`**：子进程可以使用 `exit()` 来退出自己，通知操作系统资源释放。

### 4. **父子进程的特点**

- **资源继承**：子进程继承父进程的大部分资源（如文件描述符、内存映像等），但它有自己的进程ID。
- **并行执行**：父子进程可以并行执行，父进程可以继续执行自己的任务，子进程可以独立处理自己的任务。
- **孤儿进程**：如果父进程在子进程终止之前终止，子进程将变成孤儿进程，通常会被系统的`init`进程收养。
- **僵尸进程**：当子进程结束，但父进程尚未通过 `wait()` 等系统调用回收其退出状态时，子进程会成为僵尸进程，占用少量系统资源，直到父进程清理它。

### 5. **父子进程的示例**

在 Linux/Unix 系统中，父子进程的创建通常通过 `fork()` 实现。例如，父进程通过 `fork()` 创建子进程，子进程可以执行不同的任务。

```c
#include <stdio.h>
#include <sys/types.h>
#include <unistd.h>

int main() {
    pid_t pid = fork(); // 创建子进程

    if (pid < 0) {
        // 错误处理
        perror("fork failed");
        return 1;
    }

    if (pid == 0) {
        // 子进程执行
        printf("This is the child process, PID: %d\n", getpid());
    } else {
        // 父进程执行
        printf("This is the parent process, PID: %d, Child PID: %d\n", getpid(), pid);
    }

    return 0;
}
```

在这个例子中，父进程调用 `fork()` 创建一个子进程。子进程会输出其PID，父进程会输出自己的PID和子进程的PID。

### 总结

父子进程的机制使得操作系统能够有效地管理多个并行执行的任务。通过父子进程的关系，操作系统能够协调任务的执行和资源管理，进而实现多任务和多进程处理。

### 🔹 什么是父子进程？

- 在操作系统中，一个进程通过**系统调用 fork() 或类似机制**创建另一个进程，这个新进程称为**子进程**，原始进程称为**父进程**。
- 父子进程拥有**相同的代码段、打开文件描述符、环境变量等资源的拷贝**。
- 每个进程都有一个唯一的进程号（PID），并记录其父进程号（PPID）。

------

### 🔹 父子进程的创建方式（以 Linux 为例）

#### 1. C 语言示例：

```c
pid_t pid = fork();

if (pid == 0) {
    // 子进程
    printf("This is the child process. PID: %d\n", getpid());
} else {
    // 父进程
    printf("This is the parent process. PID: %d, Child PID: %d\n", getpid(), pid);
}
```

#### 2. Python 示例（使用 `os.fork()`）：

```python
import os

pid = os.fork()

if pid == 0:
    print(f"Child Process: PID={os.getpid()}, Parent PID={os.getppid()}")
else:
    print(f"Parent Process: PID={os.getpid()}, Child PID={pid}")
```

> 后端服务 fork 子进程处理耗时任务（如图像处理、数据转换），避免阻塞主逻辑。

### 🔹 进程管理注意点

#### 僵尸进程（Zombie Process）：

- 子进程退出后，其资源需由父进程调用 `wait()`/`waitpid()` 回收。
- 如果父进程没调用，子进程状态仍保留，占用 PID —— 这就是僵尸进程。

#### 孤儿进程（Orphan Process）：

- 如果父进程先退出，子进程变为孤儿，由 **init（PID=1）** 接管。

| ------- | -------------------------- | ------------ |
| C       | `fork()`, `wait()`         | POSIX 调用   |
| Python  | `os.fork()` / `subprocess` | 高级封装管理 |
| Go      | `exec.Cmd`                 | 自动回收     |
| Node.js | `child_process.fork()`     | 事件监听     |
