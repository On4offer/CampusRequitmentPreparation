------

## 一、什么是“引用”？

在 Java 中，**引用（Reference）就是指向对象内存地址的变量**。它并不是对象本身，而是对象在堆内存中的“门牌号”。

### 举例说明：

```java
Person p1 = new Person("Alice");
Person p2 = p1;
```

- `p1` 是 `Person` 对象的引用，指向堆内的一块内存。
- `p2 = p1` 表示 p2 和 p1 **指向的是同一个对象**。
- 所以修改 p2 的内容，p1 也会看到变化。

------

## 二、引用与值的区别

| 概念     | 描述                           | 示例                   |
| -------- | ------------------------------ | ---------------------- |
| 值类型   | 变量直接存储的是值本身         | `int x = 10;`          |
| 引用类型 | 变量存储的是对象的地址（引用） | `User u = new User();` |

------

## 三、引用的核心特点

1. **多个引用可以指向同一个对象**

```java
User a = new User();
User b = a;
```

此时 `a == b` 是 `true`，因为它们指向同一个对象。

1. **修改引用指向的对象，所有指向该对象的引用都可见变动**

```java
a.setName("Tom");
System.out.println(b.getName()); // 也会输出 "Tom"
```

1. **引用本身可以为空（null）**

```java
User u = null;
```

表示此引用目前没有指向任何对象。

------

## 四、图示（内存模型）简化

```java
User a = new User();   // 假设地址是0x1234
User b = a;

堆内存:
0x1234 -> User{name="John"}

栈内存:
a --> 0x1234
b --> 0x1234
```

------

## 五、应用场景

- **对象参数传递**：方法参数传引用可以修改原对象内容。
- **设计模式中的共享、缓存**：多个模块共享一个对象的引用。
- **Spring中的依赖注入**：通过引用将对象注入到另一个对象中。

------

## 六、与“指针”的区别（对比C/C++）

| 概念             | 指针（C/C++）              | 引用（Java）                 |
| ---------------- | -------------------------- | ---------------------------- |
| 能否直接操作地址 | 可以（如 `*ptr`, `&x`）    | 不可以                       |
| 是否可以为空     | 可以（null）               | 可以（null）                 |
| 是否可变指向     | 是（指针可以指向多个对象） | 是（引用变量可以被重新赋值） |

------
