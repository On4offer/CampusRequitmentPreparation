## ✅ 二、什么是 RabbitMQ？

**RabbitMQ** 是一个基于 **AMQP（Advanced Message Queuing Protocol）** 协议实现的 **开源消息中间件**，使用 **Erlang** 语言开发，特点是：

> **高可靠性、易于使用、支持多种消息模型，适合企业级消息解耦与异步通信。**

------

## ✅ 三、RabbitMQ 的核心组件与架构

| 组件                   | 说明                                      |
| ---------------------- | ----------------------------------------- |
| **Producer**           | 消息生产者，发送消息                      |
| **Consumer**           | 消费者，处理消息                          |
| **Broker**             | RabbitMQ 服务端，负责接收、缓存、投递消息 |
| **Exchange（交换机）** | 接收来自生产者的消息，并路由到队列        |
| **Queue（队列）**      | 缓存消息，等待消费者消费                  |
| **Binding**            | 定义队列与交换机之间的路由规则            |
| **Routing Key**        | 决定消息如何从 Exchange 分发到 Queue      |

------

## ✅ 四、RabbitMQ 的 4 种核心消息模型

| 模型        | 描述                               | 使用场景       |
| ----------- | ---------------------------------- | -------------- |
| **Direct**  | 精确匹配 routing key               | 日志分类投递   |
| **Fanout**  | 广播（不看 routing key）           | 消息通知、群发 |
| **Topic**   | 模糊匹配 routing key（支持通配符） | 多层级订阅     |
| **Headers** | 根据消息头属性路由                 | 灵活但较少用   |

------

## ✅ 五、典型使用场景案例

### 🧩 1. 系统解耦 & 异步处理（最常见）

> 订单服务 → 消息队列 → 库存服务、邮件通知服务

优点：降低服务耦合、异步处理不阻塞主流程

------

### 🧩 2. 削峰限流（秒杀/抢购）

> 大量请求先写入队列，再由后台消费者限速处理，保护数据库

------

### 🧩 3. 日志收集 & 消息广播

> 日志统一收集服务，使用 Fanout 模式进行广播式推送

------

### 🧩 4. 延迟消息、定时任务

> 通过 TTL + 死信队列 实现延迟消费（如 15 分钟未支付取消订单）

------

## ✅ 六、RabbitMQ 的可靠性机制

| 机制                                  | 说明                                             |
| ------------------------------------- | ------------------------------------------------ |
| **消息确认（ack）**                   | 消费者收到消息后主动确认                         |
| **持久化（durable）**                 | 队列和消息都可设置持久化（防止 Broker 重启丢失） |
| **死信队列（DLX）**                   | 消费失败、过期、被拒绝的消息进入死信队列         |
| **发布确认机制（publisher confirm）** | Producer 可收到消息是否入队成功的反馈            |

------

## ✅ 七、RabbitMQ 的缺点或注意点

- 依赖 Erlang 和 BEAM 虚拟机，部署稍复杂
- 性能相比 Kafka 等吞吐型 MQ 略低
- 消息顺序性不如 Kafka 保证
- 如果消息堆积严重，内存可能飙升

------

## ✅ 八、使用代码示例（Java + SpringBoot）

```java
// 生产者发送消息
rabbitTemplate.convertAndSend("order.exchange", "order.created", orderId);

// 消费者监听消息
@RabbitListener(queues = "order.queue")
public void handle(String orderId) {
    System.out.println("处理订单：" + orderId);
}
```
