“**哈希值**”是指通过**哈希函数（Hash Function）\**将任意长度的输入（如字符串、对象等）映射成一个固定长度的数值，这个数值就是\**哈希值（Hash Code）**，在哈希表中用来决定数据的存储位置。

------

### 一、哈希值的基本定义

- 哈希值是一个**整数**，通常用于：
  - 快速定位数据（如 HashMap 的索引）
  - 唯一标识对象（如重写 `hashCode()` 方法）
  - 校验完整性（如文件的 MD5、SHA1 值）

------

### 二、哈希函数的特点

一个好的哈希函数需要具备以下特性：

| 特性       | 描述                             |
| ---------- | -------------------------------- |
| **确定性** | 相同输入必须返回相同的哈希值     |
| **均匀性** | 哈希值应尽可能均匀分布，避免冲突 |
| **高效性** | 计算速度快，适用于高频调用       |
| **抗冲突** | 不同输入尽可能产生不同哈希值     |

------

### 三、Java中的哈希值

在 Java 中，每个对象都有一个默认的哈希值，由 `Object.hashCode()` 提供：

```java
String str = "hello";
int hash = str.hashCode();  // 生成哈希值
```

你也可以重写 `hashCode()` 方法，自定义对象的哈希策略。

------

### 四、哈希值在哈希表中的作用

在 `HashMap` 或 `Hashtable` 中，键的哈希值用于：

1. 计算桶的位置（数组下标）：

```java
int index = hash(key.hashCode()) % array.length;
```

1. 确定是否存在哈希冲突（相同下标多个元素）

------

### 五、哈希值 vs 哈希码 vs 散列值

这几个词在中文语境下经常通用使用，通常都指代 Hash Code。若要区分：

| 中文术语 | 英文       | 含义                                 |
| -------- | ---------- | ------------------------------------ |
| 哈希值   | Hash Value | 一般指具体的散列结果值               |
| 哈希码   | Hash Code  | Java 中常用术语，`hashCode()` 返回值 |
| 散列值   | Hash Value | “散列”是“哈希”的另一种翻译           |
