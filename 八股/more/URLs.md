## ✅ 二、什么是 URL？

**URL（Uniform Resource Locator，统一资源定位符）** 是互联网上**资源的地址**，用于标识和定位 Web 上的各种资源（如网页、图片、API 接口等）。

📌 URL 是 URI（统一资源标识符）的一种，是实际可定位资源的“地址”。

------

## ✅ 三、URL 的结构与组成

一个完整的 URL 格式如下：

```
scheme://user:password@host:port/path?query#fragment
```

| 组件      | 示例                | 含义                                         |
| --------- | ------------------- | -------------------------------------------- |
| scheme    | `https`             | 协议类型，如 http、https、ftp、mailto        |
| user:pass | `user:123@`（可选） | 登录信息（不推荐使用，常见于 FTP）           |
| host      | `www.example.com`   | 主机名或 IP 地址                             |
| port      | `:8080`（可选）     | 端口号（默认可省略）                         |
| path      | `/docs/index.html`  | 服务器上的资源路径                           |
| query     | `?id=123&sort=desc` | 查询字符串，用于传参                         |
| fragment  | `#section2`         | 锚点，定位页面内部位置，**不会发送到服务器** |

📌 示例 URL：

```
https://user:pass@www.example.com:443/docs/index.html?id=123#section2
```

## ✅ 四、URL 与 URI、URN 的区别

| 项目                      | 说明                                                        |
| ------------------------- | ----------------------------------------------------------- |
| **URI（统一资源标识符）** | 抽象概念，唯一标识一个资源（包含 URL 和 URN）               |
| **URL（统一资源定位符）** | 可以定位资源位置（“在哪里”）                                |
| **URN（统一资源名称）**   | 表示资源的名字（“是什么”），如 `urn:isbn:978-7-302-13105-5` |

📌 URL 是 URI 的子集，可以理解为 “能访问的 URI”。

## ✅ 六、URL 编码与安全风险

### ✅ 1. URL 编码（百分号编码）

用于处理特殊字符（空格、中文、@ 等），如：

- 空格 → `%20`
- 中文“你” → `%E4%BD%A0`

编码原因：

- URL 仅支持 ASCII 字符
- 防止语义冲突（如 &、=、#）

### ✅ 2. 安全风险示例

- **URL 重定向漏洞（Open Redirect）**

  ```
  http://example.com/redirect?url=https://malicious.com
  ```

- **参数注入、XSS、CSRF 利用**

  - 通过 `query` 参数篡改行为
  - 伪造钓鱼链接、诱导点击

🔒 防护建议：

- 编码特殊字符
- 校验 `redirect` 参数白名单
- 避免信任 URL 中用户可控部分
