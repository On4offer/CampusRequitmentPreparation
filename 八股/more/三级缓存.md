Spring 中的“**三级缓存**”是指 Spring 容器在解决**单例 Bean 循环依赖问题**时，内部使用的三层缓存机制。这套机制只适用于 **singleton（单例）作用域**，其目的是**提前暴露 Bean 引用，以打破构造死循环**。

------

### 一、为什么需要三级缓存？

在构造单例 Bean 时，若两个 Bean 互相依赖（例如 A 依赖 B，B 依赖 A），而它们都还没完全初始化，就会造成循环依赖。
 为了解决这个问题，Spring 引入了三级缓存，让 Bean 能在尚未完成初始化时就“提前暴露”一个引用给对方。

------

### 二、三级缓存的三个层次（以 `DefaultSingletonBeanRegistry` 为例）

| 缓存级别 | 名称                    | 存储内容                                          | 类型                            |
| -------- | ----------------------- | ------------------------------------------------- | ------------------------------- |
| 一级缓存 | `singletonObjects`      | 完整实例化并初始化完成的 Bean                     | `Map<String, Object>`           |
| 二级缓存 | `earlySingletonObjects` | 实例化但未初始化完成的 Bean（提前暴露的早期引用） | `Map<String, Object>`           |
| 三级缓存 | `singletonFactories`    | 用于创建早期 Bean 引用的 ObjectFactory            | `Map<String, ObjectFactory<?>>` |

------

### 三、工作原理（简化流程）

1. 创建 A 时，发现它依赖 B，于是去创建 B。
2. 创建 B 时，发现它又依赖 A。
3. 此时 A 尚未完全创建，为了避免死循环：
   - Spring 会将 A 的**ObjectFactory**放入三级缓存 `singletonFactories`；
   - B 需要 A 时，Spring 会从三级缓存中获取 ObjectFactory，创建早期的 A 引用；
   - A 的早期引用被放入二级缓存 `earlySingletonObjects`，并从三级缓存移除；
4. 等 B 创建完成，A 再继续完成创建。
5. 最后完整的 A、B 都被放入一级缓存 `singletonObjects`。

------

### 四、示意图

```
                +------------------+
                |  Bean A (创建中) |
                +--------+---------+
                         |
                         v
     +---------------------------------------------+
     |    singletonFactories (3级缓存，ObjectFactory)|
     +---------------------------------------------+
                         |
                         v
     +---------------------------------------------+
     | earlySingletonObjects (2级缓存，早期引用)     |
     +---------------------------------------------+
                         |
                         v
     +---------------------------------------------+
     | singletonObjects (1级缓存，完成的Bean)        |
     +---------------------------------------------+
```

------

### 五、总结

| 级别     | 目的                                            |
| -------- | ----------------------------------------------- |
| 三级缓存 | 保存创建“早期引用”的工厂，用于懒加载或AOP代理等 |
| 二级缓存 | 存已创建但未初始化的Bean实例（防止死循环）      |
| 一级缓存 | 存真正可用的、完整初始化的Bean                  |
