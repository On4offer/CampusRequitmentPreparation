好的，我们来讲讲非常常用的**Redis**，它是现代开发中不可或缺的一种高性能组件，广泛用于缓存、消息队列、排行榜、分布式锁等场景。

------

## 一、什么是 Redis？

**Redis（Remote Dictionary Server）** 是一个**开源的、基于内存的、高性能键值对（Key-Value）数据库**，支持丰富的数据结构。它本质上是一个**运行在内存中的数据结构服务器**。

------

## 二、Redis 的核心特点：

| 特点                 | 说明                                                   |
| -------------------- | ------------------------------------------------------ |
| **高性能**           | 所有数据操作都在内存中，**读写速度非常快**（百万 QPS） |
| **支持多种数据结构** | 不只是字符串，还支持列表、集合、哈希、ZSet 等          |
| **持久化机制**       | 支持将数据保存到磁盘（RDB 和 AOF 两种方式）            |
| **发布/订阅**        | 可以做消息队列、实时通信等                             |
| **分布式支持**       | 支持主从复制、哨兵、集群等机制                         |
| **原子操作**         | 所有操作都是线程安全的，原子执行                       |

------

## 三、Redis 支持的数据类型

| 类型             | 命令示例              | 应用场景                 |
| ---------------- | --------------------- | ------------------------ |
| String           | `SET key value`       | 缓存、计数器、用户信息   |
| List             | `LPUSH list val`      | 消息队列、聊天记录       |
| Hash             | `HSET user name Tom`  | 存储对象（如用户、商品） |
| Set              | `SADD set val`        | 去重、标签、共同好友     |
| ZSet（有序集合） | `ZADD rank 100 user1` | 排行榜、评分系统         |

------

## 四、Redis 常见应用场景

| 场景              | 用法说明                              |
| ----------------- | ------------------------------------- |
| **缓存系统**      | 存储热点数据，减轻数据库压力          |
| **分布式锁**      | 基于 `SETNX` 实现，控制资源竞争       |
| **消息队列**      | 用 List、Pub/Sub 实现简单异步通信     |
| **会话存储**      | 结合 Token 存储用户状态（如登录信息） |
| **排行榜/计数器** | 用 ZSet 做排行榜、INCR 实现高并发计数 |

------

## 五、Redis 的持久化机制

| 类型       | 说明                                 |
| ---------- | ------------------------------------ |
| RDB        | 定时快照保存内存数据，适合冷备份     |
| AOF        | 将每次写操作记录日志，重启可重放恢复 |
| 混合持久化 | Redis 7.0 起支持 RDB+AOF 混合持久化  |

------

## 六、Redis 简单命令示例

```bash
SET name "Alice"
GET name                  # → "Alice"

HSET user:1 name "Tom" age 25
HGETALL user:1            # → name: Tom, age: 25

LPUSH msg "hello"
RPUSH msg "world"
LRANGE msg 0 -1           # → ["world", "hello"]
```

------

## 七、使用 Redis 的 Java 示例（Jedis 客户端）

```java
import redis.clients.jedis.Jedis;

public class RedisDemo {
    public static void main(String[] args) {
        Jedis jedis = new Jedis("localhost", 6379);  // 默认端口
        jedis.set("name", "Alice");
        String value = jedis.get("name");
        System.out.println("value = " + value);
        jedis.close();
    }
}
```

## 八、Redis 的架构部署（简单了解）

| 模式     | 说明                                   |
| -------- | -------------------------------------- |
| 单机模式 | 本地开发或小项目                       |
| 主从复制 | 读写分离，提高读性能                   |
| 哨兵模式 | 自动故障转移，保证高可用               |
| 集群模式 | 支持数据分片，水平扩展，适合大规模系统 |

------

## 九、Redis 的优缺点

| 优点                            | 缺点                               |
| ------------------------------- | ---------------------------------- |
| 极高性能（内存级）              | 内存成本高，数据量大时需持久化处理 |
| 数据结构丰富，灵活性强          | 不支持复杂查询（如 SQL 多表）      |
| 支持事务、Lua、过期、TTL 等特性 | 不适合用于大数据存储               |
