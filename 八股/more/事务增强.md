**事务增强**是指：在方法调用的前后，**自动插入事务控制逻辑**，比如**开启事务、提交事务、回滚事务**等。它是 Spring 框架利用 AOP（面向切面编程）实现的一种典型“增强”（Advice）行为。

------

## 一、为什么叫“增强”？

在 AOP 中，所谓“增强”（Advice）指的是：

> **在原始方法的执行过程前后动态插入附加逻辑**，而不修改原有方法本身。

**事务增强**就是将“事务控制逻辑”织入目标方法调用前后，实现自动管理事务。

------

## 二、事务增强的原理（以 Spring 为例）

Spring 通过为带有 `@Transactional` 注解的方法创建**代理对象**，在方法调用前后插入如下逻辑：

### 方法执行流程（伪代码）：

```java
try {
    beginTransaction();   // 方法执行前 —— 开启事务
    invokeTargetMethod(); // 执行业务逻辑
    commitTransaction();  // 执行成功 —— 提交事务
} catch (Exception e) {
    rollbackTransaction(); // 抛出异常 —— 回滚事务
    throw e;
}
```

------

## 三、事务增强的执行时机与类型（属于 AOP 的一种增强）

事务增强属于 AOP 的 **“环绕增强（Around Advice）”**，它能够控制：

- 方法执行前：开启事务；
- 方法执行后：
  - 正常返回：提交事务；
  - 抛异常：回滚事务。

------

## 四、事务增强的作用

| 场景             | 增强逻辑                             |
| ---------------- | ------------------------------------ |
| 方法开始执行前   | 开启数据库事务                       |
| 方法执行成功后   | 提交事务                             |
| 方法中抛出异常   | 回滚事务                             |
| 异常类型不匹配时 | 默认不回滚（需手动配置 rollbackFor） |

------

## 五、示例代码

```java
@Service
public class OrderService {

    @Transactional
    public void createOrder() {
        // 此方法将被事务增强：开启、提交或回滚
        saveOrder();     // 插入订单
        updateStock();   // 修改库存
    }
}
```

你不需要显式写 `begin`、`commit`，Spring 会在代理层帮你“增强”方法执行。

------

## 六、总结一句话：

> **事务增强就是在你调用业务方法时，自动帮你插入事务的开启、提交与回滚控制，确保数据操作符合 ACID 原则。**
