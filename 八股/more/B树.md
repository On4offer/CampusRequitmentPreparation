好的！下面为你详细介绍 **B树（B-Tree）**，它是 B+树 的基础模型，也是数据库系统、文件系统等大规模数据存储结构中广泛使用的一种 **平衡多路查找树**。

------

## 一、B树的定义（m阶 B树）

一棵 **m 阶 B 树** 是满足以下条件的 **多路平衡查找树**：

1. 每个节点最多有 `m` 个子节点。
2. 每个非根节点至少有 `⌈m/2⌉` 个子节点。
3. 每个非叶节点含有 `k` 个关键字（`k = 子节点数 - 1`），关键字按升序排列。
4. **所有叶子节点在同一层**（树高平衡）。
5. 每个关键字将子树划分成范围区间，使查找更快。

------

## 二、B树的结构示意（以3阶为例）

```
        [10 | 20]
       /     |     \
  [1 5 9] [11 15] [21 25 30]
```

- 内部节点 `10` 和 `20` 作为分割线
- 每个节点可以有多个关键字和多个子节点

------

## 三、B树的核心操作

### 1. 插入

- 找到合适的叶子节点插入关键字
- 如果节点满了（达到 m-1 个关键字），**拆分节点**，中间值上移到父节点，可能递归上升

### 2. 删除

- 从叶子节点删除
- 若节点关键字少于下限（⌈m/2⌉ - 1），通过**合并或借值**维持平衡

### 3. 查找

- 从根节点开始，比较关键字并决定走哪个子树
- 效率为 `O(log n)`，查找路径短（因节点存多个值）

------

## 四、B树 vs 二叉搜索树 vs B+树

| 特性         | 二叉搜索树 | B树         | B+树                           |
| ------------ | ---------- | ----------- | ------------------------------ |
| 节点存储个数 | 一个关键字 | 多个关键字  | 多个关键字（非叶节点只存索引） |
| 节点度       | 2          | 多叉（m阶） | 多叉（m阶）                    |
| 数据存储位置 | 所有节点   | 所有节点    | 仅在叶子节点                   |
| 范围查询     | 较慢       | 一般        | **快（叶子链表）**             |
| 是否磁盘友好 | 否         | 是          | 是                             |

------

## 五、B树的优势

1. **减少磁盘 IO 次数**：节点中存多个关键字，每次读写可获取更多信息
2. **自平衡结构**：无论插入/删除后都能维持平衡
3. **适用于大数据量存储**，非常适合磁盘型数据库

------

## 六、应用场景

- 数据库索引（如 PostgreSQL）
- 文件系统（如 HFS、NTFS）
- 内存数据库（如 Redis 的部分 skiplist 替代方案）

------

## 七、B树 vs B+树 总结

| 特性         | B树                    | B+树                   |
| ------------ | ---------------------- | ---------------------- |
| 数据存储     | 所有节点               | 仅叶子节点             |
| 范围查询     | 较差（要回溯中间节点） | **高效（叶子链表）**   |
| 查询效率     | 可能提前命中中间节点   | **一致地走到叶子节点** |
| 磁盘读取效率 | 一般                   | **更好（索引密集）**   |
