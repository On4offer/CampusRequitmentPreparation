HTTP 的**请求-响应模式**，是指 Web 浏览器（客户端）和 Web 服务器之间的通信过程是通过“**请求（Request）**”和“**响应（Response）**”两个阶段来完成的。整个流程可以简单理解为“**客户端提问，服务器回答**”。

### 一、请求（Request）

客户端（如浏览器）向服务器发送一个 HTTP 请求，请求内容包括以下几部分：

1. **请求行**：说明使用的请求方法（如 [`GET`、`POST`、`PUT`](more/`GET`、`POST`、`PUT`) 等）、请求的[资源路径（URL）](more/资源路径（URL）)和协议版本。
   - 例如：`GET /index.html HTTP/1.1`
2. **请求头（Header）**：包含浏览器信息、请求主机、语言、是否接受压缩等。
   - 例如：`User-Agent: Mozilla/5.0`，`Host: www.example.com`
3. **请求体（Body）**（可选）：主要用于 `POST`、`PUT` 这类方法，包含提交的数据（如表单数据、JSON 等）。

------

### 二、响应（Response）

服务器收到请求后进行处理，并返回一个 HTTP 响应，内容也包括以下几部分：

1. **状态行**：包括 HTTP 版本、[状态码（如 `200 OK`、`404 Not Found`）](more/状态码（如 `200 OK`、`404 Not Found`）)和状态信息。
   - 例如：`HTTP/1.1 200 OK`
2. **响应头（Header）**：描述服务器信息、响应内容类型、长度、缓存策略等。
   - 例如：`Content-Type: text/html; charset=UTF-8`
3. **响应体（Body）**：就是浏览器最终显示的内容，比如 HTML 网页、图片、视频等。

------

### 三、举个简单例子

假设你在浏览器输入网址 `http://example.com`：

1. 浏览器发送请求：

   ```
   GET / HTTP/1.1
   Host: example.com
   ```

2. 服务器返回响应：

   ```
   HTTP/1.1 200 OK
   Content-Type: text/html
   
   <html><body>欢迎访问 Example 网站</body></html>
   ```

------

这个模式的特点是：

- 一次请求对应一次响应。
- 客户端主动发起请求，服务器不会主动推送数据（除非用 WebSocket）。
