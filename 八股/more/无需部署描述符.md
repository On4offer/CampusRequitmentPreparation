### Spring Boot 的“无须部署描述符”特性

Spring Boot 的“无须部署描述符”（**No need for a deployment descriptor**）是该框架的一大特色。传统的 Java Web 应用通常需要一个 **部署描述符**（`web.xml`）来配置和管理Web应用的各种设置，比如Servlet、Filter、Listener等。而Spring Boot采用约定优于配置的原则，通过自动配置和内嵌服务器的使用，去除了传统Java Web应用中需要手动配置的`web.xml`文件，使得开发过程更加简化。

### 1. **传统Java Web应用的`web.xml`**

在传统的Java Web应用（基于Servlet规范的Web应用）中，`web.xml`（位于`WEB-INF`目录下）是一个非常重要的文件，它包含了应用的配置和元数据，如Servlet的映射、Filter的定义、Listener的配置等。这个文件是必须的，用于告知Servlet容器（如Tomcat、Jetty等）如何加载和管理应用。

一个典型的`web.xml`文件的示例如下：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://java.sun.com/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://java.sun.com/xml/ns/javaee
                             http://java.sun.com/xml/ns/javaee/web-app_3_0.xsd"
         version="3.0">
    <servlet>
        <servlet-name>dispatcherServlet</servlet-name>
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
        <load-on-startup>1</load-on-startup>
    </servlet>

    <servlet-mapping>
        <servlet-name>dispatcherServlet</servlet-name>
        <url-pattern>/</url-pattern>
    </servlet-mapping>
</web-app>
```

### 2. **Spring Boot 无须部署描述符的实现**

Spring Boot 提供了**无`web.xml`配置**的功能，通过**自动配置**和**内嵌的Web服务器**（如Tomcat、Jetty或Undertow）来管理Web应用的启动和运行，避免了手动配置和部署描述符。Spring Boot的目标是简化应用程序的开发和配置，使开发者能够专注于业务逻辑，而不是繁琐的配置文件。

#### 2.1 自动配置Web应用

在Spring Boot中，`@SpringBootApplication`注解通常是应用的入口点，并且包含了`@EnableAutoConfiguration`，该注解会自动配置Web应用所需的组件和Servlet容器。Spring Boot会根据应用的依赖自动推测并配置Web服务器及其他相关组件，无需手动创建或配置`web.xml`。

#### 2.2 内嵌的Web服务器

Spring Boot内嵌了Servlet容器（如Tomcat、Jetty或Undertow），这意味着你不再需要外部的Servlet容器部署应用。Spring Boot在应用启动时自动启动内嵌服务器，并通过`@SpringBootApplication`的配置来自动注册`DispatcherServlet`。

例如，以下是一个典型的Spring Boot应用入口类：

```java
package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class DemoApplication {

    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
```

这段代码中，`@SpringBootApplication`注解包含了`@EnableAutoConfiguration`和`@ComponentScan`，Spring Boot会自动配置和扫描所有的`@Controller`、`@Service`、`@Repository`等注解标记的类，并自动注册`DispatcherServlet`，启动内嵌的Tomcat服务器。

#### 2.3 内嵌Servlet容器的自动配置

Spring Boot根据依赖和默认配置自动配置内嵌的Servlet容器。例如，如果你添加了`spring-boot-starter-web`依赖，Spring Boot会自动引入并配置Tomcat作为默认的Web容器。如果需要切换到Jetty或Undertow，只需要排除Tomcat依赖并添加相应的Web服务器依赖。

##### 默认配置（Tomcat）

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
```

##### 切换到Jetty

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-jetty</artifactId>
</dependency>

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-tomcat</artifactId>
    <scope>provided</scope>
</dependency>
```

##### 切换到Undertow

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-undertow</artifactId>
</dependency>

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-tomcat</artifactId>
    <scope>provided</scope>
</dependency>
```

### 3. **Spring Boot 的自动配置和 `web.xml` 的替代**

Spring Boot通过自动配置和约定优于配置（Convention over Configuration）原则，减少了开发者的配置工作。以下是几个关键点，解释了如何使用Spring Boot替代传统Web应用中的`web.xml`。

#### 3.1 自动注册`DispatcherServlet`

Spring Boot会自动注册`DispatcherServlet`，这是Spring MVC应用的核心。你不需要在`web.xml`中手动配置它。

#### 3.2 自动扫描和配置控制器

Spring Boot会自动扫描类路径中的所有`@Controller`、`@RestController`、`@Service`等注解，并将它们自动注册为Spring Bean。你无需在`web.xml`中声明每个Servlet或Filter。

#### 3.3 内嵌服务器配置

Spring Boot提供了内嵌服务器（如Tomcat），并且默认启动时会启动Tomcat服务器并将应用绑定到指定端口（默认是8080）。开发者无需配置和管理外部服务器。

#### 3.4 自定义配置

虽然Spring Boot默认提供自动配置，但你依然可以通过配置文件（`application.properties` 或 `application.yml`）自定义一些配置项，例如自定义端口、上下文路径等。

```properties
server.port=8081
server.servlet.context-path=/myapp
```

### 4. **总结**

Spring Boot的“无须部署描述符”特性简化了传统Java Web应用的配置和部署。通过自动配置、内嵌的Servlet容器和约定优于配置的理念，Spring Boot避免了传统Web应用中`web.xml`文件的繁琐配置。开发者只需专注于业务逻辑，框架自动处理大部分配置和管理工作。

通过这一特性，Spring Boot不仅简化了开发过程，还减少了配置的复杂性，使得Web应用的开发、部署和维护更加高效和方便。
