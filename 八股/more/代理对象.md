## ✅ 什么是代理对象？

> 代理对象（Proxy Object）是一个**代表真实目标对象**进行方法调用的中间人，它通过控制方法调用来**添加额外逻辑**，如权限校验、日志记录、事务管理等。

通俗地说：

> **别人想找目标对象干活，必须先通过代理对象**。代理可以在干活前/后插点私货（增强功能）。

------

## 🎯 为什么使用代理？

- **解耦业务与通用逻辑**（如事务、权限、日志）
- **遵循开闭原则（OCP）**：不改源代码就能增强行为
- **实现 AOP（面向切面编程）** 的基础

------

## 🔧 代理对象的分类

| 类型     | 特点             | 实现方式          |
| -------- | ---------------- | ----------------- |
| 静态代理 | 编译期写死代理类 | 手写或代码生成    |
| 动态代理 | 运行时动态生成   | JDK Proxy / CGLIB |

------

## 🧪 示例一：静态代理

```java
interface Service {
    void doSomething();
}

class RealService implements Service {
    public void doSomething() {
        System.out.println("干正事");
    }
}

class ServiceProxy implements Service {
    private RealService target = new RealService();

    public void doSomething() {
        System.out.println("前置处理");
        target.doSomething();
        System.out.println("后置处理");
    }
}
```

调用：

```java
Service s = new ServiceProxy();
s.doSomething();
```

输出：

```
前置处理
干正事
后置处理
```

------

## 🔥 示例二：JDK 动态代理（基于接口）

```java
Service proxy = (Service) Proxy.newProxyInstance(
    Service.class.getClassLoader(),
    new Class[]{Service.class},
    (proxyObj, method, args) -> {
        System.out.println("前置");
        Object result = method.invoke(new RealService(), args);
        System.out.println("后置");
        return result;
    }
);
proxy.doSomething();
```

------

## 🚀 示例三：CGLIB 动态代理（基于类）

> 用于代理**没有接口的类**

```java
Enhancer enhancer = new Enhancer();
enhancer.setSuperclass(RealService.class);
enhancer.setCallback(new MethodInterceptor() {
    public Object intercept(Object obj, Method method, Object[] args, MethodProxy proxy) throws Throwable {
        System.out.println("前置");
        Object result = proxy.invokeSuper(obj, args);
        System.out.println("后置");
        return result;
    }
});
RealService proxy = (RealService) enhancer.create();
proxy.doSomething();
```

## 🧠 Spring 中的代理对象（核心）

| 场景           | 代理方式            |
| -------------- | ------------------- |
| 有接口         | 使用 JDK 动态代理   |
| 没有接口（类） | 使用 CGLIB 动态代理 |
| AOP 切面增强   | 依赖代理对象        |
| @Transactional | 本质是代理实现事务  |

## ✅ 一、在 Java 中：代理对象由**JVM 或 CGLIB 框架创建**

### 1. **JDK 动态代理：由 `Proxy.newProxyInstance()` 创建**

适用于接口代理。

```java
MyInterface proxy = (MyInterface) Proxy.newProxyInstance(
    classLoader,
    new Class[]{MyInterface.class},
    invocationHandler
);
```

- 创建过程是：JVM 动态生成一个新的 class（代理类），继承 `Proxy`，实现你给的接口；
- 实际字节码类是在内存中动态创建并加载的。

👉 **由 JVM 动态生成并加载**。

------

### 2. **CGLIB 代理：由 `Enhancer.create()` 创建**

适用于类代理（没有接口的情况）：

```java
RealClass proxy = (RealClass) Enhancer.create(
    RealClass.class,
    new MyMethodInterceptor()
);
```

- CGLIB 会**生成一个子类**，重写目标类的方法；
- 实际是 ASM 字节码工具动态生成 `.class`，然后通过 classloader 加载。

👉 **由 CGLIB 动态生成 class 并实例化**。

------

## ✅ 二、在 Spring 中：代理对象由**Spring 容器在 Bean 初始化后统一创建**

### 🧠 Spring 是怎么创建代理对象的？

#### 当你使用了：

- `@Transactional`
- `@Async`
- `@Cacheable`
- AOP 切面（`@Aspect`）
- Security 权限检查

这些都依赖 **代理机制**。

Spring 在 Bean 生命周期中：

### 🪄 **通过 BeanPostProcessor（特别是 `AnnotationAwareAspectJAutoProxyCreator`）统一处理**

流程是：

1. Spring 实例化 Bean（目标对象）；
2. 走 `BeanPostProcessor#postProcessAfterInitialization()`；
3. 判断这个 Bean 是否需要代理（是否有切面/事务等增强）；
4. 如果需要，就创建代理对象（JDK or CGLIB）替代原对象；
5. 最终注入的是代理对象，而不是原始对象。

👉 所以：

> 在 Spring 中，**代理对象是由容器在 Bean 初始化后统一创建的**，通过动态代理技术（JDK 或 CGLIB）生成。

------

## ✅ 总结：代理对象由谁创建？

| 场景               | 创建者/方式                              |
| ------------------ | ---------------------------------------- |
| Java JDK 代理      | JVM 动态创建实现类（基于接口）           |
| CGLIB 代理         | CGLIB + ASM 创建子类                     |
| Spring AOP、事务等 | Spring 容器通过 `BeanPostProcessor` 创建 |
| MyBatis Mapper     | 也是通过 JDK 动态代理生成代理对象        |

------

## 💬 面试答题模板

> 代理对象的创建方式取决于具体的应用场景。在原生 Java 中，JDK 动态代理通过 JVM 在运行时生成实现接口的类，而 CGLIB 通过继承目标类生成子类。在 Spring 中，代理对象是在 Bean 初始化之后由 Spring 容器通过 BeanPostProcessor 自动创建的，并使用 JDK 或 CGLIB 代理机制完成增强，是 AOP、事务等功能的基础。
