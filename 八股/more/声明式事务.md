## ✅ 什么是声明式事务？

> **声明式事务** 是通过注解或 XML 配置的方式，**让开发者无需手写事务控制代码**，由框架（如 Spring）**自动完成事务的开启、提交、回滚**。

通俗理解：

> 你只需要在方法上 **“声明”** 一下：这个方法要事务，剩下的事（开事务、遇异常就回滚、成功就提交）**Spring 自动帮你完成**。

------

## 📦 使用方式（Spring 中的标准写法）

```java
@Service
public class OrderService {

    @Transactional
    public void createOrder() {
        saveOrder();    // 开启事务
        saveLog();      // 同一个事务
        // 如果出现异常自动回滚
    }
}
```

也可以加在类上，表示**类中所有 public 方法都启用事务**：

```java
@Transactional
public class UserService { ... }
```

------

## ⚙️ 声明式事务的工作原理

1. Spring 启用事务管理器（`@EnableTransactionManagement`）；
2. 注册事务切面代理（基于 AOP）；
3. 在运行时拦截标注了 `@Transactional` 的方法；
4. 执行前开启事务；
5. 正常执行完就提交，抛异常就回滚；
6. 提交或回滚后释放资源。

📌 本质：**基于 AOP 的方法增强 + 事务管理器控制**。

------

## ✅ `@Transactional` 常用参数

| 属性名          | 说明                                                    |
| --------------- | ------------------------------------------------------- |
| `propagation`   | 事务传播行为（默认 `REQUIRED`）                         |
| `isolation`     | 事务隔离级别（如 `READ_COMMITTED`）                     |
| `timeout`       | 超时时间，超出则回滚                                    |
| `readOnly`      | 是否只读（用于优化 SELECT 语句）                        |
| `rollbackFor`   | 指定哪些异常类型触发回滚（默认仅回滚 RuntimeException） |
| `noRollbackFor` | 指定哪些异常不触发回滚                                  |

------

## ✅ 优点

- ✅ **非侵入式**：业务代码中无需写事务开启/提交逻辑；
- ✅ **统一管理**：统一配置事务属性，方便维护；
- ✅ **适合大多数常规业务场景**；
- ✅ **结合 AOP 支持横切逻辑**（比如记录事务日志）。

------

## ⚠️ 常见坑 & 注意事项

| 问题                         | 原因                                        |
| ---------------------------- | ------------------------------------------- |
| 事务不生效（自己类内部调用） | Spring AOP 不会拦截内部方法调用             |
| 异常被捕获，事务不回滚       | Spring 默认只对抛出的 RuntimeException 回滚 |
| 加在 private 方法无效        | 代理无法增强非 public 方法                  |
| 非代理类调用事务方法无效     | 直接调用未代理对象，事务不起作用            |
| 多线程/异步场景事务失效      | 线程隔离，事务上下文无法自动传递            |

------

## 🧠 适合声明式事务的场景

| 适用情况                           | 说明                           |
| ---------------------------------- | ------------------------------ |
| 绝大多数业务服务方法               | 调用 DAO/Service，涉及增删改查 |
| 方法边界清晰，事务行为统一         | 无需对事务细节手动干预         |
| AOP 生效的类中（非 final/private） | 保证代理机制能工作             |

------

## 💬 面试标准答题模板

> 声明式事务是通过注解或配置方式，由 Spring 框架自动管理事务的开启、提交和回滚，无需开发者手动控制。常用注解是 `@Transactional`，配合 AOP 实现方法增强。它适用于绝大多数业务服务逻辑，是企业开发中最常用的事务控制方式。但要注意不能在内部方法、自调用、异步线程中使用，否则事务可能失效。

------

## ✅ 一句话总结

> 声明式事务 = “用注解告诉框架要事务，剩下交给 Spring 自动搞定”。
