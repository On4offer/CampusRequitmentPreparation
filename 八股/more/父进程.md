“**父进程**”是操作系统中进程管理的一个核心概念，它指的是**创建了其他进程的进程**。

------

## 一、什么是父进程？

在操作系统中，一个程序运行时就是一个**进程（Process）**。当一个进程创建另一个进程时，前者就被称为“父进程”，后者叫“子进程”。

- 父进程可以拥有多个子进程。
- 子进程也可以再创建自己的子进程，形成“进程树”。

------

## 二、举个类比：

把操作系统想象成一个家庭：

- 父亲（父进程）生了一个孩子（子进程）。
- 这个孩子也可以长大后再生孩子（孙子进程）。

------

## 三、实际例子（以 Linux 为例）：

### 在 Linux 中：

- 每个进程都有一个 **PID（进程号）** 和一个 **PPID（父进程号）**。
- 可以用 `ps -ef` 查看：

```
UID   PID  PPID  CMD
root    1     0   /sbin/init
alice 3001  2890  python my_script.py
```

表示：

- PID 为 3001 的进程是 `python` 程序
- 它的父进程 PID 是 2890

------

## 四、在编程中的父子进程（Python 举例）

```python
import os

pid = os.fork()  # 创建子进程

if pid == 0:
    print("我是子进程，PID:", os.getpid())
else:
    print("我是父进程，PID:", os.getpid(), "我的子进程是", pid)
```

------

## 五、父进程的作用：

| 功能         | 说明                                   |
| ------------ | -------------------------------------- |
| 创建子进程   | 如调用 fork()、spawn() 等              |
| 管理子进程   | 等待子进程结束（wait）、收集返回值     |
| 控制资源     | 给子进程分配或限制资源                 |
| 传递环境信息 | 子进程继承父进程的环境变量和文件描述符 |

------

## 六、常见问题：

### 1. 如果父进程挂了，子进程会怎样？

- 在 Linux 中，**子进程会被 `init` 进程（PID=1）接管**，变成“孤儿进程”。

### 2. 父进程不等待子进程会怎样？

- 子进程退出后变“僵尸进程”，占用系统资源。
- 正常应使用 `wait()` 或 `waitpid()` 处理。
