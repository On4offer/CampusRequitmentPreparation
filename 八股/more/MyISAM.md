**MyISAM** 是 MySQL 的一种经典存储引擎，它是 InnoDB 出现之前的默认引擎，特点是结构简单、读写速度快，但功能上不如 InnoDB 强大，特别是在事务支持和并发控制方面。

------

### 一、MyISAM 的核心特点

| 特性               | 描述                                           |
| ------------------ | ---------------------------------------------- |
| **不支持事务**     | 没有事务、回滚、提交机制（不支持 ACID）        |
| **表级锁**         | 每次操作锁整个表，读写冲突时会阻塞             |
| **不支持外键**     | 无法强制表之间的数据关系                       |
| **存储结构简单**   | 数据存储在 `.MYD` 文件中，索引在 `.MYI` 文件中 |
| **读写分离性能高** | 多读少写的场景下表现非常好                     |
| **崩溃恢复差**     | 无崩溃恢复机制，写入中断容易数据损坏           |

------

### 二、适用场景

- 数据主要**读多写少**（如静态内容展示、日志查询）
- 不需要事务保证（如简单信息查询系统）
- 对性能极致追求，容忍数据一致性风险

------

### 三、MyISAM 与 InnoDB 对比

| 特性       | MyISAM                 | InnoDB               |
| ---------- | ---------------------- | -------------------- |
| 事务支持   | 否                     | 是                   |
| 锁粒度     | 表级锁                 | 行级锁（支持表锁）   |
| 外键支持   | 否                     | 是                   |
| 数据安全性 | 崩溃易损坏             | 支持崩溃恢复         |
| 读取性能   | 快                     | 稍慢（有事务开销）   |
| 并发写性能 | 差（容易[阻塞](阻塞)） | 好（行锁 + MVCC）    |
| 默认引擎   | 否（旧版本是）         | 是（MySQL 5.5 之后） |

------

### 四、基本使用

创建使用 MyISAM 的表：

```sql
CREATE TABLE example (
    id INT,
    name VARCHAR(100)
) ENGINE=MyISAM;
```

查看表的存储引擎：

```sql
SHOW TABLE STATUS LIKE 'example';
```

------

### 五、不足与淘汰趋势

- 不支持事务，是最大的短板；
- 表级锁并发性能差；
- 在高并发、复杂系统中已**基本被 InnoDB 替代**；
- 仅适用于**极简、读密集型、数据风险可容忍**的系统。

------

### 六、总结一句话：

> **MyISAM 简单、快速、轻量，但缺乏事务、并发和数据安全能力，现代项目几乎都改用 InnoDB。**
