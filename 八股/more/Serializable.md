**`SERIALIZABLE`（可串行化）** 是事务隔离级别中**最严格**的一种，它能完全避免并发事务中的所有问题：**脏读、不可重复读、幻读**，从而提供最强的数据一致性保障。

------

### 一、核心特性

| 特性           | 说明                                             |
| -------------- | ------------------------------------------------ |
| **脏读**       | 不可能发生（读取的都是已提交数据）               |
| **不可重复读** | 不可能（同一行数据在事务中始终一致）             |
| **幻读**       | 不可能（阻止插入/删除影响查询结果）              |
| **加锁方式**   | 对所有读写操作加**范围锁或表锁**，**读也会加锁** |
| **并发性能**   | 最低，事务基本是“排队”执行                       |

------

### 二、行为描述

- 所有事务仿佛按**串行顺序执行**；
- 即使是 `SELECT` 查询也会加**共享锁（S锁）**，防止其他事务插入、更新、删除相关数据；
- 保证事务之间**零交叉影响**，从根本上消除并发问题。

------

### 三、示例

```sql
-- 设置隔离级别为 SERIALIZABLE
SET SESSION TRANSACTION ISOLATION LEVEL SERIALIZABLE;

START TRANSACTION;
SELECT * FROM users WHERE age > 20;  -- 加锁，阻止其他事务插入 age > 20 的记录
```

如果其他事务尝试插入 age 为 25 的记录，会**被阻塞**直到当前事务提交。

------

### 四、与其他隔离级别对比

| 隔离级别         | 脏读 | 不可重复读 | 幻读                | 性能   | 默认（MySQL） |
| ---------------- | ---- | ---------- | ------------------- | ------ | ------------- |
| READ UNCOMMITTED | 是   | 是         | 是                  | 高     | 否            |
| READ COMMITTED   | 否   | 是         | 是                  | 较高   | 否            |
| REPEATABLE READ  | 否   | 否         | **MySQL中防止幻读** | 中等   | **是**        |
| **SERIALIZABLE** | 否   | 否         | 否                  | **低** | 否            |

------

### 五、使用场景

适用于：

- 银行转账、财务结算等**强一致性要求**的业务；
- 少量并发、对性能要求不高但不能容忍任何并发异常的系统；
- 对事务结果要求“像是串行执行”的应用。

------

### 六、小结一句话：

> `SERIALIZABLE` 是最安全但最慢的隔离级别，它通过**加强锁控制使并发事务效果如同逐个执行**，几乎消除所有并发异常，适合极端一致性场景。
