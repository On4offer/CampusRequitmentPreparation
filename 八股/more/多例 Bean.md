在Spring框架中，**Bean的作用域（Scope）**决定了Spring容器如何创建和管理Bean对象。常见的两种作用域是：**单例（Singleton）\**和\**多例（Prototype）**。

------

### 1. 单例 Bean（Singleton Scope）

**定义：**
 在Spring容器中，**默认的作用域是单例**。这意味着无论你注入多少次、请求多少次，Spring只会创建一个该Bean的实例。

**特征：**

- 全局只有一个实例（在同一个Spring容器中）。
- 在容器启动时就会创建该Bean（预初始化）。
- 用于无状态的、线程安全的组件。

**配置方式：**（默认就是单例）

```java
@Component
//@Scope("singleton") // 可写可不写
public class MySingletonBean {
}
```

------

### 2. 多例 Bean（Prototype Scope）

**定义：**
 每次从Spring容器中获取该Bean时，都会**创建一个新的实例**。

**特征：**

- 每次调用`getBean()`都会生成一个新的对象。
- 容器只负责创建，不负责管理其生命周期（如销毁）。
- 适用于有状态的、非线程安全的对象。

**配置方式：**

```java
@Component
@Scope("prototype")
public class MyPrototypeBean {
}
```

------

### 总结对比：

| 特性     | 单例Bean (singleton)             | 多例Bean (prototype)       |
| -------- | -------------------------------- | -------------------------- |
| 实例数量 | 容器中一个全局实例               | 每次获取创建新实例         |
| 生命周期 | 容器创建时初始化、容器销毁时销毁 | 每次调用都创建，容器不销毁 |
| 应用场景 | 线程安全、共享资源               | 非线程安全、有状态对象     |
| 默认类型 | 是                               | 否（需显式配置）           |
