在数据库中，**主键（Primary Key）** 是用于唯一标识一条记录（行）的一个或多个列（字段）的组合。它是关系数据库中最基本也是最重要的约束之一。

------

### 一、主键的特点

1. **唯一性（UNIQUE）**
    每一行记录的主键值必须唯一，不能重复。
2. **非空性（NOT NULL）**
    主键字段不能为 NULL，必须有值。
3. **自动索引**
    创建主键时，数据库自动在该字段上创建唯一索引，加速查询效率。
4. **每个表只能有一个主键**
    可以是单个字段，也可以是多个字段组合（称为复合主键）。

------

### 二、创建主键的方式

#### 1. 创建表时定义主键：

```sql
CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(50)
);
```

#### 2. 表已存在时添加主键：

```sql
ALTER TABLE users ADD PRIMARY KEY (id);
```

#### 3. 复合主键（多个字段联合主键）：

```sql
CREATE TABLE orders (
    order_id INT,
    product_id INT,
    PRIMARY KEY (order_id, product_id)
);
```

------

### 三、主键 vs 唯一键（UNIQUE）

| 比较点         | 主键（PRIMARY KEY） | 唯一键（UNIQUE） |
| -------------- | ------------------- | ---------------- |
| 是否允许 NULL  | 否                  | 允许多个 NULL    |
| 每张表能有几个 | 只能一个            | 可以多个         |
| 自动索引       | 是                  | 是               |

------

### 四、InnoDB 中的主键意义

- InnoDB 是**基于主键[聚簇索引](聚簇索引)（Clustered Index）**存储的。
- 每一条记录在物理上按主键顺序存储，主键越短越好。
- 如果没有定义主键，InnoDB 会：
  1. 找一个唯一且非空的列作为主键；
  2. 如果找不到，会自动生成一个隐式主键（6字节 ROWID）。

------

### 五、设计建议

- 主键应简短、稳定且不可变。
- 通常建议使用整型（如 INT 或 BIGINT）作为主键。
- 不建议使用 UUID 作为主键，除非必须（因为 UUID 长、无序，不利于索引性能）。
