好的，下面是关于 **`SETNX`** 的详细介绍，它是 Redis 中非常常用的一个命令，尤其在**实现分布式锁**时非常重要。

------

## 一、SETNX 是什么？

- **全称：SET if Not eXists**
- 用于：**仅在键不存在时设置键值对**
- 属于 Redis 的**原子操作**（不会被打断）

------

## 二、命令格式

```redis
SETNX key value
```

- 如果 `key` **不存在**，则设置 `key = value`，并返回 `1`
- 如果 `key` **已经存在**，则什么都不做，返回 `0`

------

## 三、示例

```redis
SETNX mylock "123"
```

假设 `mylock` 不存在，执行后：

- 设置成功，返回 `1`

再次执行：

```redis
SETNX mylock "456"
```

- 因为 `mylock` 已存在，返回 `0`，不会覆盖原来的值

------

## 四、常见用途：**实现简单分布式锁**

```redis
SETNX lock:order "UUID"     # 尝试上锁
EXPIRE lock:order 10        # 设置过期时间防死锁
```

或者更安全的写法（Redis >= 2.6.12）：

```redis
SET lock:order "UUID" NX EX 10
```

- `NX`：只有 key 不存在才设置（=SETNX）
- `EX`：设置过期时间（秒）

这样做可以避免死锁，也更符合真实业务中对锁超时释放的要求。

------

## 五、SETNX vs SET（NX）

| 命令               | Redis 版本    | 功能描述                    |
| ------------------ | ------------- | --------------------------- |
| `SETNX`            | 较早          | 如果键不存在则设置          |
| `SET key value NX` | Redis 2.6.12+ | 更灵活，支持 EX/PX 过期参数 |

------

## 六、注意事项

1. `SETNX` 不支持设置过期时间，必须配合 `EXPIRE` 命令，否则可能死锁
2. `SET` 命令配合 `NX EX` 是更推荐的方式，功能更完整
3. 分布式锁真正安全实现需考虑原子性 + 唯一标识 + 解锁机制，推荐用 Redisson 或自己加 Lua 脚本控制

------

## 七、总结一句话

> **SETNX 是 Redis 提供的一个“仅当 key 不存在时设置”的原子命令，常用于并发控制、分布式锁、唯一标记场景。**
