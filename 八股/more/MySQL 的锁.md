MySQL 的锁机制是数据库确保数据一致性、隔离性和并发控制的重要手段。MySQL 支持多种类型的锁，主要可以分为以下几类：

------

### 一、按作用范围分类：

#### 1. **全局锁（Global Lock）**

- 锁住整个数据库实例。
- 常用于备份操作，例如使用 `FLUSH TABLES WITH READ LOCK`。
- 所有库、表都无法进行写操作，影响大。

#### 2. **表级锁（Table Lock）**

- 锁定整个表。
- 语法：`LOCK TABLES table_name READ/WRITE;`
- **特点**：
  - 简单、开销小。
  - 并发性较低。
- 适用于 MyISAM 引擎或某些管理任务（如批量导入）。

#### 3. **行级锁（Row Lock）**

- 锁定表中的某一行或多行数据。
- InnoDB 引擎支持。
- 并发能力强，粒度小，但开销较大。
- 自动加锁，不需手动指定。

------

### 二、按锁类型分类：

#### 1. **共享锁（S 锁 / Shared Lock）**

- 多个事务可以同时读，但不能写。
- 显式语法：`SELECT ... LOCK IN SHARE MODE;`

#### 2. **排他锁（X 锁 / Exclusive Lock）**

- 一次只能被一个事务持有。
- 该行数据不能被其他事务读或写。
- 显式语法：`SELECT ... FOR UPDATE;`

------

### 三、InnoDB 的锁机制详解：

InnoDB 是 MySQL 默认的存储引擎，支持多种高级锁：

#### 1. **意向锁（Intention Locks）**

- 表级锁，用于协调行锁和表锁之间的关系。
- 自动加锁，无需手动管理。
- 包括意向共享锁（IS）和意向排他锁（IX）。

#### 2. **记录锁（Record Lock）**

- 锁住索引记录上的某一行。
- 适用于精确匹配的查询。

#### 3. **间隙锁（Gap Lock）**

- 锁住两个索引记录之间的“间隙”。
- 防止幻读（Phantom Read）。

#### 4. **临键锁（Next-Key Lock）**

- 记录锁 + 间隙锁。
- 是 InnoDB 默认使用的行锁类型。

------

### 四、锁的常见问题：

- **死锁**：两个事务互相等待对方释放锁。
- **锁等待超时**：事务长时间等待锁释放。
- **幻读**：一个事务两次读取发现中间有新数据插入。
