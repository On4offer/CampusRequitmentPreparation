### Spring Boot的外部化配置

**外部化配置**是Spring Boot的重要特性之一，它允许将应用程序的配置从代码中提取出来，并存储在外部的文件或环境中。这样做的好处是可以在不同的环境中使用不同的配置，而不需要修改应用程序的代码。Spring Boot支持多种方式来实现外部化配置，包括配置文件、环境变量、命令行参数等。

### 1. **为什么使用外部化配置**

- **环境隔离**：在不同的环境（开发、测试、生产等）中使用不同的配置，而无需修改代码。
- **灵活性**：允许根据不同的部署需求调整配置，而不必重新编译或重新部署应用。
- **增强可维护性**：将配置从代码中分离，使得应用程序更容易管理和维护。

### 2. **Spring Boot的外部化配置方式**

Spring Boot提供了多种外部化配置的方法，开发者可以根据需要选择适合的方式。

#### 2.1 配置文件（`application.properties` 或 `application.yml`）

Spring Boot默认使用`application.properties`（或`application.yml`）作为配置文件来加载应用配置。你可以将这些文件放在`src/main/resources`目录下，也可以放在外部文件系统中的任意位置。

**使用`application.properties`**：

```properties
# 配置服务器端口
server.port=8080

# 配置数据库连接
spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=root
spring.datasource.password=root
```

**使用`application.yml`**（YAML格式）：

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mydb
    username: root
    password: root
```

#### 2.2 外部化配置文件

Spring Boot允许将配置文件存储在应用外部，可以通过以下方式将`application.properties`或`application.yml`文件放到外部：

- **指定配置文件路径**：通过`spring.config.location`属性指定配置文件的位置。

  启动应用时，可以指定外部配置文件的位置：

  ```bash
  java -jar app.jar --spring.config.location=/path/to/config/application.properties
  ```

- **加载多个配置文件**：你还可以通过`spring.config.additional-location`指定额外的配置文件：

  ```bash
  java -jar app.jar --spring.config.additional-location=/path/to/extra-config/
  ```

#### 2.3 使用环境变量

Spring Boot支持从环境变量加载配置。这对于Docker容器部署和云服务平台非常有用，环境变量可以在运行时提供配置。

Spring Boot的外部配置会自动识别环境变量，格式为`SPRING_属性名`。例如，配置`server.port`的环境变量可以是：

```bash
export SPRING_SERVER_PORT=8080
```

此时，Spring Boot会自动读取环境变量并覆盖`application.properties`中的相应配置。

#### 2.4 使用命令行参数

你还可以通过命令行参数来外部化配置。这种方式适用于临时修改应用配置，例如：

```bash
java -jar app.jar --server.port=8081 --spring.datasource.url=jdbc:mysql://localhost:3306/anotherdb
```

#### 2.5 配置优先级

Spring Boot配置的加载优先级从高到低如下：

1. **命令行参数**：`java -jar app.jar --server.port=8081`
2. **环境变量**：`SPRING_SERVER_PORT`
3. **外部配置文件**：通过`--spring.config.location`加载的配置文件
4. **`application.properties` 或 `application.yml`**：默认的配置文件
5. **默认配置**：Spring Boot的默认配置

#### 2.6 `@Value`注解

在Spring Boot中，可以使用`@Value`注解直接将配置值注入到Spring的Bean中。例如：

```java
@Component
public class MyComponent {
  
    @Value("${server.port}")
    private int serverPort;

    public void printPort() {
        System.out.println("Server Port: " + serverPort);
    }
}
```

这样，`server.port`的值就会从配置文件或环境变量中读取，并注入到`serverPort`字段。

#### 2.7 `@ConfigurationProperties`注解

`@ConfigurationProperties`注解允许将一组配置映射到一个Bean类中，这样可以避免在多个地方手动注入配置值，尤其在配置项较多时非常方便。

**使用示例**：

```java
@ConfigurationProperties(prefix = "spring.datasource")
public class DataSourceConfig {
    private String url;
    private String username;
    private String password;

    // Getter and Setter methods
}
```

然后，你可以在配置文件中配置相应的属性：

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=root
spring.datasource.password=root
```

在Spring Boot应用中，`@ConfigurationProperties`将自动将这些配置注入到`DataSourceConfig`类中。

#### 2.8 Profile-specific配置

Spring Boot支持在不同的环境中使用不同的配置，通过使用**Profile**来隔离不同的配置。你可以定义多个`application-{profile}.properties`（或`application-{profile}.yml`）文件来为不同环境提供不同的配置。

- **定义开发环境配置**：`application-dev.properties`
- **定义生产环境配置**：`application-prod.properties`

然后，使用`spring.profiles.active`属性来激活特定的环境配置：

```properties
# 在开发环境中使用dev配置
spring.profiles.active=dev
```

通过这种方式，你可以轻松为不同的环境（开发、测试、生产）配置不同的参数。

#### 2.9 Spring Cloud Config

在分布式系统或微服务架构中，配置管理往往是一个挑战。Spring Cloud Config提供了集中式配置管理，可以将配置存储在Git、SVN等版本控制系统中，供多个服务共享使用。

Spring Cloud Config Server可以将配置从外部存储加载并应用到所有客户端应用中，实现集中管理和动态刷新配置。

### 3. **总结**

Spring Boot的外部化配置使得应用程序在不同的环境中更容易进行配置和管理。开发者可以通过配置文件、环境变量、命令行参数等方式提供配置，灵活应对不同的部署需求。外部化配置有助于实现配置的集中管理、简化不同环境的配置切换，并且减少了在不同环境中修改代码的需求，从而提高了应用的可维护性和灵活性。
