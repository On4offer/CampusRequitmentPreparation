AOP（面向切面编程，**Aspect-Oriented Programming**）的代理机制是实现 AOP 核心功能（如日志、权限、事务等“横切逻辑”）的关键手段。下面是对其代理机制的详细介绍：

------

## 一、AOP的核心思想

AOP 通过在不修改源代码的前提下，**在程序运行期间动态地将横切逻辑织入业务逻辑中**，实现模块化、可插拔的代码结构。

------

## 二、AOP 的两种常见代理机制

### 1. **JDK 动态代理（接口代理）**

- 要求：目标类必须实现一个或多个接口；
- 实现：使用 Java 自带的 `java.lang.reflect.Proxy`；
- 原理：生成一个实现同样接口的代理类，方法调用时转发到一个 **InvocationHandler**，在这里加入横切逻辑。

**特点**：

- 仅能代理接口；
- 动态创建，无需修改源码；
- 常用于 Spring 的默认代理方式（若目标类有接口）。

------

### 2. **CGLIB 动态代理（子类代理）**

- 要求：目标类**不需要实现接口**；
- 实现：通过字节码生成库（如 CGLIB），在运行时创建目标类的**子类**；
- 原理：重写目标类的方法，在方法前后插入横切逻辑。

**特点**：

- 可以代理普通类；
- 性能略高于 JDK 代理；
- 不支持 final 类和 final 方法（因为无法被继承/重写）。

------

## 三、Spring 中的 AOP 代理机制选择

Spring 会根据目标类是否实现接口自动选择代理机制：

| 目标类情况 | Spring 默认使用的代理方式 |
| ---------- | ------------------------- |
| 实现接口   | JDK 动态代理              |
| 无接口     | CGLIB 代理                |

你也可以通过配置 `proxyTargetClass=true` 强制使用 CGLIB。

------

## 四、代理机制工作流程简图（简化）

```
调用方法
   ↓
代理对象拦截（JDK/CGLIB）
   ↓
执行前置通知（如权限检查）
   ↓
执行目标方法
   ↓
执行后置通知（如日志、异常处理）
   ↓
返回结果
```
