`LinkedHashSet` 是 Java 集合框架中的一个 Set 实现类，它结合了 **`HashSet` 的唯一性 + 链表的有序性**，是一个**有顺序的、无重复元素的集合**。

它是 `java.util` 包中的类，继承自 `HashSet`，底层依然使用 `HashMap` 来存储数据，但在内部维护了一个 **双向链表** 来记录插入顺序。

------

### 一、LinkedHashSet 的主要特点

| 特性               | 描述                                                         |
| ------------------ | ------------------------------------------------------------ |
| **不允许重复元素** | 与 HashSet 一样，元素唯一（基于 `equals()` 和 `hashCode()`） |
| **保持插入顺序**   | 元素迭代时，顺序与插入顺序一致                               |
| **允许 null 元素** | 最多只能有一个 null                                          |
| **非线程安全**     | 多线程环境需手动同步或用 `Collections.synchronizedSet()`     |

------

### 二、底层结构

- 继承自 `HashSet`，内部通过 `LinkedHashMap` 实现。
- 使用哈希表存储元素，同时通过链表维护元素插入顺序。
- 每个节点不仅有哈希表链地址，还有前后指针（双向链表）。

------

### 三、使用示例

```java
Set<String> set = new LinkedHashSet<>();
set.add("apple");
set.add("banana");
set.add("apple"); // 重复元素，不会插入
set.add("cherry");

System.out.println(set); 
// 输出：[apple, banana, cherry]，保持插入顺序
```

------

### 四、与 HashSet 的对比

| 特性       | `HashSet`      | `LinkedHashSet`            |
| ---------- | -------------- | -------------------------- |
| 是否有顺序 | 否（哈希顺序） | 是（插入顺序）             |
| 存储结构   | HashMap        | LinkedHashMap              |
| 内存开销   | 较小           | 稍大（因为多维护一个链表） |
| 适合场景   | 仅要求元素唯一 | 需要**有序且唯一**的集合   |

------

### 五、应用场景

- 去重且**保留顺序**（比如用户访问历史、输入顺序敏感的数据）
- 实现 LRU 缓存的基础结构（`LinkedHashMap` 更适合）
- 保证数据在集合中“看起来”和“加进去”的顺序一致
