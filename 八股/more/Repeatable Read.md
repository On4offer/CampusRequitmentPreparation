**`REPEATABLE READ`（可重复读）** 是 MySQL InnoDB 默认的事务隔离级别，主要特点是：

> 在一个事务中，对同一条记录多次读取，结果始终一致，即使有其他事务修改并提交了该记录。

------

### 一、核心特性

| 特性           | 是否允许                                                     |
| -------------- | ------------------------------------------------------------ |
| **脏读**       | 否                                                           |
| **不可重复读** | 否                                                           |
| **幻读**       | 是（SQL 标准允许）但 **InnoDB 实际中通过[间隙锁](more/间隙锁)防止了幻读** |
| **并发性能**   | 较好（高于 SERIALIZABLE）                                    |

------

### 二、行为说明

- 第一次查询时会生成一个数据**快照版本**；
- 后续在同一个事务中，**所有查询看到的都是这同一个版本**；
- 即使其他事务提交了更新，也**不会影响当前事务的读取结果**。

------

### 三、示例演示不可重复读被避免

```sql
-- 事务 A
START TRANSACTION;
SELECT * FROM accounts WHERE id = 1;  -- 第一次读到 balance = 100

-- 事务 B
START TRANSACTION;
UPDATE accounts SET balance = 200 WHERE id = 1;
COMMIT;

-- 事务 A
SELECT * FROM accounts WHERE id = 1;  -- 第二次仍读到 balance = 100
COMMIT;
```

→ 在 `REPEATABLE READ` 下，**事务 A 读到的结果一致**，不会被事务 B 的更新影响。

------

### 四、InnoDB 对“幻读”的处理

在 SQL 标准中，`REPEATABLE READ` 是**不防幻读**的，但 InnoDB 通过以下手段**实际防止了幻读**：

- 使用 **Next-Key Lock（记录锁 + 间隙锁）**
- 锁住了符合查询范围的记录以及“插入位置之间的间隙”
- 从而避免其他事务在这些位置插入“幻影”记录

------

### 五、设置方式（MySQL）

```sql
SET SESSION TRANSACTION ISOLATION LEVEL REPEATABLE READ;
```

MySQL InnoDB 默认就是这个级别。

------

### 六、与其他隔离级别对比

| 隔离级别            | 脏读 | 不可重复读 | 幻读                | 性能              |
| ------------------- | ---- | ---------- | ------------------- | ----------------- |
| READ UNCOMMITTED    | 是   | 是         | 是                  | 最快（最不安全）  |
| READ COMMITTED      | 否   | 是         | 是                  | 快（Oracle 默认） |
| **REPEATABLE READ** | 否   | 否         | **InnoDB 实际防止** | 默认值，较好      |
| SERIALIZABLE        | 否   | 否         | 否                  | 最慢，最安全      |

------

### 七、适用场景

- 需要较强一致性，但不要求完全串行执行；
- 要求同一事务内多次读结果一致（如账户余额查询）；
- MySQL 默认选择，适合大多数业务系统。

------

### 小结一句话：

> `REPEATABLE READ` 保障同一事务内多次读取结果一致，InnoDB 中还通过加锁机制避免了幻读，是性能和一致性之间的黄金平衡点。
