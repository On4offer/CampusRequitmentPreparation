在 Spring 中，**Bean 的销毁**是指在应用程序停止或 Spring 容器关闭时，Spring 如何处理 Bean 的生命周期结束，释放资源或执行一些清理工作。

Spring 提供了不同的方式来进行 **Bean 的销毁**，确保 Bean 在不再使用时能够优雅地清理资源，例如关闭数据库连接、关闭文件流等。

------

## 一、Bean 销毁的主要方式

Spring 提供了三种主要的 Bean 销毁机制：

1. **通过 `@PreDestroy` 注解进行销毁**
2. **通过 `DisposableBean` 接口进行销毁**
3. **通过自定义销毁方法（XML 配置）**

------

## 二、销毁方法详解

### 1. **通过 `@PreDestroy` 注解进行销毁**

Spring 在 Bean 销毁前会调用被 `@PreDestroy` 注解标注的方法。这种方法适合基于注解的 Spring 配置。

#### 示例代码：

```java
@Component
public class MyService {

    @PreDestroy
    public void cleanup() {
        System.out.println("MyService 销毁前的清理工作！");
    }
}
```

- **使用场景**：适用于基于注解的 Spring 配置，简化了销毁方法的注册过程。
- **Spring 调用**：在容器关闭或 `@Scope("prototype")` Bean 被销毁时，`@PreDestroy` 注解标记的方法会被自动调用。

------

### 2. **通过 `DisposableBean` 接口进行销毁**

Spring 提供了 `DisposableBean` 接口，可以实现其中的 `destroy()` 方法，来进行 Bean 销毁时的清理工作。

#### 示例代码：

```java
@Component
public class MyService implements DisposableBean {

    @Override
    public void destroy() throws Exception {
        System.out.println("MyService 销毁时的清理工作！");
    }
}
```

- **使用场景**：适用于实现接口的方式，能够在销毁时执行额外的资源释放操作。
- **Spring 调用**：Spring 容器在销毁 Bean 时会调用 `destroy()` 方法。

------

### 3. **通过自定义销毁方法（XML 配置）**

在 XML 配置中，你可以为每个 Bean 指定一个销毁方法，这样 Spring 会在销毁 Bean 时调用该方法。

#### 示例 XML 配置：

```xml
<bean id="myService" class="com.example.MyService" destroy-method="cleanup" />
```

#### Java 类中的自定义销毁方法：

```java
public class MyService {

    public void cleanup() {
        System.out.println("MyService 销毁时的清理工作！");
    }
}
```

- **使用场景**：适用于 XML 配置方式的项目，能够灵活指定销毁方法。
- **Spring 调用**：在容器关闭或 `@Scope("prototype")` Bean 被销毁时，`cleanup()` 方法会被调用。

------

## 三、Bean 的销毁时机

1. **单例 Bean 销毁**：Spring 容器关闭时，所有的单例 Bean 会依次销毁。
2. **原型 Bean 销毁**：每次从容器中获取时都会创建新的实例，因此原型 Bean 只有在 **不再有引用时**才会被销毁。Spring 不会自动管理原型 Bean 的销毁过程，若需要手动销毁原型 Bean，通常可以通过 `ObjectFactory` 或 `ApplicationContext` 来管理其生命周期。

------

## 四、销毁过程的生命周期示意图

1. **单例 Bean 销毁过程**：
   - **容器关闭时**：Spring 会检查所有单例 Bean，调用它们的销毁方法。
2. **原型 Bean 销毁过程**：
   - **每个实例化的原型 Bean**：每次从容器获取时创建一个新实例，销毁通常由外部代码（如 `ApplicationContext.close()`）负责。

------

## 五、总结

| 特性         | 方法/机制                     | 适用场景                             |
| ------------ | ----------------------------- | ------------------------------------ |
| 基于注解销毁 | `@PreDestroy` 注解            | 简单场景，依赖注解方式               |
| 接口销毁     | `DisposableBean.destroy()`    | 依赖接口实现，可以在类中直接清理资源 |
| XML 配置销毁 | `destroy-method` 配置销毁方法 | XML 配置的项目                       |

------
