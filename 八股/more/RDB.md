## ✅ 二、什么是 RDB？

**RDB（Redis Database file）** 是 Redis 提供的一种**快照式（Snapshotting）持久化机制**，它会在指定时间点将内存中的所有数据**一次性写入到磁盘**，生成 `.rdb` 文件。

> 通俗地说，RDB 就是将 Redis 内存数据“打快照”保存下来，以便在 Redis 重启或宕机后恢复数据。

------

## ✅ 三、RDB 的触发机制（如何生成 RDB 文件）

### ✅ 1. **自动触发（配置文件）**

在 `redis.conf` 中配置 `save` 规则：

```ini
save 900 1    # 900 秒内有 1 个键被修改
save 300 10   # 300 秒内有 10 个键被修改
save 60 10000 # 60 秒内有 10000 个键被修改
```

### ✅ 2. **手动触发**

| 命令     | 说明                                 |
| -------- | ------------------------------------ |
| `SAVE`   | 阻塞 Redis，立即执行快照             |
| `BGSAVE` | 后台 fork 子进程异步生成快照（推荐） |

### ✅ 3. **其他触发方式**

- Redis 关闭时（配置 `stop-writes-on-bgsave-error yes`）
- 执行 `FLUSHALL`、`FLUSHDB` 等清除命令之前

------

## ✅ 四、RDB 文件存储路径和名称

- 文件名：`dump.rdb`
- 默认路径：`dir` 配置指定目录（例如 `/var/lib/redis`）

## ✅ 五、RDB 的优缺点

| 优点                   | 缺点                                   |
| ---------------------- | -------------------------------------- |
| 占用空间小，写入快     | 无法实时持久化，可能丢失最近写入的数据 |
| 支持**全量备份与恢复** | 依赖 fork 子进程，内存大时影响性能     |
| 适合灾难恢复、数据迁移 | 备份间隔越大，风险越高                 |
| 启动恢复速度快         | 每次保存全量数据，可能造成磁盘写入峰值 |

------

## ✅ 六、实战使用场景

### 🧩 场景 1：Redis 数据迁移或备份

> 定期复制 `dump.rdb` 到异地服务器或对象存储，作为**快照备份文件**，防止主实例丢失。

### 🧩 场景 2：内存数据库容灾恢复

> 系统崩溃后可通过 `dump.rdb` 文件恢复 Redis 数据到最新快照点

### 🧩 场景 3：生产环境开启双持久化（RDB + AOF）

> 开启 `appendonly yes` 和 `save`，提高可靠性

------

## ✅ 七、与 AOF 的对比（面试高频）

| 对比项                         | RDB                 | AOF                             |
| ------------------------------ | ------------------- | ------------------------------- |
| 持久化方式                     | 快照                | 命令追加                        |
| 文件大小                       | 小                  | 大（重复命令多）                |
| 数据安全性                     | 丢失最近一段数据    | 可配置为不丢失任何数据          |
| 启动恢复速度                   | 快                  | 慢（需要回放命令）              |
| 是否支持混合持久化（混合模式） | 支持（与 AOF 结合） | 支持（AOF 重写时可用 RDB 快照） |

------

## ✅ 八、最佳实践建议

- **线上建议开启 RDB + AOF 双持久化**
- 设置合适的 `save` 策略避免频繁快照导致性能波动
- 使用 `BGSAVE` 而非 `SAVE`，避免主线程阻塞
- 定期备份 `dump.rdb` 到外部存储
- 监控 Redis `rdb_last_save_time` 与 `rdb_changes_since_last_save` 等指标
