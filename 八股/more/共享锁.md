## ✅ 什么是共享锁（Shared Lock）

**共享锁（S 锁）** 是一种 **允许多个事务同时读取一条记录，但不允许修改** 的锁。

> 它确保被锁定的数据在读取期间不会被其他事务修改，从而保证“读的一致性”。

------

## 📚 举个例子理解：

假设你有如下语句：

```sql
SELECT * FROM user WHERE id = 1 LOCK IN SHARE MODE;
```

这条 SQL 会对满足条件的行加上**共享锁**，直到当前事务结束：

- 其他事务也可以读这行（加 S 锁）；
- 但不能对这行做 `UPDATE`、`DELETE` 等写操作（需要 X 锁）。

------

## 🔧 特性总结

| 特性         | 描述                                          |
| ------------ | --------------------------------------------- |
| 兼容性       | 多个共享锁可以并存                            |
| 与排他锁冲突 | 与 X 锁冲突，不能同时存在                     |
| 作用范围     | 行级（记录级）                                |
| 加锁方式     | 通过 `SELECT ... LOCK IN SHARE MODE` 显式加锁 |
| 持续时间     | 锁一直持有到当前事务 `COMMIT` 或 `ROLLBACK`   |

------

## 🧠 S 锁的作用

- 保证 **读一致性**；
- 实现 **可重复读**（Repeatable Read）；
- 在一些并发控制中防止“读-写冲突”；
- 是**MVCC（多版本并发控制）**下可选的锁策略。

------

## 🔐 与其他锁的兼容性（简表）

| 当前持有锁 | 加 S 锁 | 加 X 锁 |
| ---------- | ------- | ------- |
| 无锁       | ✅       | ✅       |
| S 锁       | ✅       | ❌       |
| X 锁       | ❌       | ❌       |

------

## 🛠️ 使用示例（MySQL 中显式加锁）

```sql
START TRANSACTION;

SELECT * FROM orders WHERE id = 100 LOCK IN SHARE MODE;

-- 其他事务现在不能更新这条记录

COMMIT;
```

------

## ⚠️ 注意事项

- 在默认的 MySQL `InnoDB` 引擎中，普通 `SELECT` 语句不会加 S 锁，而是用 **[MVCC 快照读](MVCC 快照读)** 实现并发；
- 只有明确使用 `LOCK IN SHARE MODE` 才会加共享锁；
- 在高并发写场景中使用共享锁可能会导致**[写阻塞](写阻塞)**，影响性能。

------

## 💬 面试回答模板

> 共享锁是 InnoDB 中的一种行级锁，允许多个事务同时读取同一条记录，但不允许修改。当使用 `SELECT ... LOCK IN SHARE MODE` 读取数据时，会加共享锁，防止其他事务对该行加排他锁进行修改。共享锁适用于对一致性要求较高的读操作，是事务隔离级别实现的重要手段。

------

## ✅ 总结一句话：

> 共享锁允许“多人同时读”，但不能“有人在写”，是数据库读一致性的重要保障机制。
