### **JVM 锁优化机制简介**

**背景**：在 Java 虚拟机（JVM）中，`synchronized` 是最基础的线程同步机制。为了提升它在不同场景下的性能，JVM 针对对象锁设计了一整套**“锁优化策略”**，主要包括**偏向锁、轻量级锁、重量级锁**，这些优化是 **在运行时由 JVM 自动完成的**。

------

### **一、JVM 锁的四种状态（锁膨胀过程）**

Java 对象在运行时，其锁状态可能经历以下四个阶段：

1. **无锁（No Lock）**：对象未被任何线程竞争；
2. **偏向锁（Biased Lock）**：只有一个线程使用，且长期未竞争；
3. **轻量级锁（Lightweight Lock）**：多个线程交替使用，产生少量竞争；
4. **重量级锁（Heavyweight Lock）**：多个线程同时竞争，产生阻塞。

> 这些状态是**从上往下“逐级膨胀”**，性能逐渐下降，**一旦升级就不会降级**。

------

### **二、锁优化的原理与策略**

#### 1. **偏向锁（Biased Lock）**

- **核心思想**：如果一个对象被同一个线程多次加锁，没有其他线程竞争，那么锁就偏向这个线程，不再做任何同步操作。
- **优势**：没有真正的加锁操作，性能非常高（接近无锁操作）。
- **启用时机**：JVM 启动后延迟几秒开启（通过参数控制）。
- **撤销时机**：当其他线程尝试访问该对象时，会撤销偏向状态，升级为轻量级锁。

#### 偏向锁启用参数（默认开启）：

```bash
-XX:+UseBiasedLocking
```

------

#### 2. **轻量级锁（Lightweight Lock）**

- **核心思想**：线程尝试获取锁时，如果没有竞争，就使用 **CAS（Compare And Swap）** 操作将对象头中的 Mark Word 替换为锁记录，避免了重量级锁的阻塞机制。
- **优势**：在无竞争或短暂竞争的情况下，性能优于重量级锁。
- **撤销时机**：如果有多个线程同时访问锁对象，无法通过 CAS 获得锁，升级为重量级锁。

------

#### 3. **重量级锁（Heavyweight Lock）**

- **核心思想**：使用操作系统底层的 Monitor（互斥量）机制实现锁，线程竞争激烈时使用。
- **代价高**：涉及用户态和内核态切换，线程会被**阻塞和唤醒**，性能较差。
- **应用场景**：线程大量并发访问同一个锁资源。

------

### **三、锁状态的转换流程**

```text
无锁
 ↓
偏向锁（线程1访问）
 ↓（撤销偏向）
轻量级锁（线程1和线程2交替）
 ↓（竞争加剧）
重量级锁（线程1、2并发访问）
```

------

### **四、锁优化相关技术**

1. **锁消除**（由 JIT 编译器完成）

   - 如果编译器发现锁对象不会被多个线程访问，就会**自动去掉锁**，提升性能。

   - 示例：

     ```java
     public void noSync() {
         StringBuilder sb = new StringBuilder(); // 只在当前线程用，不加锁
         sb.append("A").append("B");
     }
     ```

2. **锁粗化**

   - 如果一段代码中连续对同一个对象加锁/释放，JVM 会**合并成一次加锁，减少开销**。

   - 示例：

     ```java
     for (int i = 0; i < 100; i++) {
         synchronized (obj) {
             // do something
         }
     }
     ```

     → JVM 会将整个循环加锁一次，而非每次循环都加锁。

3. **自旋锁**

   - 在轻量级锁中，如果线程竞争失败，不立即阻塞，而是在一定时间内**循环尝试获取锁**。
   - 减少线程上下文切换，提高性能。
   - 自旋失败后才会升级为重量级锁。

------

### **五、相关 JVM 参数（可调优）**

| 参数                                 | 说明                     |
| ------------------------------------ | ------------------------ |
| `-XX:+UseBiasedLocking`              | 启用偏向锁（默认开启）   |
| `-XX:BiasedLockingStartupDelay=0`    | JVM 启动后立即启用偏向锁 |
| `-XX:+PrintGCApplicationStoppedTime` | 查看 GC/锁停顿时间       |

------

### **六、小结一句话**

> **JVM 通过偏向锁、轻量级锁、自旋锁等机制动态优化锁性能，在并发不严重时避免阻塞，极大提升了 `synchronized` 的效率，是 Java 并发性能的基础保障。**
