# 题目一：基于简历7的模拟真实面试题

> **简历依据**：侯自然 - 长安大学硕士(计算机技术)、智识神工 Java 开发实习(医美管理系统)、个人项目小众点评  
> **场景设定**：面试官手头只有你的简历，会按简历逐条追问，并结合项目延伸场景题与原理题。  
> **建议**：先自答一遍，再对照参考答案查漏补缺。

---

## 一、开场与项目选择（约 2 分钟）

**面试官**：先简单做个自我介绍，然后选一个你最有把握的项目，我们从这个项目往下聊。

**考察点**：表达清晰、项目选择是否与岗位匹配、能否自然引出技术点。

---

## 二、实习经历深挖（医美行业管理系统）

*面试官会盯着简历上的「核心功能开发与产出」逐条问。*

### 2.1 Excel 商品批量导入

1. **简历写的是「模糊匹配 + 精确匹配两步策略」，具体是怎么做的？**  
   - 模糊匹配用的是什么条件（商品名？编码？）？精确匹配又比对了哪些字段？
2. **如果用户上传的 Excel 有 5 万行，直接一行行查库会有什么问题？你怎么优化的？**
3. **Apache POI 解析 .xls 和 .xlsx 在你代码里有没有做区分？大文件解析时有没有考虑过内存占用？**

### 2.2 密码过期强制修改

4. **「3 个月过期校验」是存在哪儿的？数据库字段还是 Redis？**
5. **登录拦截器里具体怎么判断「该强制改密码」？未修改时是直接跳转改密页还是返回错误码？**
6. **密码修改接口除了校验旧密码，还做了哪些安全措施？**

### 2.3 审计日志

7. **「敏感信息脱敏」脱敏的是哪些字段？用的替换规则还是加密？**
8. **你说异步存日志，用的是 @Async、线程池还是消息队列？为什么选这种？**
9. **如果异步写日志时抛异常，日志丢了怎么办？有没有补偿或降级？**

### 2.4 入库单标识 & Bug 修复

10. **入库单标识的库表是你设计的吗？主要有哪些字段？为什么这样设计？**
11. **移动端业绩报表数据筛选出问题时，你是怎么定位的？最后根因是什么？**
12. **入库模块「完善事务管理和数据校验」具体加了哪些校验？事务传播行为用的是哪种？**

### 2.5 项目整体

13. **系统要适配原有 ASP.NET，你们和旧系统是怎么对接的？接口协议、鉴权方式有没有踩坑？**
14. **总部和门店双场景，数据是怎么同步的？有没有考虑过门店断网、延迟的情况？**

## 三、个人项目深挖（小众点评）

*简历里 Redis、Redisson、RocketMQ、Feed 流都会成为追问重点。*

### 3.1 缓存与三大问题

15. **Cache-Aside 在你项目里具体是怎么用的？读和写的步骤分别说一下。**
16. **「逻辑过期解决缓存击穿」—— 逻辑过期和物理过期有什么区别？你的逻辑过期是怎么实现的？**
17. **「缓存空值解决穿透」—— 空值你们 TTL 设多久？为什么？**
18. **「动态 TTL 解决雪崩」—— 动态 TTL 具体怎么算的？比如在什么范围内随机？**

### 3.2 Token 与用户上下文

19. **Redis 做 Token 校验的 key 是怎么设计的？value 存什么？**
20. **「滑动过期」具体逻辑是什么？每次请求都更新过期时间吗？**
21. **用 ThreadLocal 存用户信息，请求结束后有没有主动 remove？为什么必须 remove？**

### 3.3 分布式锁与秒杀

22. **Redisson 锁的 key 是按用户维度来的，那同一个用户重复点两次下单，是怎么被挡住的？**
23. **除了分布式锁，有没有用数据库的乐观锁（比如版本号）？两者分别防的是什么？**
24. **如果 Redis 主从切换，导致锁在从库还没同步就丢了，你们有考虑过吗？怎么规避？**

### 3.4 RocketMQ 与订单解耦

25. **下单和落库解耦：是「先发 MQ 再返回成功」还是「先落库再发 MQ」？选这种顺序的理由是什么？**
26. **消费者消费失败会怎样？有重试吗？会不会重复消费？你们怎么保证幂等？**
27. **你说 TPS 提升约 15 倍，是压测对比的哪两种方案？压测 QPS 大概多少？**

### 3.5 Feed 流

28. **「推模式」是指发内容时往每个粉丝的收件箱写一条吗？粉丝量很大的用户（比如 10 万）会怎么处理？**
29. **ZSet 的 score 用时间戳，那同一秒内多条内容怎么区分顺序？**
30. **「滚动分页」用的是什么命令？为什么不用 LIMIT offset？**

## 四、场景设计题（面试官临时出题）

*这些题不一定写在简历上，但会结合你的项目场景问。*

### 4.1 基于医美系统

31. **如果医美系统要做一个「双 11 秒杀某款套餐」功能，预计瞬时 1 万 QPS，你会从哪几方面设计？**（接口、缓存、限流、扣减、一致性等）
32. **业绩统计模块月末报表很慢，表数据百万级，你会从 SQL、索引、缓存、架构上怎么优化？**

### 4.2 基于小众点评

33. **如果小众点评要支持「百万用户每日签到领积分」，用 Redis 你会怎么设计？BitMap 的 key 怎么设计？**
34. **订单支付成功后要：改订单状态、扣库存、加积分，如果其中一步失败，你怎么保证一致性？**

### 4.3 通用

35. **Redis 集群宕机了，你的小众点评怎么降级？会直接打到 MySQL 吗？怎么避免把数据库打挂？**

## 五、技术原理题（简历技能栈）

*简历写了 Java、Spring、MySQL、Redis、Docker 等，都可能被抽问。*

### 5.4 Redis
