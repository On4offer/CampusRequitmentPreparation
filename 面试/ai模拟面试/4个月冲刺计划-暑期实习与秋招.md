# 4 个月冲刺计划：暑期实习 + 秋招

> 时间线：暑期实习投递已开启 → 7–8 月秋招开启（约 4 个月）  
> 目标：简历能讲透 + 基础稳 + 手撕能写 + 面试不慌

---

## 一、时间怎么切：两阶段并行

| 阶段 | 时间 | 重心 | 说明 |
|------|------|------|------|
| **冲刺暑期实习** | 当下 ~ 5 月底 | 简历深挖 + 高频基础 + 少量手撕 | 先拿到 offer，同时为秋招打底 |
| **巩固 + 秋招** | 6 月 ~ 秋招 | 补漏 + 算法加强 + 面经实战 | 用实习面试反馈补弱项，再冲秋招 |

**并行含义**：  
- **学**：按下面的「重点清单」系统过一遍，一次学到位，暑期和秋招共用。  
- **练**：暑期阶段「简历 + 基础」占比 70%，手撕 30%；秋招前一个月把算法提到 50%。  
- **面**：暑期面一场就复盘一场（哪类题卡了），立刻补那一块，秋招不再重复踩坑。

---

## 二、抓重点：什么必须稳（按优先级）

### 1. 简历项目（最高优先级）

- **原则**：简历上每一行都能展开讲 2 分钟，并能接住 2 层追问。
- **必做**：
  - 用《题目一-基于简历7的模拟面试题》把「实习经历 + 小众点评」全部自答一遍，卡壳的题记下来单独补。
  - 每个功能准备：**做了什么 → 为什么这么做 → 遇到什么问题 → 怎么解决的**（STAR 简版）。
- **时间**：每周至少 2 次「模拟自问自答」，每次 30–40 分钟，直到流畅。
- **效率**：不要泛泛背八股，只背「和简历项目直接相关」的原理（如 Cache-Aside、逻辑过期、Redisson 锁、RocketMQ 消费幂等）。

### 2. 基础八股（按简历技能栈抓）

- **优先级**：简历写了什么，就优先背什么；没写的可以后补。
- **建议顺序**（和你简历一致）：
  1. **Redis**：缓存三大问题（穿透/击穿/雪崩）、持久化、主从/哨兵、分布式锁注意点 → 直接对应小众点评。
  2. **MySQL**：事务、隔离级别、MVCC、索引、锁、慢 SQL 排查 → 实习和项目都会问。
  3. **Java**：集合（HashMap）、多线程（线程池、synchronized/Lock）、JVM 只抓「内存结构 + 常见 GC」即可。
  4. **Spring**：IOC/Bean 生命周期、循环依赖、@Transactional 失效场景 → 够应付大部分实习/秋招。
- **时间**：每天固定 30–45 分钟「小块专题」，用《第一套-查漏补缺学习文档》或自己的笔记，**只记要点 + 和项目的联系**。
- **效率**：每学一个点，立刻想「面试官会怎么问」「我能不能用小众点评/医美系统举例子」，能举例才算掌握。

### 3. 手撕 / 算法

- **暑期**：以「能写出来」为主，不追求题量。  
  - 必会：链表（反转、合并）、二叉树（遍历、层序）、二分、简单 DP（如爬楼梯）、双指针。  
  - 外加：**Redis 分布式锁伪代码**、**逻辑过期缓存伪代码**（和简历强相关，面试官爱问）。
- **秋招前 1 个月**：适当加量，LeetCode 热题 100 / 剑指 Offer 挑高频做，重点仍是「写对 + 说清思路」。
- **时间**：暑期每天 30 分钟或隔天 1 小时；秋招前可调到每天 1 小时。
- **效率**：同一道题「写 → 讲思路 → 复杂度」练一遍胜过盲目刷新题；把「手撕题」和「项目里的设计」挂钩（如：秒杀用到了限流/缓存，和哪道题思路像）。

### 4. 场景题 + 项目延伸

- **来源**：题目一里的「场景设计题」+ 网上面经里的「设计/优化类」。
- **方法**：不背答案，背**答题框架**：先问清楚需求（QPS、一致性要求）→ 再说接口/缓存/DB/消息队列/限流分别怎么设计 → 最后说边界（宕机、重复提交等）。
- **时间**：和简历模拟一起做，每周练 2–3 道，用录音或文字写下来，再让我或别人帮你分析。
- **效率**：每道场景题尽量往「小众点评/医美」上靠，用自己做过的技术栈回答，比泛泛说更加分。

## 三、提升效率：具体怎么做

### 1. 用「输出」倒逼「输入」

- 学完一个知识点：**自己用 1 分钟讲一遍**（假装对面是面试官），讲不顺就回去再看。
- 每周末：从题目一或面经里抽 5 道题，**不看书口头答**，答完再对答案补漏。

### 2. 面试驱动学习

- 投了哪家，就搜一下那家的面经，把「重复出现」的题标出来，优先准备。
- 每次面试后：**当天**把「被问倒的题」记下来，下一周专门补这一块，避免同一类题再挂。

### 3. 资料做减法

- **固定 1–2 份**为主：例如《第一套-查漏补缺》+ 你自己整理的「简历对应知识点」。
- 不要同时开很多教程/书，容易东一榔头西一棒子；一个专题吃透再开下一个。

### 4. 并行用起来

- **我出题你来做**：专门练「简历深挖」和「和简历相关的原理」，保证问到项目不虚。
- **你找面经我来分析**：练「真实问法 + 随机题」，你把答案发给我，我帮你改结构、补漏、往项目上引。
- 两种交替：比如一周 2 次「题目一自测」+ 1 次「面经答题我分析」，既稳简历又贴近实战。

## 四、按周的大致节奏（可微调）

| 周 | 暑期实习侧 | 秋招打底 |
|----|------------|----------|
| 1–2 | 题目一过完第一遍，标出弱项 | Redis + MySQL 小专题每天 30 分钟 |
| 3–4 | 弱项补完 + 简历自问自答 2 轮 | Java 集合/多线程 + Spring 小专题 |
| 5–6 | 面经练 3–5 份，答完发分析 | 手撕：链表/二叉树/二分/简单 DP |
| 7–8 | 暑期面试复盘，按公司补漏 | 场景题框架 + 和项目挂钩 |
| 之后 | — | 算法加量 + 继续面经 + 秋招投递 |

---

## 五、一句话总结

- **重点**：简历项目 > 简历上的基础八股 > 手撕能写 > 场景题有框架。  
- **效率**：输出倒逼输入、面试驱动补漏、资料做减法、题目一 + 面经分析并行。  
- **心态**：暑期能拿 offer 最好，拿不到也是「免费模拟考」+ 补漏，4 个月足够把同一套东西练到秋招稳。

需要的话可以按你当前「每天能学几小时、已投多少家」再细调一版周计划（比如只保留「必做项」的极简版）。
