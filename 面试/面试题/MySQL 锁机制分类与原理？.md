# 面试题：MySQL 的锁机制有哪些？行锁、表锁、间隙锁的区别？

------

## 1. 概念

- **锁（Lock）**：数据库用来解决并发控制问题的重要手段。
- **MySQL 常见锁分类**：
  1. **按操作粒度**：表锁、行锁、页锁（MySQL 主流是表锁和行锁）。
  2. **按读写类型**：共享锁（S）、排他锁（X）。
  3. **按实现层面**：乐观锁、悲观锁。
  4. **InnoDB 特有**：间隙锁（Gap Lock）、Next-Key Lock。

------

## 2. 锁机制区别

### 1）表锁（Table Lock）

- 锁住整张表。
- 并发度低，开销小。
- 典型于 **MyISAM 引擎**。

### 2）行锁（Row Lock）

- 仅锁住访问的行记录。
- 并发度高，开销大。
- **InnoDB 支持，基于索引实现**（无索引可能退化为表锁）。

### 3）间隙锁（Gap Lock）

- 锁定一个范围，但不包含记录本身。
- 用于防止 **幻读**（如在 `id BETWEEN 10 AND 20` 的范围中禁止插入）。
- 只有在 **可重复读（RR）隔离级别**下才会生效。

### 4）Next-Key Lock

- **行锁 + 间隙锁** 组合。
- 锁定行本身和其前后的间隙，彻底解决幻读问题。

------

## 3. 案例

```sql
-- 表 user(id int primary key, name varchar)
-- 假设表中已有 id=5, id=10 的数据

-- 事务 A：
SELECT * FROM user WHERE id=5 FOR UPDATE;

-- InnoDB 加的锁：
-- 行锁：锁住 id=5
-- 间隙锁：锁住 (5,10) 区间，禁止插入 id=6~9
```

👉 这样事务 B 即使尝试 `INSERT INTO user VALUES(7,'Tom')` 也会被阻塞，避免了幻读。

------

## 4. 使用场景

- **表锁**：DDL 操作、大批量更新/导入（MyISAM 常见）。
- **行锁**：OLTP 系统，电商下单、库存扣减。
- **间隙锁 / Next-Key Lock**：防止并发插入导致的幻读。

------

## 5. 面试模板回答

> MySQL 的锁机制包括表锁、行锁、间隙锁、Next-Key Lock 等。
>
> - 表锁锁住整张表，开销小但并发度低；
> - 行锁只锁住数据行，并发度高，但依赖索引；
> - 间隙锁锁定区间，避免幻读；
> - Next-Key Lock 是行锁+间隙锁的结合，用于彻底解决幻读。
>    一般情况下，InnoDB 主要依赖行锁和间隙锁来保证事务隔离性。
