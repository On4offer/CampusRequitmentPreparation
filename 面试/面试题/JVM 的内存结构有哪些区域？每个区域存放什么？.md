## 📌 面试题：JVM 的内存结构有哪些区域？每个区域存放什么？

### ✅ 回答思路

答题要点是 **分区 → 存放内容 → 线程共享 or 私有 → 特殊说明**，最后可以结合 **JDK8 之后的变化** 提升层次。

------

### ⭐ 标准回答

JVM 在运行时会把内存划分为不同区域，每个区域有不同的职责，大体分为以下几块：

1. **程序计数器（Program Counter Register）**
   - **线程私有**
   - 记录当前线程正在执行的字节码指令地址（行号指示器）。
   - 如果执行的是本地方法（`native`），计数器值为 `undefined`。
   - **作用**：保证多线程切换后能恢复到正确的执行位置。
2. **虚拟机栈（Java Virtual Machine Stack）**
   - **线程私有**
   - 每个方法调用对应一个 **栈帧**，包含：
     - 局部变量表（基本类型、对象引用、返回地址）
     - 操作数栈
     - 动态链接（指向常量池的方法引用）
     - 方法出口
   - 常见错误：`StackOverflowError`（栈深度过大）
3. **本地方法栈（Native Method Stack）**
   - **线程私有**
   - 与虚拟机栈类似，但服务于 `native` 方法。
4. **堆（Heap）**
   - **线程共享**
   - 存放对象实例，是垃圾回收的主要区域。
   - JDK8 默认使用 **分代模型**：
     - 新生代（Eden + Survivor）
     - 老年代
   - 常见错误：`OutOfMemoryError: Java heap space`
5. **方法区（Method Area，JDK8 之后为 Metaspace）**
   - **线程共享**
   - 存放类的元数据信息：类结构、方法信息、运行时常量池、静态变量等。
   - JDK7 之前在 JVM 内存里，叫做 **永久代 PermGen**；
   - JDK8 之后移到 **本地内存 Metaspace**，更灵活，减少 OOM。
6. **运行时常量池（Runtime Constant Pool）**
   - 属于方法区的一部分。
   - 存放编译期生成的字面量、符号引用，也支持运行时动态生成（如 `String.intern()`）。
7. **直接内存（Direct Memory）**
   - 不在 JVM 管理范围内，由操作系统分配。
   - 常用于 NIO 中的缓冲区，避免 Java 堆和本地堆之间的拷贝。
   - 错误：`OutOfMemoryError: Direct buffer memory`。
