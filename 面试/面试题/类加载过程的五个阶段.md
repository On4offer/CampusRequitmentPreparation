## 📌 面试题：类加载过程的五个阶段

### ⭐ 标准回答

JVM 中类的生命周期大体分为 **加载 → 连接（验证、准备、解析）→ 初始化 → 使用 → 卸载**，其中类加载过程的 **五个核心阶段**是：

------

### 1. **加载（Loading）**

- **做的事**：
  1. 通过类的全限定名（包名 + 类名）获取字节码文件。
  2. 将字节码文件（可能来自文件系统、JAR 包、网络等）读入内存。
  3. 在内存中生成一个代表该类的 **Class 对象**。
- **特点**：
  - 由类加载器（ClassLoader）完成。
  - 双亲委派模型在这里起作用。

------

### 2. **验证（Verification）**

- **做的事**：保证字节码文件符合 JVM 规范，不会破坏 JVM 的安全性。
- **验证内容**：
  - 文件格式验证（魔数、版本号）。
  - 元数据验证（类继承是否合法）。
  - 字节码验证（指令是否合法、栈深度正确）。
  - 符号引用验证（是否能正确解析）。

------

### 3. **准备（Preparation）**

- **做的事**：为类变量（即静态变量）分配内存并设置 **零值**（默认值），不执行任何用户逻辑。
- **注意**：
  - 只为 **static 变量** 分配空间。
  - `final static` 常量在编译期就会分配到常量池，而不是这里。

------

### 4. **解析（Resolution）**

- **做的事**：将常量池中的符号引用（如类名、方法名、字段名）替换为 **直接引用**（内存地址或偏移量）。
- **例子**：`com.demo.User` → 内存中对应的 Class 对象引用。

------

### 5. **初始化（Initialization）**

- **做的事**：执行类构造器 `<clinit>()` 方法，按顺序初始化 **静态变量** 和 **静态代码块**。
- **触发时机**：
  - 主动使用类时（如 new 对象、调用静态方法、反射）。
  - 被动引用不会触发（如访问 `final static` 常量）。

------

### 📊 流程图总结

```
加载 → 连接(验证 → 准备 → 解析) → 初始化 → 使用 → 卸载
```

------

### 🔍 面试追问

1. **为什么要分验证、准备、解析？**
    👉 提高 JVM 安全性，保证类结构正确，避免恶意字节码。
2. **什么时候会触发类的初始化？**
    👉 主动使用时，如调用静态方法、new 对象。
3. **静态变量在准备阶段和初始化阶段的区别？**
    👉 准备阶段赋零值，初始化阶段赋用户定义值。
