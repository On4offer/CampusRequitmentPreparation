# 面试题：Docker 镜像和容器的区别

### 一、概念解释

- **Docker 镜像（Image）**
   镜像是一个 **只读的模板**，包含运行应用所需的环境和依赖，比如文件系统、库、配置、代码。它相当于容器的“蓝图”。
- **Docker 容器（Container）**
   容器是镜像的 **运行时实例**，在镜像的基础上加上读写层，可以运行、停止、删除，相互之间是隔离的。

------

### 二、原理剖析

1. **镜像本质**
   - 分层存储：基于 **UnionFS**（联合文件系统）实现分层，每层只读，可以复用，节省存储空间。
   - 构建方式：通过 **Dockerfile** 构建，每条指令生成一个只读层。
2. **容器本质**
   - 镜像运行时会在最上层加一个 **可写层**（容器层），应用的运行状态和数据写入其中。
   - 容器层删除后，数据也会丢失（除非挂载 Volume）。

------

### 三、对比分析

| 对比维度     | Docker 镜像              | Docker 容器                      |
| ------------ | ------------------------ | -------------------------------- |
| **定义**     | 应用和依赖的静态只读模板 | 镜像的运行时实例                 |
| **状态**     | 静态，不会改变           | 动态，可运行、停止、删除         |
| **存储**     | 分层只读文件系统         | 镜像层 + 容器可写层              |
| **数量关系** | 一个镜像可创建多个容器   | 多个容器可基于同一镜像           |
| **持久化**   | 不会变化，版本可控       | 数据随容器销毁而丢失（需挂载卷） |

------

### 四、案例说明

- **镜像**：
   比如 `openjdk:17-jdk` 镜像，里面包含了 Java 环境。
- **容器**：
   当我们执行 `docker run openjdk:17-jdk`，启动后就是一个容器，里面可以运行 SpringBoot 项目。

在 **苍穹外卖** 项目中：

- 打包成 `waimai:v1` 镜像（只读蓝图）。
- 在测试环境、生产环境分别运行多个 `waimai:v1` 容器实例，互不干扰。

------

### 五、使用场景

- **镜像**：
  - 用于交付与分发，比如上传到 DockerHub、公司私有仓库。
  - 用于版本管理（v1、v2）。
- **容器**：
  - 用于实际运行应用，支持快速启动和销毁。
  - 结合编排工具（K8s、Compose）批量部署。

------

### 六、面试标准回答（简洁版）

> 镜像是静态的只读模板，相当于程序的“安装包”；容器是镜像运行后的实例，相当于“正在运行的进程”。镜像可以创建多个容器，容器则依赖镜像运行。
