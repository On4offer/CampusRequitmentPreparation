# 📌 面试题：什么时候用 CopyOnWriteArrayList？

## 一、核心回答

1. **定义**
   - `CopyOnWriteArrayList` 是一种 **线程安全的 List**，底层基于 **写时复制** 思想。
   - 读操作不加锁，写操作（add/remove）时会 **复制一份新数组**，然后在新数组上修改，最后替换旧数组。
2. **使用场景**
   - **读多写少** 的场景：
     - 读操作无锁，性能高。
     - 写操作成本高（复制整个数组），所以写频繁时不适合。
3. **典型应用**
   - 读操作远远多于写操作，比如：
     - 系统中的 **黑名单/白名单**（偶尔更新，但经常查询）。
     - **在线用户列表**（读多写少）。
     - **配置类数据**（如系统常量、权限规则等）。

------

## 二、扩展回答（可以展开说的点）

- **优点**
  - 读操作无锁，线程安全。
  - 迭代时不会抛出 `ConcurrentModificationException`（因为迭代的是快照）。
- **缺点**
  - 写操作开销大（复制数组 O(n)）。
  - 不能保证读到的是最新数据（存在弱一致性）。

------

## 三、项目场景举例

- 在 **黑马点评项目** 中，如果维护一个 **城市白名单列表**（只在节假日更新，但用户访问时频繁查询），用 `CopyOnWriteArrayList` 比加锁的 `ArrayList` 更高效。
- 在 **苍穹外卖项目** 中，如果维护一个 **活动菜品 ID 列表**（写少读多，前台不断查询），适合用 `CopyOnWriteArrayList`。

------

## 四、常见追问

1. **CopyOnWriteArrayList 为什么适合读多写少？**
   - 因为写时会复制整个数组，开销大，但读无锁性能很好。
2. **CopyOnWriteArrayList 的迭代器是 fail-fast 还是 fail-safe？**
   - 是 **fail-safe**，因为迭代基于快照，不会抛异常。
3. **和 Vector 的区别？**
   - Vector 通过 synchronized 实现线程安全，读写都有锁；CopyOnWriteArrayList 读无锁，写时复制，性能更优。

------

✅ **一分钟口述版总结**
 CopyOnWriteArrayList 是线程安全的 List，采用写时复制机制。它非常适合 **读多写少** 的场景，比如黑名单、配置表、在线用户列表。读操作无锁、性能高，但写操作开销大，不适合频繁修改的场景。
