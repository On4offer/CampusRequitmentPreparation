## 面试题：Linux 权限管理的原理是什么？

### 1. 概念

Linux 是一个 **多用户、多任务** 的操作系统，为了保证系统安全和资源合理使用，引入了 **权限管理机制**。权限控制的核心是：

- **谁**（用户/用户组）
- **对什么**（文件/目录/设备）
- **能做什么**（读/写/执行）

------

### 2. 权限模型原理

#### 📌 基本权限（rwx）

- `r (read)`：读取权限，允许查看文件内容/列出目录内容
- `w (write)`：写入权限，允许修改文件/目录
- `x (execute)`：执行权限，允许执行文件/进入目录

#### 📌 三类主体

- **Owner（属主）**：文件创建者，通常对文件有最大权限
- **Group（属组）**：与文件属组相同的用户，共享同一组权限
- **Others（其他用户）**：不属于前两类的所有其他用户

#### 📌 权限表示方式

1. **字符形式**：如 `-rwxr-xr--`
   - 第 1 位：文件类型（`-` 普通文件，`d` 目录，`l` 链接）
   - 后 9 位：每 3 位一组，分别对应 **用户/组/其他人** 的 rwx 权限
2. **数字形式**：八进制表示
   - `r=4, w=2, x=1`
   - 例如：`chmod 755 file` → `rwxr-xr-x`

#### 📌 特殊权限

- **SUID**：用户执行文件时临时获取文件所有者权限（如 `passwd`）
- **SGID**：目录中文件继承组权限
- **Sticky Bit**：目录下文件只能由自己删除（如 `/tmp`）

------

### 3. 常见命令

- `ls -l`：查看权限
- `chmod`：修改权限（字符法/数字法）
- `chown`：修改文件的所有者
- `chgrp`：修改文件的属组
- `umask`：新建文件的默认权限掩码

------

### 4. 案例演示

```bash
# 查看权限
ls -l test.sh
# -rwxr-xr--  1 user group 1234 Sep 30 10:00 test.sh

# 给所有用户添加执行权限
chmod a+x test.sh

# 修改属主和属组
chown root:root test.sh

# 设置 SUID
chmod u+s test.sh
```

------

### 5. 使用场景

- **项目协作**：开发人员所在组共享目录，用 SGID 保持一致属组。
- **系统安全**：限制 `/etc/passwd` 文件写权限，防止被篡改。
- **公共目录**：`/tmp` 使用 Sticky Bit，避免用户互删文件。
- **Web 服务**：限制 Nginx/MySQL 日志目录权限，避免越权访问。

------

### 6. 标准回答（面试简洁版）

> Linux 的权限管理基于 **用户（owner）-组（group）-其他人（others）** 三类主体和 **rwx（读、写、执行）** 权限。
>  文件权限既可以用字符表示（如 `-rwxr-xr--`），也可以用数字表示（如 `755`）。
>  修改权限常用 `chmod`、`chown`、`chgrp`，特殊权限包括 **SUID、SGID、Sticky Bit**，分别用于提升执行权限、继承组权限、保护共享目录。
>  这种机制保证了多用户环境下的安全与资源隔离。
