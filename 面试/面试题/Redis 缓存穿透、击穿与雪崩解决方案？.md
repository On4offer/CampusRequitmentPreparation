# 面试题

**什么是缓存穿透、击穿和雪崩？分别如何解决？**

------

## 1️⃣ 概念

- **缓存穿透**：查询一个 **不存在的数据**，缓存和数据库都没有，每次请求都打到数据库，可能被恶意利用造成数据库压力。
- **缓存击穿**：某个 **热点 Key 恰好过期**，瞬间有大量并发请求同时查询，直接打到数据库，造成数据库压力。
- **缓存雪崩**：大量 **Key 在同一时间过期或 Redis 故障**，导致请求瞬间全部打到数据库，可能压垮系统。

------

## 2️⃣ 原理剖析

- **穿透**：缓存里没有 + 数据库里没有，缓存形同虚设。
- **击穿**：某个超级热点 Key 过期，短时间内千万级请求涌向数据库。
- **雪崩**：缓存层大规模失效，所有请求回源，形成“雪崩效应”。

------

## 3️⃣ 解决方案

### 缓存穿透

- **布隆过滤器（BloomFilter）**：提前判断 Key 是否可能存在，不存在直接拦截。
- **缓存空值**：查询不到的数据写一个空对象到缓存，并设置较短 TTL。
- **参数校验**：前置过滤非法 ID。

### 缓存击穿

- **互斥锁（Mutex / SingleFlight）**：只有一个线程去重建缓存，其他线程等待或返回旧值。
- **逻辑过期**：缓存存逻辑过期时间，过期后仍返回旧值，同时后台线程异步刷新。
- **热点数据永不过期**：由后台定时任务异步刷新。

### 缓存雪崩

- **过期时间随机化**：TTL 加随机抖动，避免同时过期。
- **热点数据预热**：系统启动/重启时提前写入缓存。
- **多级缓存**：JVM 本地缓存（如 Caffeine）+ Redis 二级缓存。
- **限流/降级**：数据库 QPS 超阈值时，部分请求直接返回兜底数据。

------

## 4️⃣ 项目实践（黑马点评 / 苍穹外卖）

- **黑马点评**：
  - 店铺缓存：逻辑过期方案解决击穿问题（只允许一个线程构建缓存）。
  - 店铺不存在时：缓存空值防止穿透。
  - 系统启动：预热热点店铺数据，避免雪崩。
- **苍穹外卖**：
  - 秒杀库存：使用 Redis + 互斥锁，避免击穿。
  - 大量菜品缓存：过期时间加随机值，防止雪崩。

------

## 5️⃣ 面试标准回答模板

> 缓存异常主要分为三类：
>
> - **穿透**：查询不存在的数据，解决方案是布隆过滤器、缓存空值；
> - **击穿**：热点 Key 过期瞬间高并发涌入，解决方案是互斥锁、逻辑过期、热点 Key 永不过期；
> - **雪崩**：大量 Key 同时过期或 Redis 故障，解决方案是随机过期时间、缓存预热、多级缓存、限流降级。
>    在项目中我用逻辑过期方案解决黑马点评店铺的缓存击穿问题，用缓存空值防止缓存穿透。
