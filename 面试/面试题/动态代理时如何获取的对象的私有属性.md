动态代理中获取目标对象私有属性的核心是**依托 Java 反射机制**（突破访问权限限制），结合动态代理的核心逻辑（获取目标对象引用→反射操作属性）实现。以下从「原理 + 步骤 + 代码示例 + 注意事项」展开，贴合面试答题逻辑：

### 一、核心原理

动态代理的核心是生成代理类并持有**目标对象（被代理对象）** 的引用，而 Java 反射的`Field`类提供了`setAccessible(true)`方法，可突破访问修饰符限制（包括`private`），从而读取 / 修改私有属性。

### 二、具体实现步骤

无论 JDK 动态代理还是 CGLIB 动态代理，获取私有属性的流程一致，核心步骤：

1. 在代理逻辑中获取目标对象引用：
   - JDK 动态代理：通过`InvocationHandler`的`invoke`方法参数，或代理类持有的目标对象成员变量获取；
   - CGLIB 动态代理：通过`MethodInterceptor`的`intercept`方法参数，或增强器持有的目标对象获取。
2. 通过反射获取目标类的私有 Field：
   - 用`Class.getDeclaredField(String name)`（获取当前类声明的所有字段，包括私有），而非`getField`（仅获取 public 字段）；
   - 若私有属性在父类，需递归获取父类的`Class`对象，再调用`getDeclaredField`。
3. **突破访问权限**：调用`Field.setAccessible(true)`，关闭 JVM 的访问检查。
4. 读取 / 修改私有属性值：
   - 读取：`field.get(Object targetObj)`；
   - 修改：`field.set(Object targetObj, Object value)`。

### 三、代码示例（分 JDK/CGLIB 两种代理）

#### 示例 1：JDK 动态代理获取私有属性

```java
// 目标接口
public interface UserService {
    void doSomething();
}

// 目标类（含私有属性）
public class UserServiceImpl implements UserService {
    private String privateField = "我是私有属性"; // 私有属性
    private Integer privateNum = 100;

    @Override
    public void doSomething() {
        System.out.println("目标方法执行");
    }
}

// JDK动态代理实现
public class JdkProxyHandler implements InvocationHandler {
    private final Object target; // 持有目标对象引用

    public JdkProxyHandler(Object target) {
        this.target = target;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        // 步骤1：通过反射获取目标对象的私有属性
        // 1.1 获取私有字段（单个）
        Field privateField = target.getClass().getDeclaredField("privateField");
        privateField.setAccessible(true); // 突破访问限制
        String fieldValue = (String) privateField.get(target); // 读取值
        System.out.println("读取私有属性privateField：" + fieldValue);

        // 1.2 修改私有属性
        Field privateNumField = target.getClass().getDeclaredField("privateNum");
        privateNumField.setAccessible(true);
        privateNumField.set(target, 200); // 修改值
        System.out.println("修改后私有属性privateNum：" + privateNumField.get(target));

        // 执行目标方法
        return method.invoke(target, args);
    }

    // 创建代理对象
    public static Object createProxy(Object target) {
        return Proxy.newProxyInstance(
                target.getClass().getClassLoader(),
                target.getClass().getInterfaces(),
                new JdkProxyHandler(target)
        );
    }

    // 测试
    public static void main(String[] args) {
        UserService target = new UserServiceImpl();
        UserService proxy = (UserService) createProxy(target);
        proxy.doSomething();
    }
}
```

#### 示例 2：CGLIB 动态代理获取私有属性

```java
// 目标类（无需接口，含私有属性）
public class OrderService {
    private String orderNo = "ORDER_001"; // 私有属性

    public void processOrder() {
        System.out.println("处理订单：" + orderNo);
    }
}

// CGLIB动态代理实现
public class CglibProxyInterceptor implements MethodInterceptor {
    private final Object target; // 持有目标对象引用

    public CglibProxyInterceptor(Object target) {
        this.target = target;
    }

    // 创建代理对象
    public Object createProxy() {
        Enhancer enhancer = new Enhancer();
        enhancer.setSuperclass(target.getClass());
        enhancer.setCallback(this);
        return enhancer.create();
    }

    @Override
    public Object intercept(Object proxy, Method method, Object[] args, MethodProxy methodProxy) throws Throwable {
        // 步骤1：获取私有属性
        Field orderNoField = target.getClass().getDeclaredField("orderNo");
        orderNoField.setAccessible(true);
        // 读取
        String oldOrderNo = (String) orderNoField.get(target);
        System.out.println("原私有属性orderNo：" + oldOrderNo);
        // 修改
        orderNoField.set(target, "ORDER_002");

        // 执行目标方法
        Object result = methodProxy.invoke(target, args);
        return result;
    }

    // 测试
    public static void main(String[] args) {
        OrderService target = new OrderService();
        OrderService proxy = (OrderService) new CglibProxyInterceptor(target).createProxy();
        proxy.processOrder(); // 输出：处理订单：ORDER_002
    }
}
```

### 四、特殊场景：获取父类的私有属性

若私有属性定义在目标类的父类中，需递归遍历父类 Class：

```java
// 工具方法：递归获取私有属性（含父类）
private static Field getDeclaredField(Object target, String fieldName) throws NoSuchFieldException {
    Class<?> clazz = target.getClass();
    while (clazz != null) {
        try {
            return clazz.getDeclaredField(fieldName);
        } catch (NoSuchFieldException e) {
            clazz = clazz.getSuperclass(); // 向上找父类
        }
    }
    throw new NoSuchFieldException("字段" + fieldName + "不存在");
}
```

### 五、面试加分：注意事项

1. **访问权限风险**：`setAccessible(true)`会绕过访问控制，可能破坏封装性，生产中需谨慎使用（仅在必要场景如框架底层、日志监控中使用）；
2. **性能问题**：反射操作比直接访问属性慢，高频调用需缓存`Field`对象（如存入静态 Map）；
3. **final 私有属性**：若私有属性被`final`修饰，反射可读取，但修改会受限制（基本类型 / 字符串无法修改，对象类型可修改内部属性）；
4. **模块化影响（Java 9+）**：若目标类位于不同模块，需在`module-info.java`中开放权限（`opens 包名 to 调用模块`），否则反射会抛出`IllegalAccessException`；
5. **代理对象 vs 目标对象**：务必操作**目标对象（target）**，而非代理对象（proxy），代理对象是动态生成的子类实例，不含目标类的私有属性。

### 六、总结（面试答题话术）

动态代理中获取私有属性的核心是「反射突破访问限制」：

1. 先在代理逻辑中拿到目标对象的引用；
2. 通过`Class.getDeclaredField()`获取私有字段，调用`setAccessible(true)`关闭访问检查；
3. 用`Field.get/set()`读取 / 修改属性；
4. 若属性在父类，需递归遍历父类 Class；
5. 注意反射的性能和封装性问题，生产中需谨慎使用。

该知识点本质是「动态代理」与「Java 反射机制」的结合，也是 Spring AOP 底层（如获取目标对象私有属性做日志 / 监控）的常见场景。
