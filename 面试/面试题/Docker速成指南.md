### **Docker 面试速成指南：核心概念与应答策略**

#### **第一部分：Docker 核心概念（必须理解并背诵）**

这是面试的基石，能清晰地说出这些概念的区别，面试就成功了一半。

**1. 镜像（Image） vs 容器（Container）**

- **镜像 (Image)**：一个**只读的模板**。它包含了运行某个软件所需的所有内容：代码、运行时、库、环境变量和配置文件。比如一个 `MySQL`镜像、一个 `Redis`镜像。 **比喻**：镜像是**软件的安装包**（如 `mysql.msi`）或者是**一个类（Class）**。
- **容器 (Container)**：镜像的一个**运行实例**。当你运行一个镜像时，Docker 会在隔离的环境中创建这个实例。 **比喻**：容器是**安装好的软件**（正在运行的 MySQL 程序）或者是**一个对象（Object）**。
- **标准话术**：“镜像是一个静态的定义，而容器是镜像运行时的实体。一个镜像可以同时运行多个容器实例，就像同一个安装包可以装在很多电脑上一样。”

**2. 仓库（Repository）**

- 用于存放镜像的地方，类似 Git 的仓库。最著名的是 **Docker Hub**。
- 一个仓库里可以有多个镜像，通常通过**标签 (Tag)** 来区分不同版本。
- **镜像全名格式**：`[仓库地址]/[用户名]/[镜像名]:[标签]` 例如：`nginx:latest`(官方Nginx最新版)， `mysql:5.7`， `private.reg.com/myapp:v1.0`(私有仓库镜像)。

**3. Dockerfile**

- 一个文本文件，里面包含了一条条的**指令**。每一条指令都会在镜像上创建一个层，最终构建出一个完整的镜像。
- **核心思想**：用代码（Dockerfile）来定义环境和应用，实现**基础设施即代码**。

#### **第二部分：Docker 高频命令（按使用流程记忆）**

**1. 镜像相关命令**

- `docker pull`：从仓库拉取（下载）一个镜像。例如：`docker pull nginx:latest`。
- `docker images`：列出本地已下载的所有镜像。
- `docker rmi`：删除一个本地镜像。
- `docker build -t .`：**根据当前目录的 Dockerfile 构建一个镜像**，并用 `-t`给它打上标签。这是最关键的命令之一。

**2. 容器生命周期管理**

- `docker run [OPTIONS]`：**创建并启动**一个容器。 **必背核心参数**： `-d`：后台运行（守护进程模式）。 `-p 宿主机端口:容器端口`：端口映射。例如 `-p 8080:80`将容器的80端口映射到宿主机的8080端口。 `-v 宿主机目录:容器目录`：数据卷挂载，实现数据持久化。 `--name`：给容器起一个名字。 **经典启动命令**：`docker run -d -p 8080:80 --name my-nginx nginx`
- `docker ps`：查看正在运行的容器。 `docker ps -a`：查看所有容器（包括已停止的）。
- `docker stop`：优雅地停止一个容器。
- `docker start`：启动一个已停止的容器。
- `docker rm`：删除一个已停止的容器。 `docker rm -f`：强制删除一个容器（包括正在运行的）。

**3. 容器运维与排查**

- `docker logs`：查看容器的日志输出。**这是排查容器问题的最主要命令！** `docker logs -f`：实时追踪日志，类似 `tail -f`。
- `docker exec -it  /bin/bash`：**进入一个正在运行的容器内部**。 `-it`是交互式终端的意思。这是调试容器内部情况的利器。
- `docker inspect`：获取容器（或镜像）的底层详细信息（如IP地址、配置等）。

#### **第三部分：Dockerfile 核心指令（必须掌握）**

一个典型的 Dockerfile 看起来像这样，你需要能解释每一行的作用：

```
# 基础镜像
FROM openjdk:11-jre-slim
# 维护者信息（可选）
LABEL maintainer="yourname@email.com"
# 设置工作目录
WORKDIR /app
# 将宿主机的文件复制到容器内
COPY target/myapp.jar app.jar
# 暴露端口
EXPOSE 8080
# 启动容器时运行的命令
CMD ["java", "-jar", "app.jar"]
```

- `FROM`：指定基础镜像，必须为第一个指令。
- `RUN`：在**构建镜像**时执行的命令，常用于安装软件包。例如：`RUN apt-get update && apt-get install -y vim`。
- `COPY`/ `ADD`：将文件从构建上下文复制到镜像中。`ADD`比 `COPY`功能多（如自动解压），但更推荐使用语义清晰的 `COPY`。
- `WORKDIR`：设置工作目录，相当于 `cd`到这个目录。
- `EXPOSE`：声明容器运行时监听的端口，这只是一个文档性说明，实际映射要靠 `docker run -p`。
- `CMD`：指定**容器启动时**默认运行的命令。一个 Dockerfile 中只能有一个 `CMD`。
- `ENTRYPOINT`：和 `CMD`类似，但不会被 `docker run`后面的命令覆盖。通常与 `CMD`结合使用，`ENTRYPOINT`写死命令，`CMD`作为参数。

#### **第四部分：核心场景与标准应答（如何解决实际问题）**

**1. 如何部署一个 Java Spring Boot 应用？**

- **标准流程话术**： **编写 Dockerfile**：就像上面那个例子，基于 Java 镜像，将打包好的 JAR 包复制进去，设置启动命令。 **构建镜像**：在项目根目录（Dockerfile 所在目录）执行 `docker build -t my-java-app:latest .`。 **运行容器**：执行 `docker run -d -p 8080:8080 --name my-app my-java-app`。 **验证**：访问 `http://宿主机IP:8080`看应用是否正常。

**2. 数据如何持久化？容器删除后数据不就没了吗？**

- **回答**：“是的，容器本身是易变的。为了实现数据持久化，我们使用**数据卷**。”
- **两种方式**： **绑定挂载 (Bind Mount)**：`-v /宿主机/绝对路径:/容器内路径`。直接将宿主机的目录挂载到容器。**适合挂载配置文件或代码进行开发调试。** **卷 (Volume)**：`-v 卷名:/容器内路径`。由 Docker 管理，存储在宿主机的一个特定区域（通常是 `/var/lib/docker/volumes/`）。**更适合数据库数据等生产环境。**
- **例句**：“部署 MySQL 时，我会用 `-v mysql_data:/var/lib/mysql`，这样即使容器被删除，数据也安全地保存在宿主机上。”

**3. 如何查看容器日志和进入容器调试？**

- **标准排查话术**： **首先看日志**：`docker logs -f container_name`，查看应用本身的错误输出。 **如果日志看不出问题，进入容器内部**：`docker exec -it container_name /bin/bash`，然后可以检查配置文件、进程状态、网络连接等，就像操作一台小Linux一样。

**4. Docker 和虚拟机的区别是什么？（必考题！）**

| 特性           | 虚拟机 (VM)            | Docker 容器                |
| -------------- | ---------------------- | -------------------------- |
| **虚拟化级别** | 硬件级虚拟化           | 操作系统级虚拟化           |
| **核心**       | 每个VM有独立的Guest OS | 所有容器共享主机OS内核     |
| **性能**       | 启动慢，占用资源多     | **启动秒级，资源占用极少** |
| **隔离性**     | 强                     | 较弱（但通常足够）         |
| **镜像大小**   | 庞大（GB级别）         | **小巧（MB级别）**         |

- **标准话术**：“虚拟机模拟了整个硬件，在上面运行一个完整的操作系统，再跑应用，所以很重。而Docker容器是进程级的，它利用宿主机的内核，通过命名空间和控制组等技术实现隔离，只打包应用和依赖，所以非常轻量和快速。”

**5. 什么是 Docker Compose？**

- **回答**：“它是一个用于定义和运行多容器Docker应用的工具。通过一个 `docker-compose.yml`文件来配置所有服务，然后用一条命令（`docker-compose up`）就能启动整个应用栈。”

- **使用场景**：当你的应用需要多个服务协作时，比如一个Web应用需要 `Nginx`、`Java App`、`MySQL`、`Redis`。

- **简单示例**：

  ```
  version: '3'
  services:
    web:
      image: my-app:latest
      ports:
        - "8080:8080"
    mysql:
      image: mysql:5.7
      environment:
        MYSQL_ROOT_PASSWORD: secret
      volumes:
        - mysql_data:/var/lib/mysql
  volumes:
    mysql_data:
  ```
