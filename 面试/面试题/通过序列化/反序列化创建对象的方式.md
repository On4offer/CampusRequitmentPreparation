# Java 序列化与反序列化

### 一、序列化/反序列化创建对象的核心原理

序列化（Serialization）是将对象的状态信息转换为可存储/传输的字节序列的过程；反序列化（Deserialization）则是将字节序列还原为对象的过程。通过反序列化创建对象时，**JVM 会直接从字节流中恢复对象，无需调用构造方法**（这是与 `new`、反射调用构造方法的核心区别），本质是重建对象的内存结构。

### 二、实现条件

要让类支持序列化，必须满足：

1. 类实现 `java.io.Serializable` 标记接口（无抽象方法，仅作标识）；
2. 若有父类，父类要么实现 `Serializable`，要么提供无参构造方法（否则反序列化父类属性时会报错）；
3. 类的所有非 transient 成员变量都需支持序列化（基本类型默认支持，引用类型需实现 `Serializable`）；
4. 可选：显式声明 `serialVersionUID`（避免类结构微调后反序列化报 `InvalidClassException`）。

### 三、代码示例

#### 1. 定义可序列化的类

```Java
import java.io.Serializable;

// 实现Serializable接口
public class User implements Serializable {
    // 显式声明serialVersionUID（建议固定值，避免自动生成导致版本不一致）
    private static final long serialVersionUID = 1L;

    private String name;
    private int age;
    // transient修饰的变量不会被序列化
    private transient String password;

    // 构造方法（反序列化不会调用）
    public User(String name, int age, String password) {
        System.out.println("调用构造方法");
        this.name = name;
        this.age = age;
        this.password = password;
    }

    // getter/setter
    public String getName() { return name; }
    public int getAge() { return age; }
    public String getPassword() { return password; }

    @Override
    public String toString() {
        return "User{name='" + name + "', age=" + age + ", password='" + password + "'}";
    }
}
```

#### 2. 序列化与反序列化工具类

```Java
import java.io.*;

public class SerializationUtil {
    // 序列化：将对象写入文件
    public static void serialize(Object obj, String filePath) throws IOException {
        try (ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream(filePath))) {
            oos.writeObject(obj);
        }
    }

    // 反序列化：从文件读取对象（核心：创建对象的过程）
    public static Object deserialize(String filePath) throws IOException, ClassNotFoundException {
        try (ObjectInputStream ois = new ObjectInputStream(new FileInputStream(filePath))) {
            return ois.readObject(); // 反序列化创建对象，不调用构造方法
        }
    }
}
```

#### 3. 测试代码

```Java
public class Test {
    public static void main(String[] args) {
        String filePath = "user.ser";
        // 1. 创建原始对象（调用构造方法）
        User originalUser = new User("张三", 25, "123456");
        System.out.println("原始对象：" + originalUser); // 输出：User{name='张三', age=25, password='123456'}

        try {
            // 2. 序列化
            SerializationUtil.serialize(originalUser, filePath);

            // 3. 反序列化（创建新对象，无构造方法调用）
            User deserializedUser = (User) SerializationUtil.deserialize(filePath);
            System.out.println("反序列化对象：" + deserializedUser); 
            // 输出：User{name='张三', age=25, password='null'}（password是transient，未序列化）

            // 验证：两个对象是不同实例
            System.out.println("是否为同一对象：" + (originalUser == deserializedUser)); // false
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

### 四、关键知识点（面试高频）

1. **构造方法调用**：反序列化创建对象时，不会调用类的任何构造方法（包括无参构造），JVM 直接通过字节流重建对象。
2. **transient 关键字**：修饰的成员变量不会被序列化，反序列化后该字段为默认值（如 String 为 null，int 为 0）。
3. **serialVersionUID 作用**：
   1. 序列化时，JVM 会将类的 `serialVersionUID` 写入字节流；
   2. 反序列化时，会对比字节流中的 `serialVersionUID` 与当前类的 `serialVersionUID`，不一致则抛出 `InvalidClassException`；
   3. 建议显式声明（如 `1L`），避免类结构微调（如新增字段）后自动生成的 `serialVersionUID` 变化导致反序列化失败。
4. **父类序列化规则**：
   1. 若父类实现 `Serializable`，则父类的所有字段会被序列化；
   2. 若父类未实现 `Serializable`，则父类必须提供无参构造方法，否则反序列化时抛出 `InvalidClassException`（此时父类的字段不会被序列化）。
5. **不可序列化的场景**：
   1. 类未实现 `Serializable` 或 `Externalizable`；
   2. 类的成员变量引用了不可序列化的对象（且未用 transient 修饰）；
   3. 静态变量不会被序列化（序列化保存的是对象的状态，静态变量属于类的状态）。

### 五、扩展：Externalizable 接口（进阶）

除了 `Serializable`，还可通过 `Externalizable` 接口自定义序列化逻辑，该接口强制实现 `writeExternal()` 和 `readExternal()` 方法：

```Java
public class User implements Externalizable {
    private static final long serialVersionUID = 1L;
    private String name;
    private int age;
    private String password;

    // 必须提供无参构造（Externalizable反序列化时会调用）
    public User() {}

    public User(String name, int age, String password) {
        this.name = name;
        this.age = age;
        this.password = password;
    }

    // 自定义序列化逻辑（可手动控制哪些字段序列化）
    @Override
    public void writeExternal(ObjectOutput out) throws IOException {
        out.writeObject(name);
        out.writeInt(age);
        // 手动序列化password（即使是transient也能序列化）
        out.writeObject(password);
    }

    // 自定义反序列化逻辑
    @Override
    public void readExternal(ObjectInput in) throws IOException, ClassNotFoundException {
        this.name = (String) in.readObject();
        this.age = in.readInt();
        this.password = (String) in.readObject();
    }

    @Override
    public String toString() {
        return "User{name='" + name + "', age=" + age + ", password='" + password + "'}";
    }
}
```

**注意**：`Externalizable` 反序列化时会调用类的无参构造方法，再执行 `readExternal()`，这是与 `Serializable` 的核心区别。

### 六、面试总结

通过序列化/反序列化创建对象的核心是：**基于字节流重建对象，绕过构造方法，依赖** **`Serializable`** **标识接口，可通过 transient/serialVersionUID 控制序列化行为**。回答时需结合原理、代码示例、关键规则（如构造方法、transient、serialVersionUID）展开，体现对序列化机制的深度理解。
