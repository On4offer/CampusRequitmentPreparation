# 面试题：Docker 网络模式有哪些？分别适合什么场景？

### 一、概念解释

- **Docker 网络模式**：决定容器与容器、容器与宿主机之间如何通信。
- Docker 提供了多种网络驱动（Driver），通过 `--network` 参数指定。

------

### 二、常见网络模式与原理

1. **bridge（默认，桥接网络）**
   - 每个容器分配一个独立 IP，连接到虚拟网桥 `docker0`。
   - 容器间可通过 **容器名/别名** 互相通信。
   - 对外通信需 **端口映射（-p 宿主机端口:容器端口）**。
2. **host（主机网络）**
   - 容器与宿主机共享网络命名空间。
   - 容器 IP 与宿主机一致，性能较高。
   - 无需端口映射，直接使用宿主机 IP:端口。
3. **none（无网络）**
   - 容器只有 loopback 接口，完全隔离。
   - 适合对网络无需求或手动配置的场景。
4. **container（共享网络）**
   - 新容器与指定的已有容器共享网络命名空间。
   - 多个容器使用相同的 IP 和端口空间。
5. **自定义网络（用户定义 Bridge/Overlay）**
   - 用户可通过 `docker network create` 创建网络。
   - 支持容器之间 DNS 解析、跨主机通信（Overlay）。

------

### 三、对比分析

| 网络模式       | 特点                                | 适用场景                       |
| -------------- | ----------------------------------- | ------------------------------ |
| **bridge**     | 默认模式，独立 IP，端口映射访问外部 | 单机部署、微服务之间通信       |
| **host**       | 共享宿主机网络，性能最好            | 高性能应用、需要宿主机端口直通 |
| **none**       | 完全隔离，只有 loopback             | 对网络无需求、安全测试         |
| **container**  | 共享其他容器网络                    | sidecar 模式（日志、监控代理） |
| **自定义网络** | 更灵活，支持 DNS、跨主机            | 微服务集群、Compose/K8s 场景   |

------

### 四、案例说明

- **苍穹外卖项目**：
  - 使用 **bridge 模式**，MySQL、Redis、SpringBoot 容器在同一自定义网络 `waimai-net`，容器间通过服务名通信。
  - 使用 `-p 8080:8080` 将 SpringBoot 暴露给外部用户。

```bash
docker network create waimai-net
docker run -d --name mysql --network=waimai-net mysql:8
docker run -d --name redis --network=waimai-net redis:7
docker run -d -p 8080:8080 --name waimai --network=waimai-net waimai:v1
```

------

### 五、使用场景

- **bridge**：开发测试环境，单机多容器项目。
- **host**：对网络延迟敏感的服务（如高并发 Nginx、游戏服务器）。
- **none**：安全沙箱、离线任务。
- **container**：Sidecar 模式（监控、日志收集容器与主应用共享网络）。
- **自定义网络**：企业级微服务项目，容器之间用服务名直接访问。

------

### 六、面试标准回答（简洁版）

> Docker 网络模式主要有 bridge、host、none、container 和自定义网络。
>
> - 默认是 **bridge**，适合单机容器互联；
> - **host** 模式性能高，适合高并发场景；
> - **none** 完全隔离；
> - **container** 可与其他容器共享网络；
> - **自定义网络** 适合微服务架构和跨主机场景。
