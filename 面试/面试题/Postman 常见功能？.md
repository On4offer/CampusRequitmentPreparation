## 面试题：Postman 的常见功能有哪些？

### 1. 概念

Postman 是一款 **常用的接口测试工具**，支持 REST、SOAP、GraphQL 等多种接口协议，提供可视化操作界面，帮助开发者和测试人员快速完成接口调试、自动化测试和文档管理。

------

### 2. 常见功能

1. **接口调试**：
   - 发送 GET、POST、PUT、DELETE 等 HTTP 请求
   - 设置请求头（Header）、请求参数（Params）、请求体（Body）
2. **环境管理**：
   - 通过 **Environment 变量** 管理不同环境（开发/测试/生产）下的 baseURL、Token 等
3. **用例集合（Collection）**：
   - 接口分组保存，支持批量执行
   - 支持团队共享接口文档
4. **断言与测试脚本**：
   - 使用 JavaScript 编写测试断言（如校验响应状态码、响应体字段）
   - 内置 `pm.test()` 方法
5. **自动化与持续集成**：
   - Collection Runner 批量运行测试
   - Newman（命令行工具）支持在 CI/CD 流水线中集成
6. **Mock Server**：
   - 模拟接口返回结果，用于前后端分离或后端未完成时调试
7. **接口文档**：
   - 自动生成 API 文档
   - 团队协作共享

------

### 3. 使用场景

- **开发调试阶段**：快速验证接口正确性
- **测试阶段**：编写接口自动化测试，减少回归测试成本
- **前后端联调**：通过 Mock Server 提供接口数据
- **上线前压测/冒烟测试**：结合 Runner 批量执行测试用例

------

### 4. 项目实践（苍穹外卖 / 黑马点评 示例）

- **苍穹外卖**：
   使用 Postman 测试 **登录接口**，设置请求体为 JSON（手机号 + 验证码），断言返回的 `code == 1`。
- **黑马点评**：
   使用 Postman 测试 **商户查询接口**，带上 Token 作为请求头，检查返回的店铺信息是否正确。
