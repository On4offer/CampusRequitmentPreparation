# 面试题

**Redis 哨兵机制的作用是什么？如何保证高可用？**

------

## 1️⃣ 概念

- **Redis Sentinel（哨兵）**：Redis 提供的 **高可用解决方案**，用于监控主从架构中的主节点状态，并在主节点故障时自动完成故障转移（Failover）。

------

## 2️⃣ 哨兵机制的作用

1. **监控（Monitoring）**：实时检测主从节点是否正常运行。
2. **通知（Notification）**：当节点出现异常时，通知其他应用或管理员。
3. **自动故障转移（Automatic Failover）**：当主库宕机时，选举新的主库，并让其他从库重新复制新主库。
4. **服务发现（Configuration Provider）**：客户端通过哨兵获取当前主库地址，自动感知主从切换。

------

## 3️⃣ 工作原理

1. **主观下线（SDOWN）**：某个哨兵发现主库长时间无响应，标记为“主观下线”。
2. **客观下线（ODOWN）**：多个哨兵达成一致（基于投票），确认主库确实下线。
3. **领导选举**：哨兵集群通过 Raft 类似的投票算法，选出一个 Leader 哨兵。
4. **故障转移**：Leader 负责选出一个从库作为新主库，并通知其他从库切换复制对象。
5. **配置传播**：哨兵更新配置，并将新的主库信息同步给客户端。

------

## 4️⃣ 如何保证高可用

- **多哨兵部署**：至少 3 个哨兵节点，避免单点故障。
- **投票机制**：多数哨兵确认主库下线后才会触发 Failover，防止误判。
- **自动 Failover**：保证主库宕机后系统能快速恢复，减少不可用时间。
- **客户端重连机制**：客户端不直接连主库，而是通过哨兵发现当前可用的主节点。
- **主从架构**：哨兵依赖主从复制实现高可用。

------

## 5️⃣ 使用场景（项目实践）

- **黑马点评**：
  - 使用哨兵监控 Redis 主从集群，保证热点数据缓存服务高可用。
  - 客户端通过哨兵发现新主库，避免因主库宕机导致缓存不可用。
- **苍穹外卖**：
  - 订单数据缓存和秒杀库存必须高可用，采用 **Redis Sentinel + 主从复制**，保证宕机自动恢复。

------

## 6️⃣ 面试标准回答模板

> Redis 哨兵的作用是保证 Redis 高可用，它能监控主从节点、通知故障、并在主库宕机时自动完成故障转移。
>  它通过 **主观下线 + 客观下线** 两级检测机制避免误判，再由哨兵集群选举 Leader 执行故障转移，将从库提升为主库。
>  客户端可以通过哨兵动态获取主库地址，实现自动化切换。
>  在实际项目中，我通常部署至少 3 个哨兵，配合主从复制，保证缓存系统的高可用和自动恢复能力。
