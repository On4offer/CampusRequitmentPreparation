## 📌 面试题：常见的垃圾回收算法有哪些？

### ⭐ 标准回答

Java 垃圾回收的本质是回收 **不可达对象** 所占用的堆内存，通过回收不再被应用程序引用的对象，从而释放堆内存并避免内存泄漏。在 Java 中，堆内存是用于动态分配对象的区域，当某个对象不再被引用或无法从根对象（GC Root）访问时，该对象就成为 **不可达对象**，也就是垃圾回收的候选者。

常见的 GC 算法有以下几种：

1. **标记-清除（Mark-Sweep）**
   - **原理**：先标记出所有要回收的对象，然后统一清除。
   - **优点**：实现简单，不需要额外内存。
   - **缺点**：效率低；清理后会产生大量**内存碎片**。
   - **应用**：早期 JVM、老年代的基础算法。
2. **复制（Copying）**
   - **原理**：将内存分为两块，每次只使用其中一块；GC 时，把存活对象复制到另一块，然后清空原区域。
   - **优点**：没有碎片，分配连续内存快。
   - **缺点**：需要双倍内存空间；对象存活率高时，复制成本大。
   - **应用**：新生代（Eden + Survivor 区）。
3. **标记-整理（Mark-Compact）**
   - **原理**：先标记存活对象，再让存活对象向一端移动，最后清理边界外的内存。
   - **优点**：没有内存碎片。
   - **缺点**：移动对象开销大，效率低于复制。
   - **应用**：老年代（如 CMS 的 Full GC，G1 的混合回收）。
4. **分代收集（Generational Collection）**
   - **原理**：根据对象存活周期不同，把堆分为 **新生代** 和 **老年代**，针对不同区域采用不同算法：
     - 新生代：对象生命周期短，存活率低 → 采用 **复制算法**。
     - 老年代：对象存活率高 → 采用 **标记-清除** 或 **标记-整理**。
   - **优点**：综合性能最佳，是现代 JVM 的**主流方案**。

------

### 📊 总结对比表

| 算法      | 优点         | 缺点         | 典型应用场景     |
| --------- | ------------ | ------------ | ---------------- |
| 标记-清除 | 实现简单     | 碎片、效率低 | 老年代（早期）   |
| 复制      | 高效、无碎片 | 内存浪费     | 新生代           |
| 标记-整理 | 无碎片       | 移动开销大   | 老年代           |
| 分代收集  | 性能综合最佳 | 实现复杂     | JVM 主流 GC 策略 |
