## 5. 如何实现锁的可重入？

**可重入**：同一线程在已持有锁的情况下，再次请求同一把锁时可以直接进入，而不会死锁。

**实现思路**：锁内部记录「当前持有锁的线程」和「重入次数」；同一线程再次加锁时次数 +1，解锁时次数 -1，减到 0 才真正释放锁。

- **synchronized**：JVM 在对象头 Mark Word 和 Monitor 里会记录持有锁的线程以及重入次数，同一线程重入时只需增加计数，因此天然可重入。
- **ReentrantLock**：基于 AQS，用 **state** 表示锁状态和重入次数，用 **exclusiveOwnerThread** 记录当前持有锁的线程；同一线程再次 `lock()` 时 state 递增，`unlock()` 时递减，到 0 才释放并唤醒后继节点。

**小结**：synchronized 和 ReentrantLock 都支持可重入；分布式场景下 Redisson 的锁也是可重入的，通过 Redis 里存线程标识和重入次数实现。

## 6. 如何实现分布式锁？

**目的**：在分布式环境下，保证同一时刻只有一个节点能操作某共享资源（如扣库存、防重复下单）。

**基于 Redis 的常见实现**：

1. **加锁**  
   - 使用 **SET key value NX EX 过期时间**（Redis 2.6.12+ 一条命令），保证「不存在才设置」和「设置过期时间」的**原子性**，避免 SETNX 成功但未设置过期时间导致死锁。  
   - **value** 一般设为**唯一标识**（如 UUID 或 requestId），用于释放时校验，避免误删其他客户端的锁。

2. **释放锁**  
   - 不能直接 `DEL key`，要先**判断 value 是否为本客户端**，再删除；否则可能删掉别人的锁。  
   - 用 **Lua 脚本**把「判断 value + DEL」放在一起执行，保证**原子性**。

3. **常见问题与应对**  
   - **锁超时**：业务执行时间超过 TTL，锁可能提前释放。可用 **看门狗**（如 Redisson）定时续期。  
   - **误删**：必须用 value 校验 + Lua 释放。  
   - **不可重入**：需要自己维护重入次数，或直接用 **Redisson** 的可重入锁。

**项目中的用法**（小众点评）：秒杀场景用 **Redisson** 实现**用户维度**的分布式锁（如 `lock:order:{userId}`），结合 Redis Lua 做库存校验与扣减、以及乐观锁扣 DB，保证重复下单率和超卖率为 0。

## 7. Lua 脚本的作用

**在 Redis 里的作用**：  
Lua 脚本在 Redis 中**原子执行**，执行期间其他命令不会插入，因此可以把多条 Redis 命令组合成一段逻辑，保证**多步操作的原子性**。

**典型场景**：  
- **秒杀**：需要「查库存 → 判用户是否已下单 → 扣库存 → 记录用户」多步，若分多条命令执行，中间会有并发窗口，可能超卖或重复下单。用一段 Lua 在 Redis 里顺序执行这些步骤，等价于一条命令，避免并发问题。  
- **分布式锁释放**：需要「判断 value 是否等于当前客户端 → 再 DEL」，用 Lua 保证判断和删除原子执行，避免误删。

**项目中的用法**（小众点评）：  
秒杀时用 Lua 脚本在 Redis 里完成：判断库存是否大于 0、判断用户是否在已下单 Set 中、扣减库存、将用户加入 Set；返回 0 表示有资格，1 表示库存不足，2 表示重复下单。Java 侧根据返回值决定是否发 MQ 消息做异步落库。

## 9. 事务的隔离级别

**概念**：事务隔离级别用来控制并发事务之间的可见性，解决脏读、不可重复读、幻读等问题。

**SQL 标准四种隔离级别**：  
1. **读未提交（Read Uncommitted）**：可以读到其他事务未提交的数据，可能**脏读、不可重复读、幻读**，一般不用。  
2. **读已提交（Read Committed，RC）**：只能读到已提交的数据，避免脏读，但可能**不可重复读、幻读**；Oracle 默认。  
3. **可重复读（Repeatable Read，RR）**：同一事务内多次读同一行结果一致，避免脏读和不可重复读；**幻读**在 MySQL InnoDB 中通过 **MVCC + Next-Key Lock** 在很大程度上被解决；**MySQL 默认**。  
4. **串行化（Serializable）**：事务串行执行，无上述问题，但性能最差。

**常见并发问题**：  
- **脏读**：读到其他事务未提交的数据。  
- **不可重复读**：同一事务内两次读同一行，结果不一致（被其他事务提交更新）。  
- **幻读**：同一事务内两次范围查询，结果集行数不一致（被其他事务插入/删除）。

**项目中的使用**：  
实习和小众点评里都用 MySQL，默认 RR；秒杀扣库存用乐观锁（如 `update ... set stock = stock - 1 where ... and stock > 0`），在 RR 下配合唯一约束和业务校验，保证不超卖。

## 10. MySQL 的索引？为什么使用 B+ 树索引？

**MySQL 索引**：  
索引是一种加速查询的数据结构。InnoDB 默认使用 **B+ 树**作为索引结构（主键、普通索引、联合索引等）；Memory 引擎支持 Hash 索引；还有全文索引等。

**为什么用 B+ 树**：  
1. **多路平衡**：每个节点有多个子节点，树高较低（如 2～4 层可存百万级数据），**磁盘 IO 次数少**。  
2. **叶子节点存数据且有序链表**：所有数据在叶子节点，叶子节点按顺序用链表连起来，**范围查询**（如 `BETWEEN`、`ORDER BY`）只需在叶子层遍历，效率高。  
3. **非叶子节点只存索引**：单节点可容纳更多 key，扇出大，树高进一步降低，**缓存命中率**更好。  

**和其他结构对比**：  
- **二叉搜索树/红黑树**：树高较大，磁盘 IO 多，不适合做数据库索引。  
- **Hash**：等值查询 O(1)，但**不支持范围查询、排序**，不适合做通用索引。  
- **B 树**：非叶子节点也存数据，范围查询不如 B+ 树在叶子层顺序扫描高效。

**小结**：MySQL（InnoDB）用 B+ 树，兼顾**等值查询、范围查询、排序**，且 IO 次数少，适合磁盘存储的数据库场景。

## 算法题

### 1. 两个数字字符串的加法（大数加法）

**题意**：给定两个用字符串表示的非负整数，求它们的和，结果也以字符串形式返回。不能直接把字符串转成 int/long（可能溢出）。

**思路**：模拟竖式加法，从低位到高位逐位相加，用变量记录进位。

**示例**：`"123" + "456"` → `"579"`；`"1" + "99"` → `"100"`。

```java
public String addStrings(String num1, String num2) {
    int i = num1.length() - 1, j = num2.length() - 1, carry = 0;
    StringBuilder sb = new StringBuilder();
    while (i >= 0 || j >= 0 || carry > 0) {
        int a = i >= 0 ? num1.charAt(i--) - '0' : 0;
        int b = j >= 0 ? num2.charAt(j--) - '0' : 0;
        int sum = a + b + carry;
        sb.append(sum % 10);
        carry = sum / 10;
    }
    return sb.reverse().toString();
}
```

- 时间 O(max(m,n))，空间 O(max(m,n))（结果长度）。

### 2. 删除有序数组中的重复项 II（保留最多 2 个相同元素）

**题意**：给定非降序数组 nums，原地删除重复项，使每个元素最多出现 **2 次**，返回新长度。  
LeetCode 80.

**思路**：双指针。`slow` 表示「当前写到哪里」，`fast` 表示「当前检查到哪里」。因为允许最多 2 个相同，所以只要 `nums[fast]` 和 `nums[slow-2]` 不同，就可以把 `nums[fast]` 写到 `slow`，然后 `slow++`；这样相同元素最多保留 2 个。

```java
public int removeDuplicates(int[] nums) {
    if (nums.length <= 2) return nums.length;
    int slow = 2;
    for (int fast = 2; fast < nums.length; fast++) {
        if (nums[fast] != nums[slow - 2]) {
            nums[slow++] = nums[fast];
        }
    }
    return slow;
}
```

- 时间 O(n)，空间 O(1)。
