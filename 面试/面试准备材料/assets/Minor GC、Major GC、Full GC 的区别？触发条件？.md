## 📌 面试题：Minor GC、Major GC、Full GC 的区别？触发条件？

### ⭐ 标准回答

在 JVM 中，GC 按回收区域和范围不同，可以分为 **Minor GC、Major GC、Full GC**：

------

### 1. **Minor GC（Young GC）**

- **回收范围**：新生代（Eden + Survivor 区）。
- **触发条件**：Eden 区满时触发。
- **特点**：
  - 采用 **复制算法**（Eden → Survivor）。
  - 频繁发生，但回收速度快。
  - 大多数对象在新生代死亡，所以效率高。

------

### 2. **Major GC（Old GC）**

- **回收范围**：老年代。
- **触发条件**：
  - 老年代空间不足。
  - 大对象直接进入老年代，老年代装不下时。
- **特点**：
  - 采用 **标记-清除** 或 **标记-整理** 算法。
  - 发生频率低，但停顿时间比 Minor GC 长。
- ⚠️ 注意：有些 JVM 实现里，Major GC 和 Full GC 常混用，但严格区分时 Major GC 只指 **老年代 GC**。

------

### 3. **Full GC**

- **回收范围**：整个堆（新生代 + 老年代）+ 方法区（Metaspace）。
- **触发条件**：
  - `System.gc()` 调用（只是建议，可能触发）。
  - 老年代空间不足。
  - 方法区（Metaspace）空间不足。
  - CMS GC 失败（Concurrent Mode Failure）。
- **特点**：
  - 停顿时间最长，对性能影响最大。
  - 通常是 JVM 性能调优时最需要避免的。

------

### 📊 对比表

| GC 类型  | 回收范围                 | 触发条件                           | 算法           | 特点         |
| -------- | ------------------------ | ---------------------------------- | -------------- | ------------ |
| Minor GC | 新生代 (Eden + Survivor) | Eden 区满                          | 复制算法       | 频繁，快     |
| Major GC | 老年代                   | 老年代空间不足                     | 标记-清除/整理 | 慢，停顿长   |
| Full GC  | 整个堆 + 方法区          | 老年代/Metaspace 不足、System.gc() | 混合算法       | 最慢，代价大 |

------

### 🔍 面试追问

1. 为什么要尽量减少 Full GC？
    👉 因为会 Stop-The-World，影响响应时间。
2. 大对象为什么直接进入老年代？
    👉 避免在新生代频繁复制，降低 GC 开销。
3. Minor GC 时对象晋升老年代的条件有哪些？
    👉
   - 对象年龄达到阈值（`-XX:MaxTenuringThreshold`）。
   - Survivor 区放不下，直接晋升。
