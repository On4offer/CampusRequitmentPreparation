# 面试题

**Redis 集群（分片集群）如何实现数据分片？哈希槽的作用是什么？**

------

## 1️⃣ 概念

- **Redis Cluster**：Redis 的分布式部署方案，通过数据分片（sharding）来扩展存储能力和吞吐量，并提供部分高可用能力。
- **目标**：解决单机 Redis 内存有限、QPS 瓶颈、单点故障的问题。

------

## 2️⃣ 数据分片原理

- Redis 集群将整个键空间分为 **16384 个哈希槽（hash slots）**。
- 每个节点负责一部分槽（例如 A: 0–5000，B: 5001–10000，C: 10001–16383）。
- Key 存储时，通过 CRC16 算法对 key 进行哈希，再对 16384 取模，得到槽号 → 决定落在哪个节点。

------

## 3️⃣ 哈希槽的作用

1. **定位数据**：避免全局遍历，客户端只需计算 key 的槽号，就能直接定位到目标节点。
2. **均衡分布**：通过槽平均分配，保证数据在集群中较为均衡。
3. **迁移方便**：当集群扩容或缩容时，可以按槽为单位迁移，避免大规模数据重分布。
4. **客户端路由**：客户端计算槽号，若访问错误节点，Redis 返回 `MOVED` 或 `ASK` 重定向，客户端可自动重试。

------

## 4️⃣ 使用场景

- **高并发**：通过多节点分担请求，提高吞吐量。
- **大数据量**：单机内存不足时，可用集群横向扩展。
- **高可用**：集群中每个主节点都可以有从节点，主挂了从顶上。

------

## 5️⃣ 项目实践（黑马点评 / 苍穹外卖）

- **黑马点评**：
  - 热点店铺数据分布在不同节点，利用哈希槽机制保证扩展性。
  - 采用 Redis Cluster 解决缓存数据量大、单点 Redis 容易成为瓶颈的问题。
- **苍穹外卖**：
  - 高峰期订单量巨大，使用 Redis Cluster 分片存储订单缓存，避免单机内存和 QPS 撑爆。

------

## 6️⃣ 面试标准回答模板

> Redis 集群通过 **哈希槽机制** 实现数据分片：
>
> - 集群定义了 16384 个槽，key 通过 CRC16 算法取模映射到具体槽。
> - 每个节点负责一部分槽，保证数据均匀分布。
> - 当节点扩容或缩容时，可以以槽为单位迁移数据，保证灵活扩展。
>    哈希槽的核心作用是 **定位数据、负载均衡和数据迁移**，使得 Redis 集群能够支持大规模数据存储和高并发访问。
