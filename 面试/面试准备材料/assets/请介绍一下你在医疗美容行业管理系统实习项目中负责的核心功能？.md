# 请介绍一下你在医疗美容行业管理系统实习项目中负责的核心功能？

## 答案要点

我在医疗美容行业管理系统实习项目中主要负责了以下四个核心功能模块的开发：

### 1. Excel商品批量导入功能

**功能描述：**
- 实现了入库管理模块的Excel批量导入商品信息功能
- 支持总部入库和门店入库两种业务场景
- 支持 `.xls` 和 `.xlsx` 两种文件格式

**核心实现：**
- 采用"模糊+精确两步商品匹配策略"：先通过数据库LIKE查询进行模糊匹配，再在内存中精确匹配（去除空格）
- 创建了 `ExcelImportUtils` 工具类，实现代码复用，总部和门店入库共用同一套解析逻辑
- 实现了部分成功机制：逐行验证，单行错误不影响其他行，提供详细的错误信息和相似商品推荐
- 解决了FormData上传问题：修复了请求拦截器中 `Qs.stringify()` 破坏FormData的问题

**技术亮点：**
- 使用Apache POI进行Excel解析，支持多种单元格类型和日期格式转换
- 性能优化：支持100+条商品数据导入，响应时间<10秒
- 用户体验：友好的错误提示、相似商品推荐、导入结果统计

### 2. 密码过期强制修改功能

**功能描述：**
- 实现了员工密码每3个月强制修改一次的安全策略
- 密码超过3个月未修改时，禁止登录并跳转到密码修改页面

**核心实现：**
- 数据库设计：新增 `PasswordChangeTime` 字段记录密码最后修改时间，NULL视为过期
- 登录验证逻辑：先验证密码正确性，再检查是否过期，避免密码错误时也提示过期
- 匿名修改接口：提供无需Token的密码修改接口（`[Anonymous]`），仅验证旧密码
- 统一更新机制：通过 `Employee.ModifyPassword()` 方法统一更新密码修改时间，确保所有密码修改路径都更新

**技术亮点：**
- 跨后端实现：C# .NET后端和Java Spring Boot后端共享同一数据库字段
- 安全设计：密码错误时不提示过期，提升安全性
- 前端交互：自动跳转到密码修改页面，修改成功后返回登录页

### 3. 审计日志功能

**功能描述：**
- 实现了完整的审计日志记录功能，满足审计要求
- 记录登录日志（成功/失败/登出）和操作日志（新增/编辑/删除/导出/下载/上传）

**核心实现：**
- 登录日志：.NET后端登录成功后通过HTTP调用Java后端接口记录日志
- 操作日志：通过Spring拦截器（`OperationLogInterceptor`）自动记录操作日志
- 异步处理：使用 `CompletableFuture.runAsync()` 和 `Task.Run()` 实现异步日志记录，不阻塞主流程
- 智能识别：通过接口路径和HTTP方法智能识别操作类型（新增/编辑/删除等）
- 敏感信息脱敏：对密码、token等敏感信息进行脱敏处理

**技术亮点：**
- 跨后端通信：.NET后端通过HTTP调用Java后端接口
- 容错机制：日志记录失败不影响业务功能
- IP地址识别：支持代理场景下的IP地址识别（X-Forwarded-For等）
- 操作描述友好化：自动生成友好的操作描述（如"预约-新增"、"账单-编辑"）

### 4. 入库单标识功能

**功能描述：**
- 为入库管理系统添加入库单标识功能
- 支持创建入库单时选择是否标识，列表支持按标识状态筛选，修改入库单时支持修改标识

**核心实现：**
- 数据库设计：为门店入库表和总部入库表添加 `isMarked` 字段（TINYINT(1)，默认0）
- 后端接口：保存、更新、查询接口都支持 `isMarked` 参数
- 前端UI：使用 `el-switch` 组件实现标识开关，列表使用星号图标显示标识状态（红色实心星=已标识，灰色空心星=未标识）
- 多类型支持：支持直接入库、退货入库、自行采购三种入库类型

**技术亮点：**
- 向后兼容：默认值为0，历史数据无需特殊处理
- UI一致性：筛选组件样式统一，用户体验良好
- 代码复用：前后端逻辑清晰，易于维护

### 5. Bug修复工作

**主要修复的Bug：**
- **移动端业绩报表数据筛选问题**：修复了消耗业绩报表、开单业绩报表、退款业绩报表中移动端数据被误过滤的问题
  - 问题原因：`isExclusionChannels = 1` 的过滤逻辑未考虑移动端数据（`is_network_order = 1`）
  - 修复方案：在过滤条件中排除移动端数据，确保移动端数据始终参与报表统计
  - 修复范围：3个Mapper文件，10个修复位置

- **客户推荐功能参数校验**：补充和完善了参数校验逻辑

- **入库模块事务管理和数据校验**：完善了事务管理和数据校验逻辑
