# 🎯 面试题

**线程池的核心参数有哪些？底层工作原理？**

------

## ✅ 标准回答

### 1. 线程池的核心参数（ThreadPoolExecutor 构造方法）

```java
ThreadPoolExecutor(
    int corePoolSize,        // 核心线程数
    int maximumPoolSize,     // 最大线程数
    long keepAliveTime,      // 非核心线程存活时间
    TimeUnit unit,           // 存活时间单位
    BlockingQueue<Runnable> workQueue,  // 任务队列
    ThreadFactory threadFactory,        // 线程工厂
    RejectedExecutionHandler handler    // 拒绝策略
)
```

**核心参数说明**：

1. **corePoolSize**：核心线程数，常驻线程数，任务来了优先使用。
2. **maximumPoolSize**：最大线程数 = 核心线程 + 非核心线程。
3. **keepAliveTime**：非核心线程空闲多久会被回收。
4. **workQueue**：阻塞队列，存放等待执行的任务。
5. **threadFactory**：创建线程的工厂（可定制线程名、优先级）。
6. **handler**：拒绝策略（任务满了后的处理方式）。

------

### 2. 线程池的底层工作原理

线程池执行任务的流程：

1. **提交任务**（`execute/submit`）
   - 如果线程数 < corePoolSize → 创建核心线程执行任务。
   - 否则 → 尝试加入任务队列（workQueue）。
   - 如果队列满了 & 当前线程数 < maximumPoolSize → 创建非核心线程执行。
   - 如果达到 maximumPoolSize 且队列也满了 → 触发拒绝策略。
2. **任务调度**
   - 工作线程从 **任务队列**取任务执行。
   - 如果线程空闲时间超过 keepAliveTime 且线程数 > corePoolSize，则回收该线程。
3. **关闭线程池**
   - 调用 `shutdown()`：不接收新任务，等待执行完已有任务。
   - 调用 `shutdownNow()`：试图中断正在执行的任务，并返回队列中的未执行任务。

------

## 📌 使用场景（结合项目）

- **黑马点评**：缓存重建异步任务、短信验证码异步发送，用线程池避免 new Thread 过多。
- **苍穹外卖**：高并发下单时，订单持久化异步处理，用线程池削峰填谷。

------

## ⚠️ 易错点

- 不要用 `Executors.newFixedThreadPool` 等工厂方法（可能导致 OOM），要用 `ThreadPoolExecutor` 显式配置。
- 队列选型要慎重：
  - `LinkedBlockingQueue`（无界队列，可能堆积任务导致 OOM）。
  - `ArrayBlockingQueue`（有界，推荐）。
- 线程池必须 `shutdown()`，否则可能导致程序无法退出。

## 🔍 追问

1. **核心线程和非核心线程的区别？**
    👉 核心线程默认不会回收，非核心线程空闲后会被回收。
2. **常见拒绝策略有哪些？**
    👉 AbortPolicy、CallerRunsPolicy、DiscardPolicy、DiscardOldestPolicy。
3. **execute() 和 submit() 有什么区别？**
    👉 `execute()` 无返回值，`submit()` 返回 Future，可获取执行结果/异常。
