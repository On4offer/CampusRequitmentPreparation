# 面试题

**Redis 如何保证数据一致性和持久性？**

------

## 1️⃣ 概念

- **一致性（Consistency）**：主从节点之间的数据保持相同，避免读写分离下出现脏数据。
- **持久性（Durability）**：数据写入 Redis 后，即使宕机重启，也能恢复到最近一次可靠状态。

------

## 2️⃣ 一致性保证（Replication）

Redis 是 **异步复制**，主从数据可能暂时不一致，但提供一些机制提升可靠性：

- **复制偏移量 + 积压缓冲区**：从库断线后可通过增量复制追上主库。
- **半同步复制（wait 命令）**：主线程阻塞，等待指定数量的从库确认写入后再返回（提升一致性，但降低性能）。
- **哨兵/集群模式**：主库宕机时自动选举从库为主库，保证服务不中断。

⚠️ Redis 复制默认是 **最终一致性**，而非强一致。

------

## 3️⃣ 持久性保证（Persistence）

Redis 提供两种持久化机制：

- **RDB（快照）**：定期将内存快照写入磁盘，优点是文件小、恢复快，但可能丢失快照后写入的数据。
- **AOF（日志）**：追加记录写命令，支持 `everysec`（默认）模式，每秒刷盘一次，最多丢失 1 秒数据。
- **混合持久化（RDB + AOF，Redis 4.0+）**：RDB 快照作为基础，AOF 记录增量操作，兼顾恢复速度和数据安全性。

------

## 4️⃣ 项目实践（黑马点评 / 苍穹外卖）

- **黑马点评**：
  - 店铺缓存可从数据库重建，不强依赖持久化，一般只开启默认 RDB。
- **苍穹外卖**：
  - 订单、库存等关键数据需要更高可靠性 → 开启 **AOF everysec**，并定期 `AOF rewrite` 压缩。
  - 集群模式下，通过主从复制 + 哨兵保证一致性和高可用。

------

## 5️⃣ 面试标准回答模板

> Redis 的一致性主要依赖 **主从复制机制**，通过复制偏移量和积压缓冲区保证断线重连后的数据追赶，一般保证最终一致性。必要时可以用 `wait` 命令提高强一致性。
>  Redis 的持久性主要通过 **RDB 和 AOF** 两种方式：RDB 快照文件恢复快但可能丢数据，AOF 日志更安全但文件大、恢复慢。Redis 4.0 后支持混合持久化，兼顾性能和安全。
>  在实际项目中，对于缓存型数据只保留 RDB，对于订单等核心数据则开启 AOF，保证可靠性。
