# 抽象类和接口的区别？抽象类可以被继承吗？

## 一、抽象类和接口的区别

| 特性 | 抽象类 | 接口 |
|------|--------|------|
| **关键字** | `abstract class` | `interface` |
| **继承方式** | 单继承（`extends`） | 多实现（`implements`） |
| **方法** | 可以有抽象方法和具体方法 | JDK8+ 可以有默认方法和静态方法，JDK9+ 可以有私有方法 |
| **属性** | 可以有各种修饰符的属性 | 只能是 `public static final` |
| **构造方法** | 有构造方法 | 无构造方法 |
| **实例化** | 不能直接实例化 | 不能直接实例化 |
| **设计目的** | 为了代码复用 | 为了行为约定 |
| **使用场景** | 有共同的实现逻辑 | 有共同的行为规范 |

## 二、详细对比分析

### 2.1 抽象类（Abstract Class）

**抽象类**是一种不能被实例化的类，用于定义子类的共同行为和属性，提供代码复用。

**特点**：
- 使用 `abstract` 关键字修饰
- 可以包含抽象方法（没有实现的方法）和具体方法
- 可以包含各种修饰符的属性
- 有构造方法，但不能直接实例化
- 子类通过 `extends` 继承抽象类

**示例**：
```java
public abstract class Animal {
    // 普通属性
    protected String name;
    
    // 静态属性
    public static final int MAX_AGE = 100;
    
    // 构造方法
    public Animal(String name) {
        this.name = name;
    }
    
    // 抽象方法（必须被子类实现）
    public abstract void makeSound();
    
    // 具体方法（可以被子类继承或重写）
    public void eat() {
        System.out.println(name + " 在吃饭");
    }
}

// 子类继承抽象类
public class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }
    
    @Override
    public void makeSound() {
        System.out.println(name + " 汪汪叫");
    }
}

public class Cat extends Animal {
    public Cat(String name) {
        super(name);
    }
    
    @Override
    public void makeSound() {
        System.out.println(name + " 喵喵叫");
    }
}
```

### 2.2 接口（Interface）

**接口**是一种行为规范，定义了一组方法签名，实现该接口的类必须实现这些方法。

**特点**：
- 使用 `interface` 关键字定义
- JDK8 之前：只能包含抽象方法
- JDK8+：可以包含默认方法（`default`）和静态方法
- JDK9+：可以包含私有方法
- 属性只能是 `public static final`
- 没有构造方法
- 类通过 `implements` 实现接口
- 一个类可以实现多个接口

**示例**：
```java
// 定义接口
public interface Flyable {
    // 常量（默认 public static final）
    int MAX_HEIGHT = 10000;
    
    // 抽象方法（默认 public abstract）
    void fly();
    
    // 默认方法（JDK8+）
    default void land() {
        System.out.println("着陆");
    }
    
    // 静态方法（JDK8+）
    static void takeoff() {
        System.out.println("起飞");
    }
}

// 实现接口
public class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟在飞翔");
    }
}

public class Plane implements Flyable {
    @Override
    public void fly() {
        System.out.println("飞机在飞翔");
    }
    
    // 重写默认方法
    @Override
    public void land() {
        System.out.println("飞机着陆");
    }
}
```

### 2.3 多继承与多实现

**Java 不支持类的多继承**，但支持接口的多实现。

```java
// 接口1
public interface Runnable {
    void run();
}

// 接口2
public interface Serializable {
    // 标记接口
}

// 实现多个接口
public class MyThread implements Runnable, Serializable {
    @Override
    public void run() {
        System.out.println("线程运行");
    }
}
```

## 三、抽象类可以被继承吗？

**是的，抽象类可以被继承**。

抽象类的设计目的就是为了被继承，子类通过继承抽象类来获得其公共的属性和方法，并实现抽象方法。

### 3.1 继承抽象类的规则

1. **必须实现所有抽象方法**：子类必须实现父类的所有抽象方法，否则子类也必须声明为抽象类。

2. **可以继承具体方法**：子类可以继承父类的具体方法，也可以重写它们。

3. **调用父类构造方法**：子类构造方法必须调用父类的构造方法（显式或隐式）。

**示例**：
```java
// 抽象类
public abstract class Shape {
    protected String color;
    
    public Shape(String color) {
        this.color = color;
    }
    
    public abstract double area();
    
    public void display() {
        System.out.println("颜色: " + color);
    }
}

// 继承抽象类并实现抽象方法
public class Circle extends Shape {
    private double radius;
    
    public Circle(String color, double radius) {
        super(color);  // 调用父类构造方法
        this.radius = radius;
    }
    
    @Override
    public double area() {
        return Math.PI * radius * radius;
    }
    
    // 重写父类方法
    @Override
    public void display() {
        super.display();
        System.out.println("半径: " + radius);
        System.out.println("面积: " + area());
    }
}

// 继承抽象类但不实现所有抽象方法（必须声明为抽象类）
public abstract class Rectangle extends Shape {
    protected double width;
    protected double height;
    
    public Rectangle(String color, double width, double height) {
        super(color);
        this.width = width;
        this.height = height;
    }
    
    // 没有实现 area() 方法，所以必须声明为抽象类
}

// 继承抽象子类并实现所有抽象方法
public class Square extends Rectangle {
    public Square(String color, double side) {
        super(color, side, side);
    }
    
    @Override
    public double area() {
        return width * height;
    }
}
```

## 四、抽象类和接口的应用场景

### 4.1 抽象类的应用场景

**场景1：医疗美容系统中的服务基类**

```java
public abstract class Service {
    protected String name;
    protected double price;
    
    public Service(String name, double price) {
        this.name = name;
        this.price = price;
    }
    
    // 抽象方法：执行服务
    public abstract void perform();
    
    // 具体方法：获取价格
    public double getPrice() {
        return price;
    }
    
    // 具体方法：获取服务名称
    public String getName() {
        return name;
    }
}

public class FacialService extends Service {
    public FacialService() {
        super("面部护理", 299.0);
    }
    
    @Override
    public void perform() {
        System.out.println("执行面部护理服务");
    }
}

public class BodyService extends Service {
    public BodyService() {
        super("身体护理", 399.0);
    }
    
    @Override
    public void perform() {
        System.out.println("执行身体护理服务");
    }
}
```

### 4.2 接口的应用场景

**场景2：医疗美容系统中的支付接口**

```java
public interface Payment {
    // 支付方法
    boolean pay(double amount);
    
    // 获取支付方式名称
    String getPaymentName();
    
    // 默认方法：验证支付
    default boolean validate() {
        System.out.println("验证支付信息");
        return true;
    }
}

public class WeChatPayment implements Payment {
    @Override
    public boolean pay(double amount) {
        System.out.println("微信支付: " + amount);
        return true;
    }
    
    @Override
    public String getPaymentName() {
        return "微信支付";
    }
}

public class AlipayPayment implements Payment {
    @Override
    public boolean pay(double amount) {
        System.out.println("支付宝支付: " + amount);
        return true;
    }
    
    @Override
    public String getPaymentName() {
        return "支付宝支付";
    }
}
```

### 4.3 抽象类和接口的结合使用

```java
// 抽象类：提供基础实现
public abstract class BaseService {
    protected String name;
    
    public BaseService(String name) {
        this.name = name;
    }
    
    public abstract void execute();
    
    public void log() {
        System.out.println("执行服务: " + name);
    }
}

// 接口：定义额外行为
public interface Cancellable {
    void cancel();
}

// 实现类：继承抽象类并实现接口
public class AppointmentService extends BaseService implements Cancellable {
    private String appointmentId;
    
    public AppointmentService(String name, String appointmentId) {
        super(name);
        this.appointmentId = appointmentId;
    }
    
    @Override
    public void execute() {
        log();
        System.out.println("执行预约服务，预约ID: " + appointmentId);
    }
    
    @Override
    public void cancel() {
        System.out.println("取消预约，预约ID: " + appointmentId);
    }
}
```

## 五、JDK8+ 接口的新特性

### 5.1 默认方法（Default Methods）

默认方法允许在接口中提供方法的默认实现，解决了接口演进的问题。

```java
public interface Collection<E> {
    // 抽象方法
    boolean add(E e);
    
    // 默认方法（JDK8+）
    default boolean removeIf(Predicate<? super E> filter) {
        Objects.requireNonNull(filter);
        boolean removed = false;
        final Iterator<E> each = iterator();
        while (each.hasNext()) {
            if (filter.test(each.next())) {
                each.remove();
                removed = true;
            }
        }
        return removed;
    }
}
```

### 5.2 静态方法（Static Methods）

静态方法允许在接口中定义静态方法，提供工具方法。

```java
public interface MathUtils {
    // 静态方法
    static int max(int a, int b) {
        return a > b ? a : b;
    }
    
    static int min(int a, int b) {
        return a < b ? a : b;
    }
}
```

### 5.3 私有方法（Private Methods，JDK9+）

私有方法允许在接口中定义私有方法，用于默认方法之间的代码复用。

```java
public interface Calculator {
    // 默认方法
    default int add(int a, int b) {
        validate(a, b);
        return a + b;
    }
    
    default int subtract(int a, int b) {
        validate(a, b);
        return a - b;
    }
    
    // 私有方法
    private void validate(int a, int b) {
        if (a < 0 || b < 0) {
            throw new IllegalArgumentException("参数不能为负数");
        }
    }
}
```

## 六、面试标准回答（2分钟）

「抽象类和接口都是 Java 中实现抽象的机制。抽象类使用 abstract class 关键字，支持单继承，可以包含抽象方法和具体方法，有构造方法；接口使用 interface 关键字，支持多实现，JDK8+ 可以有默认方法和静态方法，没有构造方法。抽象类的设计目的是代码复用，接口的设计目的是行为约定。抽象类可以被继承，子类必须实现所有抽象方法，否则子类也必须声明为抽象类。在医疗美容系统中，我们可以使用抽象类定义服务的基类，使用接口定义支付等行为规范。」

## 七、常见追问

**Q1：什么时候使用抽象类，什么时候使用接口？**

- 当需要代码复用时，使用抽象类
- 当需要定义行为规范时，使用接口
- 当需要多继承时，使用接口

**Q2：抽象类可以有构造方法吗？**

可以有构造方法，但不能直接实例化抽象类。构造方法用于子类调用。

**Q3：接口中的属性是什么修饰符？**

接口中的属性默认是 `public static final`，必须初始化。

**Q4：抽象类和接口的区别在 JDK8 之后有什么变化？**

JDK8 之后，接口可以有默认方法和静态方法，增加了接口的灵活性。

**Q5：一个类可以同时继承抽象类和实现接口吗？**

可以，先继承抽象类，再实现接口。

## 八、小结表

| 对比项 | 抽象类 | 接口 |
|--------|--------|------|
| 关键字 | abstract class | interface |
| 继承方式 | extends（单继承） | implements（多实现） |
| 方法 | 抽象方法 + 具体方法 | 抽象方法 + 默认方法 + 静态方法 |
| 属性 | 各种修饰符 | public static final |
| 构造方法 | 有 | 无 |
| 实例化 | 不能 | 不能 |
| 设计目的 | 代码复用 | 行为约定 |
| 继承性 | 可以被继承 | 可以被实现 |
