# 🎯 面试题

**volatile 关键字的作用？能保证原子性吗？**

------

## ✅ 标准回答

1. **volatile 的作用**

- **保证可见性**：一个线程修改变量，其他线程能立即看到。
- **禁止指令重排**：通过 **内存屏障**（Memory Barrier）禁止特定指令重排序，保证有序性。
- **不保证原子性**：仅能保证单次读/写的原子性，**复合操作（i++）不安全**。
- `volatile` 关键字只能作用于 **变量**，并且只能修饰 **成员变量** 或 **局部变量**，不能用于 **方法** 或 **类**。

------

## ⚙️ 底层原理

1. **可见性**：

- `volatile` 写操作时，JVM 会插入 **Store 屏障**，强制刷新主内存。
- 读操作时，插入 **Load 屏障**，强制从主内存读取最新值。

1. **有序性**：

- `volatile` 修饰的变量读/写前后，会加入内存屏障，禁止 JVM/CPU 乱序优化。

1. **原子性**：

- `volatile` **只保证单次读写**原子性，但不能保证复合操作：

  ```java
  volatile int count = 0;
  // 线程不安全：读 -> +1 -> 写，可能丢失更新
  count++;
  ```

------

## 📌 使用场景（结合项目）

- **状态标志**：

  ```java
  private volatile boolean running = true;
  while (running) { ... } // 线程间安全停止
  ```

- **单例模式**：DCL（双重检查锁定）必须配合 volatile，防止指令重排。

- **黑马点评/苍穹外卖**：异步下单任务里，用 `volatile` 作为 “关闭任务标志”，可以安全地通知工作线程结束。

------

## ⚠️ 易错点

- 不能替代锁，不能保证 **复合操作原子性**。
- `volatile` 修饰引用时，只能保证**引用本身可见性**，不保证引用对象内部字段线程安全。
- 用在高并发计数器上会丢失更新，必须配合 **CAS/AtomicInteger**。

------

## 🔍 追问

1. **i++ 用 volatile 修饰安全吗？**
    👉 不安全，`i++` 是复合操作（读→加1→写），需要 CAS 或锁保证原子性。
2. **volatile 和 synchronized 的区别？**
    👉 `volatile` 保证 **可见性+有序性**，不保证原子性；`synchronized` 三者都保证。
3. **DCL 单例为什么必须用 volatile？**
    👉 防止 `new Singleton()` 被重排序（先分配内存→赋给引用→执行构造），导致其他线程拿到未初始化的对象。
