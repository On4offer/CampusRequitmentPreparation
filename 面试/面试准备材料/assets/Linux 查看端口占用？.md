## 面试题：Linux 如何查看端口占用？

### 1. 概念

在服务器开发与运维中，经常会遇到 **端口被占用** 的问题，比如启动服务失败、部署冲突。Linux 提供了多种工具来排查某个端口的占用情况。

------

### 2. 常见命令

#### （1）`netstat`（较老，依赖 `net-tools` 包）

- `netstat -tunlp | grep 8080`
  - `-t`：TCP
  - `-u`：UDP
  - `-n`：显示数字地址和端口（不反查域名）
  - `-l`：监听中的端口
  - `-p`：显示进程 PID

#### （2）`lsof`（推荐，常用）

- `lsof -i:8080`
   显示占用 8080 端口的进程信息。

#### （3）`ss`（新工具，替代 `netstat`，性能更好）

- `ss -tunlp | grep 8080`

#### （4）`fuser`

- `fuser -n tcp 8080`
   显示使用该端口的进程 PID。

------

### 3. 案例演示

```bash
# 查看 8080 端口占用
netstat -tunlp | grep 8080

# 使用 lsof 查看
lsof -i:8080

# 使用 ss 查看
ss -tunlp | grep 8080

# 根据 PID 杀掉进程
kill -9 <PID>
```

------

### 4. 使用场景

- **Web 服务冲突**：Nginx/Apache 占用 80 端口，SpringBoot 启动失败。
- **数据库服务冲突**：MySQL/PostgreSQL 启动时端口被占用。
- **排查高并发异常**：某端口连接数过多导致拒绝服务。

------

### 5. 标准回答（面试简洁版）

> 在 Linux 中，可以通过以下命令查看端口占用情况：
>
> - `netstat -tunlp | grep <端口号>`
> - `lsof -i:<端口号>`
> - `ss -tunlp | grep <端口号>`
> - `fuser -n tcp <端口号>`
>    如果确认端口被占用，可以结合 `kill -9 PID` 结束相关进程，或者修改服务配置文件中的端口号。
