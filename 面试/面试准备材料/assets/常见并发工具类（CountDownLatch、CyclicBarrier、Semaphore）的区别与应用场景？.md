# 🎯 面试题

**常见并发工具类（CountDownLatch、CyclicBarrier、Semaphore）的区别与应用场景？**

------

## ✅ 标准回答

### 1. CountDownLatch（倒计时器）

- **原理**：内部维护一个计数器（state），每次调用 `countDown()` → 计数减 1，直到为 0 时释放所有等待线程。
- **特性**：一次性，不能重置。
- **应用场景**：
  - 主线程等待多个子任务完成后再继续执行。
  - 黑马点评项目：缓存预热，主线程等待所有缓存加载完毕后再对外提供服务。

------

### 2. CyclicBarrier（循环栅栏）

- **原理**：设定一个屏障点，线程调用 `await()` 进入屏障，当等待的线程数达到指定值时，所有线程同时继续。
- **特性**：可循环使用，每次触发后计数器重置。
- **应用场景**：
  - 多线程分段计算，最后汇总结果。
  - 苍穹外卖项目：多骑手同时出发模拟测试 → 所有线程到达起点后统一出发。

------

### 3. Semaphore（信号量）

- **原理**：内部维护许可（permits），`acquire()` 获取一个许可，`release()` 释放一个许可。
- **特性**：控制同时访问某个资源的线程数。
- **应用场景**：
  - 限制并发访问数量，如限流。
  - 黑马点评项目：限制同时访问“热门商户详情页”的线程数，避免数据库被打挂。
  - 苍穹外卖项目：限制并发下单请求数，保护后端服务。

------

## 📌 区别总结

| 工具类             | 核心作用           | 特性                      | 应用场景                         |
| ------------------ | ------------------ | ------------------------- | -------------------------------- |
| **CountDownLatch** | 等待一组线程完成   | 一次性，计数减到 0 才放行 | 主线程等待缓存预热、任务全部完成 |
| **CyclicBarrier**  | 让一组线程同时出发 | 可循环使用，触发后重置    | 多线程分段计算，批量统一执行     |
| **Semaphore**      | 控制并发访问数     | 基于许可，支持公平/非公平 | 限流、限资源访问（数据库连接数） |

------

## ⚠️ 易错点

- CountDownLatch 不能重置 → 如果要循环使用，应选 CyclicBarrier。
- CyclicBarrier 中任一线程中断/超时，会导致其他线程抛 `BrokenBarrierException`。
- Semaphore 的 `release()` 必须匹配 `acquire()`，否则可能导致 permits 数量失衡。

------

## 🔍 追问

1. **CountDownLatch 和 CyclicBarrier 的区别？**
    👉 CountDownLatch 用于 **一个线程等待多个线程**；CyclicBarrier 用于 **多个线程相互等待**。
2. **Semaphore 和 synchronized 的区别？**
    👉 synchronized 是排他锁，最多 1 个线程进入；Semaphore 可以允许 N 个线程同时访问。
3. **这三个工具类底层实现？**
    👉 都是基于 AQS（AbstractQueuedSynchronizer）实现的。
