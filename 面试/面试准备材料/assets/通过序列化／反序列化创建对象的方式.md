# 通过序列化/反序列化创建对象的方式

## 一、序列化/反序列化概述

### 1.1 定义

**序列化（Serialization）**：
- **定义**：将对象转换为字节流的过程
- **目的**：便于对象在网络传输、持久化存储等场景中使用
- **特点**：对象的状态被保存为字节序列

**反序列化（Deserialization）**：
- **定义**：将字节流转换回对象的过程
- **目的**：从字节流中恢复对象的状态
- **特点**：通过反序列化可以创建新的对象实例

### 1.2 核心概念

| 概念 | 说明 |
|------|------|
| **序列化** | 对象 → 字节流 |
| **反序列化** | 字节流 → 对象 |
| **Serializable 接口** | Java 序列化的标记接口 |
| **ObjectOutputStream** | 序列化输出流 |
| **ObjectInputStream** | 反序列化输入流 |

## 二、Java 原生序列化实现

### 2.1 基本实现步骤

**步骤**：
1. 实现 `Serializable` 接口
2. 使用 `ObjectOutputStream` 序列化对象
3. 使用 `ObjectInputStream` 反序列化对象

### 2.2 示例代码

```java
import java.io.*;

// 1. 实现 Serializable 接口
class Person implements Serializable {
    private static final long serialVersionUID = 1L;  // 序列化版本号
    
    private String name;
    private int age;
    private transient String password;  // transient 修饰的字段不会被序列化
    
    public Person(String name, int age, String password) {
        this.name = name;
        this.age = age;
        this.password = password;
    }
    
    // getter/setter 方法
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
    public String getPassword() { return password; }
    public void setPassword(String password) { this.password = password; }
    
    @Override
    public String toString() {
        return "Person{name='" + name + "', age=" + age + ", password='" + password + "'}";
    }
}

// 序列化和反序列化示例
public class SerializationExample {
    public static void main(String[] args) {
        Person person = new Person("张三", 20, "123456");
        
        // 序列化
        try {
            FileOutputStream fos = new FileOutputStream("person.ser");
            ObjectOutputStream oos = new ObjectOutputStream(fos);
            oos.writeObject(person);
            oos.close();
            fos.close();
            System.out.println("对象序列化成功");
        } catch (IOException e) {
            e.printStackTrace();
        }
        
        // 反序列化（创建新对象）
        try {
            FileInputStream fis = new FileInputStream("person.ser");
            ObjectInputStream ois = new ObjectInputStream(fis);
            Person deserializedPerson = (Person) ois.readObject();
            ois.close();
            fis.close();
            
            System.out.println("反序列化对象: " + deserializedPerson);
            System.out.println("原对象: " + person);
            System.out.println("是否为同一对象: " + (person == deserializedPerson));  // false
        } catch (IOException | ClassNotFoundException e) {
            e.printStackTrace();
        }
    }
}
```

### 2.3 内存中的序列化/反序列化

**不写入文件，直接在内存中操作**：
```java
public class InMemorySerialization {
    
    /**
     * 序列化对象到字节数组
     */
    public static byte[] serialize(Object obj) throws IOException {
        ByteArrayOutputStream bos = new ByteArrayOutputStream();
        ObjectOutputStream oos = new ObjectOutputStream(bos);
        oos.writeObject(obj);
        oos.close();
        return bos.toByteArray();
    }
    
    /**
     * 从字节数组反序列化对象
     */
    public static <T> T deserialize(byte[] data, Class<T> clazz) 
            throws IOException, ClassNotFoundException {
        ByteArrayInputStream bis = new ByteArrayInputStream(data);
        ObjectInputStream ois = new ObjectInputStream(bis);
        Object obj = ois.readObject();
        ois.close();
        return clazz.cast(obj);
    }
    
    public static void main(String[] args) throws Exception {
        Person person = new Person("李四", 25, "password");
        
        // 序列化到内存
        byte[] bytes = serialize(person);
        System.out.println("序列化字节数组长度: " + bytes.length);
        
        // 从内存反序列化（创建新对象）
        Person newPerson = deserialize(bytes, Person.class);
        System.out.println("反序列化对象: " + newPerson);
        System.out.println("是否为同一对象: " + (person == newPerson));  // false
    }
}
```

## 三、序列化的关键点

### 3.1 Serializable 接口

**标记接口**：
- `Serializable` 是一个标记接口，没有方法
- 实现此接口表示类可以被序列化
- 不实现此接口会抛出 `NotSerializableException`

```java
// 必须实现 Serializable 接口
class Person implements Serializable {
    // ...
}
```

### 3.2 serialVersionUID

**序列化版本号**：
- 用于标识类的版本
- 反序列化时会检查版本号是否一致
- 如果不一致会抛出 `InvalidClassException`

```java
class Person implements Serializable {
    // 显式声明 serialVersionUID（推荐）
    private static final long serialVersionUID = 1L;
    
    // 如果不声明，JVM 会根据类结构自动生成
    // 但类结构改变后，自动生成的 UID 会变化，导致反序列化失败
}
```

**生成 serialVersionUID**：
```java
// 使用 serialver 工具生成
// serialver Person

// 或使用 IDE 自动生成
// IDEA: Alt + Enter → Add 'serialVersionUID' field
```

### 3.3 transient 关键字

**transient 的作用**：
- 标记字段不被序列化
- 常用于敏感信息（如密码）或不需要持久化的字段

```java
class Person implements Serializable {
    private String name;
    private transient String password;  // 不会被序列化
    
    // 反序列化后，password 为 null
}
```

### 3.4 自定义序列化

**实现 writeObject 和 readObject**：
```java
class Person implements Serializable {
    private static final long serialVersionUID = 1L;
    private String name;
    private int age;
    
    // 自定义序列化方法
    private void writeObject(ObjectOutputStream oos) throws IOException {
        oos.defaultWriteObject();  // 默认序列化
        // 可以添加自定义逻辑
        oos.writeUTF(name.toUpperCase());  // 序列化时转换为大写
    }
    
    // 自定义反序列化方法
    private void readObject(ObjectInputStream ois) 
            throws IOException, ClassNotFoundException {
        ois.defaultReadObject();  // 默认反序列化
        // 可以添加自定义逻辑
        name = ois.readUTF().toLowerCase();  // 反序列化时转换为小写
    }
}
```

## 四、反序列化创建对象的特点

### 4.1 不调用构造方法

**重要特性**：
- 反序列化创建对象时，**不会调用类的构造方法**
- 对象是通过直接分配内存和恢复字段值创建的

```java
class Person implements Serializable {
    private static final long serialVersionUID = 1L;
    private String name;
    
    public Person() {
        System.out.println("构造方法被调用");
    }
    
    public Person(String name) {
        this.name = name;
        System.out.println("带参构造方法被调用");
    }
}

// 测试
Person p1 = new Person("张三");  // 输出: 带参构造方法被调用
byte[] bytes = serialize(p1);
Person p2 = deserialize(bytes, Person.class);  // 不输出任何内容（构造方法未被调用）
```

### 4.2 字段初始化

**字段初始化顺序**：
1. 分配对象内存
2. 恢复字段值（从字节流读取）
3. 不执行构造方法
4. 不执行字段初始化器（如果有）

```java
class Person implements Serializable {
    private static final long serialVersionUID = 1L;
    private String name = "默认值";  // 字段初始化器
    
    public Person() {
        System.out.println("构造方法");
        this.name = "构造方法设置的值";
    }
}

// 序列化时 name = "构造方法设置的值"
// 反序列化后 name = "构造方法设置的值"（从字节流恢复，不是"默认值"）
```

### 4.3 对象引用关系

**引用类型字段**：
- 如果对象包含引用类型字段，反序列化时会递归反序列化这些对象
- 如果多个字段引用同一对象，反序列化后会保持引用关系

```java
class Person implements Serializable {
    private static final long serialVersionUID = 1L;
    private String name;
    private Address address;
}

class Address implements Serializable {
    private static final long serialVersionUID = 1L;
    private String city;
}

// 序列化
Address addr = new Address("北京");
Person p1 = new Person("张三", addr);
Person p2 = new Person("李四", addr);  // 共享同一个 Address

byte[] bytes1 = serialize(p1);
byte[] bytes2 = serialize(p2);

// 反序列化
Person p1Copy = deserialize(bytes1, Person.class);
Person p2Copy = deserialize(bytes2, Person.class);

// p1Copy 和 p2Copy 的 address 是不同的对象（因为分别序列化）
```

## 五、实际应用场景

### 5.1 场景1：对象深拷贝

```java
/**
 * 通过序列化实现深拷贝
 */
public static <T extends Serializable> T deepCopy(T original) 
        throws IOException, ClassNotFoundException {
    ByteArrayOutputStream bos = new ByteArrayOutputStream();
    ObjectOutputStream oos = new ObjectOutputStream(bos);
    oos.writeObject(original);
    oos.close();
    
    ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
    ObjectInputStream ois = new ObjectInputStream(bis);
    T copy = (T) ois.readObject();
    ois.close();
    
    return copy;
}

// 使用示例
Person p1 = new Person("张三", 20, "pwd");
Person p2 = deepCopy(p1);  // 深拷贝，完全独立的对象
```

### 5.2 场景2：对象持久化

```java
/**
 * 将对象保存到文件
 */
public static void saveObject(Object obj, String filename) throws IOException {
    FileOutputStream fos = new FileOutputStream(filename);
    ObjectOutputStream oos = new ObjectOutputStream(fos);
    oos.writeObject(obj);
    oos.close();
    fos.close();
}

/**
 * 从文件加载对象
 */
public static <T> T loadObject(String filename, Class<T> clazz) 
        throws IOException, ClassNotFoundException {
    FileInputStream fis = new FileInputStream(filename);
    ObjectInputStream ois = new ObjectInputStream(fis);
    T obj = clazz.cast(ois.readObject());
    ois.close();
    fis.close();
    return obj;
}
```

### 5.3 场景3：网络传输

```java
/**
 * 通过网络传输对象
 */
public static void sendObject(Object obj, Socket socket) throws IOException {
    ObjectOutputStream oos = new ObjectOutputStream(socket.getOutputStream());
    oos.writeObject(obj);
    oos.flush();
}

/**
 * 从网络接收对象
 */
public static <T> T receiveObject(Socket socket, Class<T> clazz) 
        throws IOException, ClassNotFoundException {
    ObjectInputStream ois = new ObjectInputStream(socket.getInputStream());
    T obj = clazz.cast(ois.readObject());
    return obj;
}
```

### 5.4 场景4：缓存对象

```java
/**
 * 将对象缓存到 Redis（序列化为字节数组）
 */
public void cacheObject(String key, Object obj) throws IOException {
    byte[] bytes = serialize(obj);
    redisTemplate.opsForValue().set(key, bytes);
}

/**
 * 从 Redis 获取对象（反序列化）
 */
public <T> T getCachedObject(String key, Class<T> clazz) 
        throws IOException, ClassNotFoundException {
    byte[] bytes = (byte[]) redisTemplate.opsForValue().get(key);
    if (bytes == null) {
        return null;
    }
    return deserialize(bytes, clazz);
}
```

## 六、注意事项和限制

### 6.1 必须实现 Serializable

**限制**：
- 所有需要序列化的类都必须实现 `Serializable` 接口
- 包括所有引用类型字段的类
- 否则会抛出 `NotSerializableException`

```java
class Person implements Serializable {
    private Address address;  // Address 也必须实现 Serializable
}

class Address implements Serializable {  // 必须实现
    // ...
}
```

### 6.2 静态字段不会被序列化

**静态字段**：
- 静态字段属于类，不属于对象
- 不会被序列化
- 反序列化后使用类的静态字段值

```java
class Person implements Serializable {
    private static String country = "中国";  // 不会被序列化
    private String name;  // 会被序列化
}
```

### 6.3 transient 字段

**transient 字段**：
- `transient` 修饰的字段不会被序列化
- 反序列化后值为 `null`（基本类型为默认值）
- 常用于敏感信息或不需要持久化的字段

```java
class Person implements Serializable {
    private String name;
    private transient String password;  // 不会被序列化
    
    // 反序列化后 password 为 null
}
```

### 6.4 性能考虑

**性能开销**：
- 序列化/反序列化有性能开销
- 比直接创建对象慢
- 字节数组较大，占用内存

**优化建议**：
- 只序列化必要的字段
- 使用 `transient` 排除不需要的字段
- 考虑使用更高效的序列化框架（如 Protobuf、Kryo）

## 七、常见面试追问

### Q1：反序列化时是否会调用构造方法？

**答**：
- ❌ **不会调用构造方法**：反序列化创建对象时，不会调用类的任何构造方法
- **创建方式**：对象是通过直接分配内存和恢复字段值创建的
- **原因**：反序列化需要恢复对象的状态，而不是重新初始化

### Q2：序列化版本号 serialVersionUID 的作用？

**答**：
- **版本标识**：用于标识类的版本
- **兼容性检查**：反序列化时会检查版本号是否一致
- **版本不一致**：如果版本号不一致，会抛出 `InvalidClassException`
- **建议**：显式声明 `serialVersionUID`，避免类结构改变后自动生成的 UID 变化

### Q3：transient 关键字的作用？

**答**：
- **不序列化**：`transient` 修饰的字段不会被序列化
- **使用场景**：敏感信息（如密码）、不需要持久化的字段、临时字段
- **反序列化后**：`transient` 字段的值为 `null`（基本类型为默认值）

### Q4：序列化能否序列化静态字段？

**答**：
- ❌ **不能序列化静态字段**：静态字段属于类，不属于对象
- **原因**：序列化是针对对象实例的，静态字段是类级别的
- **反序列化后**：使用类的静态字段值（不是序列化时的值）

### Q5：如何实现自定义序列化？

**答**：
- **实现 writeObject 和 readObject**：在类中定义这两个私有方法
- **writeObject**：自定义序列化逻辑
- **readObject**：自定义反序列化逻辑
- **调用默认方法**：在自定义方法中调用 `defaultWriteObject()` 和 `defaultReadObject()`

### Q6：序列化创建的对象和 new 创建的对象有什么区别？

**答**：

| 特性 | new 创建 | 序列化创建 |
|------|---------|-----------|
| **构造方法** | ✅ 调用 | ❌ 不调用 |
| **字段初始化** | ✅ 执行 | ❌ 不执行（从字节流恢复） |
| **对象状态** | 初始状态 | 序列化时的状态 |
| **性能** | 快 | 慢 |
