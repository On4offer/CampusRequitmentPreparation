# 面试题

**Redis 主从同步的原理是什么？全量复制和增量复制如何实现？**

------

## 1️⃣ 概念

- **主从复制（Replication）**：Redis 通过主从架构（Master-Slave）实现数据同步，从库可以作为主库的副本，提供读服务和高可用支持。
- **作用**：读写分离、容灾备份、为哨兵/集群提供高可用基础。

------

## 2️⃣ 原理流程

1. **建立连接**：从库向主库发送 `PSYNC` 命令，请求数据同步。
2. **全量复制**：从库初次连接或主从偏移量差距过大时，主库生成 RDB 文件并发送给从库。
3. **增量复制**：主从间维护 **复制偏移量（offset）** 和 **复制积压缓冲区（repl_backlog_buffer）**，从库断线重连时可增量追上。
4. **命令传播**：主库执行写命令后，通过 **复制流** 异步发送给从库。

------

## 3️⃣ 全量复制实现

- 触发条件：
  - 从库初次连接。
  - 主从数据差距过大，积压缓冲区数据已丢失。
- 实现步骤：
  1. 从库发送 `PSYNC ? -1` 请求全量同步。
  2. 主库执行 `BGSAVE` 生成 RDB 快照，并同时在缓冲区记录后续写命令。
  3. 主库将 RDB 文件传输给从库，从库加载到内存。
  4. 主库再将缓冲区中积累的写命令发给从库，使数据保持一致。

------

## 4️⃣ 增量复制实现

- 触发条件：
  - 主从断开后短时间重连。
  - 从库提供上次复制的 **复制偏移量（offset）** 和 **runid**。
- 实现步骤：
  1. 从库发送 `PSYNC {runid} {offset}` 请求。
  2. 如果偏移量在主库的 **复制积压缓冲区** 中仍然存在，主库只发送缺失的部分数据（增量命令）。
  3. 从库应用这些命令，即可追上主库。
- 如果偏移量不在缓冲区范围 → 自动退化为 **全量复制**。

------

## 5️⃣ 特点

- **异步复制**：主库写命令不会阻塞等待从库确认。
- **半同步保证**：从库可能延迟，主从数据有短暂不一致。
- **网络波动**：短暂断开可用增量复制恢复；长时间断开则退化为全量复制。

------

## 6️⃣ 项目实践（黑马点评 / 苍穹外卖）

- **黑马点评**：读多写少 → 部署读写分离架构，主库负责写，从库分担读操作。
- **苍穹外卖**：订单数据敏感 → 主从架构 + 哨兵，保证高可用；通过增量复制减少网络抖动时的全量压力。

------

## 7️⃣ 面试标准回答模板

> Redis 的主从同步分为 **全量复制** 和 **增量复制**：
>
> - **全量复制**：从库初次连接或数据差距过大时触发，主库生成 RDB 快照发送给从库，再发送缓冲区命令保证一致。
> - **增量复制**：当从库短暂掉线后重连，主库会根据复制偏移量，从复制积压缓冲区中只发送缺失部分命令，减少开销。
>    整个主从同步基于 `PSYNC` 协议实现，保证主从数据尽量一致。在项目中，我用主从 + 哨兵实现高可用，读写分离提高性能。
