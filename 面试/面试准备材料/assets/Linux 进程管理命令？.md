## 面试题：Linux 中常见的进程管理命令？

### 1. 概念

在 Linux 中，**进程管理** 是运维和开发的核心技能之一。进程（Process）是程序的运行实例，通过进程管理命令我们可以实现 **查看、监控、终止和调度** 进程。

------

### 2. 常见命令分类

#### 📌 进程查看

- `ps`：查看静态进程信息
  - `ps -ef`：查看所有进程（标准格式）
  - `ps aux`：查看所有进程（BSD 格式）
- `top`：动态监控进程资源占用（CPU、内存）
- `htop`：增强版 `top`（需安装，支持交互界面）
- `pidof <进程名>`：查看某个进程的 PID

#### 📌 进程终止

- `kill <PID>`：结束指定进程
- `kill -9 <PID>`：强制杀死进程
- `killall <进程名>`：结束所有指定名称的进程
- `pkill <关键字>`：模糊匹配结束进程

#### 📌 进程优先级

- `nice`：启动进程时设置优先级
- `renice`：修改运行中进程的优先级

#### 📌 后台运行

- `&`：命令后台运行
- `jobs`：查看后台任务
- `fg %n`：将后台任务调回前台
- `bg %n`：让暂停的任务在后台继续运行
- `nohup`：忽略挂起信号，任务不中断

------

### 3. 案例演示

```bash
# 查看所有进程
ps -ef | grep nginx

# 实时监控进程
top

# 杀死进程
kill -9 12345

# 后台运行并忽略退出
nohup java -jar app.jar > app.log 2>&1 &

# 修改进程优先级
renice -n 5 -p 12345
```

------

### 4. 使用场景

- **排查服务异常**：用 `ps` + `grep` 找到服务 PID，再结合 `kill` 重启。
- **性能监控**：用 `top`/`htop` 查看 CPU 占用过高的进程。
- **长时间运行任务**：用 `nohup` 或 `screen` 保证程序不因退出终端而中断。
- **任务调度**：通过 `nice`/`renice` 调整进程优先级，避免资源抢占。

------

### 5. 标准回答（面试简洁版）

> Linux 常见的进程管理命令有：
>
> - **查看进程**：`ps`、`top`、`htop`、`pidof`
> - **终止进程**：`kill`、`killall`、`pkill`
> - **优先级管理**：`nice`、`renice`
> - **后台运行**：`&`、`jobs`、`fg`、`bg`、`nohup`
>    它们结合使用，可以实现对进程的全生命周期管理。
