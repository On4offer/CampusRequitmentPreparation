# 面试题：Git 和 SVN 的区别是什么？

### 1. 概念解释

- **SVN（Subversion）**：集中式版本控制系统（CVCS），代码仓库在中央服务器，开发者需要联网才能提交、更新。
- **Git**：分布式版本控制系统（DVCS），每个开发者本地都有完整仓库，支持离线操作。

------

### 2. 核心区别（对比分析）

| 对比维度     | Git                            | SVN                           |
| ------------ | ------------------------------ | ----------------------------- |
| **架构模式** | 分布式（每人一份完整仓库）     | 集中式（中央仓库 + 客户端）   |
| **网络依赖** | 支持离线提交和版本回溯         | 必须联网提交                  |
| **分支管理** | 分支轻量，创建/合并成本低      | 分支较重，合并复杂            |
| **性能**     | 本地操作快，提交/切换分支高效  | 操作依赖服务器，速度慢        |
| **可靠性**   | 无中心也能保留完整历史         | 服务器挂掉会导致无法提交      |
| **存储机制** | 基于快照（snapshot），高效存储 | 基于差异（delta），有时性能差 |
| **适用场景** | 开源协作、大型项目、频繁分支   | 小团队、流程管控严格的公司    |

------

### 3. 案例说明（结合项目）

在 **黑马点评** 或 **苍穹外卖** 项目中：

- 使用 **Git + GitHub/Gitee** 进行协作开发，每个人本地都有完整仓库，分支合并方便。
- 如果用 SVN，每次提交都要依赖中央仓库，断网时无法提交代码，也不利于多人并行开发。

------

### 4. 使用场景

- **Git**：开源项目、互联网公司敏捷开发、需要频繁分支合并的项目。
- **SVN**：传统企业内部项目、代码安全性和流程控制严格的场景。

------

### 5. 面试标准回答模板

> Git 和 SVN 最大的区别在于 **架构模式**：Git 是分布式的，每个开发者本地都有完整仓库，支持离线提交和快速分支管理；而 SVN 是集中式的，依赖中央服务器，分支操作成本高。
>  在实际项目中（如黑马点评），我们一般用 Git，分支轻量，合并高效，适合多人并行协作。
