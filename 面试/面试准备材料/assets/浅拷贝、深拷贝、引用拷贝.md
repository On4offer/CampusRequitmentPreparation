# 浅拷贝、深拷贝、引用拷贝

## 一、概述

### 1.1 定义

**引用拷贝（Reference Copy）**：
- **定义**：仅复制对象的引用（内存地址），不创建新对象
- **特点**：原对象和拷贝引用指向同一块内存空间
- **本质**：给同一个对象起了"新名字"

**浅拷贝（Shallow Copy）**：
- **定义**：创建新对象，新对象的基本数据类型属性复制原值，引用类型属性仅复制引用
- **特点**：新对象和原对象是不同的对象，但引用类型属性共享同一块内存
- **本质**：对象独立，但引用类型属性共享

**深拷贝（Deep Copy）**：
- **定义**：创建新对象，且新对象的所有属性（包括基本类型、引用类型）都复制全新的独立空间
- **特点**：新对象和原对象完全独立，修改任意属性都互不影响
- **本质**：完全独立的对象，递归创建新的引用类型对象

### 1.2 核心对比

| 类型 | 是否创建新对象 | 基本类型属性 | 引用类型属性 | 独立性 |
|------|---------------|-------------|-------------|--------|
| **引用拷贝** | ❌ 否 | 共享 | 共享 | 完全不独立 |
| **浅拷贝** | ✅ 是 | 独立副本 | 共享引用（指向同一对象） | 部分独立 |
| **深拷贝** | ✅ 是 | 独立副本 | 独立副本（递归创建新对象） | 完全独立 |

## 二、引用拷贝

### 2.1 实现方式

**直接赋值**：
```java
Person p1 = new Person("张三", 20);
Person p2 = p1;  // 引用拷贝，p1 和 p2 指向同一个对象
```

### 2.2 示例代码

```java
// 引用类型属性类
class Address {
    private String city;
    
    public Address(String city) {
        this.city = city;
    }
    
    public String getCity() {
        return city;
    }
    
    public void setCity(String city) {
        this.city = city;
    }
    
    @Override
    public String toString() {
        return "Address{city='" + city + "'}";
    }
}

// 主实体类
class Person {
    private String name;
    private int age;
    private Address address;
    
    public Person(String name, int age, Address address) {
        this.name = name;
        this.age = age;
        this.address = address;
    }
    
    // getter/setter 方法
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
    public Address getAddress() { return address; }
    public void setAddress(Address address) { this.address = address; }
    
    @Override
    public String toString() {
        return "Person{name='" + name + "', age=" + age + ", address=" + address + "}";
    }
}

// 引用拷贝示例
public class ReferenceCopyTest {
    public static void main(String[] args) {
        Address addr = new Address("北京");
        Person p1 = new Person("张三", 20, addr);
        
        // 引用拷贝：p2 和 p1 指向同一个对象
        Person p2 = p1;
        
        // 修改 p2 的属性，p1 同步变化
        p2.setAge(25);
        p2.getAddress().setCity("上海");
        
        System.out.println(p1);  // Person{name='张三', age=25, address=Address{city='上海'}}
        System.out.println(p2);  // Person{name='张三', age=25, address=Address{city='上海'}}
        System.out.println(p1 == p2);  // true（同一对象）
        System.out.println(p1.getAddress() == p2.getAddress());  // true（同一对象）
    }
}
```

### 2.3 特点

- ✅ **不创建新对象**：只是复制了引用
- ✅ **完全共享**：所有属性都共享
- ✅ **修改同步**：修改任意一个引用，另一个也会变化
- ✅ **内存效率高**：不占用额外内存

## 三、浅拷贝

### 3.1 实现方式

**实现 Cloneable 接口，重写 clone() 方法**：
```java
class Person implements Cloneable {
    // ...
    
    @Override
    protected Object clone() throws CloneNotSupportedException {
        return super.clone();  // Object 的 clone() 默认是浅拷贝
    }
}
```

### 3.2 示例代码

```java
// Person 类实现 Cloneable 接口
class Person implements Cloneable {
    private String name;
    private int age;
    private Address address;
    
    public Person(String name, int age, Address address) {
        this.name = name;
        this.age = age;
        this.address = address;
    }
    
    // getter/setter 方法
    // ...
    
    // 浅拷贝实现
    @Override
    protected Object clone() throws CloneNotSupportedException {
        return super.clone();  // Object 的 clone() 默认浅拷贝
    }
}

// 测试浅拷贝
public class ShallowCopyTest {
    public static void main(String[] args) throws CloneNotSupportedException {
        Address addr = new Address("北京");
        Person p1 = new Person("张三", 20, addr);
        
        // 浅拷贝：创建新的 Person 对象
        Person p2 = (Person) p1.clone();
        
        // 修改基本类型属性：互不影响
        p2.setAge(25);
        System.out.println(p1.getAge());  // 20
        System.out.println(p2.getAge());  // 25
        
        // 修改引用类型属性：互相影响（共享 Address 对象）
        p2.getAddress().setCity("上海");
        System.out.println(p1.getAddress().getCity());  // 上海
        System.out.println(p2.getAddress().getCity());  // 上海
        
        System.out.println(p1 == p2);  // false（不同对象）
        System.out.println(p1.getAddress() == p2.getAddress());  // true（同一 Address 对象）
    }
}
```

### 3.3 特点

- ✅ **创建新对象**：原对象和拷贝对象是不同的对象
- ✅ **基本类型独立**：基本类型属性互不影响
- ❌ **引用类型共享**：引用类型属性指向同一对象
- ⚠️ **部分独立**：修改引用类型属性会互相影响

### 3.4 浅拷贝的局限性

**集合类型的浅拷贝**：
```java
class Person implements Cloneable {
    private List<String> hobbies;
    
    @Override
    protected Object clone() throws CloneNotSupportedException {
        Person clone = (Person) super.clone();
        // 注意：hobbies 仍然是共享的
        return clone;
    }
}

// 测试
Person p1 = new Person();
p1.setHobbies(new ArrayList<>(Arrays.asList("读书", "游泳")));
Person p2 = (Person) p1.clone();

p2.getHobbies().add("跑步");  // 修改 p2 的集合
System.out.println(p1.getHobbies());  // [读书, 游泳, 跑步]（p1 也被影响）
```

## 四、深拷贝

### 4.1 实现方式1：递归 clone

**步骤**：
1. 所有引用类型属性类实现 `Cloneable` 接口
2. 在 `clone()` 方法中递归调用引用类型属性的 `clone()` 方法

**示例代码**：
```java
// 第一步：Address 实现 Cloneable
class Address implements Cloneable {
    private String city;
    
    public Address(String city) {
        this.city = city;
    }
    
    // getter/setter 方法
    public String getCity() { return city; }
    public void setCity(String city) { this.city = city; }
    
    @Override
    protected Object clone() throws CloneNotSupportedException {
        return super.clone();
    }
}

// 第二步：Person 的 clone() 中递归拷贝 Address
class Person implements Cloneable {
    private String name;
    private int age;
    private Address address;
    
    public Person(String name, int age, Address address) {
        this.name = name;
        this.age = age;
        this.address = address;
    }
    
    // getter/setter 方法
    // ...
    
    @Override
    protected Object clone() throws CloneNotSupportedException {
        // 第一步：浅拷贝 Person 对象
        Person clonePerson = (Person) super.clone();
        // 第二步：深拷贝引用类型属性（Address）
        if (this.address != null) {
            clonePerson.setAddress((Address) this.address.clone());
        }
        return clonePerson;
    }
}

// 测试深拷贝
public class DeepCopyTest {
    public static void main(String[] args) throws CloneNotSupportedException {
        Address addr = new Address("北京");
        Person p1 = new Person("张三", 20, addr);
        Person p2 = (Person) p1.clone();
        
        // 修改引用类型属性：互不影响
        p2.getAddress().setCity("上海");
        System.out.println(p1.getAddress().getCity());  // 北京
        System.out.println(p2.getAddress().getCity());  // 上海
        
        System.out.println(p1.getAddress() == p2.getAddress());  // false（不同 Address 对象）
    }
}
```

### 4.2 实现方式2：序列化（推荐）

**步骤**：
1. 所有类实现 `Serializable` 接口
2. 通过序列化和反序列化实现深拷贝

**示例代码**：
```java
import java.io.*;

// 第一步：所有类实现 Serializable 接口
class Address implements Serializable {
    private static final long serialVersionUID = 1L;
    private String city;
    
    public Address(String city) {
        this.city = city;
    }
    
    // getter/setter 方法
    // ...
}

class Person implements Serializable {
    private static final long serialVersionUID = 1L;
    private String name;
    private int age;
    private Address address;
    
    public Person(String name, int age, Address address) {
        this.name = name;
        this.age = age;
        this.address = address;
    }
    
    // getter/setter 方法
    // ...
    
    // 深拷贝工具方法
    public Person deepCopy() throws IOException, ClassNotFoundException {
        // 序列化
        ByteArrayOutputStream bos = new ByteArrayOutputStream();
        ObjectOutputStream oos = new ObjectOutputStream(bos);
        oos.writeObject(this);
        oos.close();
        
        // 反序列化（生成新对象）
        ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
        ObjectInputStream ois = new ObjectInputStream(bis);
        Person copy = (Person) ois.readObject();
        ois.close();
        
        return copy;
    }
}

// 测试序列化深拷贝
public class SerializationDeepCopyTest {
    public static void main(String[] args) throws Exception {
        Address addr = new Address("北京");
        Person p1 = new Person("张三", 20, addr);
        Person p2 = p1.deepCopy();
        
        p2.getAddress().setCity("上海");
        System.out.println(p1.getAddress().getCity());  // 北京
        System.out.println(p2.getAddress().getCity());  // 上海
        
        System.out.println(p1.getAddress() == p2.getAddress());  // false
    }
}
```

### 4.3 实现方式3：使用工具类

**Apache Commons Lang**：
```java
import org.apache.commons.lang3.SerializationUtils;

// 所有类实现 Serializable
Person p1 = new Person("张三", 20, addr);
Person p2 = SerializationUtils.clone(p1);  // 深拷贝
```

**JSON 序列化**（如 Jackson、Gson）：
```java
import com.fasterxml.jackson.databind.ObjectMapper;

ObjectMapper mapper = new ObjectMapper();
String json = mapper.writeValueAsString(p1);
Person p2 = mapper.readValue(json, Person.class);  // 深拷贝
```

### 4.4 特点

- ✅ **创建新对象**：原对象和拷贝对象是不同的对象
- ✅ **完全独立**：所有属性（基本类型和引用类型）都独立
- ✅ **递归拷贝**：引用类型属性递归创建新对象
- ⚠️ **性能开销**：深拷贝需要更多时间和内存

## 五、详细对比

### 5.1 内存结构对比

**引用拷贝**：
```
栈内存：
p1 ──┐
     ├──> 堆内存：Person对象
p2 ──┘
```

**浅拷贝**：
```
栈内存：
p1 ──> 堆内存：Person对象1 ──> Address对象
p2 ──> 堆内存：Person对象2 ──┘
```

**深拷贝**：
```
栈内存：
p1 ──> 堆内存：Person对象1 ──> Address对象1
p2 ──> 堆内存：Person对象2 ──> Address对象2
```

### 5.2 适用场景

| 拷贝类型 | 适用场景 |
|---------|---------|
| **引用拷贝** | 方法参数传递、临时引用、无需独立对象 |
| **浅拷贝** | 对象仅含基本类型、引用类型只读、性能要求高 |
| **深拷贝** | 多线程操作不同对象、对象修改不影响原数据、需要完全独立 |

## 六、实际应用场景

### 6.1 场景1：对象传递（引用拷贝）

```java
// 方法参数传递使用引用拷贝
public void processPerson(Person person) {
    // person 是引用拷贝，修改会影响原对象
    person.setAge(30);
}

Person p = new Person("张三", 20, addr);
processPerson(p);  // p 的 age 被修改为 30
```

### 6.2 场景2：配置对象复制（浅拷贝）

```java
// 配置对象，引用类型属性只读
class Config implements Cloneable {
    private String name;
    private Map<String, String> properties;  // 只读配置
    
    @Override
    protected Object clone() throws CloneNotSupportedException {
        return super.clone();  // 浅拷贝即可
    }
}
```

### 6.3 场景3：多线程安全（深拷贝）

```java
// 多线程环境下，每个线程需要独立的对象副本
class Task implements Serializable {
    private List<Data> dataList;
    
    public Task deepCopy() throws IOException, ClassNotFoundException {
        // 深拷贝，确保线程安全
        ByteArrayOutputStream bos = new ByteArrayOutputStream();
        ObjectOutputStream oos = new ObjectOutputStream(bos);
        oos.writeObject(this);
        
        ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
        ObjectInputStream ois = new ObjectInputStream(bis);
        return (Task) ois.readObject();
    }
}

// 多线程使用
ExecutorService executor = Executors.newFixedThreadPool(10);
for (int i = 0; i < 10; i++) {
    Task taskCopy = originalTask.deepCopy();  // 每个线程使用独立副本
    executor.submit(() -> processTask(taskCopy));
}
```

### 6.4 场景4：集合拷贝

```java
// ArrayList 的拷贝
List<String> list1 = new ArrayList<>(Arrays.asList("a", "b", "c"));
List<String> list2 = new ArrayList<>(list1);  // 浅拷贝（元素引用共享）

// 深拷贝集合
List<String> list3 = new ArrayList<>();
for (String item : list1) {
    list3.add(new String(item));  // 创建新对象
}
```

## 七、常见面试追问

### Q1：Object.clone() 是浅拷贝还是深拷贝？

**答**：
- **默认是浅拷贝**：`Object.clone()` 方法默认实现是浅拷贝
- **实现方式**：复制对象的基本类型字段，引用类型字段仅复制引用
- **深拷贝需要**：需要手动重写 `clone()` 方法，递归拷贝引用类型属性

### Q2：如何实现集合的深拷贝？

**答**：
- **方式1：手动遍历**：
  ```java
  List<Person> list1 = new ArrayList<>();
  List<Person> list2 = new ArrayList<>();
  for (Person p : list1) {
      list2.add((Person) p.clone());  // 每个元素深拷贝
  }
  ```
- **方式2：序列化**：
  ```java
  List<Person> list2 = SerializationUtils.clone(list1);
  ```
- **方式3：Stream API**：
  ```java
  List<Person> list2 = list1.stream()
      .map(p -> (Person) p.clone())
      .collect(Collectors.toList());
  ```

### Q3：序列化实现深拷贝有什么限制？

**答**：
- **必须实现 Serializable**：所有相关类（包括多层引用）都必须实现 `Serializable` 接口
- **transient 字段不拷贝**：`transient` 修饰的字段不会被序列化，因此不会被拷贝
- **性能开销**：序列化和反序列化有性能开销，比直接拷贝慢
- **静态字段不拷贝**：静态字段属于类，不属于对象，不会被拷贝

### Q4：浅拷贝和深拷贝的性能差异？

**答**：
- **浅拷贝**：性能好，只需复制对象本身和引用
- **深拷贝**：性能差，需要递归创建所有引用类型对象
- **性能差异**：深拷贝通常比浅拷贝慢 10-100 倍（取决于对象复杂度）
- **建议**：如果不需要完全独立，优先使用浅拷贝

### Q5：如何判断拷贝是浅拷贝还是深拷贝？

**答**：
- **判断方法**：修改拷贝对象的引用类型属性，检查原对象是否受影响
- **浅拷贝**：修改后原对象受影响（`p1.getAddress() == p2.getAddress()` 返回 `true`）
- **深拷贝**：修改后原对象不受影响（`p1.getAddress() == p2.getAddress()` 返回 `false`）

### Q6：final 字段能否被拷贝？

**答**：
- **可以拷贝**：`final` 字段的值可以被拷贝到新对象
- **不能修改**：拷贝后的 `final` 字段值不能修改（因为 `final` 的特性）
- **深拷贝**：深拷贝会创建新的 `final` 字段，值相同但可以独立存在

## 八、面试回答模板

### 8.1 核心回答（1分钟）

"引用拷贝只是复制引用，不创建新对象，两个引用指向同一对象。浅拷贝创建新对象，基本类型属性独立，但引用类型属性共享，修改引用类型属性会互相影响。深拷贝创建新对象，所有属性都独立，递归创建引用类型的新对象，完全独立。Java 中实现浅拷贝通过实现 Cloneable 接口重写 clone 方法，实现深拷贝可以通过递归 clone 或序列化反序列化。选择哪种拷贝方式取决于是否需要完全独立的对象。"

### 8.2 扩展回答（3分钟）

"三种拷贝方式的本质区别在于复制的深度。引用拷贝只是复制引用，不创建新对象，相当于给对象起别名。浅拷贝创建新对象，但只复制基本类型和引用，引用类型属性仍然指向原对象，所以修改引用类型属性会互相影响。深拷贝创建新对象，并且递归创建所有引用类型的新对象，完全独立。Java 中 Object.clone() 默认是浅拷贝，需要手动重写实现深拷贝。深拷贝可以通过递归 clone 或序列化实现，序列化方式更通用但性能较差。选择拷贝方式要考虑场景：如果只需要基本类型独立用浅拷贝，如果需要完全独立用深拷贝。"
