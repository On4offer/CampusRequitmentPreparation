# 面试题：MySQL 的慢查询优化有哪些常见方法？

------

## 1. 概念

- **慢查询（Slow Query）**：执行时间超过设定阈值（默认 10 秒）的 SQL，或需要扫描大量数据的 SQL。
- MySQL 提供 **慢查询日志（Slow Log）** 来记录。

------

## 2. 优化方法

### 🔹 1）索引优化

- **单列索引、联合索引** → 避免全表扫描。
- **覆盖索引** → 只查索引字段，避免回表。
- **避免索引失效**：如函数计算、隐式类型转换、`LIKE '%xx'`。

### 🔹 2）SQL 语句优化

- 避免 `SELECT *`，只取需要的字段。
- 避免 `IN` 大量数据，可改为 `JOIN` 或 `EXISTS`。
- 拆分大 SQL → 分批处理。
- 利用 `LIMIT` + 索引优化分页。

### 🔹 3）表结构与数据设计优化

- 选择合适的数据类型（尽量小，避免 TEXT/LOB）。
- 反范式设计：适当增加冗余字段减少 JOIN。
- 分库分表：水平拆分大表。

### 🔹 4）缓存机制

- Redis / Memcached 缓存热点数据，减少数据库压力。
- MySQL 查询缓存（8.0 已移除）。

### 🔹 5）执行计划分析

- 使用 `EXPLAIN` 分析 SQL：查看索引是否命中、扫描行数、执行顺序。

### 🔹 6）其他手段

- 主从读写分离，缓解单库压力。
- 定期归档冷数据，减少活跃表数据量。

------

## 3. 案例

**分页查询优化**：

```sql
-- 慢：OFFSET 很大时需要扫描前面的所有数据
SELECT * FROM orders LIMIT 100000, 20;

-- 优化：利用索引+范围条件
SELECT * FROM orders WHERE id > 100000 LIMIT 20;
```

👉 性能提升显著。

------

## 4. 使用场景

- 电商订单表：高并发下分页、搜索需要索引+缓存优化。
- 日志系统：大表拆分 + 归档避免慢查询。
- 报表统计：使用冗余字段或缓存，避免频繁大范围聚合查询。

------

## 5. 面试模板回答

> MySQL 的慢查询优化主要有几个方向：
>
> - **索引优化**（联合索引、覆盖索引、避免索引失效）；
> - **SQL 语句优化**（避免 SELECT *、优化分页、大 SQL 拆分）；
> - **表结构优化**（字段类型选择、适度冗余、分库分表）；
> - **缓存机制**（Redis 缓存热点数据）；
> - **执行计划分析**（EXPLAIN 排查 SQL 问题）。
>    这些方法可以结合业务场景使用，比如分页优化、电商订单查询就是常见案例。
