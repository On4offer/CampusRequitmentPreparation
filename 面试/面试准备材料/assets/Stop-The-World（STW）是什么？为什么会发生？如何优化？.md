## 📌 面试题：Stop-The-World（STW）是什么？为什么会发生？如何优化？

### ⭐ 标准回答

#### 1. **STW 是什么？**

- **Stop-The-World（世界停顿）** 指的是：
   在 JVM 执行 GC（或其他安全点操作）时，**所有用户线程都会被暂停**，只剩下 GC 线程在工作。
- 本质：为了保证垃圾回收过程中对象图不被修改，确保 **内存一致性和安全性**。

------

#### 2. **为什么会发生 STW？**

- GC 需要在 **一致的对象图** 上工作，否则对象可能在标记/复制过程中被修改，导致回收错误。
- STW 触发场景：
  1. **GC 事件**：如 Minor GC、Major GC、Full GC。
  2. **类加载**：类加载、类卸载时。
  3. **JIT 编译**：C2 编译优化时。
  4. **偏向锁撤销**、安全点（Safepoint）等操作。

------

#### 3. **如何优化 STW？**

- **减少 Full GC 发生**
  - 优化内存分配，避免大对象直接进入老年代。
  - 增加堆大小，调优新生代和老年代比例。
- **选择合适的 GC 收集器**
  - CMS（并发回收，减少停顿）。
  - G1（Region 化，支持可预测停顿）。
  - ZGC / Shenandoah（低延迟，停顿时间 <10ms）。
- **调优 GC 参数**
  - `-XX:MaxGCPauseMillis`：设置最大停顿时间目标。
  - `-XX:GCTimeRatio`：调整吞吐量与停顿时间的平衡。
  - 合理设置 `-Xms`、`-Xmx` 保持堆大小一致，减少扩缩容开销。
- **代码层面优化**
  - 避免频繁创建临时对象（减少 GC 压力）。
  - 使用对象池（如连接池、线程池）。
  - 谨慎使用 `System.gc()`，它会触发 Full GC。

------

### 📊 总结

| 方面     | 说明                                                         |
| -------- | ------------------------------------------------------------ |
| **定义** | JVM 在 GC 或安全点操作时，所有用户线程都会暂停               |
| **原因** | 确保内存一致性（GC、类加载、JIT 编译等）                     |
| **优化** | 减少 Full GC，选用低延迟收集器（G1/ZGC），调优 GC 参数，减少对象分配 |

------

### 🔍 面试追问

1. **为什么 Minor GC 也会产生 STW？**
    👉 因为要移动对象到 Survivor/老年代，必须暂停用户线程，避免引用错误。
2. **Safepoint 和 STW 的关系是什么？**
    👉 STW 在 Safepoint 发生，只有所有线程到达 Safepoint，才能暂停执行。
3. **在你的项目里，有遇到过 STW 吗？怎么优化的？**
    👉 结合实际案例说：例如电商秒杀系统中调优 G1 GC，把停顿时间从 500ms 降到 100ms。
