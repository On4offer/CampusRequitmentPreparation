## 📌 面试题：强引用、软引用、弱引用、虚引用的区别？应用场景？

### ⭐ 标准回答

在 Java 中，引用的强弱直接影响对象的生命周期和垃圾回收行为，主要有以下四类：

1. **强引用（Strong Reference）**
   - **定义**：`Object obj = new Object();`
   - **特性**：只要存在强引用，GC 永远不会回收该对象。
   - **缺点**：容易引发内存泄漏，哪怕 OOM 也不会回收。
   - **应用场景**：普通对象引用。
2. **软引用（SoftReference）**
   - **定义**：通过 `new SoftReference<>(obj)` 创建。
   - **特性**：内存不足时才会回收；内存足够时保留。
   - **应用场景**：缓存，如图片缓存（Android LruCache）、本地缓存。
3. **弱引用（WeakReference）**
   - **定义**：通过 `new WeakReference<>(obj)` 创建。
   - **特性**：只要发生 GC，就会被回收，不管内存是否紧张。
   - **应用场景**：
     - ThreadLocal 的 Key（避免内存泄漏）。
     - WeakHashMap，用于存放临时数据。
4. **虚引用（PhantomReference）**
   - **定义**：必须与 `ReferenceQueue` 一起使用。
   - **特性**：不能通过虚引用获取对象，只能在对象被回收时收到通知。
   - **应用场景**：
     - 管理堆外内存（DirectByteBuffer 回收）。
     - 做对象销毁前的清理工作。

------

### 📊 对比总结表

| 引用类型 | GC 回收时机              | 是否可通过引用访问对象 | 应用场景                  |
| -------- | ------------------------ | ---------------------- | ------------------------- |
| 强引用   | 永不回收（除非引用断开） | ✅                      | 常规对象                  |
| 软引用   | 内存不足时回收           | ✅                      | 缓存                      |
| 弱引用   | 一旦 GC 就回收           | ✅                      | ThreadLocal key、临时数据 |
| 虚引用   | 回收前通知               | ❌                      | 管理堆外内存、资源清理    |

------

### 🔍 面试追问

1. 为什么 ThreadLocal 要用弱引用保存 Key？
    👉 避免线程不结束导致 Key 强引用存在，内存泄漏。
2. 软引用比弱引用更适合缓存的原因？
    👉 软引用在内存充足时能保留数据，提高缓存命中率。
3. 能否用虚引用来代替 finalize()？
    👉 可以，用虚引用 + ReferenceQueue 实现更安全的资源回收通知机制。
