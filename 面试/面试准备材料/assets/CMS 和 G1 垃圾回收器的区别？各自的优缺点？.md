## 📌 面试题：CMS 和 G1 垃圾回收器的区别？各自的优缺点？

### ⭐ 标准回答

### 1. **CMS（Concurrent Mark-Sweep）回收器**

- **特点**
  - 基于 **标记-清除算法**。
  - 追求 **最短停顿时间**，适合对 **响应速度要求高** 的应用（如 Web 服务器）。
- **工作过程**
  1. 初始标记（STW，标记 GC Roots 直接可达对象）。
  2. 并发标记（GC 线程和用户线程并发执行，标记可达对象）。
  3. 重新标记（STW，修正并发期间变动的对象）。
  4. 并发清除（清理垃圾对象，和用户线程并发执行）。
- **优点**
   ✅ 并发收集，停顿时间短。
- **缺点**
   ❌ 采用标记-清除，容易产生 **内存碎片**。
   ❌ 需要更多 CPU 资源。
   ❌ 在高并发场景下可能发生 **“Concurrent Mode Failure”**（老年代空间不足触发 Full GC）。

------

### 2. **G1（Garbage-First）回收器**

- **特点**
  - JDK7u4 引入，JDK9 以后成为默认 GC。
  - 将堆划分为多个大小相等的 **Region**，而不是固定的新生代/老年代。
  - 基于 **标记-整理+复制算法**，避免碎片。
  - 通过 **优先回收垃圾最多的 Region**（Garbage-First），提升效率。
- **工作过程**
  1. 初始标记（STW）。
  2. 并发标记（并发追踪存活对象）。
  3. 最终标记（STW，处理少量遗留对象）。
  4. 筛选回收（按 Region 的垃圾价值回收，采用复制算法，避免碎片）。
- **优点**
   ✅ 支持 **可预测停顿时间**（可配置 `-XX:MaxGCPauseMillis`）。
   ✅ 无内存碎片（标记-整理 + 复制）。
   ✅ 更适合大内存（>4GB）场景。
- **缺点**
   ❌ 实现复杂，调优难度高。
   ❌ 在小内存场景下性能不一定优于 CMS。

------

### 📊 对比总结表

| 对比项       | CMS                | G1                       |
| ------------ | ------------------ | ------------------------ |
| **算法**     | 标记-清除          | 标记-整理 + 复制         |
| **堆划分**   | 新生代 / 老年代    | Region 化，逻辑分代      |
| **停顿时间** | 停顿短，但不可预测 | 可预测（可指定停顿时间） |
| **碎片问题** | 有碎片             | 无碎片                   |
| **适用场景** | 中小内存，低延迟   | 大内存，可预测停顿       |

------

### 🔍 面试追问

1. 为什么说 G1 比 CMS 更适合大内存？
    👉 CMS 扫描和清理时代价过高，G1 的 Region + 并行策略更高效。
2. 如果 CMS 产生碎片，如何解决？
    👉 使用 `-XX:+UseCMSCompactAtFullCollection`，Full GC 时整理内存。
3. JDK9 默认 GC 为什么选择 G1？
    👉 G1 综合性能更优，支持大堆，可预测停顿，能替代 CMS。
