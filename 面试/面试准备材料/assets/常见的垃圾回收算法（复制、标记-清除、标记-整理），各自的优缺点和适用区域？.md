# 常见的垃圾回收算法（复制、标记-清除、标记-整理），各自的优缺点和适用区域？

## 一、垃圾回收算法总览

| 算法 | 核心思想 | 优点 | 缺点 | 适用区域 |
|------|---------|------|------|----------|
| **复制算法** | 将内存分为两块，复制存活对象 | 无内存碎片，效率高 | 内存利用率低 | 新生代 |
| **标记-清除算法** | 标记死亡对象，直接清除 | 内存利用率高 | 有内存碎片，效率低 | 老年代 |
| **标记-整理算法** | 标记存活对象，整理到一端 | 无内存碎片，内存利用率高 | 效率低 | 老年代 |

## 二、详细算法分析

### 2.1 复制算法（Copying）

#### 2.1.1 工作原理

1. **内存划分**：将内存分为大小相等的两块（From空间和To空间）
2. **标记阶段**：标记From空间中的存活对象
3. **复制阶段**：将存活对象复制到To空间
4. **清理阶段**：清空From空间
5. **交换角色**：From空间和To空间互换角色

**示意图**：
```
┌─────────────┐     ┌─────────────┐
│   From区    │     │   To区      │
│  (使用中)   │     │  (空闲)     │
└─────────────┘     └─────────────┘
       ↓ 复制存活对象       ↓
┌─────────────┐     ┌─────────────┐
│   From区    │     │   To区      │
│  (空闲)     │     │  (使用中)   │
└─────────────┘     └─────────────┘
```

#### 2.1.2 代码模拟

```java
public class CopyingGC {
    private static final int MEMORY_SIZE = 10;
    private Object[] fromSpace = new Object[MEMORY_SIZE];
    private Object[] toSpace = new Object[MEMORY_SIZE];
    
    public void gc() {
        int toIndex = 0;
        // 标记并复制存活对象
        for (int i = 0; i < MEMORY_SIZE; i++) {
            if (fromSpace[i] != null && isAlive(fromSpace[i])) {
                toSpace[toIndex++] = fromSpace[i];
            }
        }
        // 清空From空间
        fromSpace = new Object[MEMORY_SIZE];
        // 交换角色
        Object[] temp = fromSpace;
        fromSpace = toSpace;
        toSpace = temp;
    }
    
    private boolean isAlive(Object obj) {
        // 简化判断，实际由GC根可达性分析
        return true;
    }
}
```

#### 2.1.3 优缺点

**优点**：
- **无内存碎片**：复制后内存连续
- **效率高**：只处理存活对象
- **实现简单**：逻辑清晰

**缺点**：
- **内存利用率低**：只有50%的内存可用
- **复制开销**：需要复制存活对象

**适用区域**：新生代（对象存活率低）

### 2.2 标记-清除算法（Mark-Sweep）

#### 2.2.1 工作原理

1. **标记阶段**：从GC Root开始，标记所有可达对象
2. **清除阶段**：清除所有未标记的对象

**示意图**：
```
┌─────┬─────┬─────┬─────┬─────┐
│  A  │  B  │  C  │  D  │  E  │  初始状态
└─────┴─────┴─────┴─────┴─────┘
       ↓ 标记存活对象（A、C、E）
┌─────┬─────┬─────┬─────┬─────┐
│  A* │  B  │  C* │  D  │  E* │  标记后
└─────┴─────┴─────┴─────┴─────┘
       ↓ 清除未标记对象
┌─────┬─────┬─────┬─────┬─────┐
│  A  │     │  C  │     │  E  │  清除后（有碎片）
└─────┴─────┴─────┴─────┴─────┘
```

#### 2.2.2 代码模拟

```java
public class MarkSweepGC {
    private static final int MEMORY_SIZE = 10;
    private Object[] memory = new Object[MEMORY_SIZE];
    
    public void gc() {
        // 1. 标记阶段
        boolean[] marked = new boolean[MEMORY_SIZE];
        for (int i = 0; i < MEMORY_SIZE; i++) {
            if (memory[i] != null && isAlive(memory[i])) {
                marked[i] = true;
            }
        }
        // 2. 清除阶段
        for (int i = 0; i < MEMORY_SIZE; i++) {
            if (!marked[i]) {
                memory[i] = null;
            }
        }
    }
    
    private boolean isAlive(Object obj) {
        // 简化判断，实际由GC根可达性分析
        return true;
    }
}
```

#### 2.2.3 优缺点

**优点**：
- **内存利用率高**：不需要额外空间
- **适用于大对象**：避免了复制大对象的开销

**缺点**：
- **内存碎片**：清除后会产生内存碎片
- **效率低**：需要标记和清除两个阶段
- **停顿时间长**：需要遍历所有对象

**适用区域**：老年代（对象存活率高）

### 2.3 标记-整理算法（Mark-Compact）

#### 2.3.1 工作原理

1. **标记阶段**：从GC Root开始，标记所有可达对象
2. **整理阶段**：将存活对象压缩到内存的一端
3. **清理阶段**：清除边界外的内存

**示意图**：
```
┌─────┬─────┬─────┬─────┬─────┐
│  A  │  B  │  C  │  D  │  E  │  初始状态
└─────┴─────┴─────┴─────┴─────┘
       ↓ 标记存活对象（A、C、E）
┌─────┬─────┬─────┬─────┬─────┐
│  A* │  B  │  C* │  D  │  E* │  标记后
└─────┴─────┴─────┴─────┴─────┘
       ↓ 整理存活对象
┌─────┬─────┬─────┬─────┬─────┐
│  A  │  C  │  E  │     │     │  整理后（无碎片）
└─────┴─────┴─────┴─────┴─────┘
```

#### 2.3.2 代码模拟

```java
public class MarkCompactGC {
    private static final int MEMORY_SIZE = 10;
    private Object[] memory = new Object[MEMORY_SIZE];
    
    public void gc() {
        // 1. 标记阶段
        boolean[] marked = new boolean[MEMORY_SIZE];
        int aliveCount = 0;
        for (int i = 0; i < MEMORY_SIZE; i++) {
            if (memory[i] != null && isAlive(memory[i])) {
                marked[i] = true;
                aliveCount++;
            }
        }
        
        // 2. 整理阶段
        int compactIndex = 0;
        for (int i = 0; i < MEMORY_SIZE; i++) {
            if (marked[i]) {
                memory[compactIndex++] = memory[i];
            }
        }
        
        // 3. 清理阶段
        for (int i = compactIndex; i < MEMORY_SIZE; i++) {
            memory[i] = null;
        }
    }
    
    private boolean isAlive(Object obj) {
        // 简化判断，实际由GC根可达性分析
        return true;
    }
}
```

#### 2.3.3 优缺点

**优点**：
- **无内存碎片**：整理后内存连续
- **内存利用率高**：不需要额外空间
- **适用于大对象**：避免了复制大对象的开销

**缺点**：
- **效率低**：需要标记、整理、清理三个阶段
- **停顿时间长**：需要移动存活对象
- **实现复杂**：逻辑较复杂

**适用区域**：老年代（对象存活率高）

## 三、算法对比分析

### 3.1 时间复杂度

| 算法 | 标记时间 | 清理时间 | 总时间复杂度 |
|------|---------|---------|-------------|
| 复制算法 | O(n) | O(k)（k为存活对象数） | O(n) |
| 标记-清除 | O(n) | O(n) | O(n) |
| 标记-整理 | O(n) | O(n) | O(n) |

### 3.2 空间复杂度

| 算法 | 额外空间 | 内存利用率 |
|------|---------|-----------|
| 复制算法 | O(n) | 50% |
| 标记-清除 | O(1) | 100%（但有碎片） |
| 标记-整理 | O(1) | 100% |

### 3.3 适用场景

| 算法 | 适用对象存活率 | 适用内存区域 | 适用垃圾回收器 |
|------|----------------|--------------|----------------|
| 复制算法 | 低（<10%） | 新生代 | Serial GC、Parallel GC、G1 GC |
| 标记-清除 | 高（>50%） | 老年代 | CMS GC |
| 标记-整理 | 高（>50%） | 老年代 | Serial Old GC、Parallel Old GC、G1 GC |

## 四、医疗美容系统中的应用

### 4.1 新生代垃圾回收

**场景**：系统处理大量短期对象（如HTTP请求、临时对象）

**选择算法**：复制算法

**原因**：
- 短期对象存活率低，复制成本小
- 无内存碎片，分配效率高
- 停顿时间短，响应速度快

**参数调优**：
```bash
# 设置新生代大小和比例
java -XX:NewSize=512m -XX:MaxNewSize=1024m -XX:SurvivorRatio=8 YourApp
```

### 4.2 老年代垃圾回收

**场景**：系统处理长期对象（如缓存、单例对象）

**选择算法**：标记-清除或标记-整理

**原因**：
- 长期对象存活率高，复制成本大
- 内存利用率高，适合大对象
- 可根据具体场景选择合适的算法

**参数调优**：
```bash
# 使用CMS GC（标记-清除）
java -XX:+UseConcMarkSweepGC -XX:CMSInitiatingOccupancyFraction=75 YourApp

# 使用G1 GC（标记-整理）
java -XX:+UseG1GC -XX:MaxGCPauseMillis=200 YourApp
```

## 五、垃圾回收器与算法的关系

### 5.1 各垃圾回收器使用的算法

| 垃圾回收器 | 新生代算法 | 老年代算法 |
|------------|------------|------------|
| Serial GC | 复制算法 | 标记-整理算法 |
| Parallel GC | 复制算法 | 标记-整理算法 |
| CMS GC | 复制算法 | 标记-清除算法 |
| G1 GC | 复制算法 | 标记-整理算法 |
| ZGC | 标记-整理算法（区域化） | 标记-整理算法（区域化） |

### 5.2 算法选择的影响因素

1. **对象生命周期**：短期对象用复制算法，长期对象用标记-清除/整理算法
2. **内存大小**：小堆用Serial GC，大堆用G1 GC
3. **响应时间要求**：低延迟用CMS GC或G1 GC，高吞吐量用Parallel GC
4. **硬件资源**：多核CPU用并行回收器，单核CPU用Serial GC

## 六、面试标准回答（2分钟）

「常见的垃圾回收算法有复制算法、标记-清除算法和标记-整理算法。复制算法将内存分为两块，复制存活对象，无内存碎片但内存利用率低，适合新生代；标记-清除算法标记死亡对象后直接清除，内存利用率高但有碎片，适合老年代；标记-整理算法标记存活对象后整理到一端，无碎片且内存利用率高，但效率低，适合老年代。在医疗美容系统中，新生代使用复制算法处理短期对象，老年代使用标记-清除或标记-整理算法处理长期对象，根据系统特点选择合适的垃圾回收器和算法。」

## 七、常见追问

**Q1：为什么新生代使用复制算法？**

因为新生代对象存活率低（80-90%的对象会被回收），复制算法只需要复制少量存活对象，效率高，且无内存碎片。

**Q2：为什么老年代不使用复制算法？**

因为老年代对象存活率高，复制算法需要复制大量对象，开销大，且内存利用率低（只有50%）。

**Q3：标记-清除算法的内存碎片问题如何解决？**

可以使用标记-整理算法或CMS GC的并发标记-清除算法，或者定期进行Full GC来整理内存碎片。

**Q4：G1 GC使用什么算法？**

G1 GC采用区域化内存管理，新生代使用复制算法，老年代使用标记-整理算法，结合了多种算法的优点。

| 算法 | 核心思想 | 优点 | 缺点 | 适用区域 | 代表回收器 |
|------|---------|------|------|----------|------------|
| 复制算法 | 复制存活对象 | 无碎片，效率高 | 内存利用率低 | 新生代 | Serial、Parallel、G1 |
| 标记-清除 | 清除死亡对象 | 内存利用率高 | 有碎片，效率低 | 老年代 | CMS |
| 标记-整理 | 整理存活对象 | 无碎片，利用率高 | 效率低，停顿长 | 老年代 | Serial Old、G1 |
