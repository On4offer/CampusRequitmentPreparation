# 面试题：MySQL 的存储引擎有哪些？InnoDB 和 MyISAM 有什么区别？

------

## 1. 概念

- **存储引擎**：MySQL 数据表的底层实现方式，不同存储引擎在事务、锁、索引等机制上有所不同。
- MySQL 常见存储引擎：
  1. **InnoDB**（默认，支持事务）
  2. **MyISAM**（早期默认，不支持事务）
  3. **Memory**（数据存储在内存中，速度快，适合临时计算，临时表中间结果表）
  4. **Archive**（只支持 INSERT 和 SELECT，适合日志归档）
  5. **NDB Cluster**（分布式引擎，电信级应用）

------

## 2. 原理 & 区别

### 🔹 InnoDB

- 支持 **事务（ACID）**
- 支持 **行级锁**（并发高）
- 支持 **MVCC（多版本并发控制）**
- 支持 **外键约束**
- **聚簇索引**：数据与主键索引放在一起

### 🔹 MyISAM

- 不支持事务、外键
- 只支持 **表级锁**（并发差）
- **非聚簇索引**：索引和数据分开存储
- 读写性能偏向 **读多写少** 的场景
- 崩溃后数据恢复能力差（没有 Redo Log）

------

## 3. 案例

- **电商下单系统**：需要事务保证库存扣减和订单一致性 → **InnoDB**
- **日志查询系统**：写入少、读多、且对事务要求不高 → **MyISAM/Archive**
- **临时排名表**：频繁计算，数据不重要 → **Memory**

------

## 4. 使用场景

- InnoDB：金融、电商、订单、支付等 **强一致性场景**。
- MyISAM：全文检索、日志系统、只读应用（如报表）。
- Memory：缓存类数据（快速但断电丢失）。

------

## 5. 面试模板回答

> MySQL 常见存储引擎有 InnoDB、MyISAM、Memory、Archive 等，其中 InnoDB 是默认引擎。
>  它和 MyISAM 的主要区别在于：
>
> - InnoDB 支持事务、行锁、外键，具备更强的并发性能和数据安全性；
> - MyISAM 不支持事务和行锁，只能表锁，适合读多写少的场景。
>    因此，线上业务几乎都使用 InnoDB。
