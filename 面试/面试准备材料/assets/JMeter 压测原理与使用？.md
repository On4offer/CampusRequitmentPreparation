## 面试题：如何用 JMeter 进行压力测试？

### 1. 概念

**JMeter** 是 Apache 提供的开源性能测试工具，支持 HTTP、HTTPS、JDBC、FTP、MQ 等多种协议。它可以模拟并发用户，对系统进行 **压力测试、负载测试和性能分析**，帮助定位性能瓶颈。

------

### 2. 压测原理

- **模拟并发**：通过 **线程组（Thread Group）**，设定并发用户数、循环次数、Ramp-Up 时间（启动速率）。
- **请求模拟**：使用 **取样器（Sampler）** 发送 HTTP 请求、数据库请求等。
- **数据收集**：通过 **监听器（Listener）** 收集响应时间、吞吐量、错误率等指标。
- **断言验证**：使用 **断言（Assertion）** 判断接口返回是否符合预期。
- **结果分析**：生成图表和报告，用于性能瓶颈分析。

------

### 3. 使用场景

- **压力测试**：验证系统在高并发下是否能稳定运行。
- **负载测试**：确定系统的最大承载能力。
- **稳定性测试**：长时间运行，观察内存、CPU、连接数变化。
- **数据库/接口瓶颈分析**：发现 SQL 慢查询或接口性能问题。

------

### 4. 项目实践（苍穹外卖 / 黑马点评 示例）

- **苍穹外卖**：
   模拟 1000 个用户同时下单，验证系统是否能抗住高峰流量，重点观察 **订单接口的响应时间和失败率**。
- **黑马点评**：
   对商户查询接口进行压测，设置线程数 500，循环 10 次，验证缓存是否能有效降低数据库压力。

------

### 5. 面试模板回答

> JMeter 是常见的性能测试工具，用来做压力测试的步骤大致如下：
>
> 1. 创建 **测试计划**，添加 **线程组**，设定并发用户数和请求次数。
> 2. 添加 **HTTP 请求取样器**，配置接口地址、请求参数。
> 3. 设置 **监听器**（如查看结果树、聚合报告、图形结果），收集性能数据。
> 4. 添加 **断言**，校验返回结果是否正确。
> 5. 运行测试并分析响应时间、吞吐量、错误率等指标，从而判断系统的性能瓶颈。
>     在实际项目中，比如苍穹外卖下单接口，我会用 JMeter 模拟并发请求，验证在秒杀活动中是否存在超卖或接口超时问题。
