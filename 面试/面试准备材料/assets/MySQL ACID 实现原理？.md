# 面试题：MySQL 的事务特性 ACID 分别指什么？是如何实现的？

------

## 1. 概念（ACID 四大特性）

1. **A - 原子性（Atomicity）**
   - 事务中的所有操作要么全部执行成功，要么全部不执行。
   - 不能出现“扣了钱但没加钱”的情况。
2. **C - 一致性（Consistency）**
   - 事务执行前后，数据必须满足完整性约束。
   - 例如转账完成后，A+B 的总金额不变。
3. **I - 隔离性（Isolation）**
   - 多个事务并发执行时，互不干扰。
   - 通过隔离级别控制（如可重复读、读已提交）。
4. **D - 持久性（Durability）**
   - 一旦事务提交，结果永久写入数据库，即使宕机也不会丢失。

------

## 2. 实现原理

1. **原子性**
   - 依赖 **Undo Log（回滚日志）**。
   - 事务失败时，可以通过 Undo Log 回滚到事务开始前的状态。
2. **一致性**
   - 依赖 **约束（主键、外键、唯一约束）+ 原子性、隔离性保证**。
   - 保证数据不出现逻辑上的错误。
3. **隔离性**
   - 依赖 **锁机制（行锁、表锁、间隙锁）+ MVCC（多版本并发控制）**。
   - 不同隔离级别决定读写冲突时的处理方式。
4. **持久性**
   - 依赖 **Redo Log（重做日志）+ Binlog（归档日志）**。
   - Redo Log 保证数据落盘，Binlog 用于主从复制和灾备。

------

## 3. 案例

**银行转账：**
 用户 A 向 B 转账 100 元：

- A 扣 100
- B 加 100
- 提交事务
   保证 **要么同时成功，要么同时失败**，不可能只扣不加。

------

## 4. 使用场景

- **转账、支付**：高要求一致性。
- **库存扣减**：避免超卖。
- **订单处理**：持久化日志 + 回滚能力保证可靠。

------

## 5. 面试模板回答

> MySQL 事务具备 **ACID** 特性：
>
> - 原子性由 Undo Log 保证，
> - 一致性依靠约束 + 事务机制，
> - 隔离性由锁机制和 MVCC 实现，
> - 持久性依赖 Redo Log 和 Binlog。
>    InnoDB 引擎是 MySQL 默认存储引擎，通过日志系统和并发控制机制保证 ACID。
