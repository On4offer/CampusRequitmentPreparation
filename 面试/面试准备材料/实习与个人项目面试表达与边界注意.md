## 一、总原则（两条线都要）

| | **实习（医美 HIS）** | **个人项目（小众点评）** |
|---|----------------------|---------------------------|
| **定位** | 遗留系统里**迭代**，小步交付 | 学习/练手、**仿业务场景**，不是商业线上服务 |
| **原则** | 如实说约束（文档少、双后端） | 如实说环境（本机/压测），**数字不吹成线上大盘** |

## 二、实习：项目环境怎么说（不抱怨、不甩锅）

### 建议用语（中性）

- 「遗留系统」「历史包袱」「迭代节奏紧」「文档当时不完整」
- 「需求主要靠**问同事、跟代码、对接口**」
- 「改动尽量**局部、可回滚**，对齐现有技术栈」

### 避免

- 对面试官**大量**情绪化吐槽；只骂环境、**不提自己做了什么**

### 收尾句式

「环境是这样，但我在这个约束下完成了 **A / B / C**（对应简历或 `实习产出/`），如果文档更全，迭代会更快。」

## 三、实习：技术点如实说 + 一句边界（§3.1～3.16）

### 3.1 密码与加密（`强制更新密码.md`）

- **事实**：与现有体系一致时可能仍用 **MD5**。
- **别说**：「MD5 最安全」。
- **推荐**：「老账号体系已这样存，本次在**不改算法**的前提下做过期与流程；新系统一般会 **BCrypt / Argon2**。」

### 3.2 匿名改密

- 依赖 **HTTPS、频控、防爆破**；可补「生产上会限流/验证码/审计」。

### 3.3 Excel 导入（`商品信息excel导入.md`）

- 名称+规格为主；可加**编码列**减少歧义；大数据量→批量查库/流式，见产出复盘。
- **不吹**高并发。

### 3.4 审计异步（`日志审计.md`）

- 不阻塞业务 vs 极端情况日志可能失败；可补重试/队列/关键操作同步记。

### 3.5 双后端 .NET + Java

- 按模块 **BaseURL**；**别**说成「我设计了双后端架构」。

### 3.6 导入 vs 保存落库

- 解析返回与**正式入库**可能两步；**别**说成点导入瞬间全部落库（以代码为准）。

### 3.7 POI 与大文件

- 有 **10MB/行数** 语境；超大→SXSSF/EasyExcel/异步。

### 3.8 审计异步 + ThreadLocal

- 异步线程**拿不到**请求线程 ThreadLocal；理想是**主线程取出 userId 再异步**。

### 3.9 拦截器范围

- 多**排除**查询类路径；**别**答「每个请求都记审计」。

### 3.10 总部 vs 门店

- URL/权限可能不同；**别**混成一个接口。

### 3.11 SQL 注入

- 说**参数绑定**；**别**说「框架绝对防注入」。

### 3.12 Bug 范围

- 只讲**产出文档里**的报表/推荐/入库；**别**夸大成重写体系统。

### 3.13 幂等与连点

- 以当时实现为准；理想：**防抖 + 幂等键**。

### 3.14 合规 / 等保

- 说**业务审计要求**；**别**乱对等保级别。

## 四、个人项目（小众点评）：定调 + 环境

### 4.1 怎么说「这是什么项目」

- **推荐**：「仿大众点评场景的**学习项目**，主要练 **Spring Boot + Redis + MQ + 并发与一致性**。」
- **避免**：「线上服务、服务百万用户、商业项目」——除非真有。

### 4.2 简历上的数字（与实习 §3.13 呼应，但**个人项目更要强调**）

简历中如：**查询减少约 97%、命中率 95%、TPS 约 15 倍、响应时间约降 94%、Token &lt;5ms、超卖/重复下单 0、万级粉丝 1 秒内触达**等。

- **统一口径**：**本地或本机压测 / 测试环境**，**非**长期线上监控、**非**多机房真实流量。
- **一句边界**：「数字用来对比**优化前后**或**压测脚本下的相对效果**；**不能**等同于生产 SLA。」

### 4.3 与教程/课程的关系（若被问）

- 基于常见 **Redis / 秒杀 / MQ** 教程拓展时：**可以说**跟做过哪些、**自己改了哪些**（如异常回滚、参数调优）。
- **诚实**优于装「完全原创」。

## 五、个人项目：技术点如实说 + 一句边界

### 5.1 缓存（店铺 Cache-Aside、穿透/击穿/雪崩）

- **怎么说**：能说清**概念 + 你项目里怎么落地**（逻辑过期、空值、动态 TTL）。
- **边界**：**缓存与 DB 一致性**是经典难题；**别说**「强一致永远无延迟」——通常是**短不一致可接受 + TTL + 删缓存策略**。

### 5.2 Token + ThreadLocal + 滑动过期

- **怎么说**：拦截器校验 Redis、续期、ThreadLocal 存用户上下文。
- **边界**：**线程池 / 异步 / @Async** 里 ThreadLocal **可能失效或串**——若被问，答「同步请求内可用；异步要**显式传 userId**」或「以代码为准」。

### 5.3 秒杀：Lua + Redisson + 乐观锁 + 0% 超卖/重复下单

- **怎么说**：Redis 里**原子校验/预扣**，MQ **异步落库**，DB **乐观锁**防超卖。
- **边界**：  
  - 「0%」指**你压测场景**下未复现；**别**说成数学上绝对、全宇宙任何流量。  
  - 能说清 **MQ 发送失败是否回滚 Redis**（若你实现了）——体现一致性意识。

### 5.4 RocketMQ（解耦、削峰）

- **怎么说**：接口快速返回，订单**异步**写库，**削峰**。
- **边界**：  
  - 一般是**最终一致**；**别**吹成「分布式强一致事务」除非真用事务消息且能说清。  
  - **重试、死信、消息堆积**可被追问——准备一两句。

### 5.5 Feed 流 ZSet 推模式（时间戳 score、滚动分页）

- **怎么说**：发笔记时**写粉丝收件箱** ZSet，拉取时按 score 分页。
- **边界（重要）**：  
  - 「万级粉丝、1 秒触达」多为**简化/压测语境**；**别**包装成已解决**千万粉大 V** 的读写扩散。  
  - 大 V 场景标准答法：「推模式会**写放大**；业界会**推拉结合、分桶、限流**等，我项目里**未完整做**，知道问题在哪。」

### 5.6 Redis 部署与规模

- **简历未写集群**时：**别**主动吹「集群、哨兵我都线上配过」——除非你真做了。单机/本机 **Redis** 练手完全够用。

### 5.7 JVM / 内存 / 连接池

- 个人项目被问 OOM、连接数：**可以说**本机 `-Xmx`、连接池配置、**压测时注意**——不必装线上调优专家。
