# 出库类型筛选功能开发文档

## 一、需求分析

### 1.1 业务需求
在库存管理 > 出库管理页面，增加“出库类型”筛选，支持按类型（领用/销售/盘盈）筛选出库记录。

### 1.2 功能要求
- 筛选框：下拉选择，包含“领用出库”“销售出库”“盘盈出库”
- 筛选逻辑：选择类型后仅显示对应类型的出库记录
- 清除筛选：支持清除筛选条件，显示全部记录
- 表格显示：表格中显示“出库类型”列

### 1.3 出库类型枚举值
根据代码分析，出库类型枚举值如下：
- 0：领用出库
- 1：销售出库
- 2：调整出库
- 3：盘盈出库
- 4：调拨出库
- 5：拆包出库

本次需求仅涉及：领用出库(0)、销售出库(1)、盘盈出库(3)

## 二、技术架构分析

### 2.1 项目结构
系统采用前后端分离架构：
- 前端：Vue.js + Element UI
- 后端：Java Spring Boot + C# ASP.NET Web API（双后端）

### 2.2 接口调用关系
前端实际调用的是 C# 后端的接口：
- 前端 API：`/repertory/getrepoutlist` (GET)
- C# 后端：`RepertoryController.GetRepOutList`
- Java 后端：`RepertoryoutController.index`（备用，本次也同步修改）

### 2.3 数据库表结构
主表：`tbl_repertoryout`
- 字段：`OutRepType` (int(11) NOT NULL DEFAULT 0)
- 存储出库类型枚举值

## 三、实现思路

### 3.1 前端实现
1. 在筛选区域添加“出库类型”下拉框
2. 在数据模型中添加 `outRepType` 字段
3. 查询时将 `outRepType` 作为参数传递给后端
4. 表格列标题改为“出库类型”

### 3.2 后端实现
1. C# 后端：
   - 在 `RepertoryOutRequestModel` 中添加 `OutRepType` 属性
   - 在 `RepertoryOutServiceImp.GetRepertoryOutList` 中添加筛选逻辑
2. Java 后端（同步修改）：
   - 在 `RepertoryoutController` 的三个接口中添加筛选逻辑
   - `/index`：列表查询
   - `/itemindex`：明细查询
   - `/itemindexExport`：导出功能

### 3.3 筛选逻辑设计
- 当 `outRepType` 有值时，添加筛选条件
- 当 `outRepType` 为空/未选择时，不添加筛选条件，显示全部
- 使用条件判断，避免影响现有功能

### 4.3 Java 后端代码修改

// 出库类型筛选
int outRepType = ActionUtils.strToInt(request.getParameter("outRepType"), -1);
if (outRepType >= 0) {
    hqlFilter.addFilter("QUERY_t#outRepType_L_EQ", String.valueOf(outRepType));

- 使用 HQL 过滤器：`QUERY_t#outRepType_L_EQ`（L_EQ 表示 Long 类型等于）

## 五、关键技术点

### 5.1 参数传递
- 前端：通过 GET 请求的 `params` 传递 `outRepType`
- C# 后端：通过模型绑定自动映射到 `RepertoryOutRequestModel.OutRepType`
- Java 后端：通过 `request.getParameter("outRepType")` 获取

### 5.2 筛选条件构建
- C#：使用 LINQ 的 `WhereIf` 扩展方法
- Java：使用自定义的 `HqlFilter` 工具类

### 5.3 兼容性处理
- 未选择类型时，不传参数或传 `null`/`undefined`
- 后端判断参数是否存在，不存在时不添加筛选条件
- 确保不影响现有功能
