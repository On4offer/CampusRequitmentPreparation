## 1. 需求分析

### 1.1 业务需求
为满足审计要求，系统需记录：
- 每个用户的登录日志
- 每个用户的操作日志（编辑、删除、下载、导出等）

### 1.2 功能需求

#### 1.2.1 登录日志记录
- 记录字段：
  - 操作人ID
  - 操作人姓名
  - 登录账号
  - 操作人门店ID
  - 门店名称
  - 公司ID
  - 登录时间
  - 登录类型（成功/失败/登出）
  - 登录IP地址
  - 浏览器信息
  - 失败原因（登录失败时）

#### 1.2.2 操作日志记录
- 记录字段：
  - 操作人ID
  - 操作人姓名
  - 操作人门店ID
  - 门店名称
  - 公司ID
  - 操作类型（新增/编辑/删除/导出/下载/上传）
  - 功能接口路径
  - 操作描述
  - 操作时间
  - 操作结果（成功/失败）

### 1.3 非功能需求
- 性能：日志记录不阻塞主流程（异步）
- 可靠性：日志记录失败不影响业务
- 安全性：敏感信息脱敏（密码、token等）
- 可维护性：代码结构清晰，易于扩展

## 2. 方案设计

### 2.1 技术方案对比

#### 方案一：AOP切面
- 优点：代码侵入小，可精确控制
- 缺点：需引入AOP依赖，配置复杂

#### 方案二：Spring拦截器（最终选择）
- 优点：
  - 无需新依赖
  - 与现有拦截器模式一致
  - 配置简单
  - 可统一处理HTTP请求
- 缺点：需配置拦截规则

### 2.2 架构设计

#### 2.2.1 系统架构
```
┌─────────────────┐
│   前端应用      │
└────────┬────────┘
         │
    ┌────┴────┐
    │        │
┌───▼───┐ ┌──▼──────┐
│.NET   │ │ Java    │
│后端   │ │ 后端    │
└───┬───┘ └──┬──────┘
    │        │
    │        │
    └───┬────┘
        │
┌───────▼────────┐
│   MySQL数据库   │
│  sys_login_log  │
│sys_operation_log│
└─────────────────┘
```

#### 2.2.2 登录日志流程
```
.NET后端登录接口
    │
    ├─→ 登录成功/失败
    │
    ├─→ LoginLogHelper.RecordLoginSuccess/Fail
    │
    ├─→ HTTP POST → Java后端 /syslog/loginLog/record
    │
    ├─→ SysLogController.recordLoginLog
    │
    ├─→ LoginLogUtils.recordLoginSuccess/Fail/Logout
    │
    └─→ 异步保存到 sys_login_log 表
```

#### 2.2.3 操作日志流程
```
用户操作请求
    │
    ├─→ OperationLogInterceptor.preHandle (记录开始时间)
    │
    ├─→ Controller处理业务逻辑
    │
    ├─→ OperationLogInterceptor.afterCompletion
    │
    ├─→ 判断是否需要记录日志
    │
    ├─→ 提取用户信息、操作信息
    │
    └─→ 异步保存到 sys_operation_log 表
```

## 3. 详细逻辑设计

### 3.1 登录日志记录逻辑

#### 3.1.1 登录成功记录
```
1. .NET后端登录成功
2. 提取用户信息（EmpId, Account, ClinicId, CompId）
3. 获取IP地址和User-Agent
4. 调用 LoginLogHelper.RecordLoginSuccess()
5. 异步发送HTTP请求到Java后端
6. Java后端接收请求，调用 LoginLogUtils.recordLoginSuccess()
7. 异步查询员工姓名和门店名称
8. 构建 SysLoginLog 对象
9. 保存到数据库
```

#### 3.1.2 登录失败记录
```
1. .NET后端登录失败
2. 提取账号和失败原因
3. 获取IP地址和User-Agent
4. 调用 LoginLogHelper.RecordLoginFail()
5. 异步发送HTTP请求到Java后端
6. Java后端接收请求，调用 LoginLogUtils.recordLoginFail()
7. 构建 SysLoginLog 对象（不包含员工信息）
8. 保存到数据库
```

#### 3.1.3 登出记录
```
1. .NET后端登出接口
2. 从Token或Session提取用户信息
3. 获取IP地址和User-Agent
4. 调用 LoginLogHelper.RecordLogout()
5. 异步发送HTTP请求到Java后端
6. Java后端接收请求，调用 LoginLogUtils.recordLogout()
7. 异步查询员工姓名和门店名称
8. 构建 SysLoginLog 对象（loginType=3）
9. 保存到数据库
```

### 3.2 操作日志记录逻辑

#### 3.2.1 拦截器工作流程
```
1. preHandle: 记录请求开始时间
2. Controller处理业务逻辑
3. afterCompletion: 记录操作日志
   ├─ 判断是否需要记录（排除查询接口）
   ├─ 获取用户信息（从JWT Token）
   ├─ 判断操作类型（新增/编辑/删除/导出/下载/上传）
   ├─ 获取操作描述
   ├─ 获取IP地址
   ├─ 获取请求参数（脱敏处理）
   ├─ 计算操作耗时
   ├─ 判断操作结果（成功/失败）
   └─ 异步保存到数据库
```

#### 3.2.2 操作类型判断逻辑
```
优先级从高到低：
1. 删除操作：包含 /delete, /remove, /del, /zuofei, /cancel
2. 导出操作：包含 /export
3. 下载操作：包含 /download
4. 上传/导入操作：包含 /upload, /import
5. 编辑操作：包含 /update, /edit, /changeStatus, /set
6. 新增操作：包含 /save, /add, /create
7. 默认：根据HTTP方法判断
   - POST → 新增
   - PUT/PATCH → 编辑
   - DELETE → 删除
```

#### 3.2.3 接口过滤逻辑
```
需要记录的接口：
- 包含操作关键字：/save, /add, /create, /update, /edit, /delete, /remove, 
  /export, /download, /import, /upload, /pay, /refund, /audit, /ruku, 
  /chuku, /gofahuo, /shouhuo, /zuofei, /cancel, /set, /notice 等

排除的接口：
- 查询接口：/index, /show, /get**, /query**, /list**
- 日志查询接口：/syslog/**, /log/**
- 静态资源：/static/**, /css/**, /js/**, /images/**, /fonts/**
- OPTIONS请求
```

#### 3.2.4 IP地址获取逻辑
```
优先级从高到低：
1. X-Forwarded-For 请求头（代理场景）
2. X-Real-IP 请求头
3. Proxy-Client-IP 请求头
4. WL-Proxy-Client-IP 请求头
5. request.getRemoteAddr()（直接连接）
6. 特殊处理：IPv6 localhost (::1, 0:0:0:0:0:0:0:1) → 127.0.0.1
```

#### 3.2.5 请求参数获取逻辑
```
1. 获取URL参数和表单参数（request.getParameterMap()）
   - 跳过敏感字段：password, pwd, token
   - 限制单个参数值长度（200字符）

2. 如果是POST/PUT/PATCH且Content-Type为application/json：
   - 优先从 CourseSignatureRequestWrapper.getBody() 获取
   - 其次从 request.getAttribute("requestBody") 获取
   - 再次从 request.getAttribute("cachedBody") 获取
   - 对JSON进行脱敏处理（password, pwd, token, accessToken）
   - 限制总长度（2000字符）

3. 返回参数字符串或null
```

#### 3.2.6 操作描述生成逻辑
```
1. 先从映射表查找（API_PATH_DESC_MAP）
2. 如果未找到：
   - 提取业务模块名称（从接口路径）
   - 获取操作类型名称（新增/编辑/删除等）
   - 组合生成：模块名-操作类型
   - 例如：预约-新增、账单-编辑
3. 如果无法提取模块名，使用路径特征匹配
4. 默认返回：操作类型-接口路径最后部分
```

**OperationLogInterceptor.java - 核心拦截逻辑**
```java
@Override
public void afterCompletion(HttpServletRequest request, HttpServletResponse response, 
                           Object handler, Exception ex) throws Exception {
    try {
        // 判断是否需要记录日志
        if (!needLog(request, handler)) {
            return;
        }

        // 获取用户信息
        CurrentEmployeeDTO employee = CurrentEmployeeUtils.analysisEmployee(request);
        if (employee == null || employee.getId() == null) {
            return;
        }

        // 异步记录日志
        CompletableFuture.runAsync(() -> {
            try {
                saveOperationLog(request, response, handler, employee, ex);
            } catch (Exception e) {
                log.error("保存操作日志失败", e);
            }
        });
    } catch (Exception e) {
        log.error("操作日志拦截器异常", e);
    }
}
```

public static void RecordLoginSuccess(string empId, string account, string clinicId, 
                                     string compId, string loginIp = null, 
                                     string userAgent = null) {
    Task.Run(() => {
        try {
            string url = $"{JavaApiBaseUrl}/syslog/loginLog/record";
            NameValueCollection data = new NameValueCollection {
                { "empId", empId },
                { "account", account },
                { "clinicId", clinicId },

            SendPostRequest(url, data);
        } catch (Exception ex) {
            logger?.Warn($"记录登录成功日志失败: {ex.Message}", ex);
        }
    });
}
