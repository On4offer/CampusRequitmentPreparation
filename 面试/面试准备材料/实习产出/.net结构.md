
## 项目框架结构详细介绍

这是一个基于 .NET Framework 4.5.2 的企业级 Web API 后端项目，采用 CQRS + DDD + Event Sourcing 架构。

---

## 一、整体架构模式

### 1. 架构模式
- CQRS（命令查询职责分离）：命令与查询分离
- DDD（领域驱动设计）：以领域模型为核心
- Event Sourcing（事件溯源）：通过事件序列重建状态
- 分层架构：清晰的分层与职责划分

### 2. 核心框架
- Apworks：CQRS/ES 框架
- Entity Framework 6：ORM
- Unity：依赖注入容器
- OWIN：中间件管道

---

## 二、项目分层结构

### 1. His.Api（表现层）
职责：HTTP API 入口、路由、认证授权

主要目录：
```
His.Api/
├── Controllers/          # API控制器
│   ├── BaseController.cs      # 基础控制器
│   └── V1/                    # V1版本API控制器（35个控制器）
├── App_Start/            # 启动配置
│   ├── ApworksConfig.cs        # Apworks框架配置
│   ├── WebApiConfig.cs         # Web API路由配置
│   ├── MapperConfig.cs         # AutoMapper配置
│   ├── FilterConfig.cs         # 过滤器配置
│   └── HisAuthAttribute.cs     # 认证授权特性
├── Middleware/           # OWIN中间件
├── Provider/            # OAuth提供者
├── Permission/          # 权限相关
├── Formats/             # 格式化器
└── Validation/          # 验证相关
```

特点：
- 使用 OWIN 启动
- JWT 认证
- 权限控制
- CORS 支持
- WebSocket 实时通信

### 2. His.Application（应用层）
职责：应用服务、业务编排、DTO 转换

核心类：
- `ApplicationService`：应用服务基类
  - 仓储上下文管理
  - 缓存策略（Memcached）
  - 日志记录
  - 会话管理
  - WebSocket 服务

主要功能：
- 业务逻辑编排
- DTO 与领域模型转换（AutoMapper）
- 事务管理
- 缓存操作

### 3. His.Commands（命令层）
职责：定义命令对象

结构：
```
His.Commands/
├── CreateCompayCommand.cs      # 创建公司命令
├── Appointment/                # 预约相关命令
├── Bill/                       # 账单相关命令
└── Course/                     # 课程相关命令
```

特点：
- 命令对象实现 `ICommand`
- 命令包含执行所需数据
- 命令不可变（Immutable）

### 4. His.Commands.Handlers（命令处理层）
职责：处理命令，执行业务逻辑

结构：
```
His.Commands.Handlers/
├── CommandHandlerBase.cs        # 命令处理器基类
├── CreateCompayCommandHandler.cs
├── Appointment/
├── Bill/
└── Course/
```

处理流程：
1. 接收命令
2. 获取领域仓储
3. 执行业务逻辑
4. 保存聚合根
5. 提交事务

### 5. His.Domain（领域层）
职责：核心业务逻辑、领域模型

结构：
```
His.Domain/
├── Model/                      # 领域模型（171个实体）
│   ├── Company.cs                  # 公司聚合根
│   ├── Customer.cs                 # 客户聚合根
│   ├── Employee.cs                 # 员工聚合根
│   ├── Bill.cs                     # 账单聚合根
│   └── Enums/                      # 枚举类型（41个）
├── Repositories/               # 仓储接口（116个接口）
│   ├── ICompanyRepository.cs
│   ├── ICustomerRepository.cs
│   └── ...
└── Events/                     # 领域事件（6个）
```

特点：
- 聚合根实现 `IAggregateRoot`
- 领域事件实现 `IDomainEvent`
- 业务规则封装在领域模型中
- 仓储接口定义在领域层

### 6. His.Domain.Repositories（仓储实现层）
职责：实现数据访问

结构：
```
His.Domain.Repositories/
├── EntityFramework/            # EF实现（240个文件）
│   ├── CompanyRepository.cs
│   ├── CustomerRepository.cs
│   └── ...
├── Specifications/             # 规约模式（42个）
└── Migrations/                # 数据库迁移（175个迁移文件）
```

特点：
- 使用 Entity Framework 6
- MySQL 数据库
- Code First 迁移
- 规约模式封装查询条件

### 7. His.Events.Handlers（事件处理层）
职责：处理领域事件

结构：
```
His.Events.Handlers/
├── CompanyEventHandler.cs
├── CustomerCreateEventHandler.cs
├── Appointment/
├── Bill/
└── Course/
```

事件处理示例：
- 创建客户后发送邮件
- 订单创建后更新库存
- 异步处理耗时操作

### 8. His.Infrastructure（基础设施层）
职责：基础设施实现

核心类：
- `HisCQRSEventSourceDomainRepository`：事件溯源仓储实现
  - 继承 `EventSourcedDomainRepository`
  - 集成事件存储和事件总线
  - 支持快照

### 9. His.Common（公共层）
职责：通用工具和基础设施

主要模块：
```
His.Common/
├── Cache/                     # 缓存策略（7个文件）
│   └── MemCached/                # Memcached实现
├── Log/                       # 日志（log4net）
├── JWT/                       # JWT令牌处理（3个文件）
├── WebSocket/                 # WebSocket服务（8个文件）
├── Permission/                # 权限管理（15个文件）
├── Exception/                 # 异常处理（8个文件）
├── Enums/                     # 枚举（9个文件）
├── Model/                     # 通用模型（135个文件）

└── Extensions/                # 扩展方法（5个文件）
```

### 10. His.DataObjects（数据传输对象层）
职责：DTO 定义

结构：
- 96 个 DTO 类
- 用于 API 数据传输
- 与领域模型分离

### 11. His.Services（服务层）
职责：业务服务接口与实现

结构：
- 70 个服务类
- 封装复杂业务逻辑
- 供应用层调用

### 12. His.Subscribe.SynchronizationService（订阅服务）
职责：事件订阅与同步

功能：
- 订阅领域事件
- 同步数据到其他系统
- 后台服务

## 三、核心技术组件

### 1. Apworks 框架集成
- Command Bus：`DirectCommandBus`
- Event Bus：`MSMQEventBus`
- Event Storage：`MySqlDomainEventStorage`
- Snapshot Provider：`EventNumberSnapshotProvider`
- Domain Repository：`HisCQRSEventSourceDomainRepository`

### 2. 数据访问
- ORM：Entity Framework 6
- 数据库：MySQL
- 连接字符串：`HisDb`
- 事件存储：`his_eventdb`

### 3. 依赖注入
- 容器：Unity
- 生命周期：PerRequestLifetimeManager（请求级别）
- 注册位置：`ApworksConfig.cs`

### 4. 缓存
- 策略：Memcached
- 接口：`ICacheStrategy`
- 实现：`MemCachedStrategy`

### 5. 日志
- 框架：log4net
- 配置：`log4net.config`
- 工厂：`Log4NetLoggerFactory`

### 6. 认证授权
- JWT
- OAuth 2.0
- 权限管理：`PermissionManager`
- 特性：`HisAuthAttribute`

### 7. 实时通信
- WebSocket（WebSocketSharp）
- 服务：`HisWebSocketServer`
- 消息推送：`MessagePush`

## 四、数据流

### 命令流程
```
1. 客户端请求 → His.Api.Controllers
2. Controller → ApplicationService
3. ApplicationService → CommandBus.Publish(Command)
4. CommandBus → CommandHandler.Handle()
5. CommandHandler → DomainRepository.Save()
6. DomainRepository → EventStorage + EventBus
7. 领域事件发布 → EventHandlers处理
```

### 查询流程
```
1. 客户端请求 → His.Api.Controllers
2. Controller → ApplicationService
3. ApplicationService → Repository.Query()
4. Repository → EF Context → MySQL
5. 返回DTO → Controller → JSON响应
```

## 七、总结

这是一个企业级后端项目，采用 CQRS + DDD + Event Sourcing，分层清晰，职责明确。通过 Apworks 实现事件溯源，使用 Entity Framework 进行数据访问，Unity 管理依赖，Memcached 提供缓存，WebSocket 支持实时通信。适合复杂业务场景，便于扩展和维护。
