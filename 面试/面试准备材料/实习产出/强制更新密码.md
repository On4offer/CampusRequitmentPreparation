## 1. 需求分析

### 1.1 业务需求
- 员工密码需每3个月强制修改一次
- 密码超过3个月未修改时，禁止登录
- 登录时检测密码修改时间，过期则跳转到密码修改页面
- 密码修改页面只需输入旧密码和新密码，无需短信验证
- 修改成功后可使用新密码登录

### 1.2 功能要求
- 登录时校验密码修改时间
- 密码过期时禁止登录并提示
- 提供匿名密码修改接口（无需Token）
- 前端自动跳转到密码修改页面
- 修改密码后自动更新密码修改时间

### 1.3 技术约束
- 后端：C# .NET Web API + Java Spring Boot
- 前端：Vue.js + Element UI
- 数据库：MySQL（共享）
- 密码加密：MD5（大写）

### 2.2 核心流程

#### 2.2.1 登录流程
```
用户输入账号密码
    ↓
验证密码是否正确
    ↓
密码错误 → 返回错误信息
    ↓
密码正确 → 检查密码修改时间
    ↓
密码过期 → 返回PasswordExpired错误码(13)
    ↓
前端捕获错误 → 跳转到密码修改页面
```

#### 2.2.2 密码修改流程
```
用户输入旧密码和新密码
    ↓
验证旧密码是否正确
    ↓
验证新密码格式
    ↓
调用ModifyPassword方法
    ↓
自动更新PasswordChangeTime = DateTime.Now
    ↓
保存到数据库
    ↓
返回成功 → 跳转到登录页面
```

### 2.3 关键设计决策

1. 密码过期判断时机：登录时，密码验证通过后
2. 密码修改接口：匿名接口，无需Token，仅验证旧密码
3. 时间字段：`PasswordChangeTime`，NULL视为过期
4. 过期时间：3个月（90天）
5. 错误码：`ResultEnum.PasswordExpired = 13`

## 3. 数据库设计

### 3.1 字段设计

**表名**: `employees`

**新增字段**:
```sql
ALTER TABLE `employees`
ADD COLUMN `PasswordChangeTime` datetime NULL DEFAULT NULL 
COMMENT '密码最后修改时间，用于判断密码是否过期（超过3个月）' 
AFTER `LoginFailedExpired`;
```

### 3.2 字段说明

| 字段名             | 类型     | 可空 | 说明                                 |
| ------------------ | -------- | ---- | ------------------------------------ |
| PasswordChangeTime | datetime | YES  | 密码最后修改时间，NULL表示从未修改过 |

### 3.3 业务规则

- `PasswordChangeTime` 为 NULL → 视为过期，强制修改
- `PasswordChangeTime < DateTime.Now.AddMonths(-3)` → 过期
- 每次修改密码时自动更新为当前时间

#### 4.1.6 强制修改密码接口

```csharp
/// <summary>
/// 强制修改密码（密码过期时使用，无需Token和短信验证）
/// </summary>
[Anonymous]  // 匿名接口，无需Token
[HttpPost, Route("password/forcechange")]
public async Task<ApiResult> ForceChangePassword(ForceChangePassword input)
{
    ApiResult result = new ApiResult();
    try
    {
        if (await this._employeeService.ForceChangePasswordAsync(input))
            result.setSuccess("修改成功，请使用新密码登录");
        else
            result.setResult("修改失败", ResultEnum.Failed);
        return result;
    }
    catch (HisApiException ex)
    {
        result.setResult(ex.Message, ResultEnum.Failed);
        return result;
    }
    catch (HisNoDataException ex)
    {
        result.setResult(ex.Message, ResultEnum.NotFound);
        return result;
    }
    catch (Exception ex)
    {
        Logger.Warn("强制修改密码失败", ex);
        result.setResult(ex.Message, ResultEnum.Excepiton);
        return result;
    }
}
```

**关键点**:

#### 4.1.8 服务实现

**文件**: `His.Application/Implementation/EmployeeServiceImp.cs`

```csharp
public async Task<bool> ForceChangePasswordAsync(ForceChangePassword input)
{
    // 1. 参数验证
    Check.NotNull(input, "参数");
    Check.NotEmpty(input.CompNo, "公司编号");
    Check.NotEmpty(input.Account, "账号");
    Check.NotEmpty(input.OldPassword, "旧密码");
    Check.NotEmpty(input.NewPassword, "新密码");

    // 2. 验证新密码不能与原密码相同
    if (input.OldPassword == input.NewPassword)
        throw new HisApiException("新密码不能与原密码相同");

    // 3. 查找公司
    var company = this._companyRepository.Find(
        Specification<Company>.Eval(s => s.CompayNo == input.CompNo));
    if (company == null || company.IsDeleted)
        throw new HisNoDataException("编号：" + input.CompNo + "的公司不存在或已经删除");

    // 4. 查找员工
    var employee = this._employeeRepository.FindAll(
        Specification<Employee>.Eval(s => 
            s.CompId == company.ID && 
            s.Account == input.Account && 
            !s.IsDelete)).FirstOrDefault();
    if (employee == null)
        throw new HisNoDataException("账号:" + input.Account + "不存在或已经删除");

    // 5. 验证旧密码
    if (employee.Password != input.OldPassword)
        throw new HisNoDataException("您输入的旧密码不正确");

    // 6. 修改密码（自动更新PasswordChangeTime）
    employee.ModifyPassword(input.NewPassword);
    employee.UpdateTime = DateTime.Now;
    employee.UpdateUser = "系统"; // 匿名接口，使用"系统"作为更新人
    
    // 7. 保存到数据库
    this._employeeRepository.Update(employee);
    await this.Context.CommitAsync();
    
    // 8. 更新缓存
    try
    {
        this.MemcachedStrategy.AddObject(
            CacheKey.GenerateEmployeeCacheKey(employee.ID.ToString()), 
            employee);
    }
    catch (Exception ex)
    {
        Logger.Warn("添加员工的缓存时发生异常：" + ex.Message, ex);
    }
    
    return true;
}
```

**关键点**:
- 调用 `ModifyPassword` 自动更新时间
- 更新缓存确保数据一致性

```javascript
signIn({ commit }, userInfo) {
    return new Promise((resolve, reject) => {
        signIn(userInfo).then(response => {
            if (response.data.success === 0) {
                // 登录成功处理
                // ...
                resolve(response)
            } else {
                // 检查是否是密码过期错误
                if (response.data.success === 13) { // PasswordExpired = 13
                    reject({ 
                        type: 'passwordExpired', 
                        message: response.data.message, 
                        userInfo: userInfo 
                    })
                } else {
                    reject(response.data.message)
                }
            }
        }).catch(error => {
            // 如果axios返回错误，检查是否是密码过期错误
            if (error && error.response && error.response.data) {
                const res = error.response.data
                if (res.success === 13) {
                    reject({ 
                        type: 'passwordExpired', 
                        message: res.message, 
                        userInfo: userInfo 
                    })
                    return
                }
            }
            reject(error)
        })
    })
}
```

```javascript
import { forceChangePassword } from '@/api/users'
import crypto from 'crypto'

export default {
    data() {
        return {
            changePasswordForm: {
                compNo: '',
                account: '',
                oldPassword: '',
                newPassword: '',
                confirmPassword: ''
            },
            // ... 验证规则 ...
        }
    },
    created() {
        // 从路由参数获取公司编码和账号
        this.changePasswordForm.compNo = this.$route.query.compNo || ''
        this.changePasswordForm.account = this.$route.query.account || ''
    },
    methods: {
        submitForm() {
            this.$refs.changePasswordForm.validate(valid => {
                if (valid) {
                    const postData = {
                        compNo: this.changePasswordForm.compNo,
                        account: this.changePasswordForm.account,
                        oldPassword: crypto.createHash('md5')
                            .update(this.changePasswordForm.oldPassword)
                            .digest('hex').toUpperCase(),
                        newPassword: crypto.createHash('md5')
                            .update(this.changePasswordForm.newPassword)
                            .digest('hex').toUpperCase()
                    }
                    forceChangePassword(postData).then(response => {
                        if (response.data.success === 0) {
                            this.$message.success('密码修改成功，请使用新密码登录')
                            this.$router.push({ path: '/login' })
                        } else {
                            this.$message.error(response.data.message)
                        }
                    }).catch(error => {
                        this.$message.error(error)
                    })
                }
            })
        }
    }
}
```

## 6. 关键逻辑说明

### 6.1 密码过期判断逻辑

```csharp
// 过期条件：
// 1. PasswordChangeTime 为 NULL（从未修改过）
// 2. PasswordChangeTime < DateTime.Now.AddMonths(-3)（超过3个月）

if (emp.PasswordChangeTime == null || 
    emp.PasswordChangeTime < DateTime.Now.AddMonths(-3))
{
    // 密码已过期
}
```

**业务规则**:
- NULL视为过期，强制首次修改
- 3个月（90天）为过期阈值
- 判断时机：密码验证通过后

### 6.2 密码修改时间更新机制

**统一更新入口**: `Employee.ModifyPassword()` 方法

```csharp
public Employee ModifyPassword(string newPassword)
{
    this.Password = newPassword;
    this.PasswordChangeTime = DateTime.Now; // 统一更新
    return this;
}
```

**所有密码修改路径都调用此方法**:
1. `ForceChangePasswordAsync` - 强制修改密码
2. `ChangePasswordAsync` - 普通修改密码
3. `ChangePasswordByCodeAsync` - 通过验证码修改密码

**优势**: 统一管理，避免遗漏

### 6.3 登录验证顺序

```
1. 验证账号是否存在
   ↓
2. 检查登录失败次数限制
   ↓
3. 验证密码是否正确 ← 关键：先验证密码
   ├─ 密码错误 → 返回 NoAccount，不检查过期
   └─ 密码正确 → 继续
       ↓
4. 检查密码是否过期 ← 关键：密码正确后才检查
   ├─ 已过期 → 返回 PasswordExpired
   └─ 未过期 → 继续
       ↓
5. 检查员工状态
   ↓
6. 生成Token，登录成功
```

**设计原因**: 避免密码错误时也提示过期，提升安全性

### 6.4 匿名接口设计

**为什么需要匿名接口？**
- 密码过期时用户无法登录，无法获取Token
- 需要提供无需Token的密码修改接口

**安全措施**:
- 验证旧密码（MD5）
- 验证公司编码和账号
- 验证新密码不能与旧密码相同
- 记录更新人为"系统"

### 6.5 前端错误处理流程

```
API返回 success = 13
    ↓
响应拦截器：登录接口不拦截，直接返回
    ↓
Store signIn：检查 success === 13
    ↓
reject({ type: 'passwordExpired', message: ... })
    ↓
登录页面 catch：检查 error.type === 'passwordExpired'
    ↓
显示提示消息 + 跳转到密码修改页面
```
