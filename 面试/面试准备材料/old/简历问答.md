# 项目经历

### 登录模块与用户鉴权功能分析

1. 整体设计架构

项目采用了**JWT + 拦截器 + ThreadLocal**的设计模式来实现登录认证和用户鉴权功能，支持员工端（管理后台）和用户端（微信端）两种登录场景。

2. 登录模块设计

2.1 员工登录流程

- **控制器层**：通过`EmployeeController`的`login()`方法处理登录请求
- **服务层**：`EmployeeServiceImpl`验证用户名和密码（MD5加密比对）
- **JWT生成**：验证通过后，使用`JwtUtil`生成包含员工ID的JWT令牌
- **返回数据**：包含员工信息和JWT令牌的`EmployeeLoginVO`对象

2.2 用户登录流程

- **控制器层**：通过`UserController`的`login()`方法处理微信登录请求
- **服务层**：`UserServiceImpl`使用微信登录code换取openid，实现自动注册和登录
- **JWT生成**：同样使用`JwtUtil`生成包含用户ID的JWT令牌
- **返回数据**：包含用户信息和JWT令牌的`UserLoginVO`对象

3. 自定义线程变量实现

项目使用`BaseContext`类实现线程级别的用户ID存储：

```java
public class BaseContext {
    // 使用ThreadLocal存储当前线程的用户ID
    public static ThreadLocal<Long> threadLocal = new ThreadLocal<>();
    
    // 设置当前线程的ID
    public static void setCurrentId(Long id) {
        threadLocal.set(id);
    }
    
    // 获取当前线程的ID
    public static Long getCurrentId() {
        return threadLocal.get();
    }
    
    // 移除当前线程的ID，防止内存泄漏
    public static void removeCurrentId() {
        threadLocal.remove();
    }
}
```

**关键设计点**：
- 使用`ThreadLocal`确保多线程环境下用户身份的隔离
- 提供静态方法简化调用
- 包含清理方法防止内存泄漏

4. 登录拦截器实现

项目实现了两个拦截器分别处理不同端的请求：

4.1 员工端拦截器 (`JwtTokenAdminInterceptor`)

- 拦截`/admin/**`路径的请求（排除登录接口）
- 从请求头获取JWT令牌
- 校验令牌有效性并解析员工ID
- 将员工ID存入`BaseContext`线程变量
- 验证失败返回401状态码

4.2 用户端拦截器 (`JwtTokenUserInterceptor`)

- 拦截`/user/**`路径的请求（排除登录和店铺状态接口）
- 从请求头获取JWT令牌
- 校验令牌有效性并解析用户ID
- 将用户ID存入`BaseContext`线程变量
- 验证失败返回401状态码

5. 用户鉴权流程

1. **登录认证**：用户/员工通过登录接口获取JWT令牌
2. **请求携带令牌**：后续请求在请求头中携带令牌
3. **拦截器校验**：拦截器拦截请求，校验令牌有效性
4. **设置线程变量**：校验通过后，将用户ID存入ThreadLocal
5. **业务层获取用户ID**：业务逻辑中通过`BaseContext.getCurrentId()`获取当前用户ID
6. **权限控制和数据隔离**：基于当前用户ID进行权限校验和数据过滤

6. 核心代码组件

- **JwtUtil**：JWT令牌的创建和解析工具类
- **JwtProperties**：JWT相关配置（密钥、过期时间等）
- **BaseContext**：线程级别的用户ID存储
- **JwtTokenAdminInterceptor**：员工端JWT校验拦截器
- **JwtTokenUserInterceptor**：用户端JWT校验拦截器
- **WebMvcConfiguration**：注册拦截器的配置类

这种设计实现了无状态的用户认证，便于系统水平扩展，同时通过ThreadLocal优雅地解决了多线程环境下的用户身份传递问题。

基于 Redis + SpringCache 优化商品模块缓存，降低数据库访问压力，提高了系统查询性能和响应速度。  

基于 WebSocket 实现订单状态实时推送，实现商户与用户的即时交互，提升用户下单体验。  

使用 SpringTask 实现订单状态的定时清理与更新，保障系统数据准确性与业务流程闭环。
