## 核心原理

### 1. 动态代理的本质

**动态代理**是在运行时动态生成代理类，而不是在编译期。代理类会拦截目标对象的方法调用，在调用前后执行额外逻辑。

### 2. 获取私有属性的核心机制

**Java反射机制** + **setAccessible(true)** = 突破访问权限限制

```java
// 核心步骤
Field field = target.getClass().getDeclaredField("privateField");  // 1. 获取字段对象
field.setAccessible(true);                                          // 2. 突破访问限制
Object value = field.get(target);                                   // 3. 读取值
field.set(target, newValue);                                        // 4. 修改值
```

### 3. 关键点：操作目标对象，不是代理对象

- ✅ **正确**：`field.get(target)` - 操作目标对象
- ❌ **错误**：`field.get(proxy)` - 代理对象不包含目标类的私有属性

**原因**：
- JDK代理：代理对象是动态生成的类，实现接口，不包含目标类的字段
- CGLIB代理：代理对象是目标类的子类，但私有属性在父类中，需要通过父类引用访问

## JDK动态代理实现

### 1. 实现原理

```java
// 创建代理对象
Object proxy = Proxy.newProxyInstance(
    target.getClass().getClassLoader(),    // 类加载器
    target.getClass().getInterfaces(),     // 目标接口数组
    new InvocationHandler() {              // 调用处理器
        @Override
        public Object invoke(Object proxy, Method method, Object[] args) {
            // 在这里获取私有属性
            Field field = target.getClass().getDeclaredField("privateField");
            field.setAccessible(true);
            // ... 操作私有属性
            return method.invoke(target, args);
        }
    }
);
```

### 2. 关键步骤

1. **实现InvocationHandler接口**
   - 持有目标对象引用（target）
   - 在`invoke()`方法中处理所有方法调用

2. **通过反射获取私有属性**
   - 使用`getDeclaredField()`获取字段（包括私有）
   - 调用`setAccessible(true)`突破访问限制
   - 使用`get()`/`set()`读取/修改值

3. **执行目标方法**
   - 使用`method.invoke(target, args)`调用原始方法

### 3. 面试要点

**Q: JDK动态代理的局限性？**
- ✅ 只能代理实现了接口的类
- ✅ 代理对象实现接口，不继承类
- ✅ 通过接口类型调用方法

**Q: 为什么必须操作target对象？**
- 代理对象是动态生成的类，只实现接口，不包含目标类的字段
- 私有属性在目标对象中，必须通过目标对象访问

## CGLIB动态代理实现

### 1. 实现原理

```java
// 创建代理对象
Enhancer enhancer = new Enhancer();
enhancer.setSuperclass(target.getClass());  // 设置父类（目标类）
enhancer.setCallback(new MethodInterceptor() {
    @Override
    public Object intercept(Object proxy, Method method, Object[] args, 
                           MethodProxy methodProxy) {
        // 在这里获取私有属性
        Field field = target.getClass().getDeclaredField("privateField");
        field.setAccessible(true);
        // ... 操作私有属性
        return methodProxy.invoke(target, args);
    }
});
Object proxy = enhancer.create();
```

### 2. 关键步骤

1. **创建Enhancer增强器**
   - 设置父类为目标类
   - 设置回调拦截器

2. **通过反射获取私有属性**
   - 同样使用`getDeclaredField()`和`setAccessible(true)`

3. **执行目标方法**
   - 使用`methodProxy.invoke(target, args)`（比`method.invoke()`性能更好）

### 3. 面试要点

**Q: CGLIB与JDK动态代理的区别？**

| 特性 | JDK动态代理 | CGLIB动态代理 |
|------|------------|--------------|
| 代理方式 | 实现接口 | 继承目标类 |
| 是否需要接口 | ✅ 必须 | ❌ 不需要 |
| 代理对象类型 | 实现接口的类 | 目标类的子类 |
| 性能 | 较慢（反射调用） | 较快（MethodProxy优化） |
| 限制 | 只能代理接口 | 不能代理final类/方法 |

**Q: CGLIB如何生成代理类？**
- 使用**ASM框架**操作字节码
- 动态生成目标类的子类
- 子类重写父类方法，调用拦截器

**Q: Java 9+模块化问题？**
- CGLIB需要访问`java.base`模块的内部API
- 需要添加VM参数：`--add-opens java.base/java.lang=ALL-UNNAMED`
- 这是Java模块系统的安全限制

## 获取父类私有属性

### 1. 实现原理

```java
// 递归查找字段（包括父类）
public static Field getDeclaredField(Object target, String fieldName) {
    Class<?> clazz = target.getClass();
    while (clazz != null) {
        try {
            return clazz.getDeclaredField(fieldName);
        } catch (NoSuchFieldException e) {
            clazz = clazz.getSuperclass();  // 向上查找父类
        }
    }
    throw new NoSuchFieldException("字段不存在");
}
```

### 2. 关键点

- `getDeclaredField()`只能获取当前类声明的字段
- 需要递归遍历父类才能获取父类的私有字段
- 使用`getSuperclass()`向上查找

### 3. 面试要点

**Q: 为什么getField()不行？**
- `getField()`只能获取public字段
- `getDeclaredField()`可以获取所有字段（包括private）

**Q: 如何获取父类的私有属性？**
- 递归调用`getSuperclass()`向上查找
- 对每个父类调用`getDeclaredField()`

## 关键面试问题

### Q1: 动态代理中如何获取私有属性？

**标准答案**：
1. 在代理逻辑中持有目标对象的引用（target）
2. 通过反射获取私有字段：`target.getClass().getDeclaredField("fieldName")`
3. 突破访问限制：`field.setAccessible(true)`
4. 读取/修改值：`field.get(target)` / `field.set(target, value)`

**关键点**：
- 必须操作target对象，不是proxy对象
- 使用`getDeclaredField()`而不是`getField()`
- 必须调用`setAccessible(true)`

### Q2: 为什么必须操作target对象？

**答案**：
- **JDK代理**：代理对象是动态生成的类，只实现接口，不包含目标类的字段
- **CGLIB代理**：代理对象是目标类的子类，但私有属性在父类中，需要通过父类引用访问
- 私有属性存储在目标对象的内存中，必须通过目标对象访问

### Q3: setAccessible(true)的作用？

**答案**：
- 关闭Java的访问检查机制
- 允许反射访问private、protected、package-private成员
- 不调用会抛出`IllegalAccessException`

**注意**：
- 会破坏封装性，生产环境需谨慎使用
- Java 9+模块化系统可能有额外限制

### Q4: JDK动态代理和CGLIB的区别？

**答案**：

| 维度 | JDK动态代理 | CGLIB动态代理 |
|------|------------|--------------|
| **实现方式** | 实现接口 | 继承目标类 |
| **是否需要接口** | ✅ 必须 | ❌ 不需要 |
| **代理对象类型** | 实现接口的类 | 目标类的子类 |
| **性能** | 较慢（反射） | 较快（MethodProxy） |
| **限制** | 只能代理接口 | 不能代理final类/方法 |
| **使用场景** | Spring AOP（接口代理） | Spring AOP（类代理） |

### Q5: 如何获取父类的私有属性？

**答案**：
1. 使用`getDeclaredField()`获取当前类的字段
2. 如果找不到，调用`getSuperclass()`获取父类
3. 递归向上查找，直到找到或到达Object类
4. 找到后调用`setAccessible(true)`并操作

### Q6: 动态代理的性能问题？

**答案**：
- **反射操作慢**：比直接访问属性慢10-100倍
- **优化方案**：
  1. 缓存Field对象（避免重复查找）
  2. 使用`MethodProxy`（CGLIB优化）
  3. 减少反射调用次数

### Q7: final字段能否修改？

**答案**：
- **基本类型和String**：不能修改（编译期优化，值内联）
- **对象类型**：不能修改引用，但可以修改对象内部属性
- **反射限制**：即使调用`setAccessible(true)`，final字段的修改也可能失败

## 常见陷阱

### 1. 操作proxy对象而不是target对象

```java
// ❌ 错误
Field field = proxy.getClass().getDeclaredField("privateField");
field.get(proxy);  // 会失败，代理对象没有这个字段

// ✅ 正确
Field field = target.getClass().getDeclaredField("privateField");
field.get(target);  // 成功
```

### 2. 使用getField()而不是getDeclaredField()

```java
// ❌ 错误
Field field = target.getClass().getField("privateField");  // 只能获取public字段

// ✅ 正确
Field field = target.getClass().getDeclaredField("privateField");  // 可以获取所有字段
```

### 3. 忘记调用setAccessible(true)

```java
// ❌ 错误
Field field = target.getClass().getDeclaredField("privateField");
field.get(target);  // 抛出IllegalAccessException

// ✅ 正确
Field field = target.getClass().getDeclaredField("privateField");
field.setAccessible(true);  // 必须调用
field.get(target);  // 成功
```

### 4. Java 9+模块化问题

```java
// ❌ 错误：没有VM参数
// CGLIB会抛出InaccessibleObjectException

// ✅ 正确：添加VM参数
// --add-opens java.base/java.lang=ALL-UNNAMED
```

## 性能与安全性

### 性能问题

1. **反射操作慢**
   - 比直接访问慢10-100倍
   - 需要安全检查、方法查找等开销

2. **优化方案**
   ```java
   // 缓存Field对象
   private static final Map<String, Field> fieldCache = new ConcurrentHashMap<>();
   
   Field field = fieldCache.computeIfAbsent("privateField", name -> {
       Field f = target.getClass().getDeclaredField(name);
       f.setAccessible(true);
       return f;
   });
   ```

### 安全性问题

1. **破坏封装性**
   - `setAccessible(true)`绕过访问控制
   - 可能暴露敏感数据

2. **生产环境建议**
   - 仅在必要场景使用（框架底层、日志监控）
   - 添加权限检查
   - 记录访问日志

3. **Java 9+模块化限制**
   - 需要显式开放模块权限
   - 增强安全性，防止未授权访问

## 实际应用场景

### 1. Spring AOP

Spring AOP底层使用动态代理：
- **JDK代理**：代理实现了接口的Bean
- **CGLIB代理**：代理没有接口的Bean

在AOP中，可以通过反射获取目标对象的私有属性，用于：
- 日志记录
- 性能监控
- 数据校验

### 2. 框架底层实现

- **MyBatis**：通过反射获取实体类属性
- **Hibernate**：通过反射操作实体对象
- **Jackson**：通过反射序列化/反序列化

### 3. 测试框架

- **Mockito**：通过反射创建Mock对象
- **JUnit**：通过反射执行测试方法
- **PowerMock**：通过反射绕过final限制

### 4. 日志和监控

- 记录方法执行前后的对象状态
- 监控私有属性的变化
- 调试和问题排查
