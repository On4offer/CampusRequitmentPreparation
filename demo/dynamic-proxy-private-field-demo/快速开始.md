# 快速开始指南

## 一、项目简介

本项目演示了如何在动态代理中获取和修改目标对象的私有属性，包含以下核心内容：

- ✅ JDK动态代理获取私有属性
- ✅ CGLIB动态代理获取私有属性  
- ✅ 获取父类私有属性
- ✅ 反射工具类的使用
- ✅ 详细的输出日志，便于观察和理解

## 二、快速运行

### 方式1：使用批处理文件（推荐）

双击运行 `run.bat` 文件即可自动编译和运行。

### 方式2：使用Maven命令

```bash
# 进入项目目录
cd dynamic-proxy-private-field-demo

# 编译
mvn clean compile

# 运行（pom.xml已配置JVM参数）
mvn exec:java
```

### 方式3：使用IDE

1. 在IDE（如IntelliJ IDEA、Eclipse）中打开项目
2. 等待Maven自动下载依赖（CGLIB）
3. **重要**：如果使用Java 9+，需要在运行配置的VM options中添加：
   ```
   --add-opens java.base/java.lang=ALL-UNNAMED
   ```
4. 运行 `DynamicProxyDemo.java` 的 `main` 方法

## 四、关键观察点

运行程序时，注意观察以下关键信息：

1. **代理对象的类型**
   - JDK代理：`$Proxy0` 等
   - CGLIB代理：`OrderService$$EnhancerByCGLIB$$...`

2. **目标对象 vs 代理对象**
   - 注意日志中明确区分了 target 和 proxy
   - 理解为什么必须操作 target 对象

3. **反射操作的过程**
   - 查找字段 → 设置可访问 → 读取/修改值
   - 每一步都有详细的日志输出

4. **属性值的变化**
   - 初始值 → 代理修改后的值
   - 通过getter方法验证修改是否成功

## 六、常见问题

### Q1: 运行时出现 `InaccessibleObjectException` 错误？
**A:** 这是Java 9+的模块化限制。解决方案：
- **使用Maven运行**：`pom.xml`已配置好JVM参数，直接运行 `mvn exec:java` 即可
- **在IDE中运行**：在运行配置的VM options中添加：`--add-opens java.base/java.lang=ALL-UNNAMED`
- **手动运行**：在java命令后添加：`--add-opens java.base/java.lang=ALL-UNNAMED`

### Q2: 编译失败，找不到CGLIB类？
**A:** 确保Maven已正确下载依赖，运行 `mvn clean install` 重新下载。

### Q3: 运行时找不到主类？
**A:** 确保已编译成功，运行 `mvn clean compile` 后再运行。

### Q4: 如何查看更详细的日志？
**A:** 代码中已包含详细的 `System.out.println` 输出，可以直接查看控制台。

### Q5: 如何调试？
**A:** 在IDE中设置断点，特别是在 `invoke` 和 `intercept` 方法中，观察对象状态。
