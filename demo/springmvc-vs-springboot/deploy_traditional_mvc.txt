# 传统SpringMVC项目部署指南

## 说明
传统SpringMVC项目需要部署到外部Tomcat服务器，以下是详细步骤：

## 步骤1: 构建项目

```bash
# 进入传统SpringMVC项目目录
cd springmvc-traditional

# 使用Maven构建WAR包
mvn clean package -DskipTests
```

构建成功后，会在 `target` 目录生成 `springmvc-traditional-1.0-SNAPSHOT.war` 文件。

## 步骤2: 部署到Tomcat

1. **下载Tomcat**
   - 从官网 https://tomcat.apache.org/ 下载Tomcat 9.x或10.x版本
   - 解压到本地目录，例如：`D:\apache-tomcat-9.0.80`

2. **部署WAR包**
   - 将构建好的WAR包复制到Tomcat的`webapps`目录
   - 例如：`D:\apache-tomcat-9.0.80\webapps\springmvc-traditional-1.0-SNAPSHOT.war`

3. **启动Tomcat**
   - 运行Tomcat的启动脚本：`D:\apache-tomcat-9.0.80\bin\startup.bat`
   - Tomcat会自动解压WAR包

## 步骤3: 访问应用

启动成功后，可以通过以下地址访问：

- **首页**: http://localhost:8080/springmvc-traditional-1.0-SNAPSHOT/traditional/index
- **JSON API**: http://localhost:8080/springmvc-traditional-1.0-SNAPSHOT/traditional/api/data
- **健康检查**: http://localhost:8080/springmvc-traditional-1.0-SNAPSHOT/traditional/health

## 对比SpringBoot的优势

1. **部署复杂度对比**:
   - SpringMVC: 需要下载Tomcat → 配置 → 部署WAR包 → 启动Tomcat
   - SpringBoot: 一行命令 `mvn spring-boot:run` 或 `java -jar` 直接运行

2. **配置文件对比**:
   - SpringMVC: 需要web.xml、spring-mvc.xml等多个配置文件
   - SpringBoot: 零配置或仅需简单的application.yml

3. **开发体验对比**:
   - SpringMVC: 修改配置后需要重启Tomcat，开发效率低
   - SpringBoot: 支持热部署，开发体验更佳

通过对比可以直观感受到SpringBoot带来的开发效率提升！
