# Java 拷贝机制演示 (引用拷贝、浅拷贝、深拷贝)

本项目用于演示 Java 中常见的三种拷贝方式，并整理了相关的面试高频知识点。

## 目录结构
- `Address.java`: 地址实体类（引用类型成员）
- `Person.java`: 包含地址的人员实体类，实现了 `Cloneable` 接口
- `CopyDemo.java`: 演示程序入口

## 如何运行
在 `copy_demo` 目录下执行：
```bash
javac -d . *.java
java copy_demo.CopyDemo
```

## 概念详解

### 1. 引用拷贝 (Reference Copy)
*   **定义**：只是增加了一个指向原对象的指针（引用），并没有创建新对象。
*   **特点**：两个变量指向同一个堆地址。修改其中一个，另一个同步改变。
*   **代码示例**：`Person p2 = p1;`

### 2. 浅拷贝 (Shallow Copy)
*   **定义**：创建一个新对象，但新对象中的非基本类型属性（即引用类型）仍然指向原对象中属性的地址。
*   **特点**：
    *   基本数据类型：拷贝值。
    *   引用数据类型：拷贝引用地址。
*   **实现方式**：实现 `Cloneable` 接口并重写 `clone()` 方法，直接调用 `super.clone()`。

### 3. 深拷贝 (Deep Copy)
*   **定义**：创建一个新对象，且新对象中的所有属性（包括引用类型）都进行了递归式的创建。
*   **特点**：原对象和拷贝对象完全独立，修改任何一方的任何属性都不会影响另一方。
*   **实现方式**：
    1.  手动在 `clone()` 方法中递归克隆所有引用属性。
    2.  利用序列化与反序列化（最简便但不一定最高效）。

## 面试知识点总结

### Q1: `clone()` 方法是哪里定义的？
`clone()` 是 `java.lang.Object` 中的 `protected` 方法。

### Q2: 为什么实现克隆要实现 `Cloneable` 接口？
`Cloneable` 是一个**标记接口**（内部没有方法）。如果没有实现该接口而调用 `clone()`，会抛出 `CloneNotSupportedException`。

### Q3: 浅拷贝 and 深拷贝的区别？
*   **核心区别**：是否对成员变量中的“引用类型”进行了真正的“对象创建”。
*   **内存影响**：深拷贝会消耗更多内存，但对象之间更安全、更独立。

### Q4: 谈谈深拷贝的几种实现方式？
1.  **手动重写 clone**：逐层调用成员变量的 clone。
2.  **序列化 (Serialization)**：通过 `ObjectOutputStream` 和 `ObjectInputStream`（要求所有涉及类都实现 `Serializable`）。
3.  **构造函数**：通过拷贝构造函数（Copy Constructor）显式创建新对象。
4.  **第三方库**：如 Apache Commons Lang 的 `SerializationUtils` 或 JSON 序列化。
5.  **深拷贝工具类**：比如使用 `BeanUtils` 等（注意有些工具类默认是浅拷贝）。

### Q5: String 类型的拷贝属于哪种？
虽然 `String` 是引用类型，但它是 **Immutable（不可变的）**。在浅拷贝中，虽然拷贝的是地址，但由于无法修改原字符串内容（任何修改都会产生新字符串），其表现行为类似于基本类型的拷贝，通常不需要特殊处理。
